# 可扩展性与性能

当单个服务器的 CPU、内存或 I/O 资源达到极限时，就必须增加服务器以提升容量。术语“可扩展性”常被与“性能”混用，意味着增加容量等同于提升性能。然而，容量与性能之间的关联并非直接。

例如，一个单线程的 CPU 密集型程序在双 CPU 服务器上需要 15 分钟完成。假设服务器 CPU 未饱和，再增加两个 CPU 并不会使该进程运行得更快。如果它一次只能在一个 CPU 上运行，那么其执行速度就受限于单个 CPU 的处理能力。只有当增加更多 CPU 能让进程在处理器上获得更长的无中断运行时间时，性能才会提升。同样，即使在四节点集群上运行它，其速度也不会加快。

然而，**横向扩展**到四个服务器意味着我们可以同时运行该程序的四个副本，从而在相同的 15 分钟内完成大约四倍的工作量。总结来说，横向扩展为你的系统增加了容量。它是否提升性能，取决于你的应用程序的可扩展性以及当前系统的负载程度。

请记住，对于编写良好的应用程序，Oracle RAC 的扩展性极佳。反之，编写不佳的应用程序往往扩展性很差。
