# Exadata 安装与升级指南

## 使用 install.sh 脚本

`install.sh` 脚本有多种使用方式。例如，以下命令行选项会处理安装过程的全部 20 个步骤，仅在某个步骤失败时停止：
```bash
# ./install.sh -cf <client><cluster>.xml -r 1-20
```
每个步骤必须成功完成才能进行下一步。Oracle 建议一次运行一个步骤，在继续下一步之前审查每个步骤的输出。`install.sh` 通过 `-s` 命令行选项提供了此功能。例如，安装过程可能如下所示：
```bash
# ./install.sh –cf <client>-<cluster>.xml -s 1
```
检查输出是否有错误...
```bash
# ./install.sh –cf <client>-<cluster>.xml -s 2
```
检查输出是否有错误...
```bash
# ./install.sh –cf <client>-<cluster>.xml -s 3
```
检查输出是否有错误...
```bash
# ./install.sh –cf <client>-<cluster>.xml -s 4
```
依此类推...

`install.sh` 接受您之前使用 Exadata 部署助手生成的文件中的参数作为输入。每次调用 `install.sh` 执行配置步骤时都会创建日志文件，并且对于某些步骤，它会动态生成并执行一个 shell 脚本来完成所有必需的任务。查看这些文件对于确定步骤失败的原因非常有用。日志文件和动态生成的 shell 脚本存储在 `/opt/oracle.SupportTools/onecommand/linux-x64/log` 和 `/opt/oracle.SupportTools/onecommand/linux-x64/tmp` 目录中。

各个安装步骤生成的输出差异很大。但通常，`install.sh` 会显示一些头部信息，告知您正在运行的步骤、日志文件的位置以及是否成功完成。继续完成所有步骤，直到部署摘要完成。

## 升级 Exadata

随着越来越多的公司采用 Exadata 平台，硬件升级成为一个日益热门的话题。我们进入 Exadata 领域始于六年前的一台 Exadata V2 四分之一机架配置。大约一年后，我们将系统升级到了半机架。当然，V2 设备已不再用于生产，因此我们的升级形式是两台 X2-2 数据库服务器和四个存储单元。我们考虑的配置选项如下：

*   将新的 X2 设备配置为一个独立的 RAC 集群和存储网格，在同一个 Exadata 机柜内创建两个略有不同的四分之一机架配置。Oracle 将此称为“split rack”配置。
*   将新的 X2 设备添加到我们现有的四分之一机架集群中，有效地将其升级为半机架。

### 创建新的 RAC 集群

Exadata 部署助手不直接支持以这种方式升级系统，但通过一些调整，它可以用来生成 OneCommand 执行安装所需的所有文件。

一旦参数和部署文件上传到 Exadata，您应该能够毫无问题地运行所有配置步骤，而不会影响现有集群。一位同事实际上使用此过程在新设备上创建了一个独立的 Exadata 配置，同时保持现有系统不变。

在大多数情况下，您只需像创建匹配机架总大小的配置一样填写 Exadata 部署助手。在选择集群中的组件时，通过选择新组件来创建一个额外的集群。使用 Exadata 部署助手进行此类 Exadata 升级时，需要考虑以下几点：

*   **名称前缀**：虽然不是必须的，但您可能希望将“数据库机器名称”、“数据库服务器基本名称”和“存储服务器基本名称”的值设置为与您现有的 Exadata 配置匹配。这样，如果您决定将这些服务器合并到旧集群中，就不必更改主机名。例如，我们的四分之一机架配置的数据库主机名为 `enkdb01` 和 `enkdb02`。添加新服务器后，继续使用名称 `enkdb03` 和 `enkdb04`。同样，存储单元主机名继续使用 `enkcel04` 到 `enkcel07`。
*   **客户端访问 SCAN 名称**：此过程将创建一个新的 RAC 集群，因此您需要一个新的 SCAN 名称。您还需要确保它以及所有其他新主机名在您公司的 DNS 服务器中正确注册（就像您现有 Exadata 系统安装时那样）。
*   **国家代码/时区**：当然，这些设置应与您现有的 Exadata 系统匹配。
*   **NTP 和 DNS 服务器**：这些也应与您现有的 Exadata 环境匹配。
*   **Oracle 数据库机器模型**：此设置决定配置器在创建主机名和 IP 地址时将使用多少计算节点和存储单元。
*   **网络 IP 地址**：您应继续使用为要升级的 Exadata 机架配置的网络。在 Exadata 部署助手中输入主机的起始 IP 地址时，请确保在“审阅详细信息”屏幕上进行必要的 IP 地址调整。
*   **操作系统用户和组账户**：虽然不是必须的，但在配置新集群时，应使用相同的用户/组名和用户/组 ID。如果这些用户账户有可能在新系统和旧系统之间交互，则尤其如此。OneCommand 不会为您建立新旧服务器之间的用户等效性。因此，这必须在升级完成后手动完成。

完成输入设置后，像通常一样生成文件。您会注意到每个集群都有单独的 XML 文件，还有一个用于定义整个机架的 XML 文件。

从此时起，该过程与全新安装没有区别。首先，您需要使用之前讨论的 Firstboot 配置网络组件，然后运行回收磁盘空间的程序。完成后，只需以 root 用户身份登录到第一台新的计算节点，运行 `install.sh` 配置步骤并调用特定于新集群的 XML 文件。完成后，您将拥有一个新的 RAC 集群，其中包含一个初始数据库。

#### 升级现有集群

如果您要将 Exadata 升级到半机架或全机架配置，并希望将新服务器和单元集成到您现有的 RAC 集群中，则必须手动配置新服务器和单元。《Exadata 数据库机器扩展与多机架布线指南》中有一章题为“扩展 Oracle 数据库机器”，详细讨论了该过程。首先，我们将了解配置新计算节点的基本步骤。然后，我们将了解如何将新单元添加到您现有的存储网格中。

> **注意**
>
> 本节中的步骤并非旨在作为全面的指南，并且可能会发生变化。请参阅您的 Exadata 文档以获取特定于您 Exadata 版本的详细信息。
