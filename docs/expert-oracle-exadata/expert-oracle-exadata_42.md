# 单元救援选项

要访问单元救援选项，需通过带外管理卡连接到虚拟控制台并重启单元。当屏幕上出现 `grub` 菜单时，按任意键调出引导选项。您将看到存储单元上系统磁盘对的几个不同选项。

可以通过最终选项 `CELL_USB_BOOT_CELLBOOT_usb:in_rescue_mode` 访问 `CELLBOOT` USB 设备。从 `CELLBOOT` USB 设备引导后，您有两个初始的恢复选项：进入救援 `shell` 或重新映像存储服务器。如果只需要恢复少数文件，或者可以通过命令行解决故障，救援 `shell` 会很有帮助。通常情况下，您需要重新映像单元。虽然这听起来像是一个极端的选项，但 `Oracle` 已确保对重建操作系统至关重要的所有内容都已备份。这意味着您无需输入 `IP` 地址、主机名或重置 `root` 密码。

选择重新映像存储单元后，系统会询问您是否要擦除所有数据分区和磁盘。此决定取决于您需要执行的恢复类型。

*   **擦除数据分区和数据磁盘**：如果选择此选项，重新映像过程将删除系统磁盘上的分区，并从非系统磁盘中移除单元磁盘元数据。这将使存储单元处于具有重新映像的操作系统且存储服务器配置中无单元磁盘的状态。
*   **不擦除数据分区和数据磁盘**：存储单元仍将被重新映像，但只有系统卷分区会受到影响。此选项保留所有单元磁盘元数据完整，这意味着各个单元磁盘仍然可用，并包含重新映像前其上的所有数据。单元磁盘仍需导入，但这可以通过在 `CellCLI` 中使用一条简单的 `import celldisk all force` 命令完成。如果该单元的 `ASM` 磁盘尚未从 `ASM` 磁盘组中删除，此选项很有价值。导入单元磁盘将避免需要进行 `I/O` 密集型的重平衡操作。

那么，如果由于某种原因，内部的 `CELLBOOT` `USB` 闪存盘无法用于救援过程，会发生什么？如果发生这种情况，您可以遵循计算节点重新映像过程，并从 `Oracle Software Delivery Cloud` 下载存储单元映像制作软件。创建一个可引导的 `USB` 设备并使用它来引导存储单元。请记住，您需要输入特定于该单元的所有网络信息，因为您使用的映像不包含此数据。所有单元磁盘和网格磁盘也需要重新创建。希望这种情况永远不会发生，因为存储单元会执行持续的验证检查。较新版本的 `Exadata Storage Server` 软件包含对 `CELLBOOT` `USB` 设备的定期检查，如果设备损坏，将自动重建该设备。
