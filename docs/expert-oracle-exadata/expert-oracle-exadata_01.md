# 引言

非常感谢您购买《Expert Oracle Exadata》第二版。我们几位作者在编写本书时，始终站在巨人的肩膀上。每当为一本成功的书籍编写第二版时，作者都面临着压力，即需要创作出至少与第一版同样好、甚至更好的版本。而第一版确实非常出色。我们希望，亲爱的读者，我们已为您提供了关于 Exadata 的合适介绍。事实上，我们的希望是给您足够的信息来开始使用 Exadata。我们常常发现数据库管理员在接触 Exadata 后，会问这样一个问题：“现在该怎么办？”我们尝试构建本书的结构来帮助您回答这个问题。在深入探讨使 Exadata 在 Oracle 数据库处理硬件领域如此独特的各种优化之前，您会先了解 Exadata 是什么。尽管部分内容，特别是较早的章节，描绘了一幅宏观图景，但我们逐渐深入了许多细节。拥有一台 Exadata 开发系统对您理解更高级的材料大有裨益。我们已竭尽全力让您能够跟上进度，但请记住，Exadata 平台绝非一成不变；硬件和软件的新版本可能会改变已记录的命令和 SQL 语句的执行结果。我们将在网站 [`http://www.expertoracleexadata.com/`](http://www.expertoracleexadata.com/) 以及我们附录 B 中列出的个人博客上尝试说明主要的差异。

请注意，我们使用了各种未记录的下划线参数和功能来演示软件各个部分的工作原理。请不要将此视为管理系统生产环境的推荐方法！事实上，通常没有理由偏离默认设置。仅当获得 Oracle Support 的明确许可，并且是您提出的服务请求建议的一部分时，才允许设置下划线参数。请记住，我们曾有机会接触许多系统，可以放心地拆解分析，而无需过多担心操作后果。这为我们研究 Exadata 在不同硬件代际中的工作方式提供了巨大优势。

## 目标读者

本书假定您已经熟悉 Oracle。除了与 Exadata 平台相关的部分，我们不会深入详细解释 Oracle 的工作原理。这意味着我们对读者的知识做了一些假设。例如，我们不假设您是 Oracle 性能调优专家，但我们期望您能熟练编写 SQL 语句，并对 Oracle 架构有良好的理解。由于 Exadata 是一个硬件和软件平台，在一些与硬件更密切相关的章节中，您不可避免地会看到对 Linux 管理的引用。请不要感到畏惧——作为 Exadata 管理员，在日常平台管理中，您只需要掌握少数几个命令。



## 一个不断变化的目标

我们在本书第一版的引言中就有完全相同的这一节，其传达的信息至今依然如故。时至今日，仍让我们惊叹不已的是 Exadata 平台的发展速度。不仅硬件在不断演进，紧跟新技术的发展，软件也在持续突破可能的极限。新的软件版本发布并不要求您升级硬件。除了最早的 Exadata 系统外，当前的 Exadata 软件版本与每一代硬件都兼容。

前一段提到的变化包括大量新增功能，这些功能在附录 A 的《Exadata 数据库一体机系统概览》中清晰可见。正如您能想到的，试图跟上 Oracle 快速发布的步伐是本项目最困难的部分。每当新的硬件和软件发布时，每一章都不得不经历多次修订。我们在本书中试图涵盖的最新版本是 Oracle 12.1.0.2.2 RDBMS 配合 12.1.2.1.x 版本的 cell 软件。与本书第一版出版时 Oracle 11.2.0.2 是当前版本不同，现在 Exadata 在技术上支持的版本已经相当多了。从 Oracle 支持的角度来看，目前您可能应该正处于向 Oracle 12c 迁移的阶段。这是我们即使在许多用户尚未迁移到该版本的情况下，仍在书中为最新 RDBMS 版本留出大量篇幅的原因之一。撰写本书时的另一个考虑是，我们必须非常谨慎地引用新功能被引入时的正确版本。如果您刚开始接触 Exadata，可能会对这些版本号感到困惑；然而，一旦您有了最初几周的 Exadata 管理经验，您就会发现引用 Exadata cell 软件版本会变得自然而然。

Exadata 的演进方式无疑会使本书的某些内容变得过时，因此，如果您观察到本书涵盖的内容与您所见存在差异，很可能是由于版本不同所致。尽管如此，我们欢迎您的反馈，并将处理您发现的任何不一致之处。
