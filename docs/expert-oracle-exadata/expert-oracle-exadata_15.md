# 压缩

到目前为止，我们主要关注的是如何通过更长期或长期归档来降低成本。压缩这个话题尚未涉及，但由于本书是关于 Exadata 的，因此不可避免地要提及它。

在 Exadata 平台上，你不会像在传统硬件上那样使用这些存储层级。Exadata 管理员无需担心基于闪存/SAN 的二级和三级存储/网络解决方案的确切正确组合。存储是随系统一同提供的。如果你想将所有数据保留在存储服务器上，同时数据量又在不断增长，那么你应该认真考虑**混合列压缩**（HCC）。你在之前的章节中读到过，HCC 能提供非常好的压缩比，从而节省存储空间。除此之外，你还可以在 HCC 压缩数据上使用**智能扫描**，与非 Exadata 平台相比，这显著提升了吞吐量。例如，通过 InfiniBand 访问 HCC 压缩数据，应该比通过直接 NFS（dNFS）文件器访问数据更高效。
