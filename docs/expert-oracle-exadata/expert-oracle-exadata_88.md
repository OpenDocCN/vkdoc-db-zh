## 诊断脚本

在本书中，您已经见识了大量有用的诊断脚本。虽然其中许多脚本的内容已在正文中展示，但有些脚本过长，我们决定不将其内容打印在代码清单中。这些脚本都可以在 [`www.expertoracleexadata.com`](http://www.expertoracleexadata.com/) 在线获取。务必确保您理解特定脚本的功能，并在使用前进行彻底测试。这包括对照许可证进行检查。

表 C-1 包含脚本列表及其简要说明。

表 C-1. 本书使用的诊断脚本



| 脚本名称 | 描述 |
| --- | --- |
| `as.sql` `cdb_` `as.sql` | AS 是 Active SQL（活跃 SQL）的缩写。此脚本显示当前实例上所有活跃的 SQL 语句，信息来源于 `V$SESSION`。请注意，由于这种快速而简单的方法可能无法“捕捉”到执行飞快的语句，您可能需要执行多次才能了解系统上正在发生的情况。 |
| `calibrate_io.sql` | 此脚本为 `DBMS_RESOURCE_MANAGER.CALIBRATE_IO` 过程提供了一个简单的封装。在 Oracle 12cR1 之前的版本中，必须运行此过程，Oracle 才会允许您启用自动并行度（Auto DOP）。 |
| `check_px.sql` | 此脚本包含一个对 `V$PX_PROCESS_SYSSTAT` 的简单查询，用于显示当前正在使用的并行服务器进程数量。 |
| `comp_ratio.sql` | 这是一个简单的脚本，它根据输入值（原始表大小）计算压缩比率。 |
| `create_display_raw.sql` | 此脚本在 ENKITEC 模式（schema）中创建 `display_raw()` 函数，该函数将 raw 数据类型的值转换为各种其他数据类型（最初由 Greg Rahn 编写）。 |
| `dba_tables.sql` | 这是一个用于查询 `DBA_TABLES` 的简单脚本。它显示行数、数据块数和默认并行度。 |
| `cdb_tables.sql` | 这是一个用于查询 `CDB_TABLES` 的简单脚本。它显示整个 CDB 或特定 PDB 的行数、数据块数和默认并行度。 |
| `display_raw.sql` | 这是一个简单的脚本，用于将 raw 值转换为指定的数据类型格式，例如 `NUMBER` 或 `VARCHAR2`。它依赖于 `create_display_raw.sql` 脚本创建的 `display_raw()` 函数。 |
| `dplan.sql` | 此脚本显示共享池中某个 SQL 语句的实际执行计划。这是一个非常简单的脚本，它会提示输入 `SQL_ID` 和 `CHILD_NO`，然后调用 `dbms_xplan.display_cursor`。 |
| `dump_block.sql` | 此脚本使用 `ALTER SYSTEM DUMP DATAFILE` 将一个数据块转储到跟踪文件。它会提示输入 `fileno`（文件号）和 `blockno`（块号）。 |
| `esfc_keep_tables.sql` | 此脚本显示 `CELL_FLASH_CACHE` 属性被设置为 `KEEP` 的对象。自 Exadata 11.2.3.3 版本起，应该不再需要修改存储子句；此脚本可帮助您识别需要更改的段（segment）。 |
| `flush_pool.sql` | 此脚本使用 `ALTER SYSTEM FLUSH SHARED_POOL` 从共享池中清除所有 SQL 语句。在生产环境中使用此脚本需格外小心，因为它可能导致（硬）解析风暴。 |
| `flush_sql.sql` | 此脚本使用 `DBMS_SHARED_POOL.PURGE` 从共享池中清除单个 SQL 语句。它仅适用于 10.2.0.4 及更高版本。 |
| `fs.sql` | 此脚本允许您使用部分 SQL 文本或 `SQL_ID` 在 `V$SQL` 中进行搜索。（FS 是 Find SQL 的缩写。）该脚本会报告一些统计信息，例如平均耗时和平均 LIOs。 |
| `fsx.sql` | FSX 代表 Find SQL eXadata。此脚本基于 SQL 语句文本或特定的 `SQL_ID` 搜索共享池 (`V$SQL`)，并报告语句是否被下推（offload），如果被下推，则报告节省的 I/O 百分比。请注意，本书中使用了此脚本的几个变体版本（`fsx2.sql`、`fsx3.sql` 和 `fsx4.sql`）。这些版本缩减了输出宽度，以便更容易地适应纸质图书格式的打印限制。 |
| `fsxo.sql` | 此脚本与 `fs.sql` 脚本类似，但仅列出那些已被下推的语句。它可以与 `offload_percent.sql` 脚本结合使用，以深入分析导致其计算出下推百分比的各个语句。 |
| `gather_table_stats.sql` | 这是一个使用 `DBMS_STATS.GATHER_TABLE_STATS` 过程收集表统计信息的简单脚本。 |
| `get_compression_ratio.sql` `get_compression_ratio_12c.sql` | 此脚本是对内置压缩建议器功能 (`DBMS_COMPRESSION.GET_COMPRESSION_RATIO`) 的一个封装。它会提示输入表名和压缩类型，然后通过实际压缩表的一个行子集来估计预期的压缩比率。 |
| `get_compression_type.sql` | 此脚本为 `DBMS_COMPRESSION.GET_COMPRESSION_TYPE` 过程提供了一个封装。它可用于识别特定行使用的实际压缩类型。它会提示输入表名和 ROWID，并返回该行的实际压缩类型，而不是分配给表的压缩类型。 |
| `mystat.sql` | 这是一个用于查询 `V$MYSTATS` 的简单脚本，不要与后面描述的 Adrian Billington 的 `mystats` 脚本混淆。 |
| `mystats.sql` | 在第 11 章中有详细介绍，Adrian Billington 的 `myststs.sql` 允许您捕获 SQL 语句执行期间会话统计信息的变化。可从 [`www.oracle-developer.net`](http://www.oracle-developer.net/) 下载。 |
| `old_rowid.sql` | 此脚本创建 `old_rowid()` 函数。`old_rowid()` 函数接受一个 `rowid` 并返回 `fileno`、`blockno` 和 `rowno`（旧的 `rowid` 格式）。 |
| `obj_by_hex.sql` | 此脚本将十六进制格式的 `object_id` 转换为对象名称。十六进制值包含在数据块转储中。 |
| `offload_percent.sql` | 此脚本可用于快速检查 Exadata 平台上语句是否正在被下推。它允许评估所有执行时间超过最小值或逻辑 I/O 超过最小值的语句，并计算已被下推的语句百分比。 |
| `parms.sql` | 此脚本显示数据库参数及其当前值。包含一个开关，用于显示或隐藏参数。要求您以 SYSDBA 身份登录。 |
| `parmsd.sql` | 此脚本显示数据库参数及其描述。包含一个开关，用于显示或隐藏参数。要求以 SYSDBA 身份登录。 |
| `part_size2.sql` | 此脚本显示 `DBA_SEGMENTS` 报告的分区大小。 |
| `pool_mem.sql` | 此脚本提供了一个对 `V$SGASTAT` 的简单查询，显示分配给各种“池”的内存量。 |
| `queued_sql.sql` | 这个简单的脚本查询 `V$SQL_MONITOR`，查找因并行语句排队功能而被排队的语句。 |
| `report_sql_monitor.sql` | 这是一个调用 `DBMS_SQLTUNE.REPORT_SQL_MONITOR` 的脚本。 |
| `si.sql` | 此脚本从 `V$MYSTATS` 中显示统计信息 `Cell Physical IO Bytes Saved by Storage Index` 的当前值。它提供了一种快速检查存储索引使用情况的方法。 |
| `snapper.sql` | 这是本书中用到的最为健壮的脚本。它实际上更像是一个监控程序，可以报告关于活动会话的极其广泛的信息。该脚本和文档可以在 Tanel Poder 的博客 [`http://blog.tanelpoder.com`](http://blog.tanelpoder.com/) 上找到。 |
| `ss_off.sql` | 此脚本通过 `alter session` 关闭智能扫描（Smart Scans）（即设置 `CELL_OFFLOAD_PROCESSING=FALSE`）。 |
| `ss_on.sql` | 此脚本通过 `alter session` 开启智能扫描（Smart Scans）（即设置 `CELL_OFFLOAD_PROCESSING=TRUE`）。 |
| `table_size.sql` | 此脚本显示 `DBA_SEGMENTS` 报告的对象大小。还有另一个版本 (`table_size2.sql`)，基本上是相同的脚本，但输出的列数更少。 |
| `valid_events.sql` | 此脚本显示与文本字符串匹配的等待事件列表。要求您以 SYSDBA 身份连接。 |
| `whoami.sql` `whoami_12c.sql` | 此脚本显示当前会话信息，包括 SID、Serial#、上一个 SQL 哈希值和操作系统影子进程 ID。12c 版本还会报告线程 ID，以防您以 `threaded_execution = true` 方式操作。 |
