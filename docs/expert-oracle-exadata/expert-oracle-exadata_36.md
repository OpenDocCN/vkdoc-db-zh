# 配置数据库服务器

以下流程描述了如何配置新的计算节点：

1.  将您的 IB 交换机固件升级到当前版本或最新补丁。为您安装新硬件的 Oracle 硬件技术人员可以为您完成此操作，或者您可以自行下载最新补丁并进行安装。推荐的固件补丁可在 MOS 注释中找到：`888828.1`。
2.  将现有设备上的 Exadata 存储服务器软件版本更新为与新主机一致。这也包括更新计算节点上的操作系统。
3.  如果可能，请将 Oracle Grid Infrastructure 和数据库软件更新到与您现有系统运行的软件版本相对应的最新捆绑补丁。理想情况下，软件应运行在最新版本和捆绑补丁上。
4.  使用 Exadata 部署助手为新计算节点和存储单元生成 IP 地址和主机名。请勿包含与现有主机相关的信息。
5.  在您的 DNS 服务器中注册新的主机名和 IP 地址。
6.  逐个启动您的新计算节点。首次启动时，`ipconf`将自动启动，允许您输入网络设置。
7.  在数据库服务器上，将以下文件从您现有的某个数据库服务器复制到新的数据库服务器：
    *   `/etc/security/limits.conf`
        此文件在所有计算节点上都是相同的。
    *   `/etc/profile`
        此文件在所有计算节点上都是相同的。
8.  将用于运行 Exadata 部署助手的 OEDA 版本安装到第一个新计算节点上，包括步骤 4 中创建的文件。
9.  在第一个新计算节点上运行`install.sh`，直到但**不包括**`CreateGridDisks`步骤。
10. 按照《所有者指南》中的步骤，将 Grid Infrastructure 和数据库主目录克隆到新服务器。该过程类似于向任何 Oracle RAC 系统添加新节点。

**注意**

决定是扩展现有集群还是创建新集群可能是一个艰难的选择。对于许多客户而言，选择取决于许多不同的因素；新设备与现有设备之间的硬件差异（例如 V2/X2-2 组合与 V2/X3-2 组合）、数据库大小、整合工作负载以及停机时间要求都是决策的考量因素。虽然事后逆转决定并非难事，但这确实涉及额外的工作，并且可能对正在被重新配置的设备上运行的数据库产生影响。一如既往，在决定升级方向之前，请坐下来仔细权衡每种方法的利弊。
