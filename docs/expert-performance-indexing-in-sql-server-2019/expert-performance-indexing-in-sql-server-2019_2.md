# 引言

当今世界，一切都围绕着数据。从管理我们生活的应用程序，到指导我们决策的分析，数据无处不在。数据的背后，数据库提供了获取数据的引擎，但如果没有合适的索引，我们就缺乏高效访问数据的燃料。

当谈到索引时，没有哪种单一结构比索引更能帮助从数据库中检索数据。SQL Server 的新特性提供了利用和访问数据的新颖独特方式，但如果不回归基础并确保数据被正确索引，你会发现即使是新特性也无法达到预期效果。

索引既代表了数据的存储方式，也代表了从数据库中检索数据的访问路径。没有索引，数据库就是一团无序的混乱，缺少了找到你所需信息的路线图。

在我处理数据平台的经验中，为性能调优和应用故障提供的最常见解决方案之一，就是为底层数据库提供正确的索引。通常情况下，为数据库中最常访问的表添加一两个索引就能带来显著的性能提升——这比针对单条 SQL 语句调优数据库的效果要好得多。这是因为一个索引会影响到针对数据库运行的许多 SQL 语句，从而在整个工作负载上提升性能。

管理索引看似简单。不幸的是，它们表面上的简单性往往正是它们被忽视的关键。开发人员常常假设数据库管理员会负责索引工作。或者数据库管理员假设开发人员在开发应用功能时会构建必要的索引。虽然这些主要是沟通不畅的情况，但人们需要知道如何确定哪些索引是必要的，以及这些索引的价值。本书将提供这些信息。

除了上述情况，还有一个事实是应用程序及其使用的数据会随时间变化。创建的特性和用于调优数据库的设置可能不如预期的有用，或者一个小小的功能变更可能导致应用程序和底层数据库的使用方式发生巨大变化。所有这些变化都会影响数据库以及需要访问的内容。随着时间的推移，需要对数据库及其索引进行审查，以确定当前的索引是否适应新的负载。本书也提供了这方面的信息。



## 本书内容是什么？

从头到尾，本书提供了帮助你从索引新手成长为专家所需的知识。各章节的安排使你可以在任何部分开始阅读，以填补知识空白，并由此扩展。无论你是几乎不会拼写“索引”、需要理解基础知识，还是需要构建索引方法论，都能在本书中找到所需信息。

第 1 章涵盖索引基础。它为所有后续章节奠定了基础。本章提供了关于 SQL Server 中可用索引类型的信息，涵盖了一些主要的索引类型，定义了它们是什么以及如何创建。本章还探讨了可用于更改索引结构的各种选项。从填充因子到包含列，所有可用的属性都得到了定义和解释。

第 2 章承接上一章内容。除了定义可用的索引类型外，本章还着眼于索引的物理和逻辑结构以及构成索引的组件。这种对索引的内部理解，为掌握索引在某些情况下为何以特定方式表现提供了基础。在研究索引结构的过程中，你将熟悉可用于自行深入探究这些结构的工具。

在理解了可用索引及其构建方式之后，第 3 章探讨了存储在索引上的统计信息以及如何使用这些信息；这些统计信息有助于洞察 SQL Server 如何利用索引。本章还提供了解析索引为何可能未被选用以及为何以某种方式表现所需的信息。你将通过动态管理视图，更深入地理解 SQL Server 如何收集这些信息以及哪些数据值得审阅。

并非所有索引类型都在第一章中进行了全面讨论；未涉及的类型将在第 4、5 和 6 章中介绍。除了行存储和列存储索引结构外，还有其他一些索引类型，例如可扩展标记语言（XML）索引、空间索引、全文索引和语义搜索索引。这些索引适用于特定场景。在这些章节中，你将探究这些其他索引类型，了解它们能提供什么功能，并审视应在何种情况下实现它们。

与前三章类似，第 7 章深入探讨了内存优化表。内存优化表是 SQL Server 2014 的新功能，提供了独特的能力，当表联机驻留在内存中时可以提升性能。本章将研究索引如何在这些类型的表上运作，以及仍然存在哪些限制。

第 8 章识别并破除了一些关于索引的常见误解。同时，它概述了关于索引表的一些最佳实践。当你在后续章节中开始使用工具和策略构建索引时，牢记这些信息将非常重要。

掌握了索引选项后，接下来需要解决的是维护问题。在第 9 章中，你将了解在环境中维护索引时需要考虑的事项。我们将探讨索引的碎片以及支持 SQL Server 如何确定索引可用性的底层统计信息。

SQL Server 提供了自动化构建索引能力的工具。第 10 章探讨了这些工具，并审视了如何能够以最少的努力开始在你的环境中构建索引。讨论的四个工具是：缺失索引动态管理视图（DMVs）、数据库引擎优化顾问（DTA）、查询存储和自动数据库调节。你将了解这些工具的益处和问题，并获得关于如何在你的环境中有效使用它们的指导。

仅靠这些工具并不能提供索引数据库所需的一切。在第 11 章，你将开始研究如何确定数据库和表所需的索引。选择在数据库中构建哪些索引有多种策略。它们可以根据查询优化器的建议来构建，也应构建以支持诸如外键之类的元数据结构。对于每种索引策略，在决定是否构建索引时，都有许多因素需要考虑。

有效索引的一部分是编写能够利用索引的查询。第 12 章讨论了多种索引策略。有时在查询数据时，你认为会使用的索引最终并未被使用。这些情况通常与查询的结构或所检索的数据有关。索引可能因为 SARGability 问题（即查询未能正确利用索引的选择性）而被跳过。也可能因为临界点问题而被跳过，例如从索引中检索数据的读取操作数量可能超过扫描该索引或其他索引的读取操作数量。这些问题既影响索引选择，也影响某些索引的有效性和合理性。

如今的 DBA 不再只面对单个需要索引的表。一个数据库可能包含数十、数百甚至数千个表，所有这些表都需要有合适的索引。从第 13 章开始，你将学习一些方法，不仅为单个数据库，也为服务器上乃至你环境中的所有数据库进行索引设计。

## 本版有何新内容？

自本书上一版以来，SQL Server 已发布了三个新版本，关于索引如何应用于数据库和数据的方式发生了重大变化。涉及索引的 SQL Server 一些关键变化如下：

*   对内存优化表和列存储索引的索引限制的更改
*   索引维护流程的改进，包括增强的处理能力以及暂停和重启索引重建的能力
*   用于审查查询执行以识别和自动化索引选择的新工具
*   分区和统计信息的改进
*   可用动态管理对象（DMOs）的变化，增强了检查索引和数据页的能力

所有这些以及更多的变化分散在本书的所有章节中。尽管自上一版以来 SQL Server 已经发布了多个版本，但主要焦点将放在作为 SQL Server 当前状态的 SQL Server 2019 上。在适用的情况下，会包含信息以指出自上一版（即 SQL Server 2016 和 2017）以来新增的功能。

## 总结

如前所述，数据很重要，而索引为你提供了访问这些数据的途径。通过阅读本书各章，你将掌握了解环境中索引所需的知识。你还将学会如何找到所需信息以提升环境的性能。
