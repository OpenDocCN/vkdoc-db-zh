# Veridata 比较工作原理

默认情况下，Veridata 比较活动分为两个步骤：初始比较和确认比较。如果数据在比较期间预计不会更改，或者数据已通过分区 SQL 谓词进行筛选，则确认比较是可选的。初始比较会生成一份不同步报告 (`OOS`)。确认不同步报告 (`COOS`) 则是在确认比较过程中创建的。

默认情况下，Veridata 会对所有主键列进行逐字（值对值）比较。对于所有非键列，它使用哈希值进行比较。行哈希值是一种压缩的唯一数字签名，用于比较两个源值。例如，一个 2000 字节的行长度可以转换为 12 字节的哈希值。这并非绝对，但两个原始值相同的概率非常高。这减少了从 Veridata 代理到 Veridata 服务器的网络流量。比较方法可以在“列映射”屏幕中更改。

### Veridata 如何为您提供帮助

您可以通过多种不同方式使用 Veridata，我无法在此一一列举。以下是一些关键特性和优势：

*   它与数据复制工具（如 GoldenGate 单向复制和双向同步）无缝协作。GoldenGate 仅捕获和传送数据，但除非您添加一些自定义代码进行验证，否则它不会为您检查数据。Veridata 可以进行两阶段比较，报告关于不良数据以及 GoldenGate 跳过的任何数据的数据差异。
*   在所有系统中增强企业数据质量，避免不良数据导致决策失误、服务级别协议失败，并最终带来运营、财务和法律风险。借助 Veridata 持续验证所有关键业务数据，您可以在业务用户报告之前识别出不良数据。Veridata 支持 Oracle 物化视图、视图以及 Veridata 特定的分区，使其能够比较不同粒度和基数的数据。
*   由于高性能，减少了数据比较时间。Veridata 通过在代理的提取层使用哈希值来优化比较，然后在单独的服务器中比较数据。因此，它消除了源数据库和目标数据库上的开销。
*   它是无侵入性的，因此可以在数据变更时实现实时数据比较，且零停机。我们可以在两个 OLTP 系统的数据正在变更时比较它们之间的数据。Veridata 将对数据同步进行两阶段确认。
*   支持数据或数据库迁移项目中的初始复制和实时同步阶段，以确保数据准确性。我们可以使用 GoldenGate 进行不停机迁移，但通过在后台运行 Veridata 来交叉检查源数据库和迁移后数据库之间的数据，我们无需在迁移的任何阶段编写昂贵的查询来进行数据比较。
*   支持 Oracle、SQL Server、Teradata、HP Enscribe 和 HP SQL/MP 数据库之间的跨数据库比较。
*   易于使用。IT 和业务用户均可使用。我们无需编写单个 SQL 语句即可设置大多数比较。
*   它是一个独立工具，不依赖于 Oracle GoldenGate。您完全无需安装 GoldenGate 即可运行 Veridata。
*   它具有基于角色的安全性，以控制访问权限并保护敏感数据。
*   选择性比较可比较选定的列。通过适当设置，还可以进行增量比较。
*   它具有灵活的命令行界面，可用于自动化和调度比较作业。
