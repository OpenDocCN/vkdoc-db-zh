# 附录 A 数据库审计选项比较

## 特性比较

*   审计文件是否会保留在磁盘上？ **否**
*   是否会删除历史记录？ **是**
*   磁盘位置是否可配置？ **是**
*   是否可以在启用和运行时对其进行删除和修改？ **是** **否**
*   **保存位置**
    *   `event_file` 作为磁盘上的 `.xel` 文件
    *   磁盘上的 `.sqlaudit` 文件
    *   `ring_buffer`
    *   应用程序日志 (`Application Log`)
    *   `event_counter`
    *   安全日志 (`security Log`)
    *   `histogram`
    *   `pair_matching`
    *   `etw_classic_`
    *   `sync_target`
*   **能否自定义文件的数量、位置和大小？** **是** **是**
*   **能否在不解析 XML 的情况下进行查询？** **否** **是**
*   **是否提供关于所做更改的主机信息？** **是** **仅在 SQL Server 2017 及更高版本中**
*   **模板** **是** **否**
*   **能否筛选捕获的内容？** **是** **是**
*   **能否审计某个用户的具体操作？** **是** **是**
*   **能否捕获服务器指标，如等待统计信息或连接跟踪？** **是** **否**
*   **能否在服务器上设置多个？** **是** **是**
*   **使其工作所需的项数** **一** **二到三**

## 用例

*   **数据库服务器上发生的所有事情**
    *   **扩展事件** – 无筛选地捕获 `rpc_completed` 和 `sql_batch_completed`。请注意，这可能会产生大量审计数据。更多信息，请查阅第[7 章](https://doi.org/10.1007/978-1-4842-8634-0_7)“通过 GUI 实现扩展事件”和第[8 章](https://doi.org/10.1007/978-1-4842-8634-0_8)“通过 SQL 脚本实现扩展事件”。
*   **整个服务器的 DDL 和安全更改**
    *   **SQL Server 审计** – 这将需要一个审计和一个服务器审计规范。更多信息，请查阅第[4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过 GUI 实现 SQL Server 审计”和第[5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实现 SQL Server 审计”。
*   **一个数据库上的 DDL 和安全更改**
    *   **SQL Server 审计** – 这将需要一个审计和一个数据库审计规范。更多信息，请查阅第[4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过 GUI 实现 SQL Server 审计”和第[5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实现 SQL Server 审计”。
*   **一个表上的 DDL 更改**
    *   **SQL Server 审计** – 这将需要一个审计和一个数据库审计规范，该规范需指定这一个表以及您想要捕获的相关操作。更多信息，请查阅第[4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过 GUI 实现 SQL Server 审计”和第[5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实现 SQL Server 审计”。
*   **服务器级别和仅一个数据库的 DDL 及安全更改**
    *   **SQL Server 审计** – 这将需要一个审计、一个服务器审计规范和一个数据库审计规范。更多信息，请查阅第[4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过 GUI 实现 SQL Server 审计”和第[5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实现 SQL Server 审计”。
*   **某个用户的所有操作**
    *   **SQL Server 审计** – 这将需要一个带有筛选器的审计，以仅捕获该用户的活动，以及一个服务器审计规范。更多信息，请查阅第[4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过 GUI 实现 SQL Server 审计”和第[5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实现 SQL Server 审计”。
    *   **扩展事件** – 捕获 `rpc_completed` 和 `sql_batch_completed`。为您选择的每个事件在 `server_principal_name` 上设置筛选器。更多信息，请查阅第[7 章](https://doi.org/10.1007/978-1-4842-8634-0_7)“通过 GUI 实现扩展事件”和第[8 章](https://doi.org/10.1007/978-1-4842-8634-0_8)“通过 SQL 脚本实现扩展事件”。
*   **一个数据库中发生的所有事情**
    *   **SQL Server 审计** – 这将需要一个带有筛选器的审计，以仅捕获该数据库的活动，以及一个服务器审计规范。

## 附录 A：数据库审计选项对比

*   **配置更改**

#### SQL Server 审计
这将需要一个审计和一个数据库审计规范，用于捕获对 `sp_configure` 的 `execute` 操作。更多信息，请查阅[第 9 章](https://doi.org/10.1007/978-1-4842-8634-0_9)“跟踪 SQL Server 配置更改”。

*   **DML 更改**

#### SQL Server 审计
如果您不需要跟踪实际的数据更改本身，而是 DML 语句本身。这将需要一个审计和一个数据库审计规范，用于在您想要审计的每个表上捕获 `INSERT`、`UPDATE` 和/或 `DELETE` 操作。更多信息，请查阅[第 4 章](https://doi.org/10.1007/978-1-4842-8634-0_4)“通过图形界面实施 SQL Server 审计”和[第 5 章](https://doi.org/10.1007/978-1-4842-8634-0_5)“通过 SQL 脚本实施 SQL Server 审计”。

### 更改跟踪
如果应用程序需要跟踪数据是否更改，而不是数据的确切值。更多信息，请查阅[第 10 章](https://doi.org/10.1007/978-1-4842-8634-0_10)“其他 SQL Server 审计和跟踪方法”。

#### 变更数据捕获
如果您需要跟踪数据更改前后的确切值。更多信息，请查阅[第 10 章](https://doi.org/10.1007/978-1-4842-8634-0_10)“其他 SQL Server 审计和跟踪方法”。

*   **成功和失败的登录**

#### SQL Server 审计
您将需要一个审计来存储审计数据，以及一个服务器审计规范来收集登录信息。更多信息，请查阅[第 10 章](https://doi.org/10.1007/978-1-4842-8634-0_10)“其他 SQL Server 审计和跟踪方法”。

#### 扩展事件
您将需要创建一个使用 `login` 事件和 `error_reported` 事件的会话。更多信息，请查阅[第 10 章](https://doi.org/10.1007/978-1-4842-8634-0_10)“其他 SQL Server 审计和跟踪方法”。
