#### C2 审计与通用准则合规性

`C2`和`通用准则合规性`是国际认可的、在审计时需遵循特定安全指南的标准。这类审计是一种全面的日志记录，涵盖数据库服务器上的所有活动。如果没有审计员要求你开启此项，请保持关闭状态。它可能对服务器性能产生重大影响。系统会为每组相关的审计活动分配一个唯一的生成 ID。你可以在服务器级别启用这些功能。图 2-11 展示了启用位置。

#### 时态表

此内置数据库功能允许你查看表中数据在任意时间点的状态。它是一个系统版本控制表，保留了数据更改的完整历史。每行的有效性由系统（即数据库引擎）管理。版本控制通过一对表实现：当前表和历史表。这两个表都包含两个`datetime2`类型的列，用于定义每行的有效期，称为`PERIOD`列。当前表包含每行的当前值。历史表包含每行的每个先前值以及其有效开始时间和结束时间。

#### 审计 Azure SQL 数据库

审计 Azure SQL 数据库与审计 SQL Server 数据库有很大不同，尤其是在涉及`SQL Server 审计`时。`SQL Server 审计`在`Azure SQL Database`中不可用。你需要使用 Azure 门户来设置审计，不能使用`SQL Server 审计`。
