## 引言

人们通常认为审核既可怕又无聊。可怕是因为审计员会看到您做错了事，您会惹上麻烦。无聊是因为您想象的是文员在成堆的文件中寻找一些小错误。

我曾在一个有严格审核要求的金融公司工作。我尝试使用 `SQL Server 审核`和扩展事件（`XEvents`）来实现 `PCI` 合规性。当持怀疑态度的 `PCI` 审计员审查了我的实现方案并说



**出色地**通过了。这加深了我对审计日益增长的热爱。

#### 本书内容

本书通过具体示例向你展示审计如何让你的工作生活更轻松。

设置审计后，你并非**一无所知**。我在当前工作中设置审计，是因为我厌倦了人们将问题归咎于数据库变更。由于我没有进行审计，我完全不知道是否有任何变更发生。现在，我可以明确说出具体发生了什么变更，并与用户协作，判断此变更是否破坏了某些东西。

本书首先从高层次概述了为何需要审计以及存在哪些审计类型。接着详细介绍了 SQL Server、Azure SQL 和 AWS RDS SQL Server 中可用的审计类型。你将学习如何使用 SQL Server 审计和扩展事件来跟踪架构、安全性和配置的变更。并非所有变更都能通过这些功能捕获，因此本书还涵盖了其他功能，如变更数据捕获（CDC）、时态表和通用标准合规性。

接下来，本书介绍了 Azure SQL 数据库和 Azure SQL 托管实例中的审计方法。与 SQL Server 相比，审计这些服务的方式有所不同。此外，你还将学习如何在 AWS RDS SQL Server 中使用 SQL Server 审计和扩展事件。

阅读本书后，你将掌握许多用例和脚本，助你实施自己的审计策略。
