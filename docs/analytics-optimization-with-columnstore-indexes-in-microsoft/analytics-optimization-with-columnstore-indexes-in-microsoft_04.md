# 非结构化数据

并非所有数据都应该放入 **SQL** 数据库或任何结构化数据源。主要由文本和标记语言组成的数据在事务型数据库的限制下难以进行搜索，即使启用了诸如 **`SQL Server`** 的 **`全文搜索`** 等功能。

当处理的数据是金额、时间或指标时，存储位置有很大的灵活性。但大量的文本和文件更适合存储在**非结构化数据存储库**中，例如基于 **`Hadoop`**、**`Hive`** 或某个软件供应商提供的类似技术构建的数据湖。

如果需要一种数据存储解决方案，而源数据又是大型的文本数据，请考虑事务型数据库之外的目的地。虽然本书不旨在探讨这些解决方案，但了解何时应该考虑它们，有助于避免为数据选择了错误的技术架构。
