# 引言

分析数据对于开发人员、分析师和数据专业人员而言是一个持续存在的挑战。其快速增长，加上组织对复杂问题的快速解答的持续需求，引出了一个问题：如何高效地存储海量数据，以实现实时分析。

## 本书简介

本书使用列存储索引作为工具，来解决高效存储不同类型分析数据的挑战。诸如数据仓库、日志记录、报告、分析和数据科学等应用程序，都可以通过有效利用列存储索引来解决。此外，借助此工具还能实现实时操作分析。

*《Microsoft SQL Server 中的列存储索引分析优化》*一书在结构上分为三个功能不同的部分：

-   章节 1 和 2：介绍分析数据及其挑战
-   章节 3 和 4：列存储索引基础与架构
-   章节 5–15：深入探讨特性、架构与最佳实践

目标是提供足够的理论、架构、用例和代码，以便立即运用此功能解决实际问题。

每章都从一个高级概述开始，解决特定的功能或数据架构需求，然后深入探讨详细内容。虽然并非所有专业人员都需要确切了解列存储索引如何压缩数据，但掌握这些知识将在未来需要提升性能时体现价值。

本书中的大多数示例都源自 `WideWorldImportersDW` 和 `WideWorldImporters` 数据库中的数据。这些数据库本身提供的数据不足以构成有意义的分析数据示例。为此，我们创建、复制并修改了新表，以生成足够大且有趣的对象，用于有效演示列存储索引。虽然现实世界中的表可以轻松包含数百亿行数据，但本书中的示例采用了约 2500 万行。这在有效测试数据与因过大而构建耗时的表格之间取得了一个良好的平衡。

## 目标读者

本书面向数据专业人员、架构师和开发人员，旨在作为一种工具，帮助他们快速、经济、高效地掌控报告和分析数据。本书涵盖的主题从入门级到相当高级，因此既可作为初学者的工具，也可供拥有多年数据架构经验的人士使用。

对列存储存储技术经验不多的读者将学习首次实施此功能所需的一切知识，并开始受益。数据专家可以利用本书中提供的特性与架构细节，以及贯穿全书的演示，作为优化现有数据结构的工具。

列存储索引不仅适用于报告和分析数据。它们也可用于满足组织对实时操作分析的复杂需求。当分析工作负载与事务工作负载混合时，存在延迟、阻塞、锁定和死锁的风险。处理混合工作负载的开发人员和数据专业人员可以通过使用列存储索引来有效覆盖每种工作负载，从而极大地受益。除了提高性能外，这些努力还可以避免为管理混合工作负载的部分内容而购买或构建新系统的需要。

## 为何选择列存储索引？

市面上有许多解决方案有助于掌控分析数据，但大多数都需要在硬件、软件、技术和架构上投入巨资。除非有真正令人信服的理由，否则并非所有组织都能（或应该）被迫在新解决方案上投入大量资源。

列存储索引可以在 SQL Server 中基于现有硬件和许可证实施，使分析数据解决方案得以实现，而无需在时间和资源上进行大量投入。这便于对该功能进行质量保证和测试，而无需预先承诺。

除了提供稳健的解决方案外，列存储索引还基于成熟的技术，该技术多年来一直是许多其他数据存储产品（如 PowerPivot 和 SSAS）的基础。尽管已经成熟，SQL Server 中的列存储索引仍在随着每个新版本的发布而不断发展和完善。
