# 前言

本书旨在鼓励他人将其 `T-SQL` 开发提升到新的水平。目标不仅是让大家更好地理解如何编写更优秀的查询，还要了解为什么某些方法会优于其他方法。创建 `T-SQL` 语句时具有极大的灵活性。在许多情况下，可以通过多种方式实现相同的 `T-SQL` 结果。本书面向对 `T-SQL` 有一定了解但希望提升整体代码质量的数据库开发人员和数据专业人士。在开始阅读本书之前，您应理解 `T-SQL` 语法，并知道如何编写 `SELECT`、`INSERT`、`UPDATE` 或 `DELETE` 语句。*《Pro T-SQL 2022》* 将帮助您编写一致且性能更佳的代码。您还将学习如何通过使用源代码控制和改进数据库部署流程来保护您的 `T-SQL` 代码。总体而言，本书的目标是为您提供一个编写更优秀 `T-SQL` 代码的框架。作为数据专业人士，我们可能会遇到要求高或期限紧的情况。*《Pro T-SQL 2022》* 的编写旨在帮助您编写能够在未来节省时间和精力的代码。

本书分为五个部分。第一部分涵盖如何提高 `T-SQL` 代码的可读性。其中概述了 `T-SQL` 的各种数据类型，并提供了如何最佳使用这些数据类型的指导。这一部分解释了 SQL Server 中各种数据库对象的优缺点。其他章节讨论了 `T-SQL` 代码的标准化和设计。第二部分解释了如何编写高效的 `T-SQL` 代码。本部分包括使用基于集合的设计来编写 `T-SQL` 代码，以及如何理解硬件与 `T-SQL` 设计之间的关系。您还将学习如何使用执行计划和 SQL Server 2019 中的新功能来提升 `T-SQL` 代码的性能。第三部分讨论了如何管理您的 `T-SQL` 代码。本部分包含的章节涵盖了制定编码标准和使用源代码控制存储代码。为了进一步管理您的 `T-SQL` 代码，您还将学习一些测试和部署数据库代码的方法。第四部分阐述了如何编写能够长期保持可持续性的 `T-SQL` 代码。最后一部分概述了您可以实施的不同选项，以帮助使您的数据库更加安全。这些章节包括安全添加新功能的方法、记录数据库内数据更改的方法以及随时间推移管理数据增长的方法。
