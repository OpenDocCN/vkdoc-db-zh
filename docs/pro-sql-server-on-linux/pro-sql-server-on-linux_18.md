# 第一章：为何在 Linux 上运行 SQL Server？

## 一种新兴技术：容器

本书后续章节将详细探讨如何将 SQL Server 与容器结合使用的技术细节。在此，我将概述 SQL Server 与容器的关系及其与 Linux 的关联。

容器技术领域当前的主要推动力无疑是 Docker。相较于自行定义容器，我更倾向于引用 Docker 官方的解释（[`www.docker.com/what-container`](https://www.docker.com/what-container)）：

> *“容器镜像是一个轻量级、独立、可执行的软件包，包含运行它所需的一切：代码、运行时、系统工具、系统库和设置。容器化的软件无论环境如何，始终以相同方式运行，适用于基于 Linux 和 Windows 的应用程序。容器将软件与其周围环境隔离开来，例如开发与测试环境之间的差异，并有助于减少团队在同一基础设施上运行不同软件时的冲突。”*

阅读之后，你可能会想：为什么我们不直接采用容器，而要基于 SQLPAL 架构呢？

## 为什么 SQL Server 要运行在 Linux 上？

这主要有以下几个原因：

*   Linux 上的 Docker 引擎仅支持包含 Linux 操作系统的镜像，因为请记住：容器需要与主机操作系统交互。因此，如果我们仅采用容器，将相当于回到起点，需要将所有代码移植到 Linux 上。
*   我们希望为 Linux 操作系统提供原生能力，以获得最佳性能。
*   尽管我们希望通过 Docker 容器及其带来的新场景来全面支持 SQL Server，但我们并不希望依赖它来支持 Linux，尤其是如果容器存在某些在 Linux 原生环境下不存在的限制。

我对 Docker 容器的未来，以及 SQL Server 在使容器成为新场景（包括可移植性、一致性和 DevOps 场景）的可行技术方面所扮演的重要角色，感到十分兴奋。

##### 数据库容器

我记得不久前曾问过 Travis Wright（他是将 SQL Server for Linux 推向发布的团队的关键项目经理之一）关于 SQL Server 对 Windows 容器的支持问题。他最初的回应让我感到惊讶。他说：“Bob，你为什么关心这个？”他的意思是，如果我们能进入一个以包含 SQL Server、数据库以及应用程序任何依赖部分的*数据库*容器为中心的世界，而不是担心容器镜像中的操作系统，难道不是很好吗？我从未那样想过，我们可能尚未准备好，但他的想法非常正确。这正是容器的承诺之一。

##### 平台独立性、可移植性与一致性

这里的“平台”指的是运行容器的宿主机。Docker 提供了这种独立性，因为我可以*编排*（你将在关于容器的章节中学到这个含义）一个包含 SQL Server 的 Docker 镜像，并将该镜像作为容器运行在 Windows、Linux、MacOS 或其他容器云环境中，例如 Azure 容器服务 (AKS)、亚马逊弹性容器服务 (ECS)、谷歌云平台、红帽 OpenShift 和 SUSE CaaS。而且由于它是一个容器，在所有环境中运行的是*相同的镜像*。这就是可移植性！它提供了一个可靠的、*一致*的包，包含已知的 SQL Server 版本以及你的数据库、脚本或任何你放入镜像中的依赖项。

##### 持续集成/持续部署

也许你听说过*DevOps*这个词？这个已存在多年的概念，其核心是将软件开发与运营的角色及任务结合起来。它之所以流行，是因为它提供了一种机制来缩短开发生命周期，并允许更频繁地部署应用程序。

持续集成与持续部署 (CI/CD) 是实现这一目标的方法……



一个更高效的 DevOps 生态系统。利用容器，开发者可以持续集成其应用程序变更，并将其部署到测试和生产环境中。开发者已经使用这些技术一段时间了，但像 SQL Server 这样的数据库平台通常并未纳入其生命周期。一个典型的 SQL Server 会托管在服务器上，开发者很难将其容器与应用程序一起维护，从而连接到解耦的 SQL Server 和数据库（以及像 SQL Agent 作业和脚本这样的对象）。现在，SQL Server 已拥抱容器世界，它可以成为包含在 CI/CD 流水线中的容器的一部分。

##### Kubernetes

容器的前景在可移植性、一致性和效率方面具有巨大潜力。但是，如何在 CI/CD 流水线或大规模生产环境中协调许多数据库容器的执行呢？一种名为 Kubernetes 的技术就是为了解决这个问题而创建的。Kubernetes 是一个开源系统，旨在帮助在一个完整的生态系统中管理、部署和编排众多容器。其优势之一是为容器提供了一种天然的高可用性解决方案，这恰好满足了许多 SQL Server 用户为生产数据库提供高可用性解决方案的需求。Kubernetes 已成为大规模容器管理和部署的领先解决方案，并已被几乎每一个支持容器执行和部署的公共云供应商所采纳。
