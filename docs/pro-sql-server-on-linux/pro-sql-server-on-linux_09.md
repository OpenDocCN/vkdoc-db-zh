### 前言

将 SQL Server 引入 Linux 是一次充满冒险的征程。我们从启动到最终产品发布用了不到两年时间。然而，在项目成功之前的第一年里，许多事情必须顺利推进。过去，SQL Server 团队曾多次尝试进行这项工作，但每次都碰壁而退。大约在 2010 年，Hal Berenson 甚至发表了一篇博客，阐述为什么将 SQL Server 移植到 Linux 终究不是一个好主意。

尽管如此，在 2014 年底，团队决定再次着手将 SQL Server 移植到 Linux。最初由两位项目经理——Joe Idziorek 和 Tobias Ternstrom——组成的团队专注于评估商业机会。他们很快确信，存在的商业机会不止一个，而是多个。然而，要说服公司内部的其他人却遇到了困难。为了获得认真对待，他们必须找出一种工程实现方法。过去的工程估算令人望而生畏：预计需要 10 到 20 名工程师全职投入，耗时五年。团队知道他们既没有这么多工程师，也没有这么多时间。他们需要一个新的工程计划。

恰在那时，我正在考虑新的职业机会。SQL Server 团队的朋友们将我介绍给了 Joe 和 Tobias，我们开始了交流。我同意团队的看法，认为直接移植并非良策。我请团队给我几天时间仔细思考实现方案。机缘巧合的是，我此前熟悉 Drawbridge 研究，并且曾在之前的项目中利用它作为平台抽象层，在另一个操作系统中运行 Windows 应用程序。因此，没过多久我就将各个点串联起来，想到我们或许可以利用 Drawbridge 技术将 SQL Server 引入 Linux。于是，我毅然决然地加入了这个项目。

Drawbridge 技术是第一个必须取得成功的关键环节。许多人，包括曾参与 Drawbridge 的研究人员和工程师，都对这种方法持怀疑态度。他们担心 Linux 和 Windows 运行时之间存在阻抗不匹配的问题。此外，几乎每个人都对性能表示质疑。就我个人而言，我对前者的担忧甚于后者。我不知道我们能否让 Windows 异常处理完全正常工作，也不知道能否让栈增长语义正常运作。然而，我对性能问题倒不太担心。我了解 SQL Server 和 Drawbridge 的内部工作原理，因此对于如何从这两者中榨取性能颇有把握。

在掌握 Drawbridge 之后，我们不到一个月就让 SQL Server 在 Linux 上运行起来了。事实证明，微软研究院的工程师 Andrew Baumann 已经有一个在 Linux 上运行的 Drawbridge 早期原型。他为了自己的研究已经完成了这项移植工作。让 Drawbridge 在 Linux 上运行起来——即使是以原型的形式——是第二个必须取得成功的关键环节。我于 2 月 2 日加入了团队。

xxi 前言


2015 年，在安德鲁的帮助下，原型在月底前已能在 Linux 上运行 SQL Server。到了 2015 年 2 月底，我们不仅完成了商业案例和工程计划，还让 SQL Server 实际在 Linux 上运行了起来。

让 SQL Server 在 Linux 上运行使项目变得真实可行。乔、托比亚斯和我发起了一场我们称之为“路演”的宣传活动。我们走访公司内部，与有影响力的高管交流，收集他们对商业案例和其他细节的反馈。有趣的是，许多高管直到我们交谈的最后阶段——在展示并讨论完商业内容后，我们演示了在 Linux 上运行的 SQL Server 时——才会认真对待整个项目。那一刻他们往往会惊叹不已。最有趣的故事之一是我们与数据部门的高管们交谈：当我们展示在 Linux 上运行的 SQL Server 时，一位技术院士甚至从椅子上摔了下来。看到 SQL Server 在 Linux 上运行，他和房间里的其他人都感到非常震惊。

出乎意料的是，为这个项目招募团队并不容易。我们发现，许多工程师本质上都是纯粹主义者。我们试图说服加入项目的许多人，更倾向于纯粹的移植方法，而不是我们采用的方法。我们花费了相当大的努力才让关键工程师们相信这种方法是可靠的。大约花了六个月时间，我们组建了由六名工程师组成的核心团队：尤金·比鲁科夫、乔治·雷尼亚、迈克尔·纳尔逊、斯科特·科纳斯曼、布莱恩·詹福卡罗和罗伯特·多尔。将这些工程师招募进项目，被证明是该项目历史上至关重要的时刻。如果这些核心成员中任何一位没有加入或离开了团队，我们成功的几率几乎为零。即使现在，我仍然认为我们非常幸运，因为所有这些工程师都在项目最需要的时候加入了。

接下来，我们的许多运气来自于工程师、高管、项目经理及其他或隐或显与项目相关的人士。很多时候，人们甚至没有意识到他们所提供的支持。大约在 2015 年底，我们迎来了一位新的公司副总裁，他成为了项目的坚定支持者，并帮助我们加快了审批流程，使我们得以在 2016 年 3 月的一次“数据驱动”活动上较早地进行了公开宣布。有些人试用了该解决方案，并在公司内成为了我们的布道者。公开运作项目对此帮助巨大。到公开宣布时，公司内大约有三百人订阅了我们的工程分发列表。可想而知，该项目受到严格的保密协议约束。不用说，我们没有发生任何泄密。

本书也给了我一个机会，分享我个人在 25 年间支持和从事`SQL Server`工作过程中的一些有趣故事，包括提及过去和现在的`Microsoft`同事。颇具讽刺意味的是，我最终选择在`SQL Server`运行于`Windows NT`平台 25 周年之际来撰写这本书，但却是写关于`Linux`上的`SQL Server`。我在加入`Microsoft`之前是一名`UNIX`开发人员，所以对我来说，能够重新认识`Linux`内核和`Shell`是一件很特别的事。
