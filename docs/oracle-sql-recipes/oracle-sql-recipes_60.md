# SQL 监控与调优

SQL 是用于从数据库创建、检索、更新和删除（CRUD）数据的语言。如果您的 SQL 语句执行缓慢，整个系统都会受到影响。无论您是开发人员还是 DBA，有效地调优 SQL 都是一项关键的工作要求。在对性能问题进行故障排除时，您必须知道如何识别和调整资源密集型的查询。

一个优秀的开发人员必须熟悉查询调优技术。理想情况下，开发人员应在查询部署到生产环境之前主动评估其潜在的性能问题。实际上，糟糕的 SQL 往往会因为测试不充分，或者测试环境无法准确反映生产环境（在硬件、数据量、机器负载等方面）而进入生产环境。在部署了低效 SQL 的场景中，开发人员必须熟悉用于识别和修复性能不佳查询的响应式技术。

DBA（想要保住工作的人）也必须擅长识别那些霸占资源的 SQL 查询。当数据库代码执行性能不佳时，DBA 通常是第一个被寻求帮助的人。

DBA 还应该能够帮助开发人员生成查询执行计划，并为性能不佳的 SQL 查询实施解决方案。

大多数应用程序性能问题都可归因于代码对数据库的访问效率低下。这可能表现为 SQL、PL/SQL、JDBC 等形式。本章重点介绍用于识别和调优资源密集型 SQL 语句的技术。如果您是 SQL 新手，本章将为关键的调优技能奠定基础。如果您已经熟悉 SQL，本章将介绍 Oracle 中与 SQL 调优相关的新功能。我们从基本任务开始，例如监控和识别资源密集型 SQL 语句，然后逐步深入到生成和解释 SQL 执行计划等主题。
