前面的练习是一个简单的示例，展示了添加索引如何提升单个查询的性能。请注意，添加一个能加速某个查询的索引，可能会对应用程序的其他方面产生不利影响。索引至少会增加额外的磁盘空间开销，以及在表进行插入、更新或删除操作时维护索引的处理开销。你应该在测试环境中添加索引，并验证它在提升性能的同时没有产生意料之外的副作用。其他需要考虑的因素包括索引类型（`B 树`、`位图`等）、存储特性以及索引所在的表空间。

**注意** 每一行的`成本`是优化器对每个步骤在系统资源（如`CPU`、`I/O`和`内存`）方面开销的加权估算值。`成本`本身并没有一个具体的度量单位；它是一个数字，优化器用来比较执行计划和步骤。一般来说，`成本`越低，操作和查询的预期性能越好。

## 工作原理

在解读执行计划的每个步骤时，计划中的每个`ID`都与一个操作相关联。

对性能影响最大的两种操作类型是：

- 访问路径
- 连接方法

`访问路径`描述了如何从数据库中检索数据。表 19-3 描述了最常见的执行计划访问路径。通常，当只需要检索表中一小部分数据时，应使用索引访问路径。当需要返回表中的大部分数据时，全表扫描可能更合适。
