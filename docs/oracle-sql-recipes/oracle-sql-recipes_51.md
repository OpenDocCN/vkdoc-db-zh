## 为表和索引使用独立的表空间

数据库管理员们对于将表和索引分离到不同表空间中的优劣存在争论。如果你有从头开始设置存储系统的条件，并且能够设置拥有独立磁盘和控制器组的挂载点，那么通过指定表建立在基于一个挂载点上数据文件的表空间中，而索引建立在基于另一个独立挂载点上数据文件的表空间中，你可能会看到一些 I/O 性能上的好处。

然而，如今常见的实际情况是，你的存储管理员只会给你存储区域网络（SAN）中的一大块存储空间，并且无法保证数据和索引会物理地存储在不同的磁盘（和控制器）上。尽管如此，将索引表空间与表表空间分离仍然有一些合理的理由：

• 你可能希望为表和索引实现不同的物理存储特性（例如区大小），并且你更倾向于让表和索引从表空间继承其存储属性。

• 你倾向于拥有能够分别备份、恢复和还原表表空间和索引表空间的选项。

• 在运行维护报告时，你发现当报告按表空间分段时，管理表和索引更为容易。

如果这些原因中任何一个对你的环境有效，那么值得付出额外的努力来为表和索引使用不同的表空间。
