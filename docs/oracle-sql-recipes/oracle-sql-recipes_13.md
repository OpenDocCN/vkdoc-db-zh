# 引言

## 选择工具

我们在《Oracle SQL Recipes》中包含的所有解决方案都设计为无需借助特殊工具或应用程序即可工作。您应该能够通过 `SQL*Plus` 以即席方式选取一个解决方案并使用它，或者在调用 `Oracle OCI` 或类似客户端库的应用程序代码中包含它。

我们这些长期热心使用 `SQL*Plus` 的人都很欣赏它那种“复古”的格式化和美学处理方式——好吧，我们指的是带有 `Courier` 字体所能提供的所有美感的 1980 年代风格的输出——但这有时会让解决方案，尤其是那些展示输出格式化技巧的解决方案，显得有些失色。多年来，Oracle 一直提供 `SQL Developer`，我们一直是它的热心用户。从 Oracle Database 11g 开始，`SQL Developer` 已成为标准安装的一部分，因此您可能已经拥有它而不自知！我们很乐意您在处理这些解决方案时使用任何喜欢的工具，但如果您有任何疑虑，现在就开始使用 `SQL Developer`，您会感到满意的。

## SQL 解决方案的示例数据

在《Oracle SQL Recipes》中，我们将使用的许多示例都将利用 Oracle 作为每次安装的一部分提供的优秀示例模式——一直可以追溯到 Oracle Database 9i。古老的 `Scott` 模式已经不再使用，所以您可以忘记密码 `"tiger"`。取而代之的是以下六个模式，它们展示了 Oracle 数据库的许多技术功能：

- **`BI`**：商业智能模式是 Oracle Database 11g 的新特性，用于说明 Oracle 中日益增强的 BI 功能。该模式将吸引那些正在使用或考虑使用 `OBIEE` 产品的用户。

- **`HR`**：这是最基础的模式，模拟了非常常见的员工及其工作部门的概念。细心的读者会注意到，这其实就是旧的 `Scott` 模式演变而来的。

- **`IX`**：信息交换模式展示了 Oracle 高级队列和 IPC 技术的能力。我们将在本书末尾探讨这一点。

- **`OE`**：订单输入是另一个经典的示例模式，涉及销售、订单、订单项、客户等概念。任何处理过销售流程、零售系统或在线商务的人都会感到熟悉。Oracle 已经增强了这个模式，以至于它还有自己的子模式，在线目录（`OC`），并且它们之间存在相当多的相互关系。

- **`PM`**：在产品媒体模式中，Oracle 在多媒体对象操作方面的优势得到了突显。这包括用于大型对象操作的高级功能，我们将在第 16 章中探讨。


**SH** 销售历史表可视为一个迷你仓库或数据集市，非常适合用于数据挖掘、分析工具（如数据立方体与上卷操作）的实践。

我们需在此预先说明：我们将对示例数据进行大量自由改动，包括修改数据及其载体对象、进行切片切块操作，甚至删除销毁。请不必担心，因为当你读完本书时，你将会愉快地对更多模式（而不仅仅是示例模式）执行类似操作。
