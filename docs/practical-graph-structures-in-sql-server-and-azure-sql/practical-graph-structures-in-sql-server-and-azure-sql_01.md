*献给 Val，我们共同走过的人生如此精彩……愿未来继续相伴。*

我 23 年前就开始撰写我的第一本书，内容是关于关系数据库设计的。那时我从 David Rozenshtein 博士的课程中学到了一些关于图的知识（他 1997 年的著作*The Essence of SQL : A Guide to Learning Most of SQL in the Least Amount of Time*至关重要）。他的课程极具启发性，教会了我很多如何思考 SQL 问题的方法。那是很久以前的事了（很明显），但在同一门课中，有一个章节专门讲解 SQL Server 中的树结构。我从此便迷上了这个主题。

在我 2012 年出版的数据库设计书籍中，我开始将层次结构作为主题之一纳入。在我最新的一本书中（在 SQL Server 2019 发布后问世），我承诺要写一本关于 SQL Graph 的书。您手中的这本书，就是对那个挑战的回应。

这是我多年来第一本以编程为核心的书籍。通常我更感兴趣的是帮助您塑造设计，但在这种情况下，我想向您展示使用 SQL Server 构建图数据库解决方案的机制，至于之后如何运用，则更多地留给您自己决定。部分原因在于该主题的新颖性和相对复杂性，但也因为图本身被设计为非常灵活的结构……远比标准关系数据库灵活得多。

本书各章节内容如下：

*   第 1 章：什么是图以及图的使用方式。我触及了一些使图之所以为图的基础概念，并简要介绍了数学家用来描述和处理图数据结构的理论。
*   第 2 章：图如何实现以及用于处理图的算法。虽然图的基本结构非常简单，但理解图结构在编码中是如何构建的，并对本书后续内容有所预期，是有价值的。
*   第 3 章：Microsoft 为操作存储在 SQL Server 表中的图数据而实现的语法。它与您处理关系型表时可能已经熟悉的语法相似，但功能要强大得多。在本章中，我将教您如何使用 Microsoft 提供的语法以有趣的方式查询图表。
*   第 4 章：在第 3 章中我展示了基本查询技术，而在本章，我将展示一些有助于您加载 SQL Graph 表中的数据并保护其完整性的方法。
*   第 5 章：使用 SQL Graph 对象构建的树结构，包括在构建生产系统时加载和操作这些节点所需的代码。在本章中，您将探索在 SQL Server 中创建和管理树的代码，以及一个展示其工作原理的示例。
*   第 6 章：在本章中，我将深入探讨性能问题。您将研究一种新的树实现方法，用于与 SQL Graph 对象进行比较，并构建一些可用于编写查询的对象，以更快地报告树中的数据。然后，您将构建一些大型随机数据集，并比较这些方法在特定较大数据集上的性能表现。
*   第 7 章：本章的目标是构建一个能够展示处理非树结构的有向图时某些问题的数据结构。您将构建一个相当简单的物料清单数据结构，以演示处理这些与树相似但仍大不相同的结构时所需的技术。
*   第 8 章：在本章中，您将在 SQL Server 的图对象中对较大数据集进行一些查询。为此，您将实现一个图结构和数据生成工具，以便在符合您预期需求的大型数据集上进行试验。最后，您将探索一组处理图对象的性能提示。

SQL Server 的关系引擎可能永远无法完全替代像 CosmosDB 这样的专用图数据库系统，但即使在其相对初期的阶段（在 SQL Server 生命周期中仅经历了几次迭代），它已经成为一种在现有关系数据结构中快速、轻松扩展数据结构的绝佳方式。
