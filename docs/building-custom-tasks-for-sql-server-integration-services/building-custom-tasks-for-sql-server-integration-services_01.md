# 前言

阅读这本书以准备撰写这篇前言，对我来说是一次怀旧之旅。自定义任务的一些函数名称、代码、`GAC`、将自定义任务加入设计器的各种步骤——所有这些都唤起了我丰富的回忆，那是我们最初构建 `Integration Services` 时的样子。2020 年标志着我们开始编写 `SSIS` 代码已有 20 周年。

`SSIS` 起初是一个小团队，旨在用更多的东西取代 `Data Transformation Services (DTS)`。具体是什么，并不清晰。但我们都知道，新产品需要一个更健壮的数据处理引擎、更好的控制流和更丰富的设计时与运行时体验，尤其是，我们需要解决 `DTS` 的一些设计缺陷。在此不过多怀旧，可以说我们致力于创建一个持久、灵活、可扩展且功能丰富的产品，以充分满足企业数据处理需求。

我们最终完全废弃了 `DTS`，直到后来我们意识到 `DTS` 实际上做了一些我们未支持的很酷的事情，而且由于我们为了弥补 `DTS` 缺陷而做出的一些设计决策，我们严重限制了 `DTS` 客户升级其现有包的能力。答案是什么？将 `DTS` 嵌入到 `SSIS` 中。这是 `SSIS` 中有意且深思熟虑的可扩展性设计（用于支持创建自定义任务）不仅重要而且至关重要的多次实例中的第一次。我们能够在几天内以编程方式将整个 `DTS` 产品嵌入到一个自定义任务中，这就是后来广为人知的 `执行 DTS 包任务`！

Andy 在这里提到了我对 `SSIS` 文档编年史的贡献。在强烈推荐 Andy 的书之前，我想先分享一些相关的事情。

首先，我对 `SSIS` 项目的贡献常常被高估了。我注意到，因为我的名字与那本书相关联，我有时主要与该产品的成功联系在一起。团队中还有许多人做了了不起的事情：Ashvini Sharma、Matt David（我书籍的技术编辑）、Mike Blaszczak、Kamal Hathi、Donald Farmer、Slobodan Bojanic（我书籍的代码评审员）、James Howey、Michael Entin、Runying Mao、Silviu Guea、Nick Berezansky、Sergei Ivanov、Mark Durley、Euan Garden、Anjan Das、Bill Baker、Jeff Bernhardt、Bryan Hartman、Craig Guyer、Ranjeeta Nanda、Ovidiu Burlacu，以及我确信我遗漏的其他几位！`SSIS` 确实是一个团队努力的成果，它产生了一个持续进化、改进、远超我们最初设想并表现卓越的产品。成为该团队的一员是我的荣幸。

当我开始写那本书时，我顾虑重重；那种自我怀疑，我后来了解到在作者中相当普遍。从一开始，我就问自己这类问题：“这值得花时间吗？”“真的会有人买这本书吗？”“我做这件事有什么价值吗？”“人们是只会看到错误和缺点，还是会欣赏我倾注其中的努力和热情？”

在写作早期，一群也想写一本 `SSIS` 书的人联系了我，问我是否可以将我的目录作为他们书的起点。我说，当然可以。我的理由是，写 `SSIS` 的人越多，它的采用率就会越好。

在我准备出版时，`SSIS` 用户教育团队找到了我。由于各种原因，他们在编写 `SSIS` 文档方面落后了，需要一个启动帮助。他们问我是否可以将我的手稿作为 `SSIS` 联机丛书文档的起点。我说是的，当然可以。我感到很荣幸。我写的大部分内容在我甚至还没出版我的书之前，就变成了联机丛书的内容。

人们为什么写技术书籍？原因有很多。有些理由比其他的更合理。这肯定`不是`为了赚很多钱。对我个人而言，我对 `SSIS` 充满热情，并希望社区在吸收这个产品时能得到尽可能多的帮助。因此，当在亚马逊上收到诸如“这本书里的所有内容都能在联机丛书中找到”和“这不是一本教程”的评论时，就显得很讽刺。

那么，Kirk，你想说什么呢？在我认识的技术作者中，以及我有幸交往过的作者，他们都对他们所文档化的产品怀有真诚的热情。真正驱使他们，也驱使我写作的，不是名声、恶名、职业晋升或金钱，而是对产品的热爱以及帮助人们采用并取得成功的愿望。

Andy 也不例外。没有人比 Andy Leonard 更了解、使用、教授和理解 `SSIS`。我知道他亲自撰写这些书籍，是出于他对于帮助人们使用 `SSIS` 在工作中取得成功的热情。

所以，当你阅读、使用并从这本书中学习时，请记住这一切。如果你发现你认为这里有错误或缺点，请记住，这是一项爱的劳动。它是热情与善意结合的产物，旨在帮助你取得成功。撰写技术参考书就是一项艰苦的工作！它不仅仅是文字，还是代码；它不仅要经受你这位读者的批判性审视，还要经受演示代码编译器的严苛考验。

而且，当你发现这本书在你的 `SSIS` 之旅中至关重要时，记得向 Andy 表达感谢，感谢他所有的辛勤工作。这是一个技术作者所能体验到的最好回报。

### 作者初版感想

这本小书最初于 2012 年作为一系列博客文章发表在 Linchpin People 网站上。

在撰写本文时，我提供 SQL Server Integration Services (SSIS) 解决方案已超过十年。在 SSIS 之前，我使用 Data Transformation Services (DTS) 工作了几年。尽管 SSIS 自 2005 版本以来就支持自定义任务和组件，但我最初建议我的咨询客户*不要*开发自定义 SSIS 任务和组件。为什么？

1.  源代码需要在使用这些自定义任务和组件的 SSIS 解决方案的整个生命周期内进行维护和支持。
2.  需要投资购买 Visual Studio Professional（约 1000 美元）来构建代码。
3.  需要相当扎实的.Net 开发技能来理解构建 Visual Studio 工具箱组件的复杂性。

我的第一个担忧仍然存在，但入门门槛已大大降低——至少在成本方面。

当微软®发布免费版本的 Visual Studio（Visual Studio Express 和更近期的 Visual Studio Community Edition）后，一旦我了解到可以使用免费工具来构建自定义 SSIS 任务（本书的主题），我的第二个担忧就消失了。

本书解决了我的第三个担忧。

撰写这个博客系列并重新审视该系列以创作本书，对我来说是件乐事。我祈祷您阅读时能像我写作时一样享受。

> —安迪
