# 1. 本书的故事

偶尔，我的工程好奇心会战胜我，我会问自己一些问题，比如，“自己，我觉得某某事情可能吗？”大多数时候，这些问题只会带来一些脑力练习，没什么其他用处。但有时，努力会带来一些好处。

2012 年夏末的一天，我问自己：“你认为有可能使用`Visual Basic Express`创建一个自定义的 SSIS 任务吗？”疯狂的问题，对吧？

简短的答案是：是的。

为什么这很重要？因为[Visual Studio IDE](https://www.visualstudio.com/vs/)（集成开发环境）现在是免费的。这意味着，如果你拥有 SQL Server 的许可证，就无需再花费更多钱来通过自定义任务和组件扩展 SSIS。当然，你可能*想*花钱获取 Visual Studio 其他版本的一些工具和功能。

我对博客系列进行了修订，创作了本书。有经验的开发者会注意到我使用了`Visual Studio 2019 Community Edition`，并以`SQL Server 2019 Integration Services (SSIS 2019)`版本的`.Net Framework`为目标。


## 部落知识

我在这个过程中遇到了一些问题。起初，我不确定这些问题是否与免费的 `Express Edition` 有关。当你不确定某件事是否可行时，解决问题会很困难。这需要时间和决心，但我解决了足够多的问题，从而撰写了一系列博客文章，这些文章成为了本书的灵感来源。

在 2012 年，我承认自己是一个生疏的软件开发人员。在 `.Net` 时代之前，我曾是一名微软认证解决方案开发专家 (`MCSD`)。对于你们这些年轻的后生来说，我是在那些年份以“1”开头的美好旧时光里获得那个认证的，那时我们得自己用木头雕刻芯片。#GetOffMyLawn ☺

我遇到的一些事情对于经验丰富的软件开发人员来说显而易见。经过多年开发软件，这些经验丰富的开发人员拥有[部落知识](http://en.wikipedia.org/wiki/Tribal_knowledge)——他们不记得自己所知道的全部内容。因此，当一个像我这样的 `.Net` 菜鸟（`.Net n00b`）提问时，他们会默认我已经通过某种相当于肌肉记忆的智力过程，完成了他们本来会做的所有事情。

我没有完成所有那些事情（我提过我是个 `.Net` 菜鸟吗？）。

结果之一是，让我栽跟头的事情相对微小。一旦我知道了诀窍，一切都很好，但这些诀窍被认为是如此基础，以至于人们在回答论坛问题时很少分享它们。人们假设，如果你构建 `Visual Studio` 插件和针对全局程序集缓存 (`GAC`) 的程序集，你必然知道这些基础知识。

如果你不是 `.Net` 菜鸟，你可能知道这些基础知识。但这并不适用于开始时的我。

现在，在编码方面我并非无能。但在 2012 年，我也不再是职业软件开发人员。在 `.Net` 出现之前，我开发过软件。我已经提到了 `MCSD` 认证。我在 1975 年学习了 `BASIC`，并在 1992 年 `Visual Basic` 2.0 版发布时开始编写代码。许多 `VB` 程序员从未跃迁到首先由 `VB 4`（如果没记错的话）支持的面向对象编程 (`OOP`)，但我做到了。通过类来建模对象对我来说很有意义，并且在随后的岁月里让我受益匪浅。它甚至引导我走向了设计模式的概念，这些模式在[使用 SSIS 进行数据集成开发](http://smile.amazon.com/Server-Integration-Services-Design-Patterns/dp/1484200837)中有一定的应用。

此后，我跃迁到了 `C#`，使用 `C#` 开发了在[DILM Suite](https://dilmsuite.com/)中找到的工具和实用程序。

本版与前一版的一个区别是，我将源代码切换到了 `C#`。

## 一个起点

我在本书中的目标是创建一个模板，任何人都可以将其作为起点，使用免费的 [`Visual Studio IDE`](https://www.visualstudio.com/vs/) 创建自定义的 `SSIS` 任务。在接下来的章节中，我将演示实现这一点的一种方法。

对于开发，我使用的是运行 `Windows Server 2019 Datacenter` 的虚拟机。该虚拟机配置了四个 `CPU`（虽然你不需要四个 `CPU`）和 `32GB` `RAM`（在本书的第一版中，虚拟机有 `8GB` `RAM`；我建议最少 `4GB` `RAM`）。安装了 `SQL Server 2019 Developer Edition` 以及 [`Visual Studio 2019 Community`](https://www.visualstudio.com/vs/)。

## 我们试图解决的问题

我们试图解决什么问题？这始终是一个重要的问题。我们同时试图解决*两个*问题：

1.  记录构建一个相对简单的自定义 `SSIS` 任务所需的步骤。
2.  构建一个有用的自定义 `SSIS` 任务。

`SSIS` 目录 (`SSIS Catalog`) 是一个出色的软件工程作品。然而，`SSIS` 目录提供的功能存在空白——特别是对于实践[数据集成生命周期管理](https://dilmsuite.com/) (`DILM`) 和/或 `DevOps` 的企业而言。其中一个空白是 `SSIS` 执行包任务 (`SSIS Execute Package Task`) 的一个限制。开发人员无法选择另一个项目中的 `SSIS` 包。

我们将在这次练习中构建的执行目录包任务 (`Execute Catalog Package Task`) 将允许我们做到这一点：启动一个驻留在 `SSIS` 目录中的 `SSIS` 包。这是 `SSIS` 目录包执行功能的一个完整、生产就绪的实现吗？天哪，不是！这是一个自定义 `SSIS` 任务的基本示例。但既然你在构建这个功能，你可以随意扩展它。我*希望*你扩展它！

## 2019 版的更新

2019 版与之前的版本有一些差异。其一，示例代码是用 `C#` 编写的，而不是 `Visual Basic`。其二，在本版中，`Azure DevOps` 取代了源代码控制。这意味着：

*   `Git` 替代了 `Team Foundation Server`。
*   使用 `Git` 影响了章节的顺序和流程；本地 `Git` 仓库（`repo`）是*首先*创建的。

那么，请继续阅读，学习使用现代版本控制工具 `Git` 在 `C#` 中创建自定义任务。
