# 导出设置

*   `导出支持对象定义`：决定是否将已上传的任何支持对象随应用程序一起导出。完整描述请参见本章后面的“支持对象”部分。
*   `导出公共交互式报表`：决定由最终用户保存并标记为公共的报表定义是否作为应用程序的一部分导出。
*   `导出私有交互式报表`：决定由最终用户保存并标记为私有的报表定义是否作为应用程序的一部分导出。
*   `导出交互式报表订阅`：决定交互式报表的用户订阅信息是否作为应用程序的一部分导出。
*   `导出开发者注释`：决定开发者针对 APEX 组件输入的任何注释是否作为应用程序的一部分导出。
*   `导出翻译`：决定翻译映射信息是否作为应用程序的一部分导出。无论此处选择何种设置，翻译文本消息和动态翻译始终包含在应用程序导出中。
*   `使用原始 ID 导出`：决定导出文件应包含应用程序组件当前的 ID，还是包含此应用程序上次导入时的 ID。

选择适当的设置后，单击 `导出` 按钮以生成应用程序导出。系统会提示您将其保存到本地计算机。导出文件名由字母 `f` 后跟应用程序 ID 组成，扩展名为 `.sql`。例如，ID 为 `9239` 的应用程序命名为 `f9239.sql`。

下载的文件包含一个大型文本脚本，定义了在 APEX 中构建的应用程序的所有内容。除了应用程序页面外，还包括共享组件，如身份验证方案、授权方案、应用程序中的主题、UI 设置、报表等。此脚本可以导入到同一工作区、不同的工作区，甚至不同的服务器中。
