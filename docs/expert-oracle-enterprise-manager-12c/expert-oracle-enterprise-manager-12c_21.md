# 配置报告

在撰写本文时，`Enterprise Manager` 处于一个过渡阶段，正从之前版本中可用的技术 `Information Publisher` 转向 `Business Intelligence Publisher` 报告，后者提供了大大增强的格式、调度和分发选项。在 `12c Release 2` 中，您有两个不同的选项，第一个是链接到 `Information Publisher` 报告。（这是在 `12c Release 2` 之前唯一的选项），但已被 `BI Publisher Reports` 取代。`Oracle` 为 `Enterprise Manager` 提供了 `BI Publisher` 的嵌入式许可证，这显然是前进的方向。然而，目前存在一些限制，我们希望在未来的版本中看到放宽。



目前，仅支持 `BI Publisher 11.1.0.6`（是的，版本号有四位小数），并且仅限于将其加载到与 Cloud Control 相同的 `WebLogic` 域中。出于所有实际目的，这意味着您要么需要扩展域以包含一个新的 `BI Publisher` 服务器，要么将 `BI Publisher` 安装到现有的 `OMS` 服务器上。无论哪种方式，在复杂性和管理方面都是一项重大投入——并且确实会带来额外的 CPU、内存和磁盘需求。希望在未来，对同一域的依赖可以放宽，`BI Publisher` 模板可以安装到现有的企业 `BI Publisher` 环境或独立的 `BI Publisher` 实例中。最后，Oracle 目前不保证您开发的 `BI Publisher` 报表的向后兼容性。因此，本书不涵盖 `BI Publisher` 报表，因为我们认为这些限制意味着在撰写本书时采用 `BI Publisher` 是不明智的。

## Services 菜单

Services 是此组中的最后一个菜单选项，也是 Enterprise Manager 服务级别管理能力的基础。一个*服务*被简单地定义为向一个或多个用户提供有用的功能。这与广泛使用的 ITIL 定义非常相似，后者侧重于向客户交付有用的成果。如前所述，系统可以交付服务，但您也可以定义不基于系统的服务（也许您不拥有或不监控提供它们的 IT 基础设施）。Enterprise Manager 中可用于监控的服务类型的确切列表取决于您安装中的插件。

下一组菜单项由您已配置监控的各种受管目标类型的首页组成。例如，您可能有主机、数据库和中间件的条目。点击这些条目中的任何一个，都会将您带到受管资产的视图，按指定类型的所有目标进行过滤。每个页面都允许您查看该特定类型的现有目标，通过发现过程添加新目标，或在目标使用寿命结束时删除旧目标。

![image](img/sq.jpg) **提示** 如果您的工作角色是特定类型目标的专用管理员——例如，如果您是 Fusion Middleware 管理员——最好选择并在必要时定制一个合适的首页。

最后，在此菜单中，您有指向正在管理的应用程序的链接。这些不是预设的目标类型，而是由您从监控组件构建起来的。此功能很复杂，并且是站点和应用程序特定的。该主题在文档集 *Oracle® Enterprise Manager Cloud Control Getting Started with Oracle Fusion Middleware Management* 中有涵盖，可在此处访问：`http://docs.oracle.com/cd/E24628_01/install.121/e24215/bussapps.htm#sthref170`。

### 自定义

Cloud Control 设计为可根据单个管理员的需求进行定制，您首次登录时就会感受到这一点。登录后，您将看到一个页面，该页面为您提供机会，根据多个预定义的工作角色之一来选择主页。本书的大多数读者可能希望从两个选项中选择一个。如果您主要担任数据库管理员或开发人员，您可能会选择 `数据库` 页面。如果您从事运维或支持工作，您可能希望从 `事件` 页面开始，该页面专注于为企业中管理运维事件的员工设计。

![image](img/sq.jpg) **注意** 您在此阶段不必选择任何主页，我们建议您不要选择。相反，请审阅所有可用页面，以找到适合您角色的起点。然后，您可以在稍后更具体地定制您的主页。

由于本书的主要读者是数据库专业人员，除非另有说明，我们将以数据库特定页面为例重点介绍本节内容。但是，此处所述的内容适用于整个产品。图 4-62 展示了数据库目标的主页。
