# 自动补丁管理

为了最小化应用程序停机时间，确保所有建议的补丁都得到及时应用非常重要。打补丁可能是一项复杂且耗时的活动。`EM12c` 通过自动化补丁部署来解决这个问题，它使用 `生命周期管理包` 确保补丁被应用到您的 `Oracle Exadata 数据库一体机` 或任何其他集成系统。`生命周期管理包` 为 Oracle 数据库和基础设施组件（包括 `网格基础设施`（`Clusterware`、`ASM` 和 `Oracle RAC`））自动化部署 Oracle 补丁。

通过与 `My Oracle Support` 的集成界面，Oracle 企业管理员可以分析、推荐并下载您环境所需的补丁。然后，您可以将它们应用到测试环境，提升到暂存区，部署到生产环境，并验证它们是否正确应用。

在部署了 `Oracle 真正应用集群` 的环境中，某些补丁可以以滚动方式应用于集群中的每个节点，从而无需为了打补丁而停机。`真正应用集群` 部署过程用于在节点间以滚动方式协调关闭、打补丁和启动，以最小化停机时间。

## 总结

`EM12c` 提供了对集成系统的全面管理能力。尽管应用程序和数据库在集成系统上的运行方式与在独立系统上非常相似，但硬件和软件更紧密的集成带来了额外的管理和运维能力。Oracle 企业管理员解决了这些需求，并提供了一个单一控制台的无缝管理，该控制台可在整个 IT 生命周期中管理完整的 Oracle 技术栈。
