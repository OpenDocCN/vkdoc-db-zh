# 由 Rohan Kumar 撰写的前言

当今的商业环境正在迅速演变。跟上变化的速度要求公司进行数字化转型，其中包含更新其应用程序的计划。一些组织选择将其遗留软件和业务流程更新为云解决方案。另一些组织则选择混合方法，结合本地部署和云功能的优势。与此同时，云原生解决方案也在激增。IDC 预测未来五年内将创建超过 5 亿个新应用程序。¹

这一趋势为当今的开发者社区带来了巨大的机遇——以及一些重大挑战。开发者希望尽快将其解决方案推向市场，并与其他软件供应商区分开来。为了满足客户日益增长的需求，开发者需要一个经过验证的后端数据库解决方案，以简化编码过程。

我们喜欢将 Azure SQL Database 描述为一个“全包式”数据库。之所以这么说，是因为该平台提供了简化并加速应用程序开发所需的组件和功能。应用程序供应商需要基本功能，如出色的数据库引擎、高度响应的性能、卓越的可扩展性和高安全性。他们还需要一个支持现代应用程序构造（如地理空间、图形和 JSON）的解决方案，并以他们熟悉的 T-SQL 语言表达。该平台还必须支持多种部署选项。开发者需要自由地将其应用程序设计为在弹性池、超大规模方案、托管实例或无服务器环境中运行。诸如此类的优点促使数千名开发者选择 Azure SQL Database 作为他们的首选解决方案。

我们要向 SQL 社区中多年来分享反馈并帮助我们成就今日 Azure SQL 的许多人表示衷心的感谢。在微软，我们的团队始终专注于推进 Azure SQL Database，为开发者提供更多价值。感谢您给我们一个支持您努力和创新的机会！
