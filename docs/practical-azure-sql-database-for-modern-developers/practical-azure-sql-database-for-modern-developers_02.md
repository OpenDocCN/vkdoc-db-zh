# 由 Mark Souza 撰写的前言

如今，拥抱数字化转型和云计算是客户成功的关键。组织和技术需要发展以满足新的业务需求（以及为应对当前 COVID 危机而调整的优先事项）和新的数据生态系统，这一点至关重要。Azure SQL Database 一直是这一演变的核心技术，现代开发者需要跟上步伐。如果您是一位经验丰富的数据库软件工程师，或是一位希望在此领域提升知识和技术深度的现代开发者，那么本书正适合您。

凭借我在科技行业专注于客户和数字化转型超过 30 年的经验，我非常高兴看到这项工作取得成果。事后看来，过去 25-30 年数据库领域的巨大演变和增长并不令人惊讶。

我于 1993 年 7 月加入微软，帮助建立微软的数据库业务。当时，运行于 Windows Server（当时称为 Windows NT 3.1）上的 Microsoft SQL Server 刚刚发布。那时，我们只有 17 名 Microsoft SQL Server 软件工程师。如今，人数已超过 2300 名。2000 年，我转到产品部门，在微软工程部门内建立了首批高度技术性、面向客户的团队之一，名为 SQLCAT——SQL 客户咨询团队。该团队技术超群，直接与我们全球的顶级客户互动。我们的工程师倾听并学习自这些客户。然后，他们将这些收获转化为客户反馈、功能建议和错误报告，推动了重要的产品改进，并制定了最佳实践指南，供所有客户（无论大小）使用。2011 年，SQLCAT 团队转变为 AzureCAT，专注于帮助客户加速其上云之旅。

在过去的 25 多年里，Microsoft SQL Server 经历了显著的演变。我们见证了 SQL 平台新增强功能和能力的诞生，例如图形、地理空间、JSON、列存储压缩、内存优化表、时态表等等。其最重大的演变发生在微软将本地计算和存储服务发展到云端之时，SQL Server 成为了基于云的 Azure SQL 产品（如 Azure SQL DB、SQL MI 等等）的基础。SQL Server 的开发者体验已经真正发展起来，如今支持所有流行的语言：Python、Node.js、Go、Ruby、PHP、Java、原生 C/C++，当然还有 .NET 语言，如 C#、F# 等。

本书也非常适合新一代的数据分析开发者，特别是如果您正专注于自己的数字化转型之旅。
