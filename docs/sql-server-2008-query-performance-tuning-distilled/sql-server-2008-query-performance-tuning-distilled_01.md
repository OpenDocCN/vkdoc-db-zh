# 引言

技术总是在不断变化，而且变化越来越快。Azure SQL Database 几乎每周都会引入新功能，而自本书上一版出版以来，SQL Server 本身也经历了两次版本更新。新型数据库也层出不穷。面对所有这些变化，你眼前最迫切的问题应该是：我们是否仍然需要进行查询调优？

答案非常简短且响亮：是的。

尽管 SQL Server 和 Azure 数据平台内置了各种功能和能力，查询调优不仅依然是一项重要的技能，它实际上还成为一种为组织节省资金的方式。知道如何让查询运行得更快，从而需要更少的资源，在诸如 Azure SQL Database 这样的平台即服务产品中，这直接成为降低成本的一种途径。

然而，这不仅仅是关于金钱的问题。像 Entity Framework 这样的对象关系映射工具生成的代码可能非常出色，直到它不再如此。那时，你将不得不着手创建自定义脚本、生成数据结构和索引，以及所有传统的查询性能调优工作。

尽管技术发展迅猛且深远，但根本的需求仍然是让查询运行得更快，以更低的服务器开销完成更多任务。这正是本书发挥作用的地方。这是一份你可以利用的资源，确保你使用手中所有的工具，来保证你构建、开发和维护的数据库能够持续运行得更快。

## 本书适合谁？

如果你编写或生成 T-SQL，你将需要让它运行得更快。因此，本书适合数据分析师、开发人员、编码员、数据库设计师、数据库开发人员，以及作为公司信息最后堡垒的数据库管理员。你们所有人都会在某个时刻需要理解索引的工作原理、在哪里追踪性能指标，以及确保查询尽可能快运行的方法和机制。
