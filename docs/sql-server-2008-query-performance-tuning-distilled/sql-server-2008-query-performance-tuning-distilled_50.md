# 23. 逐行处理

常见的情况是，数据库应用程序使用游标一次处理一行。开发人员倾向于以逐行的方式思考数据处理。Oracle 甚至使用一种称为游标的机制作为高速数据访问方式。SQL Server 中的游标则不同。因为在 SQL Server 中通过游标进行数据操作会带来显著的额外开销，数据库应用程序应避免使用游标。T-SQL 和 SQL Server 的设计初衷是处理数据集，而不是一次处理一行。Jeff Moden 著名地将这种处理方式称为 RBAR（发音为 "ree-bar"），意思是“逐行痛苦地处理”。然而，如果必须使用游标，那么请使用成本最低的游标。

在本章中，我将涵盖以下主题：

*   游标的基础知识
*   游标不同特性的成本分析
*   默认结果集相对于游标的优势和缺点
*   最小化游标成本开销的建议
