# 第一版前言

对我来说，在微软工作的最大乐趣之一就是引导新产品从概念走向发布。然而，更令人欣慰的是见证新的用户社区的诞生和发展，因为没有用户的产品算什么？不过是磁盘上的比特和字节。作为 SQL Server Integration Services 团队的集团产品经理，我很荣幸见证了 SSIS 应用程序及其用户社交网络的演变。

在 Kamal Hathi 的卓越领导下，Integration Services 团队在 2005 年交付了一款产品——SQL Server Integration Services——它不仅旨在成为一个强大的应用程序本身，更希望成为一个平台，让客户和合作伙伴能够随着数据集成需求随时间的变化和增长而进行扩展和扩展。多年来（经历了该产品的多个版本），SQL Server Integration Services 已发展成为行业领先的技术。

当我们开始开发用户现在所称的 SSIS 时，任何构建数据仓库的人只有两个选择：昂贵、高度专业的提取、转换和加载 (ETL) 工具，或者繁琐、难以维护的自定义编码。通过 SSIS，我们希望突破这些传统限制：提供一个真正可扩展的工具，对初学者足够简单，但对专家而言，又具备平台的扩展性和可编程性。

我们几乎没有预料到 SQL Server 用户社区会如此热情地接纳这个工具！我们的用户群迅速增长，如同任何群体努力一样，自然的领导者涌现出来。这本杰出著作的作者们，简而言之，是 SSIS 社交网络中最杰出的贡献者之一。他们之所以成为领导者，不仅因为他们的技能，更因为他们不知疲倦地支持和承诺帮助他人。本书提炼了这些学习成果和社区焦点，汇编成一卷，可以放在你的键盘旁多年。

像 SSIS 这样的工具面临的挑战是，用户面前有太多的可能性。如果我可以选择一个预先构建的组件，我该选哪一个？如果我可以用脚本扩展功能，我应该在什么时候这样做？在加载缓慢变化维度表或处理 XML 的众多方法中，我该如何选择？

*SQL Server 2012 Integration Services 设计模式* 不仅提供了此类问题的解决方案；更有用的是，这本书将作者的广泛经验引导成*模式*。近年来，设计模式已经证明了它们对软件开发人员的价值，作为解决重复出现但仍需特定实现细节问题的灵活模板。*SSIS 设计模式* 独特地将这种方法带入了数据仓库和 ETL 的世界。

其结果是专家们的协作成果，适合初学者和高级用户。

尽管我在几年前就已经离开了 SSIS 团队，也离开了微软，但能与我非常钦佩的用户社区保持联系，对我来说是件乐事。能够向你介绍这本备受期待且有价值的书，是我的荣幸。

集成愉快！

—Donald Farmer  
QlikTech 产品管理副总裁
