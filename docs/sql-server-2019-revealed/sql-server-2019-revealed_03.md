# 引言

和我第一本书 `Pro SQL Server on Linux` 一样，您即将阅读的这些书页也经历了一些里程。我在 2019 年的出行比我一生中任何一年都多。这意味着我需要做好随时随地写作的准备。这包括在西雅图、伦敦、曼彻斯特（英国）、纳什维尔、拉斯维加斯（多次）、圣安东尼奥、奥斯汀、休斯顿、奥兰多、圣卢西亚（那是度假）、科罗拉多州的吉尼西、查尔斯顿、波士顿、迪拜、约翰内斯堡（南非）、南卡罗来纳州的格林维尔和印第安纳波利斯等城市的航班、酒店、火车和汽车旅程中，以及在我位于德克萨斯州北里奇兰山的家中的办公室里熬夜写作。

我以为在完成第一本书后，我不会再准备写另一本，但我无法抗拒讲述 SQL Server 2019 故事的机会。这本书确实代表了那句名言“爱的劳动”。我全身心地投入到学习、教学、抱怨、破坏、记录、测试和使用 SQL Server 2019 中。这本书代表了所有这些，甚至更多。

我写这本书是为那些对 SQL Server 有基本了解，但希望在一本书中全面了解 SQL Server 2019 的数据专业人员和开发人员。本书有大量的例子、图表和参考资料来指导您。我写这本书不仅是为了让您完整理解 SQL Server 2019，也是为了使其成为您可以随时查阅的参考书。

虽然每一章都是独立的，但我强烈建议您从第 1 章开始，因为它介绍了该版本的历史和背景。我还为 SQL Server 2019 的所有关键功能奠定了基础，并解释了为什么我认为它是一款引人注目的产品。从那里开始，您可以按章节顺序阅读本书，也可以跳跃式阅读。有一点是肯定的，为了从第 10 章关于大数据集群的内容中获得最大收益，您必须先阅读第 6、7、8 和 9 章。

本书主要分为以下几个部分：

*   第 1 章介绍历史和整个 SQL Server 2019 版本的概述。
*   第 2、3 和 4 章涵盖性能、安全性和可用性。仅这几章的内容就足以让您对 SQL Server 2019 感到兴奋。
*   第 5 章独立面向开发人员。
*   第 6、7 和 8 章全是关于 Linux、容器和 Kubernetes 的。
*   第 9 章通过 Polybase 向您介绍数据虚拟化。
*   第 10 章是一个大章节，讨论一个重要主题：大数据集群。
*   第 11 章通过讨论其他新功能和迁移来结束本书。

我喜欢“通过示例学习”，因此我在书中几乎为每一章都包含了许多示例（在某些情况下，我会解释如何使用已创建的示例）。您可以在本书的参考链接 [`www.apress.com/9781484254189`](http://www.apress.com/9781484254189) 或我的 GitHub 仓库 [`https://aka.ms/bobsqldemos`](https://aka.ms/bobsqldemos) ([`https://github.com/microsoft/bobsql`](https://github.com/microsoft/bobsql)) 上找到本书的所有示例。

我还建议您看看我们的团队构建的免费培训资源 [`https://aka.ms/sqlworkshops`](https://aka.ms/sqlworkshops)。这包括使用 SQL Server 2019 的免费动手实验培训！
