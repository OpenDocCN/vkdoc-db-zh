# 前言

当我开始 IT 职业生涯时，我们在物理服务器上运行应用程序和数据库。头几年，我们转向了虚拟化，这是一次伴随着诸多变革的根本性转变。当时的巨大转变如今已变得司空见惯——甚至成了常态。

如今，正在发生另一次范式转变——向容器化的迁移。我在开发者构建的新应用程序中看到了这一点，他们渴望获得跨基础设施的轻量级、可重复部署。这也正在影响数据领域，因为这些开发者希望他们容器化的应用程序能连接到同样轻量级且易于部署的数据源。

这种转变给数据专业人士带来了新的挑战——在容器化的世界中，随着公司快速采用一个或多个云，你如何确保跨这些环境的数据服务的安全、管理和监控？多年来，微软一直在并行推进两条路线：首先是将 SQL Server 引入 Linux 和容器，其次是使 Azure SQL Database 成为世界上最好的基于云的操作数据库。现在，这两条道路已通过 Azure Arc-enabled Data Services 合并，它允许你在你选择的 Kubernetes 发行版上、在你选择的基础设施中，运行 Azure 平台即服务的数据服务。

我很高兴两位专家通力合作，为您带来这本书，旨在让您尽快开始使用 Azure Arc-enabled Data Services。他们理解这一新范式带来的挑战，并将其分解为易于理解的知识块。

我第一次看到 Anthony Nocentino 演讲是在 2020 年俄亥俄州克利夫兰的 SQLSaturday 活动上——那是我在疫情前参加的最后一次线下活动。那时我正深入研究 SQL Server on Linux 和容器的世界，很想听听他对容器的看法。他的深厚知识和演讲风格让我欣喜。

我第一次看到 Ben Weissman 演讲时很遗憾是在线的，因为我们相隔一片大洋。他当时在谈论一种我从第一次听说就为之着迷的技术——SQL Server Big Data Clusters。Ben 对这项技术充满热情，讲解通俗易懂，而且风趣随和。

借此，我希望您能像我一样，享受学习和使用这项技术的乐趣！我迫不及待想听到您的故事，讲述您如何使用 Azure Arc-enabled Data Services，以及它们如何使您作为一名 IT 专业人士能做更多事情！

— Jes Schultz
Azure Arc-enabled Data Services 项目经理
