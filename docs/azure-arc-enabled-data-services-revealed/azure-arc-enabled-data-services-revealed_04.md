# 书籍结构

我们将本书分为九个独立的章节，它们将循序渐进地为你描绘出 Azure Arc-enabled Data Services 的全貌：

*   第 1 章，“Kubernetes 入门”：Kubernetes 是每个 Azure Arc-enabled Data Services 安装的架构支柱。本章介绍 Kubernetes，描述其在现代应用程序部署中的角色、提供的优势及其架构。
*   第 2 章，“Azure Arc-enabled Data Services”：本章将向你介绍 Azure Arc-enabled Data Services！我们将介绍核心的 Azure Arc-enabled 资源，包括服务器、Kubernetes、SQL Server 和 Data Services。然后我们将深入探讨 Azure Arc-enabled Data Services 是什么、其架构、工作负载如何部署和管理，并讨论关键的部署注意事项，如计算和存储容量规划。
*   第 3 章，“为部署做准备”：在本章中，我们将引导你完成在开始部署你自己的 Azure Arc-enabled Data Services 之前需要处理的必要步骤。
*   第 4 章，“安装 Kubernetes”：在本章中，我们将指导你完成 Kubernetes 集群的设置过程，这是部署 Azure Arc-enabled Data Services 所必需的。
*   第 5 章，“在间接模式下部署数据控制器”：接下来，我们将指导你完成所需的操作——无论是使用图形用户界面还是命令行——以在你的 Kubernetes 集群中部署一个不持续连接到 Azure 云的 Azure Arc-enabled 数据控制器。
*   第 6 章，“在直接模式下部署数据控制器”：在本章中，我们将指导你如何将数据控制器部署到连接到 Azure 云的 Kubernetes 集群中——我们甚至会向你展示如何从 Azure 门户进行部署。
*   第 7 章，“部署 Azure Arc-enabled SQL 托管实例”：随着我们的数据控制器准备就绪，我们现在可以继续开始部署第一个数据库实例，以便我们可以开始使用我们的 Arc 实例。与数据控制器的部署类似，我们可以直接使用命令行和 `azure-cli`，或者在 Azure Data Studio 中使用向导来完成此操作。
*   第 8 章，“部署 Azure Arc-enabled PostgreSQL Hyperscale”：虽然第 7 章处理的是 SQL 托管实例，但本章将指导你在使用 PostgreSQL Hyperscale 时完成必要的步骤。
*   第 9 章，“监控与管理”：在这最后一章中，我们将重点介绍如何通过利用本地管理服务和 Azure 的管理功能来监控和管理你的 Azure Arc-enabled Data Services。
