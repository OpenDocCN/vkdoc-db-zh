# 引言

当我们最初讨论撰写一本关于 Azure Arc-enabled Data Services 的书时，该产品尚处于早期私有预览阶段。我们都非常兴奋地看到 Azure Arc-enabled Data Services 作为一个产品即将成为的样子：这正是所有近期引入的技术——如 SQL on Linux 和 SQL on Kubernetes——汇聚的地方。这就是微软做这一切的缘由的答案。

但即使没有那个背景，我们都看到了 Azure Arc-enabled Data Services 将要提供的巨大价值。企业在云采用方面面临的一个巨大挑战是被锁定在供应商的基础设施中。锁定在某个软件中是可以接受的——老实说，例如从你选择一个数据库平台的那一刻起，这基本上就一直是情况——但基础设施则是另一回事。你可能只是无法或不愿意迁移到公有云，或者可能已经投资于其他云提供商的基础设施。通过 Azure Arc（包括 Azure Arc-enabled Data Services），我们现在能够将这些以前仅对微软 Azure 云可用的解决方案，部署到任何云、任何基础设施上！

对我们来说，显而易见的是，随着 Azure Arc-enabled Data Services 正式发布，我们必须更新这本书，并添加自本书第一版以来加入该产品的所有新功能。

本书将向您介绍 Azure Arc-enabled Data Services 及其提供的强大功能。您将学习如何从 Azure 提供的集中式管理、补丁和更新的自动滚动推出等功能中受益。

本书是任何为其数据资产寻找混合或多供应商云策略的人的完美选择。我们将逐步引导您了解将诸如 Azure SQL Managed Instance 或 PostgreSQL Hyperscale 等服务部署到 Azure 之外的可能性和要求，使它们能够为那些无法迁移到云或不想完全使用微软云的公司所用。本书所描述的技术对于那些被要求将敏感服务（如医疗数据库）远离公有云，但仍希望受益于 Azure 云及其所支持的集中式管理的人员尤其有用。
