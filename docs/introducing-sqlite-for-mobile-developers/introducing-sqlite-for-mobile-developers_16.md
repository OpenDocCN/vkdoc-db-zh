# 第 2 章

## 理解 SQLite 是什么

第 1 章概述了关系数据库和 SQL，这些内容适用于大多数现代数据库，无论它们处于何种环境（Oracle、MySQL、Microsoft SQL Server 或其他）。这种通用性很重要，因为 SQLite 是其中的一部分。然而，在本章中，重点将只放在 SQLite 本身以及你可能在其他数据库环境中找不到的 SQLite 特性上。尽管第 1 章的许多内容对于长期的数据库用户来说可能很熟悉，但本章的大部分内容可能并非如此，因为即使不是 SQLite 独有，本章描述的许多特性在其他数据库环境中并不存在。（平心而论，其他数据库环境的许多特性在 SQLite 中也找不到。）

### 正确看待数据库

关系数据库通常存在于某种容器对象内部。通常，这个容器是一个 `数据库管理系统` (`DBMS`)。（你有时可能会看到 `关系数据库管理系统` (`RDBMS`) 的提法。在当今世界的大多数情况下，大多数 `DBMS` 都是 `RDBMS`。）

像 Oracle 这样的数据库管理系统提供的功能超出了数据库本身，例如用户界面或开发者界面；诊断、调试和维护工具；甚至复杂的数据显示功能。

在某些情况下，数据库是某种语言、面向对象的类或框架的一部分，而这个框架本身可能是一个 `DBMS`，也可能不是。例如，`PHP` 就有一个数据库类，就像 `Android SDK` 也有一个一样（是的，一个是语言，一个是操作系统，但它们都可以提供面向对象的数据库类。）。你将在第 6 章和第 7 章中找到关于这些类的更多信息。

另一方面，`Cocoa` 和 `Cocoa Touch` 为 `iOS` 和 `Mac` 提供了一个 `Core Data` 框架。我将在后面的第 8 章和第 9 章中描述 `Core Data`。

在本书的大部分内容中，重点是 SQLite 本身，但从第 6 章开始，你将看到当它位于 `DBMS`、类或框架内部时如何访问它。

## 定义 SQLite

`SQLite` 是一个用 C 语言编写的软件库。它由 D. Richard Hipp 于 2000 年开发，最初是美国海军合同的一部分，由通用动力公司实施。如今，它由大约 184,000 行代码组成。
