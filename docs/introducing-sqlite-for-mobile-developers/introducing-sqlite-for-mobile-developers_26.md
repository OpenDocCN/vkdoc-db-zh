# 使用 PHP 操作 SQLite

SQLite 以及 SQL 本身的基础很简单。这是一个重复的流程：为表中的每个数据元素创建列来定义表。然后为每一行在每个列中添加值。你可以使用基本的 `SELECT` 语句从单个表中检索数据。通过使用 `WHERE` 子句将一个表中的值与另一个表（或在自连接的情况下是同一个表）中的相应值进行匹配，来连接两个表。

SQL 及其所有实现（包括 SQLite）的复杂性来自于这些基本构建块的组合。此外，我们经常处理非常大的表（数百万甚至数十亿行）也是增加复杂性的因素，但基础是简单的。

> **注意** SQLite 的单个数据库限制是 140TB，这个限制往往在行数成为问题之前就达到了。sqlite.org 有一个关于限制的页面：[www.sqlite.org/limits.html](http://www.sqlite.org/limits.html)。它指出这些限制在很大程度上未经测试，部分原因是 SQLite 开发人员无法访问能够管理如此大数据量的硬件。如果你真的考虑管理如此大量的数据，你可能需要对硬件和软件解决方案进行认真的分析，但请记住，无论你最终选择什么解决方案，SQLite 都可以成为一个完美的原型设计工具。
