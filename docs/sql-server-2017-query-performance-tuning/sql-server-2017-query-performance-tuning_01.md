# 引言

技术一直在变化，而且越来越快。Azure SQL Database 中几乎每周都会引入新功能，而自本书上一版出版以来，SQL Server 本身也已经历了两个版本的发布。新型数据库层出不穷。面对所有这些变化，你眼前的问题应该是：我们是否仍然需要做查询调优？

答案是非常简短而响亮的：是的。

尽管 SQL Server 和 Azure 数据平台内置了丰富的功能与能力，但查询调优不仅仍然是一项重要技能，它实际上已成为一种为组织节省资金的方式。知道如何让查询运行得更快，从而需要更少的资源，在诸如 Azure SQL Database 这样的平台即服务 (PaaS) 中，这直接成为降低成本的方法。

然而，这不仅仅是关乎金钱。由诸如 Entity Framework 这样的对象关系映射工具生成的代码可能非常出色，直到它出现问题。那时，你就需要着手创建自定义脚本、生成数据结构和索引，以及进行所有传统的查询性能调优工作。

尽管技术已经飞速发展，但我们仍然有一个根本需求：让查询运行得更快，并以服务器上更少的开销做更多的事。这正是本书的意义所在。这是一份资源，你可以用它来确保你运用手中所有的工具，使你构建、开发和维护的数据库能够持续运行得更快。

## 本书适合谁阅读？

如果你编写或生成 T-SQL，你就需要让它运行得更快。因此，本书适合数据分析师、开发人员、编码人员、数据库设计人员、数据库开发人员，以及保护公司信息的最后一道堡垒——数据库管理员。你们都会在某个时刻需要了解索引的工作原理、在哪里追踪性能指标，以及确保查询尽可能快运行的方法和机制。
