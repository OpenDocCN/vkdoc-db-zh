# 总结

在本章中，就管理 Docker 和容器而言，我们涵盖了很多内容。但这还不是全部。这足以让您从 SQL Server DBA 的角度开始管理容器和 Docker 主机。随着您更多地使用 Docker，您将更深入地接触 Linux 操作系统。我相信您从`第 3 章`开始已经学到了很多关于 Linux 的知识。而我们只是触及了表面。我可以保证，当您读完本书时，您将能够处理 Linux 文件系统、配置守护进程、管理系统资源等等。当您在 Docker 容器中运行更多 SQL Server 时，您别无选择，只能精通 Linux。

在“在主机和容器之间共享文件”一节中，我介绍了使用卷（volumes）使 Docker 主机上的目录在容器内部可用。下一章将更详细地介绍这一点，以及如何为有状态的应用程序（特别是 SQL Server）管理和持久化容器内的数据。此外，如果无法在其中持久化数据，关系数据库又有何用？我们将确保这种情况不会发生在您容器内的 SQL Server 数据库中。
