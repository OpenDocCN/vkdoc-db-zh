# 总结

当我为本书准备大纲时，本章甚至不在初稿中。直到我想起处理任何与云相关的网络栈是多么具有挑战性。我清楚地记得，我曾浪费数小时在 AWS 和 Microsoft Azure 上部署 SQL Server 时排查问题，结果发现问题与 SQL Server 无关——是网络问题。无论你使用的是公有云还是私有云，理解网络栈的基础在今天都尤为重要，尤其是当你面对所谓的网络即服务提供的所有抽象层时。因此，我包含了单主机部署的 Docker 网络基础知识。本章介绍了 Docker 如何利用 Linux 桥接来实现其网络功能，容器如何通过端口映射暴露给外部网络，以及即使 Microsoft 文档明确说明不可能，如何通过容器在单个 Linux 机器上运行多个 SQL Server on Linux 实例。事实是，TCP/IP 和网络基础知识保持不变，只是其实现方式发生了变化。所以，下次当你在 Docker、VMWare、Microsoft Azure 或 AWS 等云平台上遇到网络问题时，回归基础。你会很高兴你学习了这些基础知识。
