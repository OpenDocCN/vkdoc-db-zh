# 引言

在我担任 Oracle 数据库管理员的二十多年里，我曾无数次担任导师或被指导者。本书汇集了我所学到的对我有效的方法，以及多年来别人给我的建议和我给予他人的建议。在标准的导师/被指导者关系中，双方通常身处同一地点，通过面对面接触携手合作。当今世界常常要求导师/被指导者关系更为疏远。我们不断地向他人学习如何提升我们的技艺，通常是从那些没有正式师徒关系的陌生人那里。例如，我们可能通过在讨论论坛中给出建议的人获得指导；我们可能通过阅读备受推崇的 Oracle 专业人士的博客获得指导；我们甚至可能仅仅是通过阅读某人花时间写在书中的建议而获得指导，正如您今天即将开始的那样。感谢您的阅读！

本书面向那些刚开始其 Oracle 数据库管理员职业生涯的人。即使是拥有几年经验的人，也能在这里找到有用的信息。如果您想晋升到该领域的顶级水平，本书中的信息将助您一臂之力。我知道许多数据库管理员都希望他们当年有这本书来启动他们的职业生涯。非 Oracle 数据库管理员也会发现本书很有用。除了 Oracle 特有的信息外，本书中的其他建议可用于提升您的职业生涯。通过遵循我在此给出的建议，我能够快速轻松地学会管理 SQL Server 和其他数据库平台。

*Oracle DBA 导师*不仅仅教您如何管理 Oracle 数据库。我本可以轻松地写一本只说“做这个，做那个，再做另一个”的书，但那样它就会和市面上所有的其他书一样。本书以及我迄今为止所见过的任何其他书籍的最大区别在于，本书教会您在阅读完成后如何扩展您的知识。当您了解到“这个、那个和另一个”不能解决您的问题后，您下一步该转向哪里？本书会帮助您弄清楚这一点。

我在本书中大量使用“我”、“你”和“我们”这些词。这种风格是有意为之，因为我希望您感觉我们仿佛身处一个真正的导师/被指导者关系中，尽管我并未亲临现场。在我写下这些文字时，我正直接与您——读者——对话。

## 书籍结构

第 1 章和第 2 章介绍了一些软技能，以帮助您在这份工作中取得成功。对我们许多人来说，这些信息可能已经知道了。但为确保万无一失，我确保您理解数据库管理员最重要的角色是成为数据守护者，并就您在整个职业生涯中如何与他人互动提出了建议。

在第 3 章到第 8 章中，我们搭建一个测试平台服务器、安装 Oracle、创建一个数据库，并实施安全措施和备份。我们需要这个系统，以便在阅读本书后续章节时能够进行更深入的探索。

第 9 章到第 12 章向您展示如何使用 Oracle 数据库。大多数书籍会告诉我们连接到 Oracle 数据库的步骤。我们将看到这一切是如何工作的。第 10 章讨论测试平台和开发测试用例的重要性。我们为自己的用途创建了一个测试平台，现在是时候开始使用它了。使用测试平台是我们帮助自己学习更多 Oracle 知识的第一步。

第 13 章到第 16 章是本书的核心内容。在这里，您将学习如何在您可能购买的所有 Oracle 书籍中找不到答案的问题时，通过不同途径获取答案。正如您将发现的，我强烈建议您阅读 Oracle 文档，因为您的大多数答案都写在那里。第 13 章帮助您解锁庞大的文档集。第 14 章探索数据字典，第 15 章解释 My Oracle Support，第 16 章探讨社交媒体的益处。

第 17 章到第 19 章帮助我们在系统搭建完成后维护数据库系统。我们在第 17 章探讨如何应用补丁。我们在第 18 章讨论如何升级数据库。第 19 章涵盖容量规划。

第 20 章和第 21 章结束本书。它们分别概述了 Oracle 架构和高级选项。这里本可以写更多内容。目的是让您了解一些高级概念，以便将来需要相关信息时，您已经有所入门。
