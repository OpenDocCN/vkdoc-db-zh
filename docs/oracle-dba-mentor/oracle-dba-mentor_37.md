# 棘手的问题

假设你已经掌握了本章目前展示的关于 `CPU`、`内存` 和 `磁盘` 的各项指标。你的经理找到你，问你的数据库能否支持将并发用户数量翻倍。这个问题听起来很具体，但实际情况远非如此简单。要知道答案，你需要确定新用户是会执行与现有用户完全相同的活动，还是会在数据库中执行全新的操作。如果是全新的操作，具体是什么？如果你能定义这些操作，能否将其转化为对这三大资源影响的分析？将这个看似简单、具体的问题转化为一项容量规划活动可能极其困难。问题在于变量太多，未知数也太多。这有点像做长期天气预报，我们都知道那些预报有多少次与实际情况不符。

回答这类问题最简单的方法是根据我们的图表进行推断，然后加上一个重要的前提条件。例如，如果我们查看 `图 19-7` 中的 `内存` 使用率，可以看到使用率远低于 `20%`，因此我们可以轻松应对服务器 `内存` 需求翻倍的情况。如果我们查看 `图 19-6` 中的 `CPU` 使用率，可以很容易地判断 `CPU` 无法处理额外用户带来的翻倍请求。因此，问题的答案是 `内存` 可以应对新增的用户，但 `CPU` 资源无法应对。如果我们需要将并发用户数翻倍，那么就需要提前增加 `CPU` 资源。

你可以向经理展示类似的图表并解释你的发现。然后，你必须为这个答案提供一个重要的前提条件，即“在其他条件不变的情况下”。如果这些额外用户与现有用户相似，那么资源利用率将如图所示。但如果这些用户不同，他们可能带来新的需求，而这些图表无法确定性地涵盖这些需求。

在大多数情况下，如果能将棘手的问题转化为一个可以应用于图表的因素，我们就能提供容量规划的答案。这种转化可以很简单，例如，将数据库的需求增加某个量，会将图表中的数值乘以相同的因子。用户翻倍意味着图表数值翻倍。数据量翻四倍意味着将数据库增长图表乘以四。让容量规划保持简单。你无法考虑到每一个细节和每一个变量，如果试图这样做，容量规划将成为一份全职工作。数据库管理员除了容量规划之外还有其他职责要履行。

这些图表也可用于预测资源何时会耗尽。例如，`图 19-7` 中的 `内存` 使用率低于 `20%`。我们知道 `100%` 是 `20%` 的五倍，因此我们的因子是五。我们可以预测，以当前的 `内存` 配置，我们可以支持五倍于现在的并发用户数。如果我们使用了总 `磁盘` 容量的 `30%`，那么如果数据增长翻倍，我们可以预测何时会耗尽 `磁盘` 空间。

到现在你可能在心里嘀咕，我把问题想得太简单了，你是对的。你可能在想我忽略了很多细节，你再次说对了。有些人不愿承认的是，所有关于如何规划系统容量的细节、那些细枝末节，在大多数情况下并不重要。我使用高层次的概要细节，而且每次都行得通。没有任何一位数据库管理员能够将一个要求——比如将并发用户数乘以十——转化为详细的分析，并指出类似“逻辑读取请求将增长 X 倍，进而物理读取请求将增长 Y 倍”这样的结论。并发用户数 `10` 倍的增加可能会使逻辑读取请求同样增长 `10` 倍，而物理读取请求数量净增长为零；或者物理读取可能翻倍，或翻三倍。正如我之前所说，没有人能准确预测每个指标将因此如何变化，因为未知数和变量太多了。

与其花费大量宝贵的时间试图获得尽可能准确的答案，不如用应用于当前指标的简单因子来回答这些棘手的问题，然后继续处理其他重要的工作。等到你的组织采购新硬件时，新硬件将具有你未曾考虑到的新规格，这会使你之前的任何计算偏离正轨。每一次硬件采购都意味着新设备比旧设备更大、更强、更快。

云计算可以使容量规划变得更容易，同时也更困难。如果云提供商不允许你获取资源使用率指标，云中的容量规划可能会更困难。如果他们允许你查看资源使用情况，其形式可能不允许你在较长时间内存储指标值。与你的云提供商合作，了解如何最好地获取高层次的资源指标以满足你的容量规划需求。云计算使容量规划变得更容易的地方，与其说是在规划阶段，不如说是在计划的实施阶段。需要两倍的 `CPU` 资源？我相信你的云提供商只需在你的环境中添加更多 `CPU`。需要更多 `内存`？这很容易解决。增加资源变得如此容易，以至于许多 IT 管理员干脆完全放弃了规划。有什么比直接停止容量规划活动更容易的呢？我们不再试图确定未来需要多少资源，而是让云提供商动动手指，就能给我们更多资源。云提供商在他们的销售宣传中经常暗示这一点。我不喜欢这种方法，因为那些额外的资源需要你的组织额外付费。我工作过的每个组织都有预算，并且需要规划开支。虽然增加 `CPU`、`内存` 和 `磁盘` 资源很容易，但为之付款可能就不那么简单了。

## 继续前行

本章介绍了容量规划。从高层次上讲，我们需要捕获 `CPU`、`内存` 和 `磁盘` 的使用率指标，并随时间绘制图表。一旦我们有了数据点，就可以轻松添加趋势线，以很好地了解未来资源利用率的走向。保持简单。有太多的变量和未知数需要考虑。通过将容量规划工作保持在非常高的层次，你可以更明智地利用你的时间。
