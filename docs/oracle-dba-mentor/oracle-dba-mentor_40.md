# 调整 SGA 组件的大小

过去，Oracle 数据库管理员需要指定不同的初始化参数来设置 SGA（系统全局区）各组件的大小，包括共享池（Shared Pool）、缓冲区高速缓存（Buffer Cache）等。如今的 Oracle DBA 使用 `SGA_TARGET` 初始化参数来定义 SGA 的总大小，并让 Oracle 实例在符合该参数设置的前提下，自行决定各独立组件的最佳大小。使用此参数有两大好处。首先，调整 SGA 大小变得更容易，因为只需准确设置一个参数。其次，Oracle 实例可以根据工作负载的变化动态调整各个组件。如果缓冲区高速缓存需要更多空间，可以从共享池中借用——前提是共享池有充足的可用空间。

与为 PGA（程序全局区）确定合适的大小类似，Oracle 提供了 `V$SGA_TARGET_ADVICE` 视图。在清单 20-5 中，大小因子等于 1 的行就是我们当前的 SGA 设置。

```sql
SQL>  select sga_size_factor,estd_db_time,estd_physical_reads
  2  from v$sga_target_advice;

SGA_SIZE_FACTOR ESTD_DB_TIME ESTD_PHYSICAL_READS
--------------- ------------ -------------------
            .25        53470             4175955
             .5       686730            42630654
            .75        77000            29370744
              1        60692             4175955
           1.25        59958             3005852
            1.5        59703             3005852
           1.75        59581             3005852
              2        59521             3005852

清单 20-5
SGA 目标建议
```

在上面的输出中，我们当前的 SGA 估计的数据库时间值为 `60,692`，估计的物理读次数为 `4,175,955`。如果我们将 SGA 增加 25%（因子=`1.25`），估计的数据库时间和物理读次数都会略有下降。请注意，如果我们将 SGA 增加 50%（因子=`1.5`），估计的物理读次数并没有变化。这个视图告诉我们，将 SGA 大小翻倍所带来的性能提升，并不会比适度增加 25% 好很多。

Oracle 还包含了另一个参数 `MEMORY_TARGET`，用于同时设置 SGA 和 PGA 的大小。这个单一参数使得数据库管理员为支持 Oracle 实例而调整内存结构变得更加容易。但是，如果您的数据库服务器已为共享内存分配启用了大页（Huge Pages），则无法使用 `MEMORY_TARGET`。如果您在 Linux 上运行 Oracle，那么您很可能需要使用大页，这意味着您需要为 `SGA_TARGET` 和 `PGA_AGGREGATE_TARGET` 初始化参数确定合适的值。
