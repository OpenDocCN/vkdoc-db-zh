# 会议和用户组

Oracle 专业人士遍布世界各地生活和工作。我们在本章讨论的所有社交媒体平台上互动，但没有什么能比得上面对面的交流。这就是会议和用户组发挥作用的地方。

会议能吸引大量人群。2014 年，Oracle 一年一度的`OpenWorld`大会吸引了超过 6 万名与会者。这真是很多人都在谈论 Oracle！`OpenWorld`是获取关于 Oracle 最新、最重要消息的绝佳场所。你可以在演示区与 Oracle 员工进行一对一的交流。Oracle 员工会很乐意带你演示特定产品，你可以随时提问。你可以参加许多会议环节，向该领域的专家讲者学习。你还可以与在社交媒体上互动的人见面。当你在某个论坛的讨论帖中与某人互动时，你们的对话通常局限于帖子的主题。当你亲自见到他们时，你们的对话可能会转向其他方向，与最初的话题有所偏离。这是促进讨论一些在常规场合下可能永远不会谈论的话题的好方法。如果你无法亲自参加会议，可以看看他们是否提供通过主题演讲和分组会议的直播在线参与的功能。

独立 Oracle 用户组(`IOUG`)也举办一年一度的会议，名为`COLLABORATE`。你会在`IOUG`的`COLLABORATE`大会上发现许多与`OpenWorld`相同的供应商和数据库专家。这两个会议的主要区别在于，`OpenWorld`的内容由 Oracle 公司定义，而`IOUG`的内容则由`IOUG`成员定义。

许多地区都有自己的、专注于 Oracle 的独立用户组。要找到离你近的用户组，可以在谷歌上搜索。例如，如果你住在达拉斯，搜索“达拉斯 Oracle 用户组”，你会看到`DOUG`的链接。许多大城市都有自己的用户组。加入这些小组是与你所在地区人士建立联系的好方法。你无需长途跋涉就能参加他们的活动。大多数地区用户组都有为期一或两天的小型会议供你参加。这些会议的参与者通常来自当地。一些地区用户组举办规模更大的会议，例如丹佛地区的落基山 Oracle 用户组(`RMOUG`)的“培训日”会议，吸引了来自全国各地的与会者。用户组会议不仅限于美国。世界许多地区都有 Oracle 用户组和会议。

花点时间找一个你所在地区的 Oracle 用户组。成为会员，并开始与你的 Oracle 同行互动。你不仅能学到更多关于 Oracle 的知识，还能建立一个宝贵的人际资源网络，在你的职业生涯中可以依靠他们。这个网络可以帮助你解决问题，甚至找到工作。

随着你职业生涯的发展，如果你一直遵循本章的建议，你应该开始在讨论论坛上回答问题并撰写博客文章。随着时间的推移，你的 Oracle 知识会增长，你将准备好分享更多你所学到的东西。下一步就是在 Oracle 会议上发表演讲。会议总是在寻找演讲者。他们希望为与会者提供大量信息。简单来说，他们需要有人来提供这些信息。在会议之前，组织者会征集*摘要*。这些是你可能有兴趣在会议上发表的演讲的简短描述。通常，摘要是一段话，解释你想讨论什么以及它对与会者有何价值。请时刻留意会议摘要征集通知。当你准备好在会议上发言时，开始填写摘要征集表，看看你是否被选中。从在当地用户组会议上发表演讲开始，然后逐步提升。在会议上发表演讲与写博客或回答问题有同样的好处。如果你要向某人解释某事，你会比不解释时学得更好。另一个很大的好处是，许多组织者会因为你发表演讲而免费让你参加其余会议。大型会议还会为演讲者提供免费礼物。你还会开始因你的技能而被社区中的其他人所认可。

## 继续前进

在过去的几章里，我们讨论了如何了解更多关于 Oracle 的知识，甚至解决问题。这始于阅读 Oracle 文档。文档提供了海量的信息。使用 Oracle 文档最难的部分是试图找到你想要的信息，因为它篇幅巨大。你使用 Oracle 文档越多，就越容易浏览。

然后我们讨论了 Oracle 数据字典。数据字典帮助你了解特定数据库的详细信息。数据字典是数据库内容的文档记录。

如果文档或数据字典无法帮助你，下一个通常求助的地方是`My Oracle Support`。你可以搜索其广泛的知识库以寻找答案。你可以提交服务请求并从支持分析师那里获得帮助。

如果你已经穷尽了所有这些来源仍无法解决问题，那么接下来可以转向社交媒体。正如我们在本章所涵盖的，有许多社交媒体渠道可供你使用。
