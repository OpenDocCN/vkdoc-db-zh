# SQLT 入门指南

![image](img/frontdot.jpg)

理解 `SQLT` 将显著提升您解决调优问题的能力，并为您提供一个针对新代码和旧代码的有效检查清单。

调优问题是最复杂的技术难题之一。`SQLTXPLAIN` 是一个出色的工具，能帮助您解决这些问题。准备好被它折服吧。

欢迎来到使用 `SQLTXPLAIN`（通常简称为 `SQLT`）进行快速 Oracle SQL 调优的世界。从未听说过 `SQLT`？您并不孤单。在我加入甲骨文之前，作为一名从业多年的 DBA，我也从未听说过它。这就是我撰写本书的原因。`SQLT` 是一个极好的工具，因为它能帮助您快速诊断调优问题。我指的是什么？我的意思是，最多在半天时间内，您就能从一条运行缓慢的 SQL 开始，理解其为何运行异常，并最终知道如何修复它。

`SQLT` 会直接修复您的 SQL 吗？不会。修复 SQL 需要更长时间。有些表非常大，收集统计信息可能就需要数天。搭建测试环境或将修复方案部署到生产环境也可能耗时良久。关键点在于，使用 `SQLT` 工作半天，就能为您提供一份解释。您将知道 SQL 为何缓慢，或者能够解释它为何无法再快。

您需要了解 `SQLT`，因为它会让您的工作更轻松。但让我先稍微回退一下，详细介绍一下 `SQLT` 是什么、它是如何诞生的、您可能为何没听说过它，以及您为何应该将其用于 Oracle SQL 调优。

## 什么是 SQLT？

`SQLT` 是一套程序包和脚本，用于生成 `HTML` 格式的报告、一些 SQL 脚本以及一些文本文件。所有信息被打包在一个 zip 文件中，通常会发送给 Oracle 支持团队，但您也可以自己查看这些文件。`SQLT` 中包含十几个程序包和过程（称为“方法”）。这些程序包和过程会根据您的情况收集不同的信息。我们稍后将讨论适用于多种情况的程序包。

## SQLT 的由来？

俗话说，需要是发明之母，`SQLT` 的诞生正是如此。Oracle 支持工程师每天处理大量的调优问题；问题是，旧有的线性分析方法实在太慢了。您需要快速把握全局，以便聚焦细节并告知客户问题所在。因此，当时的支持工程师 Carlos Sierra（现为 Oracle 专家中心成员——Oracle 内部的一个专家团队）创建了 `SQLT`。这些例程在多次客户现场服务中不断演进，最终能够快速有效地收集*所有*必需的信息。随后，他提供了易于使用的例程来报告这些问题。

`SQLT` 的天才创作者 Carlos Sierra，如今大部分时间都用于改进 `SQLT` 代码，并使 `SQLT` 代码适配新版数据库管理系统。他还通过 `SQLT` 这一媒介，协助 Oracle 性能调优工程师进行 SQL 调优。

## 你为何没听说过 SQLT？

如果它如此有用，为什么您没听说过 `SQLT`？ Oracle 曾尝试向 DBA 社区宣传 `SQLT`，但我仍然会接到支持电话，并与从未听说过 `SQLT` 的 DBA 交流——或者即使听说过，也从未使用过。这个神奇的工具对支持客户是免费的，因此没有成本。 DBA 需要经常查看有问题的 SQL，而 `SQLT` 无疑是最快的解决问题的方法。学习曲线可能较陡，但远不及其他替代方案那样高：解析原始的 `10046` 跟踪文件或 `10053` 跟踪文件。在统计信息表中大海捞针般查找问题？猜测可能的修复方法并尝试？不了，谢谢。`SQLT` 就像一枚巡航导弹，穿越世界直击目标。

也许 DBA 太忙，没时间学习一个 Oracle 发行说明中都未提及的工具。它不在文档集中，也不是产品套装的官方组成部分。它只是一个由才华横溢的支持工程师编写的工具，却碰巧比市场上任何其他工具都好。让我重申一遍：它是免费的。

也有可能，一些 DBA 过于专注于调优中晦涩的细节，而忘记了修复 SQL 的现实世界。当你可以谈论那些你永远遇不到的情况的深奥隐藏参数时，为何要谈论一个易于使用的程序包呢？`SQLT` 是一个非常实用的工具。

无论原因为何，如果您以前从未使用过 `SQLT`，那么我在本书中的使命就是让您尽可能快、尽可能轻松地上手并运行。我向您保证，安装和使用 `SQLT` 很容易。只需掌握几个简单的概念，30 分钟内您就可以开始使用。

## 开始使用 SQLT

开始使用 `SQLT` 再简单不过了。我将这个过程分解为三个简单的步骤。

1.  下载 `SQLT`
2.  安装 `SQLT`
3.  运行您的第一份 `SQLT` 报告

`SQLT` 可在许多不同的平台上运行。我的许多示例将基于 Microsoft Windows，但 Linux 或 Unix 同样易于使用，并且 `SQLT` 的使用在各平台之间几乎没有差异。如果有，我会在文中注明。

### 如何获取 SQLT 副本？

如何下载 `SQLT`？这简单又轻松。我刚刚还为自己计时了一下。花了两分钟。以下是在目标机器上准备好 `SQLT` 程序包的步骤：

1.  找一个网页浏览器并登录 `My Oracle Support` (`http://support.oracle.com`)
2.  进入知识库部分，在搜索框中键入“`SQLT`”。标题为“`SQLT` (`SQLTXPLAIN`) – 帮助诊断性能不佳的 SQL 语句的工具 [ID 215187.1]”的 `Note 215187.1` 应该位于列表顶部。
3.  滚动到该说明的底部，选择适合您环境的 `SQLT` 版本。目前有适用于 9i 到 11g 的版本。
4.  下载 zip 文件（我下载的版本是 2MB）。
5.  解压 zip 文件。

您现在拥有了 `SQLT` 程序，可以安装到任何合适的数据库上。如果需要，您可以将 zip 文件下载到 PC，然后复制到服务器。
