# 附录 B

## CBO 参数 (11.2.0.1)

SQLT 内部包含了一个完整的、影响查询性能的参数列表。它将此列表用于 XPLORE 方法。有些参数是隐藏的（仅在各种地方有零星文档记录），而有些则是非隐藏的（在 Oracle 在线文档中有记载：数据库参考手册 `http://docs.oracle.com/cd/B28359_01/server.111/b28320/initparams.htm#i102439`）。你可以通过从 `SQLTXPLAIN` 发出以下查询来获取此列表：

```sql
SQL> select name, description from sqlt$_v$parameter_cbo order by name;
```

本章后面“参数完整列表”部分中包含的参数列表，来源于我先前使用的一个碰巧为 11.2.0.1 版本的数据库。我本可以在此处放置来自先前或后续版本的不同列表，但这个列表的目的并非详尽或权威。它在此处是作为部分参考，并指出你可以通过 SQLT 查询获取最新的参数集。然而，这个列表是一个很好的起点，其中许多参数在 12c 中仍在使用。但是，你不应将此列表视为可能改善你 SQL 的参数清单：默认的参数设置有其存在理由：它们提供了最佳的总体结果。但如果你的 XPLORE 报告突出显示了某些可能产生影响的参数，此列表可以为你提供线索，结合描述，帮助你理解查询中正在发生的事情。请记住，仅仅知道隐藏参数的名称也是有用的，因为它给了你机会去 My Oracle Support 上搜索更多线索。

处理隐藏参数（它们被隐藏是有原因的）可能是一件棘手的事情。本附录中此列表的目的是告知，而非提供实验的模板。我将给出一些指导方针、注意事项以及对某些重要隐藏参数的更详细描述，但一般规则是：联系支持人员。这些参数的名称和效果可能随着任何版本发布而改变。Oracle 支持人员将拥有最新的参数集及其效果。

## 处理隐藏参数的一般方法

在一般情况下，仅在极少数情况下或在特殊环境中才需要使用隐藏参数。例如，建议 Exadata 安装使用少量的隐藏参数。除了这些特殊情况外，你有时会被 Oracle 支持人员要求设置一些隐藏参数。当支持人员建议使用隐藏参数时，通常是为了确认某个影响性能（错误的执行计划）或导致结果错误、甚至引发错误（如 `ORA-00600` 或 `ORA-07445`）的缺陷。这些参数的设置通常遵循以下步骤：

1.  发生特定错误（错误结果、糟糕的计划、`ORA-00600`、`ORA-07445`）。
2.  错误结果的特征、糟糕计划的步骤或 `ORA-00600` 的跟踪文件、或 `ORA-07445` 的跟踪文件暗示了应采取的行动方案。
3.  通常，建议的缺陷会有一个变通方法，其中包括设置某些参数（有时是隐藏参数），以减轻问题（避免错误）、将执行计划更改回正确的计划或确保检索到正确的数据。
4.  如果建议的更改达到了预期效果且缺陷得到确认，通过此技术及其他技术，你接下来可以选择在系统级别或（如果可能）会话级别设置这些隐藏参数，作为短期的变通方法，或者在等待缺陷修复期间容忍该错误。这将取决于许多因素，包括你的特殊情况以及参数对你的系统产生的普遍影响。

这就是为什么我针对所有这些参数都明确说明，你应该与 Oracle 支持人员合作。在不了解全部影响的情况下更改隐藏参数可能是灾难性的（让人联想到数据库损坏）。以下的描述和列表将使你能够更明智地与 Oracle 支持人员合作。如果你有一个可牺牲的数据库，那么你可以对这些参数进行一些实验；但在未经 Oracle 支持人员检查的情况下，绝不应将它们应用于生产、开发、QA 或测试数据库。我想警告已经足够了。

## 一些隐藏参数的更详细描述

作为一般信息，我在下面列出了一些可能在特定情况下用于缓解问题的隐藏参数。任何信息都应在 Oracle 支持人员的协助下使用。设置这些参数有时会产生不可预见的影响。如果 Oracle 支持人员建议使用其中一些参数进行探索性调查，通常会被要求在会话级别执行此操作，以确定效果，并确定你是否受到特定缺陷的影响或评估缺陷修复的有效性。默认值基于 11.2 版本，这些值在未来的 Oracle 版本中可能会改变，或者实际上可能被完全移除。

`_and_pruning_enabled`

对于复合分区表，你可能受到某些缺陷的影响，如果它们涉及“AND”分区修剪，则可以通过将此参数设置为 `FALSE` 来修复。默认值为 `TRUE`。

`_bloom_filters_enabled`

布隆过滤器是一种内存高效的方式，用于确定一个特定值是否属于某个集合。这对于 Oracle 查询和大型数据集非常有效。如果你的执行计划显示了一个布隆过滤器（BF），并且你得到了错误结果或 `ORA-00600` 错误，你可能需要向 Oracle 支持人员提出服务请求，并讨论将此参数设置为 `FALSE`。默认值为 `TRUE`。

`_complex_view_merging`

在某些情况下，合并复杂视图的过程可能会导致问题。默认情况下，此参数设置为 `TRUE`（自 Oracle 8i 起），以允许视图查询转换发生。如果你遇到错误结果或 `ORA-00600` 错误，你可能需要咨询 Oracle 支持人员，检查将其设置为 `FALSE` 是否可能有所帮助。默认值为 `TRUE`。

`_disable_function_based_index`

函数索引包含一个函数（显然如此）。一个例子是：

```sql
sql> create index fbi_1 on t (upper(col1));
```

这里的函数是 `upper()`。在极少数情况下，你可能会得到错误结果，这可以通过将此值设置为非默认值 `FALSE` 来临时修复（直到你获得缺陷修复）。
