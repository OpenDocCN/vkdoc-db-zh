# 我与 SQLT 的故事：从被拒绝到广泛采用

## 与 Abel Macias 的初次碰撞

自从我加入 ST CoE 后，我便肩负着一项个人使命：我希望服务器技术支持组能了解 `SQLTXPLAIN`，并像采用 `EBS` 那样尽可能多地采用它。因此，我请求与性能支持团队的一位传奇人物兼领导者会面。就这样，我得以结识 Abel Macias。当我向 Abel 询问在 ST 性能问题上使用 `SQLTXPLAIN` 时，他的回答很生硬。我不记得他的原话了，但大意是这样的：“我们不会用它，因为它烂透了。”

Abel 以其时而直截了当（但正确）的回答而闻名。他为人非常谦逊，但观察问题过于直接，这可能让人望而生畏。最初的震惊过后，我问他为什么这么想。这为我打开了一扇门，了解到 `SQLTXPLAIN` 存在的一系列虽短但重要的缺陷。我离开时感到沮丧，但面前也有了一个新的挑战：我要把这些建设性的批评全部收集起来，转化为对 `SQLTXPLAIN` 的重大改进。经过数周的辛勤工作（以及喝了太多杯咖啡），我再次见到 Abel，并向他展示了所有这些对 `SQLTXPLAIN` 的“调整”。直到那时，他才喜欢上它，而 Abel 和我也成了朋友！如今，我们一起骑车驰骋数英里。即使我们不想谈，但在我们享受山地骑行，或者喝着一杯……或两杯啤酒时，我们经常会就 SQL 调优和 `SQLTXPLAIN` 进行简短而有意义的交谈。

## 第三次重写与`SQLT`的独立

根据 Abel 的意见，`SQLT` 变得独立于应用程序，安装在自己的模式中，并且支持 `RAC`。由于 `SQLTXPLAIN` 这个名字非常独特，它成为了安装所有工具对象的应用程序模式的名称。一切看起来都不错，但有件事一直困扰着我，那就是 `SQLTXPLAIN` 被大幅修改，并且需要越来越多的维护。是时候进行第三次全面重写了。由于代码量已经很大，我知道我需要大量不受打扰的专属时间。我决定利用我所有累积的假期，加上一些节假日；于是在 2009 年 12 月和 2010 年 1 月期间，我几乎都待在我的“男人洞穴”里开发 `SQLT` 的第三个主要版本。

## 工具的本质与社区的认可

我在 2010 年 4 月完成了所有的细节和测试。此时的 `SQLT` 已经包含了多个程序包，代码行数接近 10 万行，所有这些都是用两根手指开发的（我至今仍用两根手指打字）。有些人认为 `SQLT` 是由一个大型团队开发和维护的。如果你这么想，那很抱歉让你失望了：`SQLT` 诞生的初衷是作为支持人员的工具，为了让他们的工作更轻松，现在在某种程度上也被许多其他从事 SQL 调优这一复杂任务的人共享和使用。然而，这个工具的精神始终未变：一个帮助诊断性能不佳（或产生错误结果）的 SQL 语句的工具。

2010 年 4 月发布的第三个主要版本的 `SQLTXPLAIN`，迅速被支持部门内的服务器技术性能团队采纳。自从我加入 ST CoE 以来，我还很荣幸地开发了两个为期一周的内部 SQL 调优研讨会。在全球范围内向 700 多名 Oracle 专业人士交付了数十次这些研讨会后，我收集并整合了数百条在课堂上口头交流中获得的改进建议。

## 致谢杰出贡献者

在所有为我提供如何改进 `SQLT` 的想法的好心人和聪明人中，我想特别提到两个人的名字，他们除了 Abel 之外，还为该工具的功能集做出了卓越贡献：Mauro Pagano 和 Mark Jefferys。Mauro 凭借其持续的洞察力，启发了 `SQLTXPLAIN` 的 `XPLORE` 方法，该方法使用强力分析来缩小升级后出现性能回退的可能原因范围。Mark 凭借其深厚的数学和科学背景，向我指出了我犯的几个错误，尤其是在统计和测试用例创建方面。我由衷地认为，`SQLT` 能有今天的成就，要归功于我从众多聪明人那里不断获得的建设性反馈。有时反馈很直接，但大多数时候只是要求我在这里或那里加入一点东西，或者修复一个小错误。作为改进的例子，我想提及一些大的方面：在 `SQLT` 中增加对 `Siebel` 和 `PeopleSoft` 的感知能力，以及它执行的所有健康检查。

## 个人的感言

我认为自己是一个非常幸运的人，自 1996 年以来就有机会在像 `Oracle` 这样伟大的公司工作；我还遇到了一些非常棒的经理，他们非常支持我采取我认为正确的策略来诊断困难的 SQL 调优问题。对于我为这个 `SQLTXPLAIN` 工具所投入的个人时间，我一分钟都不后悔。起初，它是为了我自己，也来源于我自己。现在，它来源于我和其他许多聪明的头脑，基本上服务于任何希望深入进行严肃 Oracle 数据库 SQL 调优的人。

当 Stelios Charlambides 询问我关于他撰写一本以 `SQLXPLAIN` 为核心的书的意见时，我感到荣幸和高兴。许多人问我：“我们什么时候能有一本关于 `SQLTXPLAIN` 的书？”如果你也是其中之一，请尽情享受 Stelios 利用他个人时间所开发的成果，并感谢他写了这本书！我真诚地希望你能享受 `SQLTXPLAIN`，甚至享受 SQL 调优的过程。

我总是说，SQL 调优就像寿司，你要么爱它，要么恨它！我希望随着你了解得越来越多，你会像我在 20 世纪 90 年代那样爱上它。同时请记住，SQL 调优的学习永无止境。扎实的基础、好奇心、毅力和经验——这些只是有朝一日能从容进行 SQL 调优的部分要素。干杯！

—Carlos Sierra，《`SQLTXPLAIN`》作者
