`FF_SRC_PopulationDensity` 和 `FF_SRC_Apportionment` 是平面文件源，它们使用的属性与 HelloWorld 包的源组件相同，除了源文件。由于这些文件包含关于相同州的不同人口普查数据，我们希望将它们合并成一个数据集。在提取过程中处理此需求的唯一选择是使用 `合并连接` 组件。为了使用此组件，数据需要按键值（连接列）排序。SSIS 对大小写和空格敏感，因此我们必须在排序之前准备连接列。`派生列` 组件为每个数据流处理此操作。`派生列` 组件通过为通过数据流路径的每一行计算表达式来创建一个新列。新创建的列可以作为新列添加，也可以替换数据流路径中相同数据类型的现有列。



#### 数据流组件配置与操作

如图 3-25 所示，`Derived Column`组件使用字符串函数来确保来自不同平面文件的列能够匹配。要将其添加到数据流中，您可以双击 SSIS 工具箱中的对象，或者单击并将其拖放到流中。添加后，我们建议您做的第一件事就是使用规范重命名它。将其添加到流中后，必须连接数据路径到它。连接数据路径后，您可以双击它以打开编辑器。在编辑器中，我们添加了表达式`UPPER(TRIM(STATE_OR_REGION))`并使用了“添加为新列”选项。我们还将列重命名为`MatchColumn`，以体现该列的用途。

顶部窗格使您可以轻松引用派生列可用的输入以及可以执行派生的函数。“变量”树显示了该组件所属数据流作用域内可用的所有变量。由于我们没有为此示例定义任何变量，因此唯一可用的变量是所有包都包含的系统变量。“列”树允许您快速单击并将可用列拖放到表达式中。函数根据它们返回的数据类型进行分类。借助这两个参考窗格，您可以轻松地开始修改输入。

`TRIM()`函数移除所包含字符串对象的前导和尾随空格。由于 SSIS 不会像 SQL Server 那样可以设置忽略尾随空格，因此在使用 SSIS 进行连接或查找时，我们需要采取额外的预防措施。`UPPER()`函数将所包含字符串对象的大小写更改为大写。SSIS 也是区分大小写的，而 SQL Server 有可以禁用此功能的选项。通过强制列采用大写并生成一个新列来保存此大写字符串，我们实际上并没有修改原始数据。这将允许我们按原样将数据插入数据库而无需更改。由于这是一个 Unicode 字符串列，因此只填充了`Length`列；对于数值数据类型，会涉及精度和小数位数；对于字符串数据，可以指定代码页。数据将基于`STATE_OR_REGION`值关联在一起，因此我们稍后必须在管道中将此列用作键值。

`Sort`组件执行其名称所暗示的确切任务：它对列进行升序和降序排序。您应在将其添加到数据流后立即重命名它。当数据流路径从`DER_CapitalizeMatchColumn`连接到它之后，它可以对多个列进行排序，每个列按其在排序顺序中的位置标记，如图 3-26 所示。排序执行的字符串比较也有可用的选项。由于我们通过派生列中的表达式处理了大小写和空格问题，因此我们不需要担心比较标志。“输出别名”允许您重命名已排序的列。未排序的列仅被标记为传递。通过在此组件中勾选列，我们实际上可以修剪管道，但对于这个特定示例，我们希望获取所有存在的数据。底部有一个复选框将删除具有重复排序值的行。此功能将任意删除重复记录。除非管道中的所有列都包含在排序中，否则没有理由允许这种非确定性行为；那样的话，此功能将允许您有效地删除重复项。

*图 3-26. 排序转换编辑器*

图 3-27 中的`Merge Join`组件将所有数据汇集在一起。它将利用元数据自动检测哪些列已排序，并使用它们的排序顺序进行连接。我们希望执行内连接，以便只有两个平面文件中都存在的状态才会传递下去。该组件支持内连接、左连接和完全外连接。有一个“交换输入”按钮，可以通过将指向其输入的输出交换来添加右连接功能。顶部窗格中的表直观地表示了左右输入。它们源的名称作为表的名称出现。

我们只需要一个状态名称列传递下去，并且因为我们执行的是内连接，所以使用哪个输入的状态名称并不重要，只要我们只选择其中一个即可。我们勾选了管道中存在的所有列，但两个输入的`MatchColumn`以及右输入的`STATE_OR_REGION`除外。

我们强烈建议您尽可能在数据流的高处修剪您的管道。这使您可以将缓冲区空间仅用于所需数据。“输出别名”允许您在连接后重命名组件包含在其输出中的每一列。`Merge Join`组件在将组件包含在数据流中时，有一个重要的属性需要设置。属性名称是`TreatNullsAsEqual`。默认情况下它设置为`True`，但我们建议您将其设置为`False`。保留此属性的默认值会带来一些影响，主要是可能产生类似交叉连接的数据集。

*图 3-27. 合并连接转换编辑器*

`Unpivot`组件（其编辑器如图 3-28 所示）旨在将反规范化数据集转换为更规范化的数据集。它的工作方式类似于 T-SQL 中的`UNPIVOT`运算符。`STATE_OR_REGION`列包含了我们希望将其余列作为行围绕其进行逆透视的数据。为了确保组件忽略该列，我们将其设置为仅传递通过组件。其余列作为输入传递给组件。这些列将直接在数据流路径中逆透视为行。“目标列”创建一个列来存储每个列中报告的统计信息，并在将其转换为行值时将其作为列的一部分维护。“透视键值”代表将成为数据集一部分以显示统计信息所表示内容的字符串。在“透视键值列名”的文本字段中，您可以命名将存储透视键值的列。对于我们的数据集，透视键代表年份和统计信息名称的组合，因此我们将其命名为`YearAndStatisticName`。

`NOTE`：使用逆透视转换任务时，您会看到记录计数增加。该计数可以根据被逆透视列的唯一记录计数（*n*）和将被透视的列数（*m*）进行数学计算。结果记录计数将是*n* × *m*。在我们的示例中，有 51 条州的唯一记录（这包括整个美国的一行），并且我们有 66 列需要逆透视。结果行数将为 3,366。

当您选择要作为输入传递给组件的列时，`Pivot Key Value`会自动填充。它采用的默认值是列名本身。您还可以为每个透视键定义特定的列来填充；出于我们的目的，我们只需要所有统计信息都填充到一个列中。透视键列需要列名。
