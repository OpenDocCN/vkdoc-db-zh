# 第 7 章  源和目标适配器

您可以通过从 SSIS 工具箱中拖放数据流任务到 BIDS 设计器界面上来将其添加到控制流中。因为数据流任务是一个控制流任务，所以数据流代表了一个功能强大的、自包含的控制流子集。拥有数据流的目的是将数据移动和操作（ETL 的主要目的）与工作流管理分开。如图 7-2 所示，任何给定的数据流都可能相当复杂。您还可以在单个控制流中包含多个数据流任务。我们将在本章和下一章中介绍示例数据流中显示的源、目标和转换，以及更多内容。

*图 7-2. 由多个数据流组件组成的数据流*

构成数据流的组件可分为三类：将数据拉入数据流的源适配器；允许您操作、修改和引导数据的转换；以及允许您将数据推出数据流到持久化存储或其他系统进行进一步处理的目标适配器。在本章中，我们将考虑 SSIS 提供的用于将数据拉入和推出数据流的源和目标。

从技术上讲，所有数据流组件都派生自一个名为 PipelineComponent 的单一基类。此类公开了诸如 BufferManager 之类的属性，该属性提供对二维存储对象（即表格行集）的访问，该对象在您的数据流经数据流时保存数据。PipelineComponent 类还有诸如 AcquireConnection 之类的方法，该方法通过连接管理器建立到数据源的连接。在本章讨论的现成组件中，这些属性和方法已经为您完全开发，因此您无需直接处理它们。
