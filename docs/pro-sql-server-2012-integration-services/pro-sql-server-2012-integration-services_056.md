# 第 7 章：源适配器与目标适配器

`原始文件`是一种专门的二进制格式文件，它消除了与普通平面文件相关的许多转换开销。`原始文件`以二进制格式存储您的数据，使得那些 4 字节整数恰好存储在 4 个字节中。`原始文件`还存储有关列的元数据，因此原始文件中的二进制数据不会意外地被错误格式读入。`原始文件`目标适配器允许您将数据流中的数据输出到原始文件；`原始文件`源适配器则允许您将原始文件读回数据流。原始文件为序列化临时结果提供了一种高效的中间格式。

仅举一个使用原始文件的例子，我们最近遇到一种情况：必须使用 32 位`Jet 4.0`驱动程序从 Microsoft Access 数据库转储大量数据，然后使用 64 位驱动程序将数据导入`SQL Server`数据库。您不能在同一个包中混合使用 32 位和 64 位驱动程序，因此解决方案是创建两个包：一个用于从 Access 提取数据到原始文件，另一个用于将数据导入`SQL Server`。我们使用`SSIS` 32 位运行时运行 Access 提取包，并使用`SSIS` 64 位运行时运行`SQL Server`导入包。这只是其中一个例子，但还有很多其他情况，原始文件可以高效地临时存储中间结果。每当您需要临时存储部分处理过的数据时，请考虑使用原始文件。

#### XML 文件

`XML 源`适配器允许您使用带有 XML 架构文档（`XSD`）的 XML 文件作为数据流的源。`XML 源`使用您提供的`XSD`从 XML 文件中提取数据值，并将其作为列输出。如果您没有`XSD`文件，此组件将为您生成一个简单的。`XML 源`适配器对于相对简单的 XML 文件，或者从更复杂的 XML 文件中提取定义良好的数据子集非常有用。

**注意：** 我们推荐 Matt Masson、Tim Mitchell、Jessica Moss、Michelle Ufford 和 Andy Leonard 所著的*SSIS 设计模式*（Apress，2012）一书，该书专门用一整章介绍如何使用`SSIS`加载 XML 文件。

### 专用适配器

`SSIS`有两个专用目标适配器，它们提供额外的功能，例如写入内存记录集或`SQL Server Analysis Services`（`SSAS`）。这些适配器如下：

**记录集：** `SSIS`的`记录集`目标适配器允许您填充内存记录集。内存记录集最常见的用途之一是通过数据流填充它，然后使用`Foreach 循环容器`对其进行迭代。

**DataReader：** `DataReader`目标适配器以与`.NET DataReader`接口兼容的格式输出数据，该格式可以被能够读取`DataReader`的应用程序接收和使用。例如，`SQL Server Reporting Services`（`SSRS`）报表可以设置为使用`DataReader`接口接收`SSIS`包的输出。

#### Analysis Services

除了本章介绍的所有源和目标适配器外，`SSIS`还提供了用于`SQL Server Analysis Services`（`SSAS`）的目标适配器。本节概述了`SSAS`的功能，第 14 章将详细介绍`SSIS`的`Analysis Services`功能。

表 7-5 列出了`SSIS`的`Analysis Services`目标。

**表 7-5. Analysis Services 目标**

| 目标名称 | 描述 |
| :--- | :--- |
| 数据挖掘模型训练 | 将数据传递通过数据挖掘模型算法以训练数据挖掘模型 |
| 维度处理 | 加载和处理`SQL Server Analysis Services`维度 |
| 分区处理 | 加载和处理`SSAS`分区 |
