# 第 12 章 – 数据剖析与清洗

### 模糊分组

模糊分组组件尝试基于匹配条件减少数据集中的重复项数量。该组件仅接受字符串数据类型（Unicode 和代码页）来执行匹配操作。转换本身将在服务器上创建临时表和索引，以便高效地执行匹配操作。对于此示例，我们将使用与模糊查找示例相同的源组件。模糊分组转换的配置与模糊查找非常相似，因为它允许您选择`匹配类型`、`最低相似度`、`比较标志`以及其他选项。此外，模糊分组允许您通过提供`数字`选项来指定数据是否可能包含数字字符。`数字`选项使您能够对字母数字信息执行更好的匹配，其中数字数据在字符串开头或结尾具有重要影响。此选项对于街道地址信息或字母数字邮政编码非常有用。对于我们的模糊分组示例，我们将使用清单 12-4 中所示的表作为匹配结果的目标表。

**提示：** 为了获得一些性能提升，如果可能，让执行包的主机作为模糊分组转换的 `OLE DB` 连接。直接影响模糊匹配转换性能的因素是数据集大小（行数和列数）以及匹配列中词项的平均出现频率。

**清单 12-4. 模糊分组目标**

```sql
CREATE TABLE dbo.StateFactGrouping
(
    _key_in int,
    _key_out int,
    _score real,
    Name nvarchar(50),
    Capital nvarchar(50),
    Flag nvarchar(10),
    Date date,
    Fact nvarchar(500),
    Long float,
    Lat float,
    Fact_clean nvarchar(500),
    _Similarity_Fact real
);
GO
```

模糊分组转换在管道中添加了一些列来跟踪分组。一些列跟踪分组背后的统计信息，而另一些则在记录流经管道时跟踪它们：

`_key_in` 跟踪原始记录流经数据流任务时的情况。此列实际上是一个标识列。数据集中的每一行都接收其自己唯一的值。

`_key_out` 将一个分组行连接到它最相似的匹配行的 `_key_in` 值。

`_score` 显示该行与匹配行之间的相似度。

`Fact_clean` 代表匹配行的 `Fact` 列的值。

`_Similarity_Fact` 显示该行的 `Fact` 列与匹配行的 `Fact` 值之间的相似度。

清单 12-5 显示了分组如何根据关于各州的事实记录来匹配它们。



第一个查询中的连接条件展示了模糊分组添加的列如何让你将数据追溯到其原始形式。第二个查询允许你用模糊分组变换计算出的最接近结果替换原始州事实。

#### 代码清单 12-5. 模糊分组结果

```sql
SELECT sfg1.Name AS GroupedState,
       sfg1.Fact AS GroupedFact,
       sfg1._score AS GroupedScore,
       sfg2.Name AS MatchingState,
       sfg2.Fact AS MatchingFact
FROM dbo.StateFactGrouping sfg1
INNER JOIN dbo.StateFactGrouping sfg2
ON sfg1._key_out = sfg2._key_in
AND sfg1._key_in <> sfg2._key_in
;
GO

SELECT sfg.Name,
       sfg.Capital,
       sfg.Fact_clean
FROM dbo.StateFactGrouping sfg
;
GO
```

图 12-29 和 12-30 展示了代码清单 12-5 所示查询的结果。你甚至可能通过查看图 12-28 就能确定用于匹配的词元。例如，对于科罗拉多州和路易斯安那州，用于匹配的词元很可能是“唯一州”。至于亚利桑那州和蒙大拿州，“国家公园”是最可能的候选词元。这些匹配的分数相对较低，因为我们设置了 0.08 的相似度阈值。匹配到的事实通常是较短的字符串匹配到包含大量相关词元的较长字符串。

### 数据预览

SSIS 最有用的功能之一是能够查看数据在数据流任务中流动的过程。有几种选项可让你抽样数据。数据查看器让你能以各种形式查看数据流任务中两个不同组件之间的数据。行采样和百分比采样变换则允许你直接从数据集中提取样本数据。

#### 数据查看器

*数据查看器* 选项让你能在数据流通过数据流任务时快速查看数据流的快照。查看器可以在数据流任务中任意两个组件之间启用。

图 12-31 展示了如何在数据流任务上定义数据查看器。要启用数据查看器，你需要右键单击两个组件之间的连接并选择“启用数据查看器”。

*图 12-31. 启用数据查看器*

启用数据查看器后，所选连接上会出现一个小图标。该图标是一个小放大镜，表示对管道中某一时间点的深入查看。

图 12-32 展示了一个启用了数据查看器的连接。数据查看器能在开发过程中为你提供极大帮助，因为它可以让你在不将数据提交到表或对象变量的情况下查看数据。此对象有助于数据分析的一个方面是数据转换。假设你的数据转换变换因没有非常具体的错误消息而失败。数据查看器可以帮助你识别导致错误的行，让你能够采取行动修复错误数据或忽略它。数据查看器还可用于评估查找组件或合并连接变换的成功程度。

*图 12-32. 已启用的数据查看器*
