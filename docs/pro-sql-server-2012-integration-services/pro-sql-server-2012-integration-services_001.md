## 第 1 章：Integration Services 简介

*我是将一切凝聚在一起的粘合剂。*
——歌手奥蒂斯·威廉姆斯 (Otis Williams)

你的业务分析师已经完成了业务需求的收集。数据库架构师已经设计并构建了一个堪称艺术品的数据库。商业智能架构师们正在设计他们的 OLAP 多维数据集以及为其提供数据的维度数据集市。另一方面，也许你是一个全能型选手，自己设计并构建了所有这一切。无论哪种情况，唯一缺失的环节就是一个能将所有部分整合起来的工具。于是，SQL Server Integration Services (SSIS) 应运而生。

就像“诱惑乐队”（The Temptations）的联合创始人奥蒂斯·威廉姆斯一样，SSIS 是将一切凝聚在一起的粘合剂。不仅如此，SSIS 还是你的数据仓库和商业智能解决方案的循环系统。SSIS 通过移动数据——你组织的生命线——从分散的源系统，沿着清晰的路径，直接注入你系统的核心，从而为你的技术解决方案注入生命。在此过程中，SSIS 可以验证、清洗、操作、转换和丰富你的数据，以实现最大效能。

在本书中，我们将带你游览 SSIS，从构建你的第一个 SSIS 包，到无缝实现复杂的多包设计模式。本章向你介绍 SSIS 以及提取、转换和加载（ETL）流程背后的概念。让我们从头开始，简要回顾一下微软风格的 ETL 历史。

### 微软 ETL 简史

在我们深入探讨微软当前一代 ETL 处理解决方案的细节之前，了解 ETL 究竟是什么非常重要。正如我们所说，*ETL* 是 *提取 (Extract)、转换 (Transform) 和加载 (Load)* 的首字母缩写，这是对现代数据操作和移动过程非常直白的描述。

当我们谈论 ETL 时，我们具体指的是：（1）从源（如数据库或平面文件）*提取* 数据；（2）在数据前往目的地的途中对其进行*转换*，即操作和丰富数据；以及（3）将数据*加载*到其目的地，通常是数据库。

多年来，几乎在你所能想到的任何行业中，对数据处理的业务需求都变得更加复杂，与此同时，需要处理的数据量也呈指数级增长。不幸的是，一天中的小时数在同一时期保持相当恒定，这意味着你每天只有同样有限的处理窗口来传输和处理不断增长的数据量。为了响应这些在性能、灵活性和质量方面日益增长的数据处理需求，ETL 解决方案已变得越来越复杂和稳健。

因此，我们将通过考察 ETL 在 SQL Server 世界中的演变来开启我们的 SSIS 之旅。直到 SQL Server 6.5，*大容量复制程序 (bcp)* 一直是将数据加载到 SQL Server 数据库的主要工具。作为一个命令行实用程序，`bcp` 使将基本文本文件加载到数据库表中变得相当简单。

不幸的是，这种简单性的另一面是，你只能使用 `bcp` 从平面文件加载数据，并且在加载过程中无法对数据执行额外的验证或转换。使用 `bcp` 的常见数据库到数据库的 ETL 场景可能包括：将数据从数据库服务器提取到分隔文本文件，将文件导入 SQL Server 数据库，最后使用 T-SQL 对数据库中的数据进行转换。`bcp` 实用程序仍随所有版本的 SQL Server 提供，并且偶尔仍用于从平面文件进行简单的一次性数据加载。

为了响应 ETL 处理日益增长的需求，数据转换服务（Data Transformation Services，DTS）在 SQL Server 7 中首次亮相。使用 DTS，你可以从各种源获取数据，即时进行转换，并将其加载到数据库中。尽管 DTS 是比 `bcp` 复杂得多的工具，但它仍然缺乏开发企业级 ETL 解决方案所需的许多功能。
