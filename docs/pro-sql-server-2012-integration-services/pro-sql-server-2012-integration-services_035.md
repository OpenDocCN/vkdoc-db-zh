# 第 5 章 控制流基础

*图 5-44. 设置断点选项*

此对话框允许你指定“断点条件”、该条件的发生次数（将增加计数）以及在执行暂停前可容忍的计数。本节介绍 SSIS 发出的事件。我们将在第 11 章更详细地讨论它们。这些事件包括：

- `OnPreExecute` 在目标对象开始执行前立即创建一个事件。
- `OnPostExecute` 在目标对象完成执行后立即创建一个事件。
- `OnError` 在目标对象执行过程中发生错误时创建一个事件。
- `OnWarning` 在目标对象未处于错误状态但设计并非最优时创建一个事件。
- `OnInformation` 在目标对象发出有关执行的信息时创建一个事件。
- `OnTaskFailed` 在目标对象执行失败时创建一个事件。
- `OnProgress` 在目标对象执行过程中创建一个事件。
- `OnQueryCancel` 在目标对象执行过程中被停止时创建一个事件。
- `OnVariableValueChanged` 在目标对象执行期间变量值发生变化时创建一个事件。
- `OnCustomEvent` 在目标对象执行期间引发自定义事件时创建一个事件。

“设置断点”对话框中的“命中次数类型”列定义了导致目标对象暂停执行的条件。选项包括“始终”、“命中计数等于”、“命中计数大于或等于”和“命中计数倍数”。“始终”命中计数将始终暂停目标的执行，每次启用的断点条件事件被引发时都会如此。此选项会禁用“命中计数”列。“命中计数等于”选项仅在条件事件引发的次数达到定义的命中计数时才暂停目标。“命中计数大于或等于”选项在条件事件引发时都会暂停执行，从事件发生次数达到命中计数时开始。“命中计数倍数”选项将在断点条件事件引发的次数达到定义的命中计数的倍数时，导致目标暂停其执行。
