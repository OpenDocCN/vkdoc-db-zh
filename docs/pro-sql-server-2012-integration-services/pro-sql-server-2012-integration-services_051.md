# 第七章 - 源和目标适配器

我们将从“添加新源”对话框中选择“平面文件”源和`<新建>`连接管理器。然后使用平面文件连接管理器编辑器来设置平面文件的属性并选择一个源文件。如图 7-4 所示，我们选择了一个名为`zips.csv`的逗号分隔文件，更新了连接管理器名称，并在“常规”页面上勾选了“首行数据包含列名”复选框。我们将区域设置、代码页和其他选项保留为默认值。

在编辑器的“列”页面上，对于平面文件有两个选项。你可以更改行分隔符（即结束平面文件每一行的字符或字符组合），也可以更改列分隔符（即分隔文件中字段的字符或字符组合）。在本例中，我们选择了默认的`{CR}{LF}`（回车/换行）组合作为行分隔符，并为`.csv`文件选择了逗号（`,`）作为列分隔符。“列”页面还提供了源数据的预览。图 7-5 在编辑器的“列”页面上突出显示了这些选项。

> **注意：** `{CR}{LF}` 组合是 Windows 文本文件常见的行分隔符。在其他一些操作系统（如 Unix）上生成的文件仅使用 `{LF}` 字符作为行分隔符。处理由非 Windows 系统生成的平面文件时，请记住这一点。

*图 7-5. 在平面文件连接管理器编辑器的“列”页面中编辑属性*

“高级”页面是编辑器的第三个页面。在此页面上，你可以更改数据源的列特定属性。你可以为单个列更改 `ColumnDelimiter` 属性，以便一个或多个列可以使用特殊的分隔符（例如，当你有一个需要拆分为两部分的输入列时，这很有用）。你还可以更改列的 `DataType`（默认为字符串，即 `DT_STR`）和输出列宽度（字符串的默认值为 50）。还有 `DataPrecision` 和 `DataScale` 附加属性，它们分别决定数值列（`DT_NUMERIC`）中的总位数和小数点后的位数。图 7-6 展示了平面文件连接管理器编辑器的“高级”页面。

### 精度和小数位数

平面文件连接管理器编辑器“高级”页面中的 `DataPrecision` 和 `DataScale` 属性仅适用于数值列（`DT_NUMERIC`）和小数列（`DT_DECIMAL`）数据类型。这些列的精度是可以通过该列处理的数字所包含的总位数。例如，数字 123.45 的精度是 5。列的小数位数是小数点后允许的位数，因此对于 123.45，小数位数是 2。

对于数值列，精度可以设置为 0 到 38 之间的数字，小数位数也可以设置为 0 到 38 之间的值。小数数据类型的精度始终为 29（无法更改），小数位数可以设置为 0 到 28。

### 编辑数据流组件

将数据流组件添加到设计器界面后，你可以随时通过以下方式对其进行编辑：(1) 双击它，(2) 右键单击并选择“编辑”选项，或 (3) 右键单击并选择“显示高级编辑器”选项。大多数组件的标准编辑器通常是一个简化的用户界面，只提供对最常修改属性的访问。而高级编辑器则提供对数据流组件所有可用属性的访问，例如直接访问输出列和错误输出列属性。我们将在本章后面更详细地介绍高级编辑器的功能。

### 目标助手

除了新的源助手，SQL Server 12 SSIS 还包含一个新的目标助手。

源助手指导你将源适配器添加到数据流，而目标助手则引导你添加目标适配器。只需将目标助手拖到设计器界面，就会出现“添加新目标”对话框，如图 7-8 所示。与源助手对话框一样，你可以为新目标选择类型和连接管理器。

你还可以切换“仅显示已安装项”复选框，以列出仅当前已安装的目标类型或所有支持的类型。

*图 7-8. 目标助手的“添加新目标”对话框*

在此示例中，我们选择了 SQL Server 目标类型和`<新建>`连接管理器。SQL Server 目标类型为我们提供了 OLE DB 连接管理器编辑器，并选择了 Native OLE DB\SQL Server Native Client 11.0 提供程序。此编辑器允许你选择服务器名称、提供身份验证凭据并选择要登录的数据库，如图 7-9 所示。

*图 7-9. 配置 OLE DB 连接管理器*

编辑连接管理器属性后，BIDS 会将适当的目标适配器添加到数据流中，本例中是一个 OLE DB 目标。如果目标适配器上有一个红色`x`，如图 7-10 所示，则意味着你需要配置目标适配器的其他属性。在本例中，我们需要将绿色的平面文件源适配器输出路径箭头拖动并连接到 OLE DB 目标适配器，并配置目标适配器的属性以指向数据库中的表。

> **注意：** 由目标助手创建的 OLE DB 连接管理器的默认名称格式为`server_name.database_name`。你可能希望将其更改为更友好的名称。只需在“连接管理器”选项卡中右键单击连接管理器，然后从上下文菜单中选择“重命名”。在我们的示例中，我们将数据库连接管理器重命名为`Source_DB`。
