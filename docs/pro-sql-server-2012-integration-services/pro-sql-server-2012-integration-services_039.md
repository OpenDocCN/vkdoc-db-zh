# 第 6 章：高级控制流任务

#### 传输数据库任务

`DestinationDatabaseFiles` 指定目标数据库的 `.mdf` 和 `.ldf` 文件的文件名。省略号按钮允许您使用对话框来指定这些文件的位置。也可以指定每个文件位置的网络文件共享。对话框将自动使用服务器设置来填充 `.mdf` 和 `.ldf` 文件的文件夹路径。您需要指定网络文件共享。

`DestinationOverwrite` 指定如果服务器上已存在目标数据库，是否应覆盖它。

`Action` 将任务的操作定义为源数据库的复制或移动。

`Method` 定义传输数据库任务将源数据库置于联机模式还是脱机模式。对于脱机模式，执行包的用户必须对服务器拥有 `sysadmin` 权限。此权限是必需的，因为此模式将创建当前 `.mdf`、`.ldf` 和 `.ndf` 文件的副本。对于联机模式，执行包的用户必须是系统管理员或源数据库的所有者。它将保持数据库可操作，并使用 `.NET SMO` 命名空间。

`SourceDatabaseName` 指定要从源服务器复制或移动的数据库的名称。

`SourceDatabaseFiles` 指定与指定源数据库关联的文件。省略号按钮打开一个对话框，允许您定位文件。对话框将根据文件位置的数据库设置自动填充文件的信息。您需要指定网络文件共享。

`ReattachSourceDatabase` 指定如果执行过程中发生错误，任务是否应重新附加源数据库。

#### 传输错误消息任务

一组对 ETL 过程可能至关重要的数据库对象是用户定义的错误消息。可以使用“传输错误消息任务”在这些服务器之间传输这些消息。该任务通过标识符和错误消息语言的组合来唯一标识用户定义的错误消息。基于此组合，任务可以通过覆盖现有错误消息、在发现现有错误消息时使任务失败或忽略现有错误消息来管理它们。图 6-13 显示了该任务在控制流中的外观。图标显示了一个带有警告标志和箭头的简短消息。箭头表示此类消息的传输。

*图 6-13. 传输错误消息任务*

**注意：** 传输错误消息任务不会复制系统错误消息，即标识符值低于 50,000 的消息。

传输错误消息任务编辑器的“消息”页允许您修改任务的属性，以便在所需的服务器和对象上执行传输。该任务使用 `SQL Management Object 连接管理器` 连接到所需的服务器作为用户定义错误消息的源。“常规”页允许您修改任务的 `名称` 和 `描述`。“表达式”页（如图 6-14 所示）允许您定义可以修改任务属性值的表达式。

*图 6-14. 传输错误消息任务编辑器 - 消息页*

传输错误消息任务的可修改属性如下：

- `SourceConnection` 列出可用作用户定义错误消息源的 `SMO 连接管理器`。
- `DestinationConnection` 列出可用作用户定义错误消息目标的 `SMO 连接管理器`。执行包的用户必须对服务器拥有 `sysadmin` 或 `serveradmin` 权限才能创建用户定义的错误消息。
- `IfObjectExists` 列出当传输的错误消息在目标服务器上已存在时可用的选项。选项为 `FailTask`、`Overwrite` 和 `Skip`。`FailTask` 选项在遇到重复消息时导致任务出错。`Overwrite` 选项覆盖目标服务器上的错误消息。`Skip` 选项忽略目标服务器上的重复错误消息。
- `TransferAllErrorMessages` 定义任务是复制所有用户定义错误消息还是仅复制指定的错误消息列表。此属性的两个选项是 `True` 和 `False`。
- `ErrorMessageList` 允许您创建要复制的用户定义错误消息列表。
- `ErrorMessageLanguageList` 允许您创建要为其创建错误消息副本的语言列表。消息在存储为其他语言之前必须具有 `us_english` 错误消息版本。`us_english` 的代码页是 1033。

#### 传输作业任务

另一组对 ETL 过程至关重要且需要在 SQL Server 实例之间移动的对象是 SQL Agent 作业。“传输作业任务”正是为此需求而设计。它利用 `SQL Management Object 连接管理器` 来传输这些对象。要使用此任务，执行包的用户需要在源实例和目标实例的 `msdb` 数据库上拥有 `sysadmin` 访问权限或 SQL Agent 角色。图 6-15 展示了在控制流中出现的传输作业任务。任务的图标显示了一个由应用程序控制的进程流。该图标表示 SQL Server Agent 作业按给定顺序运行进程。

*图 6-15. 传输作业任务*

传输作业任务编辑器的“作业”页允许您配置在两个 SQL Server 实例之间复制 SQL Server Agent 作业所需的所有属性。图 6-16 显示了此页上所有可用的属性。与之前的传输任务一样，传输作业任务使用 `SMO 连接管理器`。“常规”页允许您修改任务的 `名称` 和 `描述`。“表达式”页允许您定义可以修改任务属性值的表达式。

*图 6-16. 传输作业任务编辑器 - 作业页*

“作业”页使您能够配置以下属性：

- `SourceConnection` 列出包中定义的 `SMO 连接管理器`。选择指向您需要用作 SQL Server Agent 作业源的实例的连接。
- `DestinationConnection` 标识指向您需要用作 SQL Server Agent 作业目标的实例的 `SMO 连接管理器`。
- `TransferAllJobs` 允许您选择传输源实例上存在的所有 SQL Server Agent 作业或传输定义列表中的作业。选择 `False` 允许定义 `JobsList` 属性。
- `JobsList` 枚举要复制到目标实例的特定 SQL Server Agent 作业。
- `IfObjectExists` 允许您确定当 SQL Server Agent 作业已在目标实例上存在时要执行的操作。选择是 `FailTask`、`Overwrite` 和 `Skip`。`FailTask` 在目标实例上遇到相同作业时导致任务出错。`Overwrite` 使用源实例中的定义在目标实例上重新创建重复作业。`Skip` 完全忽略重复作业。
- `EnableJobsAtDestination` 允许您在目标实例上启用复制的作业。两个选项是 `True` 和 `False`。
