开发`SSIS`包通常侧重于已知因素——数据源、转换和目标——而未能解决未知问题——模式更改、数据类型不匹配以及不断变化的网络环境。显然，所有开发都必须从一个起点出发——即已知。然而，为了确保`SSIS`包提供可移植性、稳定性和弹性，处理未知情况并记录未识别和不一致的行为至关重要。本章概述了在`SSIS`包中计划和处理意外情况所需的步骤。

### 什么构成健壮的解决方案

通常，创建`SSIS`解决方案是为了在特定且静态的环境中满足一个易于定义的过程。源和目标模式被认为是固定不变的，没有考虑变化或错误。这种类型的项目创建起来轻而易举，部署也很快，但开发人员往往未能在这种情况下进行`假设分析`，从而无法为以下方面提供支持：

- 跟踪：
  - 性能
  - 错误信息
  - 历史指标
- 处理以下方面的变化：
  - 源
  - 连接
  - 模式
- 可移植性以及轻松部署到不同环境

在尝试创建健壮的解决方案时，需要考虑几个因素：

- **弹性**：包主动处理错误、记录错误并仍然能够完成的能力。
