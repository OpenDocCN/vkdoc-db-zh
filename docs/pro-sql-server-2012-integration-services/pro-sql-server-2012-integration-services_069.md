# 第 9 章  变量、参数和表达式

#### 默认值

对于大多数编程语言，您需要初始化变量，或至少在语句中使用前为其赋值。`值`列允许您为 SSIS 中的变量指定*默认值*。某些数据类型会自动提供默认值，例如`DateTime`和`Int32`，而其他类型如`Object`数据类型则不接受任何默认值。`DateTime`变量将使用其创建时的日期和时间作为默认值，而`Int32`变量的默认值是`0`。如果在设置默认值时未提供有效值，您将收到类似图 9-10 所示的错误对话框。变量值最常发生变化的位置是`执行 SQL`任务和`脚本`任务。

#### 命名空间

*命名空间*是组织变量的最佳方式之一。它们允许您以任何您认为合适的方式对变量进行分组。我们在图 9-8 中展示的约定，根据变量在 ETL 流程中所起的作用对它们进行了分组。当需要在提取阶段使用的变量时，我们使用`Extract`命名空间；当需要用于数据转换的变量时，使用`Transformation`命名空间；如果在加载数据前需要修改数据，则使用`Load`命名空间。用户定义变量的默认命名空间是`User`，并且由于我们创建变量时总是带有非常明确的目的，我们应该充分利用这一特性。

使用命名空间的一个小缺点是，在`表达式生成器`或任何以下拉列表形式显示变量列表的编辑器中，变量列表是按名称而非扩展名称（`命名空间::名称`）排序的。在`表达式生成器`中，`RunTime`变量将被称为`@[Audit::RunTime]`。在显示变量的下拉列表中，它将显示为`Audit::RunTime`。命名空间区分大小写，因为它们是变量标识符的一部分。

### 系统变量

SSIS 提供了变量，使您可以查看包及其对象的细微详情。这些变量称为*系统变量*，它们属于`System`命名空间。这些变量在大多数作用域中都可访问，但根据访问它们的作用域不同，值可能会有所不同。要访问这些变量，您必须使用`@[System::<<变量名>>]`。
