在上一章中，我们向你介绍了 SQL Server 12 的开发工具 BIDS 和 SSMS。

在深入讨论细节之前，我们将通过一些示例包来演示这个强大工具集的功能。作为一名开发人员，在亲自尝试这些工具之后，你更容易理解它们。本章教你如何从最简单的包开始，该包由一个仅包含源和目标的数据流任务组成。在引导你创建这个简单的包之后，我们将指导你完成一个更复杂的、现实生活中包的例子。这个包的目标将是模拟一些可能成为你 ETL 流程一部分的潜在需求，因此作为一个相对复杂的例子在第三章中介绍。

在引导你创建第一个包之前，我们没有涵盖所有细节，原因是 SSIS 拥有大量的工具集，其中大多数不会同时一起使用。例如，大多数项目只会使用少数几个连接管理器，而 SSIS 有更多可用的。根据项目的要求，由 SSIS 处理的流程可能必须与其他工具分开。本章快速浏览了 SSIS 的一些功能。

在本章中，我们的数据来自美国人口普查局网站，[`2010.census.gov/2010census/data/`](http://2010.census.gov/2010census/data/)。我们在本章中使用了 Apportionment 和 Population Density 数据集。我们对数据所做的唯一更改是删除了一些页眉和页脚记录，以使开发 SSIS 12 包的介绍保持简单。实际数据本身未被修改。

*Apportionment* 指的是根据人口将众议院的总席位分配给 50 个州。*Population Density* 报告了该州每平方英里的平均人口。

### Integration Services 项目

正如我们在前一章所讨论的，SSIS 开发的关键是 BIDS 中的 Integration Services 项目。项目文件可以组织与 ETL 过程相关的所有文件。当你首次创建项目时，Visual Studio 会自动为项目创建第一个包，并将其命名为 `Package.dtsx`。第一个包将完全为空，初始设计器将显示控制流。一旦生成包，你就有能力重命名它。

### 关键包属性

包具有根据其提供的功能组织的属性。这些属性执行 ETL 过程的功能性和非功能性需求。功能性需求包括提取、转换和加载（ETL）的原则，而非功能性需求负责 ETL 过程的健壮性、性能和效率、可维护性、安全性、可扩展性、可靠性以及整体质量。在项目中创建新包时，我们建议你查看以下属性：

**名称** 可以通过 Visual Studio 的解决方案资源管理器修改。通过 Visual Studio 的解决方案资源管理器修改的好处，与通过文件系统或包中的属性字段修改相比，包括项目文件的自动识别和包名称属性的自动重命名。创建时，该值默认为 Visual Studio 分配的名称。

**DelayValidation** 确定包对象是否在包打开进行编辑时进行验证。默认情况下，此属性设置为 `False`，意味着所有对象在包打开时都将验证。通过将此属性设置为 `True`，你将仅在运行时进行验证。如果在开发阶段存在数据定义语言（DDL）更改或源或目标的结构更改，这样做可能是危险的。在开发大型包时，通过允许包在运行时验证，它可能很有用。
