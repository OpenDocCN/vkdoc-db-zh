# 数据流转换

##### 百分比采样

**百分比采样转换**会创建两个输出数据集——一个代表选定的行样本，另一个包含数据集中未被选作样本的行。图 8-40 展示了一个简单的数据流，该数据流从本节示例中使用的航空平文件中对行进行采样。

##### 图 8-40. 数据流中的百分比采样转换

如数据流所示，`Percentage Sampling`转换将你的输入数据集拆分为两个独立的输出：
- **采样选择输出**：包含组件为样本集选择的所有行。
- **采样未选择输出**：包含组件未选入样本集的剩余行。

`Percentage Sampling Transformation Editor`使得编辑此转换变得非常简单。只需选择你希望采样的行百分比（默认是 10%），并根据需要更改输出名称。你还可以为转换设置随机种子。如果不设置随机种子，组件将使用操作系统计时器计数器来随机选择输入数据中的样本行，因此样本数据集在每次运行时都会改变。如果你设置了随机种子，那么每次运行都会生成相同的样本数据集（当然，假设源数据在运行之间没有变化）。图 8-41 展示了`Percentage Sampling Transformation Editor`。

##### 图 8-41. 编辑百分比采样转换

当你想要训练和测试数据挖掘模型，或者想要使用输入数据的代表性样本来高效测试你的 ETL 过程时，百分比采样非常有用。

> **提示**：`Percentage Sampling Transformation Editor`允许你选择要包含在样本中的行百分比，但请记住，由于 SSIS 使用的采样算法，你的样本数据集可能会比这个百分比略大或略小。此转换类似于带有`PERCENT`选项的`T-SQL TABLESAMPLE`。

##### 行采样

**行采样转换**类似于百分比采样转换，不同之处在于你选择要采样的行数，而不是行的百分比。图 8-42 展示了数据流中的行采样转换。

##### 图 8-42. 数据流中的行采样转换

图 8-43 所示的`Row Sampling Transformation Editor`与百分比采样编辑器非常相似。唯一的真正区别在于，对于行采样，你选择的是行数而不是采样的行百分比。

##### 图 8-43. 行采样转换编辑器

百分比采样和行采样转换之间存在细微的行为差异。使用百分比采样，你最终得到的样本集可能比你请求的百分比样本大小略大或略小。然而，使用行采样，只要你请求的样本大小小于输入数据集中的行数，你将获得恰好你所请求的行数。如果你的样本大小等于（或大于）输入数据集中的行数，则所有行都会被选入样本。行采样转换在功能上类似于带有`ROWS`选项的`T-SQL TABLESAMPLE`子句。

#### 逆透视

**逆透视转换**用于规范化非规范化的数据集。此转换从输入数据集中获取选定的列，并将它们逆透视为更规范化的行。以我们在透视转换示例中使用的非规范化航空数据为例，我们可以将其逆透视回更规范化的版本，如图 8-44 所示。

##### 图 8-44. 对非规范化数据集进行逆透视

图 8-45 所示的示例数据流中展示了逆透视转换。

##### 图 8-45. 示例数据流中的逆透视转换

配置逆透视转换需要选择要逆透视的列，并为新逆透视的目标列和透视键值列命名。图 8-46 展示了`Unpivot Transformation Editor`。

##### 图 8-46. 逆透视转换编辑器

在编辑器中，你从“可用输入列”部分选择要进行透视的列。任何你不想透视的列会被勾选为“直通”列。在右下角，你选择一个透视键值列名——这是你希望放置透视键值的列的名称。在此示例中，我们选择了列名`Month`，透视键值是各列对应的月份名称。

如果你愿意，可以将透视键值更改为其他内容，但它们必须都是唯一的。在我们的示例中，我们使用了默认值，即与输入列名相同的值。例如，我们可以使用完整的月份名称（January、February 等）作为透视键值。

**目标列**是你希望放置逆透视值的列的名称。在本例中，名称是`TotalMinorInjuries`。每个逆透视列的值都添加在这个新创建的列中。

> **提示**：所有输入列的目标列名必须相同。

### 拆分与联接

SSIS 包含多种转换，允许你在数据流中拆分、合并和联接数据集。这些转换对于组合多个数据集、复制数据集、执行查找以及创建数据集副本非常有用。

#### 查找

**查找转换**允许你通过从参考数据集中检索相关数据来丰富你的数据。查找在功能上类似于 SQL Server 中的联接操作。在`Lookup Transformation Editor`中，你选择源数据列和来自参考数据集的相关列。你还选择希望从参考数据集添加到数据流中的列。图 8-47 是一个示例数据流，它从平文件中提取国家橄榄球联盟（NFL）比赛数据，并针对存储在 SQL Server 数据库表中的球队数据进行查找。

##### 图 8-47. 示例数据流中的查找转换

此数据流中有两个查找：一个查找获胜球队的名称，另一个查找失败球队的名称。`LKP – Winner`查找是使用图 8-48 所示的`Lookup Transformation Editor`配置的。

> **提示**：如果你的参考数据在联接键列中有多个具有相同值的行，则会删除重复项。遇到重复项时，SSIS 将生成警告消息。参考数据行按查找组件检索它们的顺序进行考虑。
