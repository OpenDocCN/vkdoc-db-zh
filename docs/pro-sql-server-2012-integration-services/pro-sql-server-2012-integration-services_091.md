# 第 12 章

## 数据画像与清洗

*编码，加载，然后爆炸。*
—史蒂夫·希契曼

需要将来自多个来源的数据整合在一起的项目——例如，数据仓库、数据集市或操作数据存储（ODS）项目——极为常见。你可能会花费数月时间收集业务需求、整理技术规范、设计目标数据库、并对 ETL 流程进行编码和测试。你也可能会在“临时维护模式”中花费大量时间，重写那些无法处理意外的坏数据或不合规数据的大段代码。这种场景是由于未能正确规划和执行数据集成项目而导致的结果——这种现象被称为*编码，加载，然后爆炸*。

任何时候，当你需要将来自两个或多个系统的数据整合在一起时，你就面临一个*数据集成*项目。正确的数据集成需要大量基础工作。不幸的是，这是任何项目中最不令人兴奋和缺乏魅力的环节之一——全面的规划和流程形式化、源系统调查、源数据探查、生成大量文档。由于偶然或有意为之，在企业项目中，数据集成需求的范围以及未正确执行的成本常常被低估。数据集成需要大量通常较为低调的分析，但未能正确执行可能导致高调的项目失败。本章将展示 SSIS 如何帮助你正确地画像和清洗你的数据。

## 数据画像

*数据画像*，其核心是收集关于你数据的统计信息的过程。目的是通过回答以下类似问题，确保你的源数据符合你的预期：

我的源数据包含空值吗？你可能期望某些源数据列不包含空值，或包含特定数量的空值（例如，少于 10%）。对源数据进行画像将揭示空值数量是否符合你的期望。

我的源数据是否全部为有效值？这涉及为你的源数据定义*域*，即有效值的集合。例如，你可能期望一个`Gender`（性别）列只包含`Male`（男性）或`Female`（女性）值，或者你可能期望一个`Price`（价格）列只包含介于 0.01 到……之间的正数值


以及 500.00。数据画像将告诉你源数据是否落在其预期域内。
