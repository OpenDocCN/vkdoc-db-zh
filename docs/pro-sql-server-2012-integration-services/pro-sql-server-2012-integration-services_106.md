# 第 14 章 - 异构源和目标

### 高级属性编辑器

源组件的一个重要组成部分是高级属性编辑器。“输入和输出属性”选项卡（如图 14-3 所示）尤其允许您调整提取的数据。运行时常见的一个错误是字符串截断错误。当列长度超过 SSIS 分配的缓冲区时，通常会发生此错误。使用高级编辑器，您可以手动修改字符串列的长度，或数值列的精度和小数位数。

另一个可以使用高级编辑器修改的关键属性是 `IsSorted` 属性。该属性通知 SSIS 即将到达的数据是已排序的。除非查询中包含 `ORDER BY` 子句，否则我们不建议修改此属性。在 OLE DB 源输出级别设置该属性后，您可以修改 `ORDER BY` 子句中枚举列的 `SortKeyPosition` 属性。第一列应将该属性设置为 1，第二列设置为 2，依此类推。如果有任何列被定义为按降序排序，则需使用其排序位置的负值来通知 SSIS。例如，如果 `ORDER BY` 子句中的第二列按降序排序，则 `SortKeyPosition` 属性的适当值是 -2。

**图 14-3. OLE DB 源的高级编辑器 — 输入和输出属性**

> **提示：** 对于显式数据转换，我们建议在源查询中使用 `CAST()` 或 `CONVERT()` 函数。另一个选项是使用数据转换转换组件。

### 将数据加载到 SQL Server

从 SQL Server 数据库提取数据后，通常希望将数据加载到另一个 SQL Server 数据库，甚至是同一数据库中。SSIS 为 SQL Server 数据库提供了目标组件。对于 SQL Server 数据库，连接管理器几乎总是 OLE DB 连接管理器。加载数据时，该目标组件提供了根据所有可见列的元数据在数据库上创建表的能力。

该组件甚至允许您预览目标表（即要加载数据的表）中已有的数据。图 14-4 显示了 OLE DB 目标编辑器及其允许的选项。

**图 14-4. OLE DB 目标编辑器**

目标编辑器上的按钮用于确保您为数据集使用了正确的目标。这些按钮执行以下功能：

*   **新建表或视图** 根据传递到目标组件的元数据，生成用于创建对象的脚本。
*   **查看现有数据** 打开一个对话框，显示指定为目标表中的 200 个样本行。
*   **快速加载** 选项允许 SSIS 在行通过缓冲区时进行批量加载。**表或视图** 选项实际上为到达目标组件的每一行执行一条 `INSERT` 语句。**行批处理大小** 和 **最大插入提交大小** 是可以在数据提交期间修改以提升性能的选项。

> **注意：** 如果您单击 **表或视图名称** 下拉列表旁边的 **新建** 按钮，然后在包含 `CREATE TABLE` 脚本的模式窗口上单击 **确定**，Visual Studio 将尝试在数据库上创建表。这依赖于用户帐户拥有对数据库的适当权限。

目标组件的另一个可用选项是能够执行 SQL 命令。此 SQL 语句将对通过管道的每一行执行。它可用于 `UPDATE` 或 `INSERT` 功能。根据数据量，可能有其他方法可以优化该数据的加载。

目标组件的“映射”页面（如图 14-5 所示）显示输入列如何与 SQL 中的目标列匹配。此映射将左侧的输入列与右侧的目标表列配对。如果存在数据类型或定义不匹配，您可以选择修改映射以允许不同的列。在处理数据转换转换组件时，此页面特别方便。因为 SSIS 尝试基于列名自动匹配，所以转换后的列通常不是那些被映射到目标列的列。

**图 14-5. OLE DB 目标编辑器的映射页面**

除了 OLE DB 目标外，SSIS 还提供了一个 SQL Server 特定的批量加载目标组件，即 SQL Server 目标。该组件将数据批量加载到表和视图中，但前提是它们位于执行包的同一台机器上。这是一个巨大的限制，因此在该目标的说明中，Microsoft 建议改用 OLE DB 组件。该组件的编辑器与 OLE DB 目标非常相似，只是它除了连接管理器、要加载数据的表或视图以及映射页面外，不提供任何其他选项。

### 其他 RDBMS 源和目标

对于 SQL Server 以外的其他关系数据库管理系统，Microsoft 提供了专门为提取数据而调整的驱动程序。对于 SSIS，这些其他系统的提供程序由 Attunity 开发。（在早期版本中，Attunity 为 Oracle 连接提供驱动程序。）如图 14-6 所示的源助手为您提供有关如何获取默认不附带的连接提供程序的信息。对于其中一些提供程序，所有者可能会收取驱动程序使用费。

**图 14-6. 异构源的提供程序**

使用非 SQL Server 连接的一个陷阱是忘记确保提供程序能识别 SQL 方言。一些提供程序不识别语句终止符（`;`），而其他提供程序则解析提供的查询时，会将其视为单行代码，即使文本编辑器显示了换行和格式。在第二种情况下，您必须注意如何注释掉代码行。您将必须使用多行注释而不是单行注释。

当使用不同的存储系统作为数据源时，我们建议在源组件后立即使用数据转换转换组件。这将允许您使用 Union All 转换和其他转换，而不必担心数据集之间的数据类型不匹配。

### 平面文件源和目标

平面文件通常作为数据转储的输出过程生成。这些平面文件可用作数据存储，但它们缺乏关系数据库管理系统提供的许多优势。然而，保留平面文件的一个吸引力在于安全性不一定像数据库系统那样严格。它们可以提供对数据的快速访问，而不会对托管数据库系统中的数据的服务器产生影响。平面文件的缺点是它们不能被查询，无法维护数据完整性，并且不存在数据质量维护措施。如果存在此类系统，所有这些优势都必须在作为平面文件来源的数据库系统上强制执行。
