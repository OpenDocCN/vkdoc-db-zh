# 前言

`SQL` 是一种强大的语言。寥寥数行代码，便能解答复杂的查询。

`PL/SQL` 包含许多扩展功能，使得 `SQL` 更易于使用。这种紧密的集成使这些语言非常适合数据管理。

然而，面对问题时，*究竟应该使用* `SQL` 或 `PL/SQL` 的*哪些*特性，有时并不清晰。Oracle 数据库的文档体量庞大。希望通过浏览文档找到能帮助你的语法或特性，是一项无止境的任务。

不幸的是，这意味着许多应用程序的代码长达数页，仅仅是为了重新实现 Oracle 数据库已有的功能。这通常是因为开发人员未能意识到这些功能已触手可及。

因此，看到 Alex 和 Patrick 撰写此书，我感到非常兴奋。他们长期以来一直致力于分享知识，帮助他人使用 Oracle 数据库。他们基于实际经验并乐于助人理解 Oracle `SQL` 和 `PL/SQL` 的意愿，使他们双双赢得了当之无愧的 Oracle ACE Director 认可。这使他们成为撰写此指南的理想人选。

阅读本书将教会你如何使用更强大的数据库选项，其中许多是其他平台所不具备的。书中充满了示例，向你展示这些 `SQL` 和 `PL/SQL` 语言特性如何工作。但最重要的是，这些脚本有助于说明何时以及为何应该使用这些选项。

在使用任何技术时，我认为你应该了解它提供了哪些功能。利用内置功能可以节省你的时间并简化你的代码。Alex 和 Patrick 在展示如何做到这一点方面做得非常出色。

多年来，我从他们两位身上学到了很多。希望这本书也能帮助你编写出更好的 `SQL` 和 `PL/SQL`。

—Chris Saxon, Oracle Developer Advocate

#### 引言

在撰写一本关于数据库技术的书籍时，你需要构思一个数据集来编写示例。我们本可以使用在 `SCOTT` 模式中存在已久的那套标准 `EMP` 和 `DEPT` 表。然而，由于这是一本关于现代 Oracle 数据库编程的书，我们希望使用一个不同的数据集。

因为我们两人都是一级方程式赛车的粉丝，而 Oracle 正好是红牛车队（我们国家的车手 Max Verstappen 恰好效力于该车队）的重要赞助商，我们便开始寻找一个与一级方程式相关的数据集。当然，本书中的示例并非全部与 Max 有关；许多其他车手和赛事结果也扮演着重要角色。

Ergast Developer API ([`https://ergast.com/mrd/`](https://ergast.com/mrd/)) 允许你查询各种赛车数据，并以多种格式获取。该网站还以 CSV 格式提供了所有表格，因此我们创建了一个 `F1DATA` 模式来导入这些数据。原始数据集是 MySQL 数据库，因此我们不得不对数据模型做一些微小的调整。

本书的源文件可在 GitHub 上找到：[`https://github.com/Apress/modern-oracle-database-programming`](https://github.com/Apress/modern-oracle-database-programming)。
