# 第 1 章 SQL Server 数据类型

SQL Server 2017 提供了广泛的基本数据类型来存储数据，例如字符串、数值数据、二进制数据和日期。在本章中，我将回顾 SQL Server 中可用的每种基本数据类型，探讨相似数据类型之间的差异。我还将讨论为每个用例使用正确数据类型的重要性。

**提示：** 以下各节讨论了每种数据类型的可接受范围。然而，值得注意的是，所有数据类型也可以设置为 `NULL`，表示该值未知。

## 数值数据类型

SQL Server 中的一些数值数据类型是精确的，而另一些是近似的。如果数据类型是精确的，它会以固定精度存储数字。如果数据类型是近似的，在许多情况下，它会存储一个非常接近的近似值，而不是数字本身。

## SQL Server 数据类型

### 精确数值类型

#### NUMERIC 和 DECIMAL

`NUMERIC` 和 `DECIMAL` 在功能上是等价的。它们用于存储具有固定精度和小数位数的数字。

使用这些数据类型时，必须指定精度（`precision`）和小数位数（`scale`）。
*   **精度** 是可以存储的总位数。
*   **小数位数** 是小数点后的位数。

例如，数字 `123.456` 可以用精度 `6` 和小数位数 `3` 来存储。

##### 存储空间

所需存储空间取决于精度，如下所示：
*   1-9 – 5 字节
*   10-19 – 9 字节
*   20-28 – 13 字节
*   29-38 – 17 字节

### 近似数值类型

#### FLOAT、REAL

`REAL` 是 `FLOAT(24)` 的同义词。`FLOAT` 用于存储具有近似精度的数字。
*   如果未指定精度，`FLOAT` 使用双精度（8 字节）。
*   如果精度在 1 到 24 之间，则使用单精度（4 字节），也称为 `real`。
*   如果精度在 25 到 54 之间，则使用双精度（8 字节）。

单精度近似于 7 位数字，存储空间为 4 字节。双精度近似于 15 位数字，存储空间为 8 字节。

**近似数值类型范围：**
*   单精度：`-1.79E+308` 到 `-2.23E-308` 以及 `2.23E-308` 到 `1.79E+308`
*   双精度：`-1.79E+308` 到 `-2.23E-308` 以及 `2.23E-308` 到 `1.79E+308`

### 精确数值类型（货币）

#### SMALLMONEY

存储小的货币值，精度为四位小数。范围从 `-214,748.3648` 到 `214,748.3647`，存储空间为 4 字节。

#### MONEY

存储大的货币值，精度为四位小数。范围从 `-922,337,203,685,477.5808` 到 `922,337,203,685,477.5807`，存储空间为 8 字节。

**提示**：`^`（脱字符）运算符表示幂运算。例如，`2¹⁵-1` 表示 2 的 15 次方减 1，等于 `32,767`。

### 数据类型转换函数

为了了解一个数字在不同数值格式下的表现，我们将使用数字 `2.5888712345678923456789` 并将其转换为每种数值数据类型，以检查其行为。在 SQL Server 中，有两个函数可用于更改值的数据类型：`CAST` 和 `CONVERT`。

#### CAST 函数

`CAST` 函数的语法如下：
```
CAST( expression AS datatype[(length)] )
```
其中 `expression` 可以是要转换的值、表中的列或任何其他有效表达式。`datatype` 应是表达式的目标数据类型。`length` 是可选参数，具体取决于目标数据类型。例如，如果使用 `CAST` 将 `INT` 的数据类型更改为 `DECIMAL`，则 `length` 应包含所需的精度和小数位数。如果将值转换为字符串，则 `length` 将是字符串的最大长度。

#### CONVERT 函数

`CONVERT` 函数的语法如下：
```
CONVERT( datatype[(length)], expression [, style] )
```
`CONVERT` 函数的 `style` 选项允许开发人员提供有关如何转换表达式的附加信息。
