# 第 2 章 理解 XML

许多 SQL Server 数据结构以 XML 格式存储，包括：
*   扩展事件
*   DDL 触发器中的事件信息
*   事件通知
*   数据收集器

对 XML 的原生支持使开发人员和管理员能够与此数据交互。虽然你可能认为 JSON 是现代首选的格式，但 XML 仍具有 JSON 不支持的功能，例如：
*   架构验证
*   `XPath`
*   命名空间
*   `XLST`

## 小结

XML 是一种类似 HTML 的标记语言，但其标签由文档作者定义。一个 `XML 文档` 由元素和属性组成，以半结构化的树格式存储数据。

SQL Server 支持 `XML` 片段、格式正确的 `XML` 文档以及有效（绑定架构的）`XML` 文档。要格式正确，一个 `XML` 文档必须包含单个根节点，并且除非格式正确，否则 `XML` 文档不能绑定架构。短语 `XML 文档` 和 `XML 实例` 常被互换使用，尽管严格来说，`XML 文档` 指的是格式正确的 XML。

`XML` 文档可以绑定到 DTD（`文档类型定义`）或 XSD（`XML 架构定义`）架构。SQL Server 仅通过 `XML 架构集合` 支持存储 XSD 架构。
