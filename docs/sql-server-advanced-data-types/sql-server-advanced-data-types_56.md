# 第 6 章 理解 JSON

## 分析日志数据

传感器或 RFID（射频识别）等设备会产生海量数据。这意味着数据架构师通常会选择将这些数据存储在 NoSQL 解决方案中。通常，JSON 被用作此类日志的文件格式。

当日志以 JSON 格式存储时，SQL Server 原生的 JSON 支持意味着这些日志可以轻松读入 SQL Server，并使用 T-SQL 进行分析，而无需任何复杂的解析要求。这可以缩短针对日志数据的报告的上市时间。

## 总结

JSON 是一种轻量级的数据交换格式，受许多 REST API 和 Web 服务端点的支持。它与 XML 相似，因为它是一种可扩展的、自描述的、分层的文档格式，但在以下方面有所不同：

• 它无法根据模式进行验证。
• 它没有闭合标签。
• 它更短。
• 它更容易解析。
• 它支持数组。

JSON 的基本格式是一系列由逗号分隔的名称/值对。JSON 对象始终用花括号括起来，数组始终用方括号括起来。JSON 对象可以嵌套在其他 JSON 对象内部。在这种情况下，应使用反斜杠转义字符，例如双引号和花括号。

JSON 在 SQL Server 中有许多用例，包括简化与后端数据库交互的 REST API。当必须将 NoSQL 解决方案与 SQL Server 集成时（例如必须分析设备日志或将 NoSQL 半结构化数据与结构化数据一起存储时），JSON 也是一个不错的选择。当 DBA 或平台工程师实施“配置即代码”解决方案时，JSON 也非常有用。
