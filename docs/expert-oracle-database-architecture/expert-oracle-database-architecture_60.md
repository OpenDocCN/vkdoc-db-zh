# 11. 索引

索引是您的应用程序设计和开发的关键环节。索引太多，修改（插入、更新、合并和删除）的性能会受到影响。索引太少，DML（包括选择、插入、更新和删除）的性能也会受到影响。找到合适的平衡点对您的应用程序性能至关重要。

我经常发现，索引在应用程序开发中是事后才考虑的事情。我认为这是错误的方法。如果您从流程一开始就理解数据将如何使用，就应该能够提出您将在应用程序中使用的代表性索引集。很多时候，方法似乎是先把应用程序部署出去，然后再看哪里需要索引。这意味着您没有花时间去理解数据将如何使用以及最终需要处理多少行数据。随着数据量随时间增长，您将不得不持续地为这个系统添加索引（即，您将执行`reactive tuning`）。您会拥有冗余且从未使用的索引；这不仅浪费空间，也浪费计算资源。在开始时花几个小时恰当地考虑何时以及如何为您的数据创建索引，将在后续为您节省许多小时的调优时间（请注意，我说的是*会*，而不是*可能*，为您节省许多小时）。

本章的基本目的是概述 Oracle 中可供使用的索引，并讨论您可能在何时何地使用它们。本章在风格和格式上与本书其他章节不同。索引是一个庞大的主题——您完全可以就此主题写一整本书——部分原因在于索引跨越了开发人员和 DBA 的角色。开发人员必须了解索引，索引如何应用于他们的应用程序，何时使用索引（以及何时不使用）等等。DBA 关注的是索引的增长、索引内的存储使用情况、其他物理属性以及数据库的整体性能。我们将主要从索引在应用程序中的实际使用角度来探讨它们。本章前半部分传达了我认为您需要的基本知识，以便就何时索引以及使用何种类型的索引做出明智的选择。本章后半部分回答了一些关于索引最常见的问题。

本章中的各种示例需要 Oracle 的不同功能版本。当特定示例需要 Oracle 企业版或个人版中才有，而标准版中没有的功能时，我会特别说明。
