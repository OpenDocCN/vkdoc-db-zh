# 数据仓库系统

在数据仓库/决策支持系统中，分区不仅是一项出色的管理工具，还是能够加快处理速度的技术。例如，你可能有一个需要执行即席查询的大表。你总是按销售季度进行即席查询，因为每个销售季度包含数十万条记录，而你总共拥有数百万条在线记录。因此，你希望查询整个数据集中相对较小的一个切片，但基于销售季度为其建立索引并不真正可行。这种索引将指向数十万条记录，以这种方式进行索引范围扫描效果会很差（有关此点的更多详细信息，请参阅第 11 章）。你的许多查询需要进行全表扫描，但最终你不得不扫描数百万条记录，其中大部分记录并不符合你的查询条件。使用智能分区方案，你可以按季度隔离数据，这样当你查询任何特定季度的数据时，只会对该季度的数据进行全扫描。这是所有可能方案中的最佳选择。

此外，在数据仓库/决策支持系统环境中，并行查询被频繁使用。在这里，诸如并行索引范围扫描或并行快速全索引扫描等操作不仅有意义，而且对我们有益。我们希望最大化利用所有可用资源，而并行查询就是实现这一目标的一种方法。因此，在这种环境中，分区极有可能加快处理速度。

## 在 OLTP 系统中减少争用

分区最后一个普遍的好处是，可能通过减少 OLTP 系统中的争用来提高并发性。分区可用于将单个表的修改操作分散到多个物理分区上。其思路是，如果你有一个经历高争用的段，将其转换为多个段可能会按比例减少争用，从而产生副作用。

例如，你可能拥有 20 个表分区和 20 个索引分区，而不是拥有 1 个表段和 1 个索引段。这就像拥有 20 个表而不是 1 个（以及 20 个索引而不是 1 个）；因此，在修改操作期间，对这个共享资源的争用将会减少。
