# NOLOGGING 总结

可以在 `NOLOGGING` 模式下执行的操作如下：

*   索引的创建和 `ALTER`（重建）。
*   使用直接路径 `INSERT`（例如通过 `/*+ APPEND */` 提示或 SQL*Loader 直接路径加载）对表进行的批量 `INSERT`。表数据不会生成重做日志，但所有索引修改都会生成（此非日志记录表上的索引将生成重做日志）。
*   `LOB` 操作（对大对象的更新不必记录）。
*   通过 `CREATE TABLE AS SELECT` 创建表。
*   各种 `ALTER TABLE` 操作，例如 `MOVE` 和 `SPLIT`。

在 `ARCHIVELOG` 模式数据库中适当使用 `NOLOGGING`，可以通过显著减少生成的重做日志量来加速许多操作。假设你有一个需要从一个表空间移动到另一个表空间的表。你可以将此操作安排在备份发生之前立即进行——你可以先将表 `ALTER` 为 `NOLOGGING`，然后移动它，重建索引（同样不记录日志），然后再将表 `ALTER` 回 `LOGGING` 模式。现在，一个可能需要 X 小时的操作可能只需要 X/2 小时（我并非承诺运行时间减少 50%！）。适当使用此功能需要让 DBA 或负责数据库备份与恢复或任何备用数据库的人员参与进来。如果该人员不知道你正在使用此功能，并且发生了媒体故障，你可能会丢失数据，或者备用数据库的完整性可能会受到损害。这是需要认真考虑的事情。

## 为什么我无法分配新日志？

我经常收到这个问题。你可能会收到类似这样的警告信息（这可以在你服务器的 `alert.log` 中找到）：

```
Thread 1 cannot allocate new log, sequence 1466
Checkpoint not complete
Current log# 3 seq# 1465 mem# 0: /.../...redo03.log
```

它可能显示 `Archival required` 而不是 `Checkpoint not complete`，但效果基本相同。这实际上是 DBA 应该注意的事情。每当数据库尝试重用一个在线重做日志文件但发现无法重用时，此信息就会被写入服务器的 `alert.log`。当 `DBWn` 尚未完成检查点写入受重做日志保护的数据，或者 `ARCn` 尚未完成将重做日志文件复制到归档目标时，就会发生这种情况。此时，从最终用户的角度来看，数据库实际上停止了。它完全停止。`DBWn` 或 `ARCn` 将被优先处理，以将数据块刷新到磁盘。检查点或归档完成后，一切恢复正常。数据库暂停用户活动的原因很简单：没有地方记录用户正在做的更改。Oracle 正在尝试重用一个在线重做日志文件，但由于要么该文件在发生故障时用于恢复数据库（`Checkpoint not complete`），要么归档器尚未完成复制（`Archival required`），Oracle 必须等待（最终用户也将等待），直到该重做日志文件可以安全地重用。

如果你的会话花费大量时间等待 "log file switch"、"log buffer space" 或 "log file switch checkpoint or archival incomplete"，你很可能遇到了这个问题。如果你的日志文件大小设置不当，或者需要由 DBA 或系统管理员调整 `DBWn` 和 `ARCn`，你可能会在数据库长时间修改期间注意到它。我经常在未经定制的 "starter" 数据库中看到此问题。"starter" 数据库通常将重做日志文件设置得非常小，无法满足任何显著的工作量（包括数据字典本身的初始数据库构建）。一旦开始向数据库加载数据，你会注意到前 1000 行加载很快，然后事情开始时快时慢：快 1000 行，然后挂起，再快 1000 行，再挂起，如此反复。这些都是你遇到此情况的迹象。

你可以采取以下几件事来解决此问题：

*   让 `DBWn` 更快：让你的 DBA 通过启用 `ASYNC` I/O、使用 `DBWn` I/O 从属进程或使用多个 `DBWn` 进程来调整 `DBWn`。检查系统上的 I/O，看看是否有一个磁盘或一组磁盘很“热”，因此你需要分散数据。同样的通用建议也适用于 `ARCn`。这种方法的优点是，你在这里“免费获得了一些东西”——无需真正更改任何逻辑/结构/代码即可提高性能。这种方法确实没有缺点。
*   添加更多重做日志文件：在某些情况下，这将推迟 `Checkpoint not complete`，并且过一段时间后，它会将 `Checkpoint not complete` 推迟到可能不再发生（因为你给了 `DBWn` 足够的空间进行检查点操作）。这同样适用于 `Archival required` 消息。这种方法的好处是消除了系统中的“暂停”。缺点是消耗更多磁盘空间，但好处远大于任何缺点。
