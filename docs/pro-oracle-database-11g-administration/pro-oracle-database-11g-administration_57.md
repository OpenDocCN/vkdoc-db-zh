# echo oracle >> /etc/cron.allow
```

一旦 `oracle` 条目被添加到 `/etc/cron.allow` 文件中，你就可以使用 `crontab` 工具来调度任务了。

> **注意：** 你也可以使用编辑工具（如 `vi`）向 `cron.allow` 文件添加条目。

`root` 用户始终可以使用 `crontab` 工具调度任务。其他用户必须被列在 `/etc/cron.allow` 文件中。如果 `/etc/cron.allow` 文件不存在，那么该操作系统用户不得出现在 `/etc/cron.deny` 文件中。如果既不存在 `/etc/cron.allow` 文件，也不存在 `/etc/cron.deny` 文件，那么只有 `root` 用户可以访问 `crontab` 工具。

> **注意：** 在某些 Unix 操作系统（例如 Solaris）上，`cron.allow` 和 `cron.deny` 文件位于 `/etc/cron.d` 目录中。

## **理解 cron 表条目**

你的 cron 表是一个由数字和命令组成的列表，`cron` 后台进程（`cron` 守护进程）将在指定的时间和计划下运行它们。`crontab` 工具要求条目遵循明确定义的格式。

在你的 `crontab` 文件开头添加一行注释来记录所需格式是一个好主意：



# 第 21 章 ■ 自动化工件

# 分(0-59) 时(0-23) 日(1-31) 月(1-12) 周(0/7-6) 命令或脚本

在上一行中，cron 文件中的井号 (`#`) 表示注释的开始。输入到 `#` 之后的任何文本都会被 cron 忽略。

`crontab` 中的每个条目都是包含六个字段的单行。前五个字段指定了执行时间和频率。这些条目可以用逗号或连字符分隔。逗号表示条目的多个值，而连字符表示一个值范围。条目也可以是星号 (`*`)，表示所有可能的值都生效。下面有一个例子来帮助澄清。

以下条目在星期一至星期五的上午 8 点到下午 4 点半之间，每半小时发送一封主题为“wake up”的电子邮件：

```
0,30 8-16 * * 1-5 echo "wake up" | mailx -s "wake up" larry@oracle.com
```

在一些 Linux 系统上，您可以通过在条目后跟上 `/<整数>` 来在某个范围内跳过一个值。

例如，如果您想每隔一分钟运行一个任务，可以在分钟列使用 `0-59/2`。您也可以将斜杠 (`/`) 与星号结合使用来跳过值。例如，要每四分钟运行一个任务，您会在分钟列使用 `*/4`。

`crontab` 中的第六个字段可以是一个或多个 Linux 命令或一个 shell 脚本。或者换句话说，第六列可以是您在 Linux 命令行中能一行运行的任何命令组合或一个脚本。

`cron` 工具有几个需要进一步解释的特殊之处。第五列是星期几。星期日由 `0` 或 `7` 指定，星期一由 `1` 指定，星期二由 `2` 指定，依此类推，直到星期六，用 `6` 表示。

第二列中的小时数采用 24 小时制，范围从 `0` 到 `23`。第四列（月份）和第五列（星期几）可以用数值表示，也可以用三个字母的缩写表示。例如，`crontab` 中的以下条目使用了月份和星期的三字母缩写：

```
0,30 8-16 * Jan-Dec Mon-Fri echo "wake up" | mailx -s "get up" larry@oracle.com
```

似乎还有一些重叠的列，例如第三列（日）和第五列（星期几）。这些列允许您为需要在特定时间表（如每月的 1 号和 15 号或每周二）运行的任务创建灵活的计划。在您不使用的列中放入一个星号。如果您需要在 1 号、15 号和每周二运行一个任务，那么就填写这两个列。

如果您从 `cron` 运行一个包含对 Oracle 实用程序（如 `sqlplus` 或 `rman`）调用的 shell 脚本，请确保您实例化（source）任何必需的操作系统变量，例如 `ORACLE_SID` 和 `ORACLE_HOME`。

如果您不 source 这些变量，当您的 shell 脚本从 `cron` 运行时，您会看到如下错误：

```
sqlplus: command not found
```

当 `cron` 以用户身份运行脚本时，它不会运行该用户的启动或登录文件（如 `.bashrc`）。因此，任何（从 `cron` 运行的）脚本都需要显式设置任何必需的变量。您可以在脚本中直接设置变量，或者调用另一个导出这些变量的脚本（如 Oracle 的 `oraenv` 脚本）。

**提示** 不要将在 `cron` 中输入的所有任务都安排在同一时间运行。相反，应将它们分散开，以免在任何特定时间点拖慢 `cron` 或系统。
```
