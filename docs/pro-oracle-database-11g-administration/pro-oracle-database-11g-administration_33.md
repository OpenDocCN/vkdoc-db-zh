# 查看警报日志

函数`valert`定义如下：

```bash
function valert {

echo $ORACLE_HOME | grep 11 >/dev/null

if [ $? -eq 0 ]; then

lower_sid=$(echo $ORACLE_SID | tr '[:upper:]' '[:lower:]')

view $ORACLE_BASE/diag/rdbms/$lower_sid/$ORACLE_SID/trace/alert_$ORACLE_SID.log else

view $ORACLE_BASE/admin/$ORACLE_SID/bdump/alert_$ORACLE_SID.log

fi

} # valert
```

#-----------------------------------------------------------#

如果你曾怀疑一个快捷方式是别名还是函数，可以使用`type`命令来验证命令的来源。以下示例验证了`bdump`是一个函数：

```bash
$ type bdump
```

输出如下：

```bash
bdump is a function

bdump ()

{

echo $ORACLE_HOME | grep 11;

if [ $? -eq 0 ]; then

lower_sid=$(echo $ORACLE_SID | tr '[:upper:]' '[:lower:]');

cd $ORACLE_BASE/diag/rdbms/$lower_sid/$ORACLE_SID/trace;

else

cd $ORACLE_BASE/admin/$ORACLE_SID/bdump;

fi

}
```

## 快速重新运行命令

当数据库服务器出现问题时，你需要能够从操作系统提示符快速运行命令。你可能遇到了某种性能问题，想要运行命令导航到包含日志文件的目录，或者你可能想要不时地显示消耗资源最多的进程。在这些情况下，你不想浪费时间重新输入命令序列。

## 第三章 ■ 配置高效环境

Bash shell 的一个省时特性是它提供了多种编辑和重新运行先前执行过的命令的方法。以下列表突出显示了可用于操作先前输入命令的几种选项：

*   使用上下箭头键滚动
*   使用`Ctrl+P`和`Ctrl+N`
*   列出命令历史记录
*   反向搜索
*   设置命令编辑器

以下各节将简要描述这些技术。

## 使用上下箭头键滚动

你可以使用向上箭头在最近的命令历史记录中向上滚动。当你滚动浏览先前运行的命令时，可以通过按`Enter`或`Return`键来重新运行所需的命令。

如果你想编辑一个命令，请使用`Backspace`键删除字符，或使用左箭头键导航到命令文本中的所需位置。通过命令堆栈向上滚动后，可以使用向下箭头键向下滚动回先前查看过的命令。

**注意** 如果你熟悉 Windows，通过命令堆栈滚动类似于使用`DOSKEY`实用程序。

### 按下 Ctrl+P 和 Ctrl+N

`Ctrl+P`按键（同时按下`Ctrl`和`P`键）显示你先前输入的命令。如果你已经多次按下`Ctrl+P`，可以通过按`Ctrl+N`（同时按下`Ctrl`和`N`键）向下滚动回命令堆栈。

## 列出命令历史记录

你可以使用`history`命令来显示用户先前输入的命令：

```bash
$ history
```

根据先前执行了多少命令，你可能会看到一个很长的堆栈。

你可以通过向命令提供一个数字来限制输出为最后*n*个命令。例如，以下命令列出最后运行的五个命令：

```bash
$ history 5
```

以下是示例输出：

```bash
273 cd -
274 grep -i ora alert.log
275 ssh -Y -l oracle 65.217.177.98
276 pwd
277 history 5
```

要运行输出中列出的先前命令，使用感叹号（`!`，有时称为*bang*），后跟历史编号。在此示例中，要运行第 276 行的`pwd`命令，使用`!`如下：

```bash
$ !276
```

要运行你运行的最后一个命令，使用`!!`，如下所示：

```bash
$ !!
```

### 反向搜索

按`Ctrl+R`，你会看到 Bash shell 的反向搜索实用程序：

```bash
$ (reverse-i-search)`':
```

在反向搜索提示符下，当你键入每个字母时，该工具会自动搜索先前运行过的、具有与你输入的字符串相似文本的命令。一旦出现你想要的命令匹配项，你可以通过按`Enter`或`Return`键重新运行该命令。要查看所有匹配字符串的命令，请重复按`Ctrl+R`。要退出反向搜索，请按`Ctrl+C`。

### 设置命令编辑器

你可以使用`set -o`命令将你的命令行编辑器设置为`vi`或`emacs`。此示例将命令行编辑器设置为`vi`：

```bash
$ set -o vi
```

现在，当你按`Esc+K`时，你将进入一种模式，可以在其中使用`vi`命令来搜索先前输入的命令堆栈。

例如，如果你想向上滚动命令堆栈，可以使用`K`键；类似地，可以使用`J`键向下滚动。在此模式下，可以使用斜杠（`/`）键，然后键入要在整个命令堆栈中搜索的字符串。

**提示** 在尝试使用命令编辑器功能之前，请确保你非常熟悉`vi`或`emacs`编辑器。

一个简短的例子将说明此功能的强大之处。假设你知道大约一小时前运行了`ls -altr`命令。你想再次运行它，但这次不带`r`（反向排序）选项。

要进入命令堆栈，请按`Esc+K`：

```bash
$ Esc+K
```

你现在应该看到你执行的最后一条命令。要在命令堆栈中搜索`ls`命令，键入`/ls`然后按`Enter`或`Return`：

```bash
$ /ls
```

最近执行的`ls`命令出现在提示符处：

```bash
$ ls -altr
```

要删除`r`选项，使用右箭头键将提示符定位到屏幕上的`r`上，然后按`X`从命令末尾删除`r`。编辑命令后，按`Enter`或`Return`键执行它。

## 开发标准脚本

我曾在一些地方工作，那里的数据库管理团队开发了数百个脚本和实用程序来帮助管理环境。一家公司有一个小型的 DBA 团队，其工作职能就是维护环境脚本。我认为这有点过头了。我倾向于使用一组小型、专注的脚本，每个脚本通常少于 50 行。如果你开发的脚本是其他 DBA 无法理解或维护的，那么它可能太复杂了。

**注意** 本章中的所有脚本均可从 Apress 网站 www.apress.com 下载。转到本书的目录页面。在书籍封面图像下方查找“Source Code”链接。

本节包含几个简短的 shell 函数、shell 脚本和 SQL 脚本，可帮助你管理数据库环境。这绝不是一个完整的脚本列表——而是提供了一个你可以构建的起点。每个小节标题就是一个脚本的名称。

**注意** 在尝试运行 shell 脚本之前，请确保它是可执行的。使用`chmod`命令来实现：`chmod 750 <script>`

### dba_setup

通常，你会为每个数据库服务器设置一组常见的操作系统变量和别名。在服务器之间导航时，你应该以一致且可重复的方式设置这些变量和别名。这样做有助于你（或你的团队）在每个环境中高效操作。例如，当你使用几十台不同的服务器时，以一致的方式设置操作系统提示符非常有用。这有助于你快速识别你所在的机器、你登录的操作系统用户等。

一种技术是将这些标准设置存储在一个脚本中，然后在登录服务器时自动执行该脚本。我通常创建一个名为`dba_setup`的脚本来设置这些操作系统变量和别名。你可以将此脚本放在像`HOME/bin`这样的目录中，并通过启动脚本（在本章后面的“组织脚本”部分介绍）自动执行。以下是一个典型的`dba_setup`脚本的内容：

```bash
# set prompt
PS1='[\h:\u:${ORACLE_SID}]$ '
#
export EDITOR=vi
export VISUAL=$EDITOR
export SQLPATH=$HOME/scripts
set -o vi
#
# list directories only
alias lsd="ls -p | grep /"
```
