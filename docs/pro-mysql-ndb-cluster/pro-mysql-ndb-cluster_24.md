# 10. 重启

对于 MySQL NDB 集群的 DBA 来说，更常见的日常任务之一是重启集群节点。由于 MySQL NDB 集群是为高可用性而设计的，大多数重启都可以在集群整体保持在线的状态下进行。这意味着在 MySQL NDB 集群中，重启扮演着特殊的角色，值得特别关注。

在 MySQL NDB 集群中，重启用于多种原因，例如：

*   配置变更
*   添加管理节点、数据节点或 API/SQL 节点
*   从崩溃中恢复
*   升级（在第 11 章讨论）

### 重启类型

如第 2 章所述，MySQL NDB 集群有四种重启类型：

*   节点重启
*   初始节点重启
*   系统重启
*   初始系统重启

对于两种节点重启类型（节点重启和初始节点重启），重启期间集群保持在线，而系统重启类型则要求集群离线。本节其余部分提供了关于这四种重启类型的更多信息。

#### 节点重启

节点重启是指一个或多个节点一起重启，在此过程中，始终保持剩余的在线节点拥有所有数据。也就是说，每个节点组中始终至少有一个数据节点在线。当整个集群以这种方式重启时，也称为滚动重启（参见下一节）。

滚动重启的优点是，您可以在集群保持在线的同时重启整个集群。滚动重启的典型用例是更改数据节点的配置。只有在数据有多个副本（`NoOfReplicas` 大于 1）时，节点重启才可能，因为每个节点组必须至少有一个节点保持在线。

#### 初始节点重启

初始节点重启——如果包括整个集群，则也称为初始滚动重启——是节点重启的一种变体。区别在于，每个数据节点在重启开始时将丢弃其所有数据，然后在重启期间从同一节点组中的另一个节点恢复数据。重做日志文件在初始重启中总是会被重新创建。

注意

虽然初始节点重启开始时会删除数据节点的所有数据，但用于磁盘数据的撤消日志和表空间文件会被保留。如果您也想重新创建这些文件，必须在重新启动数据节点之前手动删除它们。

初始节点重启的优点是，可以进行一些在其他情况下无法进行的配置更改。例如，更改重做日志设置的唯一方法是让数据节点从头开始。初始滚动重启的另一个用途是从文件系统损坏中恢复。缺点是重启时间会更长，因为所有数据必须从同一节点组中的其他节点复制。
