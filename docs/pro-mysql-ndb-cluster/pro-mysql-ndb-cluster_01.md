# 引言

MySQL NDB Cluster 是 MySQL 产品家族的一部分。很少有数据库能像 MySQL NDB Cluster 那样集如此多热门概念于一身：高可用性、SQL、NoSQL、内存数据库、自动故障处理、分片等等。本书将探讨这些特性如何被整合到 MySQL NDB Cluster 中，以及它们不仅仅是流行词，而是实实在在的功能。

## 本书读者对象

本书面向那些计划部署 MySQL NDB Cluster 或已在生产环境中拥有集群，并希望提升知识以处理日常管理任务和故障排除的数据库管理员。尽管我们力求在写作时假定读者具备尽可能少的预备知识，但至少具备一些 SQL 数据库背景将是一个优势。

## 本书结构

章节分为五个部分，每部分涵盖相关主题。本书从 MySQL NDB Cluster 工作原理的一些背景知识开始，接着介绍安装与配置、日常任务与维护、监控与故障排除，最后以开发与性能调优收尾。

### 第一部分

第一部分包含两章，从技术角度探讨 MySQL NDB Cluster 的工作原理。重点不在于实际任务，但部分理论通过案例研究进行了示例说明。这两章分别是：

*   第 1 章：架构与核心概念。第一章从高层次介绍集群的工作原理，并详细阐述管理节点和 API 节点这两种节点类型的具体细节。
*   第 2 章：数据节点。第二章专门讨论数据节点，数据存储于此，查询也在此执行。

### 第二部分

第二部分解释如何部署 MySQL NDB Cluster，从系统考量开始，然后讨论配置、安装和复制。这四章分别是：

*   第 3 章：系统规划。本章讨论选择硬件时的考量因素，以及如何规划网络分区和可扩展性。其中还包含了多个典型拓扑的示例。
*   第 4 章：配置。本章展示如何配置 MySQL NDB Cluster，并介绍最重要的配置选项。
*   第 5 章：安装。终于到了安装 MySQL NDB Cluster 的时候。本章包含针对 Linux、Microsoft Windows 和 MacOS 的安装指南。
*   第 6 章：复制。复制允许将一个集群的数据复制到另一个集群或一个 `InnoDB` 数据库。本章也包含了 MySQL NDB Cluster 用于两个集群之间主动-主动复制的冲突解决功能。

### 第三部分

集群已启动并运行，因此第三部分转向一系列侧重实践的章节，描述如何执行日常任务和维护。本部分包含七章：

*   第 7 章：NDB 管理客户端及其他 NDB 实用工具。本章讨论 MySQL NDB Cluster 安装中提供的实用工具。重点放在 NDB 管理客户端上，它可用于执行一系列任务，如启动、停止和重启节点，以及创建备份、获取集群状态等。
*   第 8 章：备份与恢复。数据的价值在于其能在灾难发生时恢复。本章深入探讨备份与恢复的方方面面。
*   第 9 章：表维护。本章介绍在线与离线模式变更功能、碎片整理以及索引统计。
*   第 10 章：重启。在一个旨在实现自身高可用性的产品中，在线重启非常重要。本章阐述滚动重启和系统重启的概念，并包含几个需要重启的任务的详细示例。
*   第 11 章：升级与降级。本章讨论并给出如何执行在线和离线升级与降级的实用示例。
*   第 12 章：安全考量。安全性必须是设计决策的首要部分，从初始规划开始，并贯穿集群的整个生命周期。本章介绍 MySQL NDB Cluster 最重要的安全方面。
*   第 13 章：MySQL 集群管理器。本章为 MySQL 企业版产品 MySQL 集群管理器（MCM）提供教程，该产品提供了一种更简便的管理集群的方法。本章包含从初始安装到升级的主要步骤。



### 第四部分

数据库日常工作中一个特殊的部分是监控与故障排除。这正是第四部分的主题。本部分包含四章：

*   第 14 章：监控解决方案与操作系统。本章探讨了监控解决方案，特别是 MySQL 企业监控器，如何用于预防和解决问题，最后讨论了在操作系统层面需要监控的内容。
*   第 15 章：监控数据来源。本章介绍了 MySQL 中用于收集和监控数据的传统来源，包括信息模式（Information Schema）、性能模式（Performance Schema）、`sys`模式、`SHOW`语句以及 MySQL 错误日志。
*   第 16 章：监控 MySQL NDB 集群。MySQL NDB 集群提供了一些额外的监控来源：`ndbinfo`模式以及管理和数据节点上的日志。
*   第 17 章：典型问题与解决方案。无论数据库管理得多么谨慎，总会有出问题的时候。本章介绍了一些适用于 MySQL NDB 集群的通用故障排除技术以及一些典型问题。

### 第五部分

最后一部分转向 MySQL NDB 集群的开发部分。讨论了使用 SQL 和 NoSQL 进行开发以及性能调优。第五部分的三章内容是：

*   第 18 章：使用 SQL 开发 MySQL NDB 集群应用程序。本章介绍了表和索引的创建，以及在开发中使用 SQL 语句时的其他注意事项，例如错误处理。
*   第 19 章：作为 NoSQL 数据库的 MySQL NDB 集群。MySQL NDB 集群支持多种 NoSQL API。本章讨论了 NDB-memcached、C++ NDB API 和 ClusterJ。本章还包含几个代码示例。
*   第 20 章：MySQL NDB 集群与应用程序性能调优。本书的最后一章涵盖了系统级别和 SQL 级别的性能调优。系统级别包括，例如，将数据节点线程绑定到 CPU 和磁盘类型。SQL 级别包括优化连接和分区、混合使用 NoSQL 与 SQL 等等。
