# 设置 Java 堆大小

调整 Java 垃圾回收（GC）是优化 Java 应用程序最重要的单一措施。Cassandra 运行在 JVM 中，因此设置合适的堆大小对性能至关重要。

Java 有多种垃圾回收算法。最常见的是并发标记扫描（CMS）垃圾收集器。然而，在较新版本的 Java 中，G1（垃圾优先）垃圾收集器通常是推荐的垃圾回收算法。

关于选择 Java 垃圾收集器，您需要了解以下几点：

*   CMS 需要更多时间和精力来调优，而 G1 易于配置且大多能自我调优。
*   CMS 更适合固定的工作负载，而 G1 更适合频繁变化、集群运行不同进程的工作负载。
*   对于使用小于 14GB 堆大小且延迟是主要关注点的较小环境，推荐的收集器是 CMS。
*   对于使用 14GB-64GB 之间堆大小的环境，G1 优于 CMS。与 CMS 在垃圾回收期间停止所有应用线程不同，G1 垃圾收集器的线程扫描堆的各个区域，收集器在应用程序线程运行时执行堆压缩。
*   您必须将堆大小设置为运行 Cassandra 实例的服务器总 RAM 的 0.25 到 0.5 之间。
