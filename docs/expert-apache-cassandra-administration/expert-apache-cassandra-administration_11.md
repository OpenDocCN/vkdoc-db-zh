# 禁用交换空间

为避免在低可用内存条件下因交换导致的高延迟，您应完全禁用交换空间。操作方法如下：
```
$ sudo swapoff –all
```

为使此更改在重启后依然生效，您还必须从 `/etc/fstab` 文件中移除与交换文件相关的条目。

默认情况下，大多数 Linux 操作系统的默认 `swappiness` 设置为 60。如果将 `swappiness` 设置为零，Linux 将避免使用磁盘，除非内存用尽；而将其设置为 100 则意味着操作系统会立即将程序交换到磁盘。可以看出，设置为 60 意味着当内存使用量达到操作系统 RAM 分配的大约一半时，操作系统就会开始频繁使用磁盘上的交换文件。如果您将 `swappiness` 调低至 10，那么操作系统只会在 RAM 使用率达到约 90% 时才使用磁盘上的交换文件。

Linux 管理员可以通过在 `/etc/sysctl.conf` 文件中添加以下内容来更改系统的 `swappiness` 值：
```
vm.swappiness=10
```

管理员必须重启服务器才能使新的 `swappiness` 设置生效。关于应将 `swappiness` 级别设置多低，并无固定规则；许多专家建议将其设置为 1。
