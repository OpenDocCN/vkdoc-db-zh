# 第 30 章 ■ 事务日志内部原理

预写日志记录机制保证脏数据页在保存到数据文件之前，其对应的日志记录必须先在事务日志中硬化。然而，反之则不成立。`CHECKPOINT` 进程是异步的，日志记录硬化与数据文件中的页更新之间存在延迟。此外，`CHECKPOINT` 不会分析修改数据页的事务是否实际提交。因此，数据文件中的某些页面反映了未提交事务的更改。

恢复过程的目标是使数据库在事务上保持一致。`SQL Server` 分析事务日志，确保已提交事务的所有更改都保存到数据文件中，并且未提交事务的所有更改都回滚。
