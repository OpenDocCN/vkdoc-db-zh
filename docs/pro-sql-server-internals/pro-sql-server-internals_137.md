# 第 31 章 ■ 备份与恢复

对于使用 `BULK LOGGED` 恢复模型的数据库，日志备份还会存储在最小日志记录的大容量复制操作（例如 `CREATE INDEX`、`ALTER INDEX REBUILD`、`BULK INSERT`、`INSERT INTO`、`INSERT SELECT` 等）期间分配的区的数据。这些区由另一组称为 `大容量更改映射 (BCM)` 的分配映射页跟踪。SQL Server 必须能够访问这些区的数据文件，日志备份才能成功。

与差异备份相反，日志备份是增量的。每个后续日志备份存储从前一个日志备份结束点开始的事务日志部分。在还原过程中，您必须按顺序应用所有日志备份。

日志备份序列包含了自上次完整备份以来 SQL Server 执行的所有操作的日志记录。这允许您重做工作并恢复数据库，即使数据库文件已损坏或不可用。此外，它支持时间点恢复，并允许您将数据库恢复到特定时间。一个有益的情况是发生意外删除数据或数据库对象时。我们将在本章后面讨论这种情况。

■ `重要提示` 如果事务日志包含同时运行的大容量日志操作，则 `BULK LOGGED` 恢复模型中的数据库不支持时间点恢复。

一种特殊的日志备份，称为 `尾日志备份`，在您需要在灾难后恢复数据库时使用。它备份自上次日志备份以来尚未备份的日志记录，并防止在恢复过程中潜在的数据丢失。我们将在本章后面详细讨论尾日志备份。

一个连续的备份序列称为 `备份链`。备份链以完整数据库备份开始，并且是必需的，以便将数据库还原到故障点和/或某个时间点。
