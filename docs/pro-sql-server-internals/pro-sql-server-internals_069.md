# 第 25 章 ■ 查询优化与执行

XML 执行计划将运算符表示为 XML 节点，子运算符节点嵌套在父节点中。这是执行计划最完整的表示形式，包含了在其他模式中被省略的大量属性。然而，XML 执行计划比较复杂，需要一些时间和实践才能变得易于理解。

除了实际执行计划，`SQL Server` 还可以在不运行实际查询的情况下提供 `估计执行计划`。这使你可以快速获取执行计划的形状以供分析。但是，它不包括实际的行数和执行次数，这些信息在查询性能调优过程中非常有帮助。

表 25-1 展示了在图形、文本和 XML 模式下生成估计和实际执行计划的不同命令。

**表 25-1.** 生成实际和估计执行计划的命令

| 计划类型 | 命令 / 菜单项 | 执行查询 | 包含估计的行和执行计数 | 包含实际的行和执行计数 |
|-----------|---------------------|-----------------|-------------------------------------------|----------------------------------------|
| 图形 | 显示估计的执行计划 | 否 | 是 | 否 |
| 图形 | 包含实际执行计划 | 是 | 是 | 是 |
| 文本 | `SET SHOWPLAN_TEXT ON` | 否 | 否 | 否 |
| 文本 | `SET SHOWPLAN_ALL ON` | 否 | 是 | 否 |
| 文本 | `SET STATISTICS PROFILE ON` | 是 | 是 | 是 |
| XML | `SET SHOWPLAN_XML ON` | 否 | 是 | 否 |
| XML | `SET STATISTICS PROFILE XML` | 是 | 是 | 是 |

■ **重要提示** 你应该始终关注执行计划中实际行数和估计行数之间的差异。这两个值之间的巨大差异通常是基数估计错误的迹象，而基数估计错误则源于不准确的统计信息。
