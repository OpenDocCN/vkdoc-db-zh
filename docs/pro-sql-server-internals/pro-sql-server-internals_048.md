# 模型选择与升级考虑

增强功能。最值得注意的是，它能更好地处理不断增长的索引的估算，并更充分地利用多列统计信息中的密度信息。然而，您应当将 130 模型视为对 120 模型的增强，而非一个全新的实现。

作为一般规则，对于新开发，您应始终选择最新的模型。另一方面，升级则更为复杂。即使新的基数估算模型在具有现代工作负载的系统中可能提供更好的结果，但始终存在因生成不同的执行计划而导致性能下降的可能性。不可能构建一个覆盖所有可能工作负载和数据分布的模型，因此在升级后应仔细测试系统。

在 SQL Server 2016 中，您可以使用一个名为 *查询存储* 的新组件，它会在系统中捕获并持久化执行统计信息和计划。这可以极大地简化测试过程，使您能够快速定位性能回退。显然，在测试期间您需要有代表性的负载和数据分布。我们将在第 29 章更深入地讨论查询存储。

当然，没有任何东西阻止您在升级后继续使用旧版基数估算器。然而，您应预计，未来产品中的大部分增强和改进都将属于新模型。旧版基数估算器不会从 SQL Server 中消失；然而，我严重怀疑 Microsoft 是否会继续投入大量时间和资源来改进旧模型。
