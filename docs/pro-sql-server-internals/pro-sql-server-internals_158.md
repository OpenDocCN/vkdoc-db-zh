# 第 32 章 ■ 高可用性技术

在高保护模式下，`主服务器`与`镜像服务器`之间的连接丢失不会导致`主服务器`停止工作。如果`主服务器`宕机，您必须执行手动故障转移，使`镜像服务器`成为新的`主服务器`。但有一个注意事项。如果您执行了手动故障转移，而在某个时刻，`主服务器`在没有连接到前`镜像服务器`的情况下重新上线，它将继续作为`主服务器`运行，这就产生了脑裂情况。

`.Net SQL 客户端`在连接到`主服务器`时会自动获取并缓存`镜像服务器`名称。如果在缓存了`镜像服务器`名称之后发生故障转移，客户端应用程序将能够重新连接到已成为新`主服务器`的`镜像服务器`。但是，如果故障转移发生在缓存`镜像服务器`名称之前，应用程序将无法连接到前`主服务器`，而该服务器现在作为`镜像服务器`运行，并将数据库保持在`RESTORING`状态。应用程序将无法获取关于新`主服务器`的信息，因此无法连接到数据库。

您可以通过在一个额外的连接字符串属性`Failover Partner`中指定`镜像服务器`名称来避免这种情况。仅当`SQL 客户端`无法连接到`主服务器`时，它才会尝试连接到在此属性中指定的服务器。当`主服务器`在线时，`SQL 客户端`会忽略此属性中指定的`镜像服务器`名称，并缓存从`主服务器`检索到的`镜像服务器`名称。

删除数据库镜像时需要小心。只有当`SQL 客户端`运行在连接字符串的`Server Name`属性所指定的服务器上时，它才能在镜像被删除后连接到数据库。如果它运行在被指定为`Failover Partner`的服务器上，您将会得到一个`"Database is not configured for database mirroring"`错误。

数据库镜像故障转移通常比故障转移群集故障转移更快。与重启整个`SQL Server`实例的故障转移群集不同，数据库镜像对单个数据库执行崩溃恢复。然而，故障转移过程的实际持续时间取决于重做队列的大小以及需要重放的日志记录数量。

`PARTNER TIMEOUT`数据库设置控制数据库镜像故障转移检测时间，默认为十秒。您可以使用`ALTER DATABASE SET PARTNER TIMEOUT`命令更改此设置。

```sql
ALTER DATABASE SET PARTNER TIMEOUT
```

如果`主服务器`和`镜像服务器`之间的网络延迟较高，例如服务器位于不同数据中心和/或在云中时，增加此设置是有益的。另一个应该增加此设置的例子是，当您使用一个`SQL Server`故障转移群集实例作为数据库镜像伙伴之一来设置数据库镜像时。群集故障转移过程通常超过十秒，保持默认的`PARTNER TIMEOUT`设置可能会触发不必要的数据库镜像故障转移。您应将`PARTNER FAILOVER`值设置为大于典型的群集故障转移时间。

`镜像服务器`上的数据库保持在`RESTORING`状态；因此，客户端无法访问它。但是，可以在`镜像服务器`上创建一个只读数据库快照，以便您可以出于报告目的访问它。此快照代表了`主服务器`上最后一个`CHECKPOINT`时刻的数据库状态。

**注意：** 数据库快照的涵盖范围超出了本书的范围。您可以在 `http://technet.microsoft.com/en-us/library/ms175158.aspx` 阅读更多关于此主题的内容。
