# 第 25 章 ■ 查询优化与执行

您可能已经注意到，有两种类型的运算符。第一组称为 `non-blocking operators`（非阻塞运算符），它们从子运算符消耗行并立即产生输出。第二组称为 `blocking operators`（阻塞运算符），它们必须先消耗子运算符的所有行，才能产生输出。在我们的例子中，`Sort` 运算符是唯一的阻塞运算符，因为它消耗了 `dbo.Customers` 表的所有行之后才进行排序。另一个常见的阻塞运算符是 `Hash`（哈希），它用于哈希联接和聚合运算，我们将在本章后面讨论。

尽管阻塞运算符完全正常，并且在许多情况下无法避免，但与它们相关存在几个问题。第一个问题是内存使用。每个运算符，无论是阻塞还是非阻塞，都需要一些内存来执行；然而，阻塞运算符在累积和处理行时可能会使用大量内存。该内存称为 `memory grant`（内存授权），需要在查询开始执行前分配给它们。我们将在第[28](http://dx.doi.org/10.1007/978-1-4842-1964-5_28)章“系统故障排除”中详细讨论这个过程。

正确估算内存授权大小非常重要。高估和低估都会导致问题。
