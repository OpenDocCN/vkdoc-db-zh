显然，在某些情况下，字符串属性会成为事实的一部分，应存储在事实表中。然而，在大量情况下，你可以添加另一个维度表，并用一个合成的、基于整数的 ID 键替换事实表中的字符串值，该键引用新表。

你已经在 `ArticleCategory` 数据中看到了一个这样的例子。再举一例，假设事实表需要指定销售的货币。与其在事实表中存储货币代码（`USD`、`EUR`、`GBP` 等），你可以创建一个 `dbo.DimCurrency` 维度表，并用一个 `tinyint` 或 `smallint` 的 `CurrencyID` 列来引用它。这种方法可以显著提高数据仓库环境中针对事实表的查询性能，尤其是在 2016 之前的 SQL Server 版本中。

#### 总结

列存储索引是 SQL Server 2012 引入的企业版功能。与基于每行存储数据的 B-Tree 索引不同，列存储索引以每列为基础存储未排序且压缩的数据。

列存储索引在数据仓库环境中非常有益，因为典型的查询会对事实表的数据进行扫描和聚合，仅选择表列的子集。

列存储索引减少了查询执行期间的 I/O 负载和内存使用。只有查询中引用的列才会被处理。此外，SQL Server 引入了利用列存储索引的批处理模式执行模型。在批处理模式执行中，SQL Server 不是逐行访问数据，而是对一批行执行操作，并尽可能将它们保留在快速的 CPU 缓存中。批处理模式执行可以显著提高查询性能并缩短查询执行时间。

有多种因素可以提高数据仓库数据库系统的效率。你应该努力通过使用合适的数据类型来减小行和列的大小；避免可为空的列；在适当时使用 `CHECK` 和 `UNIQUE` 约束；并尽可能避免在事实表中使用字符串列。
