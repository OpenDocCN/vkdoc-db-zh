# 查询优化器修复与跟踪标志 T4199

查询优化器的复杂性以及产品的庞大客户群带来了可支持性问题。不可能在所有可能的场景和工作负载下测试修复和改进，因此，任何查询优化算法变更都有可能引入性能问题。

随着云采用率的提高，这种情况正在改变。Microsoft Azure 中的 SQL 数据库与盒装版本共享代码库。SQL Server 与其他 Microsoft 产品一样，遵循 *云优先* 模型，其中功能在出现在盒装产品之前很久就被部署到云端。这使得 Microsoft 能够跨数百万个 SQL 数据库测试功能，从各种生产工作负载中收集遥测数据，并在 RTM 版本发布前修复任何问题。所有这些极大地提高了产品质量并减少了产品中的错误数量。

从历史上看，Microsoft 在发布查询优化器的修复方面一直非常谨慎。它们默认是禁用的，必须使用与之关联的各个跟踪标志来启用。另一个跟踪标志 `T4199` 将许多此类修复——推荐在大多数系统中启用的——组合在一个标志下。然而，大多数安装没有启用这些跟踪标志，因此无法从产品中的所有改进中受益。

从 SQL Server 2016 开始，修复分发策略由数据库兼容性级别控制，而不是由跟踪标志控制。将数据库兼容性级别设置为 130 将启用所有查询优化器修复和增强，类似于跟踪标志 `T4199`。而该标志本身，则启用了在 SQL Server 2016 RTM 版本发布后引入的修复，如表 3-4 所示。

表 3-4. SQL Server 2016 中的数据库兼容性级别与修复
```
**兼容性级别** | **TF4199** | **SQL Server 2016 RTM 之前发布的修复** | **SQL Server 2016 RTM 之后发布的修复**
--- | --- | --- | ---
<= 120 | 关闭 | 禁用 | 禁用
<= 120 | 开启 | 启用 | 禁用
130 | 关闭 | 启用 | 禁用
130 | 开启 | 启用 | 启用
```

将产品版本级别设为兼容级别，将启用先前所有 SQL Server 版本中的所有修补程序和增强功能。跟踪标志 `T4199` 将启用当前 SQL Server 版本在 RTM 发布后引入的所有修补程序。

这种行为会导致未来在不同版本的 SQL Server 中，跟踪标志 `T4199` 将控制不同的修补程序集。在完成 SQL Server 2016 升级后，依赖数据库兼容级别而非跟踪标志可能更安全，如果需要，可以通过使用 `LEGACY_CARDINALITY_ESTIMATOR` 数据库设置切换到传统基数估计器。

**注意** 您可以在 [`support.microsoft.com/en-us/kb/974006`](https://support.microsoft.com/en-us/kb/974006) 阅读更多关于查询优化服务模型的信息。

## 第三章 ■ 统计信息

### 概述

正确的基数估计是使查询优化器能够生成高效执行计划的最重要因素之一。基数估计影响索引选择、连接策略和其他参数。

SQL Server 使用统计信息来执行基数估计。统计信息的关键部分是直方图，它存储了最左侧统计信息列的数据分布信息。直方图中的每个步骤包含一个样本统计信息列值以及该直方图步骤中发生情况的信息，例如区间内有多少行、有多少唯一键值等。

SQL Server 为系统中定义的每个索引创建统计信息。此外，您可以在表中的单个列或多列上创建列级统计信息。如果数据库启用了 `自动创建统计信息` 选项，SQL Server 会自动创建列级统计信息。

统计信息有一些限制。直方图中最多存储 200 个步骤（键值区间）。因此，随着表的增长，直方图的步骤覆盖的键值区间会更大。这导致区间内的近似值更大，并且在包含数百万或数十亿行的表上基数估计的准确性降低。此外，直方图仅存储最左侧统计信息列的数据分布信息。除了多列密度外，没有关于统计信息或索引中其他列的信息。

SQL Server 跟踪对统计信息列所做的更改次数。默认情况下，当更改次数超过表中总行数的约 20% 时，SQL Server 会使统计信息过时并进行更新。因此，在大型表上很少自动更新统计信息。您需要根据计划手动考虑更新大型表上的统计信息。

在 SQL Server 2016 中，数据库兼容级别为 130 时，统计信息更新阈值是动态的，并基于表的大小，这使得大型表上的统计信息更加准确。在早期版本的 SQL Server 中，或数据库兼容级别低于 130 时，可以使用跟踪标志 `T2371`。建议在大多数系统中设置此跟踪标志。

您还应该更频繁地更新始终递增或始终递减的索引上的统计信息，因为除非您使用 SQL Server 2014 中引入的新基数估计模型，否则当参数超出直方图范围时，SQL Server 往往会低估行数。

新基数估计模型在 SQL Server 2014 和 2016 中针对兼容级别为 120 或 130 的数据库启用。该模型解决了一些常见问题，例如当统计信息未更新时，对始终递增索引的估计；然而，在某些情况下，它可能会导致计划退化。在升级 SQL Server 后启用新基数估计模型之前，您应该仔细测试现有系统。
