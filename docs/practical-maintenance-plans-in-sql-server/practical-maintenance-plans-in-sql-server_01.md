# 引言

您是否曾想过是什么让您的数据库运行顺畅？我们都希望数据库能像初次安装时那样良好运行，但究竟发生了什么？更重要的是，我们能否控制正在发生的事情？事实证明，我们确实可以控制这些事件，它们是我们用来确保数据库保持良好状态的极其强大的工具。

本书将向您介绍创建维护计划的核心概念，该计划将处理所有必要任务，使您的数据库保持 100% 的功能。您可以在自己的 SQL Server 安装中运行这些练习，通过利用 SQL Server Agent 的功能来确保更高的数据安全性和完整性。只要具备适当的预见和规划，任何经验级别的数据库管理员都能在短时间内自信地使用 SQL Server 中的可用工具创建极其多功能的维护计划。

## 本书的范围是什么？

在本书中，我将尽可能集中使用 SQL Server 2012 Management Studio 界面。我不会涉及太多手写脚本；将尽可能采用点击操作。某些领域我们会涉足未知，但我会尽量减少这种情况。需要明确的是，我将使用维护计划向导，而非维护计划设计界面。虽然有时我会直接关注设计界面，但大部分情况下，我们将使用向导。正如您即将看到的，我在此详述和概述的内容大部分都适用于这两个界面。

我们将创建一个单一的维护计划，该计划将在设定的时间表上每天自动执行。我将向您展示如何操纵数据库引擎，在我们选择的时间表上执行我们选择的维护活动。我还将向您展示如何从数据库的角度扩展报告和日志记录的概念，以便我们了解任何可能出现的问题。最终，我们将得到一个维护计划，它能一劳永逸地完成我们需要它做的一切。这将给我们提供一个助手，可以说，它在我们无需监督的情况下执行这些琐碎但重要的任务。当我想起维护计划时，我立刻会想到罗恩·波佩尔的老广告，其中说“设置它……然后忘记它！”。这正是我们在这里要做的事情。我们将花时间正确设置，并探究原因而不仅仅是方法。然后，我们将确保报告和日志记录设置正确，以便我们了解任何问题。之后，一切将一帆风顺，您可以享受您新近提升的摇滚明星 DBA 地位。



## 本书适合谁阅读？

这本书当然是为你而写的！对大多数读者来说，它应该是一门复习课程。我希望这本书能补充你已有的知识，或许还能为你阐明一些之前不了解的概念。至少，读完本书后，你应该能清楚地了解什么是维护计划、它对数据库有何益处、如何正确构建它、它是如何运行的，以及如何可能使其运行得更高效。我想这应该不是你第一次读到关于维护计划的内容，因为书中假定你对 `SQL Server` 的界面和机制有一定了解。如果你是第一次听说，那也没关系；我尽量保持一切简单明了，让任何人都能跟读并理解……连我 12 岁的女儿都可以！我努力让内容保持“轻松”，意思是我倾向于在写作中加入一些幽默元素以保持趣味性，但同时我也会涉及一些相当沉重的话题，所以有时也会比较严肃。

第 1 章 介绍了维护计划的各个组成部分，并提供了简明的示例，说明如何让维护计划为你工作。第 2 章 至 第 12 章 将详细说明维护计划中可用的每项任务，而 第 13 章 将把所有内容整合起来，让你能够根据特定需求创建一个完整的维护计划。在此过程中，我将审视那些涌现出来的众多“兔子洞”（细节岔路），因为实际上有相当多有趣的小点需要讨论。最终，你将拥有一套完整的工具以及使用这些工具的知识，从而为你的数据库取得卓越成效。

说了这么多，让我们开始这场冒险吧！你可以使用任何安装版本的 `SQL Server`，前提是你已安装 `SSIS`。这意味着需要 `SQL Server 2005` 及之后的任何版本，并且必须是 `Standard`、`Enterprise` 或 `Business Intelligence` 版本。尽管本书是专门针对 `SQL Server 2012` 编写的，但我也在 `SQL Server 2014` 上运行了所有内容，除了界面略有不同外，完全没有遇到任何问题。
