# 引言

自 20 世纪 60 年代和 70 年代计算机诞生以来，信息技术世界已经发生了 rapid 变化。这些变化推动了我们经济的许多不同方面，包括企业进行日常运营的内容和方式。随着组织，特别是内部信息技术的变化，更快、更好的实现业务目标的方法被推动和发展。

随着企业开始更加依赖存储在其系统中的数据，处理和报告数据的更快方法也随之发展。随着时间的推移，组织要求改进处理方式、实现更高的吞吐量并更快地进行报告。这最终导致了能够同时利用软件和硬件资源的系统的开发，从而催生了工程化系统。

在工程化系统（如`Exadata`）发展之后，许多组织面临一个艰难的选择：要么花费巨资购买工程化系统（`Exadata`），要么自己构建。这个决定影响了大量的中小型企业管理。Oracle 认识到了这一点，从而催生了`Oracle Database Appliance`。

## 什么是 Oracle Database Appliance？

从高层次来看，`Oracle Database Appliance`是结合了网络、集群和数据库软件及模板的服务器、存储和网络硬件。`Oracle Database Appliance`是一个完全受支持的、集成了硬件和软件组件的系统。由于它是一个集成系统，因此在软件和硬件层都经过工程化设计，配置和维护简单，并经过预配置以处理数据库工作负载。此外，它的设计旨在帮助组织最小化成本、加快采用速度并降低数据库部署和维护的风险。

## 本书的由来

本书的作者们在信息技术行业工作了多年。在此期间，我们接触并处理了跨越广泛应用和数据库的许多不同平台。然而，在这段时间里，我们没有看到比`Oracle Database Appliance`更能让组织受益的紧凑型工程化系统。本书的想法源于我们许多人在为各种客户实施`Oracle Database Appliances`的时候。当我们在不同环境中实施`Oracle Database Appliance`时，我们会遇到一两个问题，并意识到当时还没有一份关于这个一体机的单一、完整的作品。当然，有 Oracle 的文档，但在某种程度上，查找信息是一个挑战。那时我们意识到，当出现问题时，我们肯定不是唯一遇到它们的人。当我们在一次会议上相遇并开始讨论`Oracle Database Appliance`的问题时，撰写本书的愿望变得更加强烈。我们都认为，业界需要一本关于这个既复杂又简单的工程化系统的书。本书的所有作者都渴望分享我们通过使用`Oracle Database Appliance`所获得的知识。因此，本书的构想就此启动！
