# 引言

《数据库基准测试与压力测试》是为数据库管理员和负责就任何及所有可能最终影响数据库整体性能的架构决策提供建议的专业人士而写的。本书提供了一种经验方法，用以回答关于数据库性能的“如果”问题，帮助数据库管理员在虚拟化服务器和基于容器的解决方案快速变化的格局中做出合理的架构决策。当今的数据库管理员面临众多此类问题：

-   如果我们使用多租户功能整合数据库会怎样？
-   如果我们把数据库服务器虚拟化为 Docker 容器会怎样？
-   如果我们部署最新的 NVMe 闪存磁盘来加速 IO 访问会怎样？
-   压缩、分区和内存 OLTP 等功能是否物有所值？
-   如果我们把数据库迁移到云端会怎样？

作为管理员，你知道答案，或者甚至知道如何测试这些假设吗？你应该知道如何测试并找到这些问题的答案，而这正是本书的内容所在。《数据库基准测试与压力测试》将向你介绍使用行业标准测试套件（如 TCP 系列基准测试）进行数据库基准测试，这些基准测试也正是供应商所依赖的。你将学会运行这些行业标准的基准测试并收集结果，用于回答关于架构变更、技术变更甚至具体到数据库软件品牌对性能影响的问题。你将学会衡量性能并预测环境变更的具体影响。你将了解基准测试的局限性，以及基准测试与工作负载捕获/回放之间的关键区别。

《数据库基准测试与压力测试》是关于为业务和技术决策创建实证依据。它关乎在应该测量的时候不要猜测。实证测试是科学的测试，能提供可衡量的结果。首先对一个可能的架构或技术变更的影响提出假设。然后运行适当的基准测试来收集数据，并预测你正在探索的变更是否会带来益处，以及其量级如何。停止猜测。开始测量。让《数据库基准测试与压力测试》来指明道路。
