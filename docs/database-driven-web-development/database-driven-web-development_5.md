# 5. 理解文档对象模型（DOM）

既然我们已经涵盖了基础知识，现在将深入探讨文档对象模型（简称 DOM）的复杂性。顾名思义，DOM 是访问并可能更改构成 HTML 文档的单个对象属性的过程。

页面中的每个对象都有一组特定的属性与之关联。这些属性相当于页面元素的特性。每一个特性或规则都可以通过简单的 DOM 语句或更深入的 JavaScript 函数来更改或设置。我们暂时将重点放在更简单的 DOM 语句上，积累你的知识，直到 JavaScript 函数对你（读者）来说变得有意义。

随着你学习后续章节，你会发现 DOM 语言的具体实现因浏览器和操作系统平台而异。在本书的这一部分，你会看到主要浏览器的 DOM 语句在其实施上略有不同。每个浏览器的有效 DOM 语句都包含在每个对象、属性、方法和事件处理程序的参考中。

为了缓解主要浏览器各自实现 DOM 所带来的混淆，W3C 正在制定一个通用标准，希望得到主要浏览器厂商的认可。在此之前，我们只能努力应对 DOM 及其相关语言的不同用法和实现。

在 DOM 规范中，有五个独立的组件用于构建一个有效且完整的 DOM 语句。它们如下：

1.  对象
2.  方法
3.  数组
4.  属性
5.  值

虽然你很可能不会在一个语句中使用所有上述组件，但你必须了解并理解每一个的功能。最简单的学习方法就是直接实践。考虑到这一点，请看以下有效的 DOM 语句。尝试找出该语句反映的 HTML 标签和属性。借鉴前面章节的知识，你会发现这非常容易。

```
document.body.alink = "red"
```


## DOM 语句概述

此 DOM 语句会更改或设置整个页面中使用的“活动链接”的颜色。你注意到 `BODY` 标签及其属性 `alink` 了吗？这正是 DOM 的核心所在——反映已成为你网页一部分的现有命令。通过利用这些反射，你可以更改或设置几乎所有页面元素和属性的值，包括 CSS 规则。

前面语句中的 `document` 部分是一个对象，代表整个页面内容——整个“文档”。浏览器在实现 DOM 时，将完整的网页视为一个对象，该对象是其他对象、方法或数组的容器。前面语句中的 `body` 部分是 `document` 对象内的一个对象。`alink` 是 `body` 对象的一个属性，等号后面的 `red` 是属性 `alink` 的一个值。这种层次结构方法是所有 DOM 语句的通用模式。

每个 HTML 标签和每个 CSS 规则都有一个与之关联的对象。对象本身就是在声明属性值时被访问的实体——该属性是对象的值之一。DOM 规范中大约有 120 个对象，以及约 400 个用于描述要分配给对象的操作的属性。还有大约 100 个方法和 16 个数组。事件处理程序的总数约为 40 个。可用的值数量太多，难以计数，轻松可达数千个。

在使用 DOM 规范时，能够区分对象和方法、属性和对象至关重要。识别 DOM 方法很简单——该语句以开闭括号完成。括号内通常嵌套有一个值。数组以开闭方括号完成，而值则被引号包围，作为语句的终止符。通过对象名称通常与其反映功能的 HTML 元素名称完全相同这一事实，来识别语句中的对象。

区分对象和属性的最简单方法是借鉴前面章节的知识。每个页面元素都有一个对象、一个属性和一个属性。也就是说，对象的属性通常与 HTML 中使用的属性名称术语完全相同。

文档对象模型中反映的 CSS 规则和值通过 `style` 对象进行访问——任何跟随在 `style` 对象之后的语句都必须是 CSS 规则的 DOM 等效形式。规则的值放在等号之后，并用引号括起来。

### DOM 语句

使用 DOM，你可以更改颜色、设置边框、创建飞行动画、定义可拖动的屏幕对象，以及几乎任何你能想到的事情。所有 HTML 标签、属性和事件处理程序，以及所有 CSS 规则和值，都可以通过文档对象模型进行访问。使用 DOM 语句，你实际上可以完全不使用 HTML 或 CSS 命令来构建整个文档，但这并不实用，因为 DOM 在带宽方面的成本很高。DOM 本身并非用于构建整个页面——一些 HTML 和 CSS 是必需的。以下示例展示了可能是最简单的有效 DOM 语句，即如何更改文档中链接的颜色：

```
document.link = "red"
```

此 DOM 语句将页面内链接的颜色更改为指定颜色，红色。很简单。值 `"red"` 与 `link` 属性关联，该属性对于此 `document`（一个对象）是有效的。所有 DOM 语句都以一个对象开头，后跟该对象的一个属性。对象的属性后面是值，在本例中是一个颜色。所有对象和方法都通过句点与其属性分隔。所有值都通过等号与其属性分隔，并用双引号括起来。这个约定对所有 DOM 语句都是通用的，毫无例外。正是这种对语法规则的严格遵守，使得 DOM 如此易于使用且功能强大。

应用 DOM 语句的位置与语句本身同样重要。在将 CSS 与 HTML 结合使用时，你已经见过这方面的例子。将颜色属性应用于 `BODY` 元素会导致整个文档呈现该颜色。将相同颜色应用于单个元素或一组页面元素，则只会为这些元素着色。在 DOM 中也是如此，但有一些例外。

在 DOM 中，每个语句都引用页面元素的属性或样式表的规则，这些规则是你页面中对象的属性。以之前将页面内所有链接着色为红色的例子为例。这是一个全局语句，因为它应用于 `document` 对象，而没有指定任何其他对象或属性。你声明 DOM 指令的顺序非常重要，因为它定义了你的值将被应用的位置。你的值和属性的放置是逻辑且有序的。以颜色属性为例。将其与 `background` 属性一起声明将为背景着色。将其与 `link` 属性一起声明将为链接着色。

在使用 DOM 时，你会发现你使用的措辞至关重要。单独的措辞显然告诉了浏览器何时进行何种更改或修改。大多数 DOM 语句以 `document` 关键字开头，这表示后续指令将在当前文档中执行。可以在 `document` 关键字之前声明一个可选的窗口名称，以指示指令将在该名称的不同窗口或框架中执行。

为了使 DOM 在你页面中的实现逻辑有序且易于使用，在构建 DOM 语句时采用了层次结构方法。也就是说，当浏览器读取你的页面时，它会构建一个树，由页面中声明的单个 HTML 页面元素、属性和值组成。元素在树中的位置高于属性，而属性在树中的位置又高于该属性的值。出现在 HTML 文档顶部的页面元素在树中的位置高于其前面的元素。HTML 文档中声明的部分显然必须是有效的，而你指定的 DOM 语句必须符合此树结构。

### 理解文档语句层次结构

在使用对象时，了解并理解你的 DOM 语句将被应用的级联效果非常重要。最基本的 DOM 语句由三部分组成：一个对象及其一个属性，后跟要更改的值。浏览器在读取 HTML 文档时，会构建一个级联对象的层次结构，每个对象都由其在层次树中上层的对象引用。理解这种层次结构方法对于构建有效、功能性的 DOM 语句至关重要。考虑以下示例代码，它在前面的页面中已声明：

```
document.body.alink = "red"
```


## DOM 树结构规则

当浏览器读取这条语句时，它会在为文档构建的层次树中查找。如果找到匹配项，它会应用该值，在本例中是一个红色的活动链接。需要理解的是，DOM 语句必须反映浏览器在读取文档时构建的对象树的组件。有几条简单的规则可以帮助你理解这棵树及其构建方式：

1.  “document” 对象几乎总是位于树的最高层，因为 “document” 包含了网页中的其他一切。
2.  树是按照源代码顺序构建的。也就是说，在文档源代码中位置更高的对象在树中的位置也更高。
3.  HTML 页面元素在树中的位置高于该元素的属性。
4.  HTML 属性在树中的位置高于该属性的值。
5.  诸如 `BODY` 这样包含整个文档渲染部分的 HTML 元素在树中的位置更高。这包括了 `HTML`，如果 `SPAN` 标签包含了整个 `BODY` 部分，它可能也包含在内。
6.  深度嵌套的标签（如表格中的表格）在树中的位置最低。
7.  CSS 规则被认为在树中的位置较低，因为它们是通过 `style` 对象访问的，而该对象至少从树顶往下有四级之深。记住这些规则，考虑以下示例：

```
document.all.elementID.style.bgColor = "blue"
```

阅读这条语句时，你的视线自上而下地穿过了这棵层次树。树的最高层是 “document” 对象，因为 “document” 是树上其他一切内容的容器。第二层是 “all” 对象。接下来是 `elementID`，你将在本章后面的部分学到它。然后是 “style” 对象。其下是 `bgColor` 属性。属性后面是引号中的值 “blue”。你能弄清楚这条语句改变了什么吗？它改变了被分配了 `elementID` 的元素（通过使用 `ID` 或 `CLASS` 属性；别担心，我们很快会讲到）的背景颜色。

这种自上而下处理层次化文档树的方法，就是 DOM 访问浏览器自身构建到树中的对象的方式。深度嵌套的对象通过非常长（有时长到荒谬）的语句来访问。一般规则是，如果语句超过六或七个组件，就该考虑将 HTML 文档重构得更精简一些了。

## ID 和 CLASS 选择器与 DOM 语句

在文档中使用 CSS 样式规则可以大大降低成品的复杂性和大小，同时提供大量效果和用户定制选项。如今几乎所有互联网文档都以某种方式使用 CSS 样式规则。认识到这种广泛的用途，文档对象模型的创建者提供了一种动态更改或设置这些样式规则的方法。几乎每一条已创建的 CSS 规则都有一个对应的 DOM 语句。访问这些 CSS 规则的网关是 Internet Explorer 中实现的 `style` 对象。它使用 HTML 的 `ID` 和 `CLASS` 属性来了解应在何处应用你所需的更改。结合事件处理程序使用，你可以更改或设置文档的每个方面。颜色、字体、大小、对齐、边距，以及几乎所有你能想象到的东西，都可以根据用户操作动态更改。虽然这些概念将在后面的课程中涉及，但起点是学习将操作分配到一个非常具体的位置。

要指定操作放置的位置，必须声明一个唯一的 `ID` 或 `CLASS` 属性。这向浏览器表明，该属性所在的标签在文档中是唯一的。然后可以相应地分配 CSS 规则。

与 HTML 和 CSS 一样，为了将 DOM 语句应用于特定的页面元素，你需要使用 `ID` 或 `CLASS` 元素选择器。选择器作为 DOM 语句的一部分，在对象和对象属性之间声明，如下例所示：

```
document.all.banner.style.bgColor = "red"
```

这段示例代码将值 `red` 分配给了被分配了 `ID` 或 `CLASS` 标识符 `"banner"` 的页面元素。当然，你会使用一个与你正在应用值的页面元素有共同之处的值。例如，将一个颜色值分配给一个唯一明确属性是其边框宽度的元素，这毫无意义。常识在任何时候都适用。

在你继续使用 DOM 规范的过程中，你会经常使用这种为特定 `ID` 或 `CLASS` 标识符赋值的方法。利用前面提到的 `style` 对象可用的各种属性，你将能够访问和更改所有已使用 CSS 规则设置的规则。

## 通过 DOM 访问 CSS 规则

现在你已经了解了在 DOM 中使用 `ID` 和 `CLASS` 选择器的方法，是时候介绍实现了。要访问文档中的 CSS 规则，DOM 提供了 `style` 对象。与 `ID` 或 `CLASS` 选择器一起声明时，`style` 对象是连接更高的 `all` 对象和你希望更改的属性之间的桥梁。在你处理的每个独立文档中，大约有 400 个不同的属性可以更改。可以分配给这些属性的值种类多得令人眼花缭乱。

如你所学，为文本块分配 CSS 规则是带宽和时间的精简高效使用。为了通过 DOM 规范访问 CSS 规则，你需要使用 `style` 对象及其长长的可用属性列表。考虑以下通过文档对象模型访问 CSS 规则所涉及语法的示例：

```
document.all.elementID.style.CSSproperty = "value"
```

通过使用 `document.all` 和 `style` 对象，可以设置或更改级联样式表规范中可用的每条规则和值。使用 `ID` 或 `CLASS` 选择器，你可以为单个页面元素或一组页面元素分配要更改或设置的 CSS 规则。`elementID` 是你分配给该页面元素的 `ID` 或 `CLASS` 选择器名称。`CSSproperty` 是你想要设置或更改的 HTML 属性。

尽管前面语句中的 `style` 部分本身是一个对象，但它也是一个属性。也就是说，它是 `document.all` 对象组的一个属性。随着我们深入使用文档对象模型，这种有点令人困惑的命名法将变得清晰。

你使用前面提到的 DOM 属性指定的值将覆盖任何先前使用 `STYLE` 属性、`STYLE` 标签或来自外部样式表的样式规则分配的 CSS 规则。例如，外部样式表中的以下条目将在文档加载时应用：

```
table1 {color: "lightpink"};
```

然后，通过在 `BODY` 元素中使用 `onLoad` 事件处理程序，可以在文档加载完成后，用以下语句更改 `table1` 容器内文本的颜色：

```
document.all.table1.style.color = "red";
```

你可以看到，外部样式表中声明的初始 CSS 规则被覆盖了。这适用于所有样式规则，无一例外。可以使用任何事件处理程序或事件处理程序的组合，以及任何有效的 DOM 语句。

## 使用 DOM 数组

在 DOM 语言规范中，有大约 16 个数组用于访问你正在处理的 HTML 文档中对象的属性。这些数组有时被称为集合，因为它们收集了正在处理的文档中对象的信息。我们将使用的 16 个数组如下：


### DOM 数组概述

在 DOM 中，文档的各个部分被组织成一系列数组。

1.  `all[]` – 包含页面内所有项目的引用。
2.  `anchors[]` – 包含文档内每个锚点标签实例的引用。
3.  `applets[]` – 包含文档内每个 APPLET 标签实例的引用。
4.  `classes[]` – 包含文档内每个 CLASS 属性实例的引用。
5.  `children[]` – 包含文档内某个父容器每个子元素的引用。
6.  `embeds[]` – 包含文档内每个 EMBED 标签实例的引用。
7.  `filters[]` – 包含文档内每个 CSS FILTER 属性实例的引用。
8.  `forms[]` – 包含文档内每个 FORM 元素的引用。
9.  `frames[]` – 包含文档内每个框架的引用。
10. `ids[]` – 包含文档内每个 ID 属性实例的引用。
11. `images[]` – 包含通过 IMG 标签包含在文档中的每张图片的引用。
12. `links[]` – 包含文档内每个锚点标签实例的引用。
13. `plugins[]` – 包含对文档有效且运行中的每个插件的引用。
14. `scripts[]` – 包含文档内每个 SCRIPT 标签实例的引用。
15. `stylesheets[]` – 包含文档内每个使用了样式规则的实例的引用。
16. `tags[]` – 包含文档内每个有效 HTML 标签的引用。

从上面的数组列表中可以看出，数组的内容可以通过其名称推断出来——例如，`anchors[]` 数组包含文档中由 `<a>` 标签定义的所有锚点。仅使用数组，你就可以访问 HTML 文档中的所有元素。甚至还有一个“`all[]`”数组，它包含了浏览器读取的每一个元素。访问任何数组的内容都相当简单且符合逻辑。

数组中的每个项目都由一个数字表示其在数组中的顺序。把数组想象成一个项目列表，就像购物清单。然后从顶部开始给列表编号。由于大多数编程语言（包括 DOM）将数字 0 视为整数，因此从零开始编号。然后，可以通过引用包含你希望操作的页面项的索引号（index number）来访问数组项。

### 数组索引与访问

既然你知道了数组中的项目是如何排序（索引）的，我们现在可以继续讨论如何实际访问数组的内容。考虑以下示例：

```
window.document.anchors[3].style.color="red"
```

前面的示例显示了将文档中第四个锚点的颜色更改为指定颜色（红色）。`window` 语句是可选的，如果省略，浏览器会假定命令在当前窗口中执行。注意方括号内的数字是三，但它引用的是第四个锚点实例。这是因为数组是从零开始的。也就是说，编号从 0 开始并包含数字 0。你将使用的大多数语言的所有数组都以这种方式工作，所以现在就要习惯它。

在之前的数组列表中，你可以看到，使用 DOM 引擎编译的数组可以是实现所需效果的一种非常有用的方法。数组项可用于根据用户输入动态更改页面的外观。在使用数组时，不仅外观可以被影响——你还可以将数组作为强大的报告工具。考虑 `plugins[]` 数组。它包含浏览器可以使用的所有插件的列表，也就是你的用户已安装到其浏览器的所有插件。

你可以设计一个脚本来检查列表以确定所需插件是否已安装，或者你可以将列表打印到屏幕上。将列表打印到屏幕上相当简单；只需要一个 JavaScript 的 `document.write` 语句，你可以在 JavaScript 语言参考手册和 JavaScript 教程中找到它。

## 总结
在本章中，我们学习了文档对象模型（Document Object Model，简称 DOM 或 DOM）的用法，并将其应用于现实世界的情况。将 DOM 作为页面可执行代码的一部分几乎是必须的。在某种程度上，如果不使用 DOM，几乎没有方法可以呈现任何复杂度的页面。
