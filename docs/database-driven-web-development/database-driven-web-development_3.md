## MySQL 列与索引类型
MySQL 中提供了多种不同的数据类型，您可以定制数据库表以接受它们。从常见的文本字符到二进制信息，都可以提供支持。任何有效的数据类型都可以使用。有效类型例如数字或字符串。

列类型在创建表时声明。有许多不同的类型，我们将在后续章节中逐一介绍。虽然我们在项目中不会用到每一种列类型，但了解和理解可用类型的完整范围至关重要。如果您计划扩展其中一个项目（例如一个功能齐全的公告板系统），这些知识将派上用场。

### 整数列类型
我们从整数列类型开始。整数列类型是一种数字类型的列，仅用于存储数字——如果您尝试插入文本，MySQL 将抛出错误。

整数类型范围非常广泛——可用的有符号数字范围巨大。所谓有符号，是指包含负数表示。请注意，正数的范围比相应的负数范围少一。这是因为数字零仍被视为整数。零是一个值——不应与空值混淆，后者被视为值的缺失。

### 浮点数列类型
浮点数列类型是指包含小数点的列类型。小数点可以从一个位置“浮动”到另一个位置，因此得名浮点数。

当整数列类型的范围对于手头的任务来说不够高或不够低时，通常使用浮点数。需要注意的是，浮点数是近似值——MySQL 会将浮点数四舍五入到您在创建表时定义的小数位数。

### 字符串列类型
字符串类型包括 `CHAR` 或 `VARCHAR`。`CHAR` 代表“字符”，`VARCHAR` 代表“可变字符”。在创建表时会为列设置一个固定长度。如果数据的长度短于声明的 `CHAR` 或 `VARCHAR` 长度，剩余空间将用空格填充。需要注意的是，如果您插入的值本身设计上就带有尾随空格，这些空格将被移除。您必须在 Perl 脚本中处理这种情况，以添加尾随空格。

如果您需要一个二进制、区分大小写的 `CHAR` 或 `VARCHAR` 列，请使用 `BINARY` 标志进行设置。如果存在 `BINARY` 标志，则该列在用于排序或比较时区分大小写。否则，字符串不区分大小写。

### BLOB 列类型
`BLOB` 列类型具有可变宽度。`BLOB` 代表二进制大对象。它们能够存储非常大的数据集，并且不会修剪尾随空格。`BLOB` 列区分大小写。超过最大长度的值将被截断。`BLOB` 类型不能有默认值。

### 枚举或集合列类型
枚举和集合是字符串列，它们具有一组有限的可能值。如果在创建表时未声明任何值，则将插入空字符串。`ENUM` 列类型的最大值为 64 KB（65535 字节），`SET` 列类型的最大值为 64 字节。

要向 `ENUM` 列插入值，请使用字符串字面量，即用引号将字符串括起来。要向 `SET` 列插入多个值，请在开闭引号内用逗号分隔这些值。

### 时间列类型
MySQL 提供了广泛的日期和时间戳类型。时间取自 MySQL 数据库所在服务器的操作系统。

MySQL 能够在 `ORDER BY` 子句中使用时间和日期戳，因此您可以按日期和时间对数据进行排序。如果您希望按日期、时间或两者对表内容进行排序，这个强大的功能就非常方便。

所有日期和时间值都是整数。如果由于某种原因，您的日期中包含浮点数，MySQL 将四舍五入到以下最接近的有效值：月份值限制在 1 到 12。日期值限制在 1 到 31。小时值为 0 到 23，而分钟和秒的值范围是 0 到 59。

无论一项技能的深度如何，复习总是好事，因为您总能获得新的视角。在本节中，我们将回顾将在后续章节开始的项目中使用的特定技能。

从语法上讲，用 Perl 做某件事总有不止一种方法。因此，我将尝试以不止一种方式呈现本节中的每个主题。

我们声明 `$count` 并将其设置为零。然后声明 `foreach` 循环，并使用 `@col1` 作为参数。我们可以使用 `@col1` 作为参数，因为每个数组的长度相同——没有数据会在插入时被遗漏。我们声明了查询并将其存储在 `$query` 中。然后我们使用 `prepare()` 准备查询。我们使用 `execute()` 执行查询，并为循环的下一次迭代递增 `$count`，重新开始该过程并插入更多数据。

请注意，我采用了更纵向的方式来组织查询，将查询的每个元素放在单独的行上。在 Perl 中，您可以将更复杂的代码分解成更易读的形式。这是我的方式。您会找到适合您的方式。另外请注意，为了使这种方法成为可能，您不应使用双引号来定义查询——双引号仅在整个查询位于一行时才有效。将您的查询嵌套在 Perl 的 `qq{}` 语句中，以允许您的查询跨越多行。

### 循环

如你所见，创建表格是一个简单但非常强大的功能。`id`列是整数，也是表格的主键。其他`VARCHAR`列的大小范围在 20 到 250 个字符之间。限制为 20 个字符的列用于存放数字和日期，这些都不会超过 20 个字符长度。`title`列被设置为 250 个字符，因为它是主题的标题。最大字符数（在项目的另一个脚本中定义）为 250 个字符，因为它是表格的描述性列——主题名称最多可达 250 个字符。你会在我们即将处理的项目的`putnewthread.cgi`中看到这个表格创建代码，所以我暂时不再展示更多代码。请专注于 Perl 代码中的表格结构。

我们将在项目中使用的最常见功能是简单的循环。我们将通过使用循环来构造列表，该循环从数据库中选择数据并将其格式化为 HTML 片段。然后，这段 HTML 和来自数据库的数据会被添加到最终结果中，并显示在用户的浏览器中。

项目中使用两种形式的循环：`foreach()`循环和`while()`循环。每种循环的使用方式和位置略有不同，你将在接下来的讨论中看到。

#### while( ) 循环

`while()`循环用于一次选择一个数据。如果只有一个结果，它会被放入一个标量变量中。如果有多个结果，数据会被推入到一个数组中，如代码清单 3-2 所示。

```perl
$query = "SELECT id FROM $thistopic WHERE id >= 0 ";
$sth = $dbh->prepare($query);
$sth->execute;
while ($thisvalue = $sth->fetchrow()) {
push @theseids, $thisvalue;
};
```

`while()`循环会迭代，直到没有更多结果被 MySQL 推入数组。注意，主键`id`正在被选择。你会在接下来的脚本中看到很多这样的操作。

#### foreach( ) 循环

我们主要使用`foreach()`循环来构造最终将显示给用户的 HTML 标记，尽管这不是它唯一的用途。它也可以用于通过之前的数据库调用来遍历表行。使用`id`列作为限制数据库查询的手段是使用`foreach()`循环的好地方，如代码清单 3-3 所示。

```perl
$count = "0";
foreach (@theseids) {
$query = qq{SELECT title, threadid, posts, date
FROM $thistopic
WHERE id = $theseids[$count] LIMIT 30};
$sth = $dbh->prepare($query);
$sth->execute();
while (@thisvalue = $sth->fetchrow_array()) {
push @titles, $thisvalue[0];
push @threadids, $thisvalue[1];
push @posts, $thisvalue[2];
push @date, $thisvalue[3];
}
++$count;
};
```

这段代码片段取自`threads.cgi`。它展示了一个`foreach()`循环，只要`@theseids`中存在值（一个基于`id`主键先前收集的数组），它就会迭代循环。你还可以看到一个`while()`循环正被用来基于`$theseids[$count]`数组从数据库收集数据。我们将在后面的章节中更深入地探讨这种结构的运作方式。目前，这个基本结构已经作为循环方面内容的一个示例。

### 推入数组

所谓“推入”数组，是指将一个值添加到数组的末尾。数组可以比作一个列表，每个列表项都被赋予一个从零开始的索引号。每次你推入数组，你就是在向列表的末尾添加内容。推入数组的语法如下：

```perl
push @thisarray, $thisvariable;
```

标量变量`$thisvariable`被添加到列表的末尾，也就是`@thisarray`。很简单。

我们将在项目中的几乎每个脚本中使用这段代码片段，所以在我们开始详细检查代码之前，你现在理解这个过程非常重要。

### 收集内容

我们将使用的每个脚本都是已格式化为 HTML 标记的独立数据集的集合。在收集数据时，要以易于格式化的方式进行。通常，在收集数据后，会将其放入逻辑有序的 HTML 标记中，这些标记可以是表格内的文本或锚点标签链接，尽管你不仅限于这两种选项。请检查代码清单 3-4 中给出的代码片段。

```perl
$count = "0";
foreach (@these_ids) {
$oneContentElement = qq{
$thislink[$count] };
push @endarray, $oneContentElement;
++$count;
};
```

首先，声明一个`$count`并将其设置为零。然后我们使用`@these_ids`作为参数启动一个`foreach()`循环。新声明的标量变量`$oneContentElement`被填充了一些 HTML 标记。注意，这里使用了三个独立且不同的数组（`@id`、`@id2`和`@thislink`）。每个数组的索引项都被用作`@these_ids`每次迭代的唯一数据。由于`$count`在每次循环迭代时递增，三个数组（三个链接）中的数据随之变化，并被推入`@endarray`。

正是这整个 HTML 标记被推入`@endarray`，并最终显示在用户的浏览器中。此示例仅包含三个链接的代码。在接下来的项目中，我们还将添加图像、链接和其他有用的页面元素。

### 对数组排序：Perl reverse( ) vs. MySQL ASC 或 DESC

在处理数组时，有时反转它并对其应用格式是有利的。有三种可用选项：一种通过 Perl 实现，两种通过 MySQL 实现。

Perl 的`reverse()`函数就是这样一个选项。你需要对你正在处理的每个数组都应用反转。这使 MySQL 退出循环，在已经从数据库检索数据的情况下有时是有利的。使用`reverse()`方法如下：

```perl
@this_new_array = reverse(@this_array);
```

你可以看到使用`reverse()`函数是多么容易。通过 MySQL 使用`ASC`（升序）和`DESC`（降序）子句同样简单。检查以下示例：

```sql
$query = qq{SELECT fieldone, fieldtwo FROM this_table
ORDER BY id ASC};
```

或

```sql
$query = qq{SELECT fieldone, fieldtwo FROM this_table
ORDER BY id DESC};
```

每个 MySQL 语句都使用`id`主键来组织返回的数据——数据实际检索的方向由`ASC`或`DESC`子句决定。

`DESC`以降序方式排列信息。也就是说，表格顶部的数据也是最旧的数据。降序排列返回的数据组织方式是从列表顶部的旧数据到底部的新数据。`ASC`子句将旧数据放在列表底部，新数据留在列表顶部。

这三种数据排序方法各有优缺点。虽然 Perl 的`reverse()`函数很有用，但它是三种方法中最慢的。`ASC`和`DESC`方法更快，因为 MySQL 是用更快的编程语言编写的，针对此类操作进行了优化，并且已经常驻内存。使用哪种方法完全取决于你——根据你的情况量身定制。
