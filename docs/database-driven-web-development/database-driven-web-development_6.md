使用 `id` 属性将 `nameon()` 函数中的 JavaScript 命令分配给该单个页面元素。将作为 `id` 属性值的名称作为参数传递给我们用于实现目标的两个函数。我们使用两个事件来触发两个单独的动作：`onMouseOut` 和 `onMouseOver`。

## 更改页面元素的位置

在我们深入探讨 `style.position` 语句的使用之前，你必须理解该语句的两个选项之间的区别。你通常会将它们与 `ilayer` 元素一起使用，以实现下拉菜单。它们各有优缺点，并且每种都有其特定的使用方式。

### 绝对定位

第一个选项是 `absolute`。绝对定位的文档元素根据浏览器计算出的、相对于文档左上角的位置进行定位。它以像素为单位表示。被分配此定位模式的元素将保持在给定的位置，即使文档的其余部分在浏览器窗口调整大小时发生调整或移动也是如此。

例如，如果你在文档中对一个居中的表格进行了绝对定位，然后用户调整了窗口大小，那么表格中被绝对引用的那部分将保持不动——文档的其余部分将会移动。如果你想使用这种定位技术，出于实际原因，你必须将页面左对齐，`valign=top`。这是因为文档是根据文档的左上角进行绝对定位的。如果浏览器窗口被调整大小，页面不会移动，绝对定位的元素在文档中的引用位置也将保持不变。

### 相对定位

相对定位的页面元素是迄今为止最有用的。它通常被应用为表格中的一个位置。该位置被引用为一个绝对定位的点，但是参照由 `TD` 元素指定的文档区域的左上角。如果页面被调整大小，`TD` 元素将会移动，但附着在该元素左上角的内容将随着 `TD` 元素的调整而调整，从而保持表格结构的完整性。因此，调整浏览器窗口大小将会移动整个表格，包括作为表格内容给出的相对定位部分。

```javascript
function moveleft() {
    document.getElementById('header').style.position="absolute";
    document.getElementById('header').style.left="0";
}
function moveback() {
    document.getElementById('header').style.position="relative";
}
```

# 7. 安装和使用 Perl 服务器

本章内容简短，因为安装和使用 Perl 服务器需要了解的知识就这么多。在大多数情况下，操作都很简单且直观明了。

## Perl 在哪里？

如前一章所述，任何 Perl 脚本的第一行被称为 *shebang*。它是 Perl 可执行文件在硬盘上的位置。在 Linux 操作系统上，通常是 `#!/usr/bin/perl`。

在 Windows 操作系统上，Perl 的位置有点复杂且不太直观。所有可用的 Perl `.msi` 软件包都直接安装到 `c:\Program Files\Perl\bin\perl.exe`。这有一个非常恼人的缺点，即单词 Program 和单词 Files 之间的空格会导致错误，因为脚本和其他程序会被这个讨厌的空格卡住。为了解决这个问题，Perl 的路径应该是 `#!c:/progra~1/perl/bin/perl.exe`。

我建议将 Perl 安装到 C 盘的根目录，如下所示：`C:\Perl\bin\perl.exe`。您必须在安装 Perl 二进制文件时指定此路径。

## 准备您的工作站

与大多数软件安装一样，您需要关闭所有程序，并务必先保存您的工作。所有程序关闭并保存工作后，双击您下载的 `.msi` 软件包开始安装。此 `.msi` 软件包的名称取决于您决定使用的 Perl 版本。

## 安装 Perl 服务器

单击 `.msi` 软件包后，安装将被一个窗口中断，该窗口要求安装程序获得对您的操作系统进行更改并安装程序的许可。如果您拒绝，安装将立即停止。

安装程序的第一个窗口总会有一个徽标屏幕。某些版本的 Perl 将最终用户许可协议放在第一个窗口中。如果协议不在第一个窗口，那它就在第二个窗口。单击“我同意”以接受合同条款。

接下来是一个窗口，询问要安装什么或安装到硬盘的哪个位置。默认情况下，对于 Windows 工作站，此位置是 `C:\Program Files\Perl`，对于 Linux 工作站是 `#!/usr/bin/perl`。

如果第二个窗口不是安装选项，那么第三个窗口将是。我建议您安装所有内容。这包括 Perl 可执行文件和支持库，以及关于如何使用 Perl 服务器的文档。包含文档会使安装时间增加大约两分钟。

## 需要考虑的主题

安装期间需要考虑的主题之一就是 Perl 的位置。我建议您在 Windows 操作系统上的路径为 `C:\Perl`。这与默认路径相悖，但您将来实际上会从中受益。

如果您在安装时使用默认路径，某些程序不喜欢 Program 和 Files 之间的空格。有一个变通方法，但有点不直观。一些不喜欢空格的程序需要路径 `C:\Progra~@\Perl` 才能运行。安装到 `C:\Perl` 允许这些遗留程序找到 Perl，而无需添加额外的工作，例如包含 `PATH` 语句（这是对操作系统的直接更改，超出了本书的范围）。

## Perl 随时待命

Perl 最好的一点是，您无需启动服务器或服务即可使用 Perl 服务器的全部功能。它总是准备好接受工作。每个 Perl 脚本的第一行给出了 Perl 可执行文件的位置。Perl 服务器会以最小的麻烦迅速启动。

Perl 是一种内联语言，意味着执行从脚本顶部开始，向下移动，按照放置在脚本主体中的指令执行。



## 安装 Perl 模块

在 Perl 服务器上，额外的功能来自于 Perl 模块。Perl 模块是一个软件包，用于增强和扩展标准 Perl 安装的功能。最常用的 Perl 模块随安装二进制文件标准提供，并可通过其 `.pm` 扩展名识别。我们将广泛使用的两个 Perl 模块是 `DBD.pm` 和 `DBI.pm`。它们允许 Perl 脚本与数据库服务器（您将在下一章中安装）进行交互。

安装 Perl 模块有两种方法——使用 `CPAN.bat` 和 `PPM.bat`。大多数情况下它们都能正常工作，但由于有数千个 Perl 模块可供选择，您可能需要在 `CPAN` 和 `PPM` 之间切换才能成功安装。也就是说，如果一种方法不起作用，请尝试另一种。

有些模块依赖其他模块来完成其功能。例如，前面提到的 `DBD.pm` 必须在安装 `DBI.pm` 之前安装。如果您遗漏了 `DBD.pm` 而只安装了 `DBI.pm`，将会引发致命错误。大多数情况下，安装程序知道需要安装哪些模块并会自动安装它们，过程非常简便。

# 9. 安装和使用 Apache Web 服务器

Apache Web 服务器的核心功能是提供网页服务。它是一个高度专业化且强大的软件应用程序，每天用于服务数十亿网页。Apache Web 服务器是当今互联网上使用最多的 Web 服务器。

Apache 最基本的功能是提供静态的、不变的 HTML 文档。这具有易于实现的优点——HTML 是一种简单的语言。如果你的网站正如你所设想的那样是一个简单的事务，那么请务必探索这一点。如果你想要由强大的数据库驱动的动态生成的网页，那么请继续阅读。Apache 可以与 Perl 一起使用，根据 Perl 脚本中给出的规则生成网页。这比静态 HTML 方法更复杂，但额外的思考、规划和时间是非常值得的。你努力工作的回报将是一些非常强大且功能齐全的网站，它们会吸引用户的注意力并保持住。

Apache 可以与多种编程语言一起工作，无论是脚本语言还是编译语言。Apache 通过使用这些不同的编程语言提供了广泛的数据处理能力。它是一个多功能且制作精良的产品。

## Apache 和 DBI.pm

Perl 的 `DBI` 模块可能是已投入生产的最佳模块之一。借助它，您能够及时有效地与多个流行的数据库服务器进行交互。`DBI` 提供的方法使得连接和与数据库交互变得简单而有效。以下所有示例均假设您使用的是当前版本的 `MySQL`。

首先，让我们专注于连接到数据库。与所有 Perl 事物一样，有几种不同的方法可以做到这一点，并且都可以是正确的。我们将使用我所见过的连接到 `MySQL` 数据库的最常见语法。

您在使用数据库时遇到的错误原因，与使用系统调用时类似。正如您应该始终检查系统调用的返回码一样，您也应该始终检查数据库调用的返回状态。最简单的方法是将 `DBI` 的 `RaiseError` 属性设置为 `1`。首先，您需要连接到数据库并检查返回值。您将设置 `RaiseError` 属性，如代码清单 9-1 所示。

### 使用占位符

这就是连接到数据库；现在让我们来研究使用占位符而不是字面值。许多人总是建议在数据库调用中使用占位符而不是字面值。这可以加快执行速度，并通常使数据库调用更快、更可靠。请看下面的例子：

```perl
$dbh->do(qq{INSERT INTO $this_table (col1, col2, col3)
VALUES (?, ?, ?)}, undef, '$col1', '$col2', '$col3');
```

我们在一行中插入了三列：`col1`，`col2` 和 `col3`。我们所做的是将占位符用作数据库插入的值，并提供了包含实际要插入数据的变量。这是一种执行数据库调用的简单且非常快速的方法。

### 处理 SELECT 语句

现在让我们专注于 `SELECT` 语句。当你执行一个 `SELECT` 语句时，你显然希望尽快取回数据。最快的方法是使用 `bind_columns()` 和 `fetchrow_array()` 方法。`bind_columns` 将 Perl 变量绑定到 `SELECT` 语句返回的列。例如，如果你有以下 SQL 语句：

```sql
SELECT id, name, postid FROM $thistable
```

你会希望使用 `bind_columns()` 方法将三个变量绑定到相关联的列。因此，你设置好变量，然后使用 `bind_columns` 来绑定它们：

```perl
$sth->bind_columns(undef, \$id, \$name, \$postid);
```

传递给 `bind_columns` 的第一个参数实际上是一个哈希引用，用于指定要与此特定方法关联的 `DBI` 属性。我们不想为此特定方法关联任何属性，所以我们只传递 `undef`，这是完全可以的。在你对语句句柄调用 `execute` 之后，你需要获取数据。为此，请使用 `fetchrow_array()` 方法，该方法获取下一行并返回一个包含字段值的数组引用。但你并不需要使用该数组引用，因为你已经将列值绑定到了 Perl 变量，你可以直接使用它们，如 **列表 9-2** 所示。

```perl
while (@this_data = $sth->fetchrow_array()) {
    $id = $this_data[0];
    $name = $this_data[1];
    $postid = $this_data[2];
}
```
*列表 9-2 选择和检索三个数据库项*

每次你在语句句柄上调用 `fetchrow_array()` 时，绑定变量的值都会被更新。

### mod_perl 与 DBI

现在让我们看看 `mod_perl`。`mod_perl` 的部分目的是让 Perl 程序在 Apache web 服务器上运行得更快。`mod_perl` 通过将 Perl 解释器构建到 web 服务器中并编译你的程序到内存中来实现这一点。因此，当它收到一个新请求时，它会将该请求映射到一个已编译到内存中的程序，从而大大加快速度。如果你打算将 `DBI` 与 `mod_perl` 一起使用，你会希望确保你的数据库事务也很快，因为你不想让数据库成为瓶颈。实现此目标的最简单方法是使用 `Apache::DBI` 模块，它为你的程序提供持久的数据库连接。它通过覆盖 `DBI` 的 `connect` 方法并保持一个开放数据库句柄的缓存来实现这一点。`Apache::DBI` 最好的一点是，你不需要修改你现有的 `DBI` 代码来使用它。只需在你的 `use DBI;` 调用之前添加以下内容：

```perl
use Apache::DBI;
```

你甚至不需要移除调用 `disconnect` 的代码，因为 `Apache::DBI` 会覆盖这些调用并确保句柄实际上不会被断开连接。这种持久连接通过避免为脚本的连续调用而重新连接数据库来加快执行速度——因为一个开放的数据库连接已经存在。

## 启动 Apache 服务器

正确启动你的 Apache 服务器是一项简单但重要的任务。`httpd` 在被调用时做的第一件事是定位并读取配置文件 `httpd.conf`。在此文件中包含了 Apache 服务器启动的位置和设置。你将指定你的根 web 位置是什么，以及其他几十个配置指令。
在启动期间，Apache 服务器解析 `httpd.conf` 文件，并将该文件中的设置和指令应用到该 Apache 服务器实例。你的文件位置可以在当前机器上的任何位置，无论是在 Linux 操作系统还是 Windows 操作系统上。在安装 Apache 之前，默认用户已熟练掌握这两种操作系统中的一种或两种。

### 在 Windows 操作系统上启动

在 Windows 操作系统上安装时，你可以选择从源代码编译服务器的文件或使用预编译的二进制文件。如果你的 Windows 版本高于 Windows 98 并且 `MSI` 安装程序是最新的，你可以下载 `.msi` 包在你的 Windows 机器上安装 Apache。这是推荐的做法，因为它大大简化了安装过程，并确保安装程序能处理和修复可能发生的任何错误。`MSI` 安装程序是一个内部包含了在 Windows 操作系统上安装几乎所有程序规则的程序。

在 Windows 操作系统上启动是一件容易的事情。你只需指定 `Apache.exe` 文件的路径以及一两个开关。然后你给出 `httpd.conf` 文件的位置和 Apache 源文件目录的根目录：

```cmd
C:\Apache\bin\Apache.exe -w -f "C:\Apache\conf\httpd.conf" -d "C:\Apache"
```

首先，我们声明了 `Apache.exe` 可执行文件的路径。然后我们包含了启动开关。接下来，我们包含了 `httpd.conf` 文件的路径以用于启动选项。最后，我们给出了 Apache 安装的根目录位置。

## 停止或重启服务器

为了停止或重启 Apache，你必须向正在运行的 `httpd` 进程发送信号。我们将先检查在 Windows 机器上停止服务器。在大多数情况下，这就像按下 `Ctrl+C` 一样简单，因此我们不会深入探讨在 Windows 机器上停止 Apache 服务器。相反，让我们专注于命令行开关，除了可执行文件的路径外，这些开关在 Linux 和 Windows 操作系统上是相同的。
有许多开关可用于启动、停止或重启服务器。这些开关的完整列表超出了本书的范围。但是，Apache 安装附带的手册解释了所有这些开关以及更多内容——建议你在使用 Apache 固有的任何高级功能之前阅读该手册。

### 停止 Apache 服务器

停止你的 Apache 安装是一件非常容易的事情，但为了你将来能使用 Apache，必须涵盖这一点。以下命令显示了如何在 Linux 系统上停止 Apache 服务器：

```bash
apachectl -k stop
```

此示例假设你位于 Apache 的工作目录内。如果不是，那么你需要使用 `apachectl` 可执行文件的完整路径。
在 Windows 上，有两种选择可以停止已启动并在控制台中运行的 Apache web 服务器。你可以按 `Ctrl+C` 退出，或者转到任务管理器窗口并从那里退出程序。

### 优雅地重启 Apache 服务器



重启服务器有几种方式。你可以使用命令行指令，或者通过操作系统来停止 Apache 进程。然后，你需要按照本文档前面章节给出的流程来重启服务器。请检查以下命令：

```
apachectl -k graceful
```

**优雅重启**会在所有等待中的服务器调用完成后重启服务器。服务器随后会重启并接受新的调用。这样做的好处是在重启前完成所有等待中的服务器调用。通常在修改了 `httpd.conf` 文件中的配置后，你会以这种方式重启。

## 审查运行时配置指令

运行时配置指令是 Apache 用于启动服务器的命令和配置说明。这类指令大约有 200 条，因此全面解释这些指令超出了本书的范围。关于每个运行时配置指令详尽而详细的解释，请参阅 Apache 发行版附带的手册。

运行时指令大致分为九种类型。它们是：服务器配置、虚拟主机、目录、`.htaccess`、核心、MPM（多处理模块）、基础、扩展和实验性。每种运行时配置类型都是 Apache 服务器配置不可或缺的一部分，并在服务器启动时读取。

配置指令在 `httpd.conf` 文件内被分为三个基本部分：

1.  控制 Apache 服务器进程整体操作的指令（“全局环境”）。
2.  定义“主”或“默认”服务器参数的指令，该服务器响应未被虚拟主机处理的请求。这些指令也为所有虚拟主机的设置提供默认值。
3.  虚拟主机的设置，允许将 Web 请求发送到不同的 IP 地址或主机名，并由同一个 Apache 服务器进程处理。

随着服务器的进一步使用以及阅读 Apache 手册，你会逐渐了解每种类型指令的作用。

### 设置服务器根目录

服务器根目录是 Apache 可执行文件和配置文件在本地计算机上的存储位置。

```
ServerRoot "C:/Apache "
```

`ServerRoot` 指令告诉 Apache 服务器网站文档、可执行文件和配置文件的起始位置。

### 设置服务器名称

服务器名称是 Apache 服务器将为其提供文件的域名。你可以指定一个域名或 IP 地址作为服务器名称属性，但在本例中，你需要输入本地环回地址：`ServerName 127.0.0.1`

`ServerName` 指令告诉 Apache 服务器其 URL 应该是什么。如果你将 Apache 用作自己桌面计算机上的开发服务器，请使用“127.0.0.1”作为服务器名称。这是本地环回地址，允许你从本机向本机提供页面。

### 设置文档根目录

文档根目录是将为你的网站提供页面的目录位置。它以文件系统路径的形式给出。以下示例是 Windows 计算机上的文档根目录：

```
DocumentRoot "C:\Apache\htdocs"
```

`DocumentRoot` 指令告诉服务器将在哪里找到要提供的 HTML 文件，并反映了你的文件系统结构。它也用于显示用于 Perl 脚本的 CGI-BIN 的大致位置。

### 设置 CGI-BIN 位置

为了在你的 Apache 安装中使用 Perl，需要在本地计算机上安装 Perl。CGI-BIN 位置显示了你的 Apache 服务器将提供服务的 Perl 脚本的位置：

```
ScriptAlias /cgi-bin/ "C:\apache\htdocs\cgi-bin\"
```

该示例设置了 `ScriptAlias` 位置。这告诉服务器期望找到 CGI 脚本的位置。你的 `ScriptAlias` 不必位于文档根目录或服务器根目录内——你可以将其放在本地计算机上任何支持的操作系统的任何位置。

你可以将 CGI-BIN 的位置和名称指定为非标准形式。Apache 为那些不符合常规的服务器配置提供了此功能。使用这种方式进行配置虽然不是常规做法，但可以作为一种有趣的方式，按照你的规格个性化你的网站。你甚至不必让 CGI 脚本以 `.cgi` 或 `.pl` 扩展名结尾。你可以使用一些描述性的东西来符合你网站的主题。只需运用后续页面中的知识来更改 Apache 指令，以符合你的主题。

你需要做的是修改 `httpd.conf` 文件中的默认设置，该文件是 Apache 在启动时使用的主要配置文件，用于应用你选择的设置。这些设置不必包含默认设置——你可以指定任何你喜欢的内容。我遇到过几个大型网站，它们使用了这样的自定义设置。它们利用了网站主题的优势，为配置文件添加了定制化的风格，这种风格会反映在浏览器的地址栏中。
