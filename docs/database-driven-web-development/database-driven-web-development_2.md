# 2. Perl CGI 与 MySQL 基础

Perl 通用网关接口（`CGI`）模块是一个能够在你的 Perl 脚本中动态生成 HTML 标记代码的模块。它主要被用作捕获页面参数的一种手段。虽然你可以使用 `CGI` 模块中包含的方法来创建整个页面，但这过程冗长且效率不高。使用一个简单的 `print` 语句，将你的 HTML 标记作为打印内容，是 `CGI` 模块更实际的用法。

在本章中，我们将探索即将在项目中使用的 Perl 模块。我们还将使用错误控制技术，并介绍项目中用来捕获错误的方法，以使脚本能继续运行。

接着，我们将探索 Perl 和 MySQL 查询的语法与用法——这些我们最终都会遇到——并重点介绍从数据库检索信息的方法，其间会穿插一些有趣的例外情况。

但首先，让我们快速地进一步定义一下 `CGI`。

## CGI 入门

`CGI` 是一组网络标准的组合，它定义了网络服务器与脚本之间如何交换信息。`CGI` 规范目前由 `NCSA` 维护，该标准将 `CGI` 定义为外部网关程序与 `HTTP` 服务器等网络服务器交互的标准。创建 `CGI` 模块是为了将一些高级编程方法集中到一个紧凑而强大的源中。利用它，你能够控制表单、创建页面重定向，并访问 Perl 环境以获取诸如用户的 `IP` 地址等信息。

通过 `CGI` 模块的方法，你只需几个标准命令就能创建和控制 Perl 脚本的功能。我发现使用 `HTML` 表单和 `CGI.pm` 是一项不错的编程投资，因为你可以选择如何处理通过表单提交的数据。

## 运行 Perl `CGI` 程序

Perl `CGI.pm` 模块需要安装在 Perl 服务器中，才能使其库可供使用。你可以从 `CPAN.org` 下载源文件来安装该模块。互联网是一个广阔的天地。`CPAN` 库包含了数十万个 Perl 模块，以便在这个广阔的天地中工作。

基本上有两种方式可以安装 Perl 模块：`CPAN.bat` 或 Perl 包管理器（`PPM`）。两者都易于使用，只需要与配置 Perl 本身相同的知识。我的意思是，只需简单地设置指向你计算机上资源的路径，例如本例中 Perl 的位置。

### `CPAN.bat`

`CPAN.bat` 是一个传统的基于文本的批处理文件，可在 `Windows` 或 `Linux` 操作系统上运行。它设置简单，只需要 Perl 的位置以及它可能需要用于安装 Perl 模块的其他几个简单程序的位置。

由于它是一个基于文本的程序，一旦你双击 `CPAN.bat`，就会看到一个命令行实用程序框打开。你可以在构成 Perl 的目录中找到 `CPAN.bat`。它与 Perl 可执行文件（如 `perl.exe` 本身）位于同一目录中。

配置好后，你只需输入以下命令即可安装 `CGI` 模块：

```
install CGI
```

`CPAN.bat` 将联系 `CPAN.org` 以获取安装所需的可执行文件。`CPAN.bat` 的一个便利功能是，它还能安装被安装模块可能需要的任何其他 Perl 模块。也就是说，`CPAN.bat` 会安装 `CGI` 模块正常工作所需的其他 Perl 模块。

### Perl 包管理器

Perl 包管理器比 `CPAN.bat` 更易用一些。它有一个 `GUI`，主要用于显示和管理你已安装的所有 Perl 模块。

虽然你可以使用 `CPAN.bat` 查看已安装的模块，但 `PPM` 会显示完整的列表，而 `CPAN.bat` 仅根据你键入的模块名称来验证该 Perl 模块是否已安装。在管理你的 Perl 安装方面，`PPM` 是两者中更好的选择。

使用 `PPM` 安装 Perl 模块是小菜一碟。会显示一个小的命令行。你输入想要安装的 Perl 模块，它就会进行安装。我发现有些模块在 `PPM` 下比在 `CPAN.bat` 下安装得更好。例如，用于处理数据库的 `DBD.pm` 模块通过 `CPAN.bat` 安装时会遇到问题，而使用 `PPM` 安装则几乎不费吹灰之力。`CPAN` 和 `PPM` 的深入使用超出了本书的范围。

## 在你的脚本中包含 `CGI` 模块

为了让一个 Perl 脚本能使用任何已创建的模块，它首先必须用 `use` 指令声明，如下所示：

```
use CGI;
```

这一行必须放在 Perl 脚本的顶部，就在 shebang 行之下。为了开始使用 `CGI` 模块，你必须首先声明该模块的一个新实例：

```
$cgi = new CGI;
```

这会生成 `CGI` 模块的一个新实例，并允许你整体使用 `CGI` 模块中包含的强大方法。



### 使用 `param()` 方法进行管理

`CGI` 模块提供的最常用方法之一就是 `param()` 方法。它用于捕获用户正在交互的 HTML 页面发送到 Perl 脚本的参数：

```
$param1 = $cgi->param('input1');
```

如您所见，使用 `CGI` 模块捕获参数是一件简单的事情。如果您需要捕获多个参数，只需在下一行开始一个新的 `param()` 方法即可，如下所示：

```
$param1 = $cgi->param('input1');
$param2 = $cgi->param('input2');
```

第二个参数的捕获方式与第一个完全相同。两者之间唯一的区别是 Perl 变量名和参数名。很简单。

### 获取调用 IP 地址

正如我们将在本书后面的章节中看到的，有时了解谁在您的网站上做什么是必要的。在后面的章节项目中，您将把照片上传到服务器以显示为您的头像。因为此功能可能受到恶意攻击，例如上传非法或不合适的图片，所以了解是谁在上传照片至关重要。`CGI` 模块已为此情况做好准备，提供了一种检索用户 IP（互联网协议）地址的方法。

互联网上的每台计算机和服务器都有一个唯一的 IP 地址。这是必需的，因为如果有两台计算机具有相同的 IP 地址，互联网上的其他计算机将不知道将信息发送给哪一台。

`CGI` 模块通过以下简单语句提供了这个功能：

```
$userip = $cgi->remote_host();
```

在您的 Perl 脚本中使用这行简单易用的代码，可以告诉您刚刚与脚本交互的人的 IP 地址。您不必将此功能限制在一个上传脚本中。因为它如此易用，您可以在每个正在使用的脚本中记录用户的 IP 地址，并将其存入您的数据库。

#### 进行系统调用

因为 `CGI.pm` 并非包罗万象，您很快会发现需要利用服务器的操作系统来获取某些信息。首先想到的就是日期和时间。`CGI.pm` 能做很多事，但不能进行时间日期戳记。正如您在前一章所学，`MySQL` 允许时间戳记，但那只是记录在数据库中创建的时间。

要进行系统调用，您可以使用 Perl 的反引号 (\`) 操作符。下面展示了如何使用反引号操作符来查找日期：

```
$date = `date`;
```

如您所见，检索确切的日期和时间是一个相当简单直接的操作。因为反引号操作符允许执行任何系统命令，所以也可以通过脚本操作来处理文件和目录。

### 错误处理

在使用 Perl 时，您需要在脚本结构中包含错误处理。所谓错误处理，是指可能出现的不可预见的错误，这些错误可能是您脚本的缺陷，也可能是用户恶意攻击的产物。

在 Perl 中有几种处理错误的方法，我们将在本节中讨论所有这些方法。在处理错误之前，建议您首先设置脚本以正确报告错误。这就是 `CGI::Carp` 模块发挥作用的地方。

### `CGI::Carp`

这个模块对开发人员来说是一个非常宝贵的工具。通过它，Perl 抛出的错误会被回显到浏览器供您在那里阅读。如果您不使用 `CGI::Carp`，您的浏览器将显示“错误 500。发生了关键系统错误”，这并不能告诉您太多信息。每次脚本抛出致命错误时，您都必须查看 Apache 错误日志。这可能非常烦人，并且肯定会大大减慢您的速度。

`CGI::Carp` 模块必须与脚本将使用的其他 Perl 模块一起在脚本顶部声明：

```
use CGI::Carp qw(fatalsToBrowser);
```

它可以在您使用的模块列表中按任意顺序输入。这样使用时，抛出的每个错误都会被发送到浏览器。发生的错误会以通俗的语言报告，同时还会报告错误所在的 Perl 脚本行号和脚本名称。然后您可以确切地知道是什么导致了错误，并找到快速的解决方案。

### 何时何地使用错误处理

错误处理有时有点碰运气。一个致命错误可能会以您没想到会出问题的方式、从脚本的某个部分抛出。在调试脚本时，您会学到在哪里使用错误处理。

我通常会在每个数据库调用上自动包含错误处理。使用 `MySQL` 时，即使是最轻微的错误也是致命的。大多数时候，错误是由用户以您未预料到的方式使用您的脚本引起的。恶意用户也是每个网络开发者的祸害。

有几种方法可以处理脚本将抛出的错误。一种是将 Perl 代码包装在 `eval` 语句中，或者将用户重定向到不同的自定义错误页面。

#### 外部 HTML 内容模板

最终你会达到需要将 HTML 内容打印到浏览器的阶段。有几种方法可以做到这一点，其中最简单的方法是为你页面的每一行 HTML 代码使用一个`print`语句。这并不推荐，因为这是一种低效的打印页面方式。

正如我们将在后续章节中看到的，当我们实际开始显示页面时，有一种高效的方法可以完成向浏览器的多行打印——外部内容模板。

此模板保存将用于显示页面的 HTML 内容。你可以使用任意数量的模板。模板通过一个简单的`require`语句包含在脚本中：

#### 将动态内容打印到浏览器

为了将你生成的和静态的内容打印到浏览器，`content.pl`中的 HTML 标记被封装在一个大的变量`$htmlcontent`中。然后，只需使用两个易于使用的命令，就可以轻松地将动态内容打印到浏览器：

```
print qq{Content-type: text/html\n\n};
print qq{$htmlcontent};
```

第一个`print`语句告诉浏览器，发送给它的数据类型是`text/html`，这是一种更优雅的说法，表示即将发送的是 HTML 标记。第二个`print`语句打印巨大的`$htmlcontent`变量，该变量包含了来自`content.pl`的静态标记以及位于`$topcontent`、`@contentarray`和`$bottomcontent`中的数据库生成内容。

### Perl 与 MySQL 基础

Perl 和 MySQL 协同工作，为操作数据库数据实现了一些非常强大的流程。通过使用 Perl DBI 模块，可以非常及时地存储和检索数据库数据。DBI 模块是一个很好的工具，因为它包含了你可能遇到的几乎所有数据库操作的方法。

我们将探索 Perl 和我们最终都会遇到的 MySQL 查询的语法与用法。在本章中，我们将专注于从数据库检索信息，其中也有一些例外以增加趣味性。

在接下来的示例中，展示了获取数据的每一种可想象的方法。你会看到检索数据片段的方法简单且直截了当。一旦数据被检索出来，你就可以以任何你认为合适的方式对其进行操作。为了简单起见，数据将按照其在数据库中出现的形式原样存在于变量和数组中。

#### 将一个数据库项选择到一个变量中

从数据库中检索一条信息是一个直接的操作，但有一些细微之处需要解决。首先，你需要决定是使用`fetchrow()`方法还是`fetchrow_array()`方法。因为只有一条信息需要检索，所以将使用`fetchrow()`方法。

请注意，完全有可能使用`fetchrow_array()`来获取数据片段——你通常不会这么做，因为那不是它设计的用途。`fetchrow_array()`方法会创建一个数组并将其加载到内存中，而不是一个标量变量。由于我们只从数据库中取出一条信息，不建议你在这种情况下使用`fetchrow_array()`。

#### 使用两步法选择一段数据

使用以下两步方法，我们在`prepare()`方法内定义查询，并通过调用`execute()`执行查询。代替前一个示例中的`$query`变量，我们将文本数据库查询作为参数发送给`prepare()`。然后我们用`fetchrow()`方法捕获数据片段并将其放入`$thisvariable`。

```
$sth = $dbh->prepare(SELECT id FROM this_table WHERE id = 12);
$sth->execute();
$thisvariable = $sth->fetchrow();
```

正如你可能从示例中推断出的，这是一个非常基础的结构。对于简单、干净的代码，你只会将两步方法用于简单的 MySQL 查询。三步方法——在那里你有更多的行空间——更适合我们将在本书后面使用的多行结构。现在我们将通过使用`do()`方法来探索一步方法。

#### 使用一步法选择一段数据

我们使用`do()`方法来定义、准备和执行一个数据库调用，然后用`fetchrow()`方法捕获数据片段，并将其放入`$thisvariable`。如果你的同一脚本中有许多简单的数据库查询，这是一个清理代码的好方法。需要阅读的内容更少，混乱就更少，你的代码总体上更干净。

```
$dbh->do(qq{SELECT id FROM this_table WHERE id = 12});
$thisvariable = $sth->fetchrow();
```


### 将多个数据库项选择到多个变量中

如果你需要快速执行少量工作，通常会使用一步到位的 `do()` 方法。你肯定不想在一个长达百万次迭代的 `foreach` 循环中使用 `do()` 方法。

由于将来会有将大量信息选择并存入多个变量的需求，本章的示例将展示这一点。然而，在继续之前，有几个要点需要考虑。本书的这一部分将深入探讨这些需要记住的要点，并提供反映我们目标功能的示例任务。

你可以使用两个通用函数通过 Perl 的 `DBI` 模块从数据库中检索数据。其中一个是用于从数据库中提取单条数据。这就是 `fetchrow()` 方法。它用于从数据库中检索单条信息，并将其存储于一个标量变量或一个简单的数组中。因为我们将会检索多条信息，所以建议你使用 `fetchrow_array()` 方法来实现你的数据库查询。

`fetchrow_array()` 方法比 `fetchrow()` 方法更高级，使用此方法执行起来非常快速且可靠。与 `fetchrow_array()` 方法配合使用的简单构造既高效又直观，这无疑促成了 Perl `DBI` 模块的广泛流行。

数据库查询本身必须是允许检索多条数据的那种。使用 `WHERE` 子句是实现这一目标最简单、最直接的方式。例如，如果你在 `WHERE` 子句中使用一个确定的值，你将只能检索到一条数据。通过使用设计为提供多个结果的 `WHERE` 子句，脚本就能够从多个列或行中检索多个值，正如示例所示。一个简单的 `>`（大于）运算符就能很好地实现这一点，你将在接下来的示例中看到。

这些示例概述了可用于检索多条信息并将其存储于多个标量变量中的构造。你可以用数组代替这些变量，但不推荐这样做。尽可能保持代码简单，你会得到最好的结果。

和往常一样，检索到的数据应按照其存在于数据库中的形式呈现在你的变量和数组中。这有助于形成清晰简洁的数据库模型和易于理解的脚本。这也是一种非常易于实现的数据库操作方法。

### 将多个数据库项选择到一个数组中

### 从多个数组中选择多个项目

对于不熟悉的人来说，从数据库中选择大量不同的数据可能是一件棘手的事。本书的撰写，部分原因就是为了解答我们所有人关于从数据库中选择多种不同信息的疑问。

首先需要考虑的是你希望数据呈现的形式。也就是说，我们将要使用的数组的排列应完美地反映数据库数据。每一列将被选入一个数组，该列的每一行将成为该数组的一个索引项。

使用多个数组来表示多个列是一个值得遵循的良好编程规范。它使数据库数据易于理解，也更容易操作。

我在清单 2-5 中提供的示例中，数组是`@col1`、`@col2`和`@col3`。它们完美地反映了数据库中存在的原始数据。你可以看到，我们使用了一个简单的数组`push`操作，将后续的数据片段加载到这三个数组中。

通过使用`WHERE`子句，我们能够选择数据库中那些列里的每一条数据。使用`>=`（大于或等于）这样的运算符可以让 MySQL 确切地知道你需要哪些信息。`id`列是一个`AUTO_INCREMENT`（自增）列。

你可以按前面提到的`AUTO_INCREMENT` `id`列对查询进行排序，而无需将`id`列选入变量或数组——MySQL 会为你完成这项工作。你甚至不必选中`id`列；只需在你的查询中引用它，MySQL 就会相应地操作。这样做的好处是保持变量或数组的数量易于管理，并简化了整个脚本。`id`列存在于我们将要使用的每一个假设表中，并且是表的主键。

`AUTO_INCREMENT` `id`列存在于我们将要使用的每个表中，因为它是一种引用表行和表列内容的便捷方式。使用这种方法将确保你在这种 Perl、MySQL 和 Apache 的组合下，拥有最快、功能最强大的查询。请考虑以下示例：

```perl
$query = qq{SELECT col1, col2, col3 FROM $this_table WHERE id >= 0 ORDER BY id DESC};
$sth = $dbh->prepare($query);
$sth->execute();
while (@this_data = $sth->fetchrow_array()) {
    push @col1, $this_data[0];
    push @col2, $this_data[1];
    push @col3, $this_data[2];
};
```

我们构造了一个查询；在本例中，查询针对三个项目：`col1`、`col2`和`col3`。我们使用 DBI.pm 的`prepare()`方法准备查询。这会检查查询的语法是否正确。然后我们使用`execute()`执行查询，这会将查询发送到数据库。接着，我们使用 DBI.pm 的`fetchrow_array()`方法收集数据。数据现在包含在`@col1`、`@col2`和`@col3`中，并且与数据库的列和行完全匹配。

我们使用`fetchrow_array()`是因为我们想从数据库中拉取多条信息。对于`while`语句的每一次迭代，我们将数据收集到`@this_data`数组中。然后，使用一个简单的`push`语句将数据推入`@col1`、`@col2`和`@col3`。接着`while`语句再次循环，一组新的数据被推入`@this_data`，然后又被推入那三个列数组中，这个过程重复进行，直到没有更多数据可以从数据库中检索为止。

关于系统资源需要说几句。使用如此强大的结构可能会导致服务器过载。这是因为在示例中，可能会有大量数据被提取到三个列数组中。关于你的数据，一个很好的惯例是考虑服务器可用的`RAM`（内存）量。在大多数系统上，你可以使用各种程序检查平均可用内存量。我们不会深入探讨这类程序的工作原理。我们现在需要知道的是服务器上可用的`RAM`量。简单来说，就是不要获取超过你拥有的内存量的数据。

### 从一个数组插入多个数据库项

使用数组来插入数据库项有点复杂，但仍然是一项相当简单的任务。只是在使用这种方法时需要考虑几个问题。本节将解释它们。

使用一个在每次循环迭代时递增的`$count`变量，可以根据需要动态插入任意多行。你的限制是你的资源，所以必须谈谈系统资源。因为这种插入表行的方法非常强大，如果使用一个包含数据量超过服务器能够成功且及时处理能力的数组，就有可能使服务器不堪重击。你不应该使用这样的结构来一次性插入数千行表数据，尤其是当每条数据都包含许多千字节信息时。由此产生的操作，虽然仍然是安全的，但可能会让你的服务器过载。

虽然在你的结构中使用`while`循环也可以实现相同的结果，但带有`$count`变量的`foreach`循环的效率和简洁性使其成为更好的选择。使用`foreach`循环要处理的事情更少，因为我们只是遍历数组的索引项并增加计数。`foreach`循环会自动遍历整个数组，并在到达给定参数数组的末尾时停止。

使用`while`语句，你需要执行一个测试，检查`$count`是否与数组中的索引项数量匹配。这会稍微减慢速度。然而，如果你事先知道数组中的项目数量，那么请务必使用`while`语句。请记住，这仅适用于你事先知道要使用的数组长度的情况。

### 从多个数组插入多个数据库条目

当你需要将大量数据从多个数组插入到数据库时，时机将会到来。在本节中，我们将探讨如何将存在于多个数组中的数据插入到你的数据库中。引入更多数组并不是坏事；我们只是开始学习这种编程工具组合的巨大灵活性。

我们将首先定义几个虚构的数组，每个数组长度相同。数据可以是文本/ASCII 或二进制。但请注意，你必须使用正确的列类型来创建列。列类型超出了本节的范围，因此我们将假设我们处理的所有数据都是纯文本，并且被放入 VARCHAR 或 BLOB 列类型中。

将数据从多个数组插入到数据库中的最简单方法是使用 `foreach` 循环遍历这些数组。你会声明一个初始设置为零的 `$count`。这是因为 Perl 中所有的数组都是基于零的。它们按照从零开始的数字系统进行索引。在 Perl 中，零是一个完整的数字（一个整数）。

对于循环的每次迭代，都会引用多个数组。这会导致可能大量的数据被插入到数据库中。请确保将自己限制在 300 或 400 次迭代以内，尽管如果每次插入的数据量相当小并且你的服务器资源充足，可以使用更多次迭代。你可以根据需要插入任意多的行和列——这种结构具有无限的可扩展性。如果需要，请使用 `LIMIT` 声明来限制要创建的数据库行数。

你可以使用 `foreach` 循环或 `while` 循环来实现相同的结果。当你有特定数量的行要插入时，使用 `while` 循环。也就是说，如果你事先知道要插入多少行，那么使用 `while`。如果你要插入的数据库行数量不固定，请使用 `foreach` 循环来遍历你收集到的数组。
