## 日期和时间格式化
有许多不同的 Perl 模块可用于向用户展示日期和时间。这些模块通常非常庞大且运行缓慢，而且老实说，在大多数情况下它们并非必需。这是因为你可以使用反引号运算符（`` ` ``）直接从操作系统获取日期和时间。反引号运算符用于对任何你正在使用的操作系统进行系统调用。

这些模块最终都使用反引号运算符来获取日期和时间，因此你可以通过自己操作来加速。我们将专注于 Linux 的日期和时间，因为它是互联网服务器上最流行的操作系统。

你可能会问，为什么有这么多不同的 Perl 模块仅仅用于格式化日期和时间。答案很简单，因为展示日期和时间的方式有很多种。这就是日期和时间模块的作用——将它们从系统调用获取的日期和时间格式化成更美观的形式。它们之所以如此庞大，是因为它们在一个模块中提供了许多不同的日期和时间展示方式。所有这些复杂性转化为一个运行缓慢的 Perl 模块。

在 Linux 上获取日期和时间的系统调用语法非常简单。查看以下代码片段：

```
$date = `date`;
```

在 Linux 上，`“date”`命令会显示精确到秒的日期和时间。在这个例子中，这些日期和时间存储在`$date`变量中，现在它包含以下内容：

```
Fri Nov 20 11:18:44 PST 2023
```

你可以看到，它给出了日期和时间的完整表示。但这可能不是你希望看到的日期或时间格式——大多数情况下确实如此。我们现在需要做的是将日期和时间格式化成更美观的形式。你可以使用连接运算符（`.`）和`split()`函数非常简单地完成这项工作，如清单 4-1 所示。

```
@date_raw   = split(/ /, $date);
$date_complete = $date_raw[0] . ", " . $date_raw[1] . " " .  $date_raw[2] . ", " . " " . $date_raw[5];
清单 4-1
格式化 $date
```

我刚才做的是使用`split`函数按空格分割`$date`的内容，并将结果推入`@date_raw`数组。然后我创建了`$date_complete`，并将索引 0、1、2 和 5 连接起来，得到了格式化后的日期：

```
Fri, Nov 20, 2023
```

你可以看到我没有使用位于索引项 3 和 4（`$date_raw[3]`和`$date_raw[4]`）的时间和时区。你还可以从连接中看到，我在索引项 0 和 1 之间、2 和 5 之间添加了逗号。

## 网站参数
你最终需要在脚本中包含参数。捕获和使用参数是常见情况，因此你必须完全理解它们的用法。

网站参数是指文件名后问号（`?`）之后出现的字母、数字和特殊字符的组合：
`[`http://www.domain.com/cgi-bin/script.cgi?id=123456&id2=abcdefg`](http://www.domain.com/cgi-bin/script.cgi?id=123456&id2=abcdefg)`
如你所见，给出的完整域名包含两个参数和一个文件名。给出的参数是`id`和`id2`参数。它们各自持有值，并用与号（`&`）分隔。你可以根据需要使用任意多个参数，通常的上限大约是半打（6 个）参数。

这种特定的参数方法被称为 GET 方法。还有另一种提交参数的方法叫做 POST 方法。这些概念在过去的章节中简要介绍过。

为了使用 POST 方法，你必须在 HTML 表单元素内提交参数，如清单 4-2 所示。

```

清单 4-2
使用 method 设置为 POST 的 FORM
```

从前面的示例中可以看到，我将参数插入在 HTML 表单元素的开始和结束标记之间，作为类型为`“hidden”`的`INPUT`元素。当点击提交按钮时，参数将被上传到服务器。
由于使用了 POST 方法，参数将在消息体中传递给服务器，而不是在地址栏的值中。

### 捕获参数
捕获参数最简单的方法是使用`CGI`模块中的`param()`方法。为了调用`param()`方法，你必须首先创建一个`CGI`模块的新实例，如清单 4-3 所示。

```
USE CGI;
$cgi = new CGI;
$id = $cgi->param('id');
$id2 = $cgi->param('id2');
清单 4-3
使用 param() 方法捕获参数
```

你可以看到我首先包含了一个`“USE CGI;”`语句。然后通过使用`“$cgi = new CGI;”`语句创建了一个`CGI`模块的新实例。接着，使用`“$id = $cgi->param('id');”`和`“$id2 = $cgi->param('id2');”`方法捕获了`id`和`id2`参数，它们的值现在存储在`$id`和`$id2`中。需要注意的是，参数名是区分大小写的。

你可以根据需要使用任意多个参数，一个脚本中的限制大约在五到十个之间。需要注意的是，POST 方法可以比 GET 方法传递更多的参数。

### 处理参数
处理参数的方法有很多种。我发现经常使用正则表达式，并可以将其应用于参数变量中包含的数据，以抵御恶意攻击。

一个只允许参数变量中包含数字的正则表达式：

```
$id =~ s/[A-Za-z\s]+//g;
```

如你所见，在这个正则表达式中，变量`$id`中的所有字母都将被去除。这使得参数变量中只包含数字。恶意用户将没有机会入侵你的脚本，因为如果不以某种方式使用文本命令，就不可能进入脚本。

如果安全不是问题，你也可以将参数值直接放入数据库。由于参数有逻辑用途，你可以准备数据供下一个脚本使用，或者将其格式化以访问特定的数据库表，其中参数值作为唯一标识符。考虑以下示例：

```
$profileid = $id . "profile";
```

我再次使用了连接运算符将`$id`和引号中的单词`“profile”`连接起来。注意，单词`“profile”`包含在开始和结束的双引号中。这告诉 Perl 内容本质上是文本。`$profileid`变量现在持有以下值：

```
123456profile
```


由于新变量 `$profileid` 现在是一个唯一值，你可以用它来访问一个唯一的数据库表或表中的一行，在本例中，即该特定用户的网站配置文件。

## 加载链接

总会遇到使用 HTML 表单传递参数行不通的时候。并不总是能使用表单和 `POST` 方法。在这种情况下，你必须想办法通过加载链接的 `GET` 方法来传递数据。这样，你就可以用数据加载你的链接。

正如本章前面一节所提到的，数据可以通过加载的链接在页面间传递：

```
/cgi-bin/script.cgi?id1=123456id2=abcdef
```

链接加载现在对你来说应该非常熟悉了。理解如何加载链接非常重要，因为这是一项几乎用于每个 Perl 脚本的技术。
在以下部分中，我们将进一步拓展你关于加载链接的知识，以便你能够根据来自多个 Perl 数组的数据库条目动态地创建加载的链接。

## 收集信息

从你的数据库中收集所有信息到多个数组中是一个直接的过程，它使用 `fetchrow_array()` 方法。我们要做的是从一个表中收集所有信息。请检查清单 4-4 中给出的结构。

```
$query = qq{SELECT id1, id2, id3, id4 FROM thistable
WHERE id >= 0};
$sth = $dbh->prepare($query);
$sth->execute();
while (@thisvalue = $sth->fetchrow_array()) {
push @id1, $thisvalue[0];
push @id2, $thisvalue[1];
push @id3, $thisvalue[2];
push @id4, $thisvalue[3];
};
Listing 4-4
收集数据以便后续格式化
```

因此，我们从一个表中收集了所有信息，并通过使用 Perl `push` 函数将每列的内容放入其自己的数组中。我们现在在 `@id1`、`@id2`、`@id3` 和 `@id4` 数组中拥有了数据库表的完整表示。继续阅读以了解如何使用它们。

## 构造最终数组

我们将用上一节生成的数组来动态构造加载的链接。我们将使用清单 4-5 中给出的函数，将数据库信息放入一个大数组中。

```
$count = 0;
foreach (@id1) {
$oneContentElement = qq{

Message[$count]

};
push @contentArray, $oneContentElement;
++$count;
};
Listing 4-5
收集最终结果
```

我们首先声明了 `$count`。我们将其值设为 `0`。我们这样做的原因将在接下来的段落中说明。
然后我们使用 `@id1` 作为参数声明了一个 `foreach()` 循环。在这里，它被用作计数来决定 `foreach()` 循环将迭代多少次。
正如你从 `$oneContentElement` 变量内的 HTML 标记中所看到的，每个锚点元素都给出了四个参数。
`$count` 指的是每个数组内的索引项位置。
例如，`@id1` 中的第一个索引项是零，因为我们还没有将 `$count` 从其初始值零迭代。
然后这组链接被 `push` 到 `@contentArray` 上，接着我们迭代了 `$count`。`$count` 现在的值是 `1`。
由于我们现在已经到达构造的末尾，因此它再次循环，这次由于 `$count` 被迭代，所以不同的值被 `push` 到 `@contentArray` 上。
整个构造被循环处理，每次循环时 `$count` 都会被迭代。该构造将循环，直到 `@id1` 数组没有更多的索引项来触发另一次迭代并 `push` 到 `@contentArray` 上。

## 外部内容模板

既然你已经具备了生成和格式化页面内容的必要知识，你现在必须学习如何生成一个完整的 HTML 页面，以便在其中打印 `@contentArray` 的内容。
为此，你必须创建一个新的包含 HTML 标记的 Perl 脚本。将这个标记放在一个用双引号操作符 (`qq{}`) 包围的变量中。我们使用双引号操作符是因为它允许多行代码被封装在其中，就像文本双引号 (`"`) 的情况一样。

此脚本中唯一需要放入的 Perl 代码是一个变量和 shebang，shebang 用于告诉脚本在哪里找到 Perl。以下是一个常见的 shebang 示例：

```
#!/usr/bin/perl
```

我总是在这种情况下使用的变量名是 `$htmlContent`。一个非常简单的外部内容模板示例如清单 4-6 所示。

```
#!/usr/bin/perl
$htmlContent = qq{

- - - Web Site Title - - -

@contentArray

};
1;
Listing 4-6
一个外部内容模板
```

前面展示的标记经过了简化以便于学习，而不至于过于复杂。然而，它仍然是一个有效且完整的 HTML 标记页面。
我们首先从 shebang 开始，然后是 `$htmlContent` 变量，它在 `qq{}` 操作符的开始和结束之间声明。
接着我们继续一些简单的标记，并以一个数字 `1` 结束脚本。这个数字 `1` 是必需的，以便向 Perl 返回一个 `true` 值。省略这个数字会导致抛出致命错误，从而结束脚本的执行。
你可能已经注意到我在开始和结束的 `<table>` 元素之间包含了 `@contentArray` 数组。`<table>` 元素标记与我们在本章前面一节中使用的四个简单链接相契合。
以这种方式声明 `@contentArray` 将允许你拥有更复杂的页面，这些页面利用了在本章前一节详细讨论的加载链接所生成的内容。

为了让 HTML 内容模板与你脚本的其余部分一起使用，只需使用 Perl `require` 语句：

```
require "content.pl";
```

这个系统的关键在于在 Perl 脚本的最末尾声明 `require` 语句。这样，你所有的数据都已经被收集和格式化好，并准备好打印到用户的浏览器。

## 打印最终数组

将你的内容打印到用户的浏览器是简单而直接的。它是脚本中给出的最后一条命令，并将出现在脚本的最后几行：

```
print qq{Content-type: text/html\n\n};
print qq{$htmlcontent};
```

你可以看到，我们首先通过 `Content-type` 命令告诉浏览器将要接收和显示的数据类型。我们使用了 `"text/html\n\n"` 作为浏览器将要接收的内容类型。
现在，一切都整合起来了。从数据库收集的数据被格式化并 `push` 到 `@contentArray` 上。`@contentArray` 然后被放入外部内容模板 `content.pl` 中的适当位置。接着，包含所有标记的变量被打印到浏览器。很简单。

## 使用 CGI::Carp 模块

每个脚本在你完成编写后都会有一个错误。任何复杂度的脚本都至少有一个错误。

虽然 Perl 不能纠正你的错误，但它确实为你提供了一种找出问题所在的方法。为此，`CGI::Carp` 模块是一个宝贵的工具，用于帮助你找到错误。与所有 Perl 模块一样，它在脚本的顶部声明。以下是它的语法：

```
use CGI::Carp qw(fatalsToBrowser);
```



如你所见，声明它是一件很简单的事。`CGI::Carp` 模块会将错误打印到浏览器，而无需使用命令行提示符来调试脚本。这些报告非常直观且易于理解。
这些报告会通知错误信息、脚本名称以及发生致命错误的脚本行号。
通常在网站开发阶段你会使用此模块。在你解决了所有网站错误后，建议你注释掉 `CGI::Carp`，这样用户就无法推断出你的脚本或数据库模型。在活动脚本上保持 `CGI::Carp` 处于激活状态会使你面临恶意用户的攻击。

## 用户名与密码维护

用户名与密码维护是开发网站的重要组成部分。除非你使用安全外壳连接（这超出了本书的范围），否则你通常使用 FTP 将文件放置在它们应该在的位置。
每月更改一次密码会让你安心，知道其他开发者将无法访问你网站的关键文件。使用不同的用户名创建新账户也会增加你网站的安全性。
当涉及到数据库用户名和密码时，更改它们通常是一项大工程，因为除非你更改连接语句中的用户名和密码，否则所有数据库调用都将失败。你将不得不更改每个使用数据库的 Perl 脚本的用户名和密码。
如果你的站点上运行的脚本数量较少，那么更改用户名和密码是一项可行的安全措施。
与其在数据库连接脚本中更改用户名和密码，不如使用一个精心设计的密码。在这种情况下，一个好的密码长度是 10 到 12 个字符。
在密码中使用大小写字母、数字和标点符号，可以降低他人通过你的数据库连接语句获取访问权限的几率。需要注意的是，在 MySQL 上，用户名和密码都是区分大小写的。

## 每用户使用情况统计

在某些网站上，跟踪用户是必需的操作。如果网站是按服务收费的，用户根据其使用的服务付费，通常就是这种情况。

跟踪用户最容易的方式是通过他们的 IP 地址。使用 `CGI` 模块，跟踪 IP（互联网协议）地址变得很容易。考虑以下代码：

```
$userip = $cgi->remote_host();
```

这条语句将返回用户的 IP 地址，允许你随意记录它或对其进行操作。一些网站通过 IP 地址来缩小用户的地址范围，使他们能够记录你的大致位置或在销售宣传中使用它。
跟踪每个用户访问的页面是许多网站的常见功能。通过跟踪用户的活动轨迹，可以了解网站的哪些部分被使用，哪些部分没有被使用。这将使你能够对那些未被充分利用的网站功能做出明智的调整。

跟踪用户需要一两次数据库调用。有两种常用方法来记录脚本（以及你网站的一个页面）是否被访问。第一种是如清单 4-7 所示的直接的两步法。

```
$query = qq{SELECT hits FROM tracker};
$sth = $dbh->prepare($query);
$sth->execute();
$thisHit = $sth->fetchrow()
};
++$thisHit;
$query = qq{UPDATE tracker SET hits = '$thisHit'};
$sth = $dbh->prepare($query);
$sth->execute();
};
```

你可以看到执行了两次数据库调用。首先，使用上面的语句获取点击计数。然后我们递增检索到的值，并将新值更新回数据库。这是记录单个页面点击的一种良好而直接的方法。

单次调用法可能是两种方法中更优雅的一种。通过它，你只需进行一次数据库调用，并让 MySQL 为你完成工作：

如你所见，单次 Perl 调用使得脚本更简洁，同时也让执行速度更快的 MySQL 为你进行递增操作。

## 删除表

删除表可能是一个有风险的操作，特别是因为 MySQL 没有可靠的恢复功能。表一旦被删除，就永远消失了，所以必须格外小心。你通常通过脚本操作来删除表。确保在删除表之前进行了适当的有效性检查。
例如，如果用户正在删除他们的账户，在检查了删除脚本的其余部分已评估完毕并确认需要删除表之后，才应删除该表。然后你就可以成功地删除一个表了。

删除表的正确术语是“丢弃”（dropping）该表，这反映在数据库调用的语法中，如下所示：

```
$query = qq{DROP TABLE username_master};
$sth = $dbh->prepare($query);
$sth->execute();
```

如你所见，删除表很简单且几乎没有冗余操作。另外，请确保你删除的是正确的表，否则会带来麻烦。

## 删除行

删除表中的一行是一种常见做法，通常通过脚本操作完成。删除数据库中的整行可以通过匹配行中的某个值来实现，以此告诉 MySQL 要删除哪一行。
正如我之前所说，务必在每个表中都有一个 `autoincrement id` 列。由于这个 `id` 从零开始并向上递增（因此称为 `autoincrement`），即使在拥有数千条记录的表中，这也是标识要删除的单个行的非常安全的方法。
要在数千行中缩小要删除的行的范围，你会使用 `WHERE` 子句。只需通过 `id` 列匹配要删除的行，就能确保安全地删除表中预期的行。

查看清单 4-8 中删除行的示例，其中要删除的行是通过该行的 `id` 列找到的。

```
$query = qq{DELETE thisrow FROM master WHERE id = 1234};
$sth = $dbh->prepare($query);
$sth->execute();
```

另请注意，我们使用了 `id` 列名来决定删除该行。我发现，在使用任何类型的数据库时，这种方法都非常宝贵。

### 文件管理

通过脚本操作管理文件对新手开发者来说可能令人困惑。虽然有许多 Perl 模块可用于管理文件集合，但你并不一定要使用它们。再次强调，可以使用反引号（`` ` ``）运算符。

反引号运算符允许你直接向服务器的操作系统发出系统调用。大多数互联网服务器运行 Linux 操作系统，因此我们在后续章节中进行的所有数据库调用都将基于此。

在 Web 开发中，每个文件都有两个位置。第一个是 URL，即用户通过浏览器看到的文件位置。第二个是文件在服务器操作系统中的位置。

我们将主要处理后一种文件位置——即文件在服务器上存储的地点。所有 Linux 服务器在处理文件时基本使用相同的命令，因此请放心大胆地使用反引号运算符。

如你所见，表单中需要包含一些通常不会出现的元素。首先是 `ENCTYPE`，在示例中被设置为 `"multipart/form-data"`。这会告知浏览器（然后是服务器），正在上传一个多部分文件。第一部分是 `ENCTYPE`，它将包含文件名以及一个不同的数据容器，用于保存文件本身的数字编码。

表单中改变的下一部分是 `type`，它被设置为 `"file"`。这告诉浏览器显示一个包含“选择”按钮和一个小空间的界面元素，用于显示文件名。

点击“选择”按钮，会出现一个对话框，显示你计算机的文件系统。导航到所需文件，单击一次选中它，然后点击确定按钮。

选中的文件名将显示在按钮旁边。点击提交按钮，你的文件将被上传到表单中指定的脚本。

根据你使用的浏览器，你应该会在某处看到一个进度指示器，以文件总大小的百分比形式显示已上传的文件量以及仍需上传的剩余量。
