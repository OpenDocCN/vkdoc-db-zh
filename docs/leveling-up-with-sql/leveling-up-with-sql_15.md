# 总结

你的大量工作不仅会涉及真实表，还会涉及生成的虚拟表。虚拟表包括：

*   一个连接
*   一个公用表表达式或一个表子查询
*   一个视图，或在某些情况下，一个表值函数
*   一个临时表。

### 视图

视图是一个已保存的 `SELECT` 语句。你可以将其设置得尽可能复杂，然后像虚拟表一样获取它。

视图的优点包括：

*   它们可以是处理数据的一种便捷方式。
*   它们可以充当数据的接口，特别是在原始或修改后的形式不符合你的要求时。
*   当从外部应用程序访问时，它们为复杂数据提供了简单的表格视图。

### 表值函数

表值函数是一种返回虚拟表的函数。对表值函数的支持情况不一。在支持的地方，它允许你向查询中输入参数，使其比简单视图更加灵活。

### 临时表

有时，保存结果比每次都重新生成它们更好。你可以将它们保存到一个缓存表中。

优点包括：

*   无需重新计算本就相同的结果，这更高效。
*   你可能希望使用某个时间点的数据快照进行工作。

如果你的缓存计划非常短暂，你可能会使用临时表。临时表是一种会在会话结束时自动销毁的表。

无论缓存表是临时的还是永久的，你都可以使用 `SELECT` 语句将数据复制到其中。你也可以在单条语句中创建一个新表并将数据复制进去。

#### 计算列

在现代数据库管理系统中，你可以在表中创建虚拟列来给出计算结果。

`VIRTUAL` 计算列每次从表中获取数据时都会重新生成值。`STORED` 计算列（在 MSSQL 中也称为 `PERSISTED`）会缓存结果，直到其他数据发生更改。

计算列可以用于提供便利。如果它是 `STORED` 列，它还具有节省处理开销的好处。

## 即将介绍

一条 `SELECT` 语句不一定就是故事的终点。在某些情况下，它可以是一个更复杂过程中的一步。

一个 `子查询` 允许你将一条 `SELECT` 语句嵌入到另一个查询中。这可以用于从其他表中获取值，或者使用一个表来过滤另一个表。如果你想在另一个查询中纳入聚合数据，这尤其方便。
