# 总结

在本章中，我们仔细研究了如何使用**公共表表达式**。公共表表达式生成一个虚拟表，您可以稍后在主查询中使用它。过去，您会在 `FROM` 子句中使用子查询来应付。

您会使用 CTE 或 `FROM` 子查询的原因是，您可能需要准备数据，但又不想麻烦地将其保存在视图或临时表中。CTE 比临时表更短暂，因为它们根本不会被保存。

与 `FROM` 子查询相比，CTE 有一些优势：

*   您在 CTE 使用前就定义它，使查询更易读、更易于管理。
*   您可以简单地链接多个相互依赖或独立的 CTE。如果您想使用 `FROM` 子查询来实现这一点，就必须嵌套它们，这很快就会变得难以处理。
*   CTE 可以是递归的，因此您可以用它们来迭代数据。

### 简单的 CTE

CTE 最简单的用途是为后续处理准备数据。一些用途包括：

*   定义一组常量值，可以是字面值或计算值。
*   准备聚合数据，以便与非聚合查询结合使用。

### 参数名称

CTE 期望为每一列提供一个名称或别名。您可以在 CTE 内部定义这些名称，也可以将它们作为 CTE 定义的一部分来定义。

### 多个 CTE

一些查询涉及多个步骤。这些步骤可以通过链接多个 CTE 来实现。

### 递归 CTE

递归 CTE 是指引用自身的 CTE。它可以用于迭代一组数据。

递归 CTE 的一些用途包括：

*   生成一系列值。
*   通过自连接遍历层次结构。
*   将字符串拆分为更小的部分。

## 后续内容

到目前为止，我们已经处理了多个重要的核心概念。在下一章中，我们将了解一些额外的技术，您可以用它们来更智能地处理数据库：

*   **触发器 (Triggers)** 允许您在数据发生某些变化时自动执行流程。
*   **数据透视表 (Pivot tables)** 本质上是二维聚合查询。
*   **变量 (Variables)** 允许您在操作过于复杂时保存中间值。
