# 引言

在 20 世纪 70 年代初，一种基于 `E. F. Codd` 原创工作的数据库管理新设计正在开发中。该模型被称为关系模型，描述了一种基于数学原理收集数据以及访问和操作数据的方式。

在接下来的十年里，`SQL` 语言被开发出来。尽管它并未完全遵循关系模型，但它试图使用一种简单的语言使数据库易于访问。

`SQL` 语言多年来不断改进、增强和发展，并在 20 世纪 80 年代末发展成为 `ANSI`（美国国家标准学会）和 `ISO`（国际标准化组织，没错，它并不拼写为 `ISO`）的标准。

这段非常简短的历史带给我们的启示是：

*   `SQL` 已存在相当长一段时间。
*   `SQL` 基于一些坚实的数学原理。
*   存在一个官方标准，尽管没有人完全严格遵守它。
*   `SQL` 是一门不断发展的语言，新功能和新技术一直在增加。

第三点的后半部分值得强调。没有人完全遵守 `SQL` 标准。这背后有很多原因，有些合理，有些不合理。但你可能会发现各种 `SQL` 方言大约有 80-90% 是兼容的，其余的部分我们会在学习过程中为你补充说明。

在本书中，你将学习如何使用 `SQL`，达到超越基础的水平。你将了解到一些 `SQL` 中的新特性；还有一些是你可能不知道的旧特性。我们将探讨一些非标准特性，也会研究如何以更强大的方式使用你已知的功能。

本书不适合纯新手——我们假设你具备一定的 `SQL` 知识和经验。如果你是纯新手，那么阅读我的前一本书《*SQL 与数据库入门*》^(1)会让你收获更多；之后，你可以带着扎实的基础，满怀信心和热情地回来阅读本书。

如果你具备相应的知识和经验，第一章将为你快速概述你应该掌握的知识类型。

### 示例数据库

要完成练习，你需要准备以下内容：

*   一个数据库服务器和一个合适的数据库客户端。
*   在数据库上拥有任意操作的权限。如果你是在本地安装的软件，你很可能拥有所需的所有权限，但如果你是在别人的系统上操作，则需要确认权限。
*   用于生成示例数据库的脚本。

第一章将详细介绍如何准备好你的 `DBMS` 软件和示例数据库。同时也会概述示例数据库背后的故事。

## 说明

在你编写 `SQL` 操作数据时，在另一端有软件响应这些 `SQL` 语句。该软件通常被称为数据库服务器，更具体地，是它的朋友们所说的数据库管理系统，或称 `DBMS`。我们将在全书中使用这个术语。

我们将涵盖的 `DBMS` 包括 `PostgreSQL`、`MariaDB`、`MySQL`、`Microsoft SQL Server`、`SQLite` 和 `Oracle`。我们假设你使用的是这些 `DBMS` 相对较新的版本。

第 1 章 将详细介绍如何设置你的 `DBMS`，以及如何下载和安装示例数据库。
