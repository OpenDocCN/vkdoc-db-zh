# 索引

由于表不是按任何特定顺序存储的，因此搜索起来可能很耗时。可以为你经常搜索的任何列添加可选的索引，这会使搜索快得多。

## 数据操作

数据操作语句用于添加或更改数据。除了 `SELECT` 语句外，还有

*   `INSERT`：向表中添加新行
*   `UPDATE`：更改表中一行或多行的数据
*   `DELETE`：删除表中一行或多行

与 `SELECT` 一样，`UPDATE` 和 `DELETE` 语句可以用 `WHERE` 子句限定，以确定将影响哪些行。

与 `SELECT` 不同，它们有可能搞乱数据库，尤其是因为 SQL 没有撤销功能。


## 集合操作

在 SQL 中，表是行的数学集合。这意味着它们不包含重复项且是无序的。这也意味着你可以使用集合操作来组合表和虚拟表。

主要有三种集合操作：

*   `UNION` 组合两个或多个表，并返回所有行，其中任何重复项会被过滤掉。若要保留重复项，需使用 `UNION ALL` 子句。

*   `INTERSECT` 仅返回出现在所有参与表中的行。

*   `EXCEPT`（在 Oracle 中也称为 `MINUS`）返回第一个表中存在，但第二个表中不存在的行。

应用集合操作时，关于每个 `SELECT` 语句中的列有一些规则：

*   列在数量和类型上必须匹配。

*   仅使用第一个 SELECT 的名称和别名。

*   仅匹配值，这意味着如果你的各个 `SELECT` 改变了列顺序或选择了不同的列，只要它们是兼容的，它们就会被匹配。

一个 `SELECT` 可以包含任何标准子句，例如 `WHERE` 和 `GROUP BY`，但不能包含 `ORDER BY` 子句。不过，你可以在最后使用 `ORDER BY` 对最终结果进行排序。

集合操作也可用于特殊技术，例如创建示例数据、比较结果集以及组合聚合。
