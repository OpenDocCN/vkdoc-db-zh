# 总结

SQL 表中的数据应该以其最纯粹、最简单的形式存储。然而，这些数据可以重新计算以提高其可用性。

计算可以采取多种形式：

*   基于单列
*   基于多列
*   硬编码的字面值
*   子查询的结果
*   由函数计算得出

计算也可以用于 `WHERE` 和 `ORDER BY` 子句。

### 别名

所有计算值都应该使用别名重命名。`AS` 关键字是可选的，但建议使用以减少混淆。

如果新名称更合理，你也可以为非计算列指定别名。

别名在 `SELECT` 子句中给出，该子句在 `ORDER BY` 之前最后求值。对于大多数 DBMS，这意味着你不能在 `ORDER BY` 之外的任何其他子句中使用该别名。

### NULL 值

表当然可能在不同的地方包含 `NULL`。通常，`NULL` 会使任何计算失效，并在结果中留下 `NULL`。

你可以使用 `coalesce()` 函数绕过 `NULL`，该函数用一个替代值替换 `NULL`。你也可以使用 `CASE ... END` 表达式。

### 类型转换

SQL 主要处理三种数据类型：

*   数字
*   日期和时间
*   字符串

你可能需要改变数据类型。这是通过 `cast()` 函数完成的。

当在主要类型内进行转换时，效果是改变类型的精度或大小。

当在主要类型之间转换时，通常是为了兼容性。虽然转换为字符串通常是可能的且经常是自动的，但从字符串转换可能并不总是成功。不同的 DBMS 对不成功的转换有不同的反应。

### 数字计算

你可以对所有数字类型执行基本算术。不同的 DBMS 对处理整数有不同的态度。

SQL 包含多种处理数字的函数，包括：

*   数学函数
*   近似函数

还有一些格式化函数，它们生成格式化后的字符串结果。

### 字符串计算

字符串可能以各种方式存储。通常，字符串使用 ASCII 或 Unicode。字符串操作可能区分大小写，也可能不区分，这取决于数据库的排序规则（collation）。

字符串的基本简单操作是连接（concatenation）。通常有一个简单的运算符来完成此操作。

其他字符串操作涉及字符串函数。

### 日期计算

请注意，SQLite 没有日期数据类型。它确实包含一些将字符串或数字转换为日期的函数。

对于日期，以下操作很常见：

*   输入并存储日期和时间
*   获取当前日期和时间
*   提取日期或时间的一部分
*   格式化日期
*   一些简单的算术，例如日期和时间之间的差值，以及修改日期和时间

### CASE 表达式

`CASE` 表达式允许你从多个备选值中进行选择。`CASE` 表达式可用于简化值，也可用于对它们进行分组。

`CASE` 表达式有一种简单形式，可用于离散值。

`CASE` 表达式也可以嵌套以形成更复杂的表达式。
