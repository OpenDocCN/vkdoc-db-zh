# 总结

在本章中，我们研究了在查询中使用子查询的各种变体。我们在前面的章节中已经看到过一些子查询，但在这里我们更仔细地研究了它们的工作原理。

子查询可以在任何子句中使用。子查询的结果必须匹配该子句的上下文：

*   `SELECT`子句或简单`WHERE`表达式中的子查询需要返回单个值。
*   在`IN()`表达式中使用的子查询需要返回单个列。
*   `FROM`子句中使用的子查询需要返回一个虚拟表。

你也可以在`ORDER BY`子句中使用子查询，尽管你可能更希望在`SELECT`子句中使用该表达式。

你还可以将子查询与`WHERE EXISTS`表达式或`LATERAL`连接一起使用。

`FROM`子句中的子查询可以嵌套，尽管你可能更希望使用公用表表达式。

### 关联与非关联子查询

子查询可以是关联的或非关联的：

*   非关联子查询独立于主查询，并被计算一次。然后结果用于主查询。
*   关联子查询是引用主查询中数据的那种。它为每一行计算一次。

关联子查询可能代价高昂，因为它会被计算多次，因此可能存在更合适的替代方案。

### WHERE EXISTS 表达式

`WHERE EXISTS`测试子查询是否返回任何行；如果返回，则保留主查询中的行；否则，将其过滤掉。

在`WHERE EXISTS`中使用非关联子查询通常会返回主查询中的所有行或不返回任何行。使用关联子查询将过滤选择性行。

你通常可以使用`IN()`表达式作为替代方案，但有时`WHERE EXISTS`允许使用更复杂的子查询。它也绕过了`NOT IN`的怪癖。

### LATERAL 连接（亦称 CROSS APPLY）

`LATERAL JOIN`（在某些 DBMS 中为`CROSS APPLY`）允许你向主查询中的每一行添加列：

*   你可以使用`LATERAL JOIN`来添加计算列，并且可以链接它们，因此可以有多个计算和中间值。
*   你也可以使用`LATERAL JOIN`从子查询中添加多个列。

### 公用表表达式

子查询可以在`FROM`子句中使用，但它可能很复杂，特别是在需要多个子查询的情况下，这些子查询必须嵌套。

公用表表达式允许你在主查询之前定义一个虚拟表，使主查询能够像普通查询一样工作。

你也可以将多个 CTE 链接在一起，这比嵌套子查询更容易处理。

## 后续内容

在第 5 章中，我们研究了聚合数据。通常，聚合值不能与非聚合值混合，除非在其中放入几个子查询。

**窗口函数**是一组函数，其作用是将子查询应用于每一行。窗口函数主要有两大类：

*   聚合函数可用于将聚合应用于非聚合查询的每一行。它们也可用于累积或分组聚合。
*   排序函数可用于根据行在数据集中的位置生成值。它们可用于指示行位置或某些分组。它们也可用于从其他行获取值。

使用窗口函数，你将能够生成结合了纯数据与更多分析数据的数据集。
