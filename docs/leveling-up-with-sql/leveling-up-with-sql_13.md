# 总结

在本章中，我们了解了如何聚合数据集。

### 基本聚合函数

基本聚合函数有：

*   `count()`，计算列中的值数或表中的行数。
*   `sum` 和 `avg`，对列中的数字进行求和或求平均值。
*   `max` 和 `min`，查找任何类型列的最小值和最大值。实际上，它们是在按该列排序时查找第一个和最后一个值。
*   `stddev`, `stddev_samp`, `stddev_pop` (PostgreSQL, MySQL/MariaDB, Oracle) `stdev`, `stdevp` (MSSQL)。计算一列数字的总体或样本标准差；我们假设数据是正态分布的。

`sum`、`avg` 和标准差函数只能应用于数字数据。



### NULL 值

所有聚合函数都会跳过 `NULL`。这在计数值时尤其重要，计算平均值时也是如此。

`NULL` 会被跳过这一事实，也可以在计算选择性聚合时加以利用。

### 聚合过程

你聚合的数据不一定来自已保存的表。你也可以聚合由连接、公用表表达式和视图生成的虚拟表。

聚合数据时，数据可能先被 `WHERE` 子句过滤。聚合是基于过滤后剩余的数据进行的。

执行聚合操作实质上是将原始数据转换成一个或多个汇总行的虚拟表。这不再包含原始数据。你只能选择虚拟汇总表中存在的内容。因此，你无法在单个 `SELECT` 子句中混合聚合数据和原始数据。

如果你同时需要原始数据和汇总数据，你可能需要将原始表与包含汇总数据的 CTE 进行连接。

### 聚合过滤器

可以过滤用于单个聚合函数的数据。

有一个标准的 `FILTER (WHERE ...)` 子句，允许你对列进行过滤。然而，它（目前）尚未得到广泛支持。

过滤数据的常用方法是在要聚合的内容上使用 `CASE ... END` 表达式。为你想要的值设置一个值，例如 `1`，让其余的值默认为 `NULL`，并让聚合函数在后续处理中忽略它们。

你也可以在 `DISTINCT`（不同的）值上进行聚合。这在计数时最有意义。

### GROUP BY

`GROUP BY` 子句可用于生成一个组汇总的虚拟表。

在某些 DBMS 中，你可以使用 `GROUP BY ()` 来生成总计。这是没有 `GROUP BY ()` 子句时的默认行为，并且每当 SQL 看到聚合函数时都会自动完成。它从来都不是真正必需的。

你可以按基本值分组，也可以按计算值分组。

按计算值分组可能会变得复杂，因为 `SELECT` 和 `ORDER BY` 子句只能使用 `GROUP BY` 子句中的内容。由于子句的顺序，你可能会发现自己在各个子句中重复相同的计算。

由于 `SELECT` 子句只在最后阶段进行求值，并且选择和排序只能基于 `GROUP BY` 子句中的内容，你可能会发现以下技术很有帮助：

*   使用冗余分组来选择一项并按另一项排序。
*   将聚合查询放在一个 CTE 中，并将其与其他表连接以获取其余结果。

按列分组时，结果可能不会按正确顺序排列。由于组名都是字符串，按组名排序只会将它们按字母顺序排列，这并不总是合适的。但是，你也可以根据它们在另一个字符串中的位置来排序，这个字符串可以按你喜欢的任何顺序排列。

### 混合小计

总的来说，聚合查询产生的是单一层级的简单聚合。有时，你需要将它们与不同级别的小计结合起来。

你可以在单独的查询中生成小计，然后用 `UNION` 将它们组合起来。你可能需要一些额外的工作才能让结果按你希望的顺序排列。

大多数 DBMS 包含小计操作来自动创建组合结果。它们可能包括 `GROUPING SETS`、`ROLLUP` 或 `CUBE`。大多数包含 `ROLLUP`，这是最常见的变体。还有其他的分组函数来辅助排序和标记。

### 统计

一般来说，聚合函数本质上是统计性的。虽然 SQL 不像专门的统计软件那样强大，但你可以使用聚合和分组来生成一些基本的统计数据。
