# 概述

Oracle 身份与访问管理满足了企业内对高度安全、可审计且可靠的身份管理的需求。各组件协同工作，形成了一种平台化的身份管理方法，从而降低了在许多场景下的复杂性——这些场景可能随着时间推移和需求增长，原本需要集成多家供应商和多个产品。图 1-8 展示了 Oracle 身份与访问管理套件各组件之间如何共享数据。考虑到许多组织是按需构建安全系统，Oracle 确保了身份与访问组件是模块化的。这允许公司先安装所需的基础目录服务，之后再添加访问管理、单点登录（SSO）和治理能力。

![A352855_1_En_1_Fig8_HTML.jpg](img/A352855_1_En_1_Fig8_HTML.jpg)

图 1-8. Oracle 身份与访问管理技术

如图 1-8 所示，Oracle 身份与访问管理套件提供了处理组织所有安全相关需求所需的服务。访问管理提供了诸如`Access Manager`、`Identity Federation`和`Secure Token Services`等功能，通过允许应用程序共享会话信息并提供单点登录（SSO）能力，从而提高用户生产力和应用程序可访问性，减少了用户必须输入其凭据的次数。身份治理使组织能够自动化许多先前手动的用户配置任务，并缩短了员工入职时的上手时间。在治理框架内，`Identity Manager`通过允许用户进行自己的密码管理，并基于一套定义好的规则处理访问请求，从而减少了支持电话。通过运行定期的安全审计和对账任务，也降低了风险。平台安全服务允许组织为开发者提供连接和引用安全信息的标准方法。所有这些功能都使用目录服务来存储和访问身份数据。Oracle 目录服务由一个基于 Java 的身份存储`OUD`和一个基于数据库的`OID`组成。这两者都可用于存储应用程序身份数据，并能与现有的企业 LDAP 身份存储进行同步。`OVD`允许组织整合现有的身份存储，而无需复制数据，并为多个应用程序提供单一的身份数据访问点。
