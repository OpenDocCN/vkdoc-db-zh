# 配置 Oracle Access Manager 身份存储

随着 OID 扩展以支持 OAM，现在是时候配置 OAM 以使用 OID 作为系统和默认用户存储了。通过导航至 URL `http://host:wls_console_port/oamconsole` 登录到 AOM 管理控制台。图 9-7 展示了一个 OAM 管理控制台的示例。使用 `weblogic` 用户名和密码登录。在 OAM 系统和默认身份存储配置完成之前，新创建的管理员用户将无法使用。

![A352855_1_En_9_Fig7_HTML.jpg](img/A352855_1_En_9_Fig7_HTML.jpg)
*图 9-7. OAM 管理控制台*

管理控制台的登录页面被划分为多个区域，以便更容易找到配置设置。顶部是主要配置类别：应用程序安全、联合、移动安全和配置。每个类别都有多个选项，允许您配置各种组件。首先，选择屏幕右上角的“配置”选项。配置菜单如图 9-8 所示。

![A352855_1_En_9_Fig8_HTML.jpg](img/A352855_1_En_9_Fig8_HTML.jpg)
*图 9-8. 配置菜单*

完整的配置和管理超出了本书的范围。但是，本指南涵盖了完成整体集成所必需的部分。在配置菜单上，点击“用户身份存储”图标以查看当前配置。图 9-9 显示了您在创建新存储之前将看到的“用户身份存储”屏幕。

![A352855_1_En_9_Fig9_HTML.jpg](img/A352855_1_En_9_Fig9_HTML.jpg)
*图 9-9. 身份存储配置屏幕*

首次访问“用户身份存储”屏幕时，您将看到访问管理器正在使用嵌入式 LDAP 作为 `用户身份存储`，并且它被设置为默认和系统存储。嵌入式 LDAP 指的是 WLS 内置的。在本节中，您将创建一个名为 `OID` 的新身份存储，并将系统和默认存储更改为使用它。

在“创建：用户身份存储”屏幕（如图 9-10 所示）上，输入 OID 的必要连接详细信息。随后的表 9-2 列出了在此阶段预期的值。

*表 9-2. OID 身份存储连接详细信息*

| 存储名称 | 为新的身份存储指定一个将使用的名称 |
| :--- | :--- |
| 存储类型 | `OID` Oracle Internet Directory |
| 绑定 DN | `Cn=orcladmin`，或其他对用户和组搜索基准具有读取访问权限的用户 |
| 密码 | 为“绑定 DN”中列出的用户设置的密码 |
| 描述 | （可选） |
| 位置 | OID 的主机名和 LDAP 端口号 |
| 登录 ID 属性 | `CN` 是默认值；如果您使用不同的属性，请在此处输入 |
| 用户密码属性 | `userPassword` |
| 用户搜索基准 | `Cn=Users,dc=mycompany,dc=com`；此值应与 `extendOAMPropertyFile` 中使用的用户搜索基准匹配 |
| 组搜索基准 | `Cn=Groups,dc=mycompany,dc=com`；此值应与 `extendOAMPropertyFile` 中使用的组搜索基准匹配 |

![A352855_1_En_9_Fig10_HTML.jpg](img/A352855_1_En_9_Fig10_HTML.jpg)
*图 9-10. 创建：用户身份存储屏幕*

点击“测试连接”以确保您输入的详细信息正确，并且 OAM 可以连接到新的 LDAP 存储。在继续下一步之前，请确保您看到的结果类似于图 9-11。

![A352855_1_En_9_Fig11_HTML.jpg](img/A352855_1_En_9_Fig11_HTML.jpg)
*图 9-11. 测试连接*

成功创建 LDAP 存储后，您必须将 OAM 系统和默认存储值设置为新存储的名称。您将在图 9-12 中看到一个示例。

![A352855_1_En_9_Fig12_HTML.jpg](img/A352855_1_En_9_Fig12_HTML.jpg)
*图 9-12. 设置默认和系统存储*

当您更新存储值时，您将看到一条弹出消息，告知您在应用更改之前必须完成一个步骤。一个名为“访问系统管理员”的新部分将出现在页面上。在此部分中，您将添加在 `preconfigOAMProperties` 文件中创建的 `oamadmin` 用户和 `oamadministrators` 组。点击“添加”按钮，并使用搜索屏幕查找用户和组。根据您的环境添加任何可能需要的额外用户。添加用户后，点击“应用”。系统将要求您提供 `oamadmin` 用户名和密码，以验证新用户存储是否正常工作。

验证管理员用户并保存新配置后，您必须配置 LDAP 身份验证模块以使用新的身份存储。在退出管理控制台之前未能完成此步骤可能会阻止您再次登录 OAM 控制台。要继续，请选择屏幕顶部中央的“应用程序安全”选项，如图 9-13 所示。

![A352855_1_En_9_Fig13_HTML.jpg](img/A352855_1_En_9_Fig13_HTML.jpg)
*图 9-13. 应用程序安全菜单*

在“应用程序安全”菜单上，在“访问管理器”下，点击“身份验证方案”链接。图 9-14 显示了 OAM 预配置的身份验证方案列表。

![A352855_1_En_9_Fig14_HTML.jpg](img/A352855_1_En_9_Fig14_HTML.jpg)
*图 9-14. 身份验证方案*

在“搜索身份验证方案”屏幕上，点击“搜索”以显示当前可用的方案列表。点击列出的 `LDAPScheme` 以调出配置。这将显示所使用的身份验证模块的名称。图 9-15 显示了将使用的配置屏幕。

![A352855_1_En_9_Fig15_HTML.jpg](img/A352855_1_En_9_Fig15_HTML.jpg)
*图 9-15. LDAPScheme 配置屏幕*

在这种情况下，`LDAPScheme` 被配置为使用 LDAP 身份验证模块。这是您需要更改的模块，以确保使用外部 LDAP 存储（如 OID）配置的访问管理器内的正常登录和访问。图 9-16 显示了“身份验证模块”选项卡上的结果集。

![A352855_1_En_9_Fig16_HTML.jpg](img/A352855_1_En_9_Fig16_HTML.jpg)
*图 9-16. 身份验证模块*

现在您已经获得了 LDAP 身份验证方案使用的身份验证模块的名称，返回到启动页面，并在“应用程序安全”屏幕上的“插件”下点击“身份验证模块”链接。点击“搜索”以显示可用模块，然后点击“LDAP”链接。这将调出所选模块的配置参数，如图 9-17 所示。

![A352855_1_En_9_Fig17_HTML.jpg](img/A352855_1_En_9_Fig17_HTML.jpg)
*图 9-17. LDAP 身份验证模块配置*

在配置屏幕上，对于“用户身份存储”，选择 `OIDStore`，即先前创建的身份存储的名称。点击“应用”以保存更改。您现在已完成配置 AOM 以使用 OID，并准备好进入下一章配置 OIM 的下一步。现在关闭浏览器并重新启动 OAM。重新启动后，进行测试以确保您可以以创建的 `oamadmin` 用户身份登录到 `OAMConsole`。

## 总结

本章演示了配置 OAM 所需的步骤。它包括预配置 OID 模式，以包含集成 OAM 与外部 LDAP 提供程序所必需的容器、用户和组。尽管这些说明是针对 OID 的，但经过一些修改，类似的说明可用于大多数其他受支持的 LDAP 目录。无论是否配置 OAM 以与 OIM 集成，此步骤都是必需的。它将确保用户和权限包含在集中式 LDAP 存储中，而不依赖于嵌入式 WebLogic 存储。
