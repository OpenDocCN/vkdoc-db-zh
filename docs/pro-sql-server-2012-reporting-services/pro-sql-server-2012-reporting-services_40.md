# 报告设计步骤

## 布局调整

现在你已经合并了患者的全名，可以通过点击数据单元格并按`Delete`键来删除“Pat Last Name”字段。在预览报告之前，还有一些其他任务需要完成。

首先是将“Zip”列移动到“State”列的右侧。只需点击包含“Zip”的单元格，并将其拖到“State”列的右侧即可。

接下来，调整字段大小，使每条记录都能在一行内显示。在用于报告的字段中，“Diagnosis”和“Address”字段可能是最大的，因为它们是可变长度字段，通常包含 20 到 50 个字符。然而，由于我们希望每列的标题显示在一行上，因此也需要调整“Start of Care”字段的大小。

在此示例中，将“Pat ID”字段调整到约 0.5 英寸，将“Diagnosis”、“Start of Care”和“Address 1”字段扩展到约 1.5 英寸，然后预览报告，如`Figure 13-27`所示。代表数据的`x`字符数量将与数据库字段中最大值中的字符数量一样多，这使得确定如何调整每列大小变得容易。

“Address”字段显示为`NA`的原因是，这些信息要么是经过混淆处理（例如患者姓名），要么被修改为`NA`以移除可识别信息。

另请注意，在`Figure 13-27`中，每个字段在列标题级别自动包含一个交互式排序图标。尽管点击图标会重新渲染报告，但在你配置排序之前，实际上不会执行任何排序操作。下一步我们将进行配置。

![Image](img/9781430238102_Fig13-27.jpg)

**Figure 13-27.** 预览患者情况表报告

## 排序与分组

点击工具栏上的“Design Report”按钮返回报告的设计区域。接下来，右键单击“Pat ID”数据单元格，然后在工具栏上选择“Sort and Group”。这将打开“Sort”对话框。选中“Patient Census”组，选择首先按“Pat ID”排序，然后按“Patient Name”排序，最后按“Diagnosis”排序，如`Figure 13-28`所示。你还可以选择在每个组后添加分页符作为组选项。此时请不要添加分页符。单击“OK”后，你可以预览报告，并看到每个定义列的排序工作正常。

![Image](img/9781430238102_Fig13-28.jpg)

**Figure 13-28.** 排序和分组选项

关于将字段拖到报告的表区域，我们还需要提及一点：分组会自动从左到右定义。换句话说，将字段拖放到另一个字段的右侧会将该字段保留在同一组中（本例中为“Patient Census”组）；但是，如果你已经将一个字段拖到表中，然后又将另一个字段拖到它的左侧，则会根据被放置字段的名称创建另一个组。你可以在`Figure 13-29`中看到这一点，例如将“Branch Name”字段拖到“Pat ID”字段的左侧。

如果你正在进行这些操作，请继续将“Branch Name”字段拖到报告中以查看自动分组效果。同时，请将表头行调整为一行高。然后将整个表拖得离标题更近一些。执行这些步骤后，应该会得到一个类似于`Figure 13-29`所示的报告。

![Image](img/9781430238102_Fig13-29.jpg)

**Figure 13-29.** 自动创建的分组

## 保存与部署

![Image](img/9781430238102_Fig13-30.jpg)

**Figure 13-30.** 预览新的“Branch Name”组

## 创建矩阵报告

与表格报告不同，矩阵报告以二维方式显示聚合值，同时包含列组和行组。无论是总和、平均值还是计数，这些汇总值都会在行和列的分组层级相交。这创建了一个交叉表报告，类似于 Microsoft Excel 中的数据透视表。此报告的需求之一是显示患者的住院时长，从入院到出院的时间。另一个要求是报告中需显示出院原因。你可以将这两个需求完美结合到一个矩阵报告中。

那么，回到 Report Builder 1.0 的设计区域，在任务窗格的“报告布局”部分点击“矩阵（交叉表）报告”。这将打开一个新的空白矩阵模板，你可以像处理表格报告一样，将字段拖放到矩阵数据区域上。该模板不过是一个预定义的矩阵数据区域，你将向其添加几个字段，如 图 13-32 所示。

![Image](img/9781430238102_Fig13-32.jpg)

**图 13-32.** 空白矩阵报告

要开始创建此报告，请展开“字段”列表中的 `Total Length of Stay` 属性。将模型生成时计算的 `Avg Length of Stay` 属性拖到矩阵的汇总区域。对于行分组，添加 `Discharge Reason` 字段。最后，将 `Marital Status ID` 字段拖到列分组区域，使矩阵报告类似于 图 13-33。需要调整 `Discharge Reason` 字段的大小，并将 `Marital Status ID` 字段设置为居中对齐。此时，你还需要为报告添加一个标题：“平均住院时长”。将标题设为粗体，并将字体类型设置为 Arial Rounded MT Bold。

![Image](img/9781430238102_Fig13-33.jpg)

**图 13-33.** 包含数据的矩阵报告

在预览时，报告已初具雏形，如 图 13-34 所示；然而，你需要处理一些异常情况。首先，数字格式并非你所期望或想要的。默认情况下分配给 `Avg Length of Stay` 字段的数据类型是常规数字，这并不理想。你需要为数字添加格式。其次，你会看到包含空值的行和列。这可能有两个原因：首先，你包含了尚未出院的患者，即活跃患者；其次，婚姻状况并非必填字段，因此某些记录会没有值。

![Image](img/9781430238102_Fig13-34.jpg)

**图 13-34.** 矩阵报告的初始预览

你可以通过返回设计区域，右键点击最后一列（即总计列），然后选择 `Format`（格式）来处理数字格式问题。在 `Format`（格式）对话框中，将定义的格式从 `General`（常规）更改为 `1234.56`，如 图 13-35 所示，并将小数位数从 `2` 改为 `0`。然后点击 `OK`（确定）。对所有其他值字段执行相同的格式设置。

![Image](img/9781430238102_Fig13-35.jpg)

**图 13-35.** 更改数字格式

为了解决空列问题，请向报告添加一个筛选器，将数据限制为仅包含已出院且有出院原因的患者，以及定义了婚姻状况的患者。工具栏上的 `Filter`（筛选器）按钮将启动 `Filter Data`（筛选数据）对话框。在 图 13-36 中，你可以看到可以添加四个筛选条件：`MaritalStatusID`、`Discharge Date` 和 `Discharge Reason` 不为空，并且 `Discharge Year` 大于或等于 `2010`。（我们添加最后一个筛选器是为了演示报告可以提示用户输入一个作为参数的值，该参数与筛选器关联。）要将比较运算符从默认的“等于”更改为“不为空”，请点击“空”，然后从可用运算符中选择 `Not`（非）和 `Is Empty`（为空）。对于出院年份，选择 `Greater Than or Equal To`（大于或等于）运算符。将年份设置为 `2010` 后，点击 `Discharge Year`（出院年份）标签并选择 `Prompt`（提示），将其设置为报告参数，如 `Discharge Year`（出院年份）筛选器标签旁边的绿色问号图标所示。

![Image](img/9781430238102_Fig13-36.jpg)

**图 13-36.** 设置报告筛选器

现在，当你在 图 13-37 中预览矩阵报告时，可以看到数字已被清理，所有空字段都已消失；同时，系统会提示你输入出院年份。将 `Discharge Year`（出院年份）参数更改为 `2011` 以查看这些结果。像保存表格报告一样，将此报告以“平均住院时长”为名保存到报告服务器上的 Pro_SSRS 文件夹中。现在，我们将继续使用 Report Builder 1.0 创建最后一个报告：图表。

![Image](img/9781430238102_Fig13-37.jpg)

**图 13-37.** 设置报告筛选器并预览报告



### 创建图表报表

报表生成器 1.0 中最后一种可用的报表类型是图表报表。至此，你已经创建并保存了表格和矩阵报表，以满足“患者统计”报表的许多请求。现在，我们将展示一种最重要的报表，用于回答统计中的患者数量问题。“统计”意味着患者当前正在接受服务且尚未出院。在你使用的数据库中，如果患者的出院日期为 `Null`，或者用报表生成器 1.0 的术语来说是“空”，则该患者被视为活跃状态。你将创建的图表是一个简单的条形图，按分支机构显示活跃患者的计数。

回到报表生成器 1.0 的设计区域，在“报表布局”选项卡中点击图表报表。空白报表模板包含一个“图表”数据区域，如图 13-38 所示。如同之前创建表格和矩阵报表一样，你将添加一个标题：活跃患者统计。

![Image](img/9781430238102_Fig13-38.jpg)

**图 13-38.** 图表报表模板

图表有三个可以拖放数据的区域：系列、类别和数据。你需要显示活跃患者计数，因此可以将 `患者 ID` 字段拖放到“数据”区域；但是，因为 `患者 ID` 字段不像 `平均住院天数` 那样被视为可以应用聚合函数的字段，所以无法直接将其拖放到“数据”区域。图表将不接受放置于此。既然你知道既需要一个计数值，又需要一个筛选器来仅获取活跃患者，你可以通过向字段列表添加一个新字段来同时解决这两个问题。在“资源管理器”选项卡中点击“新建字段”按钮，并将新字段命名为 `患者计数`，如图 13-39 所示。用于 `患者计数` 字段的公式可以是 `COUNTDISTINCT(患者 ID)`。请注意，在“定义公式”对话框中，`患者 ID` 字段添加到公式文本框后，其下方会有一条虚线，表示链接。你可以单击该字段，在为新字段定义公式时添加一个筛选器，接下来你将进行此操作。

![Image](img/9781430238102_Fig13-39.jpg)

**图 13-39.** 为新字段 `患者计数` 定义公式

点击“未应用筛选器”链接，选择“创建新筛选器”以打开“筛选数据”对话框。你可以在图 13-40 中看到需要应用的筛选器，以仅显示活跃患者，其中你将 `出院日期` 设置为空，这将强制仅使用没有出院日期的患者进行公式计算。要实现筛选，将 `出院日期` 拖放到筛选窗格。然后通过点击“等于”链接标签选择“为空”值。将筛选器命名为 `活跃患者`，然后点击“确定”。一旦 `患者计数` 字段创建并显示在“字段”选项卡上，你就可以将其拖放到图表的“数据值字段”区域。在这种情况下，由于我们将其添加到“数据值字段”区域，只需双击 `患者计数` 字段，报表生成器就会将其放置在正确的部分。

![Image](img/9781430238102_Fig13-40.jpg)

**图 13-40.** 添加 `活跃患者` 筛选器

接下来，将 `分支机构名称` 字段拖放到图表的“类别”区域。你可以向图表添加更多的系列项或类别，但为了保持图表简洁并满足请求，你将仅通过移除图表标题来对图表应用一些格式设置，以使报表上不显示“图表标题”。要修改图表属性，请遵循以下四个步骤：

> 1.  右键单击图表，然后选择“图表选项”。
> 2.  在“标题”选项卡上，删除图表、X 轴类别和 Y 轴数值的标题文本。
> 3.  在“图例”选项卡上，取消勾选“显示图例”复选框。
> 4.  最后，在“3D”选项卡上，勾选“以 3D 视觉效果显示图表”框，将“水平旋转”设置为 25，“垂直旋转”设置为 10，“明暗”设置为“逼真”，并选择“圆柱体”作为形状，如图 13-41 所示。

![Image](img/9781430238102_Fig13-41.jpg)

**图 13-41.** 设置 3D 视觉效果

在预览报表之前的最后一个任务是设置图表上的数据标签。为此：

> 1.  右键单击图表，选择“数据系列”，然后选择“设置数据系列格式 - 患者计数”。如果你希望在图表柱形上显示患者计数值，请选择“显示数据点标签”。在本示例中，选择“显示数据点标签”选项。
> 2.  将角度设置为–10 度，使其稍微突出一些。
> 3.  点击“格式”按钮，将字体大小设置为 16。

当你点击“确定”设置数据点标签，然后预览报表时，可以看到按分支机构显示活跃患者计数的图表。随着患者出院，由于筛选器，他们将从活跃患者列表中移除，图表将自动反映当前的统计计数。图 13-42 显示了完成的图表。现在，你可以将图表报表保存为 `活跃患者统计` 到 `Pro_SSRS` 文件夹中，就像保存其他两个报表一样。

**图 13-42.** 活跃患者统计图表报表的预览

## 使用 Report Builder 2.0 向导创建报表

至此，我们已经使用 `BIDS` 和通过 `ClickOnce` 部署的 `Report Builder 1.0` 创建了 `SSRS` 报表。这两种设计环境对报表设计提供了不同级别的控制。虽然 `BIDS` 为报表创建者提供了比 `Report Builder 1.0` 大得多的灵活性，但对于非应用程序程序员的人来说，它可能不是最佳选择。相比之下，`Report Builder 1.0` 易于学习和使用以发布报表；然而，它存在限制，即只能使用预构建的模型来创建报表，这不足以满足每个人的需求。在 `SSRS 2008` 中，微软提供了 `Report Builder 2.0`，它支持真正的临时查询和报表设计，而不仅仅基于报表模型。`Report Builder 2.0` 现代化的外观和感觉，采用了类似 `Office 2007` 应用程序中的 `Ribbon` 技术，为报表创建者提供了一个熟悉的工作环境。

图 13-44 展示了 `Report Builder 2.0` 的界面。图中明显可见微软对新技术 `Ribbon` 的应用。

![Image](img/9781430238102_Fig13-44.jpg)

`图 13-44. Report Builder 2.0 的新界面`

## 使用 Report Builder 2.0 创建报表

正如我刚才提到的，创建报表时我们并非必须使用向导。此外，如果我们不想使用 `新建表或矩阵` 向导，也可以分别创建新的数据源和数据集。在本节中，我们将不使用向导来创建相同的报表。

首先，如果 `按分支机构估算服务类型成本` 报表仍然打开着，请点击屏幕左上角的圆形功能区图标。这个功能区图标的作用类似于其他 Microsoft 产品中的 `文件` 菜单选项。点击功能区图标后，选择 `新建` 来创建新报表。如果您没有打开 `Report Builder 2.0`，请从 `所有程序` 菜单中的 `Microsoft SQL Server 2008 Report Builder 2.0` 下找到 `Report Builder` 瘦客户端并打开。如果您一直跟随操作，那么 `Report Builder 2.0` 的空白设计环境对您来说应该很熟悉，它与图 13-44 中所示的一样。

要开始不使用向导从头创建报表，我们需要在进入报表设计之前先创建一个数据源和一个数据集。创建好这些后，我们就能将表格添加到设计窗格，然后开始添加列和分组。以下四个步骤将引导我们完成为报表创建数据源和数据集的过程。

1.  点击 `报表数据` 窗格上的 `新建` 按钮，然后选择 `数据源`。如果看不到 `报表数据` 窗格，请点击 `视图` 选项卡，然后选中 `报表数据` 复选框。
2.  将您的数据源命名为 `Pro_SSRS` 并设置 `使用共享连接或报表模型` 选项。我们可以使用服务器上的共享数据源，而不是嵌入式数据源。这有助于将报表部署到不同环境时管理。如果报表使用共享数据源，即使数据库服务器在报表生命周期内发生更改，唯一需要更新的对象也只是共享数据源本身。反之，如果您始终使用嵌入式数据源，那么所有使用该数据源的报表也都需要更改。这使得管理更加困难，因此，应尽可能使用共享数据源。
3.  现在，点击 `浏览…` 按钮，导航到 `Pro_SSRS` 文件夹下的 `DataSources` 文件夹，并选择 `Pro_SSRS` 数据源。点击 `打开` 返回 `数据源属性` 窗口，如图 13-51 所示。点击 `确定` 保存 `Pro_SSRS` 数据源并返回 `Report Builder`。保存数据源后，`Report Builder` 会将其添加到 `报表数据` 窗格。
    ![Image](img/9781430238102_Fig13-51.jpg)
    **图 13-51.** 创建新数据源
4.  在 `报表数据` 窗格中右键单击新的 `Pro_SSRS` 数据源，然后选择 `添加数据集`。将数据集命名为 `Emp_Svc_Cost`，并将 `查询类型` 设置为 `存储过程`。如果提示输入登录凭据，请输入相应的登录信息。然后在 `选择或输入存储过程名称` 下拉列表中，选择我们的 `Emp_Svc_Cost` 存储过程。图 13-52 显示了 `数据集属性` 窗口。点击 `确定` 保存 `Emp_Svc_Cost` 数据集。
