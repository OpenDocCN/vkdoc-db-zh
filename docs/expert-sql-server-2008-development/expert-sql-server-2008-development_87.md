# 性能比较：SQLCLR 与 TSQL

讨论了实现基于 SQLCLR 的解决方案背后的一些安全和架构考虑后，你可能想知道哪些情况实际上可以从使用 SQLCLR 中受益。

重要的是要认识到，SQLCLR 不是，也从来无意成为 T-SQL 的替代品作为 SQL Server 中的数据操作语言。为了从数据库读取数据以及执行大多数标准数据操作，T-SQL 是唯一的选择。此外，虽然我强调了创建易于在各层之间移动和共享的可移植代码的重要性，但你不应尝试使用 SQLCLR 将本应属于应用程序的逻辑移入数据库。虽然允许在层之间进行代码重用可能有好处，但请记住，数据库服务器通常是成本最高的扩展层，因此将过多逻辑移入数据库是对宝贵数据库资源的低效使用，而这些资源本可以由一个便宜得多的应用程序服务器来提供。关于数据和应用逻辑正确放置位置的进一步讨论，请回顾第 1 章。
