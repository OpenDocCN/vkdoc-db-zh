# 第 6 章：加密

## 平衡性能与安全性

*图 6-6。一种混合加密方法。*

混合加密结构的要素可以描述如下：

*   数据使用对称密钥进行加密，这提供了性能最佳的加密方法。我推荐基于`AES`算法的加密，并根据您的需求选择适当的密钥长度。密钥越长提供的保护越多，但同时也需要更多的处理开销。

*   对称密钥由一个或多个证书保护，这些证书提供了非对称加密的强度以及额外的好处，例如能够从`T-SQL`备份和恢复证书，以及为证书的有效期设置明确的开始和到期日期。我倾向于为每个需要访问对称密钥的用户或用户组创建一个证书。这些相同的证书也可用于加密该用户或用户组所需的其他密钥。

*   每个证书都用一个（强）密码保护，以便只有知道密码的个人才能访问。这意味着加密信息对系统管理员也是保密的，否则他们可以使用`DMK`来打开证书及底层的对称密钥。

*   可以在此模型中添加进一步的加密层，方法是在证书加密之前，使用额外的对称密钥来保护该对称密钥。每一层都为潜在的黑客破解设置了额外的障碍，同时也使得密钥轮换更容易实施。

为了说明如何创建上述混合加密模型，首先创建两个用户：

```sql
CREATE USER FinanceUser WITHOUT LOGIN;
CREATE USER MarketingUser WITHOUT LOGIN;
GO
```
