## 创建一个用于素数的“简单筛子”

在这个测试中，我创建了两个简单的存储过程，用于返回直到提供最大值的所有素数列表——一个用 T-SQL 实现，另一个用 SQLCLR 实现。这些测试的逻辑尽可能简化：每个过程都被提供一个最大值，该值在一个循环中递减，在循环的每次迭代中，使用模运算运算符计算该值除以每个较小数字的余数。如果除法的余数为 0（换句话说，我们找到了一个因子），我们就知道当前值不是素数。因此，循环继续测试下一个可能的值。如果内部循环测试了所有可能的除数并且没有找到任何因子，那么我们就知道这个值一定是素数。使用这种“简单筛子”算法来寻找素数依赖于基本的数学函数和过程式逻辑，这使其成为比较 T-SQL 和 SQLCLR 性能的一个良好测试。以下是 T-SQL 的实现：

```
@n int = @Limit,

-- 我们已经测试到 2，并且没有找到任何因子

IF(@m = 1)

BEGIN

PRINT CAST(@n AS varchar(32)) + ' 是一个素数'

BREAK;

END

-- 这个 @m 是这个素数的因子吗？

IF(@n%@m) <> 0

BEGIN

-- 不是因子，所以继续下一个 @m

SET @m = @m - 1;
```

```
public static void ListPrimesCLR(SqlInt32 Limit)

{

int m = (int)Limit - 1;

while(n > 1)

{

while(m > 0)

{

if(m == 1)

{

SqlContext.Pipe.Send(n.ToString() + " 是一个素数");

}

{

m = m - 1;

continue;

}

else

{

break;

}

}

n = n - 1;

m = n - 1;

}

}
```

`注意` 显然，如果你真的想要获取素数列表，你绝不会使用如此简单（天真的）方法。这里使用的例子旨在提供一个可以在 T-SQL 和 SQLCLR 中一致实现的简单过程。
