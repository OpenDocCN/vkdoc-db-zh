# 第 2 章 „ „ „ 数据库编程最佳实践

软件开发不仅仅是编码员执行的一门实践学科，也是学术研究和理论的领域。现在有大量关于软件开发的知识体系，并且已经撰写了冗长的学术论文来提出、剖析和讨论不同的开发方法。各种方法论已经出现，包括测试驱动开发 (`TDD`)、敏捷和极限编程 (`XP`) 以及防御性编程，关于这些思想流派所提供好处的争论不计其数。
本章描述的实践，以及本书其余部分所采用的方法，与防御性编程的理念最为契合。然而，这里讨论的主题同样可以轻松应用于任何环境。虽然软件理论家可能会争论不同方法论之间的细微差别（无疑，它们在某些方面确实不同），但归根结底，无论你采用哪种方法论，良好编程的基本特征始终如一。
我无意提供一份关于什么构成最佳实践的详尽、客观的指南，而是旨在强调一些我认为能够体现数据库开发者为做好工作所需专业水准的标准。我将从防御性角度来阐述每个论点的理由，但请记住，这些论点通常在其他环境中也同样有效。

## 防御性编程



防御性编程是一种软件开发中使用的方法论，它建议开发者应主动预测并为（或“防范”）不可预见的未来事件做出预案。

防御性编程的目标是创建即使在面临意外情况时也能保持健壮和有效的应用程序。

防御性编程本质上是对世界持悲观看法——如果某事可能出错，它就会出错：网络资源会在事务处理中途变得不可用；所需的文件可能缺失或损坏；用户输入数据的方式可能与预期有无数种不同，等等。防御性程序员不会听天由命，而是会预测到这些可能性，并编写相应的处理代码来检查和处理这些情况。这意味着潜在的错误状况可以在实际错误发生前被检测并处理。

请注意，防御性编程不一定能使应用程序在异常情况发生时继续运行，但它确实使系统能够以一种可预测的、受控的方式运行——实现*优雅降级*，而非冒着导致未知后果的崩溃风险。在许多情况下，可能可以识别并隔离导致故障的特定组件，从而允许应用程序的其余部分继续运行。
