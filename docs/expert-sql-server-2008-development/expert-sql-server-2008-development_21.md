# 前言

我使用 Microsoft SQL Server 已近十年，自 SQL Server 2008 的第一个公开预览版可用起便开始使用它。我注意到的一点是，随着每一个新版本的发布，SQL Server 变得越来越强大，也越来越复杂。现在有大量功能远远超出了数据库系统预期的核心功能，并且由于需要涵盖如此多的不同方面，要成为一名 SQL Server “专家”变得越来越难。SQL Server 开发人员不再仅仅需要精通编写 `T-SQL` 代码，还需要精通 `XML` 和 `SQLCLR`（并知道何时使用每种技术）。您不再只是从一个孤立的数据库执行查询以获得单个结果集，而是要处理源自分布式服务器查询的多个活动结果集。现代数据库中存储的信息类型不仅代表字符、数字和二进制数据，还代表复杂数据，如空间、层次结构和 `filestream` 数据。

试图全面涵盖这些主题中的任何一个，都足以轻松填满整本书，所以我甚至不会尝试这样做。相反，我将根据我自己的实践经验，专注于我认为您需要了解的内容来创建高质量的数据库应用程序。我不会浪费篇幅讨论一些晦涩或很少使用的功能的细节，除非我能向您展示它的真正用例。我也不会通过费力地解释基础知识来侮辱您的智力——我会假设您已经熟悉 `Books Online` 中涵盖的简单示例，并且现在希望进一步拓展您的知识。

本书中的所有示例都基于我遇到的真实场景，它们向您展示了如何处理在大多数典型 SQL Server 环境中可能面临的问题。我保证不会向您展示看似完美的解决方案，然后您发现它们只在经过人为净化的“`AdventureWorks`”世界中有效；作为开发人员，我们处理的是不完美的数据，我会尽量向您展示包含所有缺陷的示例。这些代码示例是使用安装了 Service Pack 1 的 SQL Server 2008 Developer Edition 测试的，但除非另有说明，否则应适用于 SQL Server 2008 的所有版本。

最后，我希望您喜欢阅读本书并思考所讨论的问题。我喜欢数据库开发的原因是它提出了一个永无止境的需要解决的谜题——即使您认为已经找到了问题的最佳答案，未来总是有可能找到更好的解决方案。虽然您不应让这种对完美的追求分散您对当前工作的注意力（有时，“足够好”就是您所需要的），但总有新的技术要学习，其他的方法要探索。我希望您能在接下来的页面中学到其中的一些。
