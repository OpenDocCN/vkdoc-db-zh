## 总结

应用程序对数据库的依赖远不止于单纯的数据持久化，数据库开发者必须理解整个应用程序开发过程，才能创建真正有效的数据库系统。
通过理解诸如耦合、内聚和封装等架构概念，数据库开发者可以定义模块化的数据接口，从而在持续维护和测试中获得巨大收益。数据库开发者还必须理解如何最好地将数据从面向对象系统映射到数据库结构，以便有效地持久化和操作数据。
本章介绍了这些概念。这里提出的概念将在本书后续章节的各种例子中再次讨论。
