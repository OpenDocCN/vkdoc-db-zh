# 第二章 „ 数据库编程的最佳实践

影响防御性开发成功的另一个因素是预算和项目截止日期的管理方式。软件项目交付若超出截止日期通常会产生罚款。不幸的是，当截止日期提前或预算被大幅削减时，管理层往往认为防御性实践（如严格测试）是非必要的，并且这些实践会最先被从项目范围中剔除。

要求基于不切实际的短期时间表提供快速修复解决方案的管理者，很可能会鼓励零散的编码实践，从而产生漏洞。这类代码不太可能采用防御性编程，也无法经受严格的测试。软件开发需要耐心和细致地雕琢，然而管理要求常常迫使团队必须走捷径，真正采用防御性编程的项目很少能够善始善终。基于这些原因，真正的防御性编程可能被视为一种理想，而非一个可实现的目标。

## 总结

防御性编程实践旨在提高软件应用程序在面对未预见情况时的韧性和可靠性。考虑到数据库应用程序的典型预期寿命，以及发生缺陷可能导致的严重后果，采用防御性方法以确保应用程序在长期内保持健壮，并将持续维护的需求降至最低，是合乎情理的。

在本章中，我展示了一些我认为是 `SQL` 编程最佳实践的简单示例，并说明了它们与防御性编程方法论的关系。

在本书后续部分，我将继续更详细地展示如何在一系列开发场景中采取防御性立场。
