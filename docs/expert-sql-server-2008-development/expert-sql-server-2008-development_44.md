# 第三章 „ „ „ 测试数据库例程

什么定义了伟大的开发者？是快速准确地编写复杂例程的能力吗？是在预算内按时正确实现业务需求的能力？或者，也许是通过开发者能多快地追踪并修复应用程序中的缺陷来定义——或者反过来，开发者代码中缺乏缺陷？

所有这些无疑都是伟大开发者的特质，但在大多数情况下，它们并非仅仅源于原始技能就能显现。真正伟大开发者的标志，也是让这些品质得以闪耀的原因，在于对测试重要性的透彻理解。

通过在开发过程早期创建 `单元测试`，开发者可以持续验证接口并测试异常和回归。精心设计的 `功能测试` 确保符合业务需求。而性能测试——这种似乎总是最受关注的测试类型——可以用来查明应用程序是否真的能够处理预期的流量。

不幸的是，就像在应用开发社区中已更好确立的其他各种实践一样，测试在数据库专业人士中尚未广泛普及。尽管一些开发团队会对存储过程和其他数据库代码进行性能测试，但很少看到数据库开发者编写数据特定的 `单元测试`。

数据库开发者没有理由不编写与应用程序开发者同等数量——甚至更多——的测试。测试一个数据驱动的应用程序而不验证驱动应用程序组件的数据部分，这是毫无意义的！

本章简要介绍了软件测试领域，以及如何在数据库开发场景中应用测试技术。软件测试是一个庞大的领域，有其自身的大量术语，因此我的意图是只专注于那些我认为对数据库开发者最重要的领域。
