# 第一部分：环境设置

## 1. 什么是数据库？

软件的一个主要特性是它能让您以特定方式与数据进行交互。这可以是一个购物清单应用，让您在手机上添加购物物品；一个让您可以撰写和发布文章的网站；或者是一个存储客户信息的客户关系管理程序。

所有这些程序都让您能以某种方式查看信息、编辑信息，然后将其存储在某处。如果无法存储这些信息，软件的效用就会大打折扣。想象一个购物清单应用，但当您输入需要购买的物品后，应用却不保存，一旦关闭应用清单就会消失。这样的应用就没什么用了！

软件需要一个地方来存储数据。最常见的存储方式就是使用数据库。数据库是文件和信息的结构化定义，它使用户和软件能够高效地存储、检索和更新数据。

购物清单应用、文章发布网站、客户关系管理系统以及许多其他类型的软件都会使用数据库。存储在数据库中的数据以多种方式被软件使用，并允许用户在软件中查看、保存和编辑信息。

例如，一个购物清单应用可能会在数据库中存储以下信息：
*   购物清单上每个物品的名称（您需要从商店购买的物品）
*   物品类别的列表（例如，蔬菜、乳制品、调味品）
*   您想购买的每个物品的数量（例如，3 个苹果，2 杯面粉）

那么，除了存储信息，为什么还要使用数据库呢？

首先，它很高效。数据在数据库中的存储方式使得用户和软件易于读取数据、添加新数据、更新或删除现有数据。

数据库还允许您在不影响所有其他数据的情况下，对一小部分数据进行更新（如果需要）。例如，您可以向购物清单中添加一个新物品，而无需更改数据库中的每个其他物品。

使用数据库存储数据的一个考虑是，它们需要一个“数据库管理系统”。这是一种允许用户或应用程序与文件交互的软件。它使您可以轻松输入命令，而数据库管理系统则负责处理数据的更改及其中涉及的所有复杂性。另一个优势是能够处理多个用户查看或编辑数据的情况。

### 数据库的替代方案

数据库并不是存储数据的唯一方式。数据也可以直接存储在文件中。一个例子就是文本文件。如果您打开记事本并新建一个文档，输入一些文本并保存，那么一个文件就被保存到了您的计算机上。这个例子中没有创建数据库。文件只是保存到了您的计算机上。

将数据存储在文本文件中比使用数据库更简单，如果您是唯一处理该文件的人，这也没问题。

然而，当软件开始被多人使用或随着时间的推移变得庞大时，将数据存储在文件中可能会出现问题，例如确保不同用户的更改不会被覆盖、确保数据能够快速保存和检索，以及只更新文件的部分内容。如果使用数据库，这些问题都能得到解决。

我想解释几个与数据库相关的术语，以帮助您理解什么是数据库以及如何与之交互。

### 表

表是数据库中存储相关数据的逻辑结构。数据库中的大部分数据都存储在表内。数据库通常有多个表，我们将在本书中频繁提及表。

如果您之前使用过电子表格程序，例如 Microsoft Excel，那么可以将表想象成一个工作表。信息在表中的存储方式与在工作表中的存储方式类似。您有被划分为列的多行数据。



### 行

一行代表表中单条信息记录。例如，在客户关系管理系统中，一行可以代表一个客户，或者在杂货清单应用中代表一个产品。数据库中的一行类似于电子表格文件中的一行。纯粹主义者可能不赞同我这样类比，但这个比喻是有用的。

### 列

列是一种表示表中记录或行特定信息的方式。它是每一行的一个属性或值。

例如，客户的“名”可以是一个列，然后可以为各行记录一个值。客户的“姓”可以是另一个列，可以单独记录，并且很可能与每个客户的“名”不同。

在脑海中想象电子表格文件中的列可能会有所帮助。例如：

```
First Name     Last Name
Jacinda        Ardern
Malcolm        Turnbull
Theresa        May
```

在这个例子中，每一行代表一位总理。因此，这个表将是一个总理表。表中的每一行具体代表一位总理。然后，每一列描述了一个属性——比如名和姓——这些是你期望一位总理拥有的属性。

### 查询

你已经了解到数据存储在数据库中，并且可以被添加、更新或检索。软件通过数据库管理系统以几种不同的方式与这个数据库交互：

*   从数据库中检索数据
*   更改数据库中的值
*   向数据库添加一些新数据
*   从数据库中删除一些现有数据

这些类型的命令被称为查询，它们是使用某种特定类型代码的特定命令，让软件通过数据库管理系统与数据库交互。这些查询易于阅读，我们将在本书中学习如何读写它们。

### SQL

SQL 代表结构化查询语言。它是用于在数据库上编写查询的语言或代码类型。这就是我们将在本书中学习的语言。

这个缩写可以发音为“ess cue ell”。然而，它也可以发音为“see kwel”，就好像这个词是“sequel”。无论哪种方式，SQL 都是我们将使用的语言。

### 索引

之前我们解释了什么是表。表是数据库中的一种对象。表用于存储数据。索引是数据库中的另一种对象，用于使查找表中存储的数据更容易。

索引是一个提高查询效率的对象。它有点像书本后面的索引。如果你正在查找关于特定主题的信息，你不会浏览每一页直到找到它。你会在索引中查找该主题，它会告诉你去哪一页。

### 供应商

SQL 语言和相关数据库功能遵循一个标准：ISO SQL 标准。该标准规定了语言的工作方式。

对于许多编程语言，例如 Java，只有一个供应商。然而，对于 SQL，有许多供应商。这些供应商创建了他们自己版本的使用 SQL 的数据库管理系统。

其中一些供应商实现包括：

*   Oracle Database
*   Microsoft SQL Server
*   Oracle MySQL
*   PostgreSQL

所有这些供应商实现的相似之处在于，它们都实现了 SQL 标准。标准中定义的相同代码在每个供应商的数据库中将以相同的方式工作。至少，这是希望。

然而，每个供应商都增加了额外的功能和特性，这些在不同平台上工作方式并不相同。例如，Oracle 实现了对其数据库平台有效的 SQL 扩展，但在 PostgreSQL 等中找不到。所有供应商都做同样的事情，因此在实践中，编写完美兼容所有供应商实现的 SQL 可能很困难。

有时供应商特定的改动很小，例如特定函数的名称或数据类型的名称。有时改动较大，例如新的对象类型或构建数据库的方式。

这对你意味着什么？如果你学习适用于一个供应商数据库的 SQL，那么大部分知识可以转移到其他供应商。你应该能够将你从一个数据库供应商那里学到的大部分知识应用到另一个数据库供应商。

在本书中，你将学习 Oracle SQL。这意味着你将学习使用 SQL 语言的 Oracle 数据库管理系统上的所有基本命令。如果你去尝试使用 Microsoft SQL Server 数据库，在了解差异之后，你应该能够很快掌握 Microsoft 的实现。

### 总结

数据库是应用程序使用的一种数据存储方法。数据库中的数据存储在表中，表有用于每条记录的行和用于需要存储的每种数据类型的列。

查询是在数据库上运行的以检索、添加或更新数据的命令。这些查询是用一种称为 SQL 的语言编写的，SQL 代表结构化查询语言。

几个供应商提供他们自己版本的使用 SQL 的数据库管理系统。其中一些供应商是 Oracle、Microsoft SQL Server、MySQL 和 PostgreSQL。我们将在本书中学习 Oracle。

## 2. 设置环境

本章向你介绍我们将在本书中使用的不同工具，并向你展示如何下载和设置它们。你还将创建一个表并设置一些示例数据。

### 我们需要什么工具？

我们将在本书中使用两种不同的工具来教授 Oracle SQL：Oracle Express 和 Oracle SQL Developer。Oracle Express 是 Oracle 企业级数据库的一个精简版本。快速版（Express Edition）或 Oracle Express 是免费且易于设置的。它在学习 Oracle SQL 以及在家创建自己的项目的人群中很受欢迎。

Oracle SQL Developer 是 Oracle 开发的一个程序，它让你能够连接到数据库并在数据库上运行 SQL。它也是免费且易于设置的。

最后，Oracle 提供了另一个名为 LiveSQL 的工具。它是一个包含示例数据集的网站，让你可以连接到数据库，上传自己的数据，并在其上运行 SQL 查询。你不需要安装 Oracle Express 或 Oracle SQL Developer 来使用 LiveSQL。

虽然 LiveSQL 是一个用于学习和使用 Oracle SQL 的方便工具，但本书的重点将放在 Oracle SQL Developer 和 Oracle Express 上。这是因为本书不仅教你如何使用 SQL 语言，还将教你如何使用 Oracle SQL Developer——这是处理 Oracle 数据库最广泛使用的工具之一。这种对 SQL Developer 的关注让你为 SQL 的实际应用做好更充分的准备，因为公司会使用 Oracle SQL Developer（或类似工具）来处理数据库，而不是 LiveSQL。


### 版本

本书中的示例主要围绕以下版本的 Oracle 快捷版（Express Edition）和 SQL Developer 构建。不过，SQL 是一门成熟且稳定的语言。本书开头的内容应该也适用于任何较近期的数据库版本。

*   Oracle 快捷版 18c。此版本于 2018 年初发布，其快捷版则在 2018 年末发布。
*   Oracle SQL Developer 18.2。此版本于 2018 年 7 月发布。

Oracle 的版本号历史从 2 开始，一直到 2017 年的 12c 版本。大约每 2 或 3 年会发布一个新的主版本，“12c”中的“c”代表“cloud（云）”，这是该版本的重点（之前的版本使用其他字母，如代表网格计算的“g”和代表互联网的“i”）。尽管“c”代表云，但 12c 版本既可用于本地安装，也可在云上使用。

Oracle 公司在 2017 年宣布，他们的版本发布将改为年度发布，因此版本号将改为反映发布年份。这就解释了 18c 版本的由来。Oracle 没有发布 13 到 17 的版本号，而是直接跳到了 18c。

同样的概念也应用于 Oracle SQL Developer。最新版本是 18，而之前的版本是版本 4。

您可能已经注意到 Oracle 快捷版有 Windows 和 Linux 版本，但没有 Mac 版本。Oracle 目前不提供 Oracle 快捷版的 Mac 版本下载。但是，如果您使用的是 Mac，仍然可以使用 Oracle 快捷版来学习 Oracle。我自己就使用 Mac，并且能够在其上运行 Oracle 数据库。

Mac 用户有几种选择：

*   设置一个 Windows 虚拟机，这是您计算机上运行 Windows 安装的一个区域，您可以在其中安装 Oracle。这可以使用虚拟机应用程序（如 Parallels 或 VirtualBox）来完成。

有关设置虚拟机的详细信息，请访问以下页面：
*   Oracle
*   在 Mac 上设置 Oracle： [`www.databasestar.com/oracle-mac/`](http://www.databasestar.com/oracle-mac/)

接下来的说明主要适用于 Windows 和 Linux 用户。如果您使用的是 Mac，那么您关于使用虚拟机还是 LiveSQL 的选择将会影响您需要执行以下哪些活动。例如，Oracle 预构建的虚拟机会预装 Oracle 数据库，而对于您从头创建的虚拟机，您需要自己安装数据库。

### 运行 Oracle SQL Developer

Oracle SQL Developer 的一个优点是它不需要安装。你下载的文件是一个 `ZIP` 文件。要让 SQL Developer 运行起来：

1.  浏览到你下载 SQL Developer 的位置。

2.  通过双击文件，或右键单击并选择**解压**来解压 `ZIP` 文件。

3.  文件解压后，你可以为该文件创建一个快捷方式以便更容易运行。右键单击 `sqldeveloper.exe` 并选择**发送到桌面快捷方式**。这将在桌面上放置该文件的快捷方式。

## 第二部分 查看数据

## 3. 检索数据

在本章中，你将学习如何查看数据库中的数据。你将学习你的第一个 SQL 语句——`SELECT` 语句，并在你刚创建的表上看到一些示例。

### 如何查看数据？

在上一章中，你完成了很多工作：

*   你下载并安装了 Oracle Express。
*   你下载并运行了 SQL Developer。
*   你创建了两个新连接和一个新用户。
*   你创建了一个新表并填充了示例数据。

当你创建表并添加示例数据时，你使用的是 SQL Developer 的用户界面——菜单和屏幕上出现的字段。然而，SQL 是一种基于文本的语言。你将学习如何编写 SQL 代码来处理这些数据，因为这种代码可以在其他应用程序中使用，用于开发报表或其他脚本。

SQL 语言中有一个语句或命令允许你查看数据库中的数据。它被称为 `SELECT` 语句。

### 什么是 SELECT 语句？

`SELECT` 语句是 SQL 中的一种语句类型，它允许你从数据库中读取数据。一个 `SELECT` 语句必须按照某种模式编写。这种模式被称为*语法*（syntax），你将学习到的每个语句都有不同的模式或语法。“语法”一词在 SQL 和编程中常用，用于描述代码需要如何编写。

要在 Oracle SQL 中编写 `SELECT` 语句，需要遵循以下语法：

```
SELECT your_columns
FROM your_table;
```

让我们看看这个语句的各个部分。首先，你从 `SELECT` 关键字开始。我之所以称它为*关键字*，是因为它是 SQL 中的一个特殊词汇，有特定的功能（在这个例子中，是检索一些数据）。一个 `SELECT` 语句以 `SELECT` 关键字开头。

然后你有一个空格，接着是占位符 "your_columns"。这个占位符是你列出希望在结果中显示的列的地方。当你编写 `SELECT` 语句时（你很快就会做），你需要用其他内容替换 "your_columns" 部分，通常是一个列名列表。

接下来，是 `FROM` 关键字。`FROM` 关键字是 `SELECT` 语句的一部分。它指定了数据所来源的表。`FROM` 关键字本身不是一个语句，因为一个 SQL 语句不能以 `FROM` 关键字开头。`FROM` 实际上是一个属于其他语句（如 `SELECT`）的关键字。

然后你有一个空格，接着是 "your_table" 这个词。这也是一个占位符，你可以将其替换为表的名称。

最后，你有一个分号 ";"。它指定了语句的结束。写分号是可选的，但养成这个好习惯很有益。当你开始在 SQL 窗口中拥有多个语句或开始编写更多 SQL 代码时，分号将变得很重要。尽早养成在 SQL 语句末尾添加分号的好习惯。


### 需注意的几点

你可能会对 `SELECT` 语句语法中的几处细节感到疑惑，例如大小写和换行。我将部分语法写成了大写字母，部分写成了小写字母：

```
SELECT your_columns
FROM your_table;
```

为什么这样呢？基本上，这是关于格式化的问题。当你编写 SQL 时，坚持使用一致的格式有助于他人阅读，也更易于维护。

我倾向于将 SQL 关键字（如 `SELECT` 和 `FROM`）大写。这样无论你是在 SQL Developer、其他集成开发环境（IDE）、应用程序代码还是脚本中阅读它们，关键字都会很醒目。

其他人则偏好将 SQL 关键字小写，例如 `select` 和 `from`。这取决于个人喜好，网上也不乏文章详细说明为何以及如何将 SQL 写成大写或小写。

幸运的是，SQL 并非大小写敏感的语言。对于 Oracle 数据库而言，你写 `SELECT` 还是 `select` 并不重要。代码仍会运行。

你应该如何编写 SQL 呢？我推荐关键字使用大写。然而，我认为更重要的是与你团队或应用程序的其他部分保持格式一致。如果你的团队偏好小写，那就使用小写。如果你有选择权，比如是为自己学习而非在团队中，那么我建议使用大写。

那么间距呢？为什么我要将 `FROM` 这个词放在新的一行？SQL 并不要求关键字必须换行。

然而，将 `FROM` 关键字（以及许多其他关键字）放在新的一行，将有助于使查询更易于阅读和维护。因此，我推荐你这样编写查询。

### 我们的第一个 SELECT 语句

既然你已经了解了 `SELECT` 语句的语法并解释了一些格式问题，现在让我们来编写你的第一个 `SELECT` 语句。

首先，以 `SELECT` 关键字开始：

```
SELECT
```

然后，添加你想查看的列。在这个例子中，我们将展示此表中所有可用的列。在 SQL 中，要实现这一点，你需要使用星号字符 `*`。在 `SELECT` 关键字后加一个空格，然后添加它。

```
SELECT *
```

这种 `SELECT` 语句的变体通常被称为“全选”或“select star”，因为你选择了所有的列。

接下来，另起一行并添加 `FROM` 这个词。

```
SELECT *
FROM
```

现在你需要指定数据来自哪个表。你可以使用上一章创建的表，即 `employee`。

```
SELECT *
FROM employee
```

最后，加上分号。

```
SELECT *
FROM employee;
```

这就是你的第一个 `SELECT` 语句！

那么行呢？你已经指定了要看所有列，但行呢？

数据库将使用此语句并查找所有符合你提供条件的行。默认情况下，除非你指定了需要检查的条件，否则将返回所有行。你将在本书后面学习如何显示满足特定条件的行，但目前你并未添加任何条件，因此数据库将显示所有行。

你该如何操作呢？你需要使用 SQL 工作表或编辑器在数据库上运行它。

### SQL 中的错误

运行这些查询看起来相当简单，对吧？运行一条语句，数据就会显示。然而，并非所有 SQL 语句都那么简单，也并非所有语句都会给你想要的结果。

偶尔，你运行一个查询，结果不是从数据库返回结果集，而是一个错误消息。

显示错误消息的原因有很多，例如：

*   你的语句存在语法错误，或者写法不正确（例如，关键字放错了位置）。
*   处理数据时遇到问题（例如，某一行中的某个值无法转换为数字）。
*   发生了与数据库相关的错误（例如，连接问题）。

我不知道有谁能运行 SQL 查询而不遇到错误。我们都会在 SQL 上犯错。即使我做这行有一段时间了，我的查询仍然会出错。

你刚开始时遇到的大多数错误都是第一种：查询写得不正确的语法错误。数据相关的错误有时也会发生。其他与数据库相关的错误很少见，通常超出了你的控制范围。

以下小节描述了你在编写 SQL 时会看到的一些更常见的错误。随着你学习本书的深入，你会了解更多关于可能出错的地方，以及如何发现和纠正语句中的问题。

#### 缺少关键字

我们之前的示例查询如下：

```sql
SELECT *
FROM employee;
```

假设你忘记在查询中添加 `FROM` 关键字，你的查询变成这样：

```sql
SELECT * employee;
```

如果你运行该语句，你会得到这个错误：

```
ORA-00923: FROM keyword not found where expected
00923. 00000 -  "FROM keyword not found where expected"
*Cause:
*Action:
Error at Line: 1 Column: 10
```

所有 Oracle 错误消息都以一个三字母代码开头，后跟一个连字符和五个数字。这是错误代码。大多数错误都有 ORA 错误代码，但有些会有 PLS 代码。

示例中的错误代码是 ORA-00923，消息是 “FROM keyword not found where expected”。

第二行显示了错误描述，但它与错误消息相同。通常，会显示原因和操作。但对于此错误，没有显示原因或操作。最后一行指出了发生错误的行和列：在本例中是第 1 行，第 10 列。

这个错误不言自明：“找不到 FROM 关键字”。如果你查看你的 SQL 语句，你会看到没有 `FROM` 关键字，因此该语句无法运行。

#### 表不存在

有时你会有一个类似下面的查询，并得到一个错误：

```sql
SELECT *
FROM emplyoee;
```

```
ORA-00942: table or view does not exist
00942. 00000 -  "table or view does not exist"
*Cause:
*Action:
Error at Line: 2 Column: 6
```

错误提示是 ORA-00942: table or view does not exist。

你可能会想，“这不可能，表确实存在，我刚刚创建的！” 我也犯过很多次同样的错误。这个查询的问题在于，你指定的表实际上并不存在，但这是因为它被拼写错了。查询中提到的是 “emplyoee” 而不是 “employee”。

如果你拼错了表名，就会得到这个错误。但是，此错误也可能由于角色和安全相关的其他一些原因而出现。

```sql
SELECT *
FROM employee;
```

图 4-1 显示了结果，即来自 `employee` 表的所有列和行。幸运的是，在这个例子中列数不多，所以结果不至于让人不知所措。

图 4-1

对 employee 表查询的结果

试想一下，当一个表有几十列甚至可能上百列时！有时我们不希望在查询结果中看到所有列。这可能是因为我们在应用程序的某个特定部分使用查询，而该部分只希望显示部分数据，而不是表中的所有数据。

在 SQL 中我们如何做到这一点？我们在 `SELECT` 子句中指定每一列。这种查询的语法如下：

```sql
SELECT column_name, column_name...
FROM table_name;
```

这个语法看起来与第 3 章中的非常相似。但是，我们不是指定 `*` 字符（表示所有列），而是指定我们要显示的每一列。我们用逗号分隔每个列名。语法末尾的三个点 “...” 仅表示列名语法可以重复。

还有一件事：我用到了一个我们尚未涵盖的术语，即 *子句（clause）*。子句是一行以关键字开头的 SQL 代码，并包含与该关键字相关的所有内容。

到目前为止，我们已经见过两个子句：

*   SELECT 子句，包含 `SELECT` 关键字及其后跟随的列。
*   FROM 子句，包含 `FROM` 关键字及其后跟随的任何内容，例如表名。

在本书中，我们将经常使用“子句”这个术语来指代查询的不同部分。

SELECT id,last_name

### 选择特定列还是选择所有列？

如果你想显示表中的所有列，有两种方法可以实现：

*   使用 `SELECT *`

*   使用 `SELECT` 并单独指定每一列

哪种方法更好？你应该用哪一种？我推荐单独指定每一列。这意味着你的查询应该像这样：

```sql
SELECT id, last_name, salary
FROM employee;
```

而不是像这样：

```sql
SELECT *
FROM employee;
```

我为什么这么推荐？你可能在想，如果 `SELECT *` 更容易输入而且效果一样，为什么我不推荐它。这有几个原因。

首先，只显示你感兴趣或想要查看的列是个好习惯。仅仅因为你“可以”显示所有列，或者仅仅因为它更容易输入，并不是一个好习惯。单独输入列名只需要几秒钟（如果你使用像 `SQL Developer` 这样的程序，速度会更快）。显示所有列可能会导致查询运行变慢，也会使任何使用此查询的应用程序运行变慢，因为需要处理更多的数据。

其次，如果列名发生变化，或者增加了新列，使用 `SELECT *` 会影响查询和结果。具体指定列名则可以减少这个问题。

所以，请尽量单独指定列，而不要使用 `SELECT *`。

### 总结

你可以使用 SQL 从表中选择特定的列。你可以选择一列、多列或表中的每一列。推荐单独指定列，而不是使用 `SELECT *`，因为它更易于维护，并能减轻应用程序和数据库的负载。

## 5. 限制结果

我们刚刚学习了如何显示表中的特定列。在本章中，我们将探讨如何显示表中的特定行。

### 为什么要限制数据？

在之前的例子中，我们查看了表中的所有列，以及表中的一列或两列。其中一个查询是这样的：

```sql
SELECT id, last_name, salary
FROM employee;
```

此查询将显示表中所有员工的 `id`、`last_name` 和 `salary` 列。如果你不想看到表中的所有员工呢？在我们的示例表中，有五条记录。但如果表中有一百条或一千条记录呢？在你可能遇到的工作场所的真实数据库中，拥有超过一百万条记录的表并不罕见。

那么，为什么你只想显示部分记录呢？本质上，如果你有一个需求，只想显示符合特定条件的记录，那么你只需要在数据库中找到这些记录。

一些例子可能是：

*   查找特定员工的所有信息。

*   查找客户的所有订单。

*   查找特定年份的所有学生注册信息。

你可能会想：“我们不能直接获取表中的所有记录，然后让应用程序自己找出需要的吗？”我们可以这样做，但这会非常慢。从表中找出所有数据，将其从数据库发送到应用程序，然后由应用程序丢弃它不需要的记录，这将花费大量的时间和内存。它还会影响网络流量，因为将所有数据发送到应用程序通常会产生大量网络流量，并且比数据库在发送回数据之前检查行要慢得多。

更好的方法是只查找我们想要的数据，这样能更好地利用数据库和时间。幸运的是，在 SQL 中，我们可以使用 `WHERE` 子句轻松做到这一点。

### 什么是 WHERE 子句？

`WHERE` 子句是一个子句，它让你能够过滤查询的数据，只显示符合你指定条件的记录。正如我们在前面章节提到的，子句是 SQL 关键字及其后所有相关内容的组合。

`WHERE` 子句看起来像这样：

```sql
WHERE criteria
```

这个子句看似简单，但功能强大。你将 `WHERE` 子句放入 `SELECT` 查询中，紧跟在 `FROM` 子句之后。例如：

```sql
SELECT columns
FROM table
WHERE criteria;
```

分号被移到了语句的最后，在 `WHERE` 子句之后。

### 使用文本值的示例

让我们看一些例子。我们将从文本值开始。这是我们在上一章运行的查询，显示员工表中的所有记录。

```sql
SELECT id, last_name, salary
FROM employee;
```

我们的结果看起来像这样：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

假设你想查找 `last_name` 为 `SMITH` 的员工的所有列（或字段）。

你可以这样写查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = 'SMITH';
```

这个查询有几个新的组成部分：

*   首先，你添加了 `WHERE` 关键字。

*   然后，你指定了 `last_name` 列。这是因为你想将 `last_name` 列的值与另一个值进行比较。

*   接着，你添加了一个等号 "`=`"。

*   最后，你添加了 `'SMITH'`。这是因为你想查看所有 `last_name` 为 `SMITH` 的记录。

`SMITH` 被放在单引号内，因为这是你要查找的特定值。`SMITH` 这个值被称为“字符串”，这是软件开发中表示文本值的术语。`SMITH` 的值用单引号括起来，是因为你想让数据库知道你希望它被视为文本值，而不是列名。如果你没有使用单引号，你的查询看起来会是这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = SMITH;
```

然后，Oracle 数据库会查找一个名为 `SMITH` 的列，并会显示错误，因为该列不存在。所以，任何时候你要查找文本值，请确保它用单引号括起来：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = 'SMITH';
```

这个查询是如何工作的？Oracle 数据库将运行此查询，并检查员工表中的每一行，看该行的 `last_name` 是否等于 `'SMITH'`。如果是，则该行将包含在显示给你的结果中。如果不是，则不显示。如果你运行这个查询，你的输出将是：

`ID LAST_NAME SALARY`
`2  SMITH 35000`

只显示了一条记录，因为只有一条记录的 `last_name` 值是 `SMITH`。当你使用带有文本值的 `WHERE` 子句时，**值是区分大小写的**。本书前面提到，SQL 不是区分大小写的语言，这意味着 `SELECT` 和 `select` 的功能是相同的。

但是，在处理文本值时，例如 `'SMITH'`，这种检查是区分大小写的。这意味着查找 `'Smith'` 不会显示与查找 `'SMITH'` 相同的结果。

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = 'Smith';
```

```sql
No results found.
```

这是因为表中的值等于 `'SMITH`，这与 `'Smith'` 不同。

#### 注意

文本值（那些在单引号内的）在 SQL 中是区分大小写的。所以，`'SMITH'` 不等于 `'Smith'`。

注意等号周围的空格。它们不是必需的，但没有它们，查询会更难阅读。例如：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name="Smith";
```

这个查询可以执行，但我建议在像等号这样的运算符周围加上空格，以提高可读性。良好地使用空格可以使查询更容易被下一个需要维护它的程序员阅读。


### 数值示例

你并不总是需要按文本值进行筛选。你也可以按数值进行筛选。比如说，你想找出所有薪资为 52000 的员工。你的查询会像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 52000;
```

这个查询与之前的查询有几个不同之处：

*   我们希望查看具有特定薪资的记录，因此我们指定了 `WHERE salary`。

*   我们不关心 `last_name` 的值是什么，所以我们从 `WHERE` 子句中移除了针对该列的任何条件。

*   我们还使用了等号，然后指定了一个数字：52000。这样做将显示所有薪资为 52000 的记录。

你可能会想，“美元符号怎么办？我能添加逗号作为千位分隔符吗？”你可能正尝试运行这样的查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = $52000;
```

或者像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 52,000;
```

好消息是，你不需要输入逗号和美元符号。良好数据库设计的因素之一是每列都具有合适的数据类型。虽然 `last_name` 列存储为文本值，但 `salary` 是存储为数字。

你不能在薪资列中存储货币符号（`$` 或 `€`），因为这意味着该字段不再是数字。然而，你可以将薪资存储为不含货币符号的数字，任何使用此薪资值的应用程序都可以决定如何显示该值（例如，添加货币符号）。

你也不需要存储任何逗号或其他字符作为千位分隔符。这些应该仅用于在报告或应用程序中显示数据，因此显示薪资的位置将决定是否需要任何千位分隔符。在数据库中，我们只存储数字值。你也可以在从数据库 `SELECT` 该列时添加它。

与 SQL 中的文本值不同，你不应将数值括在单引号中。指定 `salary = 52000` 将有效。

总结一下，我们的查询看起来像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 52000;
```

如果你运行查询，结果将如下所示：

```
ID   LAST_NAME   SALARY
4    SIMPSON     52000
```

此结果显示了一条符合你条件的记录：`salary` 等于 52000。没有其他记录具有相同的 `salary` 值。

### 查找不等于某个值的行

到目前为止，你一直在查看精确匹配某些条件的记录。如果你想找到所有不匹配某些条件的记录怎么办？

在 SQL 中，你可以使用一个稍有不同的符号来查找不匹配某些条件的记录，该符号代表“不等于”。“不等于”操作由 `<>` 表示。这个符号是两个尖括号紧挨在一起。

让我们看一个查找不匹配特定条件的记录的示例，你想查看所有 `last_name` 不是 SMITH 的员工。

你的查询会像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name <> 'SMITH';
```

这个查询看起来与之前查找所有名为 SMITH 的员工的查询完全相同，只是我们使用了 `<>` 而不是 `=`。因此，我们新版本的查询将显示所有 `last_name` 不等于 SMITH 的记录。

```
ID   LAST_NAME   SALARY
1    JONES       20000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

最后，还有另一个符号可用于表示不等于：`!=`。感叹号和等号也指定“不等于”。这意味着你的查询也可以这样写：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name != 'SMITH';
```

结果将是相同的：

```
ID   LAST_NAME   SALARY
1    JONES       20000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

如果有两个符号做同样的事情，你该使用哪一个？我建议使用两个尖括号 `<>`，因为它们是 SQL 标准的一部分，而 `!=` 不是。这使他人更容易理解，也便于未来的代码更改。

### 大于

我们已经看了等于和不等于。你还可以使用“大于”检查来查找记录，这意味着你可以找到某列大于特定值的所有记录。大于符号是 `>`，一个指向右边的单尖括号。

假设你想找到所有 `salary` 大于 35000 的 `employee` 记录。你的查询会像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary > 35000;
```

就像查找等于某个数字的值一样，你不应将数字括在单引号中。这适用于你处理数字的所有工作。

这个查询将查看每条记录，确定 `salary` 是否大于 `35000`，如果是，则该记录将显示在结果中。如果不是，该记录将不会被显示。大于符号不包括与你指定的值匹配的值。因此，35000 的值不会被返回，但 35001 的值会被返回。

我们的结果将如下所示：

```
ID   LAST_NAME   SALARY
3    KING        40000
4    SIMPSON     52000
```

结果显示了 `salary` 大于 `35000` 的两条记录。它没有包括 `employee 'SMITH'`，因为他们的 `salary` 正好是 `35000`，并不大于 `35000`。

### 大于等于

如果你想包含那条 `'SMITH'` 记录怎么办？你可以将查询改为 `> 34999`。另一种方法是使用“大于等于”。可以使用 `>=` 符号来包含 `'SMITH'` 记录，该符号是大于符号和等号的组合。

如果你想编写一个查询，显示所有 `salary` 大于等于 `35000` 的 `employees`，它会像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary >= 35000;
```

这个查询将显示以下结果：

```
ID   LAST_NAME   SALARY
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
```

它显示了 `'KING'` 和 `'SIMPSON'`，因为他们超过 35000，也显示了 `'SMITH'`，因为 35000 满足“大于等于 35000”的条件。它没有显示 `'JONES'` 或 `'ANDERSON'`，因为他们的 `salary` 值小于 35000。

### 小于

就像“大于”条件一样，你可以使用“小于”符号来查找小于某个值的记录。这个符号是 `<`。它的用法与大于符号相同。如果你想编写一个查询来显示所有 `salary` 小于 40000 的 `employees`，它会像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary < 40000;
```

你的结果将是：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
5    ANDERSON    31000
```

表中的五条记录里，`'KING'` (40000) 和 `'SIMPSON'` (52000) 没有被显示，因为他们的 `salary` 值不小于 40000。`'KING'` 的 `salary` 值等于 40000，这并不小于 40000。


### 小于等于

SQL 支持“小于等于”检查，其方式与“大于等于”检查类似。它使用小于符号和等于符号的组合，即“`<=`”。要查看所有 `salary` 小于或等于 40000 的员工，可以使用如下查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary <= 40000;
```

该查询的结果如下：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
5    ANDERSON    31000
```

结果显示了 `'JONES'`、`'SMITH'` 和 `'ANDERSON'`，因为他们的 `salary` 都小于 40000。结果显示了 `'KING'`，因为他们的 `salary` 等于 40000。总之，可以使用的不同符号如表 5-1 所示。

**表 5-1 可用的比较运算符**

| 含义 | 符号 |
| --- | --- |
| 等于 | `=` |
| 不等于 | `<>` 或 `!=` |
| 大于 | `>` |
| 大于等于 | `>=` |
| 小于 | `<` |
| 小于等于 | `<=` |

### 最佳实践

SQL 中的 `WHERE` 子句非常灵活，正如你在本章所见，整个 SQL 语言也是如此。你的查询的格式和结构可以与其他人编写显示相同结果的查询方式不同。

对于改进你对 `WHERE` 子句及整个查询的使用，我可以提供一些“最佳实践”或提示。这将确保你的查询易于阅读，并且易于你自己和他人将来理解。

#### 将列名放在左侧

在编写 `WHERE` 子句时，你应该将列名放在左侧，将值或其他列放在每个比较语句的右侧。这使代码保持一致且更易于阅读。这意味着你的查询应该这样写：

```sql
WHERE last_name = 'SMITH'
```

而不是：

```sql
WHERE 'SMITH' = last_name
```

两个 `WHERE` 子句的作用相同，但第一个遵循了更常见的格式，我认为更具可读性。大多数有 SQL 经验的程序员都期望列名在左侧，字面值在右侧。

#### 只选择你需要的

在 `WHERE` 子句中提到的列不必在 `SELECT` 子句中列出。你可以在不需要将数据发送回应用程序的情况下，基于某个值进行筛选。

例如，假设你想查看所有 `salary` 小于 40000 的员工 `ID`。你可以在 `SELECT` 和 `WHERE` 子句中都列出 `salary`：

```sql
SELECT id, salary
FROM employee
WHERE salary < 40000;
```

然而，通过在 `SELECT` 子句中列出 `salary` 列，当应用程序不需要这些值时，你仍然会导致所有 `salary` 值通过网络发送，或者可能暴露需要保护且保密的数据。

一个更高效的查询版本如下：

```sql
SELECT id
FROM employee
WHERE salary < 40000;
```

数据库引擎仍然可以基于 `salary` 列筛选结果。但是，只有 `ID` 值会通过网络发送到应用程序。网络流量减少了，你的应用程序用户也不会等待不需要的数据被发送，或者不会将 `salary` 数据显示给不应看到的人。

### 总结

你可以使用 `WHERE` 子句来筛选结果，它涉及 `WHERE` 关键字和一些条件。这意味着你可以只显示你感兴趣的记录。使用 `WHERE` 子句筛选数据可以通过多种方式完成，使用诸如等于、不等于、大于和小于等符号。

## 6. 比较数据

你刚刚学习了如何限制要显示的行，使用了多种方法，例如等于或大于。这些方法中的每一种都将一个列与另一个值进行比较。这个另一个值是通过“精确匹配”来检查的。检查时使用的是你指定的确切值。

例如，考虑这个 `WHERE` 子句：

```sql
WHERE last_name = 'SMITH'
```

这个 `WHERE` 子句将返回所有 `last_name` 等于 `'SMITH'` 的行。它不会返回 `last_name` 是以下任何值的记录：

*   Smith
*   SMITHERS
*   SMITHSONIAN
*   Naismith
*   SMIT

这就是精确匹配的作用。但如果你想匹配部分值呢？

### 什么是部分匹配？

在之前的例子中，你查找了所有 `last_name` 为 `'SMITH'` 的 `employee` 记录。如果你想查找所有 `last_name` 以字母 S 开头的 `employee` 记录呢？

你可以编写一个如下查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = 'S';
```

如果你运行这个查询，会得到以下结果：

```sql
No results found.
```

你没有得到任何结果，因为没有任何 `last_name` 值等于 `'S'`。“等于”和“以...开头”在 SQL 及一般的编程中是不同的概念。能够找到“以”某个值“开头”的记录的能力被称为*部分匹配*。

部分匹配是能够匹配部分值的能力。它不是检查一个值是否完全等于另一个值，而是通过部分匹配我们可以检查：

*   以某个值开头
*   以某个值结尾
*   包含某个值
*   其他值的组合

你可以通过在 SQL 中使用 `LIKE` 关键字来执行部分匹配。

### LIKE 的语法

`LIKE` 关键字可以在 `WHERE` 子句中代替等号使用：

```sql
WHERE column LIKE compare_string
```

你从像之前见过的 `WHERE` 子句开始，然后指定你想要查找的列。接着使用 `LIKE` 关键字并指定一个 `compare_string`，即你要查找的值。

如何编写这个比较字符串？你需要使用所谓的*通配符字符*。通配符字符是用来代表任何其他字符的字符。通过一个例子可以更好地解释这一点。

### 百分号通配符

在前面的例子中，您想查找所有以 “S” 开头的 `last_name`。为此，您可以使用百分号通配符，即 “%”。它代表“零个或多个字符”，即任何字母、数字或符号。

您可以使用这样的 `WHERE` 子句：

```sql
WHERE last_name LIKE 'S%'
```

这将查找 `last_name` 值以 “S” 作为第一个字符，然后后面是零个或多个字符的记录，无论这些字符是什么。

这将显示如下值：

*   SMITHERS
*   Smith
*   SIMPSON
*   Stanley
*   Sum
*   So

您能看出规律吗？所有这些值都以 “S” 开头，后面有零个或多个字符。实际跟随的字符无关紧要，只需有一个或多个即可。

您可以编写查询来显示此数据：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE 'S%';
```

您的结果将是：

```sql
ID   LAST_NAME   SALARY
2    SMITH       35000
4    SIMPSON     52000
```

您可以看到两个结果，一个是 `'SMITH'`，一个是 `'SIMPSON'`。两者都以 “S” 开头。另外有两个记录也包含 “S”（`'ANDERSON'` 和 `'JONES'`），但它们没有被显示，因为您的条件是 “S%”，意味着 `last_name` 需要以单个 S 开头，后面是其他字符。值 `'ANDERSON'` 和 `'JONES'` 不以 S 开头。

如果您想显示那些记录呢？如果您想查找所有 `last_name` 中包含 “S” 的记录呢？您可以通过在 S 前面和后面都放置 % 符号来实现：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S%';
```

您的结果将如下所示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
4    SIMPSON     52000
5    ANDERSON    31000
```

显示这些值是因为它们在 `last_name` 值的任何位置都包含一个 S。它可能在开头、中间或结尾。

您也可以查找以 S 结尾的值，通过在比较字符串的开头放置一个 “%” 符号但不在结尾放置。您的查询将如下所示：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S';
```

我们的结果将如下所示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
```

如前所示，百分号通配符对于在字符串的任何位置查找一个或一组字符很有帮助。

### 下划线通配符

Oracle SQL 中可用的另一个通配符是下划线字符 “_”。下划线字符用于表示零个或一个任意类型的字符。它与百分号字符类似，后者表示零个或多个字符，区别在于下划线在使用通配符的位置最多允许一个字符。

假设您有这样的 `WHERE` 子句：

```sql
WHERE last_name LIKE 'SA_'
```

您的结果将包括：

*   SAM
*   SAL
*   SAr
*   SA

然而，您的结果**不会**包括：

*   SALL
*   SANDERS
*   SAWS
*   Sam

这是因为下划线的位置最多只允许一个字符，而第二组例子有两个或更多字符（或者不是以 “SA” 开头）。

假设您想查找每个 `last_name` 是单个字符后跟 `'ING'`。您的查询将如下所示：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '_ING';
```

结果将是：

```sql
ID   LAST_NAME   SALARY
3    KING        40000
```

此结果显示 `'KING'` 的值，因为它包含一个单个字符，后跟字符 `'ING'`。如果表中有 `last_name` 值为 `'BING'` 或 `'LING'` 的记录，它们也会被显示。

您也可以混合使用百分号和下划线字符，或任意数量的这两种字符来查找您想要的记录。假设您想查找所有 `last_name` 中包含 ‘SO’ 但后面只有一个字符的记录。您的查询将是：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%SO_';
```

`WHERE` 子句将 `last_name` 与 `'%SO_'` 进行比较，这代表任意数量的字符，然后是字符 SO，然后是零个或一个字符。您的结果将是：

```sql
ID   LAST_NAME   SALARY
4    SIMPSON     52000
5    ANDERSON    31000
```

`'SIMPSON'` 和 `'ANDERSON'` 都被显示，因为它们包含字符 `'SO'` 并且后面有一个字符。

假设您想查找所有 `last_name` 包含字母 I 作为第二个字符并且在那之后任何位置包含字母 N 的记录。您的查询将是：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '_I%N%';
```

这个比较字符串 `'_I%N%'` 可以解释为：

*   一个下划线，代表任意单个字符
*   一个 I，位于第二个位置
*   一个 %，代表零个或多个任意字符
*   一个 N，需要在 I 之后但在任何位置
*   一个 %，代表 N 之后的零个或多个任意字符

此查询的结果将是：

```sql
ID   LAST_NAME   SALARY
3    KING        40000
4    SIMPSON     52000
```

`'KING'` 和 `'SIMPSON'` 都符合 `WHERE` 子句条件。`'KING'` 中的 N 紧跟在 I 之后，`'SIMPSON'` 中的 N 在值的末尾。


### 搜索通配符字符

在本章中，你已经学习了如何使用通配符字符 `%` 和 `_` 来查找部分匹配某个值的记录。这些 `%` 和 `_` 字符在查询运行时会被其他字符替代。但是，如果列中的值实际包含 `%` 字符或 `_` 字符怎么办？

假设你的 `employee` 表中存在一个包含下划线的值。这条记录可能是由应用程序在开发过程中放置在那里的，你需要了解更多相关信息。该记录看起来像这样：

```
ID   LAST_NAME            SALARY
11   confirmation_needed  (null)
```

“confirmation_needed” 这个 `last_name` 并不是一个有效的 `last_name`。你想编写一个查询来查找所有 `last_name` 中包含下划线字符的 `employees`。你可以这样写查询：

```
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%_%';
```

当你运行这个查询时，会得到以下结果：

```
ID   LAST_NAME           SALARY
1    JONES               20000
2    SMITH               35000
3    KING                40000
4    SIMPSON             52000
5    ANDERSON            31000
11   confirmation_needed 0
```

这显示了表中的所有记录。查询正在搜索 ‘%_%’，这意味着“所有字符，后跟一个字符，再后跟所有字符”。如何搜索包含下划线的所有字符呢？你需要指定下划线应被视为一个值，而不是通配符字符。这需要对字符进行*转义*。

转义字符意味着你希望该字符被视为列值的一部分，而不是一个特殊符号（例如通配符）。在包括 SQL 在内的许多编程语言中都可以对字符进行转义。要在 SQL 中转义一个字符，你需要做两件事：

*   在要转义的字符前立即指定一个字符
*   在你的 `WHERE` 子句中指定这个转义字符是什么

常用的转义字符是反斜杠字符 ‘\’，因为它很少在你的数据中使用。如果确实使用了，你可以选择另一个字符。

要在查询中使用转义字符，你需要这样编写查询：

```
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%\_%' ESCAPE '\';
```

这个查询与前一个查询相比有两个变化。首先，在 `LIKE` 关键字后的字符串中有一个 \ 字符。这个字符表示后续字符（下划线）被“转义”。其次，在 `WHERE` 子句末尾添加了一个关键字：`ESCAPE` ‘\’。这意味着转义字符被定义为 ‘\’ 字符。如果没有添加这个 `ESCAPE` 关键字，Oracle 将不会知道 `LIKE` 字符串中的 ‘\’ 是什么意思。

这个更新后的查询结果是：

```
ID   LAST_NAME           SALARY
11   confirmation_needed 0
```

如你所见，更新后的查询将只显示 `last_name` 值包含下划线的记录。

### 性能

关于 `LIKE` 关键字需要了解的另一点是，使用该关键字的查询性能通常不是很好。

如果你使用精确匹配，Oracle 通常可以扫描列中的所有值并找到匹配的值。然而，使用带有 `LIKE` 关键字的部分匹配需要更长时间，因为 Oracle 需要查看每个值并应用你的条件。

我们不会在本书中过多讨论 Oracle SQL 的性能，但我想指出，`LIKE` 关键字的性能表现并不好。如果确实需要，你应该使用它；如果可以用其他方法（例如“等于”）实现，就使用其他方法。

### 总结

`=` 符号和其他符号用于执行精确匹配或完全匹配指定值。在 SQL 中，另一种选择是部分匹配，它匹配值的一部分。这些值的部分可以通过 `LIKE` 关键字和使用通配符 `%` 和 `_` 来指定。

## 应用多个过滤器

在前面的章节中，你已经学习了如何限制你想查看的记录。你研究了使用诸如“等于”和“大于”之类的符号。你还学习了如何使用 `LIKE` 关键字和通配符字符进行部分匹配。

这些概念很有用。然而，你可能已经注意到，我们在每个查询中只按一个条件进行了过滤。我们基于特定的 `last_name` 值或特定的 `salary` 值进行过滤。如果我们想基于两个或更多条件进行过滤呢？

### 为什么使用多个过滤器？

为什么要在 SQL 查询中使用多个过滤器？这完全取决于查询的类型或你想要的结果。对于我们正在使用的示例，使用 `WHERE` 子句执行单个过滤可能没问题，但当你处理更多数据或想要显示不同的结果时，你就需要学习如何使用多个过滤器。

一些例子可能是：

*   查找特定客户在某个日期范围内的所有订单
*   查找所有活跃且处于特定状态的学生
*   查找所有在线订购且订单金额超过特定数额的客户

每种这样的查询都在查看两个或更多条件。你将学习如何使用你的示例数据来实现这一点。

### AND 关键字

SQL 允许你在 `WHERE` 子句中使用多个条件，以将数据限制为你想要查看的记录。有两种方法可以做到这一点。第一种方法是使用 `AND` 关键字。

`AND` 关键字让你可以在 `WHERE` 子句中指定两个条件。如果一条记录满足这两个条件，它就会被显示。如果一条记录满足零个或一个条件，那么它就不会被显示。如果你想检查查询中的多个条件，并且两个条件都需要匹配某一行，那么 `AND` 关键字非常有用。

使用 `AND` 关键字如下所示：

```
WHERE condition
AND condition
```

这可以分解为：

*   以 `WHERE` 关键字开头，表示 `WHERE` 子句的开始。
*   `condition`：一行要被显示所必须满足的条件（例如，`last_name = 'SMITH'`）
*   `AND`：表示一行还必须满足第二个条件。
*   `condition`：如前所述，一行要被显示所必须满足的条件

让我们看一些使用 `AND` 关键字的多条件示例。

#### 查找匹配姓氏和薪资的记录

假设你想查找 `last_name` 包含字母 “S” 且 `salary` 大于 32000 的记录。你的查询将如下所示：

```
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S%'
AND salary > 32000;
```

在这个查询中，`AND` 被写在新行上以使其更易于阅读。即使你将 `AND` 与 `WHERE` 放在同一行，查询仍然会运行（你可能更喜欢那样写）。这个查询的结果将是：

```
ID   LAST_NAME   SALARY
2    SMITH       35000
4    SIMPSON     52000
```

此查询显示了 `'SMITH'` 和 `'SIMPSON'` 的记录，因为它们的 `salary` 都大于 32000 且 `last_name` 中都包含 “S”。`'JONES'` 和 `'ANDERSON'` 的记录在 `last_name` 中都有 “S”，但他们的 `salary` 小于 32000。`'KING'` 的记录 `salary` 大于 32000，但 `last_name` 中没有 “S”。因此，只显示了 `'SMITH'` 和 `'SIMPSON'`。



#### 使用 AND 关键词对两列进行大于和小于查询以查找记录

使用 `AND` 关键词的另一个例子是使用除等于和 `LIKE` 之外的其他比较运算符。假设你想查找 `ID` 大于或等于 3 且 `salary`（工资）小于 40000 的记录。你的查询语句会是这样的：

```sql
SELECT id, last_name, salary
FROM employee
WHERE id >= 3
AND salary < 40000;
```

结果会是这样：

```
ID   LAST_NAME   SALARY
5    ANDERSON    31000
```

这是唯一显示的记录，因为尽管 `KING` 和 `SIMPSON` 的 `ID` 大于或等于 3，但他们的 `salary` 并不小于 40000。

### OR 关键词

在查询中使用多个筛选条件的第二种方法是使用 `OR` 关键词。`OR` 关键词的使用方式与 `AND` 关键词相同。然而，区别在于 `OR` 表示两个条件中至少有一个需要满足，而不像 `AND` 关键词那样需要两个条件都满足。

使用 `OR` 关键词的语法如下：

```sql
WHERE condition
OR condition
```

`OR` 关键词的工作方式与 `AND` 关键词相同。它用在 `WHERE` 子句中，你可以用它来指定第二个要检查的条件。如果对于某一行，两个条件中的任何一个（或两个）为真，那么该行就会包含在结果中。`OR` 关键词对于检查多个条件并查看至少有一个条件为真的结果非常有用。

#### 使用 OR 关键词按姓氏或工资查找记录

让我们看看之前使用 `OR` 而不是 `AND` 重写的例子。你想查找 `last_name`（姓氏）包含字母“S”，或者 `salary` 大于 32000 的记录。你的查询语句会是这样的：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S%'
OR salary > 32000;
```

你的结果会是这样：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

结果显示了表中的每一条记录。`last_name` 为 `'JONES'`、`'SMITH'`、`'SIMPSON'` 和 `'ANDERSON'` 的记录被显示，因为它们 `last_name` 中都包含“S”。`'KING'` 的记录被显示是因为他们的 `salary` 大于 32000。有些记录只满足两个条件中的一个（`last_name` 中包含“S” 或 `salary` 大于 32000），但这没关系；它们仍然被包含在结果中。

#### 使用 OR 关键词对两列进行大于或小于查询以查找记录

让我们看另一个使用 `OR` 的例子。你可能想查看所有 `ID` 大于或等于 4，或者 `salary` 小于 40000 的记录。

```sql
SELECT id, last_name, salary
FROM employee
WHERE id >= 4
OR salary < 40000;
```

结果会是这样：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
4    SIMPSON     52000
5    ANDERSON    31000
```

我们可以解释表中每条记录的处理方式：

*   `JONES`：显示是因为 `salary` 小于 40000。
*   `SMITH`：同样显示是因为 `salary` 小于 40000。
*   `KING`：未显示，因为 `salary` 不小于 40000 且 `ID` 不大于或等于 4。
*   `SIMPSON`：显示是因为 `ID` 大于或等于 4。
*   `ANDERSON`：显示是因为 `ID` 大于或等于 4 且 `salary` 小于 40000。

### 超过两个条件

你已经看到几个关于 `AND` 和 `OR` 带有两个条件的例子。如果你想使用超过两个条件怎么办？你可以在 SQL 中使用 `AND` 和 `OR` 来做到这一点。你只需在 `WHERE` 子句末尾添加另一个 `AND` 或 `OR`：

```sql
WHERE condition
[{AND|OR} condition]
[{AND|OR} condition]
```

当你阅读 SQL 文档时，会看到类似的语法。方括号 `[` 和 `]` 表示其中的部分是可选的。花括号 `{` 和 `}` 表示其中的部分是必需的。最后，竖线 `|` 表示必须选择其中一个选项。

因此，这个语法意味着你需要有一个 `WHERE` 条件，然后你可以有一个 `AND` 或 `OR` 条件。如果你确实有一个 `AND` 或 `OR` 条件，你需要写成 `AND condition` 或 `OR condition`。

你可以在查询中添加更多 `AND` 或 `OR` 条件。你还可以在查询中使用 `AND` 和 `OR` 的不同组合。

#### 多个 AND 条件

让我们看一些例子。你可能想查看所有 `last_name` 包含“S”且 `salary` 大于 30000 并小于 40000 的 `employees`（员工）。你的查询语句会是这样的：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S%'
AND salary > 30000
AND salary < 40000;
```

你的结果会是这样：

```
ID   LAST_NAME   SALARY
2    SMITH       35000
5    ANDERSON    31000
```

这显示了 `'SMITH'` 和 `'ANDERSON'`，因为他们满足查询中的所有三个条件：

*   `last_name` 包含“S”。
*   `salary` 大于 30000。
*   `salary` 小于 40000。

#### 多个 OR 条件

让我们看一个使用多个 `OR` 关键词的例子。假设你想查看所有 `salary` 小于 25000，或 `salary` 大于 50000，或 `last_name` 包含“G”的 `employee` 记录。你的查询语句会是这样的：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary < 25000
OR salary > 50000
OR last_name LIKE '%G%';
```

此查询的结果将是：

```
ID   LAST_NAME   SALARY
1    JONES       20000
3    KING        40000
4    SIMPSON     52000
```

`'JONES'` 的记录显示是因为 `salary` 小于 25000。`'KING'` 的记录显示是因为 `last_name` 包含 G，`'SIMPSON'` 显示是因为 `salary` 大于 50000。对于一行要显示，只需要其中一个条件为真。

#### 组合多个 AND 和 OR 条件

如果你想混合使用这些关键字，在一个查询中同时放入 `AND` 和 `OR` 呢？你可以这样做。假设你有这个查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%S%'
AND salary > 30000
OR salary < 40000;
```

你有 `condition1 AND condition2 OR condition3`。这些结果会是什么样子呢？它会是这样：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
4    SIMPSON     52000
5    ANDERSON    31000
```

如何得到这个结果？`AND` 和 `OR` 关键字按照特定的顺序进行求值。这被称为*运算顺序*（order of operations）。这个概念适用于 SQL 的多个领域，你可能在数学课上也记得它，当时一些运算比其他运算具有更高的优先级。

`AND` 和 `OR` 都具有相同的优先级。它们按顺序使用。因此，对于这个查询，逻辑是显示满足以下条件的记录：

*   `last_name` 包含 S 且 `salary` 大于 30000
*   或者，`salary` 小于 40000

现在这是两个独立的条件。`'SMITH'`、`'SIMPSON'` 和 `'ANDERSON'` 的记录被显示，因为它们满足第一个标准：`last_name` 包含 S 且 `salary` 大于 30000。`'JONES'` 被显示是因为它的 `salary` 小于 40000。`'KING'` 未被显示。



### 使用多个条件指定顺序

如果你想按特定顺序检查条件怎么办？你可以使用括号：“(” 和 “)”。在查询中添加括号将强制查询以某种方式运行。括号内的操作先运行，然后再与括号外的操作组合。让我们看这个例子：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%N%'
AND salary > 50000
OR last_name LIKE '%T%';
```

这将显示符合以下条件的记录：

*   `last_name` 包含 “N” 且 `salary` 大于 50000
*   或者，`last_name` 包含 “T”。

这将显示以下结果：

```sql
ID   LAST_NAME   SALARY
2    SMITH       35000
4    SIMPSON     52000
```

如果你想显示一组不同的记录，你会这样编写查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE '%N%'
AND (salary > 50000
OR last_name LIKE '%T%');
```

注意括号围绕着两个条件和 `OR` 关键字。这意味着这些条件先运行，然后与第一个条件组合。这将显示符合以下条件的记录：

*   `salary` 大于 50000 或 `last_name` 包含 “T”
*   并且 `last_name` 包含 “N”。

此查询的结果将是：

```sql
ID   LAST_NAME   SALARY
4    SIMPSON     52000
```

每条记录都满足这些条件。如果在 `WHERE` 子句中没有括号，结果就会不同。`'SMITH'` 的记录在第一个结果中显示而不在第二个中显示的原因在于，第一个查询显示了所有 `last_name` 包含 “T” 的记录，而第二个查询只显示了 `last_name` 包含 “T” 且 `salary` 大于 50000 的记录。

### 总结

你可以通过组合使用 `AND` 和 `OR` 关键字在 SQL 查询中使用多个筛选器。`AND` 关键字将检查记录是否满足指定的所有条件，而 `OR` 关键字将检查记录是否满足指定的至少一个条件。查询中可以添加许多 `AND` 和 `OR` 关键字，它们按书写顺序运行。如果你想确保它们按特定顺序运行，可以将条件和关键字用括号括起来。

## 8. 处理 Null 值

在本章中，你将学习 SQL 和数据库中称为“null”的概念。在你一直使用的示例表中，每条记录的每个列都有一个值。然而，在处理真实数据库时，情况并非总是如此。“null”这个概念用于数据未知的情况。

### 缺失数据

在真实数据库中，可能会出现数据缺失的情况。这可能是故意的（我们不知道值是什么），或者是某种错误（本应有值却没有）。数据库中的表不要求你在每一行每一列都存储一个值。

到目前为止，我们在本书中一直使用一个名为 `employee` 的示例表。该表如下所示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

该表有五行和三列。每行的每个列都有一个值。然而，情况并非总是如此。值可能会缺失。如果某个 `employee` 的 `salary` 未知怎么办？或者我们表中还有其他列，例如 `date of birth`，而某些 `employee` 的这些信息未知？

这是处理数据库时的一种有效场景。问题是：我们如何存储它？假设你有一个新 `employee`，但你目前不知道他们的 `salary` 是多少。可以这样表示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
6    COOPER      Unknown
```

在这个例子中，我们使用了一个文本值 “Unknown”，来指明我们不知道新 `employee` 的 `salary` 值。但这并不是一个好主意。

首先，`salary` 列是一个数字列，这意味着它只存储数字而不是文本值。在这个数字列中使用文本值 “Unknown” 会在尝试添加时导致错误。

你不应该为了允许出现这个 “Unknown” 值而将 `salary` 列的类型更改为文本值。嗯，你可以这样做，但那样你可以对数据库中的每个字段都这样做，这将意味着数据类型完全失去作用。

如果你使用 “n/a” 的值呢？出于同样的原因，这也不行。将文本值添加到数字列是不可能的。而且，有什么理由说 “n/a” 的值在这个字段或其他字段中不是合法值呢？它可能对 `salary` 无效，但如果有一个名为 `commission` 的列存储员工的佣金金额呢？“n/a” 的值可能意味着佣金不适用于该特定员工。

如果你使用 0 呢？你可以使用 0，但 0 值表示“`salary` 是零”。你想捕捉的事实是 `salary` 未知，而未知与 0 是不同的。如果你使用 0，你将不得不调整所有查看 `salary` 列的查询，忽略那些 `salary` 为 0 的记录，因为它会扭曲任何总计或平均值。

如果你使用一个空格字符 “ ” 呢？这是一个选项，但它面临着与 “Unknown” 和 “n/a” 相同的问题，因为它是一个试图添加到数字字段的文本值。

使用空字符串怎么样，这意味着引号内没有任何内容：‘ ’？你可以试试这个，但空字符串在 SQL 中的处理方式不同。它仍然是一个文本值，这意味着在尝试用于数字或日期字段时会有问题。

所以，你不能使用像 “n/a”、或 0、或 “ ”、或空字符串这样的文本值。你能用什么？你可以使用 `NULL` 值。

### 什么是 NULL？

`NULL` 这个词是一个关键字，意思是“一个未知值”。它用于表示数据库表中的未知或缺失值。提供这个关键字是因为未知值不同于空字符串、空格、零或其他可能有含义（如 “n/a” 或 “FALSE”）的文本值。

`NULL` 值在数据库中使用相当频繁，因为无法保证所有字段总是有数据。在数据库中使用 `NULL` 值（相对于零或空字符串）有什么优势？

*   `NULL` 适用于任何列类型：文本、数字、日期。你不需要为了使用它而更改列的类型。
*   `NULL` 有自己的一套用于执行逻辑的关键字，因为它是一个特殊值。
*   有一系列函数可以与 `NULL` 一起使用。
*   它不占用存储空间，这对于小表来说可能不是问题，但对于大表可能有帮助。

### 应该如何处理 NULL？

`NULL` 值需要特殊的处理方式，这可能是它们唯一的缺点。在本书中，你将学习如何处理 `NULL` 值，但你将结合目前所学查看一些与 `NULL` 值相关的例子和处理方式。

你刚刚学习了如何使用等于和大于等符号检查列值是否与其他值匹配。这对于文本、数字和日期值都适用。但是，它不适用于 `NULL` 值。一个值不能“等于” `NULL`，因为 `NULL` 是一个未知值。要检查一个值是否是 `NULL`，我们使用关键字 `IS NULL`。你的 `WHERE` 子句应该像这样：

```sql
WHERE salary IS NULL;
```

如果你使用等号来写，它不会找到任何内容：

```sql
WHERE salary = NULL;
```

让我们看一些例子来解释这一点。

### 隐藏空值

你刚刚看到了如何在结果中显示包含 `NULL` 值的记录。如果你想忽略它们该怎么办？你可以使用一个类似的关键词。就像不能使用“等于”来查找 `NULL` 值一样，也不能使用“不等于”来忽略 `NULL` 值。

要使用的关键词是 `NOT`。`NOT` 是 SQL 中的一个关键字，用于执行与它所修饰的关键字相反的操作。它在 SQL 的多个地方使用，你将在本书中学习到。目前，它可以用来忽略 `NULL` 值。你的查询将如下所示：

```
SELECT id, last_name, salary
FROM employee
WHERE salary IS NOT NULL;
```

注意你在 `IS` 后面添加了 `NOT`，所以是 `IS NOT NULL`。这意味着将显示所有 `salary` 不是 `NULL` 的记录。你的结果是：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
```

这些结果不包括那些 `salary` 为 `NULL` 的记录。你可以对 `last_name` 列执行相同的操作：

```
SELECT id, last_name, salary
FROM employee
WHERE last_name IS NOT NULL;
```

```

1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
6    COOPER      NULL
```

你可以看到 `ID` 7 没有被显示，因为它的 `last_name` 是 `NULL`。`ID` 为 6 的记录被显示了，因为 `last_name` 有值。虽然 `salary` 是 `NULL`，但你的查询没有提到 `salary`。

你可以使用 `AND` 关键字来排除这两个列中任一列为 `NULL` 的记录。

```
SELECT id, last_name, salary
FROM employee
WHERE last_name IS NOT NULL
AND salary IS NOT NULL;
```

此查询的结果是：

```

1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000

```

你可以看到，两个包含 `NULL` 值的记录没有显示出来。

### 什么是 DISTINCT？

SQL 中有一个名为 `DISTINCT` 的关键字，它允许你从结果中删除重复的行。`DISTINCT` 关键字不会从数据库中删除数据。它只是在运行 `SELECT` 查询时识别任何相同的记录，并从你的结果中删除任何重复的行。`DISTINCT` 关键字放在 `SELECT` 关键字之后：

```
SELECT DISTINCT columns
FROM table...
```

这个 `DISTINCT` 关键字将应用于整个查询。你不能为每个列指定 `DISTINCT`，因为它删除的是结果中的重复记录，而不是每个列中的重复值。这是一个重要的区别，你很快就会看到一些这样的例子。

#### 注意

`DISTINCT` 关键字从你的结果中删除重复的行，而不是每个列中的重复值。

### 查找唯一的 Last Name 值

你可以使用前面的例子来查找所有唯一的 `last_name` 值，而不需要在 `WHERE` 子句中使用 `ID`。你的查询将如下所示：

```
SELECT DISTINCT last_name
FROM employee;
```

结果将如下所示：

```
LAST_NAME
JONES
SMITH
KING
SIMPSON
ANDERSON
COOPER
(null)
PATRICK
```

你可以看到重复的记录（第二个 `last_name` 为 SMITH）没有显示。这就是 `DISTINCT` 关键字的作用。


### 查找唯一的薪资值

我们来看另一个例子。假设你需要找出表中所有唯一的 `salary` 值。这将展示 `DISTINCT` 关键字不仅对文本值有效，也适用于数值。你的查询可能一开始是这样的：

```sql
SELECT salary
FROM employee;
```

这个查询的结果是：

```
SALARY

(null)
(null)
```

这向你展示了两件事。首先，这一列中有两个 `NULL` 值。其次，薪资为 40000 的值重复了，因为有两名 `employee` 的薪资相同。你可以用 `DISTINCT` 来解决这个问题。

```sql
SELECT DISTINCT salary
FROM employee;
```

这个查询的结果是：

```
SALARY

(null)
```

使用 `DISTINCT` 消除了一条 `NULL` 记录和一条 40000 的记录。如果你不想包含 `NULL` 值该怎么办？你可以将 `DISTINCT` 关键字与上一章学过的 `WHERE` 子句结合使用。

```sql
SELECT DISTINCT salary
FROM employee
WHERE salary IS NOT NULL;
```

你的结果会像这样：

```
SALARY
```

这显示了所有唯一的 `salary` 值，但排除了 `NULL` 值。

### 查找唯一的值组合

让我们看另一个例子。假设你被要求找出此表中所有唯一的 `last_name` 和 `salary` 值组合。你的查询会是这样的：

```sql
SELECT DISTINCT last_name, salary
FROM employee;
```

注意 `DISTINCT` 关键字只出现了一次。你为整个查询定义了它，而不需要为每一列都定义一次。如果你运行这个查询，结果如下：

```
LAST_NAME   SALARY
JONES       20000
SMITH       35000
KING        40000
SIMPSON     52000
ANDERSON    31000
COOPER      (null)
(null)      (null)
SMITH       62000
PATRICK     40000
```

你可能会疑惑为什么 `'SMITH'` 出现了两次。还有为什么 `salary` 为 40000 的值也出现了两次？这是因为它们可能是重复的值，但它们不是重复的行。薪资为 35000 的 `'SMITH'` 行与薪资为 62000 的 `'SMITH'` 行是不同的行，因此该行保留在结果中。

薪资为 40000 的 `'KING'` 行和薪资为 40000 的 `'PATRICK'` 行也是如此。尽管 `salary` 相同，但 `last_name` 不同，所以行也不同。

`DISTINCT` 的设计目的不是为了从不同行的不同列中删除重复值。如果它这样做，那么每行中的数据就会被混淆，从而变得不正确。如果你需要通过消除重复项来在查询中查找唯一结果，那么 `DISTINCT` 就是实现这一目标的方法。

如何区分两个 `'SMITH'` 员工是不同的人？你可以查看 `ID` 列。或者，你可以添加一个新列来定义更多关于他们的信息，这将在后面的章节中学习。

### 总结

SQL 不会自动从你的结果中消除重复记录。你可以在查询中通过在 `SELECT` 关键字后添加 `DISTINCT` 关键字来实现这一点。它移除的是重复的记录，而不是重复的列，这两个概念截然不同。

## 10. 对值列表和范围应用过滤器

在前面的章节中，你学习了 `WHERE` 子句以及如何使用它来限制你看到的结果。你可以使用各种符号（称为 `运算符`）来实现这一点，例如“等于”和“小于”。你还可以使用 `AND`/`OR` 关键字来指定多个条件。

处理值列表带来了一个有趣的挑战：一个很长的列表可能导致查询中出现大量的 `AND` 和 `OR` 关键字。本章介绍了一些在处理大型列表时很有帮助的语法。你还将学习一些用于范围的语法。

### 连词过多

在 `WHERE` 子句中编写包含一长串相等条件的查询会变得很繁琐。让我们通过一个例子来说明，这将为你接下来要学习的语法做铺垫。

首先，到目前为止，我们的示例数据如下：

```
ID  LAST_NAME SALARY
1   JONES     20000
2   SMITH     35000
3   KING      40000
4   SIMPSON   52000
5   ANDERSON  31000
6   COOPER    (null)
7   (null)    (null)
8   SMITH     62000
9   PATRICK   40000
```

如果你想找出所有 `salary` 等于 5,000 或 10,000 倍数的记录，你可能会有这样的查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 20000
OR salary = 35000
OR salary = 40000;
```

如果你运行这个查询，得到的结果是：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
9    PATRICK     40000
```

我们忽略了薪资为 52000、31000 和其他值的记录。你得到了想要的结果，但我们的 `WHERE` 子句包含了针对同一列的三个检查：

```sql
WHERE salary = 20000
OR salary = 35000
OR salary = 40000;
```

如果你的表中有更多值，并且想检查五到十个不同的值怎么办？你可以使用更多 `OR` 关键字，但这会让查询看起来有点乱：

```sql
WHERE salary = 20000
OR salary = 35000
OR salary = 40000
OR salary = 45000
OR salary = 50000
OR salary = 55000
OR salary = 60000
OR salary = 65000;
```

有一种方法可以改进这种重复使用大量 `OR` 关键字的情况：使用 `IN` 关键字。

### IN 关键字

`IN` 关键字允许你在单个比较中指定多个值。这个关键字的工作方式与使用多个 `OR` 关键字相同。`IN` 子句看起来像这样：

```sql
WHERE column IN (value1, value2... value_n)
```

使用 `IN` 关键字来代替“等于”运算符。然后你指定一个左括号 "("，并在括号内包含所有你想要检查的值，每个值之间用逗号分隔。最后，你关闭括号。

#### 使用 IN 的示例

你可以在 `IN` 关键字内指定一个很长的值列表（实际上最多可达 1,000 个）。这种指定列表的能力对于你正在查看的示例非常有用。与其使用像这样的查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 20000
OR salary = 35000
OR salary = 40000;
```

你可以使用这样的查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary IN (20000, 35000, 40000);
```

第二个版本看起来更整洁，更易于阅读，因为所有关于薪资的条件都集中在一起。这个查询的结果是：

```
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
9    PATRICK     40000
```

这些结果与使用单独 `OR` 语句的查询结果相同。

#### 更长的值列表

你之前的例子在 `IN` 子句中包含了三个值。如果你想包含更多呢？使用几个 `OR` 语句，你的查询可能像这样：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 10000
OR salary = 20000
OR salary = 30000
OR salary = 40000
OR salary = 50000
OR salary = 60000
OR salary = 70000
OR salary = 80000
OR salary = 90000
OR salary = 100000;
```

这个查询检查薪资值是否以 10,000 为增量。每个值都有单独的 `OR` 关键字。这个查询显示以下结果：

```
ID   LAST_NAME   SALARY
1    JONES       20000
3    KING        40000
9    PATRICK     40000
```

查询结果只显示了三条记录，因为只有三个 `salary` 值与提供的列表中的值匹配。尽管列表中有十个值，但许多值没有对应的行，所以只返回了三行。

你可以使用 `IN` 子句重写查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary IN (
10000, 20000, 30000, 40000, 50000,
60000, 70000, 80000, 90000, 100000);
```

同样，`IN` 子句看起来比单独的 `OR` 关键字整洁得多，也更容易阅读。这个查询的结果也是一样的：

```
ID   LAST_NAME   SALARY
1    JONES       20000
3    KING        40000
9    PATRICK     40000
```



### 使用 `IN` 处理文本值

`IN` 关键字也可以与文本值一起使用。假设你有一个需要筛选的姓名列表。使用 `OR` 关键字的查询可能如下所示：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name = 'SIMPSON'
OR last_name = 'COOPER'
OR last_name = 'ANDERSON';
```

这个查询可以改用 `IN` 关键字来写：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name IN ('SIMPSON', 'COOPER', 'ANDERSON');
```

每个文本值都需要用单引号括起来，就像你在其他 `WHERE` 子句中看到的一样。这些文本值也都放在括号内，用逗号分隔，就像处理薪资值的查询一样。

这个查询的结果是：

```sql
ID   LAST_NAME   SALARY
4    SIMPSON     52000
5    ANDERSON    31000
6    COOPER      NULL
```

你可以看到结果显示了你指定的 `last_name` 值的记录。无论你在 `IN` 关键字内指定多少个值，这种类型的查询都有效，并且比单独的 `OR` 语句更容易输入和阅读。

### 将 `IN` 与其他子句结合使用

你已经见过一些在单个 `WHERE` 子句中查找列中几个不同值的查询。你也可以将这种方法与其他条件相结合，例如另一个 `AND` 或 `OR` 子句。

假设你想找到所有 `salary` 为 20000、25000、30000、35000 或 40000，并且 `last_name` 为 SMITH 的员工。你的查询可能如下所示：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary IN (20000, 25000, 30000, 35000, 40000)
AND last_name = 'SMITH';
```

这个查询使用 `IN` 关键字来查找 `salary` 值，并使用相等比较来检查 `last_name` 值。这种将 `IN` 关键字与其他子句结合的技术是完全有效的 SQL。运行此查询时，结果应为：

```sql
ID   LAST_NAME   SALARY
2    SMITH       35000
```

结果显示了一条记录，因为表中只有一条记录匹配指定的 `salary` 值并具有给定的 `last_name` 值。

### `IN` 与 `AND`？

你已经了解到，在 `IN` 子句中指定值等同于写出多个 `OR` 子句。为什么 `IN` 子句不等同于多个 `AND` 子句呢？当你有多个 `AND` 子句时会发生什么？

这是之前使用 `OR` 的查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 20000
OR salary = 35000
OR salary = 40000;
```

现在，考虑将所有的 OR 关键字替换为 AND，因为一个常见的错误是认为“我对 20000、35000 和 40000 的薪资值感兴趣”。很容易犯这样的错误：从英语思维中抓住“和”，并将其转换为查询中的 AND 关键字。例如：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary = 20000
AND salary = 35000
AND salary = 40000;
```

然而，这个查询不会得到你想要的结果。它本质上是在说：“显示 `salary` 等于 20000 且等于 35000 且等于 40000 的记录”。

一个列不能同时等于多个值。如果一个 `salary` 值等于 20000，它就不可能同时等于 35000。这个查询将返回零行。这就是为什么 `IN` 关键字等同于 `OR` 关键字，而不是 `AND` 关键字。

### `IN` 与 `LIKE`？

在之前的章节中，你学习了如何使用 `LIKE` 关键字执行部分匹配。例如，要找到所有 `last_name` 以“S”开头的记录，我们的查询将是：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE 'S%';
```

如果你想找到所有 `last_name` 以“S”或“P”开头的记录呢？你能为此使用 `IN` 关键字吗？你能写成这样吗？

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name IN ('S%', 'P%');
```

不幸的是，`IN` 的这种用法不会给你所需的结果。`IN` 子句对括号内的所有值进行“等于”判断，而不是 `LIKE`。因此，`IN` 子句无法对列表中的值执行部分匹配。它将返回 `last_name` 精确匹配字符“S%”或“P%”的记录，而这样的记录一条也没有。

当你有多个 `LIKE` 条件时，可以使用 `OR` 关键字将它们组合起来。例如：

```sql
SELECT id, last_name, salary
FROM employee
WHERE last_name LIKE 'S%'
OR last_name LIKE 'P%';
```

这个查询将显示以下结果：

```sql
ID   LAST_NAME   SALARY
2    SMITH       35000
4    SIMPSON     52000
8    SMITH       62000
9    PATRICK     40000
```

### 过滤值的范围

你已经学习了如何使用“等于”和“大于”等运算符通过 `WHERE` 子句过滤数据。有时你可能需要查找落在特定值范围内的数据。有一种比使用大于和小于运算符组合更好的方法来过滤值范围。

可以使用你的员工表来完成值范围的过滤。你可能需要过滤薪资值的范围。员工表如下所示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
4    SIMPSON     52000
5    ANDERSON    31000
6    COOPER      NULL
7    NULL        NULL
8    SMITH       62000
9    PATRICK     40000
```

假设你的经理想知道谁的薪资在某个范围内。你被要求提供所有薪资在 20,000 到 40,000 之间（含）的员工的详细信息。

你会怎么做？你可以使用一个 `WHERE` 子句，其中包含：

*   一个大于或等于符号，用于薪资 20,000
*   一个小于或等于符号，用于薪资 40,000
*   `AND` 关键字以确保记录同时满足这两个条件

你的查询可能如下所示：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary >= 20000
AND salary <= 40000;
```

这个查询将找到 `salary` 大于或等于 20000 且 `salary` 小于或等于 40000 的记录。你的结果将如下所示：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
5    ANDERSON    31000
9    PATRICK     40000
```

这些结果显示了 `salary` 等于 20000、40000 以及介于两者之间的所有值的记录。有另一种方法可以将相同的逻辑写入查询：使用一个称为 `BETWEEN` 的运算符。

### `BETWEEN` 运算符

在 SQL 中，`BETWEEN` 是一个运算符，它允许你为特定列指定上限和下限，行必须满足该条件才会被包含。`BETWEEN` 等同于使用“大于或等于”和“小于或等于”。使用 `BETWEEN` 的语法如下：

```sql
WHERE column BETWEEN low_value AND high_value
```

这意味着：

*   它以 `WHERE` 列开头，这是你想要查找的列。
*   然后你指定 `BETWEEN` 运算符。
*   接着是 `low_value`，即范围中的下限值。
*   然后你有 `AND` 关键字。
*   最后你有 `high_value`，即范围中的上限值。

#### 使用 Between 处理两个薪资值

你可以将前面的例子重写为一个使用 `BETWEEN` 的查询。

第一个例子是这个查询：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary >= 20000
AND salary <= 40000;
```

你可以使用 `BETWEEN` 关键字将其重写如下：

```sql
SELECT id, last_name, salary
FROM employee
WHERE salary BETWEEN 20000 AND 40000;
```

这个查询输入量稍少，并且可以说更容易阅读。结果是相同的：

```sql
ID   LAST_NAME   SALARY
1    JONES       20000
2    SMITH       35000
3    KING        40000
5    ANDERSON    31000
9    PATRICK     40000
```
