# 引言

“写一本你希望当初自己作为 DBA 新手时，能有人写好并递给你的书。”
大约 13 年前，我的朋友兼导师 Kevin Kline 对我说了这句话。当时我正在写我的第一本书 `DBA Survivor`，我向 Kevin 请教如何着手这个项目。他的回答让我豁然开朗，并且我将同样的方法应用到了你正在阅读的这本书中。

在我刚开始成为一名初级 DBA 时，Tom Davidson 那篇著名的 `SQL Server 2005 Waits and Queues` 白皮书还要几年才会发布。我所知道的关于等待的知识，都是通过使用 `DBCC` 语句针对（当时还被称为）Sybase ASE 和 SQL Server 2000 实例探索而来的。
换句话说，我知之甚少。

随着 SQL Server 2005 的发布和 Davidson 白皮书的出版，等待统计信息成为了一种可行的调优方法。管理员和开发者现在可以利用等待和队列来准确理解为什么一个查询的运行时间比预期的要长。一夜之间，我们的团队从**被动响应**查询性能问题转变为**主动理解**整个数据库工作负载最需要哪些资源。

发送到数据库服务器的每个请求都受到相同的约束：内存、处理、磁盘、网络和锁定/阻塞。无论你是否想仅依赖执行计划进行查询调优；查询面临的物理和逻辑约束始终不变：它们只是在执行计划中以不同的方式呈现。**等待和队列调优方法将查询性能调优所需的复杂性和时间降低了一个数量级**。

一旦你理解了数据库引擎如何处理请求、等待如何发生以及如何跟踪它们，你就走上了成为查询性能调优专家的道路。

而这正是本书的目标。当你读完本书时，我希望你拥有成为查询性能调优专家所需的所有技能。这就是我希望当初有人能写好并递给我这个 DBA 新手的书。

为了实现这个目标，本书被分为两个不等的部分。第一部分“等待统计分析基础”，详细介绍了数据库引擎如何处理查询（正式称为`请求`，由`会话`发送，在建立到实例的`连接`之后），然后介绍了如何通过各种 SQL Server `动态管理视图 (DMVs)` 查询等待统计信息。第一部分以 `查询存储`功能的概述和指导如何创建和收集指标以建立你自己的基线作为结束。

第二部分“等待类型”，深入探讨了具体的等待类型、其原因、一些示例以及可能的解决方案。章节按照等待类别划分，这有点棘手，因为某些等待（如 `PAGEIOLATCH`）可能涉及多个约束（内存和磁盘）。因此，章节将特定等待分解为按 CPU、IO、备份、锁、闩锁、高可用性与灾难恢复、抢占、后台与杂项以及内存中 OLTP 划分的类别。

是的，尽管后台和杂项等待对查询性能是良性的，它们也被包括在内。对你来说，了解这些等待为何（以及何时）发生，以及何时可以安全地忽略它们（通常可以，但并非总是如此），是很重要的。

需要注意的一点是，本书中的示例使用了一个名为 `GalacticWorks` 的数据库。这是我修改的 `AdventureWorks` 版本，在我授课时用于各种演示。书中的示例同样适用于 `AdventureWorks`，所以如果没有 `GalacticWorks` 也不必惊慌；使用大多数版本的 `AdventureWorks` 都没问题。

当你读完这本书时，我希望你有信心去应对任何查询性能调优问题。你将掌握成为专家所需的细节、信息和知识。也许不久之后，你也会去教导他人，甚至有一天写出你自己的书。
