# 引言

欢迎阅读*实战 Oracle Cloud Infrastructure*。我撰写本书旨在为读者提供关于 Oracle Cloud 第二代平台最重要方面的快速、实践性入门。本手册旨在通过一系列实践练习来指导您，让您有机会边做边学。阅读完本书后，您将对各种云基础设施概念有广泛的理解。此外，您将能够构建利用 Oracle Cloud 上不同类型云服务的基于云的应用程序。

我非常热衷于将自动化应用于云基础设施供应、系统配置和软件开发生命周期。在本书中，您将学习运用大量自动化技术，并采用基础设施即代码的方法。

## 本书适合谁阅读？

本书面向云架构师、顾问、工程师、计算机科学与技术专业的学生，以及任何希望了解 Oracle 公有云提供的基础设施即服务的人士。为了充分受益于本书描述的练习，您应该具备 Linux 或 macOS shell 的工作知识，对 IP 网络概念有基本理解，并有一些编程经验。

## 如何使用本书？

本书以代码驱动。在阅读过程中，您会发现我假设您正在使用随书附带的代码。因此，您应该克隆包含所有代码片段、源代码和配置文件模板的 Git 仓库。您可以通过以下链接找到相关 Git 仓库：

[`https://github.com/mtjakobczyk/oci-book`](https://github.com/mtjakobczyk/oci-book)
[`www.apress.com/source-code`](https://www.apress.com/source-code)

代码片段设计为在 macOS（终端）、Linux（Shell）或 Windows Subsystem for Linux 中执行。如果您使用 Windows 系统，请使用`Windows Subsystem for Linux`或启动一个带有 Linux 客户操作系统的虚拟机。Git 仓库由多个目录组成，每章一个。在每个目录中，您会找到该章节特定的`README.md`文件。这些`README.md`文件使用方便的 Markdown 符号，并包含所有必要的代码片段，以便您通过直接复制到终端来更轻松、更快速地使用代码。

此外，如果 Oracle Cloud 中的某些内容发生变化，Git 仓库中的代码将相应更新，以便您能够使用更新版本的代码片段。只需确保您定期检查更新并拉取最新版本即可。本书中的大部分应用程序代码是用 Python 编写的。即使您不懂 Python，也应该没问题，因为最重要的代码段都得到了解释。

我知道如今很少有人会从头到尾阅读技术书籍。然而，我强烈建议您按顺序学习本书，从第 1 章开始，一章接一章地阅读，更重要的是完成实践练习。

如果您只想获得一些基本理解，请阅读第 1 章并完成第 2 章涵盖的练习。如果您对自动化感兴趣，请继续阅读第 3 章。为了充分受益于本书包含的实践练习，建议您在继续后续章节之前，按照第 3 章和第 4 章的描述设置您的环境。这是因为第 5 章到第 9 章的练习假定第 3 章和第 4 章涵盖的环境配置仍然有效。总而言之，我的建议是按章节顺序阅读并学习本书。

### 云账户

要执行本书中的练习，您需要一个 Oracle Cloud 账户。在第 1 章末尾，您将学习如何订阅一个新的试用版云账户，该账户允许您免费使用 Oracle Cloud 30 天。如果您愿意，可以使用付费账户进行操作，但请注意某些服务会产生费用。本书的练习主要使用我个人的付费云账户创建。

## 变更管理

如今，云服务以惊人的速度发展，有时会引入破坏性变更。这对于相对较新的服务尤其重要，例如本书涵盖的 Oracle Kubernetes Engine 和 Oracle Functions。如果您发现某些操作方式与描述不符，请确保从本书关联的 GitHub 仓库拉取最新版本，并阅读特定章节的`README.md`文件。
