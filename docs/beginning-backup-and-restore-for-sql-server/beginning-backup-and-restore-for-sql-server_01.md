## 备份与恢复方法论导言

作为数据库管理员，我们的主要职责是保护所负责数据库中的数据。这些数据代表着雇佣我们的企业赖以生存的核心，如果维护不当，可能会摧毁一家企业。在此背景下，正确应用一个健全的备份和恢复解决方案对于在灾难发生时提供另一层数据安全至关重要。我们需要能够确保在紧急情况下数据可以被恢复，而确保数据能够顺利恢复的唯一方法，就是确保对需要恢复的数据进行了正确的备份。

对于未以任何形式备份的数据，是无法恢复的。幸运的是，有相当多的方法可以备份数据，既可以在 `SQL Server` 内部，也可以在其外部。在本书中，我们将严格从 `SQL Server` 管理员的角度来处理备份和恢复数据的问题，而不会从 `Windows Server` 管理员的视角来看待这一主题。此外，我们也不会深入探讨 `SQL Server Management Studio` 之外的高级实践。这样做是因为我希望尽可能将注意力集中在用于开发和维护的实际工具上。为此，我需要让本书的大部分内容与 `SSMS` 直接相关。请注意，这并不意味着你永远只能使用 `SQL Server Management Studio` 来管理数据库备份；事实并非如此。相反，我希望在本书中介绍一些概念，希望能扩展你对备份和恢复是什么的理解，以及在 `SQL Server` 的背景下如何正确维护它们。然而，这确实意味着我们将严格保持在 `SQL Server` 的范围内。需要绝对清楚的是，我们不会涉及与 `SQL Server` 如何准备并将备份解决方案传送到预定义存储位置不同的存储技术，也不会探讨 `SQL Server` 如何检索该备份解决方案以恢复损坏、丢失或受损的数据。相反，我们将专注于可用于确保你获得的备份集符合预期的各种不同设置和技术。因此，我们也将研究 `SQL Server` 中可用的各种不同恢复技术。

**注意**  
当备份由 `SQL Server` 创建时，你可以从多种选项中选择，以详细指定备份中要包含哪些数据。当备份由 `Windows` 创建时，整个数据库文件会连同文件系统一起备份。这可能是有利的，但也是对空间的浪费，有时如果经过加密，则可能无法恢复。

具体来说，本书将涵盖以下项目（以及其他许多主题）：

*   三种主要备份类型的优缺点及正确用法
    *   **完整备份**：整个数据库的备份
    *   **差异备份**：自上次完整备份以来发生变化的信息的备份
    *   **事务日志备份**：自上次备份以来的事务日志
*   用于不同规模组织的不同备份方法论
    *   本地存储与异地存储
    *   数据保留期限
    *   存储介质及其固有差异（磁带 vs. `USB` vs. `DVD`）
*   备份的压缩类型
*   备份的加密类型

需要注意的是，在本书的课程中，我们将准备许多备份解决方案，而我们创建的解决方案将位于单个物理硬盘驱动器上，该驱动器被分区为多个逻辑驱动器。如果你读过我的其他书籍，你会知道我有一个特定的方式来设置我的 `SQL Server` 文件位置，如清单 1 所示。对我来说，这有助于以逻辑方式组织文件夹结构，而不是陷入嵌套文件夹的泥潭才能到达默认的文件和文件夹位置。

```
文件夹位置                 用途
E:\SQL Server\Backups      备份（`.bak`）文件
E:\SQL Server\Data         数据（`.mdf`）文件
E:\SQL Server\Logs         日志（`.ldf` 和 `.trn`）文件
E:\SQL Server\Temp         任何其他文件类型
```
