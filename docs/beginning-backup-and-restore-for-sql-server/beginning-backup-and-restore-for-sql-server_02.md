# SQL Server 备份与恢复指南

在本书中，我们将专注于各种备份类型、不同的恢复过程（使用事务日志进行时间点恢复与加载完整备份），以及每个过程的组成部分。我们不会讨论将备份文件保存到指定备份文件夹（参见清单 1）以外的任何位置。您的具体场景可以根据需要进行调整，但这些章节中的说明将为您提供一个可行的备份和恢复解决方案的分步指南。简而言之，本书希望能解决几乎所有与备份或恢复数据相关的问题，如果它不能完全解答您的疑问，那么它或许能为您自己的解决方案指明正确的方向。

我们将从一个全新的 `SQL Server 2016` 和 `SQL Server Management Studio` 安装开始。我不会包含 `SQL Server` 或 `SSMS` 的安装说明，因为假设您已经有一个数据库在运行，并且希望为其启动一个备份/恢复解决方案，或增强您当前的备份/恢复场景。

一如既往，我强烈建议建立一个您可用于测试的开发环境。`SQL Server Express` 对开发者是免费的，因此这是一个很好的选择。如果您没有本地数据库可供使用，`Microsoft Azure` 也提供了一些极好的存储交易，而且物有所值。最新版本的 `SQL Server Management Studio` 将只连接到最“现代”版本的 `SQL Server` 引擎，因此请记住，它可以连接到任何先前版本的 `SQL Server`，如下所示：
*   `SQL Server 2008`
*   `SQL Server 2008 R2`
*   `SQL Server 2012`
*   `SQL Server 2014`
*   `SQL Server 2016`
*   `SQL Server 2017`
*   `Microsoft Azure`

几乎任何开放数据库连接 (`ODBC`) 数据源都可以通过 `SSMS` 访问，但如果我是您，我会坚持使用 `Microsoft` 的产品。为了本书的练习和大部分内容，如果您决定使用 `SQL Server` 或 `Azure` 以外的其他产品，您将很难完全跟随示例进行操作。

## 备份基础

在我们开始学习 `SQL Server` 内可用的备份和恢复方法类型之前，我们需要了解备份和恢复的各个部分，无论是单独还是结合起来，以便真正知道我们想从一个备份/恢复解决方案中实现什么。您的数据库很可能在其整个生命周期内平稳运行，您永远不需要恢复备份。我相信这对大多数数据库管理员来说是乌托邦，而且我想不起有谁曾经真正遇到过这种情况。大多数时候，墨菲定律会发生作用，它确保您要么有一个随时可用的备份可以恢复，要么您就从为什么要有一个可靠的备份解决方案这个问题上学到一个非常惨痛的教训。不幸的是，我们大多数人都属于这一类；那些数据库管理员可能因为时间限制或懒惰而放弃了备份解决方案的便利，并在后来遭受了后果。如果这是您，请理解您并不孤单。在您作为数据库管理员的职业生涯早期，犯一些新手错误是很容易的，但期望您从这些错误中学习，这样它们就不会再发生。这就是我认为本书可以发挥作用的地方。

在处理数据恢复时，很容易说备份应该总是可用的，但知道如何实际构建您需要的备份解决方案，并在需要时让该解决方案可用，则完全是另一回事。因此，`Microsoft` 在 `SQL Server` 的背景下提供了不同的备份数据的方式。让我们看看备份和恢复的各种方法，以及它们如何相互关联。

### 恢复模型

在 `SQL Server` 中，有三种不同的恢复模型可用：
*   `Full`
*   `Bulk-logged`
*   `Simple`

这些模型的目的是为您，即数据库管理员，提供关于您希望如何交付备份数据的不同选项。您不能为一个数据库选择多个选项，并且您受限于每种恢复模型可用的备份类型。因此，应非常谨慎地选择恢复模型。

可以将恢复模型视为容器，备份类型及随后的备份都位于其中。如果恢复模型不适合您的需求，那么您将永远无法获得所需的备份和恢复解决方案。

选择一个恢复模型实际上非常简单；只需决定在您的情况下可接受多少数据丢失，场景基本上就自己构建好了。如果您希望数据丢失非常少，那么简单恢复模型选项就被排除了。这就留下了完整和大容量日志恢复模型。这两种模型之间的区别是什么？基本上，完整恢复模型允许进行时间点恢复，而大容量日志恢复模型只允许备份从上次恢复点开始的数据。在这两个选项之间做出决定，那就是您期望的恢复模型。

这三种不同的恢复模型在第 1 章中有详细讨论。


##### 备份类型

与恢复模型分离的是三种不同的备份类型。这些类型如下：

*   `完整`：对整个数据库的备份
*   `差异`：自上次完整备份以来已更改信息的备份
*   `事务日志`：自上次备份以来的事务日志

这些备份类型在本章前面也介绍过，并且在本书的整个过程中都会详细讨论。因此，我现在还不会深入探讨备份类型的细节，而是让各个相关领域的章节来详细介绍关于这些类型的信息。

## 注意

恢复模型和备份类型的主要思想是，某些恢复模型只能与特定的备份类型配合工作，因此选择恢复模型是设计有效备份解决方案的第一步。

正如你所见，牢记恢复模型和备份类型之间的关系非常重要。本书的重点是巩固这方面的知识，并在此基础上建立更深入的理解，了解我们可以采取哪些措施来创建有效的备份策略，以帮助未来减轻灾难的影响。
