# 10. 差异备份和还原解决方案

既然我们在第 9 章中详细介绍了功能正常的完整备份和完整还原 `SQL Server 代理` 作业，我们需要拼凑出差异备份和差异还原部分。我们将借鉴第 2 章和第 6 章的内容来完成本章，但我们将着眼于为我们现有的“备份计划”维护计划添加略有不同的功能和定制。

我们想要做的是修改备份计划以包含备份的差异部分，然后创建一个新的差异还原 `SQL Server 代理` 作业，以便手动运行或与完整还原 `SQL Server 代理` 作业结合运行。在本章结束时，我们将为现有计划创建一个附加部分，测试我们目前为该计划创建的内容，并验证预期操作的预期结果。

最近，我接到任务，要在我的工作区域创建一个相同的开发环境。您可能明白我的意思了；由于事情的偶然性，我能够将我们现在所做的应用到真实场景中，并且它奏效了。我可以将完整数据库备份还原到一个全新的 `SQL Server` 实例，没有任何麻烦，不仅如此，我还能将差异和事务日志备份也还原到新服务器上，因此在迁移到开发服务器时基本上实现了零数据丢失。知道我能够快速、正确地备份和还原我负责保护的数据，让我感到非常安心。

我经常发现自己会构思一些“噩梦场景”，这会迫使我快速而正确地思考和行动。显然，这些场景存在局限性；它们必须完全植根于我可以影响或控制结果的情况。例如，服务器宕机的场景完全超出了我的责任范围，因为我需要依赖其他人来解决这种情况。但是，例如，如果不小心运行了一个查询并删除了一个表，或者我的数据库遭受了 `SQL` 注入攻击（它不会，但请记住这是一个噩梦场景），或者其他与 `SQL Server` 相关的问题出现，我需要知道我有技术能力来回答我需要回答的问题。在我看来，我们可以采取不同的步骤来减轻这些噩梦场景，我们希望最终能在本章中解决这些问题。

#### 在 SSMS 中创建差异还原计划

我们将使用 SQL Server Agent 在 SSMS 中创建差异还原计划，就像我们在第 9 章中处理完整还原计划一样。我们将遵循相同的基本指令集，只是在涉及还原类型的地方有所不同。清单 10-3 将详细说明我们需要采取的步骤，以确保我们的差异还原计划正确执行。

我们计划实施此操作的基本结构很简单，如清单 9-12 所示。

清单 10-3. 还原数据的结构
*   配置 `RESTORE DATABASE` 命令
*   创建 T-SQL 类型的 SQL Server Agent 作业
*   创建自上次完整备份以来最新的差异备份副本，并将其重命名为 `backrecTestDB_DIFFERENTIAL.bak`
*   正常还原数据

现在我们有了如何推进的计划，让我们开始创建此计划。

##### T-SQL 还原命令

在此示例中，我们将使用清单 10-4 中所示的 T-SQL 命令。

```sql
RESTORE DATABASE backrecTestDB
FROM DISK = N'E:\SQL Server\Backup\backrecTestDB_FULL.bak'
WITH NORECOVERY, REPLACE;
RESTORE DATABASE backrecTestDB
FROM DISK = N'E:\SQL Server\Backup\backrecTestDB_DIFFERENTIAL.bak'
WITH RECOVERY;
```
清单 10-4 `RESTORE DATABASE` 命令

回想一下，`NORECOVERY` 属性允许我们指定正在还原最后一个完整备份，然后是差异备份。如果不首先还原最近的完整备份，就无法还原差异备份，因此我们需要先运行完整还原，然后再运行差异还原。为此，我们包含了清单 10-4 中所示的完整还原语法以及差异还原语法，以正确还原数据库。请注意，这两条命令由分号分隔；这表示我们将在两个独立的 SQL Server Agent 作业步骤中使用这些命令，如下一节所示。
