# 关系数据库词典（扩展版）

#### SQL 表与关系型关系

SQL 表在几个关键方面不同于其关系型对应物：
(a) SQL 表可以包含重复行；
(b) SQL 表可以包含空值；
(c) SQL 表具有从左到右的列顺序；
(d) SQL 表可以有两个或更多同名列；
(e) SQL 表可以有实际上完全没有名称的列。

更一般地说，一个关系的图示（例如在纸上的）。**注意**：将关系与这种表格图示混淆，可能是导致人们普遍误以为关系是“扁平”或二维的根源。虽然这些图示显然是二维的，但关系通常不是；相反，一个度为 `n` 的关系是 `n` 维的，这意味着其元组对应于某个 `n` 维空间中的点（每个维度对应于所讨论关系的一个属性）。这种考虑的一个具体结果是（再次与普遍观点相反）关系完全能够表示所谓的多维数据，从而支持所谓的联机分析处理（OLAP）。*另见* `cell`；`column`；`row`。

#### TABLE_DEE 和 TABLE_DUM

两个关系常量，最好是内置的。`TABLE_DEE` 是唯一的没有属性且恰好有一个元组（必然是空元组）的关系；`TABLE_DUM` 是唯一的没有属性且根本没有元组的关系。它们可以分别解释为 `TRUE`（或*是*）和 `FALSE`（或*否*）。（更准确地说，`TABLE_DEE` 的关系谓词是任何求值为 `TRUE` 的零元谓词，`TABLE_DUM` 的关系谓词是任何求值为 `FALSE` 的零元谓词。）**注意**：这些名字可能选得不太好，因为 `TABLE_DEE` 和 `TABLE_DUM` 正是最难用表格形象化的两个关系。

#### 表与视图 / 表或视图

在 SQL 上下文中经常出现的短语，强烈暗示视图以某种方式与表不同。但视图的关键在于（用 SQL 术语来说）它们*就是*表——就像在数学中，一个集合（例如两个集合的并集或交集）的关键在于它本身就是一个集合。视图对于用户来说应该“看起来和感觉起来”都像基本表一样（可互换性原则转化为 SQL 术语）。

#### 重言式

一个谓词，无论用什么参数替换其参数，其所有可能的调用都保证求值为 `TRUE`。*对比* `contradiction`。

**示例**：令 `p1` 为谓词（实际上是命题）`2+2 = 4`；令 `p2` 为谓词 `x = x`，其中 `x` 表示任意整数；令 `p3` 为谓词 `(p) OR (NOT(p))`，其中 `p` 表示任意命题。那么 `p1`、`p2` 和 `p3` 都是重言式。

#### 时态数据库

一种数据库，其中至少一个关系变量是时态的（时态关系变量是指其标题包含至少一个某种时间戳类型的属性的关系变量，这意味着相应的关系变量谓词具有至少一个某种时间戳类型的参数）。

可以定义多种特殊运算符来帮助管理此类数据库——参见 C. J. Date、Hugh Darwen 和 Nikos A. Lorentzos 所著的 *Temporal Data and the Relational Model*（Morgan Kaufmann, 2003）一书——但归根结底，所有这些运算符都是最初定义的关系代数中可定义运算符的简写。
