# 注意

*术语* `database` *在非关系型语境中也用于表示多种其他含义：例如，物理存储的数据集合。它也常被非常频繁地用来指代数据库管理系统（DBMS），但这种特定用法被强烈反对。（如果我们将 DBMS 称为数据库，那么我们该称数据库为什么呢？）*

#### 数据库约束

1.  *（“一个”数据库约束）* 从形式上讲，指任何非类型约束的约束（这意味着除类型约束外的所有约束，包括属性约束和关系变量约束等，都只是其特例）；通俗地讲，指引用两个或多个不同关系变量的约束（也称为多关系变量约束，参见 q.v.）。*注意*：这两个定义并非在任何意义上等价——它们指的是两个不同的概念。
2.  *（“该”数据库约束）* 除类型约束外，适用于给定数据库的所有约束的逻辑与（即该数据库的“该”数据库约束——有时称为*总*数据库约束）。注意，根据此定义，每个数据库都至少受一个约束：即退化的（“默认”）约束 `TRUE`。*另见* 关系变量约束。

### 示例

首先，供应商-零件数据库定义中指定的键约束和外键约束都是数据库约束。

其次，以下是一些也可能适用于该数据库的其他数据库约束：

```sql
CONSTRAINT C1 IS_EMPTY
    ( S WHERE STATUS < 1 OR STATUS > 100 ) ;
    /* status values must be in the */
    /* range 1 to 100 inclusive */

CONSTRAINT C2 IS_EMPTY
    ( P WHERE CITY = 'London' AND COLOR ≠ 'Red' ) ;
    /* parts in London must be red */

CONSTRAINT C3 IS_EMPTY
    ( ( S JOIN SP )
    WHERE STATUS < 20 AND P# = P#('P6') ) ;
    /* no supplier with status less */
    /* than 20 can supply part P6 */
```

最后，假设为举例说明，指定的键约束和外键约束，连同上述约束 `C1`–`C3`，是适用于供应商-零件数据库的唯一数据库约束。

那么，所有这些约束与 `TRUE` 的逻辑与就是该数据库的“该”（总）数据库约束。

#### 数据库值

某个数据库的当前或某个可能的“状态”；即关系的集合，这些关系是相应关系变量的可能值。因此，从概念上讲，数据库值可以被认为是一组命题，这些命题对应于相应关系中的元组。*对比* 数据库变量。

### 示例

#### 数据库变量

宽泛地说，是关系变量的容器；更准确地说，是一个在任何给定时间其值都是数据库值的变量。严格来说，数据库值和数据库变量之间存在逻辑差异，这与关系值和关系变量之间的差异完全类似。换句话说，我们通常称为数据库的东西实际上是一个变量（通常是一个相当大的变量），更新该数据库的效果是将该变量的一个值替换为另一个值——这里涉及的值是数据库值，而涉及的变量是数据库变量。更精确地说，数据库是一个元组变量，对于该数据库中的每个关系变量，都有一个（关系值的）属性。

因此请注意，数据库变量实际上并不是一组关系变量，尽管我们在非正式场合经常这样认为。

尽管如此，考虑到所有这些，我们在本词典的其他地方遵循传统用法，使用术语*数据库*来指代数据库值和数据库变量，依靠上下文来明确所指。

*参见* 数据库。

#### 德摩根定律

1.  *（逻辑）* 谓词 *p* 和 *q* 的析取的否定等价于 *p* 和 *q* 的否定的合取；谓词 *p* 和 *q* 的合取的否定等价于 *p* 和 *q* 的否定的析取。
2.  *（集合论）* 集合 *s1* 和 *s2* 的并集的补集等于 *s1* 和 *s2* 的补集的交集；集合 *s1* 和 *s2* 的交集的补集等于 *s1* 和 *s2* 的补集的并集。

#### 反规范化

用一组关系变量 `R1, R2, . . ., Rn` 的连接 `R` 来替换它们，使得对于所有 `i`，`R` 在 `Ri` 属性上的投影保证等于 `Ri`（`i` = 1, 2, . . ., `n`）。反规范化通常出于性能原因而进行；然而，它具有增加冗余（见该词条）的效果，从而增加了（a）必须完成的完整性检查量，或（b）某些更新异常（见该词条）发生的可能性，或（c）两者。它也会增加某些查询的复杂性。
*注*：从逻辑角度看，反规范化，至少降到 5NF 以下的级别，总是不可取的。然而，鉴于当今商业产品中发现的技术水平，有时它无法避免。

**示例**：可应用于供应商-零件数据库的一个反规范化是，用关系变量 `S` 和 `SP` 的连接（假设为 `SSP`）来替换它们。然后可以通过在 `S` 的属性和 `SP` 的属性上分别投影关系变量 `SSP` 来导出关系变量 `S` 和 `SP`。

#### 导出关系

笼统地说，是用其他关系定义的关系。更精确地说，设 `s` 为一个关系集合。那么关系 `r` 是从 `s` 中的关系导出的，当且仅当它本身不出现在 `s` 中，但可以通过某个关系表达式从 `s` 中的关系得到。该表达式必须满足（a）每个关系操作数都是 `s` 中的一个关系或从 `s` 中的关系导出的关系，并且（b）每个关系运算都是本词典中定义的运算之一，`EXTEND`、`SUMMARIZE` 以及只读的（“假设”）形式的 `UPDATE` 除外。*对比* 基关系。
*注*：此定义中对合法运算符的限制目的是排除引入某种“全新的”关系（可以这么说）的可能性，而该关系无法从 `s` 中的关系导出。

**示例**：考虑表达式 `S JOIN SP`。如果关系变量 `S` 和 `SP` 的当前值分别为 `s` 和 `sp`，则此表达式定义的导出关系就是 `s` 和 `sp` 的连接。

#### 差

两个关系 `r1` 和 `r2`（按此顺序）的差，`r1 MINUS r2`，其中 `r1` 和 `r2` 是相同类型 `T` 的，结果是一个类型为 `T` 的关系，其体是包含所有满足条件“`t` 出现在 `r1` 中且不出现在 `r2` 中”的元组 `t` 的集合。



*注*：关系差运算符在某些方面与同名的数学或集合论运算符不同，详见相关条目。实际上，它是半差的一种特例，详见相关条目。

**示例**：表达式 `(S{CITY}) MINUS (P{CITY})` 表示分别对关系变量 `S` 和 `P` 的当前值在 `{CITY}` 上进行投影后得到的关系之间的差集。该差集是一个类型为 `RELATION {CITY CHAR}` 的关系。此外，如果关系变量 `S` 和 `P` 的当前值分别是 `s` 和 `p`，那么该关系的主体由所有形式为 `< c>` 且出现在 `s{CITY}` 中但不出现在 `p{CITY}` 中的元组组成——这意味着 `c` 是一个当前供应商所在城市，但它不是当前零件所在城市。注意，表达式 `(S{CITY}) MINUS (P{CITY})` 在语义上等价于表达式 `(S{CITY}) NOT MATCHING (P{CITY})`——或者更简单的表达式 `(S{CITY}) NOT MATCHING P`。

**分配律** 1. 令运算符 `OpM` 和 `OpD` 分别为一元和二元运算符，并为明确起见，假设 `OpD` 以中缀风格表示。那么，`OpM` 对 `OpD` 满足分配律，当且仅当对于所有 `x` 和 `y`，`OpM(x OpD y) = (OpM(x)) OpD (OpM(y))`。 2. 令运算符 `OpC` 和 `OpD` 均为二元运算符，并为明确起见，假设它们都以中缀风格表示。那么，`OpC` 对 `OpD` 满足分配律，当且仅当对于所有 `x, y` 和 `z`，`x OpC (y OpD z) = (x OpC y) OpD (x OpC z)`。

**示例**：1. （*一元对二元*）在普通算术中，非负平方根（"√"）对乘法（"*"）满足分配律，因为对于所有 `x` 和 `y`，`√ (x * y) = √(x) * √(y)`。类似地，在关系代数中，限制对 `UNION`、`INTERSECT` 和 `MINUS` 满足分配律。 2. （*二元对二元*）乘法对加法（"+"）满足分配律，因为对于所有 `x, y` 和 `z`，`x * (y + z) = (x * y) + (x * z)`。类似地，在关系代数中，`UNION` 和 `INTERSECT` 彼此之间都满足分配律。同样，在逻辑中，`OR` 和 `AND` 彼此之间也满足分配律。

**DIVIDEBY** *参见* 除法；大除；小除。

**域演算** 关系演算的一种形式，在语义上等价于元组演算，详见相关条目。其中，范围变量在域（类型）上变化，而不是在关系上变化，因此表示来自那些域的值。

**示例**：以下是查询“获取供应至少一种零件的供应商名称”的域演算表述（*参见* 元组演算以了解其元组演算类似形式）：

```
NX RANGES OVER { NAME } ;
SX RANGES OVER { S# } ;
PX RANGES OVER { P# } ;
NX WHERE
    EXISTS SX ( EXISTS PX ( S { S# SX, SNAME NX } AND
                           SP { S# SX, P# PX } ) )
```

用生硬的英语表达：“获取名称 `NX`，使得存在供应商编号 `SX` 且存在零件编号 `PX`，使得一个包含供应商编号 `SX` 和供应商名称 `NX` 的元组出现在关系变量 `S` 中，并且一个包含相同供应商编号 `SX` 和零件编号 `PX` 的元组出现在关系变量 `SP` 中。” 正如你所见，这个特定例子比它的元组演算对应形式（*参见* 元组演算）更笨拙一些，但也有些情况恰好相反。

**域检查覆盖** 一种临时的、逻辑上有缺陷的（因此不推荐的）机制，用于在不同类型值之间执行比较。（它存在缺陷是因为它混淆了类型和表示之间的逻辑区别。）

#### 点限定
在元组演算及基于它的语言中，一个点限定名称是形如 `rx.A` 的表达式，其中 `rx` 是一个范围变量的名称，而 `A` 是该范围变量 `rx` 所遍历的关系 `r` 的一个属性名称。这样的表达式用作属性引用，参见；它表示 `rx` 当前所指向的 `r` 的特定元组中属性 `A` 的值（或可能是指属性 `A` 本身）。点限定在元组演算中用于消除歧义——在 SQL 中也是如此——但在域演算或关系代数中则不然（后者使用属性（重）命名和名称作用域来达到等效效果）。

**示例：** 以下查询“找出至少提供一种零件的供应商”的元组演算表述，使用了两个点限定名称 `SPX.S#` 和 `SX.S#`：

```
SX RANGES OVER { S } ;
SPX RANGES OVER { SP } ;
SX WHERE EXISTS SPX ( SPX.S# = SX.S# )
```

这里是同一查询的关系代数表述，以作比较：`S MATCHING SP`。这里的“匹配”是基于属性 `S#` 进行的（因为该属性是关系变量 `S` 和 `SP` 唯一共同的属性）。

#### 对偶
1.  （*逻辑学*）`与`、`或`、`真` 和 `假` 的对偶分别是 `或`、`与`、`假` 和 `真`（`非` 是其自身的对偶）。更一般地说，令 `x` 是一个仅涉及 `非`、`与` 和 `或` 这些连接词的逻辑表达式，并令 `y` 是通过将 `x` 中出现的每一个 `与`、`或`、`真` 和 `假` 替换为其对偶而得到的；那么 `x` 和 `y` 互为对偶。*注意*：由于每个逻辑表达式都等价于一个仅涉及 `非`、`与` 和 `或` 的表达式，因此每个逻辑表达式都有一个对偶。
2.  （*集合论*）交集、并集、全集和空集的对偶分别是并集、交集、空集和全集（补集是其自身的对偶）。更一般地说，令 `x` 是一个仅涉及补集、交集和并集的集合论表达式，并令 `y` 是通过将 `x` 中出现的每一个交集、并集、全集和空集替换为其对偶而得到的；那么 `x` 和 `y` 互为对偶。*注意*：由于每个集合论表达式都等价于一个仅涉及补集、交集和并集的表达式，因此每个集合论表达式都有一个对偶。*另见* `对偶原理`。

#### 双模原理
指任何可以交互调用的关系操作，也可以从应用程序中调用的原则。

#### 对偶原理
1.  （*逻辑学*）令 `x` 和 `y` 是逻辑表达式，使得 `x` 是 `y` 的对偶；那么 `x` 是重言式当且仅当 `y` 是重言式。
2.  （*集合论*）令 `x` 和 `y` 是集合论中的陈述，使得 `x` 是 `y` 的对偶；那么 `x` 是一个定理当且仅当 `y` 是一个定理。

#### 重复
令 `a1` 和 `a2` 是在某个上下文中值 `v1` 和 `v2` 的出现，参见。那么 `a1` 和 `a2` 互为重复，当且仅当 `v1` 和 `v2` 相等（换句话说，当且仅当 `v1` 和 `v2` 是完全相同的值）。顺便说一句，从这个定义可以清楚地看出，那句众所周知的格言——关系中从不包含重复元组——其真正含义是没有关系包含*相同*元组的重复*出现*——尽管为了熟悉度，我们在本词典中倾向于使用不太精确的表述。观察到，由于（a）关系从不包含重复元组，并且（b）每个关系操作都产生一个关系，因此，每当执行关系操作时，如果需要，会消除重复元组（更准确地说，是相同元组的冗余出现）。

**示例**（*重复消除*）：给定图 1 所示的示例值，关系变量 `S` 的当前值在 `{CITY}` 上的投影基数为三，而非五；类似地，关系变量 `S` 和 `P` 的当前值在 `{CITY}` 上的投影的并集基数为四，而非十一。（在最熟悉的关系运算符中，投影和并集是唯一需要关注重复消除的两个。对于其他运算符——如选择、连接等——它实际上是一个空操作。）

#### 嵌入式依赖
由某个关系变量的某个投影所满足的依赖。

**示例：** 考虑关系变量 `CTXD`，其属性为 `C`（课程）、`T`（教师）、`X`（教材）和 `D`（天数），谓词为“教师 `T` 在课程 `C` 上花 `D` 天时间讲解教材 `X`”。假设该关系变量的唯一键是 `{C,T,X}`。同时假设对于给定课程，教师集合和教材集合是相当独立的。那么 `CTXD` 属于 6NF——除了平凡分解外，它根本无法进行无损分解。然而，它在 `{C,T,X}` 上的投影满足嵌入式多值依赖 `{C} ->> {T}` 和 `{C} ->> {X}`。
