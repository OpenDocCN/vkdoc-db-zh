# 关系数据库词典，扩展版

#### 命题变量
其值是一个命题，因此实际上只能是真或假的变量。（有些作者用该术语指代自由变量，但此用法不予推荐。）

#### 原型元组
宽松地指关系演算表达式中位于 `WHERE` 子句之前的部分。该术语是“prototype tuple”的简写；它虽有用但并非标准术语。

#### 伪变量引用
使用操作性表达式而非简单变量引用来表示某个更新操作的目标。

*注*：为便于定义，将伪变量引用视为常规变量引用是方便的（本词典即如此处理）；换句话说，宽松地说，伪变量就是变量。

*示例*：设 `CS` 是一个 `CHAR` 类型的变量，当前值为字符串 ‘Middle’，并考虑以下赋值：
```
SUBSTR ( CS, 2, 1 ) = 'u' ;
```
此处的 `SUBSTR` 是子串运算符，该赋值的效果是“击中” `CS` 内的第二个字符位置，将 ‘i’ 替换为 ‘u’（因此，更新后 `CS` 的当前值是字符串 ‘Muddle’）。赋值左侧的表达式是一个伪变量引用。

再举一例，设 `LSV` 是一个视图，定义为关系变量 `S` 中仅包含伦敦供应商的限制版本，并考虑以下 `DELETE` 语句：
```
DELETE LSV WHERE STATUS > 15 ;
```
从逻辑上讲，此 `DELETE` 语句等价于以下语句：
```
DELETE ( S WHERE CITY = 'London' )
WHERE STATUS > 15 ;
```
在此展开形式中，`DELETE` 的目标被指定为一个操作性表达式（即伪变量引用）。如本例所示，因此，更新视图在逻辑上等价于更新某个伪变量（宽松地说，因此视图就是伪变量）。

#### QBE
一种基于域演算的关系语言。其名称是 `Query-By-Example` 的缩写。

#### 量化
将量词（参见 q.v.）应用于一个自由变量，从而将该自由变量转换为约束变量（参见 binding），并将包含该自由变量的谓词转换为一个逻辑上不同于原谓词的新谓词。
*注*：如果原谓词中的每个自由变量都以这种方式被量化，那么结果就是一个命题。

#### 量词
参见存在量词；全称量词。
*注*：其他量词也是可能的——例如，恰好存在一个（参见 `UNIQUE`）；除一个外所有；存在奇数个；等等——但 `EXISTS` 和 `FORALL` 无疑是实践中遇到频率最高的。

#### QUEL
一种基于元组演算的关系语言，曾一度是 SQL 的有力竞争者。

#### 查询
一种检索请求（即，一个关系表达式，或一个请求评估此类表达式的语句）。有时被宽松地用于指代更新请求；也用于指代某些检索或更新请求的非正式自然语言对应物。

#### 查询分解
一种通过递归地将关系表达式划分为子表达式来评估它们的分治技术。

#### 查询重写
参见表达式转换。

#### 定额查询
一种对结果的基数施加期望限制或定额的查询。

*示例*：以下是定额查询“获取三个最重的零件”的一个可能表述（此处的定额为三）：
```
WITH ( P RENAME ( WEIGHT AS WT ) ) AS t1 ,
( EXTEND P ADD ( COUNT ( t1 WHERE WT > WEIGHT ) AS #_HEAVIER ) AS t2 ,
```

#### 范围（range）

1.  *参见* `函数（function）`。
2.  *参见* `范围变量（range variable）`。

#### 范围变量（range variable）

关系演算中逻辑变量的类似物；换言之，一个“取值范围”为某个特定值集的变量——在元组演算中是某个关系中的元组集合，在域演算中是某种类型的值集合——并且在关系演算表达式中既可以是绑定的也可以是自由的。

**示例**：*参见* `域演算（domain calculus）` 和 `元组演算（tuple calculus）` 下的示例。

#### 排名（RANK）

*参见* `排名运算（ranking）`。

#### 排名运算（ranking）

设 *r* 是一个没有名为 *X* 属性的关系。那么排名运算 `RANK r BY ( item, . ., item AS X )`，其中每个 `item` 由 `ASC`（升序）或 `DESC`（降序）后跟一个标识 *r* 属性的属性引用组成（并且这些 `item` 的整体序列按常规方式指定了主排序到次排序），生成一个与 *r* 相同的关系，除了 (a) 它多了一个属性 *X* 以及 (b) 该结果中任何给定元组的 *X* 值显示了该元组相对于指定排序的排名位置。

**示例**：*参见* `配额查询（quota query）`。

#### 只读操作符（read-only operator）

一个函数；即一个调用时不更新任何内容（除了可能更新该操作符实现内部的局部变量）但返回一个值的操作符，该值的类型在操作符定义时声明。因此，只读操作符调用表示一个值；即，它是一个表达式——事实上，根据定义，每个表达式都是一个只读操作符调用（*参见* `表达式（expression）`）——并且它可以出现在任何允许相应类型字面量的地方。特别地，它可以嵌套在其他表达式中。*注意*：正如本词典其他地方提到的，某些 SQL 只读操作符是“可能非确定性的”，这意味着从技术上讲，它们根本不是函数。实际上，它们确实是函数；然而，它们是故意未充分指定的，因此从用户的角度来看，它们的行为不像函数。

**示例**：*参见* `参数（argument）` 下的第一个示例。

#### 实关系（real relation）

给定实时点上一个真实关系变量的值。

#### 实关系变量（real relvar）

一个基本关系变量或一个快照（*对比* `虚拟关系变量（virtual relvar）`）。

#### 记录（record）

有时用来表示“行”的术语，具有该词的任何可能含义。然而，所有此类用法都被弃用；该术语最好保留给操作系统甚至物理级别的构造。

#### 递归查询（recursive query）

一个关系表达式——根据定义可以被视为某个关系值操作符 *Op* 的调用——其求值涉及对该同一操作符 *Op* 的进一步调用。

**示例**：这里是一个计算具有属性 A 和 B（均为 `P#` 类型）的二元关系的传递闭包（*参见* `传递闭包（transitive closure）`）的操作符的递归定义（代码效率不高，但显然可以通过多种方式改进）：
```
OPERATOR TRANCLO ( AB RELATION { A P#, B P# } )
RETURNS RELATION { A P#, B P# } ;
RETURN
( WITH ( AB UNION
( ( AB RENAME ( B AS C ) ) COMPOSE
( AB RENAME ( A AS C ) ) ) AS ttt :
IF ttt = AB THEN ttt ELSE TRANCLO ( ttt ) END IF ) ;
END OPERATOR ;
```
现在，`TRANCLO( r )` 的调用（其中 *r* 是一个适当类型的关系）可以被认为是一个递归查询，因为它的求值涉及对 `TRANCLO` 本身的进一步调用（通常是这样）。

#### 递归关系（recursive relationship）

一种关系（取该词的第三种定义，*参见* `关系（relationship）`），其中参与的两个集合是同一个集合。这个术语并不是特别贴切，因为其中并不涉及递归。

#### 归谬法（reductio ad absurdum）

“归谬法”；一种证明方法（*参见* `证明（proof）`），通过证明某物的否定会导致矛盾来确立该物为真。

#### 冗余（redundancy）

一个数据库当且仅当它包含（直接或间接地）同一命题的两个或多个不同表示时，才显示出冗余。*另见* `受控冗余（controlled redundancy）`。

#### 被引用键（referenced key）

*参见* `外键（foreign key）`。

#### 被引用关系变量（referenced relvar）

*参见* `外键（foreign key）`。

#### 被引用元组（referenced tuple）

*参见* `外键（foreign key）`。

#### 引用（referencing）

该术语的关系型含义如 `外键（foreign key）` 下所述；不应与同名操作符混淆——该操作符存在于支持指针但不支持关系模型的系统中——它给定一个变量 *V*，返回一个指向 *V* 的指针。*注意*：支持指针的系统通常也支持一个称为 `解引用（dereferencing）` 的操作符，它给定一个指针 *p*，返回 *p* 指向的变量 *V*。

#### 引用关系变量（referencing relvar）

*参见* `外键（foreign key）`。

#### 引用元组（referencing tuple）

*参见* `外键（foreign key）`。

#### 引用操作（referential action）

外键规则的操作规范部分（例如，DELETE 规则中的“cascade”）；也用于指代指定的操作本身。

#### 引用约束（referential constraint）

*参见* `外键（foreign key）`。

#### 引用环（referential cycle）

从某个关系变量到其自身的一条引用路径（*参见* `引用路径（referential path）`）。涉及此类循环的数据库设计通常是不推荐的。

#### 引用完整性（referential integrity）

粗略地说，规则是：如果对应的被引用元组不存在，则不允许任何引用元组存在。更精确地说，设 *FK* 是某个引用关系变量 *R2* 中的某个外键；设 *K* 是对应被引用关系变量 *R1* 中的对应键；并设 *K'* 是如 `外键（foreign key）` 下所述从 *K* 派生而来的值。那么，完整性规则要求不存在这样一个时刻：存在一个 *R2* 中的 *FK* 值，它不是 *R1* 中某个（必然是唯一的）元组的 *K'* 值。*另见* `外键（foreign key）`; `外键规则（foreign key rule）`。

#### 引用路径（referential path）

设关系变量 *Rz, Ry, Rx, ..., Rb, Ra* 满足：存在一个从 *Rz* 到 *Ry* 的引用约束，一个从 *Ry* 到 *Rx* 的引用约束，...，以及一个从 *Rb* 到 *Ra* 的引用约束。那么，从 *Rz* 到 *Ra* 的这一系列约束构成了一条从 *Rz* 到 *Ra* 的引用路径（并且链中约束的数量是该路径的长度）。

#### 自反性（reflexivity）

1.  （*二元逻辑操作符的*）二元逻辑操作符 *Op*（为明确起见，我们假设它以中缀风格表示）是自反的，当且仅当，对于所有 *x*，`x Op x` 为真。
2.  （*二元关系的*）二元关系 *r* 是自反的，当且仅当，对于所有 *x*，元组 `<x, x>` 出现在 *r* 中。
3.  （*函数依赖的*）*参见* `阿姆斯特朗推理规则（Armstrong's inference rules）`。
    *注意*：前两个定义略有简化，因为它们故意没有指定 *x* 的可能值范围。

    **示例**（*仅第一个定义*）：逻辑操作符 `EQUIV`；“小于或等于”操作符 `"≤"`。

#### 刷新（refresh）

*参见* `快照（snapshot）`。

#### 关系（relation）

一个关系值（*参见* `关系值（relation value）`）。*注意*：众所周知，该术语也常用于指代关系变量，但此用法因引起诸多混淆而被弃用。

#### 关系相等性

关系的相等性。关系`r1`和`r2`相等（即，关系比较`r1 = r2`的求值结果为`TRUE`），当且仅当`r1`和`r2`是完全相同的关系。

#### 关系谓词

设`r`是一个关系。那么，`r`的关系谓词（relation predicate）就是在某个特定上下文中代表用户所理解的`r`的含义的那个谓词。如果`r`的度是`n`，那么该谓词将是`n`元的（它为`r`的每个属性都有一个参数）。此外，在封闭世界假设（The Closed World Assumption）下，`r`的主体将包含且仅包含那些对应于该谓词求值结果为`TRUE`的元组。

示例：
1.  设`r`是关系变量`S`在`{S#, CITY}`上的投影（projection）。那么`r`的谓词是“存在一个名称`sn`和一个状态`st`，使得供应商`S#`已签约，其名为`sn`，状态为`st`，并且位于`CITY`”。注意，这个谓词是二元的，正如二元关系所预期的那样。
2.  考虑关系`r1`和`r2`，其中`r1`是`S{CITY}`的投影，`r2`是`P{CITY}`的投影。那么`r1`和`r2`完全有可能相等；尽管如此，它们具有不同的谓词，对应于它们各自的两个不同上下文（粗略地说，谓词分别是“存在一个位于`CITY`的供应商”和“存在一个存储在`CITY`的零件”）。

#### 关系类型

设`{ H }`是一个标题（heading）；那么`RELATION { H }`是一个关系类型（relation type），它与`{ H }`具有相同的度和属性。

示例：
*   关系变量`S`的类型是：
    ```
    RELATION { S# S#, SNAME NAME,
               STATUS INTEGER, CITY CHAR }
    ```
*   以下类型（对应于关系变量`S`的某个特定投影）也是一个关系类型：
    ```
    RELATION { CITY CHAR, SNAME NAME }
    ```

#### 关系类型生成器

参见类型生成器（type generator）。

#### 关系类型推导

确定给定关系表达式所表示的值的类型的过程。注意，这个过程完全由定义各种关系运算结果类型的规则所指定。

#### 关系值

粗略地说，是一个表（value）。更精确地说，设`RELATION { H }`是一个关系类型，并设`{ b }`是一个类型为`TUPLE { H }`的元组集合。

设`r`是`<{ H }, { b }>`这个序对。那么`r`就是一个关系值（relation value，简称关系），其类型是`RELATION { H }`，具有标题`{ H }`和主体`{ b }`，并且与`{ H }`具有相同的度和属性，与`{ b }`具有相同的基数（cardinality）。

对比：关系变量（relation variable）；关系变量（relvar）。

注意：根据这个定义，一个关系并不真正包含元组（它包含一个主体，而该主体又包含元组），但通常为了简化，会说成关系直接包含元组。另请注意，关系模型中的关系与同名的数学构造在某些重要方面有所不同。特别是，数学中的关系通常没有命名的属性；相反，它们的属性是通过从左到右的序数位置来标识的。关于其他差异，参见关系（数学）（relation (mathematics)）。

#### 关系值属性

一种类型为某种关系类型的属性。此类属性的值是指定类型的关系（有时称为嵌套关系，因为它们“嵌套”在元组中——尤其是其他关系中的元组内）。

注意：如果一个关系变量具有关系值属性，这一事实本身并不构成对任何特定范式级别（甚至第一范式）的违反；然而，这种属性通常在基本关系变量（base relvar）中是不推荐的，因为它们必然暗示某种结构上的不对称性，从而在查询、约束和更新中也引发不对称性（和复杂性）。另见分组（grouping）；去分组（ungrouping）。

#### 关系变量

粗略地说，是一个表（variable）；更精确地说，是一个类型为某种关系类型的变量。设关系变量`R`的类型是`RELATION { H }`；那么`R`具有与该类型相同的标题（因此也具有相同的属性）和度。设在某个给定时间`R`的值是`r`；那么在该时间`R`具有与`r`相同的主体和基数。注意，关系变量*不是*一组元组变量（tuple variable）（甚至不是一组相同类型的元组变量）。

对比：关系（relation）；关系值（relation value）。

#### 关系代数

一个开放的关系运算符集合，每个运算符接受一个或多个关系作为操作数，并产生一个关系作为结果。具体包含哪些运算符有些随意，但整个集合被要求至少与关系演算（relational calculus）一样强大，其意义是每一个关系演算表达式都语义等价于某个关系代数表达式。
