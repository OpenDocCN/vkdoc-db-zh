#### 连接陷阱
一些批评者用来指称关系模型中所谓缺陷的术语。考虑表达式`(S JOIN P){S#,P#}`。该表达式表示一个关系，假设为*r*，它碰巧包含元组`<S2,P5>`（因为供应商`S2`和零件`P5`都位于巴黎）。现在，显然不能从这个事实（至少不能有效地推断）推断出供应商`S2`供应零件`P5`——*r*的谓词是“供应商`S#`和零件`P#`位于同一个城市”，而不是“供应商`S#`供应零件`P#`”。然而，批评者声称，用户仍然会做出这种无效推断，因此关系模型是有缺陷的，因为它让用户陷入这个陷阱。但是，从这个例子应该清楚，缺陷不在于模型本身，而在于那些用户（或者也许是那些批评者）未能理解连接的语义。
*注意*：如例子所示，*连接陷阱*一词通常被视为与连接操作相关的问题（事实上，一些作者甚至因此将其称为连接陷阱）；然而，类似的问题也可能出现在其他操作中。
