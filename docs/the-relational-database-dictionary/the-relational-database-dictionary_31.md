#### 关系赋值

将一个关系值赋给一个 `关系变量`（同类型）的操作。*参见* `赋值`。

#### 关系演算

`谓词演算` 的一种应用形式，专门用于对关系进行操作，其特性是每个关系演算表达式在语义上等价于某个关系代数表达式。

*另见* `域演算`；`元组演算`。

#### 关系比较

一种 (`exp1`) `theta` (`exp2`) 形式的布尔表达式，其中 `exp1` 和 `exp2` 是相同类型 `T` 的关系表达式，`theta` 是任何对类型 `T` 的关系有意义的比较操作符（例如，“=”、“≠”、“⊆”）。*注*：在比较中包围 `exp1` 和 `exp2` 的圆括号在实践中可能并不需要。

#### 关系完备性

衡量一种语言表达能力的尺度；本质上，一种语言是关系完备的，当且仅当它至少和关系演算一样强大，这意味着任何可以由某个关系演算表达式定义的关系，也可以由该语言的某个表达式定义。

*示例*：`关系代数` 是关系完备的，因为每个关系演算表达式在语义上都等价于某个关系代数表达式。（事实上反过来也成立；也就是说，每个关系代数表达式在语义上都等价于某个关系演算表达式，至少对于通常定义的代数和演算来说是这样。）例如，在 `元组演算` 条目下作为示例展示的关系演算表达式，在语义上等价于以下关系代数表达式：

```
(S MATCHING SP){SNAME}
```

因此，为了证明一个给定的语言 `L` 是关系完备的，只需证明每个关系代数表达式都等价于 `L` 中的某个表达式——这通常比证明每个关系演算表达式都等价于 `L` 中的某个表达式要容易。例如，`SQL` 可以通过这种方式被证明是“近乎”关系完备的（“近乎”，因为 `SQL` 未能支持 `TABLE_DEE` 和 `TABLE_DUM`）。*注*：实际上，`SQL` 是“超越”关系完备的，因为它的表达式允许定义许多根本不是关系的对象。正如这个例子所示，超越关系完备性并不一定是件好事。
