# 赋值原则
将值 `v` 赋给变量 `V` 后，比较 `v = V` 必须求值为 TRUE。

#### 关联寻址
按值而非按位置寻址。在关系模型中，所有寻址都是关联的，这意味着其中包含明确拒绝指针（参见相关条目）等特性。

#### 结合律
设 `Op` 为一个二元运算符，并假设 `Op` 以中缀形式表示。当且仅当对于所有 `x`、`y` 和 `z`，都有 `x Op (y Op z) = (x Op y) Op z` 时，`Op` 才具有结合性。

**示例**：在普通算术中，加法（`+`）是结合的，因为对于所有数字 `x`、`y` 和 `z`，都有 `x + (y + z) = (x + y) + z`。同样，`||`（字符串连接）是结合的，因为对于所有字符串 `x`、`y` 和 `z`，都有 `x || (y || z) = (x || y) || z`。类似地，在关系代数中，`UNION` 和 `JOIN` 是结合的（相比之下，`MINUS` 不是）。同样，在逻辑中，`OR` 和 `AND` 是结合的（相比之下，`IMPLIES` 不是）。

*注意*：上述所有结合律运算符（除了 `||`）也都是可交换的（参见相关条目）。另一个具有结合性但不具有交换性的例子是二值逻辑中仅返回其第一个参数值的未命名二元连接词。*另见* 左结合性；右结合性。

#### 原子关系变量
这是一个**弃用术语**，指无法分解为独立投影的关系变量（*参见* FD 保持性）。该术语已被弃用，因为它很可能与术语*不可约关系变量*（*参见* 不可约性，第二个定义）混淆。虽然不可约关系变量总是原子的，但反过来则不一定——一个关系变量可以是原子的但不是不可约的，实际上甚至可能不在 BCNF 中。无论如何，这个概念很少需要；因此，必要时最好直接阐明其含义。

**示例**：假设关系变量 `SP` 满足附加的 FD `{QTY} → {P#}`，意味着给定发货的零件号是发货数量的函数；例如，零件 `P1`（单独）总是以数量 100 供应，零件 `P2`（单独）以数量 200 供应，依此类推（当然，此示例非常牵强，但足以说明当前目的）。此修订版的 `SP` 不在 BCNF 中（因为 `{QTY}` 不是超键），并且可以无损分解为在 `{S#,QTY}` 和 `{QTY,P#}` 上的投影。然而，这些投影虽然处于 BCNF 中，但并非独立，因为 FD `{S#,P#} → {QTY}` 在分解中没有被保留；因此，此修订版的 `SP` 不是原子的。*注意*：由此示例可知，(a) 分解为 BCNF 投影和 (b) 分解为原子投影的目标，尽管通常都是可取的，但有时可能会相互冲突。

#### 原子语句
（*编程语言*）在语法上，指内部不包含其他嵌套语句的语句（*对比* 复合语句）；在语义上，指保证要么完全执行，要么没有任何效果（可能除了返回状态码或等效值）的语句。所有语法上的原子语句在语义上也是原子的。
