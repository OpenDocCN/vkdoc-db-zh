# 关系数据库词典（扩展版）

## 相关模型中的原子性
在关系模型中，一个关系变量是原子的当且仅当其所有属性都是原子的。（顺便说一句，反之不成立；具体来说，多重赋值在语义上是原子的，但在语法上并非如此。）

#### 原子类型
**已弃用术语**，标量类型的旧称。*参见* 标量。

#### 原子值
**过时且有些弃用的术语**，指标量值。*参见* 标量。

#### 属性
宽泛地说，指一列；更精确地说，是一个 `<属性名, 类型名>` 对，尽管通常非正式地用其属性名本身来指代某个给定属性。（这种简化形式是可接受的，因为关系模型要求属性名在相关标题中必须是唯一的，因此这些名字实际上隐含了相应的类型名。）

**示例**：在供应商-零件数据库中，(a) 对 `<SNAME, NAME>` 是关系变量 `S` 的一个属性；(b) 对 `<S#, S#>` 是关系变量 `S` 和关系变量 `SP` 的共同属性。

我们也可以更简单但不太正式地说：(a) `SNAME` 是关系变量 `S` 的一个属性；(b) `S#` 是关系变量 `S` 和关系变量 `SP` 的一个共同属性。这两个属性的类型分别是 `NAME` 和 `S#`。

#### 属性赋值
*参见* 属性引用。

#### 属性约束
一种规范（概念上属于关系变量约束的一部分），用于指定给定关系变量的给定属性属于给定的声明类型。

**示例**：关系变量 `S` 的属性 `SNAME` 被声明为类型 `NAME`——也就是说，它被约束为只包含类型 `NAME` 的值。任何试图将非该类型的 `SNAME` 值引入该关系变量的操作都会立即失败。

#### 属性提取器
一种操作符，用于从指定元组中提取指定属性的值。

**示例**：令 `t` 表示图 1 中供应商 S1 的供应商元组。那么，以下表达式从该元组中提取状态值 20（一个整数）：
```
STATUS FROM t
```
这里的 `STATUS` 是一个属性引用。

## 属性 FROM
**Tutorial D** 语法中用于属性提取器的表达方式。

#### 属性引用
从语法上讲，指一个属性名（可能带点限定）。属性引用可以表示属性本身，或者该属性的值（通常但不总是出现在某个特定元组的上下文中），具体取决于上下文。需要特别注意的是，当它出现在某个 `UPDATE` 操作符调用内的“属性赋值”左侧时，它肯定表示属性本身。

**示例**：考虑以下 `UPDATE` 语句：
```
UPDATE P WHERE CITY = 'London' :
{ WEIGHT := 2 * WEIGHT , CITY := 'Oslo' } ;
```
该语句包含两个属性赋值和四个属性引用：`CITY`（两次）和 `WEIGHT`（也是两次）。设想整个 `UPDATE` 操作是按某种顺序逐个处理关系变量 `P` 的元组来执行的，令 `t` 是当前正在处理的元组。那么在整个语句中：(a) `CITY` 的第一次出现和 `WEIGHT` 的第二次出现分别表示 `t` 内的 `CITY` 值和 `WEIGHT` 值；(b) `WEIGHT` 的第一次出现和 `CITY` 的第二次出现分别表示 `t` 内的 `WEIGHT` 属性本身和 `CITY` 属性本身。*参见* `UPDATE` 以获取进一步解释。

#### 公理
被假定为真的事物，可用于推导进一步的真理（即定理）。在数据库中，基本关系中的元组可被视为公理，因为它们代表了被假定为真的命题。公理是定理的一个特例。*参见* 证明。



**示例**：关系`S`（当前为基本关系变量`S`的值）中的元组`<S1,Smith,20,London>`表示了以下这个被认为是真实的命题：

“供应商`S1`已签约，名为`Smith`，状态为`20`，且位于`London`。”

**外延公理**
集合论的一条公理，大意是两个集合相等当且仅当它们拥有相同的元素（此时它们实际上是同一个集合）。

**包**
宽松地说，是一种允许重复的“集合”；更精确地说，是一个由称为元素的个体组成的集合，其中同一个元素可以出现任意多次。一个例子是集合`(y,y,x,z,y,z)`，由于包和集合一样，其元素没有顺序，因此它等价于`(x,y,y,y,z,z)`。一个给定元素在给定包中出现的次数称为该元素（相对于该包）的多重性。

集合论中的包含、并、交、差和积（但不是补）运算都可以推广应用于包。首先是包含：令`b1`和`b2`为包，元素`x`在`b1`中恰好出现`n1`次，在`b2`中恰好出现`n2`次（`n1` ≥ 0，`n2` ≥ 0）。那么，当且仅当对于所有这样的元素`x`都有`n1` ≥ `n2`时，包`b1`包含包`b2`（`b1` ⊇ `b2`）；进一步，当且仅当`b1`包含`b2`时，`b2`被包含于`b1`（`b2` ⊆ `b1`），而当且仅当两者相互包含时，`b1`等于`b2`（`b1` = `b2`）。所有与集合包含相关的术语（超集、子集等）在包包含（超包、子包等）中都有对应的概念。

现在，令`Op`为并、交或差运算，令`b`为对包`b1`和`b2`应用`Op`（在差运算的情况下，按此顺序）所得到的包。与之前一样，元素`x`在`b1`中恰好出现`n1`次，在`b2`中恰好出现`n2`次（`n1` ≥ 0，`n2` ≥ 0）。那么，元素`x`在`b`中恰好出现`n`次，其中`n`是：
如果`Op`是并运算，则为 `MAX(n1, n2)`
如果`Op`是交运算，则为 `MIN(n1, n2)`
如果`Op`是差运算，则为 `MAX(n1 - n2, 0)`
在任何情况下，`b`都不包含任何其他元素。

现在，令元素`x1`和`x2`分别在`b1`中恰好出现`n1`次，在`b2`中恰好出现`n2`次（`n1` ≥ 0，`n2` ≥ 0），并令`b`为`b1`和`b2`（按此顺序）的积。那么，序对`<x1, x2>`在`b`中恰好出现`n1 * n2`次，且`b`不包含任何其他元素。

最后，还有两种运算，并加（union plus）和交星（intersection star）（也以其他多种名称为人所知），它们在集合论中没有对应物。

令`b`为对包`b1`和`b2`应用其中一种运算所得到的包，其中再次假设元素`x`在`b1`中恰好出现`n1`次，在`b2`中恰好出现`n2`次（`n1` ≥ 0，`n2` ≥ 0）。那么，元素`x`在`b`中恰好出现`n`次，其中`n`是：
如果`Op`是并加运算，则为 `n1 + n2`
如果`Op`是交星运算，则为 `n1 * n2`

**注意**：SQL 支持并加但不支持真正的包并。它不支持交星。

**示例**：令`b1`和`b2`分别为包`(w,w,x,x,y)`和`(x,y,y,y,z,z)`。那么以下表达式产生所示结果：

```
b1 UNION b2 = (w,w,x,x,y,y,y,z,z)
b1 INTERSECT b2 = (x,y)
b1 MINUS b2 = (w,w,x)
b2 MINUS b1 = (y,y,z,z)
b1 TIMES b2 = (<w,x>,<w,x>,<x,x>,<x,x>,<y,x>,
                <w,y>,<w,y>,<x,y>,<x,y>,<y,y>,
                <w,y>,<w,y>,<x,y>,<x,y>,<y,y>,
                <w,y>,<w,y>,<x,y>,<x,y>,<y,y>,
                <w,z>,<w,z>,<x,z>,<x,z>,<y,z>,
                <w,z>,<w,z>,<x,z>,<x,z>,<y,z>)
b1 UNION+ b2 = (w,w,x,x,x,y,y,y,y,z,z)
b1 INTERSECT* b2 = (x,x,y,y,y)
```

**包属于（对于一个元素）**
指某个元素出现在给定包中的属性；指测试该属性的操作。类似于集合属于（参见），包属于通常用符号“∈”表示；因此，布尔表达式`x ∈ b`当且仅当元素`x`确实至少在包`b`中出现一次时返回`TRUE`。

**基本关系**
在给定时间，某个给定基本关系变量的值。*对比* 导出关系。

**示例**：在任何给定时间，作为关系变量`S`、`P`和`SP`的值的关系。

**基本关系变量**
不是用其他关系变量定义的关系变量；即，独立的关系变量。*对比* 导出关系变量。*注意*：一个普遍的误解是基本关系变量是物理存储的，即它们在存储中由物理文件表示，其元组和属性在存储中由这些文件内的记录和字段表示（*参见* 直接映像）。但关系模型故意对物理存储不作任何规定；特别是，它明确地不表示基本关系变量本身是物理存储的——既不是在上述意义上，也不是在其他任何意义上。唯一的要求是必须存在从物理存储的内容到用户感知的内容（即基本关系变量或导出关系变量或两者的混合）的某个已定义映射，反之亦然。

**示例**：关系变量`S`、`P`和`SP`。

**双射**
从集合`s1`到集合`s2`的一个映射或函数，其中`s2`的每个元素恰好是`s1`的一个元素的像；等价地，一个既是单射又是满射的映射（换言之，是从`s1`到`s2`的一个严格意义上的一一对应）。也称为双射映射或“一一映上”映射。注意，如果一个给定的映射是双射，那么它也有一个也是双射的逆映射。

**示例**：从整数`x`到其后继`x + 1`的映射是从所有整数的集合到其自身的一个双射。从整数`x`到其前驱`x - 1`的逆映射同样是一个双射。

**二元**
度为二。

**绑定（逻辑）**
通过量化（参见）将自由变量转换为约束变量。

**体**
一组同类型的元组；特指出现在给定关系中，或在给定时间出现在给定关系变量中的一组元组。体的每个子集本身也是一个体。

**示例**：在任何给定时间出现在关系变量`S`中的一组元组；该组的任意子集。

**布尔**
一种标量数据类型——关系模型唯一要求的标量类型——仅包含两个值（具体是两个真值，分别由字面量`TRUE`和`FALSE`表示）。

**布尔代数**
1.  （简单情况）真值`TRUE`和`FALSE`，连同逻辑运算符`NOT`、`OR`和`AND`（参见）。
2.  （一般情况）令`s`为一个集合；令“≤”为`s`上的一个偏序（参见）；并令一个一元运算符“¬”（补）和两个不同的二元运算符“+”（加法）和“*”（乘法）在`s`上定义，使得 (a) “¬”满足闭合律和对合律；(b) “+”和“*”满足闭合律、交换律、结合律、分配律、幂等律和吸收律（特别是分配律意味着每个“+”和“*”都对另一个运算分配）；以及 (c) “¬”、“+”和“*”共同满足德·摩根定律。令`s`还包含两个元素`0`和`1`。
