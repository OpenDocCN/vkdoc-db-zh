`absorption` 设运算符 *OpC* 和 *OpD* 都是二元运算符，并且为了明确起见，假设它们以中缀风格表示。那么 *OpC* 吸收 *OpD* 当且仅当，对于所有的 *x* 和 *y*，`x OpC (x OpD y) = x`。

*示例*：在逻辑中，`OR` 和 `AND` 彼此吸收对方，因为 `x OR (x AND y)` 和 `x AND (x OR y)` 都简化为 `x`。类似地，在集合论和关系代数中，`union`（并）和 `intersection`（交）也彼此吸收对方。

`abstract data type` 类型。*注*：该术语有时用于指代某种特定类型的类型（尤其是非内置的类型），但有充分理由认为所有类型都应该是“抽象的”，至少在其物理表示对用户隐藏这个意义上如此。

`access path` 一种实现构造。典型的例子包括哈希、索引和指针链。在关系模型中没有访问路径——对关系的所有访问都是通过关联寻址（参见）进行的。

`actual operand` *参见* argument（实参）。

`ad hoc polymorphism` *参见* overloading（重载）。

`aggregate operator` 一种只读运算符，它从作为某个关系某个属性值出现的“聚合”（即集合或包）中导出单个值（通常是但不一定是标量值）——或者，在 `COUNT`（它稍微特殊一点）的情况下，是从整个关系本身这个“聚合”中导出。*对比* summary（汇总）。*注*：如果 (a) 某个聚合运算符调用所基于的关系是空的，并且 (b) 该调用本质上只是对某个二元标量运算符重复调用的简写，并且 (c) 该标量运算符存在一个幺元（参见），那么该调用的结果就是那个幺元值。例如，假设运算符 `SUM` 在一个由一组 `INTEGER` 类型值组成的聚合上调用。由于 `SUM` 本质上只是对标量运算符“+”重复调用的简写，并且对于整数上的“+”存在一个幺元值——即零，那么如果聚合为空，结果就是零。
