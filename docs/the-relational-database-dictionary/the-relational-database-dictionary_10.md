#### 代数

1.  泛指一个由一组元素和一组只读运算符构成的形式系统，这些元素和运算符共同满足特定的定律和属性（必然包括封闭性，可能包括交换律和结合律等）；亦称为代数结构或抽象代数。`代数`一词本身源自阿拉伯语`al-jebr`，意为（破碎之物的）复位或组合。*另见* `布尔代数；域（数学）；群（数学）；代数定律，The；关系代数；环（数学）。`
2.  特指关系代数（如果上下文有此要求）。

#### 替代键

非正式地指不是主键的候选键。

更精确地说，让关系变量 `R` 拥有键 `K1, K2, . . . , Kn,` 并选择某个 `Ki` (`i = 1, 2, . . . , n`) 作为 `R` 的主键；那么每个 `Kj` (`j = 1, 2, . . . , n, j ≠ i`) 都是 `R` 的一个替代键。该术语不常用。

#### 出现

*（值的）* 某个值的出现或“实例”（在某个上下文中）。注意值本身与其出现之间存在逻辑差异——例如，作为某个变量的当前值出现，或作为某个元组变量或关系变量的当前值中的属性值出现。每个这样的出现，在内部都由该值的某种物理表示组成（同一值的不同出现可能有不同的物理表示）。因此，值的出现与其物理表示之间也存在逻辑差异；同一值的不同出现所使用的物理表示之间甚至可能存在逻辑差异。然而，尽管如此，通常将*值的出现之物理表示*缩写为*值的出现*，或（更常见地）直接缩写为*值*，只要不存在歧义风险。

注意`值的出现`是一个模型概念，而`出现的物理表示`是一个实现概念——用户当然可能需要知道（例如）两个变量是否包含同一值的出现，但他们不需要知道这些出现是否使用相同的物理表示。

**示例**：令 `N1` 和 `N2` 为 `INTEGER` 类型的变量。经过以下赋值后，`N1` 和 `N2` 都包含整数值 `3` 的一个出现。对应的物理表示可能相同也可能不同（例如，`N1` 可能使用二进制表示，而 `N2` 使用十进制表示），但无论哪种情况，用户都无需关心。

```sql
N1 := 3 ;
N2 := 3 ;
```

## 实参

当调用某个运算符时，替换其某个参数的实际操作数。注意实参本身与表示它的表达式（即实参表达式）之间存在逻辑差异。实参本身要么是一个值，要么是一个变量。如果相关参数是可更新的，那么实参——事实上，必须是——一个变量，由某个变量引用表示；否则它是一个值，可以由任意复杂的表达式（可能只是一个变量引用）表示。*对比* `形参`。

**示例**：令运算符 `DOUBLE` 定义如下：

```sql
OPERATOR DOUBLE ( X INTEGER ) RETURNS INTEGER ;
    RETURN ( 2 * X ) ;
END OPERATOR ;
```

这里的 `X` 是一个形参，声明类型为 `INTEGER`。令 `N` 为 `INTEGER` 类型的变量。那么，例如 `DOUBLE(N+1)` 是对 `DOUBLE` 的一次调用，在该调用时刻表达式 `N+1` 的值是该调用的一个实参——实际上是唯一的实参。该调用本身又是一个表达式，它可以出现在任何整型字面量可以出现的地方（因为运算符 `DOUBLE` 被定义为返回 `INTEGER` 类型的值）。

现在假设 `DOUBLE` 被定义为一个更新运算符而非只读运算符：

```sql
OPERATOR DOUBLE ( X INTEGER ) UPDATES { X } ;
    X := 2 * X ;
END OPERATOR ;
```

现在形参 `X` 是可更新的，任何对应于 `X` 的实参必须是一个变量。因此，例如 `DOUBLE(N)` 是 `DOUBLE` 的有效调用，并且变量 `N`——注意，不是该变量的值——是该调用的实参。（注意，例如 `DOUBLE(N+1)` 会是语法错误，因为 `N+1` 不是变量引用。）然而，该调用 `DOUBLE(N)` 不是一个表达式，它不能出现在“任何整型字面量可以出现的地方”；相反，它只能出现在显式的 `CALL` 语句（或等效语句）中，如下所示：

```sql
CALL DOUBLE ( N ) ;
```

## Armstrong 推理规则

*（针对函数依赖 FD）* 令 `A, B,` 和 `C` 为某个关系变量头部的子集。令 `AC` 表示 `A` 和 `B` 的集合理论并集，`BC` 类似。那么 Armstrong 规则（也称为 Armstrong 公理）指出：(a) 如果 `A` 是 `B` 的超集，则 `A → B`（自反律）；(b) 如果 `A → B`，则 `AC → BC`（增广律）；以及 (c) 如果 `A → B` 且 `B → C`，则 `A → C`（传递律）。这些规则既是可靠（*见* `可靠性`）又是完备的（*见* `完备性`）。

**示例**：令 `s` 为一个 FD 集合，并且 `s` 包含 FD `A → BC`。那么 FD `A → B` 被 `s` 蕴含，并且可以容易地使用 Armstrong 规则推导如下：(a) `A → BC`（给定）；(b) `BC → B`（自反律）；因此 (c) `A → B`（传递律）。

作为第二个例子，如果集合 `s` 包含 FD `A → B` 和 `C → D`，那么 FD `AX → BD`（其中 `X` 是 `C` 和 `B` 的集合理论差集，按此顺序）被 `s` 蕴含。*注意*：这个由 Darwen 提出的例子可以被视为另一个推理规则。它有一个有趣的性质，即增广律和传递律，以及此处未讨论的其他几个规则，都是它的特例。

#### 赋值
将一个值（源，由表达式表示）赋给一个变量（目标，由变量引用表示）的运算符；也指调用该运算符时执行的操作。源和目标必须具有相同类型。*注意*：每个更新运算符的调用在语义上等价于某个赋值操作（可能是一个多重赋值，参见相关条目）。
