# 章节概览

## 第 1 章：故事讲述者的数据库

本章旨在为您进入数据和叙事的世界奠定基础。本章将数据库作为一种讲故事的工具进行介绍，阐述了叙事结构和关系模型如何帮助赋予数据意义。在学习本章的过程中，您将获得开始这段旅程所需的所有信息，为数据故事讲述的艺术与科学奠定基础。您将了解到，SQL 不仅能够查询数据，还能讲述引人入胜的故事。接下来，在搭建好故事讲述环境后，您将开始更详细地探索 SQL 的复杂性。

## 第 2 章：从 SELECT 开始

在本章中，您将学习如何使用`SELECT`语句从表中提取和探索基本数据。使用 SQL 中最常用的命令，您可以有效地检索和操作数据。这要求您学习如何编写精确的`SELECT`语句，以检索您的叙事所需的信息。这将为更高级的数据操作和分析技术奠定基础。


## 第 3 章：用 WHERE 筛选事实

在本章中，你将发现另一个 SQL 命令，它常用于通过使用 `WHERE` 条件、比较和逻辑运算符来优化数据检索。这需要创建精确的 `WHERE` 语句，根据特定条件过滤数据。这将使你能够优化数据集并提取更深层次的洞见，进而帮助你用数据讲述更丰富的故事。

## 第 4 章：用 JOIN 塑造复杂角色

本章旨在探讨如何使用 `JOIN` 连接多个表，基于不同的数据来源创建更丰富的数据叙事。讨论了 `JOIN` 操作，这是组合多个表数据的基础。当你精通此操作后，你将能创建复杂的查询，为你的记录提供更深入的洞见和更全面的分析。

## 第 5 章：聚合行为

本章旨在介绍聚合函数，如 `COUNT`、`SUM`、`AVG`、`MIN` 和 `MAX`，这些函数可用于汇总和分析分组数据。在 SQL 中，聚合行为就是将聚合函数应用于分组的数据子集。这些操作使得 SQL 能够从数据中提取有用的汇总和统计信息，用于分析和决策。

## 第 6 章：用 ORDER BY 和 LIMIT 安排情节

在本章中，你将学习如何对查询结果进行排序和限制输出，以提高可读性和性能。重点在于高效地排序和过滤查询结果。一旦掌握此操作，你将能够有意义地组织数据。为了聚焦于最相关的数据点，你可以优先考虑关键信息，并将结果限制在最相关的数据点上。

## 第 7 章：用子查询实现动态对话

本章旨在介绍子查询，它是嵌套逻辑和构建复杂、分层数据请求的强大工具。本章探讨编写子查询的艺术，以便为数据分析增加深度和维度。本章概述了子查询、其类型，以及在动态对话中使用的叙述性示例。

## 第 8 章：数据绘图中的条件逻辑

本章解释了如何利用 SQL 的条件逻辑来转变数据分析和可视化工作流程，从而实现基于逻辑的数据可视化和转换。你将学习 SQL 中的条件逻辑，对数据进行分类，应用动态过滤以提高图表相关性来增强可视化效果，为可视化创建颜色编码数据，使用条件表达式聚合数据，以及处理可视化中的缺失数据。本章的重点不在于如何可视化或绘制数据，而在于准备数据的关键过程。在本章中，SQL 用于在数据可视化之前对数据进行操作、清理和结构化。

## 第 9 章：用索引和视图优化你的脚本

本章阐明了如何使用索引来提高查询性能，以及如何使用视图来简化逻辑。处理大量数据时，优化 SQL 查询可以显著提升性能。索引和视图都是实现此类优化的强大工具。索引使数据库引擎能更高效地定位行，从而减少为检索数据而进行的全表扫描。另一方面，视图通过存储可重用的 SQL 逻辑来简化复杂查询，从而提高可读性和可维护性。

## 第 10 章：分析炼金术：点数据成金

用 SQL 将数据点石成金，需要掌握高级分析函数，帮助你从原始数据中提取更深层次的洞见，并将其转化为引人入胜的叙事。SQL 语言提供了强大的函数，可将原始数据转化为引人入胜的叙事。本章还讨论了如何将原始数据转化为一个引人入胜的故事，以及如何使用递归查询来有效地构建查询逻辑。

## 第 11 章：盛大终章：呈现你的数据故事

在本章中，你的旅程即将结束。通过逐章学习，你学会了如何从原始数据中提取深刻的洞见和信息，以使用原始数据解决复杂和高级的分析问题。本章总结了前面的章节，并就如何呈现数据分析的叙事提供了见解。
