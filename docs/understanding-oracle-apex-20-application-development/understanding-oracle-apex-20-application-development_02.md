# 为何需要本书？

使用 APEX 设计页面似乎很简单——你只需选择你想要的组件，然后为它们的属性分配合适的值即可。选择组件是直接的，但为属性赋值却并非如此。一个页面及其组件拥有众多属性，你必须了解这些属性的用途才能知道该赋什么值。这些属性的范围从必不可少的（例如报表的 `source query`（源查询））到难以理解的（例如报表的 `static ID`（静态 ID））。有些属性（例如报表列的 `HTML expression`（HTML 表达式））是挂钩，允许你在生成的网页中插入自定义的 HTML 或 JavaScript 代码。

本书旨在温和地引导你穿越这片属性的宝库。为此，本书将开发一个演示 Web 应用程序，用以说明构建典型网页功能的各种 APEX 技术。该应用程序的页面起初很简单，然后逐渐增加其复杂程度。对于每一个页面，我会引入几个新属性，讨论它们的用途，并展示其效用。到本书结束时，你将深深沉浸在 APEX 属性的世界里，从而有足够的自信去应对你自己的任何网站项目。如果你的项目需要比这里展示的更复杂的功能，你也应该能够自如地使用未涵盖的属性，或许通过查阅文档、查看 Oracle 提供的众多预打包应用程序、浏览网络论坛，甚至自己摸索出来。

在 APEX 中构建网页的另一种方法是依靠向导（wizards）。APEX 提供向导来生成常见组件，例如报表页面和数据录入表单。每个向导会询问你一系列问题（例如“页面的名称是什么？”“你想显示哪个表？”“页面是否应在导航菜单中有条目？”），然后利用你的回答生成具有相应属性的合适组件。其优势当然是你不需要了解任何关于属性的知识。缺点是向导往往会产生“一刀切”的页面，无论是在外观还是功能上。

向导只能带你走这么远。如果你想要对页面的外观、感觉和行为进行任何形式的控制，你就需要参与到其属性中来。本书提供了你所需的指导。
