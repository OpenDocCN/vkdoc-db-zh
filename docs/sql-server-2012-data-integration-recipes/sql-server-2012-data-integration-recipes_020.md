## 引言

**Microsoft SQL Server 2012** 是一个庞大的主题。这个强大而全面的数据库生态系统的一个重要部分，经过多年的显著发展，便是数据集成——或者如果你想使用另一个几乎同义的术语，那就是 **ETL**。`BCP` 作为唯一可用的数据加载或导出工具的日子早已一去不复返。就连 `DTS` 现在也已成为遥远的记忆。如今，用户在将数据导入和导出 **Microsoft RDBMS** 时，面对琳琅满目的工具和选项，可谓眼花缭乱。本书旨在阐明将数据加载到 **SQL Server** 以及从 **SQL Server** 发送到外部世界的多种方式。我也尝试针对各种不同的挑战和情况，提供一些关于哪些技术最为适用的见解。

然而，本书并不仅仅是一本 `SSIS` 手册。我对这款优秀的产品怀有深深的敬意，但并不认为它是某些开发者所认为的“一站式解决方案”。我希望向读者展示，在许多 `ETL` 场景中，通常存在可以有效应用的替代技术。事实上，我的理念是，在处理数据时，应始终应用正确的解决方案，并永远不要相信只有一个答案。因此，本书收录了关于 **SQL Server** 生态系统中许多其他工具的实践方法。有时，我特意展示了处理本质上相同挑战的不同方法。我希望通过这样做来激发你的好奇心，并提供一些实用的示例，展示如何将来自无数源的数据清晰、高效地导入 **SQL Server** 数据库。

尽管本书主要面向 **SQL Server 2012** 的用户，但我尽可能在可行时说明某个方法是否适用于该数据库的先前版本。我也尝试指出 **SQL Server 2012** 与旧版本之间的任何新特性和差异。这是因为用户不太可能只处理该 RDBMS 的最新版本，在大多数站点上很可能同时运行着多个版本。在指出数据库的演变时，我只回溯到 **SQL Server 2005**，因为该版本引入了 `SSIS`——这是基于 **SQL Server** 的 `ETL` 的一个重大转折点。

由于本书侧重于 **SQL Server**，几乎所有使用的代码都是 `T-SQL`。提供的示例有些极其简单，有些则更为复杂。所有内容都围绕 `ETL` 需求展开。因此，你不会在本书中找到基于 `OLTP` 或 `DBA` 的示例。在处理 **Analysis Services** 数据时，你会看到一些 `MDX` 的应用；在使用 `SSIS` 脚本任务时，则会用到一些 `VB.Net`。根据我的经验，在本书描述的几乎所有 `SSIS` 脚本任务中，我选择使用 `VB.Net`，因为它是许多 `T-SQL` 程序员最为熟悉的 .Net 语言。尽管如此，为了避免被指责忽视特定语言，我也添加了一两个 `C#` 代码片段（特别是在使用 `CLR` 程序集的地方）。

数据集成是一个庞大的主题。因此，为了给这个潜在巨大且分散的领域应用一点结构，本书分为两个主要部分。

`第一部分`——第 1 章 到 第 7 章——涉及将数据导入和导出 **SQL Server** 的机制。在这里，你将找到如何连接各种数据源并随后“吞咽”（ingurgitate）数据的基本细节。针对每个数据源，尽可能多地提醒你注意潜在的陷阱和圈套。

`第二部分`——第 8 章 到 第 15 章——涉及更广泛的 `ETL` 环境。在这里，我们从基础的“螺母螺栓”进展到提取、转换和（高效）加载数据的协调整体。这些章节带领读者踏上一段旅程，涵盖元数据分析、数据转换、源数据剖析、数据过程日志记录以及优化数据加载的一些方法。

在本书中，我决定避免使用无处不在的 `AdventureWorks`，而使用我自己的示例数据库。这样做有几个原因。首先，我认为 `AdventureWorks` 过于庞大和复杂，可能会分散读者对我想要解释的一些技术的注意力。我更喜欢使用极其简化的数据结构，让读者能够自由地专注于所解释内容的本质，而不是数据本身。其次，我希望避免 `AdventureWorks` 中存在的多重关联表和外键所带来的额外复杂性。最后，我不想使用需要花费时间加载的数据。这样，你就可以再次专注于过程和原理，而不会在看着时钟滴答作响、成千上万条记录涌入表中、伴随着屏幕上旋转的图像或昏暗的硬盘指示灯闪烁时，患上“`ETL`-凝视症”。因此，我宁愿使用一套极其简洁的源数据。源数据库的完整描述见附录 B。

另请注意，本书不打算作为循序渐进的自学手册。强烈建议你在实践之间删除并重建示例数据库，以确保在测试提供的示例时拥有一个干净的环境。事实上，基于实践（recipe-based）的全部理念是，你可以随时查阅以找到帮助，除非有明确的指示表明某个实践需要预先阅读或建立在先前的解释之上。

本书中的实践涵盖了极其广泛的需求，从非常简单到相对复杂都有。这是为了尽可能覆盖广泛的领域。其结果是，对于某些读者来说，一些实践可能显得过于简单，而另一些读者可能会怀疑更高级的解决方案是否与他们的工作相关。我只能希望 **SQL Server** 新手能找到简单的答案，而高级用户也能找到能增加他们知识的调整和建议。在所有情况下，我真诚地希望你会发现本书很有用。

不可避免的，一本书无法回答所有问题，也无法解决所有议题。我真诚地希望我已经涵盖了你将面临的许多基本的 `ETL` 任务，并为你可能遇到的相当数量的问题提供了解决方法。因此，对于任何没有在其特定问题上找到答案的读者，我表示歉意，但编写一本百科全书并非可行的选择。无论如何，我只能鼓励你阅读除了你感兴趣的确切主题之外的其他实践，因为你可能会在本书的其他地方找到潜在的解决方案。

我祝你使用 **SQL Server** 提取、转换和加载数据时好运。并且我真诚地希望你能从中获得与我写这本书时同样多的乐趣。

—亚当·阿斯平（Adam Aspin）
