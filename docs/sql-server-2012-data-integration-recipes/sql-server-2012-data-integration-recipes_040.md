# 工作原理

与 Excel、Access 和大多数关系数据源的情况一样，您可以将带分隔符的平面文件数据源用作链接服务器。由于链接服务器需要 OLEDB，因此您必须使用基于 OLEDB 的 ODBC，或者使用 OLEDB 提供程序。但请注意，这将在您使用的任何驱动程序上运行得**极其缓慢**——而且某些驱动程序（例如 Jet）仅限于 32 位环境。还需要注意的是，需要 `Schema.ini` 文件。此方法最好在以下情况下使用：

*   您拥有结构一致的、带分隔符的 CSV 或文本文件时。
*   您希望经常使用即席查询在 T-SQL 代码中加载平面文件数据，但又不想使用 SSIS 导入数据时。
*   源数据可能经常变化，因此需要立即访问，无需任何形式的数据导入时。
*   源数据量不太大，且快速查询不是必需品时。

我更喜欢使用 ACE 驱动程序来设置指向文本文件的链接服务器，因为它适用于 32 位和 64 位环境，并且对我来说一直有效。

如果您希望避免使用 T-SQL 来设置链接服务器，可以使用 SSMS 执行此任务。这在配方 1-13 中有说明。链接服务器的安全性对于平面文件来说无关紧要——因此，只要使用链接服务器的 SQL 帐户能够访问存储平面文件的文件夹，至少在这个层面上没有额外的复杂性。
