# 2（撰写时版本）包含以下增强功能
*   支持迁移至 SQL Server 2012。
*   用于提升规模和性能的多线程数据迁移。
*   支持将非英语数据库迁移到 SQL Server 的全球化功能。

之前我提到过会解释如何安装 SSMA，具体步骤如下：
1.  解压你下载的 zip 文件中的所有文件。
2.  使用合适的工具解压安装文件 `SSMA for Access 5.2.exe`。
3.  双击 `SSMA for Access 5.2.exe`。
4.  在欢迎页面，单击“下一步”。
5.  如果你未安装必备组件，将会出现一条消息，提示你必须先安装所需组件。请确保已安装所有必备组件（见前文列表），然后重新运行安装程序。
6.  选择“我接受协议”选项以同意条款。
7.  单击“下一步”。
8.  选择或取消选择功能报告框。
9.  单击“下一步”。
10. 在“选择安装类型”页面，单击“典型”（或根据需要选择其他类型，如“自定义”或“完全”）。
11. 单击“安装”。
12. 单击“完成”。

首次运行 SSMA 时，你需要注册许可证并下载免费许可证密钥。这需要一个 Windows Live ID。完成此操作后，指向包含已下载许可证密钥的目录，然后单击“刷新许可证”以应用许可证并启动 SSMA。

#### 提示、技巧与陷阱
*   如果你认为迁移过程会相当简单，那么使用迁移向导可能更容易。
*   SSMA 不允许你仅选择源表中的某些字段，这是一个“全有或全无”的过程。如果你只想加载表的一个子集，最快的解决方案是先运行一个 Access 生成表查询，将数据子集定义为一个新表。
*   在创建目标元数据时（通过选择“转换架构”），你可以在“Access 元数据资源管理器”窗口中选择数据库和表层次结构中的一个级别，以便为该级别下的所有对象创建架构。
*   选择“工具” ![image](img/arrow.jpg) “转换、加载和迁移”，即可在目标数据库元数据中创建表，并一气呵成地传输数据。
*   SSMA 不会使用 SQL Server 测试其生成的架构；它只是展示架构可能的样子。
*   Access 查询会被转换为 SQL Server 视图——而非数据表。若要让 SSMA 创建表，请先使用 Access 生成表查询来创建要导出的表。
*   若要预览 Access 表中的源数据，请单击右上角窗格中的“数据”选项卡。这样，你无需打开 Access 即可查看源数据。
*   可以随时保存 SSMA 项目——只需单击“文件” ![image](img/arrow.jpg) “保存”（或“保存”工具栏按钮）。当然，你也可以重新打开先前保存的项目以继续工作。
*   如果你的任何 Access 数据库被修改，这些更改不会自动反映在 SSMA 的“Access 元数据”窗口中。要更新元数据，请右键单击“数据库”（以更新所有元数据）或某个数据库名称（以更新单个数据库的元数据），然后选择“从数据库刷新”。
