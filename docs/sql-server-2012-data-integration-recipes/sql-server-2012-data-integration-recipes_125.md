# 为什么选择自定义剖析例程

如果你同意前面的前提，你可能仍然想知道：为什么要费心去开发自己的剖析例程，当已经存在一个相当好的 SSIS 任务时？嗯，你可能更喜欢自定义方法的原因有几个：

*   数据剖析任务仅适用于 SQL Server 源（2000 及以上版本），而自定义剖析可以适用于许多不同的数据源。
*   你可能希望执行剖析而不一定使用 SSIS。
*   数据剖析任务仅从 SQL Server 2008 开始可用。在某些情况下，你可能仍在使用 SQL Server 2005。
*   为了节省时间，你可能希望在数据摄取期间对现有数据流执行数据剖析，而不是运行一个独立于数据流的任务。
*   你想要超越 SSIS 中可用的“开箱即用”的剖析请求类型。
*   你想为剖析结果添加进一步的统计分析。
*   作为 ETL 过程的一部分，`XML`输出可能需要一些笨拙的变通方法来读取。
*   你想在运行耗时耗资源的 ETL 过程之前测试数据是否有足够的可能准确性。使用剖析任务时这并不容易。
*   你希望剖析捕获为统计上可疑的记录从数据流中移出，和/或输出到单独的数据目的地进行分析。
*   你想执行有针对性的子集分析——也就是说，你已经经历过倾向于引起问题的数据类型，并且你希望剖析具有某些特征的数据以减少剖析时间。

本章描述的技术不会试图自动化数据清理，因为这是一个我不会详细探讨的主题。然而，我希望表明，有效使用数据剖析技术可以是数据清理中极其有用的第一步（有些人可能会说是必不可少的一步）。

本章概述的技术都是具体的，而不是通用的。我的意思是，本章描述的每项剖析或分析技术的所有部分都必须组装到一个适应特定数据源的自定义流程中。这些方法不试图适应任何数据源，也不试图自动适应数据源。

## 前提条件

要测试本章中的示例，你必须从本书的配套网站下载示例文件，并将其安装在`C:\SQL2012DIRecipes\CH10`文件夹中。你还需要两个示例数据库`CarSales`和`CarSales_Staging`，它们也在网站上。

## 10-1\. 剖析数据属性

### 问题

你想从 SQL Server 数据源获取属性信息（计数、NULL 记录、字段长度和其他基本细节）。

### 解决方案

使用 T-SQL 函数来剖析和返回源数据的属性。这是一个例子：

```sql
SELECT COUNT(*) FROM CarSales.dbo.Stock WHERE Marque IS NULL;
```

### 工作原理

我们从使用 T-SQL 进行数据剖析开始。此代码段返回特定列有`NULL`值的记录数。

为了进一步解释这个概念，我假设你正在分析已经加载到 SQL Server 中的数据。根据我的经验，当你首次将数据加载到暂存数据库，最终将传输到 ODS（操作数据存储）或数据仓库时，这是一个常见的场景。在配方 10-3 中，你将看到如何使用它们处理尚未进入 SQL Server 的数据。幸运的是，这种方法只需要你应用一系列你可能已经知道的内置函数。它们显示在表 10-1 中。

**表 10-1. T-SQL 中的属性剖析函数**

| 函数 | 在剖析中的用途 |
| :--- | :--- |
| `NULL` | 检测`NULL`值。 |
| `MAX` | 获取最大数值——或者与`LEN`结合使用，获取最大字符串长度。 |
| `MIN` | 获取最小数值——或者与`LEN`结合使用，获取最小字符串长度。 |
| `COUNT` | 计算记录数。 |
| `LEN` | 获取字符字段的长度。 |

配合明智地使用`GROUP BY`和`DISTINCT`关键字，这些函数可能会覆盖你大多数基本数据属性剖析需求。确实，考虑到这些函数的简单性和易用性，一次性查看所有函数并同时了解核心属性剖析类型，可能比费力地逐一解释每个函数更快。我知道对许多读者来说，这一切可能显得过于简单。然而，为了完整性，表 10-2 给出了你可能会使用的代码片段。所有示例都假设你使用的是`CarSales.dbo.Stock`表。

**表 10-2. T-SQL 中的属性剖析示例**

| 剖析任务 | 代码 | 注释 |
| :--- | :--- | :--- |
| `NULL`剖析 | `SELECT COUNT(*) FROM dbo.Stock WHERE Marque IS NULL;` | 特定列中有`NULL`的记录有多少？ |
| `NULL`百分比 | `SELECT (SELECT CAST(COUNT(*)AS NUMERIC (18,3)) FROM dbo.Stock WHERE Marque IS NULL) / (SELECT COUNT(*) FROM dbo.Stock);` | 这代表总数的百分比。 |
| 最大值 | `SELECT MAX(ID) FROM dbo.Stock;` | 数值列中的最大值。 |
| 最小值 | `SELECT MIN(ID) FROM dbo.Stock;` | 数值列中的最小值。 |
| 最大值计数 | `SELECT COUNT(*) FROM dbo.Stock WHERE ID = (SELECT MAX(ID) FROM dbo.Stock);` | 数值列中具有最大值的数量。 |
| 最小值计数 | `SELECT COUNT(*) FROM dbo.Stock WHERE ID = (SELECT MIN(ID) FROM dbo.Stock);` | 数值列中具有最小值的数量。 |
