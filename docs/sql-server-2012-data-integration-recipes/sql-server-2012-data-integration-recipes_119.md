# 此过程的功能

此过程的功能是跟踪维度属性随时间的变化，方式是：
*   每当维度属性（任何包含描述性数据的列中的值）发生变化时，添加一条新记录。
*   记录变更发生的日期，从而确保每条新记录的起始日期，以及上一条记录失效的日期，都被保存下来。
*   为每个业务键标记当前记录，这可以提高报表性能。

此过程假设目标表包含所有必需的值：一个业务键（来自源数据的客户 ID）和一个将用于数据仓库的代理键。除了这些“核心”字段外，以下字段也是跟踪维度随时间演变所必需的：

*   `ValidFrom`：记录此维度记录开始有效的日期。
*   `ValidTo`：记录此维度记录曾经或当前有效的截止日期。
*   `IsCurrent`：指示这是该维度元素的当前记录（最新数据）。

前面的 SQL 代码片段基于业务键将两个表进行映射，并在目标表中不存在业务键所引用的记录时（`WHEN NOT MATCHED`），插入一条新记录，同时添加一个自增的代理键，并将今天日期设置为 `ValidFrom` 日期。

如果检测到源表和目标表之间存在任何差异（通过 `WHEN MATCHED AND`... 子句），则会触发几个事件。首先，更新一个自增的代理键，并将今天日期设置为 `ValidFrom` 日期。结果是该记录被标记为当前记录。然后，将昨天日期设置为该业务键的上一条有效记录的 `ValidTo` 日期。

所有这些可能看起来很复杂，但通过使用 `MERGE` 语句的 `OUTPUT` 子句，它变得极其简单。代码所做的是像之前一样对新记录和现有记录执行 `INSERT` 和 `UPDATE` 操作，然后（通过 `OUTPUT` 子句传递）将 `UPDATED` 的记录作为单独的 `INSERT` 语句选择出来。这样，源数据的最新修改成为一条新记录——作为单独的 `INSERT` 处理——并且任何所需的跟踪数据，如 `ValidFrom` 日期和 `IsCurrent` 标志，都在此阶段添加。

#### 提示、技巧与陷阱

*   `ValidFrom` 和 `ValidTo` 日期作为 `INT` 数据类型添加，而不是 `DATE` 或 `DATETIME`，这纯粹是考虑到未来加载到 Analysis Services 的便利性。在那里，数据类型的选择不仅能使日期更易于操作，还可以提高处理时间。如果需要，可以随意使用任何日期数据类型。
*   为了简单起见，我假设维度日期的有效性是 24 小时周期。如果你的需求更复杂，那么维度记录的有效范围可以是任何有效的日期范围。
*   你可能不需要检查所有列的数据差异。如果某些列包含的数据不被视为基本属性，那么就不要在 `WHEN MATCHED ... AND` 子句中使用它。
