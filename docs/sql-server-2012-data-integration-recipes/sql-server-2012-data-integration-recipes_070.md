# 最终数据流

## 工作原理

如有必要，你可以使用 SSIS Web 服务任务将数据插入 SSIS 包。这需要满足两个条件：

*   访问 Web 服务提供商。
*   理解 Web 服务所需的描述文件。

这两者都可以从许多免费的 Web 服务提供商处获得。我选择使用 `www.webservicex.net`。当然，你也可以使用任何可用的 Web 服务。

![image](img/sq.jpg) **注意** 由于本示例使用的是免费的第三方 Web 服务，无法保证该 Web 服务在未来仍然可用或其方法不会更改。

首先，我们定义了要调用的 Web 服务，并使用 HTTP 连接管理器与其建立连接。然后，我们将描述该 Web 服务的 Web 服务描述语言文件下载到一个现有的空文件中。有了这些元素后，我们便能够配置 Web 服务任务来调用我们所用 Web 服务的 `ConversionRate()` 方法。当然，在你自己的环境中，你可能会使用不同的 Web 服务，因此会看到不同的可用 Web 服务和方法。接着，我们将输出——即从 Web 服务返回的 XML——设置到一个变量。（也可以将其写入磁盘上的文件。）我们将目标货币的选择定义为一个变量，以使任务更易于重用。这个变量以及目标货币都是该特定 Web 服务所要求的元素。

配置好 Web 服务任务后，我们就可以使用其输出。每个 Web 服务返回的 XML 各不相同，因此在实际环境中处理输出的具体方式也会有所不同。在本示例中，我们使用了一个脚本任务来接收从 Web 服务调用返回的“原始” XML 片段，并利用 `.NET System.XML` 类的功能来读取 XML 中的值。然后，该值从脚本任务传递到一个 SSIS 变量 (`ConversionRate`)，该变量用于将从 `CarSales.Stock` 加载的源数据转换为 `CarSales_Staging.StockInDollars`。这是通过一个派生列转换来完成的，该转换不仅添加了新的美元成本，还添加了汇率。当然，你可以使用 Web 服务的输出以任何适合你需求的方式进行调用。

#### 提示、技巧与陷阱

*   如果需要，你可以将输出设置到变量。
*   在派生列转换中使用变量和列时，你可以从对话框的左上窗格将变量和列拖放到“表达式”区域。
