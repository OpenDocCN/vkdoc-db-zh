# 1-7. 作为结构化 ETL 流程的一部分加载 Excel 数据

## 问题

您需要从 Excel 工作簿执行工业级的数据加载。这将作为受控 ETL 流程的一部分定期执行。

## 解决方案

创建一个 SSIS 包来加载 Excel 源数据。这将开启 Microsoft 主要 ETL 工具的全部功能和灵活性，以管理您的数据加载。

如果您是一位经验丰富的 SSIS 专业人士，您可能已经习惯了本配方中描述的过程，甚至可能拥有自己执行包创建过程的方法。如果是这样，请随意跳过，因为本配方的目标读者是那些可能对 SSIS 不太熟悉的用户。

1.  创建一个新的 SSIS 包。
2.  在`控制流`区域添加一个`数据流`任务。
3.  双击`数据流`任务跳转到`数据流`窗格。
4.  在`数据流`区域添加一个`Excel 源`。
5.  双击`Excel 源`任务打开`Excel 源编辑器`。点击`新建`打开`Excel 连接管理器`对话框。
6.  点击`浏览`并选择您的 Excel 源文件。
7.  选择与 Excel 工作簿版本相对应的 Excel 版本（Excel 97-2003 用`.xls`，Excel 2007/2010 用`.xlsx`）。您应该会看到类似图 1-12 的内容。
8.  点击`确定`。您将返回到`Excel 源编辑器`对话框。
9.  从“Excel 工作表名称：”下拉框中选择包含您希望导入数据的 Excel 工作表或区域，如图 1-13 所示。
10. 点击`确定`返回到`数据流`窗格。
11. 在`数据流`窗格中添加一个`OLEDB 目标`任务，最好放在`Excel 源`任务下方。
12. 将绿色的连接线（或称为`优先级约束`）从`Excel 源`任务拖到`OLEDB 目标`任务上。
13. 双击`OLEDB 目标`任务打开`OLEDB 目标编辑器`。点击`新建`创建一个新的`OLEDB 连接`，然后再次点击`新建`以指定连接管理器。
14. 选择或输入服务器名称，然后选择或键入数据库名称。本例中我使用的是`CarSales_Staging`。您应该会看到一个如图 1-14 所示的对话框。
15. 点击两次`确定`返回到`OLEDB 目标编辑器`。然后选择一个现有目标表的名称，或者点击`新建`创建一个新表。您可以更改表名（如果您希望这样做）。如果您创建了新表，点击`确定`完成此步骤。您应该会看到类似图 1-15 的内容。
16. 点击`映射`来创建输入到输出的映射。将列从左拖到右进行映射。点击`删除`可移除映射。
17. 点击`确定`完成配置`OLEDB 目标`。
18. 按`F5`或点击`标准`工具栏上的绿色`开始调试`三角形来运行该包。或者，从菜单中选择`调试` → `开始调试`。

## 工作原理

对于定期导入或当您需要 ETL 工具的全部功能来调度和处理数据到 SQL Server 时，SSIS 是应使用的工具。本配方使用 SSIS 的`Excel 源`连接到 Excel 工作簿。然后，使用`OLEDB 目标`组件将所选工作表中的数据加载到 SQL Server 目标表中。

#### 提示、技巧与陷阱

*   如果您愿意，可以在添加`OLEDB 目标`之前创建`OLEDB 目标`连接管理器。然后您只需在`OLEDB 目标编辑器`对话框中的可用连接管理器列表中选择它即可。在 SSIS 2012 中，这可以是一个包级别的连接管理器。
*   如果您的目标表已存在，您可以从`表或视图的名称`中出现的列表中选择它。
*   如果您愿意，可以在添加`数据流`任务之前创建`Excel 连接管理器`。您甚至可以创建包级别的连接管理器（在 SSIS 2012 中）。然而，根据我的经验，这对于本质上与电子表格源一起使用的“单次使用”连接管理器来说很少有用。
*   如果 Excel 工作表经过筛选，那么 SSIS 将只导入筛选后的数据，而不是整个区域或工作表。

---

链接服务器假定数据集包含标题行。但是，Excel 链接服务器（与`OPENDATASOURCE`不同）接受不带标题行的命名区域。如果您的数据集是这种情况，那么请确保在`sp_addlinkedserver`命令的`@PROVSTR`参数中添加`HDR = NO`。这将列名设置为`F1`、`F2`等等。因此，如果您的某些源区域有标题行而另一些没有，您将需要设置两个链接服务器，一个带`HDR = NO`，另一个带`HDR = YES`。此外，您需要注意，指向 Excel 电子表格的链接服务器速度极慢，如果您在 ETL 流程中重用数据，那么将其加载到暂存表中整体上可能会快得多。

查询数据使用标准的 T-SQL `SELECT` 查询，您可以使用指定的列名（如果没有标题行，则使用`F1`、`F2`等）、`WHERE`子句、`ORDER BY`等来限制选择。这意味着您也可以使用`CAST`和`CONVERT`来更改数据类型，以及所有常用的文本函数（首先想到的是`LTRIM`、`RTRIM`和`LEFT`）对文本字段进行基本的数据操作。由于我在配方 1-4 和 1-5 中提供了大量这方面的例子，我请您回顾那些配方以获取更多细节。

#### 提示、技巧与陷阱

*   请务必将提供程序设置为`ACE`或`Jet`连接字符串。您还必须将`@PROVSTR`参数设置为`Excel 8.0`（用于`Jet`）或`Excel 12.0`（用于`ACE`）。
*   `@SRVPRODUCT`参数纯粹是装饰性的。
*   定义链接服务器时，Excel 文件无需存在。
*   您可以通过在 SSMS 中展开`服务器对象/链接服务器`来查看链接服务器。在 SSMS 中双击链接服务器名称可查看您使用`sp_addlinkedserver`命令设置的属性。
*   您也可以使用 SSMS 定义链接服务器。这在配方 1-13 中有描述（针对 Access）。然而，原理几乎完全相同。
*   Excel 源文件不能受密码保护。
*   请注意，在四部分名称表示法中，您不需要架构或数据库引用。只需键入三个句点。
*   如果 Excel 工作簿包含多个数据集（作为单独的工作表或命名区域），那么实际上，您只需配置一次连接（通过设置链接服务器）。然后，您只需更改`SELECT`查询中四部分名称表示法的最后一个部分（本例中是`Stock$`）所对应的工作表/区域名称，即可查询各种源数据集。
