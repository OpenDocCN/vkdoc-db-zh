# 脚本任务

脚本任务利用 .NET 中 `System.IO` 类的功能来分离文件名、扩展名和路径，然后将它们传递给存储过程。这些信息也会存储在目标表中。诚然，这种路径解析也可以用 T-SQL 完成——但使用脚本组件和现成的 `System.IO` 类要容易得多。存储过程 `pr_LoadBLOBsFromSSIS` 然后根据 Foreach 循环容器中的迭代次数执行实际的加载操作。这种方法可以非常轻松地扩展以处理多个源目录或文件类型——但这部分就留给你自己完成了。

## 提示、技巧和陷阱
*   在向 SSIS 包添加任何其他对象之前创建变量，可以避免潜在的作用域错误。当然，你也可以随时创建变量。
*   记得在脚本中使用 `Imports System.IO`，否则你将无法访问那些极其有用的内置 .NET 函数来解析文件和路径数据。
*   别忘了 SSIS 变量是区分大小写的！
