# 第四章

图 3-11. SSIS 中的 XML 验证

## 工作原理

这个相当简单的 SSIS 脚本任务根据模式文件验证 XML 文档，并且（在本例中）根据验证结果引导过程流。也就是说，如果验证成功，数据将加载；如果验证失败，流程将停止（并发送电子邮件）。此外，如果验证失败，将创建一个记录所有遇到错误的日志文件。其工作方式如下：

-   首先，设置对 XML 和 `System.IO` 库的引用。
-   然后，将要处理的文件（XML 和 XSD）分配给脚本任务中的变量。
-   读取源文件并进行验证。任何错误都由 `ValidationEventHandler` 检测到，每个错误都写入一个 `StringWriter`。错误标志被设置。
-   最后，错误标志被返回给 SSIS，错误列表被写入一个文件（如果该文件已存在，则先删除）。

#### 提示、技巧与陷阱

-   显然，如果 XML 文件和模式文件严重不同步，那么日志文件不仅会非常大，而且会极其重复。
-   当然，你不需要使用发送邮件任务——实际上，你可以什么都不做来表示验证 XML 源数据失败。

#### 总结

正如本章试图展示的，有多种方法可用于获取 XML 源文件并将其加载到 SQL Server 中。在某些情况下，选择将取决于你的目标——如果你想“原样”加载文件而不将数据分解为其组成部分，那么显然 `OPENROWSET (BULK)` 可能是最佳解决方案。然而，如果源文件被用作数据传输的媒介，那么你有更广泛的选择。如果你的 ETL 流程基于 T-SQL，你可能会发现使用 SQL Server 的 XQuery 支持是可行之道。另一方面，如果你更“以 SSIS 为中心”，那么在许多情况下，SSIS XML 任务可以是一个优秀的解决方案。对于非常大的源文件——或者速度至关重要的情况——那么 SQLXML 批量加载器可能是唯一可行的选择。
