# 工作原理

在许多情况下，你可能需要从一个目录中的文件加载数据。这些文件可能是平面文件数据（例如逗号分隔文件）、XML 文件甚至是图像文件。

现在，加载目录中的所有文件是一种标准技术，之前在食谱 12-1 中已经解释过。然而，正如许多评论者所指出的，标准的 Foreach Loop 容器除了指定文件名和扩展名模式外，不允许你做任何其他事情。有时你可能需要指定加载顺序，或者根据文件系统暴露的属性（如创建日期或文件大小等）来筛选目录中的文件。

尽管有几种方法可以对源文件（无论它们包含何种类型的数据）进行排序和筛选，但我更喜欢使用 SSIS 和脚本任务来实现。这种方法依赖于 ADO.NET 数据集来执行排序和筛选。这种方法既可靠又易于维护。它将需要创建和使用两个 .NET 数据集。一个数据集保存你用来存储待加载文件的目录中文件的名称和属性。然后创建此 .NET 数据集的一个克隆——这个克隆将被筛选和排序。此克隆数据集用于迭代要加载的文件。

为了使此过程更加灵活，我建议将以下项作为变量传入：

*   要处理的目录的 `name`。
*   文件 `filter`。
*   `sort` 列。

传入这些项目当然不是绝对必要的，但它使程序包更易于重用。你还需要一个字符串变量来保存正在加载的实际文件的文件名，以及一个对象变量来保存排序后的数据表。

以下是脚本代码的工作原理：为了使脚本更具动态性，声明了两个变量，它们的内容从 SSIS 变量传入。这些（`FileSource` 和 `FileFilter`）将传入要解析的目录和要应用的文件筛选器。声明变量以保存文件的所有必需属性。当然，如果你仅对一个或两个属性进行排序和/或筛选，那么你只需要使用相应的变量即可使代码更易于维护。最初创建一个 `Main` 数据集。该数据集又包含 `Main` 表，该表将填充来自所选目录的文件列表。定义了 `Main` 表，包含你可能使用的每个文件属性的列。
