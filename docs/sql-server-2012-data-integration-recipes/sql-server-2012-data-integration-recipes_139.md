# 13-2. 选择多个文本文件进行导入

## 问题

你希望在一次操作中选择一组特定的、结构相同的平面文件导入到 SQL Server 中。

## 解决方案

使用 `MULTIFLATFILE` 连接管理器从一个或多个不同文件夹中选择一组平面文件，并在一次操作中加载它们。以下步骤说明了如何操作。

1.  创建一个新的 SSIS 包并添加一个数据流任务。双击以激活数据流窗格。
2.  创建一个名为 `CarSales_OLEDB` 的 OLEDB 连接管理器，连接到 CarSales 数据库。
3.  在连接管理器选项卡中右键单击，选择“新建连接”。从可能的连接列表中，选择 `MULTIFLATFILE` 并单击“添加”（或双击 `MULTIFLATFILE`）。将出现“多平面文件连接管理器编辑器”。正如你将看到的，它与“平面文件连接管理器编辑器”有着惊人的相似之处。
4.  为连接管理器命名（我将其称为 `MultiFileSelection`），然后单击“浏览”。在标准的 Windows “打开”对话框中，你可以按住 Ctrl 单击（或使用 Shift 单击和 Ctrl 单击的组合）来选择多个文件。在此示例中，你需要导航到目录 `C:\SQL2012DIRecipes\CH13\MoreMultipleFlatFiles`。
5.  单击“打开”确认你的选择。按照方案 13-1 步骤 5 中的描述，完成平面文件定义的其他选项。
6.  添加一个平面文件源。使用你在步骤 2 中创建的 MultiFile 平面文件连接管理器（名为 `MultiFileSelection`）作为其连接管理器。
7.  添加一个目标任务。将源任务连接到目标。通过配置任务使用 `CarSales_OLEDB` 连接管理器连接到 `dbo.Stock` 目标表来完成包的设置。
8.  映射源列和目标列。

你现在可以运行该包，并将步骤 3 中选择的文件加载到目标表中。

## 工作原理

导入多个文件（文本或其他类型）的方法有很多，方案 12-1 描述了使用 Foreach 循环容器的“经典”方法。Foreach 循环方法在以下情况下会失效：你想加载源目录中的部分而非所有文件，并且无法使用通配符进行过滤。然而，还有另一种既简单又高效的方法可用于文本文件，它允许你指定要加载的***确切文件***。它涉及使用 `MULTIFLATFILE` 连接管理器，并且可以配置为加载选定的子集文件，或者基于一个或多个通配符定义加载文件的子集（前提是这些定义不包括你明确不希望加载的文件）。此技术假定所有源文件具有相同的格式。

> **注意**   你只能从多平面文件连接管理器浏览到一个源文件夹。要扩展选择范围，你必须在“多平面文件连接管理器编辑器”对话框的“文件名”字段中手动输入——或复制粘贴——多个文件夹。每个文件或路径必须用管道符 (`|`) 分隔。

## 提示、技巧与注意事项

-   要使用通配符进行文件选择，你需要如前所述设置连接管理器，然后选中它并按 F4（或查看 -> 属性窗口）以显示连接管理器的属性。然后，你可以编辑 `ConnectionString` 属性，在文件名中使用通配符。
-   使用 `ConnectionString` 属性，你还可以扩展文件源信息以处理多个目录——无论是否使用通配符，方法是键入或粘贴所需的路径和文件名。只需用管道符 (`|`) 分隔每个源路径/通配符元素。
-   如果你创建了一个指向多个目录的复杂路径，请避免单击“浏览”按钮（或省略号 `...`），否则你将丢失该路径，因为它会被新路径替换（而不是添加到其中）。
-   如果你用另一组文件替换现有的待加载文件组，SSIS 会要求你确认是否保留描述平面文件结构的现有元数据。假设文件格式相同，如果出现此对话框，你可以确认。
