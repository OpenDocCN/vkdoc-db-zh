## 元数据

元数据是关于数据的数据。具体来说，在数据摄取过程的上下文中，它意味着发现并了解有关源数据结构的所有必要知识。这意味着至少需要拥有以下所有详细信息：

*   使用的表
*   可用的列
*   数据类型
*   列中数据的长度
*   列的可空性

对于某些源，您还应了解：

*   主键和唯一键
*   索引
*   触发器

究竟有多少信息对您正在构建的过程至关重要，将取决于每个单独的项目。其中一些可能仅仅是有用的，但其他方面可能是成功数据加载过程的关键。

现在，虽然可以在对源元数据了解不多的情况下创建和运行高度成功的 ETL 过程，但拥有对源数据如何结构化和定义的清晰定义可能非常有用，原因包括：

*   当您定义数据流时，如果能够从一开始就指定正确的数据类型和长度，将使您的流程更加稳健，并避免代码投入生产时出现意外的包失败。
*   当您将源数据类型映射到目标数据类型时，可以对其兼容性有更大的信心。

我相信您还能找到其他理由来补充这个简短的列表。因此，在假设获取和理解支撑数据流的元数据是创建和维护稳健高效的数据摄取过程的关键之后，本章将向您展示如何：

*   获取元数据。
*   使用元数据调试数据摄取问题。
*   存储元数据并使用它来检测源数据结构的变化，例如结构修改（新增或删除列）和类型更改。
