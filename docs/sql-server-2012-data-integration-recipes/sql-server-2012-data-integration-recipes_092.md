# 提示、技巧和陷阱

*   如果您愿意，可以只为包使用一个连接管理器（OLEDB 或 ADO.NET）。我发现将变量传递给存储过程时，使用 ADO.NET 连接管理器更容易调试和维护，而对于纯粹的数据传输，OLEDB 似乎稍快一些。因此，*我的*习惯是使用两个连接管理器，其中一个纯粹是为了方便编写和调试需要为其包含的 T-SQL 传递输入和输出变量的执行 SQL 任务。
*   与其在步骤 8 中为平面文件目标任务勾选“覆盖文件中的数据”，您也可以选择添加一个初始文件系统任务来删除文件（如果您愿意的话）。
*   可以在步骤 11 设置 `SQLProc` SSIS 变量而不使用脚本任务——但这又是一个关于清晰度和可维护性的问题；我更喜欢添加一个任务组件来完成此操作。
*   需要进行一些试错来确定能实现高效处理的最佳源记录数。
*   有非常、非常多的方法可以对数据集进行分块。这是本书其他地方描述过的方法之外的又一种。当然，您可以自由选择最适合您特定数据结构的方法。
