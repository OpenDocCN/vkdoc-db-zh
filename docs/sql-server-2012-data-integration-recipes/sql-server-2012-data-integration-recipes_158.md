# 讨论日志记录的位置与方法

这些指标以及错误检测指标可用于控制流程并增强数据集成与决策步骤的结合。但在深入技术方案之前，我建议稍微退后一步，从更高的视角审视日志记录过程。到目前为止，本章已经定义了我们记录数据的“原因”。现在，我想从全局上看看“在哪里”以及“如何”记录。

假设你已经确信记录日志是个好主意，那么下一个问题是——我该在哪里记录所有这些信息？主要有三种可能性（尽管你可能知道或偏好其他方式）：

*   将信息记录到 `SQL Server` 表中。
*   将信息记录到文件（文本或 `XML`）中。
*   将信息记录到系统日志中。

每种方式都有其优缺点，详见 表 15-1、15-2 和 15-3。

## 表 15-1. 记录到 SQL Server 表

| 优点 | 缺点 |
| --- | --- |
| 易于添加到存储过程和 `SSIS` 包中。 | 对数据类型不匹配和长度溢出毫不宽容。 |
| 易于查询，可用于比较和建立基线。 | 可能与数据库资源竞争。 |

## 表 15-2. 记录到文件

| 优点 | 缺点 |
| --- | --- |
| 易于添加到 `SSIS`。 | 可能与服务器资源竞争。 |
| 相对容易添加到 `T-SQL`。 | 更难阅读、查询和划分子集。 |
| 在处理过程中处理繁重的日志记录，而不会给数据库增加额外负担。 | 可能过度增长且难以有选择地修剪。 |

## 表 15-3. 记录到系统日志

| 优点 | 缺点 |
| --- | --- |
| 易于添加到 `SSIS`。 | 更难从 `T-SQL` 查询。 |
| 易于使用 `Windows` 工具查看。 | 可能给系统日志增加过多杂乱信息。 |
| 使用 `LogParser` 非常容易查询。 |  |

冒一点过于简化的风险，我甚至不建议将指标（记录计数）和审计数据（计数器、计时器和单元测试）存储在数据库表之外的地方。我的理由如下：

*   存储的数据是数据表的理想选择，因为它类型明确（`INT/BIGINT, DATETIME, BOOLEAN`）。
*   几乎没有冗长的文本数据，这与错误日志记录的情况不同。因此数据表很窄，系统开销最小。
*   审计数据在很大程度上是 `T-SQL` 过程，因此将指标存储在数据库中使分析指标变得异常容易。
*   这些指标的存在是为了建立基线和进行比较——因此将它们放在数据表中，可以直接在此基础上构建监控系统。

最终，决定使用哪种类型的日志记录，甚至使用哪些日志类型的组合，将取决于你的系统需求。你还需要考虑这些日志将放置在哪里。这里的选择同样简单：

*   将日志放在 `SQL Server` 和/或 `SSIS` 服务器上。
*   将日志放在单独的数据库服务器或文件服务器上。

同样，最终的决定可能不像乍看起来那么明显。表 15-4 和 15-5 概述了每种方式的利弊。

## 表 15-4. 将日志放在 SQL Server 上

| 优点 | 缺点 |
| --- | --- |
| 无网络延迟。 | 系统资源竞争。 |
| 本地可访问性。 | 可能“挤占”磁盘空间。 |

## 表 15-5. 将日志放在分布式服务器上

| 优点 | 缺点 |
| --- | --- |
| 本地系统资源竞争少。 | 网络延迟。 |
|  | 存在网络问题的风险。 |

你可能会发现日志目标和日志系统的混合方式最适合你的项目。我只想提高你对这些可能性的认识。

我唯一要提出的建议——也是我在本章示例中将遵循的——是在使用 `SQL Server` 表作为日志目标时，使用一个单独的 **日志记录与审计** 数据库。原因相当明显：

*   你不会用不相关的数据弄乱暂存区或 `ODS`（操作数据存储）数据库。
*   你可以将日志数据库的数据库文件放在单独的磁盘上以避免争用。
*   你正在为一个可扩展的解决方案奠定基础，以防日志最终移动到单独的服务器——或者如果你的企业集中所有 `ETL` 和处理日志及计数器以便于报告。

此外，为了使这个数据库更清晰，我将对日志和审计过程的不同方面使用 `Log` 和 `Audit` 架构。

无论日志类型如何，你总是需要在“通过记录所有发生的事情而减慢进程”与“无法快速找到错误原因”之间找到正确的平衡。日志记录与其说是一门科学，不如说是一门艺术，每个系统都有不同的要求。计数器太多，你会只见树木不见森林；计数器太少，你看不到即将发生的事故。知道你是否正确的唯一真正方法是你的进程运行平稳，并且你可以轻松快速地跟踪错误或计数器。

因此，鉴于没有唯一的解决方案——因为没有单一的需求——本章将展示如何在三个进程级别记录日志：

*   使用 `T-SQL` 或内置 `SSIS` 日志记录到表和文件的简单日志记录。
*   从 `T-SQL` 和 `SSIS` 简单记录到集中式日志数据库。
*   从 `T-SQL` 或 `SSIS` 复杂地记录事件和计数器到集中式自定义日志数据库。

本章将探讨所有这些技术，为你提供一些想法，如何在严酷的现实世界数据集成环境中增强你的进程，使其更加健壮和有弹性。
