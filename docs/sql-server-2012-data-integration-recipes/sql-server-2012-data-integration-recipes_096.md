# 使用 SSIS 将数据导出到 Access

## 问题
你需要作为 SSIS 包的一部分，将数据导出到 Access。

## 解决方案
在 SSIS 数据流中使用 OLEDB 目标任务，步骤如下：

1.  创建一个新的 SSIS 包并添加一个 OLEDB 连接管理器。将其命名为 `CarSales_OLEDB`，并配置它连接到你的源服务器。
2.  添加第二个 OLEDB 连接管理器。单击“新建”，从对话框顶部的提供程序弹出列表中选择 **Microsoft Jet 4.0 OLEDB Provider**。浏览或输入现有 Access 数据库的路径。点击“确定”两次。将第二个连接管理器命名为 `Access_OLEDB`。
3.  添加一个数据流任务并双击进行编辑。
4.  添加一个新的 OLEDB 源组件，并将其配置为使用你添加的 `CarSales_OLEDB` 连接管理器。选择一个源表或输入 SQL 查询来定义源列。
5.  添加一个新的 OLEDB 目标组件。选择 `Access_OLEDB` 连接管理器，并选择一个现有表作为目标，或单击“新建”创建一个新表。
6.  单击“映射”，将源列映射到目标列。
7.  单击“确定”。

现在你可以运行该包了。

## 工作原理
你可能更喜欢——或者需要——使用 ACE 提供程序导出到 Access，尤其是在 64 位环境中导出时。在这种情况下，创建目标 OLEDB 提供程序（步骤 2）时，请从弹出列表中选择 **Microsoft Office 12.0 Access Database Engine OLEDB Provider**。然后，你将需要手动键入或粘贴 Access 数据库的完整路径作为服务器或文件名。如果你使用 ACE 提供程序，在创建新表时可能会收到关于数据类型问题的警告消息。在大多数情况下，可以忽略此消息。它的意思是，将使用更通用的 OLEDB 数据类型来创建表，而不是特定的 Access 限制数据类型。当然，在创建任何新的目标表时，你可以将表名从 `OLEDB Destination` 重命名为你选择的名称。

## 提示、技巧和陷阱
*   与导出到 Office 应用程序一样，你处于 32 位还是 64 位环境的问题可能会产生广泛的影响。这与**配方 1-1** 中的解释相同。
*   如果你的 Access 数据库受密码保护，你将需要在 OLEDB 连接管理器配置中输入连接详细信息。
*   如果你的源数据包含任何非 Unicode（即 `VARCHAR`）列，那么目标任务会指示存在问题。有两种解决方案：
    *   使用 SQL 语句选择源数据，并使用 `CAST` 或 `CONVERT` 将源数据类型更改为 `NVARCHAR`。
    *   在源和目标之间添加一个数据转换任务，并将所有非 Unicode 列转换为 Unicode 字符串 [DT_WSTR]。
