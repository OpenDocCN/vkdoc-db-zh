# 工作原理

我意识到这是一个很长的脚本，但观察如何利用索引元数据在目标数据库中重新创建索引是很有趣的。

在重新创建索引时，请记住：

*   如果要删除所有索引，请先删除非聚集索引，然后是聚集索引。
*   如果要同时创建聚集索引和非聚集索引，请记住先创建聚集索引。

此脚本使用 `MetaData_Indexes` 表中的数据来重新创建所有所需的脚本，以便像删除前一样重新创建索引。这不是一个简单的过程，因为必须管理复杂的键列和覆盖索引，因此需要明智地使用临时表。

脚本按以下顺序组装创建以下对象的脚本：

*   创建主键。
*   创建聚集索引。
*   创建唯一索引。
*   创建非聚集索引。

所有必需的脚本都存储在临时表 `#ScriptElements` 中。然后，通过一个游标遍历此表并执行这些语句——从而生成索引。
