# 工作原理

尽管这个解决方案更多是为了解决消除数据加载中任何手动干预的问题（而非纯粹的加速技术），但可能会遇到这样的情况：源目录中存在数十甚至数百个结构相同、需要加载到 SQL Server 表中的文件。幸运的是，SSIS 可以轻松处理这种情况，这里你可以看到具体操作方法。我意识到在关于 XML 数据源的章节中已经介绍过这个内容，但它有足够多的不同之处（并且非常常用），因此我更倾向于通过一个专门针对文本文件的方案来详细介绍。

该过程本质上是对目录中所有或选定的文件进行循环。你需要一个字符串变量来保存每个被加载文件的名称，并在进程运行时用于替换创建 SSIS 任务时定义的实际文件。在其他所有方面，这个过程与单个文件加载几乎完全相同。此技术假定所有要加载的文件格式相同，并且将加载到同一个表中。本方案不会深入探讨使用平面文件连接管理器和数据流任务将文本文件映射到 SQL Server 表的复杂细节。该信息在方案 2-2 中已涵盖。

## 提示、技巧与注意事项

-   文件和/或扩展名过滤器的用法与你在搜索操作或传统的 DOS/命令窗口中使用的完全相同。实际上，你可以在命令窗口中测试打算使用的过滤器（还记得 `DIR *.CSV` 吗？）。
-   在步骤 11 中用于保存文件名的变量始终是变量 0，因此无需更改此设置。
-   在步骤 18 中，你可以评估表达式以测试其是否有效——但如果没有任何显示，请不要惊讶。只有在变量窗格中为 `FileName` 变量输入值后，变量的内容才会显示。
-   加载目录中的所有文件时，仅限使用一种搜索模式（例如，`*.Csv`）。如果需要使用多种搜索模式——请继续阅读，因为这在方案 13-2 中有解释。
