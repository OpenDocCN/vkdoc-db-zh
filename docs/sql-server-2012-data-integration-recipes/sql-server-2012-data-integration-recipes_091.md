# 工作原理

即使 SSIS 没有 XML 目标组件，您也可以（相当）轻松地从 SQL Server 输出 XML。

再次强调，唯一的问题将是输出的大小。如果它太大而无法放入 SSIS 缓冲区，那么该过程将失败。因此，真正的技巧在于为大型数据集找到一种高效的分块机制来处理批量 XML 导出。

不幸的是，并非所有 XML 导出文件都小到足以放入 SSIS 缓冲区，因此同样有必要定义一个子集化过程，将核心数据输出为足够小的分块以便 SSIS 处理，然后将 `XML ROOT` 元素添加到输出文件的顶部和底部。

此处描述的 SSIS 包将源数据分解为“分块”，然后将输出导出为单个 XML 文件。更准确地说，步骤 4 中描述的第一个“获取分块迭代次数”执行 SQL 任务，根据您在 `@ChunkRecordNo` 变量中指定的分块大小，计算需要处理的分块数量，并相应地设置 SSIS 变量。第二个执行 SQL 任务，“准备分块分段的临时表”，使用要处理的所有记录的 ID 填充一个全局临时表。步骤 7 中描述的此任务还为每个 `recordID` 分配一个 `ChunkID`，以便每条记录作为指定分块的一部分输出。T-SQL `NTILE(?)` 中的问号将在执行时被 `@Chunks` 参数（该参数包含根据您指定的记录大小计算出的分块数）替换。“遍历分块计数器”Foreach 循环任务确保 XML 导出过程处理先前定义的每个分块。脚本任务确保为分块中的每组记录运行的 SQL 语句传递了适当的输入参数。最后，XML `<ROOT> ...</ROOT>` 元素被添加到核心过程的数据输出中。

这个过程相当复杂，并且包含几个依赖项。可以在使用它的全局临时表（`##tmp_ChunkDef`）创建之前创建 `pr_GetChunkedXML` 存储过程，但您不能在创建此表的包上下文之外运行该存储过程。
