# Access 升级向导概述

**Access 升级向导**是一个历史悠久、可靠且简单的工具，适用于将中小型 Access 数据库迁移到 SQL Server 的快速开发场景。本文仅讨论使用此工具将数据迁入 SQL Server，不涉及应用转换。

## 使用场景与限制

该工具在以下情况可能适用：

*   源数据结构简单，无复杂数据类型。
*   源数据量不大。
*   需要快速将大部分或全部 Access 数据库（包括结构和数据）迁移到 SQL Server。

## 成功关键与常见问题

迁移失败的主要原因往往不是技术问题，而是源 Access 数据库本身设计不佳（如不符合第三范式、命名混乱、数据冗余）。真正的机会在于借此契机重新设计数据库和清理数据。

为成功迁移，请遵循以下关键步骤：

1.  **务必操作源数据库的副本**。
2.  **修改副本中所有表和字段名**以符合 SQL 命名规范（移除特殊字符，并采用 SQL Server 命名约定）。
3.  **不要迁移索引、验证规则、默认值和引用完整性**——应在 SQL Server 中重新创建这些对象，以便使用自己的命名约定定义约束名。这些方面最常导致迁移失败。

#### 提示、技巧与陷阱

*   迁移过程中可以新建数据库，但为了更好地控制文件位置和属性，建议先手动创建目标数据库。
*   若需迁移视图，可先在 Access 中运行“创建表”查询基于视图生成表，再迁移该表。
*   可以使用该向导创建表结构，然后在 SSIS 中调整完善表结构后再迁移数据。此方法也允许将表移至 `dbo` 之外的架构。
*   Autoincrement 字段在 SQL Server 中不会迁移为 `IDENTITY` 字段，而是作为普通的 `INT` 字段，因此需要手动修改表结构将其指定为标识列。
*   迁移 OLE 对象时，OLE 图像数据仍保持为 OLE 对象格式（而非二进制图像数据）！
*   超链接字段将作为文本字段迁移。
*   对于 SQL Server 2008 及以上版本，请确保使用 `DATETIME2` 字段以避免日期溢出错误。
*   迁移到 SQL Server 2005 时常见溢出错误。此时应在 Access 中查询源数据，确保日期字段值在 SQL Server 日期范围（1753-01-01 至 9999-12-31）内。一个初步的解决方法是使用 Access 查询将超出上限（9999-12-31）和下限（1753-01-01）的日期设置为极限值后再进行转换。
*   导入大型数据集时可能超时。可通过注册表编辑器将 `HKEY_LOCAL_MACHINE\Software\Microsoft\Jet\4.0\Engines\ODBC\QueryTimeout`（适用于 Access 97 至 2003）和 `HKEY_LOCAL_MACHINE\Software\Microsoft\Office\12.0\Access Connectivity Engine\Engines\ODBC\QueryTimeout`（适用于 Access 2007–2010）的值设置为 0 来解决。
*   如果拥有早期版本的 Access 数据库，请先在 Access 97 或更高版本中打开并保存。转换信息可参考 Office Online 网站。

## 1-11. 导入多个 Access 表格

### 问题
你希望快速轻松地从一个 Access 数据库导入一个或多个表格，且这通常是一次性操作。

### 解决方案
使用 SQL Server 导入向导，步骤如下：

```
1. 在 SQL Server Management Studio 中，右键单击目标数据库（最好是数据要导入的数据库）。选择 任务 > 导入数据。或者，从 开始 > 所有程序 > Microsoft SQL Server 2012 > 导入和导出数据（32 位或 64 位）启动。将出现“选择数据源”窗口。
2. 选择“Microsoft Access (Microsoft Access Database Engine)”。
3. 浏览到源文件，单击“确定”。
4. 选择目标服务器和数据库，以及身份验证类型（若未使用 Windows 身份验证，则需指定用户名和密码）。单击“下一步”。出现“指定表复制或查询”窗口。
5. 接受默认选项“从一个或多个表或视图复制数据”。单击“下一步”。出现“选择源表和视图”对话框。
6. 选择要导入的表和/或视图。单击“下一步”。出现“保存并执行包”对话框。
7. 单击“下一步”。出现“完成该向导”对话框。
8. 单击“完成”。出现“执行结果”对话框。
```

### 工作原理
对于 Access（如同 Excel），有时需要快速高效地将数据从 Access 数据库导入 SQL Server 表，而无需设置 SSIS 包。数据导入向导是完成此任务的最佳工具之一。需要注意的是，导入数据和元数据可能无法避免大量的重构工作。此方法最适用于偶尔需要从 Access 导入数据，或者目标是初步分析数据的场景。
