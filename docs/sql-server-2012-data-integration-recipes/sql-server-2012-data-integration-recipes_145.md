# 提示、技巧和陷阱

*   记住对服务器和目录名称使用 UNC 路径，以确保可移植性。
*   你不需要通过基于零的索引来引用数据表中的列（就像我在本食谱的代码中所做的那样）。你可以使用列名，例如：
    ```
    MainRow("FileName") = FileName
    ```
*   你选择的变量名不必与列名完全相同——我只是发现这样编码和调试都更简单。
*   当（或者如果）进行排序时——因为这不是强制性的——请记住使用你作为 ADO.NET 表的一部分定义的列名进行排序。
*   将排序列作为变量传入意味着你必须小心处理引号字符串。如果你对排序列进行硬编码，那么只需输入 `FileSize ASC`（按 ado.NET 数据表中的 `FileSize` 列排序）就足够了。
*   筛选条件可以更复杂，可以包括多个列和标准的比较运算符。一个更复杂的例子可能是：
    ```
    "FileSize > 10000 AND LastWriteTime <= '2010-12-25'"
    ```
*   请注意，筛选条件中使用的所有文本字符串必须用单引号括起来，并且有几种可能的日期格式。我更喜欢使用 YYYY-MM-DD。
*   如果你想排序而不筛选，那么你需要运用一些技巧。.NET `Select` 方法要求在排序之前先进行筛选；也就是说，没有一个重载允许你仅输入排序参数。因此，一个很好的作弊方法是使用如下代码：
    ```
    Select
    ```
    这筛选了所有大小为 1 字节或更大的文件（这意味着在大多数情况下所有有用的文件），但它按文件创建日期排序。
*   如果你使用的是 SSIS 2005 或 2008，则需要在 `Imports` 指令中添加对 `System.Xml` 的引用。
