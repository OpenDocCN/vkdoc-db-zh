# 7-7. 将数据导出到另一个 SQL Server 数据库

## 问题
您需要将数据导出到另一个 SQL Server 数据库。您希望最小化开发时间，并尽可能快地传输数据。

## 解决方案
使用 BCP 以 SQL Server 原生格式导出和导入数据。

要导出一个完整的表（其中列结构相同），BCP 命令将类似于：

```BCP
BCP CarSales.dbo.Client OUT C:\SQL2012DIRecipes\CH07\SQL2000.bcp -N –SADAM02 –T
```

## 工作原理
这本质上是“经典”的 BCP，有一个源表（但也可以是一个查询，如配方 7-4 所示）和一个目标文件。最大的区别在于您指定了 `-N` 标志来告诉 BCP 使用原生格式。其结果是您无需确认列长度和数据类型。

原生格式导出适用于目标数据库不是不同的系统，而是另一个 SQL Server 数据库的情况。在这些情况下，您可能会发现使用 BCP/SQL Server 原生格式在 SQL Server 实例之间导出和导入更容易。我意识到这看起来可能是一条复杂的路线，但有时这是唯一实用的解决方案。在众多可能的情况中，一些可能必要的场景包括：

*   当您需要在数据库之间传输一部分表时。
*   当您的数据库使用不同版本的 SQL Server 时（即使是 SQL Server 2008 和 SQL Server 2008 R2 也被认为是不同的版本），因此备份和恢复将不起作用。
*   链接服务器违反了企业安全要求。
*   只有文件传输是允许的或可行的。

当需要映射、排除和添加字段时，您有三种选择：

*   **将整个 `SELECT` 语句添加到 BCP 命令中**，用引号括起来，并使用 `QUERYOUT`。这可能第一次很难做对，所以我总是建议先在 `SSMS` 中完善 `SELECT` 语句，然后将其复制粘贴到命令窗口中。
*   **将 `SELECT` 语句包装在视图中**，并从 BCP 调用该视图。对于将频繁导出的表或非规范化数据时，这值得付出努力。
*   **将 `SELECT` 语句包装在存储过程中**，并使用 `EXECUTE .. QUERYOUT` 从 BCP 调用该存储过程。请记住，会话范围的临时表不能使用。

原生格式的优缺点详见表 7-6。

**表 7-6 BCP 原生格式的优缺点**
| 优点 | 缺点 |
| --- | --- |
| 速度快 | 无法使用格式文件——因此输出文件和目标表中的列结构需要完全相同。 |
| 不需要格式文件 |  |

还有一些 SQL Server 到 SQL Server 的导出选项——如表 7-7 所示——也值得了解。

**表 7-7 SQL Server 到 SQL Server 导出选项**
| 选项 | 说明 |
| --- | --- |
| 原生格式 (`-n`) | 只能用于不包含任何扩展/双字节字符集 (DBCS) 字符的数据。要保留任何扩展字符，您必须指定支持您希望传输的字符的代码页。 |
| Unicode 原生格式 (`-N`) | 防止在使用不同代码页的服务器之间批量传输数据时丢失任何扩展字符。这将应用于所有 `char`、`nchar`、`varchar`、`NVARCHAR`、`text`、`varchar(max)` 和 `NVARCHAR(max)` 字段。 |
| `-V70` | SQL Server 版本 7 |
| `-V80` | SQL Server 2000 |
| `-V90` | SQL Server 2005 |
| `-V100` | SQL Server 2008/2008 R2 |

将表导出到以前版本的 SQL Server 时，唯一真正的小麻烦是可能需要某种形式的字段映射。我建议在输出阶段使用 T-SQL 进行塑形和映射数据，并确保发送到 BCP 文件的数据能自动映射到目标表。这避免了使用格式文件，并确保任何较新的数据类型以及目标服务器中不存在的字段从文件中省略。

简单来说，您需要：
*   通过不将它们添加到 `SELECT` 语句中来删除字段。
*   通过将 `CAST(NULL AS` 数据类型) `AS FieldName` 插入到 `SELECT` 语句中来添加字段。

对于在旧版本目标服务器中以另一种数据类型存在的字段，将您可以从较新版本转换的数据类型进行 `CONVERT`。然而，这并非总是可行，因此您可能必须跳过一些字段。

如果您完成了所有这些操作，并以 SQL Server 原生或宽原生格式（使用 `-n` 或 `-N` 标志）导出，同时使用 `-Vnn` 标志指定 SQL Server 版本，那么您的输出文件应该能流畅地加载到目标服务器中。

要导出一个完整的表（其中没有太新而无法转换的数据类型，且列结构相同），BCP 命令将类似于：

```BCP
BCP CarSales.dbo.Client OUT C:\SQL2012DIRecipes\CH07\SQL2000.bcp -N –SADAM02 –T –V80
```
