# 第 5 章：表与数据库状态

,{(EMPNO;102), (ENAME;'Thomas'), (JOB;'SALESMAN') }

,{(EMPNO;103), (ENAME;'Lucas'), (JOB;'PRESIDENT')}

,{(EMPNO;104), (ENAME;'Pete'), (JOB;'MANAGER') } }

`E1`和`E2`的并集是一个定义在`{EMPNO,ENAME,JOB}`上的表。它包含四个元组，而不是五个，因为员工 102 的元组同时属于这两个集合。

现在来看表`E1`和`E3`的并集（`E1∪E3`）：`E1∪E3` =

{ {(EMPNO;101), (ENAME;'Anne'), (JOB;'TRAINER') }

,{(EMPNO;102), (ENAME;'Thomas'), (JOB;'SALESMAN') }

,{(EMPNO;103), (ENAME;'Lucas'), (JOB;'PRESIDENT') }

,{(E#;101), (NAME;'Anne'), (JOB;'TRAINER'), (SAL;3000)}

,{(E#;102), (NAME;'John'), (JOB;'MANAGER'), (SAL;5000)} }

这个结果是一个函数集合，但它显然不是一个表；并非此结果集中的所有函数都共享相同的定义域。

还记得闭包性质吗（见第 2 章的“并集、交集与差集”一节）？我们只关心两个表的并集结果是另一个表的情况。显然，并集运算符通常并不在表上封闭。它只在参与运算的表具有相同标题时才在表上封闭。如果并集运算符的操作数是定义在不同标题上的非空表，那么得到的结果集将不是一个表。

注意空表作为操作数参与的特殊情况。一个给定的表与空表（`∅`）的并集总是得到给定的表；因为`∅`是定义在任何标题上的表，所以闭包性质成立。

###### 交集

两个集合的交集包含所有同时是第一个集合元素和第二个集合元素的对象。以下是表`E1`和`E2`的交集（`E1∩E2`）：`E1∩E2` = { {(EMPNO;102), (ENAME;'Thomas'), (JOB;'SALESMAN')} }

`E1`和`E2`的交集是一个表。不难看出，两个具有相同标题的表的交集将总是产生另一个表。注意，当操作数是定义在不同标题上的表时，交集在表上也是封闭的。

然而，在这些情况下，交集是无用的，因为它总是产生空表；你或许可以通过研究`E1`和`E3`（不同标题的表）的交集来验证这一点。

你可以有意义地对表`E1`和（部分）`E3`进行交集运算，但首先你必须将其中一个表转换，使其具有与另一个表相同的标题。实现这一点所需的所有概念都已在第 4 章介绍：函数限制和函数复合。

看看下面表`E4`的定义。它将表`E3`的属性`E#`和`NAME`进行了重命名。

`E4 := { e◊{(EMPNO;E#),(ENAME;NAME),(JOB;JOB),(SAL;SAL)} | e∈E3 }`

集合`E4`是一个定义在`{EMPNO,ENAME,JOB,SAL}`上的表。我们使用函数复合（`◊`；参见定义 4-8）来重命名两个属性。属性`E#`被重命名为`EMPNO`，属性`NAME`被重命名为`ENAME`。另外两个属性（`JOB`和`SAL`）保持不变。

接下来我们需要去掉额外的`SAL`属性（它也不是`E1`标题的一部分）。为此我们使用函数限制（`↓`，参见定义 4-4）。看看`E5`的定义：

`E5 := { e↓{EMPNO,ENAME,JOB} | e∈E4 }`

`E5`等于以下集合：

{ {(EMPNO;101), (ENAME;'Anne'), (JOB;'TRAINER')}

,{(EMPNO;102), (ENAME;'John'), (JOB;'MANAGER')} }

`E5`与`E1`的交集现在变得有意义了，结果为以下集合：`E5∩E1` = { {(EMPNO;101), (ENAME;'Anne'), (JOB;'TRAINER')} }

最后，我们注意空表作为操作数参与的特殊情况。一个给定的表与空表的交集总是得到空表。

###### 差集

两个集合的差集包含所有是第一个集合元素但不是第二个集合元素的对象。以下是表`E1`和`E2`的差集（`E1–E2`）：`E1–E2` =

{ {(EMPNO;101), (ENAME;'Anne'), (JOB;'TRAINER') }

,{(EMPNO;103), (ENAME;'Lucas'), (JOB;'PRESIDENT')} }



如你所见，结果是一个基于 `{EMPNO, ENAME, JOB}` 的表。具有相同表头的两个表之间的差集运算总会产生另一个表。与交集类似，当操作数是基于 `不同` 集合的表时，差集运算符在表上也是封闭的。

然而，在这里，这些情况下的差集同样是无用的，因为它总是产生第一个表；第二个表不可能包含第一个表中的元组（由于表头不同）。因此，元组永远不会从第一个集合中被“移除”。

由于差集运算符不满足交换律，我们记下当空集作为操作数参与运算时的两种特殊情况（与前面的交集和并集形成对比）。令 `T` 为一个给定的表；那么 `T–∅` 的结果总是 `T`，而 `∅–T` 的结果总是空表。

###### 投影

另一个需要引入的重要表运算是：将一个表 `投影` 到给定集合上。给定表（例如 `T`）在给定集合（例如 `B`）上的 `投影`，执行了对 `T` 中每一个元组在集合 `B` 上的 `限制`。我们使用符号 `⇓` 来表示 `投影`。`投影` 运算符可以看作是 `限制` 运算符提升到表层面的一个版本。

###### 定义 5-3：表的投影
令 `T` 为一个函数集合，`B` 为一个集合。`T` 在 `B` 上的 `投影`（记作 `T⇓B`）定义如下：

```
T⇓B := { t↓B | t∈T }
```

`T` 在 `B` 上的 `投影` 包含 `T` 中每一个被限制在 `B` 上的函数。虽然上述定义描述了任何函数集合 `T` 和任何集合 `B` 的 `投影`，但我们主要（但不仅限于）对 `T` 是一个表，且 `B` 是此类表的表头的一个（非空）真 `子集` 的情况感兴趣。

###### 清单 5-7. 表 T3
让我们看一个例子来说明 `投影` 的概念。清单 5-7 引入了表 `T3`。它包含五个元组，其域为 `{empno, ename, salary, sex, dno}`。

```
T3 := { {(empno;10), (ename;'Thomas'), (salary;2400), (sex;'male'), (dno;1)}
      ,{(empno;20), (ename;'Lucas'), (salary;3000), (sex;'male'), (dno;1)}
      ,{(empno;30), (ename;'Aidan'), (salary;3000), (sex;'male'), (dno;2)}
      ,{(empno;40), (ename;'Keeler'), (salary;2400), (sex;'male'), (dno;1)}
      ,{(empno;50), (ename;'Elizabeth'), (salary;5600), (sex;'female'), (dno;2)} }
```

这是 `T3` 在 `{salary, sex}` 上的 `投影` 结果，记作 `T3⇓{salary, sex}`：

```
T3⇓{salary,sex} =
{ {(salary;2400), (sex;'male')}
, {(salary;3000), (sex;'male')}
, {(salary;5600), (sex;'female')} }
```

请注意，`T3` 在 `{salary, sex}` 上的 `投影` 产生了另一个表：一个基于 `{salary, sex}` 的表。此表只有三个元素，而 `T3` 有五个元素。`T3` 定义中列举的第一个和第四个元组产生了相同的（限制后的）函数，`T3` 中列举的第二和第三个函数也是如此。因此，这个 `投影` 结果的集合中只剩下三个元组。

你现在可以使用 `E4⇓{EMPNO,ENAME,JOB}` 来指定在“交集”一节中引入的 `E5`。

###### 限制

表通常包含许多元组。通常你只对其中一些元组感兴趣：具有特定属性的元组。你想查看给定表中元组的一个 `子集`。你可以通过 `表限制` 来派生给定表的一个 `子集`。这个运算符不需要新的数学符号；你可以简单地使用谓词方法来指定一个基于给定表的新元组集合（即一个新表）。

使用清单 5-7 中的员工表 `T3`，假设你想将该表限制为仅包含薪资大于 2500 的男性员工。以下是你指定该操作的方式（我们将此结果命名为 `T4`）：

```
T4 := { e | e∈T3 ∧ e(sex)='male' ∧ e(salary)>2500 }
```
