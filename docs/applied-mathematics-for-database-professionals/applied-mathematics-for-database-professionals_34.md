#### 章节小结

本节总结了本章内容，以项目符号列表格式呈现。在继续下一节的练习之前，你可以使用它来检查对本章引入的各种概念的理解程度。

-   *表* 可以形式化地表示为一组函数，这些函数共享一个共同的域；你可以通过以*枚举方式*列出所有涉及的函数来指定这个集合。
-   表的元素被称为元组；元组中的有序对称为*属性-值对*。
-   表中所有元组共享的域被称为该表的*表头*。
-   我们通常将表绘制为行和列的*二维图像*（简写记法）。列标题代表表的表头，行代表表中的元组。
-   元组的绘制顺序（简写记法）或枚举顺序（集合论记法）无关紧要。列的顺序或属性-值对的顺序也无关紧要。
-   特征化的广义乘积将产生一个表。你可以使用这个给定的集合以*谓词方式*指定表；通过添加谓词，你可以进一步约束集合的内容。
-   一个*数据库状态*，包含 x 个表，可以形式化地表示为包含 x 个有序对的函数。在每个有序对中，第一个坐标代表一个表结构的名称，第二个坐标代表该表结构当前的表。
-   一个*数据库纲要*形式化地描述了一个数据库设计的结构。它引入了表结构名称以及数据库设计中涉及的所有属性的名称。
-   由于表是集合，你可以将著名的集合运算符 *并集* (`∪`)、*交集* (`∩`) 和 *差集* (`–`) 应用于表作为其操作数。
-   你可以使用 *投影* 运算符 (`⇓`) 将所有元组*限制*在表表头的一个给定子集内。
-   你可以通过以谓词方式指定一个基于给定表的新集合，从给定表中选择元组的一个子集。在这种谓词规范中添加的谓词被称为执行表的*限制*。
-   *连接* 运算符 (`⊗`) 组合来自两个给定表中的*兼容*元组。由于它基于兼容性概念，它要求用于此组合的属性具有匹配的名称。
-   *属性重命名* 运算符 (`◊◊`) 使你能够选择性地重命名表表头中的一个或多个属性。你可以使用它来确保在连接这两个表之前，它们的属性具有匹配的名称。
-   你可以使用*表扩展*向表中添加新属性（包括它们在每个元组中的值）。
-   五种聚合运算符—— *求和*、*计数*、*平均值*、*最大值* 和 *最小值*——使你能够通过对给定集合中的所有元素进行聚合来计算一个值。

## 元组、表与数据库谓词

本章将介绍元组谓词、表谓词和数据库谓词的概念（使用在第一章中建立的谓词含义）。你可以使用元组谓词、表谓词和数据库谓词来描述数据库设计的数据完整性约束。本章的目标是建立这三类不同的谓词——我们将其统称为数据完整性谓词——并为每一类谓词提供充足的示例。

> **注意：** 我们在本章介绍数据完整性谓词，并展示它们对于某个特定论元（值）成立。当然，我们并不关心这种附带属性（一个谓词对于一个给定值是否为真）；它们应该对值空间成立（也就是说，它们应该构成一种结构性属性）。这一点将在第七章进一步阐述。

数据完整性谓词处理的是数据。通过考察它们所处理的数据范围，我们将数据完整性谓词分为三类。按作用范围递增的顺序，一个谓词可以处理一个元组、一个表或一个数据库状态。

“元组谓词”一节确立了元组谓词的概念。这类数据完整性谓词有一个类型为元组的参数。你可以用它来接受或拒绝一个元组（粗略地说，一组属性值）。

下一节“表谓词”将介绍表谓词。表谓词处理一个表，因此有一个类型为表的参数。我们将使用它们来接受或拒绝一个表（一组元组）。

在“数据库谓词”一节中，我们将使你熟悉数据库谓词的概念。数据库谓词处理数据库状态。我们将使用它们来接受或拒绝一个数据库状态（粗略地说，一组表）。

> **注意：** 本书区分这三类数据完整性谓词是有原因的。在第七章，你将学习一种形式化的方法来规约数据库设计，并了解该方法采用了一种分层的方法。每一层（总共三层）分别引入一类约束：元组约束、表约束和数据库约束。你将通过数据完整性谓词来指定这些约束。

“表和数据库谓词的常见模式”一节将探讨五种在数据库设计中经常起作用的数据完整性谓词模式：唯一标识、子集要求、特化、泛化以及连接中元组谓词。

在数据库设计中，你总是会遇到（极少数情况除外）基于唯一标识和子集要求谓词的数据完整性约束。你可能会将这两种谓词模式分别与关系概念中的唯一键和外键约束联系起来。特化和泛化谓词模式可能会让你想起 ERD 中的子类型和超类型概念。连接中元组谓词模式涉及两个或多个表之间的连接，它是数据库设计中也经常遇到的另一类约束的基础。

#### 元组谓词

让我们立即解释什么是元组谓词，然后快速用几个例子演示这个概念。元组谓词是一个带有一个参数的谓词。该参数的类型是元组。清单 6-1 展示了几个元组谓词的例子。

```
P0(t) := ( t(sex) ∈ {'male', 'female'} )

P1(t) := ( t(price) ≥ 20 ⇒ t(instock) ≤ 10 )

P2(t) := ( t(status) = 'F' ∨ ( t(status) = 'I' ∧ t(total) > 0 ) )
P3(t) := ( t(salary) < 5000 ∨ t(sex) = 'female' )

P4(t) := ( t(begindate) < t(enddate) )
```

谓词 `P0` 规定元组的 `sex` 属性值应为 `'male'` 或 `'female'`。

谓词 `P1` 规定，如果一个元组的 `price`（属性）值大于或等于 20，那么其 `instock`（属性）值应小于或等于 10。`P2` 规定 `status` 值要么是 `'F'`，要么是 `'I'`，但若是后者，同时 `total` 值应大于 0。谓词 `P3` 规定 `salary` 应小于 5000 或 `sex` 是 `'female'`。最后，谓词 `P4` 规定 `begindate` 值应小于 `enddate`。

我们将使用元组谓词（在第七章）来接受或拒绝一个给定的元组。如果谓词——当用给定元组作为论元实例化后——变成一个假命题，那么我们将拒绝该元组；否则，我们将接受它。在后一种情况下，我们说给定的元组满足该谓词。在前一种情况下，该元组违反了该谓词。

再次，让我们用几个示例元组来演示（见清单 6-2）。

```
t0 := { (empno;101), (begindate;'01-aug-2005'), (enddate;'31-mar-2007') }

t1 := { (price;30), (instock;4) }

t2 := { (partno;101), (instock;30), (price;20), (type;'A') }

t3 := { (status;'F'), (total;20) }

t4 := { (total;50) }

t5 := { (salary;6000), (sex;'male'), (name;'Felix') }

t6 := { (salary;5000), (sex;'female') }
```

用元组 `t5` 实例化谓词 `P0`——形成表达式 `P0(t5)`——得到命题 `'male'∈{'male', 'female'}`，这是一个真命题，我们说 `t5` 满足 `P0`。用元组 `t1` 实例化谓词 `P1`——表达式 `P1(t1)`——得到命题 `30 ≥ 20 ⇒ 4 ≤ 10`。这也是一个真命题；因此 `t1` 满足 `P1`。请自行验证以下命题是真命题：`P0(t6)`、`P2(t3)`、`P3(t6)`、`P4(t0)`。以下是假命题：`P1(t2)`、`P3(t5)`。

#### 表谓词

在上一节中，你被介绍了一种特定类型的谓词：元组谓词。在本节中，我们将向你介绍*表谓词*的概念。表谓词处理单个表中的*多个元组*。它们是具有单个类型为表（table）的参数的谓词。列表 6-3 展示了一些表谓词的例子。

**列表 6-3.** 表谓词示例

```
P5(T) := ¬( ∃t1,t2∈T: t1≠t2 ∧ t1(partno)=t2(partno) )
P6(T) := ( (Σt∈T: t(instock)*t(price)) ≤ 2500 )
P7(T) := ( ∀t1∈T: t1(instock)=0 ⇒
            (∃t2∈T: t2(partno) ≠t1(partno) ∧ t2(name)=t1(name) ∧ t2(instock)>0 ))
```

谓词 `P5` 表明，在表 `T` 中没有两个不同的元组（零件）具有相同的 `partno` 值。假设谓词 `P6` 也处理零件表，它大致表示——由零件表 `T` 代表的总库存价值不超过 2500。谓词 `P7` 表明，每当某个零件缺货时，就应该存在另一个不同的零件（具有不同的零件编号），它具有相同的名称并且仍有库存。

不失一般性，我们也将——就像对待元组谓词一样——将注意力限制在由单个合取项组成的表谓词上。如果你可以将一个给定的表谓词 `P(T)` 重写为 `Q(T) ∧ R(T)` 形式的表达式，那么你实际上是在处理两个表谓词。以下是一个例子来说明这一点：

```
( ∀t1∈T: t1(instock)>100 ⇒
    ( (∀t2∈T: t2(partno)≠t1(partno) ⇒ t2(name)≠t1(name))
      ∧
      t1(price)<200
    )
)
```

这个谓词表明，对于一个给定的零件，如果我们有超过 100 件库存，那么不应该存在另一个具有相同名称的零件，并且该给定零件的价格也不应大于等于 200。这个表谓词是以下形式：

```
( ∀t∈T: Q1(t) ⇒ ( Q2(t,T) ∧ Q3(t) ) )
```

使用以下重写规则

```
( A ⇒ (B ∧ C) ) ⇔ ( (A ⇒ B) ∧ (A ⇒ C) )
```

连同全称量词的分配律（见列表 3-12），你可以将其重写为两个谓词的合取：

```
( ∀t∈T: Q1(t) ⇒ Q2(t,T) )
∧
( ∀t∈T: Q1(t) ⇒ Q3(t) )
```

这表明你正在处理两个谓词。第一个合取项确实是一个表谓词，而第二个实际上是一个伪装的元组谓词！
