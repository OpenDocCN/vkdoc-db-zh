# 第 1 章 逻辑：简介

如果这让你感到困惑，请不要担心；对于本书所涉及的数学应用而言，这并不重要。

可以说逻辑学是可靠的；它已经存在了许多世纪，并在这些世纪里被许多杰出的科学家研究过。

**注意：** 如果你想了解更多关于逻辑的历史，或者整个数学史，`http://en.wikipedia.org`是一个极好的信息来源。

如今，源自逻辑的形式方法不仅用于数学、信息学（计算机科学）和人工智能；它们还用于生物学、语言学，甚至法学。

基于逻辑的数据管理的重要性由 E. F. (Ted) Codd（1923–2003）在 1969 年预见，当时他首次在其 IBM 研究报告“存储在大型数据库中的关系的可推导性、冗余性和一致性”中提出了他著名的关系模型。该研究报告中引入的数据库管理关系模型已被证明是数据管理方面具有影响力的通用理论，并且仍然是他最令人难忘的成就。

每一门科学都（应该）基于逻辑：每一个理论都是一个句子（或陈述）系统，这些句子被接受为真，并且可以根据一些明确定义的规则来*推导*出新的陈述。本书的主要目标之一就是解释关系数据管理科学所基于的数学概念。

#### 值、变量和类型

你可能对*值*和*变量*这两个术语有些概念。然而，精确地定义这两个术语很重要，因为它们经常被误解和混淆。

一个*值*是一个具有确定含义的独立常量。例如，整数`42`是一个值。你不能更新一个值；如果你能，它就不再是同一个值了。

值可以使用某种编码以多种方式表示。值可以具有任意的复杂性。

一个*变量*是值的容器。变量在时间和空间的进程中会获得不同的值。我们称值的改变为变量的*更新*。变量有一个*名称*，允许你在不知道它们当前代表哪个值的情况下谈论它们。

在本书中，我们将始终使用具有给定*类型*的变量。变量被允许持有值的*值集合*被称为*该变量的类型*。

一个包含表值的数据库是变量的一个例子；在任何时间点，它都有某个（复杂的）值。*我们将在本书的第二部分以精确的方式指定数据库变量的类型。*

#### 命题和谓词

在逻辑中，我们处理的主要组成部分是命题和谓词。命题是一个*陈述句*，它要么为真，要么为假。

**注意：** 如果对于句子`S`，以下是一个恰当的英文问题：“`Is it true that S?`”，那么`S`就是一个陈述句。

如果一个命题为真，我们说它的“真值”为真；如果一个命题为假，它的真值为假。以下是一些命题的例子：

- 本书有两位作者。
- `16`的平方根等于`3`。
- 所有数学家都是骗子。
- 如果`Toon`熟悉`SQL`，那么`Lex`有三个女儿。

所有四个例子都是陈述句；如果你在它们前面加上“`Is it true that`”，就会形成一个恰当的英文问题，你可以判断这些陈述句是否为真。第一个命题的真值为真；它是一个真命题。第二个命题显然是假的；`16`的平方根等于`4`。第三个例子也是一个命题，尽管你可能觉得很难判断它的真值。但在理论上你*可以*判断它的真值。假设你对谁是谁不是数学家有一个清晰的定义，你就可以确定需要检查的人的集合。然后你必须检查每一个数学家，在这个相当大但有限的集合中，来找出这个命题的真值。如果你找到一个不说谎的数学家，那么这个命题就是假的；另一方面，如果找不到这样的数学家，那么这个命题显然是真的。在最后一个例子中，“`Toon`”和“`Lex`”是本书的作者。因此，如果你对本书作者有足够的了解，你应该能够判断这个谓词是真还是假。顺便说一句，这个命题是假的；`Toon`确实熟悉`SQL`，但`Lex`并没有三个女儿。

**注意：** 一个常见的误解是认为只有真陈述才是命题；命题也可以为假。

以下例子不是命题：

- `x + y > 10`
- `x`的平方根等于`z`。
- 你为这本书付了多少钱？
- 停止设计数据库。

前两个例子包含嵌入变量；这些句子的真值取决于这些变量当前持有的值。后两个例子不是陈述句，因此不是命题。

一个*谓词*是具有陈述句形式的东西。一个谓词包含嵌入变量，其值在当时是未知的；在不知道变量值的情况下，你无法判断所声明的内容是真还是假。我们将谓词中的嵌入变量称为谓词的*参数*。

前面列表中的前两个例子是谓词；它们包含嵌入变量`x`、`y`和`z`。以下是一些谓词的其他例子：

- `i`的值为`4`。
- `x`居住在`y`。
- 如果`Toon`熟悉`SQL`，那么`Lex`有`k`个女儿。

你无法判断第一个例子是真还是假，因为它取决于参数`i`的实际值。第二个例子也是如此；只要你参数`x`代表哪个人类以及参数`y`代表哪个城市，你就无法说出这个谓词是真还是假。最后，最后一个例子的真值取决于其参数`k`。你已经看到，如果值`3`替代参数`k`，那么这个谓词的真值就变成假。

一个有`n`个参数的谓词被称为`n`元谓词。如果你用一个值替换`n`元谓词中的一个参数，那么该谓词就变成了`(n-1)`元谓词。例如，前面的第二个例子是一个二元谓词；它有两个参数`x`和`y`。如果你用值`Lex`替换参数`x`，那么这个谓词就变成下面的表达式：

`Lex lives in y`

这个表达式代表一个一元谓词；它有一个参数。真值仍然取决于剩余参数的值。如果你现在用值“`Utrecht`”（荷兰一个城市的名字）替换参数`y`，该谓词就变成了一个零元谓词。

`Lex lives in Utrecht`


##### 谓词与命题

你现在看出这变成了一个命题了吗？你可以确定该表达式的真值。如示例所示，命题可被视为谓词的一种特殊情况；它们是无参数的谓词。通过提供参数的替换值，你可以将谓词转换为命题。这被称为用给定值*实例化*该谓词。

**注意** 对谓词的一种理解如下——此处我们引用克里斯·戴特（Chris Date）在《数据库深度剖析》（O'Reilly Media，2005）中的表述：“你可以将谓词视为一个真值函数。与所有函数一样，它有一组参数，当被调用（实例化）时，通过提供参数值来返回一个结果，并且，由于它是真值函数，该结果要么为真（TRUE），要么为假（FALSE）。”

谓词的参数也被称为该谓词的*自由变量*。

还有另一种将谓词转换为命题的方法：通过使用*量词*来*绑定*涉及的自由变量。自由变量于是变成了所谓的*约束变量*。

（在集合上的）量化是逻辑中，乃至数据管理中一个更为重要的概念；我们将在第 3 章（“量词”一节）中详细讨论它。

表 1-1 总结了谓词和命题的属性。

**表 1-1.** 谓词与命题

| **谓词** | **命题** |
| :--- | :--- |
| 声明句的形式 | 声明句 |
| 带参数 | 无参数 |
| 真值函数 | 要么真（TRUE）要么假（FALSE） |
| 需要输入以评估真/假 | 谓词的特殊情况（无需输入） |

在我们继续之前，让我们简要看看如下句子：这句话是假的。我在说谎。

这些是*自指*句子；它们谈论的是自身。这类句子可能带来麻烦，因为它们可能导致矛盾。第二个例子被称为说谎者悖论。如果你假设这些句子为真（TRUE），那么你可以推断出它们是假（FALSE）的（反之亦然）；你无法确定它们是真（TRUE）还是假（FALSE），而这对于它们成为有效命题是必需的属性。解决方法很简单，就是取消它们作为有效命题的资格。

你还必须将“格式不良”的表达式排除在有效谓词之外；即那些不符合我们语法规则的表达式，例如：3 是 4 的一个元素。n = 4 ∨ ∧ m = 5

第一个是格式不良的，因为 4 不是一个集合；它是一个数值。说 3 是它的一个元素没有任何意义。在这里，我们假设你对集合以及某物是集合元素的概念有所了解；我们将在第 2 章介绍基础集合论。

第二个表达式包含了两个连续的连接词∨和∧（在下一节解释），这是非法的。

##### 逻辑连接词

你可以通过将*逻辑连接词*应用于现有谓词来构建新谓词。最著名的连接词是*合取*（逻辑与）、*析取*（逻辑或）和*否定*（逻辑非）；我们将频繁使用的另外两个连接词是*蕴涵*和*等价*。在本书中，我们使用数学符号来表示这些逻辑连接词，如表 1-2 所示。

**表 1-2.** 逻辑连接词

| **符号** | **含义** | **也被称为** | **例子** |
| :--- | :--- | :--- | :--- |
| ¬ | 非（NOT） | 否定 | ¬（Toon 居住在乌得勒支） |
| ∧ | 与（AND） | 合取 | （n = 4） 与 （m = 2） |
| ∨ | 或（OR） | 析取 | （job = 'CLERK'） 或 （salary > 5000） |
| ⇒ | 蕴涵（IMPLIES） | 蕴涵（如果...那么...） | （x < 6） 蕴涵 （x ≠ 7） |
| ⇔ | 等价于（IS EQUIVALENT TO） | 等价（当且仅当） | （x = 10） 等价于 （x/2 = 5） |

**注意** 尽管你可能对上述列出的大多数连接词的含义有所了解，但我们必须谨慎，因为它们也在自然语言中使用。自然语言是非正式和不精确的；因此，词语的含义有时取决于它们所使用的上下文。

我们将在介绍*真值表*概念的“真值表”一节中精确定义这五个逻辑连接词的含义。

##### 简单谓词与复合谓词

逻辑连接词可以被视为*逻辑运算符*；它们接受一个或多个谓词作为其操作数，并返回另一个谓词。例如，表 1-2 中的逻辑与以 n = 4 和 m = 2 作为其操作数（输入），并返回示例列下列出的谓词作为其输出谓词。我们说与运算符是以操作数 n = 4 和 m = 2 被调用的。

通常将没有逻辑连接词的谓词称为*简单*谓词，将带有一个或多个逻辑连接词的谓词称为*复合*谓词。复合谓词中连接词的输入谓词（操作数）也被称为该谓词的*组成部分*。

另一个常见术语是复合谓词的*复杂度*，即该谓词中出现的连接词的数量。以下是一些复合谓词的更多例子，这次使用了数学符号。

**注意** 你可能难以阅读最后一个例子。这是因为在该例子中*运算符优先级*（在下一节解释）不明确。

*   n = 4 ∧ m = 5
*   x = 42 ∨ x ≠ 42
*   Toon 居住在乌得勒支 ⇒ 1 + 1 = 2
*   P ∧ Q ⇒ P ⇒ Q

在最后一个例子中，P 和 Q 是*命题变量*。它们表示一个真值未指定的任意命题。我们将使用字母 P、Q、R 等表示命题变量。你可以使用命题以及命题变量，结合前面介绍的逻辑连接词来形成新的断言。包含至少一个命题变量的断言被称为*命题形式*。如果你将命题变量替换为命题，就会得到一个命题。

类似地，我们将在像 P(x)、Q(s,t)、R(y) 这样的表达式中使用字母 P、Q、R 等来表示至少带有一个参数的任意谓词。这类谓词的参数由字母表示——在这些情况下是 x、y、s 和 t——在这些表达式的括号内用逗号分隔。例如，前面列出的第一个复合谓词可以表示为 P(n,m)。如果我将参数 n 替换为值 3，将参数 m 替换为值 5，该谓词就变成了一个命题 P(3,5)，其真值为假（FALSE）。

在数学中，所有逻辑连接词的精确含义（语义）都可以用真值表来定义，如下一节所介绍。有了这些定义，你就可以从其组成部分的真值“计算”出复合谓词的真值。

#### 使用括号和运算符优先级规则

你可以在逻辑表达式中使用括号来表示运算符优先级，就像在常规算术中一样，在那里你可以覆盖由助记符“Please Excuse My Dear Aunt Sally”定义的惯用运算符优先级。

**注意** 这些单词的首字母表示算术中的运算符优先级：括号（Parentheses）、指数（Exponents）、乘法（Multiplication）、除法（Division）、加法（Addition）、减法（Subtraction）。

即使一种语言定义了正式的运算符优先级规则——允许你在某些表达式中省略括号——为了提高可读性，使用括号仍然是一个好主意。

包含那些即使省略也不会改变含义的括号并不是错误的。

你可以将括号加入到前面的四个复合谓词示例中，如下所示：

*   （ n = 4 ） ∧ （ m = 5 ）
