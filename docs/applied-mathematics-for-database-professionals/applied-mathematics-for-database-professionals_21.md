# 量词与集合论

*   在集合 A 中存在一个值，当该值被提供给谓词 `P(x)` 作为参数 `x` 时，会使谓词 `P(x)` 成为真命题。
*   对于集合 B 中的所有值，当该值被提供给谓词 `Q(y)` 作为参数 `y` 时，谓词 `Q(y)` 都将成为真命题。

> **注意**：这两个量化表达式是否为真，显然取决于所涉及的集合（A 和 B）以及给定的谓词 `P(x)` 和 `Q(y)`。

##### 一般量词语法

3-12 节中介绍的一般存在量词和全称量词语法实际上是简写语法。

该语法依赖于集合论（它使用了“属于”运算符）。纯粹的逻辑学家不需要集合论来表达量化。事实上，集合论是逻辑学的一个分支。从逻辑学家的角度来看，一个集合表示某个谓词的外延。例如，集合 A 可能表示谓词 `A(x)` 的外延，其形式为“x 是<whatever A means>”。集合 A 如何表示谓词 `A(x)`？通过成为包含所有对象的集合，这些对象在代入 `A(x)` 中的 `x` 后，会产生一个真命题。文中给出的关于撒谎的数学家的例子，可以表示为 `∀x: "x is a mathematician" ⇒ "x is a liar"`。等价于 3-12 节介绍的简写语法且**不**依赖于集合论的通用语法现在分别是 `∃x: "x is a <whatever A means>" ⇒ P(x)` 和 `∀y: "y is a <whatever B means>" ⇒ Q(y)`。由于大多数实际的量化谓词形式正是“如果 x 是...，则 `P(x)`”，因此习惯上使用依赖于集合论的简写符号。

我们现在可以重新审视第 1 章“命题与谓词”一节中给出的一个示例命题：所有数学家都是撒谎者。这个命题实际上是一个全称量化。以下是规范指定此谓词的形式方法：

```
∀x∈M: "x is a liar"
```

这里 `M` 代表所有是数学家的人的集合。该集合用于对谓词“`x`是撒谎者”进行量化——这是一个包含名为 `x` 的参数的谓词。

此量化绑定了参数 `x`（它为该参数提供了值）。结果的量化现在是一个命题；它实际上不再具有参数。

看下面三个例子（使用第 2 章介绍的自然数集 `N`）：

```
∃x∈N: x < 42

∀y∈N: y/2 ∈ N

∀z∈N: z ≥ 0
```

第一个和第三个谓词（更准确地说，是命题，因为它们实际上没有参数）为真，第二个为假。要证明第一个（存在量化）为真，只需给出一个例子即可。在这种情况下，我们必须为 `x` 提供一个小于 42 的自然数；例如，41 是小于 42 的自然数。要证明一个全称量化为假（第二个），只需给出一个反例；例如，3（一个奇数）除以 2 得到 1.5（一又二分之一），这不是自然数。对于第三个语句，你必须检查所有自然数，并找出一个不大于或等于零的自然数，以证明该命题为假。当然，你永远找不到这样的数。最小的自然数是 0（零）——它大于或等于 0——因此，该谓词对所有自然数都为真。

当一个谓词的参数被量词绑定时，该参数的*名称*就可以自由选择了。让我们用一个例子来阐明这一点。考虑谓词“`x`住在乌得勒支”。这个谓词有一个参数，碰巧被命名为 `x`。现在让我们通过将该谓词在集合 `{Toon, Lex, Chris, Cary, Hugh}` 上进行全称量化来定义一个新命题，如下所示：

```
∀x∈{Toon, Lex, Chris, Cary, Hugh}: "x lives in Utrecht"
```

可能不难看出，这个命题等价于以下复合命题（一个包含五个合取支的合取式）：

```
"Toon lives in Utrecht"
∧ "Lex lives in Utrecht"
∧ "Chris lives in Utrecht"
∧ "Cary lives in Utrecht"
∧ "Hugh lives in Utrecht"
```

你应该意识到，参数的原始名称（`x`）不会影响构成我们新命题的全称量化的含义。如果你将参数 `x` 替换为一个名为 `y` 的参数——同时在量词和被量化的谓词内部进行替换——你将得到一个与前述五个合取支的合取式逻辑等价的命题：

```
∀y∈{Toon, Lex, Chris, Cary, Hugh}: "y lives in Utrecht"
```

你可以任意选择量化（无论是全称还是存在）中绑定参数的名称。有时为了避开混淆，为这样的参数选择一个不同的名称是方便的。

因为一个量化表达式（如前面的例子）是一个谓词，所以将其用作*复合*谓词的组成部分是完全有效的。也就是说，你可以使用第 1 章介绍的逻辑连接词将量化表达式与其他谓词组合起来。例如，以下两个表达式是有效的谓词（事实上，两者都是命题）：

```
( ∀x∈A: x > 0 ) ∧ ( ∃y∈B: y < 10 )

¬( ∀z∈A: z > 10 ) ⇒ ( ∃x∈A: x < 11 )
```

##### 量词与有限集

尽管如今的数据库越来越大，但根据定义，它们始终是有限的——这与数学中的集合不同，后者有时是无限的。如果你只处理有限集合，你总是可以将存在量词和全称量词分别视为迭代 `OR` 和迭代 `AND` 结构。迭代是在量化参数所来自的集合的所有元素上进行的；对于每个元素值，谓词中变量的出现都被该值替换。这种解释量化表达式的替代方法可能在你研究其真值时有所帮助。

例如，假设 `P` 是所有小于 15 的质数的集合：`P := {2,3,5,7,11,13}`

那么这两个命题

```
∃x∈P: x > 12

∀y∈P: y > 3
```

在逻辑上等价于以下两个命题：在 `P` 的所有元素上迭代 `x`：`2>12 ∨ 3>12 ∨ 5>12 ∨ 7>12 ∨ 11>12 ∨ 13>12`

在 `P` 的所有元素上迭代 `y`：`2>3 ∧ 3>3 ∧ 5>3 ∧ 7>3 ∧ 11>3 ∧ 13>3`

你现在可以手动计算两个谓词的真值。如果你评估第一个命题中每个析取支的真值，它变成 `FALSE ∨ FALSE ∨ ... ∨ FALSE ∨ TRUE`，结果为 `TRUE`。如果你对第二个命题的合取支做同样的事情，你得到 `FALSE ∧ FALSE ∧ TRUE ∧ ... ∧ TRUE`，结果为 `FALSE`。

##### 空集上的量化

空集（`∅`）不包含任何元素。因为它仍然是一个集合，所以你可以在空集上进行量化。以下两个表达式是有效的命题：

```
∃x∈∅: P(x)

∀y∈∅: Q(y)
```

但是，说空集中存在一个元素 `x` 使得 `P(x)` 成立是什么意思呢？显然，空集没有元素。因此，无论涉及的谓词 `P` 是什么，只要我们声称在空集中有这样的元素，我们必定是在陈述一些明显为**假**的事情，因为不可能从空集中选择一个元素。
