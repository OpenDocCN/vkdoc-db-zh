# 表的限制

`表 T4` 包含两个元组：代表员工 `'Lucas'` 和 `'Aidan'` 的元组，他们是 `T3` 中仅有的工资超过 2500 的男性员工。

`T4` 的规范是表限制的一个简单案例示例，这也是你将经常使用的一种操作。以下是这类限制的通用模式：`{ t | t∈T ∧ P(t) }`

在此表达式中，`T` 代表被限制的表，`P(t)` 代表一个以某种元组类型为单个参数的谓词。谓词 `P` 可以是一个简单谓词或复合谓词（即涉及逻辑连接词的谓词）。此谓词通常包含涉及使用参数 `t` 的函数应用的表达式；对于 `T` 的每个元组，你可以通过检查（仅）该元组的一个或多个属性值来决定它是否保留在结果集中。

表的限制可以稍微复杂一些。例如，假设我们希望将表 `T3` 限制为仅包含男性员工中工资最高的（在所有男性中）和女性员工中工资最高的（在所有女性中）。指定这种对 `T3` 的限制（我们将命名为 `T5`）的一种方式如下：`T5 := { e | e∈T3 ∧ ¬(∃e2∈T3: e2(sex)=e(sex) ∧ e2(sal)>e(sal)) }`

表 `T5` 将包含 `T3` 中每一个元组（比如 `e`），条件是在 `T3` 中不存在另一个元组（比如 `e2`），使得 `e` 和 `e2` 在 `sex` 属性上匹配，并且 `e2` 中的 `sal` 值大于 `e` 中的 `sal` 值。请注意，这个特定的限制实际上会产生一个包含三个元组的表；`T3` 中有两名男性员工的工资都是最高的。

这种更复杂的表限制符合另一种常用的模式。以下是这类限制的通用模式：`{ t | t∈T ∧ P(t,T) }`

在此模式中，`P(t,T)` 代表一个带有两个参数的谓词。第一个参数取表 `T` 中的一个元组作为值，第二个参数取实际的表 `T` 作为值。对于 `T` 的每个元组，你不仅通过检查该元组的一个或多个属性值，还通过检查同一表中的其他元组来决定它是否保留在结果集中。

限制也可以涉及其他表（除了被限制的表之外）。假设我们有一个名为 `D1` 的表，其模式为 `{dno,dname,loc}`，代表部门表（具有明显的语义）。假设我们希望将表 `T3` 限制为仅包含工资超过 2500 且受雇于一个在 `D1` 中已知且位于旧金山 (`'SF'`) 的部门的男性员工。指定这种对 `T3` 的限制的一种方式如下：`{ e | e∈T3 ∧ e(sex)='male' ∧ e(sal)>2500 ∧ (∃d∈D1: e(dno)=d(dno) ∧ d(loc)='SF') }`

这种表的限制符合以下模式：`{ t | t∈T ∧ P(t,S) }`

这里 `S` 代表某个给定的表（不同于 `T`）。对于 `T` 的每个元组，你现在不仅通过检查该元组的一个或多个属性值，还通过检查另一个表中的元组来决定它是否保留在结果集中。

正如你现在可能理解的，执行对给定表的限制还有很多其他方式；可能涉及任意数量的其他表（包括被限制的表）。

在实践中，通常也可以对作为连接（join）结果的表指定此类限制（参见下一节）。

###### 连接（Join）

在数据库中，你经常会发现两个表通过它们共享的某个属性（或属性集）相互关联。如果是这种情况，那么你可以有意义地组合这些表的元组。

这种元组的组合是通过两个元组的*兼容性*概念完成的（你可能需要回顾定义 4-3 以刷新记忆）。第一个表中的每个元组都与第二个表中与其*兼容*的元组进行组合。让我们用一个例子来演示这一点。图 5-5 和 5-6 介绍了表 `T6`（代表员工）和表 `T7`（代表部门）。

**图 5-5.** *示例表 T6*

**图 5-6.** *示例表 T7*

你可以将表 `T6` 和 `T7` 中在共享属性（本例中为 `deptno` 属性）上对应的元组组合起来。这种组合产生的表，其标题是 `T6` 标题和 `T7` 标题的并集。

图 5-7 显示了表 `T6` 和 `T7` 组合的结果。

**图 5-7.** *表 T8，表 T6 和 T7 的组合*

你可以用谓词方式正式指定此表如下：`{ e∪d | e∈T6 ∧ d∈T7 ∧ e(deptno)=d(deptno) }`

`T6` 中的每个元组都与 `T7` 中具有相同 `deptno` 值的元组组合。

两个“相关”表的元组的这种组合被称为这两个表的*连接（join）*。定义 5-4 正式定义了这个操作符。

**定义 5-4：两个表的连接** 设 `R` 和 `T` 为两个（不一定相异）的表。`R` 和 `T` 的连接（记作 `R⊗T`）定义如下：

```
R⊗T := { r∪t | r∈R ∧ t∈T ∧ "r 和 t 是兼容的" }
```

两个表（比如 `R` 和 `T`）的连接将包含每对元组 `r` 和 `t` 的并集，其中 `r` 是 `R` 中的一个元组，`t` 是 `T` 中的一个元组，且 `r` 和 `t` 的并集是一个函数（兼容性概念）。根据这个定义，你现在可以将图 5-7 中的 `T8` 指定为 `T6⊗T7`。

两个正在进行连接的表所共有的属性集被称为*连接属性集*。我们有时也会说两个元组是*可连接的*，而不是说它们是兼容的。

正如前一节关于限制的结尾所提到的，你经常会限制两个表的连接。例如，下面是表 `T6` 和 `T7` 的连接限制为仅包含那些代表位于丹佛的职员的元组：`{ t | t∈T6⊗T7 ∧ t(job)='CLERK' ∧ t(loc)='DENVER' }`

请注意，定义 5-4 并不限于操作数实际至少有一个共同属性的情况。你可以连接两个连接属性集为空的表（兼容性概念允许这种情况）；在这种情况下，第一个表中的每个元组都与第二个表中的每个元组兼容（这遵循定义 4-3）。你实际上是在将第一个表的每个元组与第二个表的每个元组进行组合。这种连接的特殊情况也被称为*笛卡尔积*。

通常，笛卡尔积在数据处理中的实际用途有限。涉及基数为 1 的表的笛卡尔积是合理且有时有用的。以下是这样一个笛卡尔积的例子：`T7⊗{ { (maxbudget;75000) } }`

这个表达式将 `T7` 中的每个元组与连接操作符的右侧参数（这是一个模式为 `{maxbudget}` 的表）中的每个元组进行连接。它实际上*扩展*了 `T7` 中的每个部门元组，添加了一个名为 `maxbudget` 的新属性。这个笛卡尔积实际上是*表扩展*的一个特例，这将在后面的“扩展”一节中介绍。

还要注意笛卡尔积的一个相反特殊情况，即你连接两个共享相同标题的表。也就是说，连接属性集等于这个共享的标题。在这种连接中，你实际上是在对这两个表求交集。这使得交集成为连接的一个特例（因此是一个冗余操作符）。

## 属性重命名

有时，两个表之间存在关联，使得连接它们是有意义的。然而，用于执行连接的属性名称并不匹配；用于连接的属性集合是空集。在这些情况下，连接操作符将无法按预期工作；它将执行一个笛卡尔积连接。连接操作的定义要求目标连接属性的名称相同。例如，假设前面表 `T7` 中的 `deptno` 属性被命名为 `dept#`。那么执行 `T6` 和 `T7` 的连接——`T6⊗T7`——将导致 `T6` 和 `T7` 的笛卡尔积。要解决此问题，您必须首先将 `T6` 中的属性 `deptno` 重命名为 `dept#`，或者将 `T7` 中的属性 `dept#` 重命名为 `deptno`。

同样，有时对两个表执行交集、并集或差集操作是有意义的；然而，两个表的标题不同，区别不在于基数，而在于属性的名称。要解决此问题，也必须首先重命名所涉及表的属性，使它们具有相等的标题。

为此，我们引入了**属性重命名**操作符；定义 5-5 正式定义了该操作符。

###### 定义 5-5：属性重命名操作符
设 `T` 是一个表，`f` 是一个函数。根据函数 `f` 对 `T` 中属性的重命名（记法 `T◊◊f`）定义如下：
```
T◊◊f := { t◊f | t∈T }。
```
您可以将属性重命名操作符视为函数复合操作符提升到表级别的一个版本。

使用此操作符，我们可以将 `T6` 中的属性 `deptno` 重命名为 `dept#`，如下所示：
```
T6◊◊{(empno;empno),(ename;ename),(job;job),(sal;sal),(dept#;deptno)}
```
请注意，重命名函数 `f` 为 `T6` 标题中的每个属性持有一个有序对。有序对中的所有第二个坐标代表表的当前属性名称。只有当需要重命名属性时，有序对的第一和第二个坐标才不同；第一个坐标将具有该属性的新名称。

###### 扩展

有时您希望向给定表添加属性。您想在表的标题中引入新属性，并为表中的每个元组提供这些新属性的值。在表上执行此类操作被称为**扩展表**或执行**表扩展**。表扩展不需要新的数学符号；您可以简单地使用谓词方法来指定一个新的元组集合（即一个新表），该新表基于给定的表。

让我们看一个表扩展的例子。图 5-8 显示了两个表。左边的一个是在 `{empno, ename}` 上的表；它是表 `T6`（见图 5-5）在 `{empno, ename}` 上投影的结果。我们将此表命名为 `T9`。右边的一个（`T10`）代表用属性 `initial` 扩展 `T9` 的结果。对于每个元组，此属性的值等于 `ename` 属性值的第一个字符。

> **注意** 我们将使用一个名为 `substr` 的函数从给定字符串中提取子串。表达式 `substr(<some string value>,n,m)` 表示 `<some string value>` 的子串，它从位置 `n` 开始，长度为 `m` 个字符。例如，表达式 `substr('I like this book',13,4)` 等于字符串值 `'book'`。

###### 图 5-8. 用新属性扩展示例
给定表 `T9`，您可以形式化地指定表 `T10` 如下：
```
T10 := { e ∪ { (initial;substr(e(ename),1,1)) } | e∈T9 }
```
对于 `T9` 中的每一个元组，例如 `e`，表 `T10` 持有一个由以下表达式表示的元组：
```
e ∪ { (initial;substr(e(ename),1,1)) }
```
此表达式向元组 `e` 添加一个属性-值有序对。添加的属性是 `initial`。附加到此属性的值是 `substr(e(ename),1,1)`，它表示 `e(ename)` 值的首字母。

这是另一个例子。图 5-9 显示了两个表。左边的一个（`T11`）代表一个在 `{deptno, dname}` 上的部门表。右边的一个（`T12`）是 `T11` 用一个 `emps` 属性扩展的结果。对于每个元组，`emps` 属性表示在表 `T6`（见图 5-5）中找到的、被分配到该元组所代表部门的雇员数量。

###### 图 5-9. 另一个用新属性扩展表的例子
给定表 `T11` 和 `T6`，您可以形式化地指定表 `T12` 如下：
```
T12 := { d ∪ { (emps;#{ e | e∈T6 ∧ e(deptno)=d(deptno)}) } | d∈T11 }
```
这里我们使用了基数运算符（符号 `#`）来统计每个部门的雇员数量。

###### 聚合

聚合运算符对一个集合进行操作。它们通过对集合中所有元素的某个表达式进行聚合，产生一个（数字）值。有五种常见的聚合运算符：`sum`（求和）、`average`（平均值）、`count`（计数）、`minimum`（最小值）和 `maximum`（最大值）。

您在第二章已经接触过求和运算符；您可能想快速回顾定义 2-12。您可以将求和运算符应用于一个表。这里有一个例子。给定表 `T6`（见图 5-5），以下表达式计算研究部门（`deptno=10`）所有雇员工资的总和：
```
(SUM x∈{ t | t∈T6 ∧ t(deptno)=10 }: x(sal))
```
前面求和聚合的结果值是 `24100`。现在看下面的表达式，它再次使用了表 `T6`：
```
{ e1 ∪ {(sumsal;(SUM x∈{ e2 | e2∈T6 ∧ e2(deptno)=e1(deptno) }: x(sal)))}
| e1 ∈ T6⇓{deptno} }
```
这里我们首先将 `T6` 投影到 `{deptno}`，然后用一个 `sumsal` 属性扩展结果表。对于此结果表中的每个元组，我们通过计算所有在该元组所代表的部门工作的雇员的工资总和来确定这个新属性的值。图 5-10 显示了此表达式的结果。

###### 图 5-10. `T6` 按部门的工资总和
我们不需要为 `count` 运算符引入新的定义。我们可以简单地使用集合论符号 `#`（基数）来统计集合的元素数量。您在前面的“扩展”部分已经看到了一个这样的例子；`T12` 的形式化规范就是 `count` 聚合运算符的一个例子。

定义 5-6 定义了平均值聚合运算符。

###### 定义 5-6：平均值运算符 (AVG)
表达式 `f(x)` 的平均值，其中 `x` 从给定的非空集合 `S` 中选择，`f` 表示在 `x` 上的任意数值函数，表示如下：
```
(AVG x∈S: f(x))
```
对于可以从集合 `S` 中选择的每个元素 `x`，平均值运算符计算表达式 `f(x)` 的值，并计算所有这些计算结果的平均值。请注意，如果 `S` 是空集，则此运算符未定义。

与前面类似，定义 5-7 定义了最大值 (`MAX`) 和最小值 (`MIN`) 聚合运算符。

###### 定义 5-7：最大值 (MAX) 和最小值 (MIN) 运算符
表达式 `f(x)` 的最大值和最小值，其中 `x` 从给定的非空集合 `S` 中选择，`f` 表示在 `x` 上的任意数值函数，分别表示如下：
```
(MAX x∈S: f(x)) 和 (MIN x∈S: f(x))
```
对于可以从集合 `S` 中选择的每个元素 `x`，最大值和最小值运算符计算表达式 `f(x)` 的值，并计算所有这些计算结果的最大值或最小值。请注意，如果 `S` 是空集，这些运算符也未定义。
