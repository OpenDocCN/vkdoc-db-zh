# 第二章：集合论：导论

#### 基数与单元素集

集合可以是*有限*的或*无限*的；然而，在处理数据库时，根据定义，你所有的集合都是有限的。对于每一个有限集合 `S`，其*基数*被定义为 `S` 中元素的个数。表示集合 `S` 基数的常见记法是 `|S|`，即在竖线之间放入集合的名称（或说明）。然而，在本书中，我们更倾向于使用 `#` 符号（如定义 2-2 中所用）来表示基数运算符，因为我们已经在谓词性集合说明和替代性集合说明中大量使用了竖线。

**■定义 2-2：集合的基数** 一个（有限）集合 `S` 的基数是 `S` 中元素的个数，记作 `#S`。

**■注** 无限集也有基数概念，但本书不予讨论。

代码清单 2-5 展示了一些集合及其基数的例子。

**代码清单 2-5.** *基数示例*
```
#{1,2,3} = 3
#{{1,2,3}} = 1
#{1,3,2,3} = 3
#{{1,3},{2,3}} = 2
#∅ = 0
#{∅} = 1
```
请注意第一个和第二个例子之间的细微差别。第三个例子表明重复元素不被计为独立元素；最后一个例子表明，由空集构成的集合并非空集——它包含一个元素，即空集本身。

##### 单元素集

代码清单 2-5 展示了两个基数为一的集合——第二个和最后一个例子。恰好包含一个元素的集合称为*单元素集*（参见定义 2-3）。

**■定义 2-3：单元素集** 单元素集是恰好包含*一个*元素的集合。

请注意，单元素集仍然是一个集合；因此，`{42}` *不等于* `42`。`{42}` 是一个集合，而 `42`——它唯一的元素——是一个整数值。注意区分；将单元素集与其包含的元素混淆是一个常见错误。

#### 选择运算符

我们引入一个特殊的运算符，允许你从单元素集中“提取”或“选择”其唯一的元素。在本书中，我们使用符号 `↵` 表示此运算符；例如：
```
↵{5} = 5
↵{{5,6}} = {5,6}
```
对于单元素集，你可以将选择运算符视为一个“去括号器”，使你能够编写原本会有些笨拙的表达式。`↵` 运算符总是期望其操作数是一个单元素集；因此，下面的表达式是一个格式不良、无意义的公式：
```
↵{5,6}
```
这里的操作数不是单元素集，而是一个基数为 2 的集合。

#### 子集

*子集*的概念相当直观，所以我们直接从定义开始，如定义 2-4 所示，该定义也引入了相反的概念：*超集*。

**■定义 2-4：子集与超集** 如果 `A` 的每一个元素都是 `B` 的元素，则 `A` 是 `B` 的子集（`B` 是 `A` 的超集）。记法为 `A ⊆ B`。
