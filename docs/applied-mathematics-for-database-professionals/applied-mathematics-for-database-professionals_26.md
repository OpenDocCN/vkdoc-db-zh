# 关于函数域与范围的介绍

函数 `F` 所有第一坐标的集合称为该函数的**定义域**；其表示法为 `dom(F)`。函数所有第二坐标的集合称为该函数的**值域**；其表示法为 `rng(F)`。这两个概念相当简单：要找到一个函数的定义域或值域，只需枚举该函数所有有序对的第一或第二坐标即可。

##### 示例函数的定义域与值域

**列表 4-2. 示例函数的定义域**

```
dom(F2) = dom(F3) = {X,Y,Z}
dom(F4) = {Z}
dom(F5) = ∅
dom(F6) = {empno,ename,born}
```

**列表 4-3. 示例函数的值域**

```
rng(F2) = {1,2}
rng(F3) = {1}
rng(F4) = {2}
rng(F5) = ∅
rng(F6) = {126,'Lex de Haan','11-aug-1954'}
```

##### 从图示确定定义域与值域

从函数的图示中，通过省略那些没有箭头发出或到达的元素，你可以确定其定义域和值域。图 4-2 展示了一个从集合 `A := {A,B,C,D,E}` 到集合 `B := {1,2,3,4,5}` 的二元关系 `F7`。如你所见，它直接表示一个函数（集合 `A` 的元素最多只有一个箭头发出），且 `dom(F7) = {A,C,D,E}` 以及 `rng(F7) = {2,4}`。

##### 函数的定义

如果满足以下两个性质，我们定义函数 `F` 是**在**集合 `A` **上**并**映射到**集合 `B` **内**的：
*   `dom(F) = A`
*   `rng(F) ⊆ B`

因为 `F` 是在 `A` 上映射到 `B` 内的函数，根据定义，所有第一坐标的集合——即 `F` 的定义域——等于 `A`。并且因为在一个在 `A` 上映射到 `B` 内的函数中，每个有序对都以 `B` 的一个元素作为第二坐标，所以 `F` 的值域将是集合 `B` 的一个子集；通常它会是 `B` 的一个真子集。

**注意** 你可能已经注意到，这里引入的集合论中“定义域”的概念与“域”一词在其他数据库教科书中的常见含义大不相同。在其他教科书中，一个“域”通常代表一个“表”中“列”的可能值集合——或者如果你愿意，可以说是它的类型。在本书中，有一个类似的概念对应于“域”的这个含义（我们将其称为**属性值集**），它将在“集合函数”一节中被精确定义。

##### 函数应用

给定一个函数永远不会有两个具有相同第一坐标的有序对，你可以明确地确定与函数的一个给定定义域元素配对的每一个值域元素（记住，从每个定义域元素恰好有一个箭头发出）。数学家有一种特殊的方式来表示一个给定定义域元素的单个值域元素。如果 `F` 是一个函数且 `x` 是 `dom(F)` 的一个元素，那么 `F(x)`（读作“F of x”或“F applied to x”）表示在函数 `F` 中与 `x` 配对的值域元素。这种表示法称为**函数应用**。我们将使用图 4-2 引入的函数 `F7` 来说明这一点。以下是 `F7` 的形式化规范：

```
F7 := { (A;2), (C;2), (D;4), (E;4) }
```

指定函数 `F7` 的另一种有效方法是使用函数应用表示法；只需为每个定义域元素指定其值域元素是什么：
```
F7(A) = 2
F7(C) = 2
F7(D) = 4
F7(E) = 4
```

你可能从过去的数学课上已经熟悉了这种表示法。这里是一个典型函数 `F` 的规范示例：`F(x) := 3x2 - 2x + 1`

给定函数 `F`，你会写像 `F(2)` 这样的表达式，在这个例子中它的求值结果为 9（三乘以四，减去二乘以二，加上一）。

同一个函数 `F` 的集合论规范如下（假设我们想在集合 **Z** 上定义此函数并映射到集合 **Z** 内）：
```
F := { (x; 3x2 - 2x + 1) | x∈Z }
```

这里我们使用了代入法以集合论的方式指定函数 `F`。

#### 恒等函数

如果集合 `A` 是函数 `f` 的定义域，并且对于集合 `A` 中的所有元素 `x`，`f` 的 `x` 都等于 `x`，则函数 `f` 称为**在 `A` 上的恒等函数**。我们使用记法 `id(A)` 来表示在 `A` 上的恒等函数。

这里有几个例子：
```
id({1,2,3}) = { (1;1), (2;2), (3;3) }
id({deptno}) = { (deptno;deptno) }
id({course,startdate}) = { (course;course), (startdate;startdate) }
```

#### 函数的子集

在本节中，我们将简要研究函数的子集。因为函数是一个集合，所以你可以构造一个函数的子集。列表 4-4 提供了前一节介绍的函数 `F2 := { (X;2), (Y;1), (Z;2) }` 的每一个（真）子集。

**列表 4-4. 函数 F2 的所有真子集**

```
{ (X;2), (Y;1) }
{ (X;2), (Z;2) }
{ (Y;1), (Z;2) }
{ (X;2) }
{ (Y;1) }
{ (Z;2) }
∅
```

如你所见，`F2` 的每一个真子集本身又是一个函数。确实，这对于每个函数都成立。对于一个给定的函数 `F`，`F` 的每一个真子集也将是一个函数。

#### 小结

至此，集合论函数的介绍结束。因为函数是一个重要的概念，将在本书的其余部分被广泛使用，我们在此做一个简要总结。
*   函数是一个二元关系，因此是一个**有序对的集合**。
*   在函数中，**每个第一坐标只出现一次**。
*   你可以使用一个函数来指定数据库设计的某个部分。

本章的标题被特意选为“关系与函数”，因为我们认为你应该被介绍给（至少是）二元关系的数学概念。

`n-元`关系（即不仅仅是二元关系）这一更一般的概念是由 E. F. Codd 引入数据库领域的一个基石概念。二元关系的概念足以满足本书中将发展的用于指定数据库设计的数学方法的需要。其他教科书处理——并在其方法中使用——`n-元`关系的概念。它们也以不同于本书处理二元关系特例（即函数）的方式来使用 `n-元`关系。为了避免混淆，本书从现在开始将明确不再使用*二元关系*或*关系*这些术语。

#### 函数上的操作

因为函数是集合，所以你可以应用集合运算符——并集、交集和差集——以函数作为其操作数。本节将探讨这些集合运算符在函数上的应用。

##### 并集、交集与差集

数学家总是对运算符的**闭包性质**感兴趣。在本章的上下文中，我们问自己：在集合上封闭的并集、交集和差集运算符，在函数上是否也是封闭的？换句话说，当你对两个函数应用这些运算符时，结果是否仍然是另一个函数？

看下面的例子，它引入了两个函数 `G1` 和 `G2`，并展示了这两个函数的并集的结果：
```
G1 := { (a;1), (b;2), (c;3) }
G2 := { (a;2), (d;4) }
G1 ∪ G2 = { (a;1), (b;2), (c;3), (a;2), (d;4) }
```

这个例子说明了两个函数的并集不一定产生一个函数，因为函数 `G1` 和 `G2` 并集的结果包含了两个具有相同第一坐标的**不同**有序对：`(a;1)` 和 `(a;2)`。

再看另一个例子：
```
G3 := { (empno;999), (ename;'Toon Koppelaars'), (born;'14-feb-1965') }
G4 := { (empno;999), (occupation;'IT architect') }
G3∪G4 = { (empno;999), (ename;'Toon Koppelaars'),
           (born;'14-feb-1965'), (occupation;'IT architect') }
```
