# 关系与函数

正如你可能在设计中所使用的那样，一种普遍的设计技术被称为实体关系图（ERD）。“关系模型”中的“关系”一词与 ERD 中的 R 没有关系。它指的是（n 元）关系的*数学*概念，这与 ERD 中的“关系”概念完全不同。前者是一个数学——集合论——概念；后者最终映射到数据完整性约束的某些情况（粗略地说）。

在本章中，你将被介绍关系——E. F. Codd 所使用含义——的集合论概念。更具体地说，你将被介绍一种特定类型的关系：*二元*关系。我们将在本书的第 2 部分处理 ERD 意义上的关系——作为数据完整性约束。

#### 有序对与笛卡尔积回顾

让我们先回顾一下第 2 章的内容。该章以关于有序对和笛卡尔积的以下两个定义结束：

- 一个*有序对*是一对元素，也称为有序对的*坐标*。符号是 `(a;b)`。
- 两个给定集合（比如 `A` 和 `B`）的*笛卡尔积*定义如下：`A×B = { (a;b) | a∈A ∧ b∈B }`

这个定义表明，两个给定集合的笛卡尔积的结果实际上是一个有序对的集合。它包含了来自第一个集合和第二个集合的所有可能的元素组合。

列表 4-1 用一个例子说明了笛卡尔积。

```
A := {X,Y,Z}
B := {1,2}
A×B = { (X;1), (X;2), (Y;1), (Y;2), (Z;1), (Z;2) }
```

第 2 章的同一部分还列出了以下性质：
`a≠b ⇒ (a;b)≠(b;a)`
`(a;b)=(c;d) ⇔ (a=c ∧ b=d)`
`A×B=B×A ⇔ A=B`

这些性质说明了以下事实：

- 有序对的坐标顺序很重要。
- 笛卡尔积不是交换算子。

这使我们能够在下一节中引入二元关系的数学概念。

#### 二元关系

从集合 `A` 到集合 `B` 的*二元关系*被定义为笛卡尔积 `A×B` 的*一个子集*。它被称为*二元*关系，因为它处理两个集合（`A` 和 `B`）。让我们从一个小例子开始，基于列表 4-1 中的相同两个集合：

```
A := {X,Y,Z}
B := {1,2}
```

看看这里显示的集合 `R1`：

```
R1 := { (X;1), (X;2), (Y;1), (Z;2) }
```

请注意，集合 `R1` 并不包含来自集合 `A` 和集合 `B` 的所有可能的元素组合；集合 `R1` 只包含四个有序对作为其元素。每个有序对也是集合 `A` 和 `B` 的笛卡尔积的一个元素，如列表 4-1 所示。形式上，这可以表示如下：

`( ∀p∈R1: p∈A×B )`

这使得 `R1` 成为集合 `A` 和 `B` 的笛卡尔积的子集，因此是从集合 `A` 到集合 `B` 的一个二元关系。定义 4-1 说明了定义二元关系的另一种方式，并使用了第 2 章引入的重要幂集算子。

**定义 4-1：二元关系**
“`R` 是从集合 `A` 到集合 `B` 的二元关系” ⇔ `R∈℘(A×B)`

给定集合的幂集包含该集合的所有可能子集。在前面的情况下，幂集包含集合 `A` 和 `B` 的笛卡尔积的所有可能子集。因此，该幂集的每个元素都是从集合 `A` 到集合 `B` 的二元关系。

**注意** 在示例关系 `R1` 中，每个 `A` 值都出现在有序对中，每个 `B` 值也是如此。
这当然并不总是成立。从 `A` 到 `B` 还有其他二元关系不是这种情况。

图 4-1 将二元关系 `R1` 显示为图片，其中四个箭头代表有序对。

**图 4-1. 二元关系 `R1` 的图示**

从二元关系 `R` 的图片中，你可以立即看到，集合 `A` 的元素 `X` 发出了两个箭头，而元素 `Y` 和 `Z` 各发出一个箭头。你也可以立即看到集合 `B` 的元素 `1` 和 `2` 都有两个箭头到达。在下一节中，为了得到函数的定义，将对每个元素发出的箭头数量施加限制。

**注意** 有些教科书可能不将二元关系声明为*从*集合 `A` *到*集合 `B`，而是声明为*在*集合 `A` *上*和*在*集合 `B` *上的。

#### 函数

*函数*是一种二元关系，它没有两个元素具有相同的第一坐标；或者换句话说，在函数中，两个不同的有序对总是具有不同的第一坐标。从图片的角度来看，这限制了左侧集合的元素发出的箭头数量最多为一个。

此外，如果左侧集合中的每个元素都恰好有一个箭头发出，那么我们说该二元关系是该集合（左侧）上的函数。

定义 4-2 说明了如何正式定义这个额外的限制。它采用了第 2 章介绍的 `π` 算子（它选择有序对的第一个坐标）。

**定义 4-2：函数**
“`F` 是定义在集合 `A` 上、取值于集合 `B` 的函数” ⇔
`F∈℘(A×B) ∧ ( ∀p1,p2∈F: p1≠p2 ⇒ π(p1)≠π(p2) ) ∧ {π(p) | p∈F } = A`

定义 4-2 中等价式右侧的第二个合取项构成了前面提到的额外限制：对于函数 `F` 中的所有有序对 `p1` 和 `p2`，如果 `p1` 和 `p2` 是不同的对，那么它们应该具有不同的第一坐标。换句话说（使用第 1 章练习 4a 的结果），如果 `p1` 和 `p2` 具有相同的第一坐标，那么它们必须是同一个对。最后一个合取项指出，为了使 `F` 是*定义在* `A` 上的函数，`A` 的每个元素都必须作为第一坐标出现。

大多数时候，我们感兴趣的是函数的第一坐标来自集合 `A`（而远非第二坐标来自集合 `B`）。我们不再说“`F` 是定义在 `A` 上、取值于 `B` 的函数”，而是说“`F` 是定义在 `A` 上的函数”。

现在看下面的例子：

```
F1 := { (X;1), (X;2), (Y;1), (Z;2) }
F2 := { (X;2), (Y;1), (Z;2) }
F3 := { (X;1), (Y;1), (X;1), (Z;1) }
F4 := { (Z;2) }
F5 := ∅
F6 := { (empno;126), (ename;'Lex de Haan'), (born;'11-aug-1954') }
```

第一个例子 `F1` 不是函数，因为它包含两个不同的对——`(X;1)` 和 `(X;2)`——它们具有相同的第一坐标 `X`。第二个二元关系 `F2` 确实是一个函数：没有两个不同的对具有相同的第一坐标。第三个 `F3` 也是一个函数。尽管列举了两个具有相同第一坐标的对（第一个和第三个对），但它们实际上是同一个对。其中一个对无需列举，可以省略而不改变集合 `F3`。示例 `F4` 也是一个函数，空集（`F5`）确实也是一个函数。它是任意两个集合的笛卡尔积的子集（空集是*每个*集合的子集）。定义 4-2 中给出的额外限制也成立。你可能需要重新审视表 3-2 中的第二条重写规则。最后一个示例二元关系 `F6` 也是一个函数。通过这个例子，你可以开始看到如何（作为构建块）使用函数来指定数据库设计。我们将在“集合函数”一节的末尾回到这一点。
