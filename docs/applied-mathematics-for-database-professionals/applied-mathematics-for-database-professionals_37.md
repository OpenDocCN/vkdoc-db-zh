# 元组、表和数据库谓词

我们之前提到过，作为给定表元素的每个元组在该表内是唯一的。使元组在表内唯一的属性是它的*值*，而不是它的属性。通常，不需要所有的属性值就能唯一地标识表中的一个元组；属性值的一个真子集就足够了。对于某个表，可以通过一种称为*唯一标识*谓词的特殊表谓词来陈述此属性。唯一标识谓词是定义 6-1 中定义的表谓词的一种。

###### 定义 6-1：唯一标识谓词
设 `T` 是一个表，并设 `A` 是 `T` 的标题的一个给定非空子集。一个表谓词是唯一标识谓词，如果它具有以下形式（或可被重写为此形式）：

```
( ∀t1,t2∈T: t1↓A=t2↓A ⇒ t1=t2 )
```

我们称 *`A` 在 `T` 中是唯一标识的*。

唯一标识谓词表明，如果表中的任意两个元组在给定属性子集上具有相同的属性值，那么它们必须是同一个元组。使用本书第一部分介绍的重写规则，您也可以用列表 6-9 中列出的形式来表达唯一标识谓词。

###### 列表 6-9：唯一标识谓词的其他表现形式
```
( ∀t1,t2∈T: t1≠t2 ⇒ t1↓A≠t2↓A )
( ∀t1,t2∈T: t1=t2 ∨ t1↓A≠t2↓A )
¬( ∃t1,t2∈T: t1≠t2 ∧ t1↓A=t2↓A )
```

列出的第一个形式可能是唯一标识谓词更直观的正式版本；表中的两个不同元组必须在给定的属性子集上有所不同。列表 6-7 中的谓词 `P11` 就是以这种形式编写的；它指出 `{dname, loc}` 在表 `T` 中是唯一标识的。同一列表中的谓词 `P12` 也是一个唯一标识谓词。它以列表 6-9 中显示的第三种形式编写，指出 `{deptno}` 在表 `T` 中是唯一标识的。列表 6-3 中的第一个谓词也使用了这种形式。

给定集合 `A` 在表 `T`（标题为 `H`）中是唯一标识的，那么 `H` 的每个作为 `A` 的真*超集*的子集当然在表 `T` 中也是唯一标识的。在唯一标识谓词中，您希望指定一个在表 `T` 中是唯一标识的、标题 `H` 的*不可约*子集 `A`。

> **注意**：一个表中可能有多个这样的不可约子集是唯一标识的。
>
> 列表 6-7 显示了两个不同的子集——`{deptno}` 和 `{dname, loc}`——它们都被认为是在部门表中唯一标识的。

如果您选择一个可约的 `H` 子集，那么得到的唯一标识谓词就太弱了。最弱的唯一标识谓词是 `T` 的标题（最大的可能超集）在表 `T` 中是唯一标识的。实际上，这对表 `T` 允许的元组集合根本没有增加任何约束；正如本节开头提到的，我们已经知道元组在表内是唯一的。

您也可以通过更正式的方法得出这个结论。看看如果将 `T` 的标题代入唯一标识谓词模式中的集合 `A` 会发生什么：

```
( ∀t1,t2∈T: t1↓dom(t1)=t2↓dom(t2) ⇒ t1=t2 )
```

注意，如果 `t` 是 `T` 的一个元素，那么您可以通过表达式 `dom(t)`（元组 `t` 的域）来引用 `T` 的标题。使用重写规则 `t↓dom(t) = t`（一个元组对其所有属性的限制与该元组本身相同），我们可以将前面的谓词重写为：

```
( ∀t1,t2∈T: t1=t2 ⇒ t1=t2 )
```

该谓词现在是一个重言式；无论 `T` 的值如何，它总是为真。因此，它根本没有以任何方式约束 `T`。

> **注意**：我们在本节中刻意不讨论*键*。在第 7 章中，我们将为数据库设计中的每个表结构定义该表结构的可接受表集合——称为*表宇宙*。在定义表宇宙时，我们将确保每个表（在宇宙中）满足一个或多个唯一标识谓词。然后我们说，一个唯一标识属性集合构成了该表宇宙中的一个*键*。

###### 子集要求谓词

您在数据库设计中捕获的不同表结构并非全部相互独立。这些表结构中的许多*对*具有一种称为*子集要求*的关系。定义 6-2 定义了子集要求谓词的概念。此定义中谓词的结构说明了它为何被称为子集要求。

###### 定义 6-2：子集要求谓词
设 `T1` 和 `T2` 是表，并设 `A` 是 `T1` 的标题的一个给定子集，同时也是 `T2` 的标题的一个子集。我们称一个数据库谓词为子集要求谓词，如果它具有以下形式：

```
{ t1↓A | t1∈T1 } ⊆ { t2↓A | t2∈T2 }
```

我们称 *`A` 在 `T1` 中引用 `T2` 中的 `A`*。

让我们用表 `EMP1` 和 `DEP1` 来演示这一点（见图 6-2 和图 6-3）。如果我们将 `EMP1` 代入 `T1`，`DEP1` 代入 `T2`，`{deptno}` 代入 `A`，我们最终得到以下表达式：

```
{ t1↓{deptno} | t1∈EMP1 } ⊆ { t2↓{deptno} | t2∈DEP1 }
```

这将计算为以下表达式：

```
{ {(deptno;10)}, {(deptno;12)} } ⊆
{ {(deptno;10)}, {(deptno;11)}, {(deptno;12)}, {(deptno;13)} }
```

这是一个真命题；`EMP1` 中的每个部门编号也出现在 `DEP1` 中。

如果 `A` 的基数等于 1（即 `A` 表示一个单元素集），那么我们通常使用函数应用而不是函数限制来指定子集要求谓词。

使用前面的例子，我们会这样写：

```
{ t1(deptno) | t1∈EMP1 } ⊆ { t2(deptno) | t2∈DEP1 }
```

这个子集要求类似于列表 6-8 中的谓词 `P13`。

子集要求谓词通常伴随着一个唯一标识谓词。

使用定义 6-2 中的名称，您经常会遇到“`A` 在 `T2` 中是唯一标识的；”在上面的例子中 `{deptno}` 确实在 `DEP1` 中是唯一标识的。这不是强制性的；第 7 章中将给出的示例数据库设计实际上有一个子集要求，其中伴随的唯一标识谓词是缺失的。

当使用函数限制来指定子集要求时，有时必须应用函数组合来重命名所涉及的一个表中的一个或多个属性。

让我们通过一个例子来说明这一点。看一下图 6-5 中显示的表 `DEP2`，它与表 `DEP1` 非常相似。

###### 图 6-5：示例表 DEP2
表 `DEP2` 与表 `DEP1` 的区别在于，用于保存部门编号的属性现在命名为 `dno` 而不是 `deptno`。您可以使用函数应用来表达从 `EMP1`（见图 6-2）到 `DEP2` 的子集要求，如下所示：

```
{ e(deptno) | e∈EMP1 } ⊆ { d(dno) | d∈DEP2 }
```

但是，如果您想使用函数限制来指定此子集要求，则必须首先将表 `EMP1` 中的 `deptno` 属性重命名为 `dno`，或者将表 `DEP2` 中的 `dno` 属性重命名为 `deptno`。以下是此子集要求的规范，其中属性重命名应用于 `EMP1` 元组：

```
{ e◊{(dno;deptno)} | e∈EMP1 } ⊆ { d↓{dno} | d∈DEP2 }
```

在此表达式中，函数 `{(dno;deptno)}` 表示一个属性重命名函数。它将表 `EMP1` 中的元组的 `deptno` 属性重命名为 `dno`。

> **注意**：在这种情况下，重命名函数也执行了元组限制（参见第 4 章“函数组合”一节中的注释）。
