# 函数集合理论

###### 函数的并集与兼容性

函数 `G3` 和 `G4` 的并集结果仍然是一个函数。这是因为 `G3(empno) = G4(empno)`，并且 `empno` 是两个函数共有的唯一定义域元素。在并集结果中，元素 `(empno,999)` 不会重复出现两次。

现在应该不难理解两个函数为了使其并集仍构成一个函数所需具备的一般性质。这个性质被称为两个函数的*兼容性*或*可连接性*。定义 4-3 正式规定了这一特定性质。

**定义 4-3：兼容（可连接）函数**
“函数 `F` 与函数 `G` 兼容” ⇔
```
( ∀c∈(dom(F)∩dom(G)): F(c)=G(c) )
```
当且仅当对于两个函数共有的每一个第一坐标（即，可以从两个函数的定义域的交集中选取的坐标），相应的第二坐标匹配时，这两个函数才是兼容的。只有当两个函数兼容时，它们的并集才会产生另一个函数。

> 注意：定义 4-3 认为两个没有共同第一坐标的函数总是兼容的。如下例所示，这样两个函数的并集确实是一个函数：
> ```
> G5 := { (deptno,1), (dname,'Natural Join'), (location,'Utrecht') }
> G6 := { (invoice,'inv123'), (amount,3500) }
> G5 ∪ G6 = { (deptno,1), (dname,'Natural Join'), (location,'Utrecht'), (invoice,'abc123'), (amount,3500) }
> ```
> 如果两个函数没有共同的第一坐标，那么它们的并集就不会包含两个具有相同第一坐标的有序对。实际上，对于两个没有共同第一坐标的函数，这个性质总是成立的。然而，在实践中，这样的并集可能是没有意义的。

###### 函数的交集与差集

现在来看两个函数的交集，这里重用示例函数 `G1` 到 `G6`：
```
G1 ∩ G2 = ∅
G3 ∩ G4 = { (empno,999) }
G5 ∩ G6 = ∅
```
函数 `G1` 和 `G2` 的交集是空集；没有有序对同时是 `G1` 和 `G2` 的元素。函数 `G3` 和 `G4` 的交集正是那个第一坐标出现在 `dom(G3)` 和 `dom(G4)` 的交集中（坐标 `empno`）且满足性质 `G3(empno) = G4(empno)` 的单元素有序对。最后，函数 `G5` 和 `G6` 的交集是空集；尽管这两个函数是兼容的，但它们没有共同的第一坐标，更不用说有一个有序对了。

两个函数的交集结果总是一个函数。一个简单的理解方式是，任何两个给定*集合*的交集总是这两个给定集合的子集；设 `A` 和 `B` 是两个集合，则 `(A ∩ B) ⊆ A` 且 `(A ∩ B) ⊆ B`。两个函数的交集将是这两个函数的子集，因此是一个函数（回顾“函数的子集”一节，及清单 4-4）。

两个函数的差集总是一个函数。这与两个函数交集的推理相同。集合 `A` 和集合 `B` 的差集——`A` 减 `B`——总是集合 `A` 的子集。

##### 函数的限制

在本书的第 2 部分，我们经常会对给定函数的一个子集感兴趣。我们只想考虑那些第一坐标可以从函数定义域的某个给定子集中选取的有序对。这引出了*函数的限制*这一概念。定义 4-4 定义了函数的限制这一概念。

**定义 4-4：函数的限制**
设 `F` 是一个函数，`A` 是一个集合。函数 `F` 在集合 `A` 上的限制；记作 `F↓A`，定义如下：
```
F↓A := { p | p∈F ∧ π₁(p)∈A }
```
函数 `F` 在集合 `A` 上的限制是 `F` 的一个子集，该子集只包含那些第一坐标是集合 `A` 元素的有序对。让我们用清单 4-5 中的几个例子来说明这一点。其中重用了前面提到的函数 `G5`。

**清单 4-5：函数限制示例**
```
{ (X,2), (Y,1) }↓{X} = { (X,2) }
{ (X,2), (Z,2) }↓{Z} = { (Z,2) }
{ (X,2), (Z,2), (Y,1) }↓{X,Y,Z} = { (X,2), (Z,2), (Y,1) }
G5↓{dname,location} = { (dname,'Natural Join'), (location,'Utrecht') }
G5↓∅ = ∅
```
注意，函数的限制*总是*会产生另一个函数；如果你在空集上进行限制，结果是空函数。

有了这个限制运算符，你就能给出两个函数兼容性的另一种定义；即，为了使两个函数的并集产生另一个函数，这两个函数应具备的性质。见定义 4-5。

**定义 4-5：函数兼容性（可连接性）的替代定义（使用限制）**
“函数 `F` 与函数 `G` 兼容” ⇔ `F↓(dom(F)∩dom(G)) = G↓(dom(F)∩dom(G))`

当两个函数被限制到其定义域的交集时，如果得到的限制是相等的，那么这两个函数就是兼容的（可连接的）。

##### 集合函数

在本书的第 2 部分中，我们将需要的另一个概念是*集合函数*的概念。集合函数将被用作*数据库全域*定义的基础：这是在第 2 部分将介绍的数据库设计的形式化规范。

如你所知，函数是一个有序对的集合，而有序对是一个顺序很重要的两个元素的对。

> **注意**：对于允许作为有序对坐标出现的元素类型，没有施加任何限制。它们可以具有任意的复杂性。例如，坐标可以是集合。

集合函数是一种函数，其中每一个有序对都以一个集合作为其第二坐标。定义 4-6 正式定义了这个概念。

**定义 4-6：集合函数**
“`F` 是一个集合函数” ⇔ `F` 是一个函数，并且 `(∀c∈dom(F): F(c) 是一个集合)`

从这个定义可以看出，集合函数是一种函数，其中每个值域元素都是一个集合（可能是空集）。清单 4-6 提供了集合函数的各种示例。

**清单 4-6：集合函数示例**
```
H1 := { (X,{1,2,3,4,5}), (Y,{1,2,3,4}) }
H2 := { (X,{1,2}), (Z,{3,4}) }
H3 := { (company,{'Natural Join','Centraal Boekhuis','Oracle','Remmen & De Brock'}), (location,{'Utrecht','Culemborg','De Meern','Eindhoven'}) }
H4 := { (empno,[1..99]), (ename,varchar(10)), (born,date), (job,{'CLERK','SALESMAN','TRAINER','MANAGER','PRESIDENT'}), (salary,[1000..4999]) }
```
第一个例子 `H1` 是一个函数（没有第一坐标出现超过一次）。它是一个集合函数，因为 `H1(X)` 和 `H1(Y)` 都求值为集合值；分别是 `{1,2,3,4,5}` 和 `{1,2,3,4}`。第二个例子，函数 `H2`，也是一个集合函数，因为 `H2(X) = {1,2}` 且 `H2(Z) = {3,4}`。

##### 特征化

你可以认为集合函数 `H3` 通过其有序对的第一坐标枚举了一个公司的两个方面：公司名称（坐标 `company`）和公司位置（坐标 `location`）。这样的第一坐标被称为*属性*。

附属于这些属性的是值集——作为有序对的第二坐标——可以被认为是代表了公司这些属性的可能值集合。同样，你可以认为集合函数 `H4` 表征了一名员工。注意 `H4` 的定义使用了一些在表 2-3 中引入的集合规范简写形式。
