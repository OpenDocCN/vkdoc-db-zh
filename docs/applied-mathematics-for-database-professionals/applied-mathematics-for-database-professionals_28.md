# 4 关系与函数

##### 特征描述

当使用一个集合函数来描述现实世界的某物时，它被称为一个*特征描述*，方法是列出相关的属性以及每个属性的可接受值的集合。相关属性是有序对的第一个坐标，可接受值集合是它们各自对应的第二个坐标。

我们将特征描述的有序对的第一个坐标称为该特征描述的*属性*，将第二个坐标称为相应属性的*属性值集合*。

如前所述，集合函数 `H4` 可被视为一个表示雇员属性的特征描述。在这种情况下，你只对表示每个雇员的编号（属性 `empno`）、姓名（属性 `ename`）、出生日期（属性 `born`）、职业（属性 `job`）和薪水（属性 `salary`）感兴趣。

###### 外部谓词

看待特征描述还有另一种方式。它们可以被视为刻画了现实世界中的一个谓词；特征描述引入了谓词的（名称）参数以及这些参数可以取值的相应值集合。例如，特征描述 `H4` 刻画了以下谓词：员工 `ENAME` 被分配员工编号 `EMPNO`，出生于日期 `BORN`，担任职位 `JOB`，并拥有每月 `SALARY` 美元的薪水。

这个谓词有五个参数（`ENAME`、`EMPNO`、`BORN`、`JOB`、`SALARY`），对应于 `H4` 中的五个第一坐标。该谓词是一个自然语言句子，描述了所涉及属性的含义和关联。在本书的第二部分，一个特征描述将作为表设计的基础。表设计的特征描述所刻画的谓词代表了用户对该表设计所理解的含义。这样的谓词被称为该表的*外部谓词*。

##### 集合函数的泛化乘积

介绍了集合函数后，我们现在可以建立集合函数的*泛化乘积*概念（乍一看，这常常被认为是一个复杂的概念）。泛化乘积以一个集合函数作为操作数，并产生一个新的*函数集合*。

泛化乘积运算符是第二个关键的垫脚石，仅次于第 2 章引入的幂集运算符。使用这两个集合运算符，我们将能够构造通常相当大的*值集合*、*元组*集合（此处引入）、*表*集合（在第 5 章引入）以及一个*数据库状态*集合（在第 5 章引入）。在第 7 章，我们将演示一种方法，该方法涉及将这些值集合定义为数据库设计的形式化规范的构建块。

一个集合函数 `F` 的泛化乘积，记作 `∏(F)`，生成一个包含所有*可能*函数的集合，这些函数与 `F` 具有相同的定义域，并且这些函数中的有序对的第二坐标*选自（即，是）在 `F` 中与相同第一坐标相关联的集合*。定义 4-7 正式定义了泛化乘积运算符。

###### 定义 4-7：集合函数的泛化乘积
设 `F` 为一个集合函数。`F` 的泛化乘积，记作 `∏(F)`，定义如下：

```
∏(F) = { f | f 是函数 ∧ dom(f) = dom(F) ∧ (∀c∈dom(f): f(c) ∈ F(c)) }
```

此定义中的最后一个合取项规定，由该运算符生成的每个函数，其第二坐标都是集合函数中相应集合（第二坐标）的元素。清单 4-7 给出了一些例子，可以快速说明当你将此运算符应用于集合函数时的效果。此清单中重用了上一节中的集合函数 `H2` 和 `H3`。

###### 清单 4-7. *泛化乘积的应用示例*
```
∏({ (a;{1,2,3}), (b;{4,5}) }) =
{ {(a;1), (b;4)}
, {(a;1), (b;5)}
, {(a;2), (b;4)}
, {(a;2), (b;5)}
, {(a;3), (b;4)}
, {(a;3), (b;5)} }

∏(H2) = { {(X;1), (Z;3)}
, {(X;1), (Z;4)}
, {(X;2), (Z;3)}
, {(X;2), (Z;4)} }

∏(H3) = { {(company;'Natural Join'), (location;'Utrecht')}
, {(company;'Natural Join'), (location;'Culemborg')}
, {(company;'Natural Join'), (location;'De Meern')}
, {(company;'Natural Join'), (location;'Eindhoven')}
, {(company;'Centraal Boekhuis'), (location;'Utrecht')}
, {(company;'Centraal Boekhuis'), (location;'Culemborg')}
, {(company;'Centraal Boekhuis'), (location;'De Meern')}
, {(company;'Centraal Boekhuis'), (location;'Eindhoven')}
, {(company;'Remmen & De Brock'), (location;'Utrecht')}
, {(company;'Remmen & De Brock'), (location;'Culemborg')}
, {(company;'Remmen & De Brock'), (location;'De Meern')}
, {(company;'Remmen & De Brock'), (location;'Eindhoven')}
, {(company;'Oracle'), (location;'Utrecht')}
, {(company;'Oracle'), (location;'Culemborg')}
, {(company;'Oracle'), (location;'De Meern')}
, {(company;'Oracle'), (location;'Eindhoven')} }
```

第一个示例将泛化乘积应用于集合函数 `{(a;{1,2,3}), (b;{4,5})}`。我们称这个集合函数为 `S1`。`S1` 的定义域是 `{a,b}`。对 `S1` 应用泛化乘积得到的*函数集合*包含六个元素，每个元素都是一个函数。第一个列举的函数是 `{(a;1), (b;4)}`。我们称这个函数为 `f1`。函数 `f1` 与集合函数 `S1` 具有相同的定义域：`dom(S1) = dom(f1) = {a,b}`。

还要注意以下命题成立：`f1(a)∈S1(a)` 和 `f1(b)∈S1(b)`。第一个命题求值为 `1∈{1,2,3}`，为真；第二个求值为 `4∈{4,5}`，也为真。这两个命题的合取对应于定义 4-7 中的全称量化。同样地，你可以验证第一个示例中列出的其他五个函数也符合定义 4-7。

第二个示例在结果集合中有四个函数。同样，你可以看到这些函数的每一个的定义域都等于 `{X,Z}`，这等同于 `dom(H2)`。

在第三个示例中，应用于特征描述 `H3` 的泛化乘积产生一个包含 16 个函数的集合。每个函数都可以被视为表示一个基于公司示例特征描述 `H3` 的实际可能公司。

我们将集合函数的泛化乘积的结果集中的函数称为*元组*。注意，这样的集合函数通常表示一个特征描述。元组的元素——有序对——被称为该元组的*属性-值对*。在关系数据库设计中，一个元组表示现实世界中的一个命题（参见边栏“封闭世界假设”）。

你看出清单 4-7 中结果集的基数是如何计算的吗？

```
#∏({ (a;{1,2,3}), (b;{4,5}) }) = #{1,2,3} * #{4,5} = 3 * 2 = 6
#∏(H2) = #H2(X) * #H2(Z) = #{1,2} * #{3,4} = 2 * 2 = 4
#∏(H3) = #H3(company) * #H3(location) = 4 * 4 = 16
```

集合函数的泛化乘积的基数等于该函数所有值域元素基数的乘积。

##### 封闭世界假设

特征描述的泛化乘积结果集中的每个元组都可以被视为表示现实世界中的一个命题。给定元组所表示的命题，是通过使用该元组的属性值来实例化由特征描述所刻画的外部谓词而形成的命题。



例如，元组 `{ (empno;100), (ename;'Smith'), (born;'14-feb-1965'), (job;'MANAGER'), (salary;5000) }` 表示这样一个命题：“员工 Smith 被分配员工编号 100，出生于 1965 年 2 月 14 日，担任经理职位，月薪为 5000 美元。”

数据库管理关系模型的其中一条原则是：如果一个给定的元组出现在表设计中，那么我们假设该元组所表示的命题在现实世界中当前是一个真命题。反之，如果一个本可以出现在表设计中的给定元组当前并未出现，那么我们假设该元组所表示的命题在现实世界中是一个假命题。

这一原则被称为“**封闭世界假设**”；数据库设计中所持有的元组描述了现实世界中所有且仅有的当前为真的命题。

##### 约束规范预览

回顾代码清单 4-6 中定义的函数 `H4`：

```
H4 = { (empno;[1..99])
      , (ename;varchar(10))
      , (born;date)
      , (job; {'CLERK','SALESMAN','TRAINER','MANAGER','PRESIDENT'})
      , (salary;[1000..4999]) }
```

你可以将 `∏(H4)` 视为一个集合，它包含了基于特征描述 `H4` 所给出的属性及相应属性值集的所有*可能*的员工表示——或者，如果你愿意，也可以称之为外部谓词的实例化。这个集合包含大量的元组，但它仍然具有有限的基数，因为集合函数 `H4` 的每个值域元素都是一个有限集。

> **注：** 为简化起见，我们假设 `varchar(n)` 集合简写仅使用字符 a 到 z（包括大小写），共计 52 个字符。`date` 的基数源于其在表 2-3 中给出的定义。

以下是这些值域元素的基数：
- `#[1..99] = 99`
- `#varchar(10) = 52¹⁰ + 52⁹ + 52⁸ + ... + 52¹ = 7516865509350965246`
- `#date = 3442447`
- `#{'CLERK','SALESMAN','TRAINER','MANAGER','PRESIDENT'} = 5`
- `#[1000..4999] = 4000`

在本书第二部分，我们将开发各种可以被视为特定变量类型的值集。对于元组变量，我们关注的不是包含所有*可能*元组（基于 `H4`）的集合，而是包含所有*可接受*元组（根据现实世界中的某些规则可接受）的集合；这是 `∏(H4)` 的某个特定*子集*。例如，以下规则可能适用于员工：
- `CLERK`（办事员）的薪资低于 2500。
- `PRESIDENT`（总裁）的薪资高于 4000。

你可以通过首先创建所有可能元组的集合，然后通过指定每个元组都应遵循的规则（作为谓词）来缩小该集合的范围，从而定义可接受元组的集合。实现这一点所需的所有要素现已介绍完毕。看一下以下集合（命名为 `t-emp`）的规范：

```
t-emp := { t | t∈∏(H4) ∧ ( t(job)='CLERK' ⇒ t(salary)<2500 )
                ∧ ( t(job)='PRESIDENT' ⇒ t(salary)>4000 ) }
```

在这个例子中，我们开始将第 1 章和第 3 章介绍的逻辑与第 2 章及本章介绍的集合论结合起来。集合 `t-emp` 是 `∏(H4)` 的子集；只有那些满足集合规范内给出的两个附加谓词的（可能）元组才属于集合 `t-emp`。这两个谓词是对那两条非形式化描述规则的形式化表示。

## 函数组合

在你最终能够进入本书第二部分并看到前四章介绍的所有数学概念的完整应用之前，还需要介绍最后一个概念——*函数组合*。你可以使用函数组合来*重命名*元组的属性；在使用*连接*（将在第二部分介绍）、并、交或差运算符时，你偶尔需要应用此技术。

函数组合是关于连续、依次应用两个函数。给定某个定义域元素，你首先应用一个函数，得到一个值域元素。然后，你将第二个函数应用到这个结果值域元素上。你可以在定义 4-8 中找到函数组合的形式化定义。

> **定义 4-8：函数组合** 设 `A`、`B` 和 `C` 为集合。设 `f` 是从 `A` 到 `B` 的函数。设 `g` 是从 `rng(f)` 到 `C` 的函数。
> 函数 `f` 和 `g` 的组合，记作 `g◊f`（读作“g of f”或“g after f”），定义如下：`g◊f := { (a; g(f(a)) | a∈dom(f) ∧ f(a)∈dom(g) }`

从定义中可以看出，两个函数的组合结果是另一个有序对集合。实际上，它产生的是另一个函数，因为 `g◊f` 的定义域等于 `dom(f)`，并且在组合 `g` after `f` 中，函数 `f` 的每个定义域元素只引入一个有序对。

让我们用一个例子来说明函数组合。代码清单 4-8 介绍了两个函数 `f` 和 `g`，并展示了将 `g` 应用于 `f` 之后（分三步推导）的结果。

**代码清单 4-8.** *示例函数 f 和 g，以及它们的函数组合 g after f*

```
f := { (num;empno), (name;ename), (occup;job) }
g := { (empno;999), (ename;'Smith'), (job;'Trainer') }
g◊f = { (num;g(f(num))), (name,g(f(name))), (occup;g(f(occup))) }
     = { (num;g(empno)), (name;g(ename)), (occup;g(job)) }
     = { (num;999), (name;'Smith'), (occup;'Trainer') }
```

这个例子具体展示了如何利用函数组合来重命名给定元组的属性（本例中是函数 `g`）。在此例中，`f` 是重命名函数，`g` 是其属性根据 `f` 进行重命名的元组。

应用第一个函数 (`f`) 会对第一坐标进行重命名（属性 `empno` 重命名为 `num`，`ename` 重命名为 `name`，`job` 重命名为 `occup`），而应用第二个函数 (`g`) 则会将第二坐标重新附加到重命名后的属性上。

> **注：** 如果重命名函数所包含的属性少于被重命名属性的元组，那么实际上你同时也在进行函数限制。例如，如果 `f`（来自代码清单 4-8）等于 `{(num;empno)}`，那么 `g◊f` 的结果是 `{(num;999)}`。

你也可以画图表示函数组合。图 4-3 展示了代码清单 4-8 中介绍的函数 `f` 和 `g` 的函数组合示意图。

**图 4-3.** *函数组合 g◊f*

你可以从这个图中直接看出，坐标 `num` 映射到值域元素 `999`，坐标 `name` 映射到 `'Smith'`，坐标 `occup` 映射到 `'Trainer'`。



#### 章节摘要

本节以项目符号列表的形式提供了本章的摘要。你可以在继续下一节的练习之前，用它来检查自己对本章所介绍的各种概念的理解。

- 从集合`A`到集合`B`的`二元关系`被定义为笛卡尔积`A×B`的一个子集。
形式化地，“`R`是从集合`A`到集合`B`的二元关系” ⇔ `R∈℘(A×B)`。
- `函数`是一种二元关系，其中没有第一个坐标出现超过一次。
形式化地，“`F`是从`A`到`B`的函数” ⇔ `F∈℘(A×B) ∧ ( ∀p1,p2∈F: p1≠p2 ⇒ π(p1)≠π(p2) ) ∧ { π(p) | p∈F } = A`。
- 函数`F`的所有第一个坐标的集合被称为函数`F`的`定义域`。
记法为`dom(F)`。因此，如果`F`是从`A`到`B`的函数，那么`dom(F) = A`。
- 函数`F`的所有第二个坐标的集合被称为函数`F`的`值域`。
记法为`rng(F)`。因此，如果`F`是从`A`到`B`的函数，那么`rng(F) ⊆ B`。
- 如果`F`是一个函数，且`x`是`dom(F)`的一个元素，那么`F(x)`（`F`应用于`x`）表示在函数`F`中与`x`配对的值域元素。
- 函数的每一个子集都是一个函数。
- 两个函数是`相容的`（或`可联合的`），当且仅当对于两个函数共有的每一个第一个坐标，相应的第二个坐标都匹配。只有当两个函数相容时，这两个函数的并集才会是一个函数。
- 函数`F`对集合`A`的`限制`是`F`的一个子集，该子集只包含那些第一个坐标是集合`A`元素的序对。记法为`F↓A`。
- `集合函数`是一种函数，其中每个元素（一个序对）的第二个坐标是一个集合。
- 如果一个集合函数被用来表示现实世界中我们想要捕捉的某事物的各个方面，那么我们将这样的集合函数称为`特征化描述`。特征化描述的定义域元素被称为其`属性`。值域元素被称为这些属性的`属性值集`。
- 集合函数`F`的`广义积`，记法为`∏(F)`，生成一个集合，该集合包含所有与`F`定义域相同，并且其序对的第二个坐标是从`F`中相同第一个坐标所关联的相应集合中选出的函数。
- 特征化描述的广义积中的元素可以被认为代表了现实世界中的命题，并被称为`元组`。
- 元组中序对的第一个坐标被称为该元组的`属性`。
- 两个给定函数`f`和`g`的`函数复合`，记法为`g◊f`（`g`在`f`之后），可用于重命名元组的属性。这里，`f`是重命名函数，`g`是其属性根据`f`进行重命名的元组。
