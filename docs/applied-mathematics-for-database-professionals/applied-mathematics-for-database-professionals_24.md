# 第四章 关系与函数

第二章奠定了集合论的基础。本章第四章将在此基础之上，引入更多将在本书第二部分中用作工具、以形式化定义数据库设计（包括所有约束规范）的集合论概念。

在“二元关系”一节中，我们将回顾有序对和笛卡尔积，以引入两个集合之间二元关系的概念。正如你将看到的，这种二元关系实际上是两个集合笛卡尔积的一个子集。

随后，“函数”一节介绍了函数这一重要概念。函数是一种特殊类型的二元关系——它符合特定的属性。在本节中，我们还将围绕（集合论中的）函数概念建立一些术语。

“函数上的操作”一节回顾了并、交、差这些集合并算符，并更仔细地探讨了当这些算符应用于函数作为其操作数时的行为。你还将被介绍到一个新的（二元）集合并算符，称为函数的限制。

接着，本章继续介绍另一个重要概念，我们称之为集函数。集函数持有一种特殊类型的有序对：其中对的第二个坐标总是一个集合。我们将揭示集函数与指定数据库设计之间的联系：它可以用来描述一个外部谓词。外部谓词描述了我们想要在数据库中表示的现实世界的某些东西。如果集函数用于此目的，我们将称其为描述。在同一小节中，你还将被介绍到一个新的单元集合并算符，称为广义积。广义积以一个集函数——或者更确切地说，一个描述——作为其操作数，并产生一个新的函数集合。广义积的概念是第二个关键的垫脚石（继第二部分中引入的幂集概念之后），当我们将在本书第二部分演示如何形式化定义数据库设计时，将会用到它。

本章最后以函数复合的概念作结，该概念将在第五章（表上的操作）中，当我们以形式化方式研究连接时被应用。

你会发现本章末尾有一个“章节摘要”，随后是“练习”部分。强烈建议你在继续学习其他章节之前，花足够时间完成这些练习。

#### 二元关系

在我们开始处理二元关系之前，请看第一章中的这段引文：*基于逻辑的数据管理之重要性由 E. F. (特德)·科德 (1923–2003) 在 1969 年预见，当时他在 IBM 研究报告“存储在大型数据银行中的关系的可导性、冗余性和一致性”中首次提出了他著名的关系模型。这项研究报告中引入的用于数据库管理的关系模型已被证明是一个有影响力的数据管理通用理论，并且仍然是他最令人难忘的成就。*

E. F. 科德在前面引文中其研究报告标题所使用的“关系”一词，创造了我们现在称为数据的关系模型的通用理论的名称。

> **注意：** 一个常见的误解是，“数据关系模型”中的“关系的”一词指的是可以存在于不同“实体类型”之间的“联系”（多对一、多对多等等）。
