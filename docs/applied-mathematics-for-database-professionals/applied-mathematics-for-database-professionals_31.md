## ■定义 5-1：表
如果`T`和`H`是集合，那么`“T 是 H 上的一个表” ⇔ ( ∀t∈T: t 是 H 上的一个函数 )`。

这个定义是通用的，因为它对集合`H`的元素没有施加任何限制。然而，在实践中，我们只对`H`是代表属性的名称集合的情况感兴趣。

表`T1`可被视为一个*零件表*。它是在`{partno, name, instock, price}`上的一个表，包含六个元组，每个元组代表一个不同零件的信息。它为每个这样的零件保存零件号、名称、库存数量和价格。

以下是更多例子：

`T2 := { { (X;2), (Y;1) }, { (Y;8), (X;0) }, { (Y;10), (X;5) } }`

`T3 := { { (partno;3), (name;'hammer') }, { (pno;4), (pname;'nail') } }`

`T4 := { { (empno;105), (ename;'Mrs. Sparks'), (born;'03-apr-1970') },{ (empno;202), (ename;'Mr. Tedesco') } }`

`T2`确实是一个表。它包含三个函数，所有这些函数共享定义域`{X,Y}`。它是一个在`{X,Y}`上的表。注意，对（在函数内部）的顺序无关紧要。`T3`不是一个表。它是一个函数集合；然而，第一个函数的定义域是`{partno, name}`，这与第二个函数的定义域`{pno, pname}`不同。同样，`T4`也不是一个表。它是一个函数集合；然而，第一个函数的定义域是`{empno, ename, born}`，这与第二个函数的定义域`{empno, ename}`不同。

表的一个元素是一个函数，每个这样的函数被特别称为一个`元组`。在第 4 章介绍表征的广义积时（参见“集合函数的广义积”一节），你们已经接触过这个术语。

表征的广义积实际上是一个表；它包含函数，所有这些函数共享相同的定义域。

表是一个集合，该集合的元素是元组。根据集合的定义，这意味着每个元组在该集合内都是唯一的；同一个表中不能出现重复的元组。

如果`T`是`H`上的一个表，那么`T`的任何真子集（包含的元组少于`T`）当然也是`H`上的一个表。

即使是空集`(∅)`也是一个表。事实上，根据定义 5-1（它量化表中的元素），空集是*任何集合上的表*。关于这一点，你可能需要回顾表 3-2 中的重写规则。

*这种表的正式定义使你能够以清晰、正式的方式处理表上的操作（本章稍后讨论）和数据完整性谓词（下一章讨论）。*

##### 表的构造

你可以通过将广义积应用于一个给定的集合函数来构造一个表。

让我们用一个例子来演示这一点。代码清单 5-2 展示了一个名为 `F1` 的集合函数。

##### **代码清单 5-2.** 集合函数 `F1`

```
F1 := { (X; {0,1,2})
      ,(Y; {0,1,2})
      ,(Z; {-1,0,1}) }
```

集合函数 `F1` 的广义积将得到一个包含二十七个函数的集合（三乘三乘三）。代码清单 5-3 展示了这一结果。

##### **代码清单 5-3.** 集合函数 `F1` 的广义积

```
∏(F1) :=
{ { (X;0), (Y;0), (Z;-1) }, { (X;0), (Y;0), (Z;0) }, { (X;0), (Y;0), (Z;1) }
, { (X;0), (Y;1), (Z;-1) }, { (X;0), (Y;1), (Z;0) }, { (X;0), (Y;1), (Z;1) }
, { (X;0), (Y;2), (Z;-1) }, { (X;0), (Y;2), (Z;0) }, { (X;0), (Y;2), (Z;1) }
, { (X;1), (Y;0), (Z;-1) }, { (X;1), (Y;0), (Z;0) }, { (X;1), (Y;0), (Z;1) }
, { (X;1), (Y;1), (Z;-1) }, { (X;1), (Y;1), (Z;0) }, { (X;1), (Y;1), (Z;1) }
, { (X;1), (Y;2), (Z;-1) }, { (X;1), (Y;2), (Z;0) }, { (X;1), (Y;2), (Z;1) }
, { (X;2), (Y;0), (Z;-1) }, { (X;2), (Y;0), (Z;0) }, { (X;2), (Y;0), (Z;1) }
, { (X;2), (Y;1), (Z;-1) }, { (X;2), (Y;1), (Z;0) }, { (X;2), (Y;1), (Z;1) }
, { (X;2), (Y;2), (Z;-1) }, { (X;2), (Y;2), (Z;0) }, { (X;2), (Y;2), (Z;1) } }
```

在这个结果集中，每个函数都有相同的定义域 `{X, Y, Z}`；因此，该结果集是一个定义在 `{X, Y, Z}` 上的表。现在来看下面这个更实际的例子。代码清单 5-4 展示了一个零件的特征描述。对于一个零件，我们感兴趣的属性是零件号（`partno`）、名称（`name`）、该零件的库存数量（`instock`）以及零件价格（`price`）。

##### **代码清单 5-4.** 零件的特征描述

```
chr_PART := { (partno; [1..999])
           ,(name; varchar(12))
           ,(instock; [0..99])
           ,(price; [1..500]) }
```

`chr_PART` 的广义积是一个相当大的表；它包含了使用 `chr_PART` 定义中给出的属性值集可以生成的所有*可能的*元组。请注意，本章开头介绍的表 `T1` 中的每一个元组都是 `∏(chr_PART)` 的一个元素。这使得 `T1` 成为 `∏(chr_PART)` 的一个子集。



**注意** `∏(chr_PART)` 的每一个子集——不仅仅是`T1`——都是在属性集 `{partno, name, instock, price}` 上的一个表。

事实上，表 `T1` 中的每个元组都是基于 `∏(chr_PART)` 定义的以下集合（名为 `T2`）中的一个元素：

```
T2 := { t | t∈∏(chr_PART) ∧ ( t(price) ≥ 20 ⇒ t(instock) ≤ 10 )
                    ∧ ( t(price) ≤ 5 ⇒ t(instock) ≥ 15 )
         }
```

在 `T2` 的定义中，引入了两个限定 `T2` 内容的谓词。第一个条件指出，如果一个零件的价格大于或等于 20，那么其库存数量应小于或等于 10。第二个条件指出，如果一个零件的价格小于或等于 5，那么其库存数量应大于或等于 15。表 `T2`——一个在 `{partno, name, instock, price}` 上的表——将包含且仅包含 `∏(chr_PART)` 中同时满足这两个条件的元组。请注意，由于 `∏(chr_PART)` 将包含许多违反其中一个或两个条件的元组，因此 `T2` 是 `∏(chr_PART)` 的一个真子集。由于 `T1` 的所有元组都符合这两个条件，因此 `T1` 是 `T2` 的一个子集。实际上，它是 `T2` 的一个真子集。

在第 7 章，我们将重新审视这种定义 `T2` 的方式；也就是说，取一个特征化（characterization）的广义乘积（generalized product）并“插入”额外的谓词。

## `数据库骨架`

要指定一个数据库状态，你不仅需要实际的表（如前面示例中的 `EMP1` 和 `DEP1`），还需要为表结构确定名称（如前面示例中的 `EMPLOYEE` 和 `DEPARTMENT`）。

你可能不会感到惊讶，数据库设计的正式规范（我们将在第 7 章演示）也包含了对所涉及的每个表设计的特征说明规范。当你为一个表设计指定特征说明时，你就选择了该表设计中涉及的属性名称。

一个 `数据库骨架` 将所有这些名称——表结构的名称和所涉及的属性名称——收集到一个单一的正式结构中。数据库骨架是一个集合函数，它为每个表结构包含一个有序对。第一个坐标引入表结构名称，第二个坐标引入所涉及的属性名称集合。

###### 清单 5-6. 数据库骨架 `SK1`

`SK1 :=`
`{ (EMPLOYEE; {empno,ename,job,sal,deptno} )`
`,(DEPARTMENT; {deptno,dname,loc,salbudget} ) }`

如你所见，集合函数 `SK1` 为员工/部门数据库设计中涉及的表结构引入了名称 `EMPLOYEE` 和 `DEPARTMENT`，并将相关属性的名称集合附加到它们之上。

> **注意** 你应该仔细选择数据库骨架引入的名称，因为它们不仅构成了你（数据库专业人员）和你的客户（用户）之间的词汇表，也是理解数据库设计含义（语义）的第一块垫脚石。在接下来的章节中，你将会看到数据完整性约束是理解数据库设计语义的另一块重要垫脚石。
