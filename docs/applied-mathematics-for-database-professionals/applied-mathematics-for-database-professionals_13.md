# 第 1 章：逻辑：导论

上一个例子的可读性无疑通过添加括号得到了提升。

对于我们目前已介绍的五种逻辑连接词，通常采用如表 1-3 所示的优先级规则（注意其中一些具有相同的优先级）。

**表 1-3.** 逻辑连接词优先级规则
| 优先级 | 连接词 |
|------|------------|
| ¬    |
| ∧, ∨ |
| ⇒, ⇔ |

这些优先级规则规定，例如，下面这个没有包含括号的复合谓词 `P ∧ Q ⇒ R`，实际上应被解释为 `(P ∧ Q) ⇒ R`——而不是解释为 `P ∧ (Q ⇒ R)`，后者是另一个不同的谓词（即具有不同的含义）。

当涉及相同优先级的连接词时，通常让它们将其操作数从右向左结合；这有时被称为结合律规则。这意味着谓词 `P ⇒ Q ⇒ R` 实际上应被解释为 `P ⇒ (Q ⇒ R)`。

考虑以下两个示例谓词，并留意括号的用法：
1. `¬(P ⇒ Q) ⇔ (P ∧ ¬Q)`
2. `((P ⇒ Q) ∧ (Q ⇒ R)) ⇒ (P ⇒ R)`

在第一个例子中，第一组括号是必需的——否则否定将仅作用于 `P`。第二组括号可以省略，因为 `∧` 的优先级高于 `⇔`。在第二个例子中，左侧最外层的括号是可选的，右侧的括号也是如此（由于结合律规则），但另外两组括号是必需的。

在本书后续部分，当一个谓词中涉及多个连接词时，我们将始终添加括号，而不依赖于优先级和结合律规则。

#### 真值表

真值表在逻辑学中主要用于建立基本的逻辑概念。你可以将它用于两个（相当不同的）基本目的：
* 定义逻辑连接词的含义
* 研究复合逻辑表达式的真值

通过使用真值研究复合逻辑表达式，你可以建立（发展）新的逻辑概念；本章末尾重写规则的发展就是一个很好的例子。一旦基本的逻辑概念建立起来，你就很少再需要使用真值表了。

每个真值表都由表头和表体组成。在定义逻辑连接词的含义时，最左侧的列标题列出与你要定义的连接词相关的变量，最右侧的列标题列出连接词本身。

当你研究复合逻辑表达式的真值时，最左侧的列标题列出复合表达式中涉及的变量，最右侧的列标题列出该复合表达式。你可以选择在中间增加额外的列，以帮助你逐步推导出最后一列中的逻辑表达式；这在研究复杂逻辑表达式时尤其有用。

看一下图 1-1。它展示了定义逻辑 AND 连接词含义的真值表。

**图 1-1.** 逻辑 AND 的真值表结构

真值表的表体总是包含一行，对应所涉及变量的每一种可能的真值组合。也就是说，如果你只有一个变量，你需要两行（一行对应 `TRUE`，一行对应 `FALSE`）；如果你有两个变量，你需要四行（`TRUE/TRUE`、`TRUE/FALSE`、`FALSE/TRUE`、`FALSE/FALSE`），依此类推。

**注意** 真值表中涉及的变量是命题变量。

¬P 为假当且仅当 P 为真。在另一种情况下，¬P 为真。

##### 表 1-5. 与（合取）的真值表

| P | Q | P ∧ Q |
| :--- | :--- | :--- |
| **T** | **T** | T |
| **T** | **F** | F |
| **F** | **T** | F |
| **F** | **F** | F |

`P ∧ Q` 为真当且仅当 `P` 和 `Q` 均为真。在所有其他情况下，`P ∧ Q` 为假。在合取式 `P ∧ Q` 中，`P` 和 `Q` 被称为*合取项*。

##### 表 1-6. 或（析取）的真值表

| P | Q | P ∨ Q |
| :--- | :--- | :--- |
| **T** | **T** | T |
| **T** | **F** | T |
| **F** | **T** | T |
| **F** | **F** | F |

`P ∨ Q` 为假当且仅当 `P` 和 `Q` 均为假。在析取式 `P ∨ Q` 中，`P` 和 `Q` 被称为*析取项*。

**注意** 此处定义的 OR 运算符称为*包含性或*。在自然语言中，我们使用“或”这个词时通常指包含性或。然而，我们有时也用“或”来表示所谓的*排他性或*（简写为 `eor`）。与包含性或的真值表（表 1-6）相比，排他性或的真值表仅在第一行不同；当 `P` 和 `Q` 均为真时，命题形式 `P eor Q` 为假。

排他性或的一个使用例子是这样的陈述：“你必须打扫你的房间，否则你就要早点上床睡觉。”显然，你不需要同时做打扫房间*和*早点上床睡觉这两件事。在本书中，我们将始终使用包含性或。

##### 表 1-7. 如果…那么（蕴涵）的真值表

| P | Q | P ⇒ Q |
| :--- | :--- | :--- |
| **T** | **T** | T |
| **T** | **F** | F |
| **F** | **T** | T |
| **F** | **F** | T |

`P ⇒ Q` 为假当且仅当 `P` 为真且 `Q` 为假。在所有其他情况下 `P ⇒ Q` 为真。在蕴涵式 `P ⇒ Q` 中，`P` 通常被称为*前件*、*假设*或*前提*，而 `Q` 被称为*后件*或*结论*。

##### 表 1-8. 当且仅当（等价）的真值表

| P | Q | P ⇔ Q |
| :--- | :--- | :--- |
| **T** | **T** | T |
| **T** | **F** | F |
| **F** | **T** | F |
| **F** | **F** | T |

逻辑等价无非是常规的 `=` 运算符在布尔值上的应用；`P ⇔ Q` 为真当且仅当 `P` 和 `Q` 具有相同的真值。表达等价的另一种常见方式是使用“必要且充分”这样的措辞。

请注意，否定（见表 1-4）是一元运算符；它只接受一个操作数。表 1-5 至表 1-8 中的其他连接词是二元运算符；它们接受两个操作数。

**注意** 其他书籍在对运算符进行分类时，有时会使用一元、二元和三元等术语，而不是一元、二元和三元。

#### 蕴涵

在现实生活中，蕴涵扮演着主要角色，因为生活充满了如果-那么的情境。数据库设计旨在代表现实生活的一部分。你将在本书的第二部分看到，蕴涵经常用于数据完整性约束的形式化规范中，这些约束在数据库设计中扮演着重要角色。因此，充分理解表 1-7 所给出的蕴涵连接词的形式定义非常重要。

可能最具“争议”的真值表条目是这一条：如果 `P` 为假而 `Q` 为真，则蕴涵 `P ⇒ Q` 为真。还记得本书引言中，我们描述了数学家如何创造自己的游戏，改变规则直到他们喜欢这个游戏吗？事实证明，前述的蕴涵定义在实践中比任何其他定义都更好用。思考蕴涵 `P ⇒ Q` 的一种方式如下：

* 这个蕴涵在 `P` 为真的情况下，说明了关于 `Q` 的真值。
* 如果 `P` *不*为真，该蕴涵就不“说明”任何事情；因此，`Q` 的真值无关紧要，该蕴涵被认为为真。

看看下面的命题：*图恩住在乌得勒支意味着莱克斯有两个女儿。*

你可以将其视为命题形式 `P ⇒ Q` 的一个实例，其中命题“图恩住在乌得勒支”被代入作为命题变量 `P` 的值，而命题“莱克斯有两个女儿”被代入作为变量 `Q` 的值。因为图恩并不住在乌得勒支，`Q` 的值甚至可以说是莱克斯有三个女儿（实际上有两个）；蕴涵 `P ⇒ Q` 仍然为真。

另一种看待这个问题的方式是：如果你相信一个谬误，你就会相信任何事情。

#### 谓词强度

蕴涵可以被认为对其作为操作数的两个谓词进行*排序*；它在两个操作数，即假设和结论之间，声明了一种顺序。这种顺序被称为谓词的*强度*。如果两个谓词——例如 `P` 和 `Q`——涉及一个蕴涵式 `P ⇒ Q`，则谓词 `P` 被认为是*强于*谓词 `Q`——或者 `Q` *弱于* `P`。

假设 `x` 是一个整型变量，看下面的蕴涵：`x > 42 ⇒ x > 0`

声明 `x` 大于 42 显然意味着 `x` 大于 0。无论你为 `x` 提供什么值，这个蕴涵都为真，并且语句 "`x > 42`" 被认为是强于语句 "`x > 0`"。

两个谓词，例如 `P` 和 `Q`，具有相等的强度，当且仅当 `P ⇒ Q` 和 `Q ⇒ P` 均为真。在这种情况下，`P` 和 `Q` 是等价的：`P ⇔ Q`。

**注意** 你并不总是能对于两个给定的谓词，例如 `P` 和 `Q`，指出哪一个更强。

如果 `P ⇒ Q` 不成立，并且 `Q ⇒ P` 也不成立，那么 `P` 和 `Q` 之间就没有顺序关系。在数学中，这种排序被称为*偏序*。

给定这种谓词的排序，你可以问自己是否存在一个谓词是所有谓词中最强的，同样，是否存在一个是最弱的？最强的谓词将是*蕴涵所有其他谓词*的那个。确实存在这样的谓词：`FALSE` 是最强的谓词，因为对于任何结论 `Q`，`FALSE ⇒ Q` 均为真。这源自定义蕴涵的表 1-7 中真值表的第三和第四项。

最弱的谓词将是*被所有其他谓词所蕴涵*的那个；`TRUE` 是*最弱的*谓词，因为对于任何假设 `P`，`P ⇒ TRUE` 均为真。这源自表 1-7 中真值表的第一和第三项。

#### 再深入一点

让我们稍微进一步探索一下逻辑连接词领域，只是为了有趣。前面的真值表展示了五个逻辑连接词。为什么是*五个*？我们总共能想出多少个具有不同含义的逻辑连接词？我们需要其中多少个？表达所有其他连接词所需的理论最小逻辑连接词集合是什么？

例如，相对容易看出，你可以定义四种不同的一元逻辑连接词和十六种不同的二元逻辑连接词。一元连接词的真值表有两行，二元连接词的真值表有四行；因为真值表的每个单元格都有两种选择（`TRUE` 和 `FALSE`），可能的总数分别是 2 * 2 = 4 和 2 * 2 * 2 * 2 = 16。因此，我们可以给所有这 20 个连接词命名，也选择符号来表示它们，并开始在我们的逻辑表达式中使用它们。显然，我们不这样做；记住所有这些运算符太困难了，我们的表达式会变得难以阅读，而且显然我们不需要所有的连接词。这引出了函数完备性的重要概念。

#### 函数完备性

一个给定的逻辑连接词集合是*真值函数完备的*，当且仅当所有可能的连接词都可以通过该集合中元素的适当组合来表达。
