# 集合论导论

## 集合的划分

*   划分的任意两个元素必须是*不相交*的。
*   划分`P`的*并集*必须等于原始集合`S`。

如果想用维恩图来理解，划分就是将原始集合分割成若干个非空、不重叠的部分，如图 2-5 所示。假设`S := {1,2,3,4,5,6,7,8,9}`且`P := {{1},{2,4},{3,6,7},{5},{8,9}}`。

那么，`S`中的每个元素都恰好出现在划分`P`的一个元素中。

**图 2-5.** *划分`P`的维恩图*

让我们看另一个简单的数值例子：

假设：`T := {1,2,3}`

那么集合`P1`到`P5`都是`T`所有可能的划分：
```
P1 := {{1},{2},{3}}
P2 := {{1,2},{3}}
P3 := {{1,3},{2}}
P4 := {{2,3},{1}}
P5 := {{1,2,3}}
```

请注意，最后一个例子表明，一般情况下`{S}`总是`S`的一个划分。总是如此吗？

#### 有序对与笛卡尔积

如果一个集合包含两个元素，这两个元素的顺序是无关紧要的。例如，集合`{1,2}`与集合`{2,1}`是相同的。另一方面，在几何学中，我们通常用`x`和`y`坐标来表示点，其中坐标的顺序很重要。如果交换一个给定点的`x`和`y`坐标（假设它们值不相同），你会得到一个不同的点。

##### 有序对

本节介绍有序对的概念，允许你对两个配对值的顺序进行区分。我们使用一种特殊的符号表示，如定义 2-10 所示。

**■定义 2-10：有序对** 一个有序对是一对元素，例如`a`和`b`，其中顺序很重要。符号表示为`(a;b)`。

有序对的概念在本书中很重要。这不仅是因为我们将使用有序对来引入笛卡尔乘积运算符；更重要的是，在第 4 章中，我们将需要用它们来引入二元关系和函数，这反过来又将用于表示*元组*（我们注意到这个词的发音与“couples”押韵）。

**■注** 通常使用逗号作为有序对元素的序列化运算符。一个有序对将被描述为`(a,b)`。然而，由于逗号已经被大量用于分隔枚举指定集合的元素，本书使用分号来分隔有序对的元素。这使得各种表达式更易于阅读。

有序对的两个元素也被称为该有序对的第一和第二*坐标*。由于有序对的坐标顺序很重要，因此成立以下两个性质：

`(a ≠ b) ⇔ ( (a;b) ≠ (b;a) )`

`( (a;b) = (c;d) ) ⇔ ( (a = c) ∧ (b = d) )`

前面等式中的第二个性质也称为“相等公理”。

使用德·摩根重写规则，你也可以如下指定此公理：
`( (a;b) ≠ (c;d) ) ⇔ ( (a ≠ c) ∨ (b ≠ d) )`

**■注** 就像集合中的元素一样，有序对中配对的两个值可以是任意复杂的。例如，`(A;{1,2,3})`是一个有效的有序对。这个对以`A`作为其第一坐标，以集合`{1,2,3}`作为其第二坐标。

我们引入运算符`π1`和`π2`来从有序对中检索第一和第二坐标。对于一个给定的有序对——例如`(a;b)`——第一坐标可以使用表达式`π1(a;b)`检索，结果为`a`；第二坐标可以通过表达式`π2(a;b)`检索，结果等于`b`。

##### 笛卡尔积

两个给定集合的笛卡尔积现在可以使用有序对来定义，如定义 2-11 所示。

**■定义 2-11：笛卡尔积** 两个给定集合`A`和`B`的笛卡尔积，记作`A×B`，等于`{ (a;b) | a∈A ∧ b∈B }`。

也就是说，笛卡尔积`A×B`是你可以通过从集合`A`中选择一个第一坐标、从集合`B`中选择一个第二坐标来构造的所有有序对的集合。让我们看一个例子。假设给定以下集合：
```
A := {1,2,3}
B := {3,4}
```
那么
```
A×B = { (1;3), (1;4), (2;3), (2;4), (3;3), (3;4) }
```
并且
```
B×A = { (3;1), (3;2), (3;3), (4;1), (4;2), (4;3) }
```
这个例子清楚地表明，`A×B`通常*不*等同于`B×A`。换句话说，笛卡尔积运算符不是交换的。你甚至可以证明，在`A×B`和`B×A`相同的情况下，`A`和`B`必须相等（反之亦然）：
`( A×B = B×A ) ⇔ ( A = B )`

另请注意，两个示例集合`A`和`B`包含*数字*作为其元素，而两个笛卡尔积的结果包含*有序对*。在这方面，笛卡尔积显然不同于我们之前讨论的运算符。并、交和差运算符都是重复（或省略）其操作数的元素；它们不会创建新类型的元素。

因为笛卡尔积从两个有限输入集中产生两个坐标的所有可能组合，所以以下性质也相当直观（也见前面的例子）：
`#A×B = #A * #B`

笛卡尔积的基数等于两个操作数基数的乘积。最后但并非最不重要的一点是，别忘了检查空集：`A×∅ = ∅×A = ∅`

#### 求和运算符

我们通过建立求和运算符（见定义 2-12）来结束这个集合论的导论章节。使用求和运算符，你可以对给定集合中的每个元素计算一个数值表达式，并计算所有这些计算结果的和。

**■定义 2-12：求和运算符（SUM）** 数值表达式`f(x)`的和，其中`x`取自给定集合`S`，表示如下：
`( SUM x∈S: f(x) )`

对于可以从集合`S`中选择的每个元素`x`，求和运算符计算表达式`f(x)`并计算所有这些计算结果的和。如果`S`代表空集，那么（根据定义）求和运算符返回数值`0`。

这里有几个例子：
```
( SUM x∈{ -1, 2, 5 }: x )
( SUM x∈{ -1, 2, 5 }: 1 )
( SUM n∈{ x| x∈N ∧ x ≤ 4 }: 2*n )
( SUM p∈{ (A;1), (B;1), (C;5) }: π2(p) )
```

第一个例子对值-1、2 和 5 求和。结果是 6。第二个例子是计算集合`{ -1, 2, 5 }`基数的某种复杂方式；它对集合中的每个元素求值 1，结果为 3。第三个例子计算和`2*0 + 2*1 + 2*2 + 2*3 + 2*4`，结果为 20。第四个例子计算`1 + 1 + 5`，结果为 7。

#### 一些方便的集合简写表示法

我们在数据库世界中经常使用一些（有限的）集合：日期、数字和字符串。因此，我们引入表 2-4 中列出的集合指定简写。

**表 2-4.** *一些方便的集合指定简写*

| 简写 | 定义 |
| :--- | :--- |
| `date` | `{ d \| d 是一个介于公元前 4712 年 1 月 1 日和公元 4712 年 12 月 31 日之间的日期 }` |
| `number(p,s)` | `{ n \| n 是一个精度为 p、标度为 s 的有效数字 }` |
| `varchar(n)` | `{ s \| s 是一个长度至少为 1、最多为 n 个字符的字符串 }` |
| `[n..m]` | `{ x∈Z \| x ≥ n ∧ x ≤ m }` |

如你所见，这种简写与 SQL 数据库管理系统支持的常见数据类型有着相当明显的相似之处。对于`date`简写，定义的边界是至少一个常见 DBMS 供应商所实现的边界。表 2-4 中的最后一个简写允许你指定整数的闭区间。


#### 章节小结

本节以项目符号列表的形式总结了本章内容。你可以在继续下一节的练习之前，用它来检查自己对本章引入的各种概念的理解程度。

*   集合完全由其元素所刻画，且仅由其元素所刻画。
*   你可以通过 `枚举法`、`谓词法`、`代换法`，或者谓词法与代换法混合的方式来指定集合。
*   `维恩图` 是集合的图形表示，因此在处理集合时可以作为视觉辅助工具；不过，请记住它们的局限性。
*   有限集合的元素数量被称为其 `基数`。我们使用符号 `#` 来表示基数运算符。
*   恰好包含一个元素的集合称为 `单元素集`；对于单元素集，我们可以使用“选择”运算符 (`↵`) 来解包其中的元素。
*   如果 `A` 的每个元素也是 `B` 的元素，则 `A` 是 `B` 的 `子集`；记法是 `A⊆B`。如果 `A` 同时不等于 `B`，则称其为 `B` 的 `真子集`（记法为 `A⊂B`）。
*   每个集合都是其自身的子集，空集是每个集合的子集。
*   最常见的集合运算符是 `并集`、`交集` 和 (`对称`) `差集`。
*   如果两个集合的交集是空集，则称它们为 `不相交`。
*   集合 `V` 的 `幂集` `℘V` 是由 `V` 的所有可能子集组成的集合。如果 `V` 包含 `n` 个元素，则幂集 `℘V` 包含 `2ⁿ` 个元素（包括空集和 `V` 本身）。
*   `集合的并集` 是二元并集运算符的推广，它允许你对任意数量的集合进行求并。
*   非空集合 `S` 的 `划分` 是一组两两不相交的、非空的 `S` 的子集，这些子集的并集等于 `S`。
*   `有序对` `(a;b)` 由两个坐标 `a` 和 `b` 组成，其顺序很重要；换句话说，`(a;b) = (c;d) ⇔ (a = c ∧ b = d)`。
*   `π₁` 和 `π₂` 运算符使你能够从有序对中提取第一和第二坐标。
*   两个集合的 `笛卡尔积` 是由所有可能的有序对组成的集合，这些有序对的第一个坐标选自第一个集合，第二个坐标选自第二个集合。
*   `求和` 运算符使你能够计算给定表达式在给定集合上的总和。
