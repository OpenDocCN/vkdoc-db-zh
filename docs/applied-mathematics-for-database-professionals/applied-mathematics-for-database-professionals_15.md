# 第二章 集合论：导论

**表 2-1. 集合论符号**

| 符号 | 含义 |
| :--- | :--- |
| ∈ | 是...的元素 |
| ∉ | 不是...的元素 |
| **Z** | 所有正整数、负整数及零构成的集合 |
| **N** | 自然数集，即大于或等于零的整数 |
| ∅ | 空集 |

**N** 和 **Z** 是来自常规算术的集合示例；以下是一些其他例子：
- 所有小于 20 的素数构成的集合（例如称为 `P`）：
  `P := {2,3,5,7,11,13,17,19}`
- 一周七天名称构成的集合（例如称为 `W`）：
  `W := {'sunday','monday', 'tuesday','wednesday','thursday','friday','saturday'}`

这里我们使用了枚举法来指定一个集合：在花括号内列出所有用逗号分隔的元素。这种方法以及其他一些指定集合的方法将在下一节介绍。

空集符号 `∅` 表示一个不包含任何元素的集合。`{}` 也用于表示空集。

一旦你为集合命名（就像我们为前面两个集合 `P` 和 `W` 所做的那样），就可以写出如下表达式：
- `-5 ∉ N`
- `-6 ∈ Z`
- `'4' ∉ N`
- `'thursday' ∈ W`
- `x ∈ P`

注意，前四个表达式实际上是命题；顺便说一下，它们都具有真值 TRUE。最后一个是一个谓词（假设 `x` 代表一个数值变量）。

你可以通过采用某些标准来保持表达式的可读性；例如，通常约定使用*小写*命名元素，使用*大写*命名集合。

但请注意，这只是一个命名约定；你不能根据标识符名称是大写还是小写来得出任何结论。考虑以下表达式：
`X ∈ Y`

该表达式*仅*在 `Y` 表示（或指代）一个集合时有定义；否则，它是一个未定义、无意义或格式错误的公式。也就是说，如果 `Y` 不指代一个集合，你就无法判断该语句是 TRUE 还是 FALSE。另一方面，`X` 和 `Y` 都可能指代集合，因为集合可以包含集合作为元素。看看以下三个例子：
- `3 ∈ 5` 既非 TRUE 也非 FALSE；它是一个*无意义*的表达式。
- `1 ∈ {1,2}` 为 TRUE，`2 ∈ {1,2}` 为 TRUE。
- `{1,2} ∈ {1,2}` 为 FALSE。
- `{1,2} ∈ {{1,2},3}` 为 TRUE。

为了澄清最后一个例子为何是一个 TRUE 命题，我们给右边的集合起个名字：
`S := {{1,2},3}`

那么集合 `S` 包含两个元素：第一个元素是集合 `{1,2}`，另一个元素是数值 `3`。因此，语句 `{1,2} ∈ {{1,2},3}` 是一个 TRUE 命题。

#### 指定集合的方法

到目前为止，你已经看到了几个集合的例子；除了集合 `Z` 和 `N` 外，它们都使用了*枚举法*来指定其元素，将所有元素列为一个用逗号分隔并放在花括号内的列表。我们将在本节讨论的其他方法基于一个给定的集合，并使用谓词来导出新的集合。

##### 枚举法

枚举（顾名思义，逐一列出）是迄今为止最简单的集合指定方法；你在花括号（也称为大括号）内列出集合的所有元素，并用逗号分隔。

参见代码清单 2-1 中的一些例子。

**代码清单 2-1. 枚举法指定集合示例**
```
E1 := {1,3,5,7,9}
E2 := {'a',2,'c',4}
E3 := {8,6,{8,6}}
```
请注意，在数学中，同一集合的元素不需要是相同类型；例如，代码清单 2-1 中的集合 `E2` 包含数字和字符的混合体。

还要注意，指定集合元素的顺序无关紧要；此外，重复多次指定同一个元素并非被禁止，但毫无意义。因此，以下三个集合是相等的：
`{1,2} = {2,1} = {1,2,1}`

当集合的元素数量增加时，枚举法的缺点变得显而易见；写下所有元素需要很多时间，而且表达式会变得太长。

##### 谓词法

对于大型集合，指定集合的另一种方法是*谓词法*。设 `S` 是一个给定的集合（例如自然数集），并设 `P(x)` 是一个给定的谓词，它恰好有一个类型为 `S` 的参数 `x`（即 `S` 中的元素构成参数 `x` 可以取的值）。那么，以下对一个名为 `E4` 的集合的定义是有效的：
`E4 := { x∈S | P(x) }`

这种方法也使用花括号来括住集合指定（就像枚举法一样），但现在你在花括号内的某处指定了一个竖线符号（`|`）。这个竖线应读作“...使得...”或“...满足以下条件为真：...”。

在竖线的右边你指定一个*谓词*，在左边你指定一个*集合*，从中为谓词中涉及的参数选择值。前面的语法应解读如下：“集合 `E4` 由 `S` 中使得谓词 `P(x)` 变为 TRUE 的所有元素 `x` 组成。”

参见代码清单 2-2 使用此方法的一些示例（假设 `EMP` 表示某个包含“员工记录”的集合，非正式地说）。第一个示例使用了 `mod`（取模）函数，你可能不熟悉；它返回第一个参数除以第二个参数的余数。

**代码清单 2-2. 谓词法指定集合示例**
```
E5 := { x∈Z | mod(x,2) = 0 ∧ x2 ≤ 169}
E6 := { e∈EMP | e(JOB) = 'SALESREP'}
```
集合 `E5` 的元素是所有偶数整数（除以二后余数为零），其平方小于或等于 169。集合 `E6` 的元素是所有 `JOB` 值等于 `'SALESREP'` 的员工记录。

##### 替换法

指定集合的第三种方法是*替换法*。参见代码清单 2-3 使用此方法的一些示例。

**代码清单 2-3. 替换法指定集合示例**
```
E7 := { x2 – 1 | x∈N }
E8 := { (e(HIRED)–e(BORN)) | e∈EMP }
```
前面集合 `E7` 的指定应解读如下：“集合 `E7` 由所有 `x2 – 1` 的值组成，其中 `x` 是 `N` 的一个元素。” 通常，两个集合指定（`E7` 和 `E8`）的左边现在包含一个*表达式*；竖线右边的右边告诉你必须从哪个集合中选择 `x` 的值代入该表达式。

##### 混合法

可选地，右边后面可以跟一个以 `x` 为参数的谓词。这使你也能如下指定集合 `E4`：
`E4 := { x | x∈S ∧ P(x) }`

左边现在只包含 `x`，而不是使用 `x` 的某个表达式。通常，对于集合指定，混合使用谓词法和替换法很方便。看一下代码清单 2-4 中的例子。

**代码清单 2-4. 混合法指定集合示例**
```
E9 := { x2 – 1 | x∈N ∧ mod(x,3) = 1 }
```
形式上，这是以下指定集合 `E9` 的两步收缩，使用中间集合定义 `T1`：
```
T1 := { x∈N | mod(x,3) = 1 }   // 首先使用谓词法
E9 := { x2 – 1 | x∈T1 }        // 然后使用替换法
```
当然，我们不会这样正式拆分集合定义。谓词法、替换法和混合法的共同点是，在枚举法变得过于繁琐的情况下，它们允许你使用一个*给定集合*和*谓词*。对于你想指定的每个集合，你只需选择最合适的方法。在本书中，混合法将是指定大型集合的主要方法。

#### 韦恩图

韦恩图（以苏格兰教授约翰·韦恩命名）是集合的*图示*。它们可以帮助你可视化多个集合之间的关系。不过请记住，韦恩图“仅仅”是示意图；它们也可能令人困惑甚至产生误导。你可能还记得学生时代数学课上见过的韦恩图。参见图 2-1，其中展示了来自代码清单 2-1 的三个集合 `E1`、`E2` 和 `E3` 的韦恩图示例。

**图 2-1.** *韦恩图示例*

集合 `E3` 的韦恩图已经有些令人困惑了，因为它包含了数字 8 和 6 两次；此外，一眼看去并不明显能看出集合 `E3` 包含三个元素。在韦恩图中如何表示一个本身是集合的元素并不明确。显然，集合 `E3` 不适合用韦恩图表示。而如果你想研究空集，你将面临如何在韦恩图中绘制空集的难题。只要牢记其局限性，韦恩图还是相当有用的。在本章后续内容中，它们将被用来阐明各种集合论概念。
