# 第三章 更多逻辑知识

##### 合取范式

一个谓词如果由通过 AND（与）连接的一个或多个成员组成，则该谓词处于合取范式（CNF）；也就是说，如果谓词，例如 `P-CNF`，具有以下格式：`P-CNF := C ∧ C ∧ C ∧ ... ∧ C_n`。

在此公式中，`C_1`, `C_2`, ... `C_n` 被称为合取项（conjunction members）——或简称合取项。每个合取项只能是简单谓词的迭代 OR（或）（注意也允许零次迭代，这会导致合取项不包含 OR 的调用）；简单谓词是指无法进一步分解为析取或合取的谓词（即，它不能是复合谓词）。这些简单谓词可以选择性地以一个否定连接词开头。注意，集合 `{∨, ∧, ¬}` 是功能完备的，如第一章所述；因此，你不需要其他连接词来表达任何任意谓词。同样，我们稍后将证明，任何任意谓词都可以转化为 CNF。

CNF 的完整详细形式如代码清单 3-17 所示。

`代码清单 3-17. 合取范式 (CNF)`

```
P-CNF := (SP_1a ∨ SP_1b ∨ SP_1c ∨ ... ∨ SP_1d) ∧
         (SP_2a ∨ SP_2b ∨ SP_2c ∨ ... ∨ SP_2d) ∧
         (SP_3a ∨ SP_3b ∨ SP_3c ∨ ... ∨ SP_3d) ∧
         ...
         ∧
         (SP_n1 ∨ SP_n2 ∨ SP_n3 ∨ ... ∨ SP_nd)
```

在代码清单 3-17 中，每个简单谓词 `SP_ij` 都可以带有一个否定连接词前缀。检查一个 CNF 命题是否为永真式（tautology）很容易，因为只要其中一个合取项为假（FALSE），整个命题就为假（FALSE）。换句话说，你可以逐个检查这些合取项。此外，因为 CNF 谓词的所有合取项都必须是仅由简单谓词构成的迭代 OR，所以永真式检查也可以表述如下：

*一个 CNF 谓词是永真式（始终为真）当且仅当每个合取项都包含一个简单谓词 P 及其否定。*

代码清单 3-18 展示了四个 CNF 谓词的例子（`P`, `Q`, `R` 和 `S` 是简单谓词）。

`代码清单 3-18. CNF 谓词示例`

```
P
P ∨ Q
P ∧ Q
(P ∨ Q ∨ ¬R) ∧ (P ∨ ¬R)
(P ∨ Q ∨ ¬P) ∧ (Q ∨ ¬Q) ∧ (¬R ∨ R ∨ S)
```

第一个谓词有一个合取项，由一个简单谓词组成。第二个谓词有一个合取项，恰好是两个简单谓词的析取；第三个和第四个例子由两个合取项组成。第五个例子是一个永真式；注意这三个合取项分别包含了简单谓词对（`P`, `¬P`）、（`Q`, `¬Q`）和（`R`, `¬R`）。

##### 析取范式

一个谓词如果由通过 OR（或）连接的一个或多个成员组成，则该谓词处于析取范式（DNF）；也就是说，如果谓词，例如 `P-DNF`，具有以下格式：`P-DNF := D ∨ D ∨ D ∨ ... ∨ D_n`。

在此公式中，`D_1`, `D_2`, ... , `D_n` 被称为析取项（disjunction members）——或简称析取项。每个析取项只能是简单谓词的迭代 AND（与）（允许零次迭代）。因此 DNF 的完整详细形式如代码清单 3-19 所示。

`代码清单 3-19. 析取范式 (DNF)`

```
P-DNF := (SP_1a ∧ SP_1b ∧ SP_1c ∧ ... ∧ SP_1d) ∨
         (SP_2a ∧ SP_2b ∧ SP_2c ∧ ... ∧ SP_2d) ∨
         (SP_3a ∧ SP_3b ∧ SP_3c ∧ ... ∧ SP_3d) ∨
         ...
         ∨
         (SP_n1 ∧ SP_n2 ∧ SP_n3 ∧ ... ∧ SP_nd)
```

在代码清单 3-19 中，每个 `SP_ij` 都可以带有一个否定连接词前缀。DNF 中的复合谓词很容易检查是否存在矛盾。对于一个 DNF 谓词成为矛盾式（contradiction），所有析取项必须都为假——因为该谓词是写成迭代 OR 结构的。遵循与 CNF 谓词相同的推理，我们可以对 DNF 谓词做出如下论断：

*一个 DNF 谓词是矛盾式当且仅当每个析取项都包含一个简单谓词 P 及其否定。*

有关 DNF 谓词的例子，请参见代码清单 3-20。

`代码清单 3-20. DNF 谓词示例`

```
P ∨ Q
P ∧ Q
(P ∧ Q ∧ ¬R) ∨ (S ∧ ¬R)
(P ∧ Q ∧ ¬P) ∨ (R ∧ ¬R ∧ S)
```


##### 求给定谓词的范式

本节我们将探讨如何将给定的谓词转换为析取范式（DNF）或合取范式（CNF）。

###### 求给定谓词的析取范式

有几种方法可以将现有谓词重写为析取范式。通常，你会尝试使用现有的重写规则，将表达式向“正确”的方向移动，直到它具有所需的格式：一个迭代的析取式，其中每个析取项是仅包含简单谓词的迭代合取式。

一个有趣的替代方法是基于使用真值表。例如，假设原始谓词如下：

`P ⇒ ( Q ⇔ ¬P )`

表 3-3 显示了对应的真值表。

**表 3-3.** 使用真值表将谓词重写为析取范式

| `P` | `Q` | `¬P` | `Q ⇔ ¬P` | `P ⇒ ( Q ⇔ ¬P )` |
| :--- | :--- | :--- | :--- | :--- |
| `t` | `t` | `f` | `f` | **`f`** |
| `t` | `f` | `f` | `t` | **`t`** |
| `f` | `t` | `t` | `t` | **`t`** |
| `f` | `f` | `t` | `f` | **`t`** |

从表 3-3 的最后一列可以看出，给定谓词在 (`P`,`Q`) 的最后三种真值组合下为真：(`t`, `f`)、(`f`, `t`) 和 (`f`, `f`)。你可以用以下对应的合取式来表示这三种真值组合：
(`t`,`f`): `C := P ∧ ¬Q`
(`f`,`t`): `C := ¬P ∧ Q`
(`f`,`f`): `C := ¬P ∧ ¬Q`

现在，如果你将这三个合取式组合成迭代析取式 `C ∨ C ∨ C`，你最终会得到一个重写后的谓词，该谓词为析取范式，并精确地表示了原始谓词的真值表：

`( P ∧ ¬Q ) ∨ ( ¬P ∧ Q ) ∨ ( ¬P ∧ ¬Q )`

如果你不确定，可以为重写的谓词建立一个真值表，并将其结果与表 3-3 中的原始谓词进行比较。顺便说一句，你注意到表 3-3 的最后一列与与非（NAND）运算符的定义完全相同吗？重写的析取范式表达式精确地表示了与非运算符；该表达式为假，当且仅当两个操作数都为真。

**注意：** 这种使用真值表的析取范式重写算法相对容易实现自动化。你可以编写一个程序来为你完成转换，而无需灵感和创造力；析取范式总是可以得到的。

###### 功能完备性再探

你还记得第 1 章中关于功能完备性的讨论吗？显然，你可以使用前面的技术将任何任意谓词（可选地包含来自大量理论连接词可能性中的任何非标准连接词）重写，以产生一个析取范式的谓词。无论原始谓词的复杂性如何，也无论它包含哪些连接词，你最终都会得到一个真值表列，该列“表示”了其变量所有给定真值组合下表达式的真值。

前面的技术利用这些信息生成一个逻辑上等价的表达式，该表达式只包含合取、析取和否定；因此，`{∧,∨,¬}` 是功能完备的。

###### 求给定谓词的合取范式

要找到给定谓词的合取范式——这也总是可以实现的——你首先应用前面使用真值表的方法来找到谓词的析取范式。然后，你应用分配律将迭代析取式转换为迭代合取式。以下是一个示例推导，将处于析取范式的谓词 `( P ∧ Q ) ∨ ( R ∧ S )` 转换为合取范式：

```
( P ∧ Q ) ∨ ( R ∧ S )
⇔ ( P ∨ ( R ∧ S ) ) ∧ ( Q ∨ ( R ∧ S ) )
⇔ ( P ∨ R ) ∧ ( P ∨ S ) ∧ ( Q ∨ R ) ∧ ( Q ∨ S )
```

我们将在本书第 2 部分讨论数据完整性约束的规范和实现指南时，再次探讨合取范式。


#### 章节摘要

本节包含本章内容的摘要，以项目符号列表格式呈现——与上一章类似。再次说明，如果你对以下任一概念感到不太熟悉，最好在继续下一节练习前回顾本章的相关小节。

-  就像常规算术运算符一样，逻辑连接词也具有某些*代数属性*，例如交换律、结合律、分配律、自反性、传递律、幂等性和吸收律。
-  你可以通过两种方式将谓词转化为命题，其中一种是通过*量化*来绑定参数。
-  有两种量词：
    -  *存在量词*允许你使用符号 `∃` 来表达诸如“存在……”的陈述。
    -  全称量词允许你使用符号 `∀` 来表达诸如“对于所有……”的陈述。
-  因为根据定义，数据库中的集合是*有限*的，所以你总是可以将数据库中某个集合上带有存在量词的谓词解释为一个迭代的 OR 构造；类似地，你可以将带有全称量词的谓词解释为一个迭代的 AND 构造。
-  对*空集*进行量化时要小心。存在量词将始终返回 FALSE，无论谓词是什么，而应用于空集时，全称量词将始终返回 TRUE。
-  量词可以*嵌套*。有时你可以在表达式中互换量词，但在许多情况下，这样做会改变表达式的含义。一般来说，如果量词类型相同，并且你所量化的集合中没有一个依赖于另一个集合所绑定的变量，那么你可以互换量词。
-  你可以对量化表达式应用许多*重写规则*。例如，你总是可以将全称量词表达式重写为存在量词表达式，反之亦然。
-  *范式*通常用于重写表达式，以便于你轻松求解、自动化定理证明过程或研究谓词是否是重言式或矛盾式。本章介绍了两种范式：
    -  *合取范式*：谓词写作迭代的 AND。当你研究重言式时，CNF 很有用。
    -  *析取范式*：谓词写作迭代的 OR。DNF 在检查矛盾时很有用。
