## 10. 给定以下两个函数

```
f := { (empno;103), (ename;'Tedesco'), (deptno;11) }
g := { (eno;103), (ename;'Tedesco'), (dno;11) }
```

指定一个函数 `h`，它将 `f` 中的 `empno` 属性重命名为 `eno`，并将 `deptno` 属性重命名为 `dno`。形式化地，指定函数 `h`，使得 `(f◊h) = g`。

## 11. 给定以下集合

```
E := { { (empno;103), (ename;'Tedesco'), (mgrno;104), (deptno;10) }
      ,{ (empno;104), (ename;'Rhodes'), (mgrno;106), (deptno;10) }
      ,{ (empno;105), (ename;'Sparks'), (mgrno;106), (deptno;20) }
      ,{ (empno;106), (ename;'Weiss'), (mgrno;102), (deptno;10) } }

D := { { (deptno;10), (dname;'Gaming') }
      ,{ (deptno;20), (dname;'Sales') }
      ,{ (deptno;30), (dname;'Research') } }

```

研究以下两个命题的真值：

**a.** `{ e◊f | e ∈ E} = { e↓{empno} | e ∈ E }`

**b.** `{ e↓{deptno} | e ∈ E } ⊂ { d↓{deptno} | d ∈ D }`

## 第二部分 应用

> Ce que l’on conçoit bien, s’énonce clairement  
> et les mots pour le dire, arrivent aisément.  
> (构思清晰的事物，便能表达明确  
> 表述它的词语，亦会自然来到。)  
> —— 尼古拉·布瓦洛 (1636–1711)

**你已经到达本书中将前四章建立的数学基础进行应用的部分。特别是，你将看到这如何应用于**

***规约数据库设计***。

**第 5 章将从一些重要的基石开始。我们将让你熟悉如何形式化地规约一个*表*和一个*数据库状态*。本章还将形式化地介绍最常见的表运算符：*并*、*交*、*差*、*投影*、*限制*、*连接*、*属性重命名*、*扩展*和*聚合*。**

**第 6 章将向你介绍*元组*、*表*和*数据库谓词*的概念。我们将使用这些类型的谓词来规约数据库设计的*数据完整性约束*。本章还将探讨一些通常在数据库设计中发挥作用的谓词*模式*。**

**第 7 章建立在前两章的基础上，介绍了一种形式化方法来完全规约一个数据库设计。这样的规约被称为*数据库宇宙*。这种方法最初由 Bert De Brock 和 Frans Remmen 于 1980 年代在埃因霍温理工大学计算机科学系联合开发。我们将使用这种方法定义一个广泛的示例数据库设计。**

**第 8 章向你介绍*事务宇宙*的概念。你可以使用这个概念来进一步规约数据库设计的语义。数据库宇宙包含了数据库设计中涉及的大部分约束：*静态约束*。事务宇宙允许你规约另一类约束，称为*动态约束*；它们在静态约束之外的其他理由上限制允许的事务。**

**第 9 章涉及数据检索。通过*查询*，你从数据库中检索数据。我们将向你介绍一种形式化规约查询的方法。本章也将是第一章使用 SQL 来进一步演示本书中介绍的数学应用的章节，给出了大量示例。**

**第 10 章介绍了一种形式化规约*事务*的方法。像第 9 章一样，它给出了各种例子来演示这一点，既通过形式化方法，也通过使用 SQL 表达式。**
