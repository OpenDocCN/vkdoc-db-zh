# 引言

在过去遥远的年代，数据由计算机以各种方式管理，并没有统一的方法。现在依然如此，这并非坏事，因为并非所有数据都能以相同方式处理。然而，确实存在大量数据可以用一种通用方法处理，于是在 20 世纪 70 年代初，关系模型中发展出了一套数学原理。

始于 IBM 的一个实验室，人们开发了用于处理关系数据的软件，并开发了一种语言来管理它。这种语言最终成为了**结构化查询语言**（SQL）。早期，不同的供应商对该语言应如何运作有各自的想法，但最终这些被整合成了一项标准。几十年来，该标准不断发展，这意味着该语言也在不断演进。

并非所有数据库供应商都遵循同一程度的标准。有些标准姗姗来迟，数据库供应商便用自己的想象填补了一些空白。其他标准实施起来比看起来更难，一些供应商“仍在努力实现中”。有时，供应商只是想以不同方式行事。

本书着眼于使用 SQL 执行基本任务。这主要意味着获取数据并可能对其进行处理。市面上有许多可用的数据库软件包，各自都有其特点，也各有其 SQL 标准的变体。本书涵盖了其中几个最受欢迎的软件包，并特别指出不仅展示标准的内容，也展示各个软件包的具体实现方式。

我们也将探讨数据库的设计，但并非因为我们将会设计任何数据库。对于任何 SQL 新用户来说，一大困惑是为何要这样或那样做事，以及数据为何是这个样子。通过理解一些设计原则，你将能更好地明白原因。

在本书中，我们不假设你具备任何先验知识，除了你对“表”是什么有个概念。你可能也使用过电子表格，有些经验。

至于你以后可能会做什么，这要视情况而定。你的工作可能是获取数据以输入到某种分析或报告软件中。你可能直接在 SQL 中进行所有分析，或者在专门的数据库应用程序中编写查询。又或者，你可能是一个想要管理博客或电子商务网站的网页开发者。
