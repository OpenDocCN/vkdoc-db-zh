# 8. 处理表格

## 摘要

除了从数据库表中简单获取数据外，你还可以使用聚合查询生成各种汇总。聚合查询使用一个或多个聚合函数，并隐含对数据的某种分组。

聚合查询有效地将数据转换为一个二级汇总表。对于总计汇总，你只能选择汇总值。你不能同时选择非聚合值。

主要的聚合函数包括：

*   `count()`，用于计算列中的行数或数值数量
*   `min()` 和 `max()`，用于按排序顺序获取列中的第一个或最后一个值

对于数字，你还可以使用：

*   `sum()`、`avg()` 和 `stddev()`，用于对数字列执行求和、平均值和标准差计算

在处理数字时，并非所有数字的用法都相同，因此并非所有数字都应该被汇总。

对于字符串，你还可以使用：

*   `string_agg()`、`group_concat()` 和 `listagg()`，用于连接列中的字符串

在所有情况下，聚合函数只对值起作用：它们都会跳过 `NULL`。

你可以控制列中哪些值被包含进来：

*   你可以使用 `DISTINCT` 来只计算每个值的单个实例。
*   你可以使用 `CASE … END` 来作为特定值的过滤器。

如果没有 `GROUP BY` 子句，或使用 `GROUP BY ()`，聚合结果就是总计：你将得到一行汇总值。

你也可以使用 `GROUP BY` 来为多个分组生成汇总。每个分组是独立的。当你这样做时，你会得到每个分组的汇总，以及包含分组值本身的额外列。

聚合并不局限于单表：

*   你可以连接多个表并聚合结果。
*   你可以将一个聚合结果连接到一个或多个其他表。

在许多情况下，分多个步骤处理聚合是有意义的。为此，将第一步放入通用表表达式（Common Table Expression）是很方便的，这是一个虚拟表，可以与下一步一起使用。

在对数据进行分组时，有时你希望过滤掉某些分组。这可以通过 `HAVING` 子句完成，你将其添加在 `GROUP BY` 子句之后。

在大多数情况下，我们使用 `SELECT` 语句从数据库中提取数据。`SELECT` 语句及其各种子句通常被称为 `DQL`——**数据查询语言**。

SQL 的功能远不止让你选择数据。除其他功能外，它还允许你操作数据。

SQL 中有三种主要的操作数据语句：

*   `INSERT` 语句是你向表中添加新数据的方式。
*   `UPDATE` 语句允许你更改表中的现有数据。
*   `DELETE` 用于从表中删除一行或多行。

总的来说，这些语句属于所谓的 `DML`——**数据操作语言**。

还有一些语句操作的不是数据，而是实际的数据库结构，例如表和视图：

*   `CREATE` 用于在数据库中创建新表、视图或其他对象。
*   `DROP` 用于移除一个 `CREATE` 的对象。如果用于表，则数据也会丢失。
*   `ALTER` 允许你对现有对象进行更改。
*   `TRUNCATE` 可用于重置表。这主要意味着删除所有行，如果有自增列则重新开始计数。

总的来说，这些语句属于 `DDL`——**数据定义语言**。
