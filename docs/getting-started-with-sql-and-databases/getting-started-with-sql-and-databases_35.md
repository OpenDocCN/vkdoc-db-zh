# 创建表

表是通过 `CREATE TABLE` 语句来创建的。例如：

```sql
CREATE TABLE customers (
id int GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
email VARCHAR(60) NOT NULL UNIQUE,
familyname VARCHAR(40) NOT NULL,
givenname VARCHAR(40) NOT NULL,
street VARCHAR(64),
town VARCHAR(48),
state VARCHAR(3) default 'VIC',
postcode CHAR(4)
CHECK (postcode SIMILAR TO '[0-9][0-9][0-9][0-9]'),
dob date
CHECK (dob < current_date - interval '18 years'),
phone char(10)
CHECK (phone SIMILAR
'[01][0-9][0-9][0-9][0-9][0-9][0-9][0-9][0-9][0-9]'),
spam boolean default false,
height decimal(4,1),
registered date NOT NULL
);
```

此语句适用于 PostgreSQL 数据库管理系统，但对于其他数据库管理系统也非常相似，我们会在后文中说明。

`CREATE TABLE` 语句，顾名思义，用于创建一个新表。它大致如下所示：

```sql
CREATE TABLE something (
--  详细信息
);
```

SQL 中有许多 `CREATE` 语句，例如 `CREATE VIEW`，但这个语句略有不同，因为其详细信息被包裹在括号内。

前面的详细信息通常定义表中的列。它们遵循以下模式：

```
名称 类型 [约束条件等 …]
```

约束条件是附加的规则，用于*约束*（限制）哪些值对是该列有效的。有时还存在一些其他属性，它们会影响列的行为。

约束条件和其他属性可以通过三种方式添加：

*   它们可以与列的定义一起*内联*添加。
*   它们可以在 `CREATE TABLE` 语句中单独添加，通常在末尾。
*   它们可以在表创建之后添加。

这是按灵活性递增和便捷性递减的顺序排列的。

要删除一个表，SQL 有点讲究。你可能认为可以 `delete` 一个表，但这行不通：`DELETE` 是为表内的行保留的。相反，你应该 `DROP` 该表：

```sql
DROP TABLE something;
DROP TABLE IF EXISTS something; --  Oracle 不支持
```

如果你尝试删除一个不存在的表，你会得到一个错误。前面的第二种形式忽略了这一点。Oracle 不支持 `IF EXISTS`，所以在那里你需要更加小心。

无论如何你都必须小心，因为删除表也会删除数据，这种错误你犯一次就可能开始找新工作了。专家也许能恢复数据，但别指望它。
