# 配置与运行分布式重放

如果你选择使用默认值，则无需修改配置文件。但如果你选择包含系统调用或更改空闲时间，那么就需要修改配置文件 `DReplay.Exe.Preprocess.config`。这是一个简单的 XML 配置文件。我正在使用的文件如下所示：

```xml
<?xml version="1.0" encoding="utf-8"?>
<Options>
<PreprocessModifiers>
<IncSystemSession>No</IncSystemSession>
<MaxIdleTime>2</MaxIdleTime>
</PreprocessModifiers>
</Options>
```

我只做了一处更改，调整了 `MaxIdleTime` 以限制重放过程中的任何停顿时间。

在运行预处理之前，请确保已安装 `DRController` 并且 `DReplay` 服务正在你的系统上运行。如果满足条件，你只需调用 `DReplay.exe` 来执行预处理。

```
dreplay preprocess –i c:\data\dr.trc –d c:\DRProcess
```

在上面的代码中，你可以看到 `dreplay` 执行了 `preprocess` 事件。输入文件由 `–i` 参数提供，用于存放输出的文件夹则通过 `–d` 参数提供。跟踪文件将被处理，输出将进入指定的文件夹。输出结果将类似于图 24-4。

**图 24-4.** 分布式重放预处理步骤的输出

预处理完成后，你就可以继续运行分布式重放过程了。

但在此之前，你需要确保已准备好一个或多个客户端系统。

### 配置客户端

客户端计算机必须进行配置才能与分布式重放控制器协同工作。首先，将你的客户端安装到不同的机器上。仅为演示目的，我在单台机器上运行所有组件；不过，即使你使用多台机器，设置过程也没有不同。你需要配置客户端以连接控制器，且一个客户端一次只能与一个控制器工作。你的系统还需要为两项内容预留空间。首先，你需要一个位置来存放每次重放时会被覆盖的工作文件。其次，如果你希望从该客户端收集执行数据，则需要为客户端输出的跟踪文件预留空间。你还需要决定客户端进程的日志记录级别。所有这些都在另一个 XML 配置文件 `DReplayClient.config` 中设置。以下是我的配置：

```xml
<Options>
<Controller>DOJO</Controller>
<WorkingDirectory>C:\DRClientWork\</WorkingDirectory>
<ResultDirectory>C:\DRClientOutput\</ResultDirectory>
<LoggingLevel>CRITICAL</LoggingLevel>
</Options>
```

目录和日志级别是清楚的。我还必须将客户端指向运行分布式重放服务的服务器。对于多个客户端工作，不需要其他设置；你只需要确保它们连接到正确的控制器系统即可。

### 运行分布式测试

至此，你已完成所有配置并捕获了数据。接下来，你需要回到命令行，使用 `Dreplay.exe` 可执行文件来运行。大部分控制通过配置文件完成，因此可执行文件本身需要的输入很少。你使用以下命令调用测试：`Dreplay replay –d c:\data –w DOJO`

你需要输入预处理输出的位置，这意味着你需要以逗号分隔的列表列出参与的客户端机器。执行的输出将类似于图 24-5。

**图 24-5.** 运行 DReplay.exe 的输出

如你所见，捕获了 365 个事件，但只重放了 244 个事件。其中，只有 151 个事件实际成功。在这种情况下，你可能需要确定关于为何某些事件失败的可能信息。为此，只需重新配置测试并再次运行即可。拥有可重复测试过程的整个理念就在于你可以反复运行它。前面的例子代表了对我本地 `AdventureWorks2012` 副本进行的一次轻负载运行，捕获过程大约持续了五分钟。然而，我配置了空闲时间的限制，因此重放仅在 58 秒内就完成了。

从这里开始，你可以根据需要重新配置测试、重置数据库并反复运行测试。

请注意，更改配置文件将需要你重启相关服务，以确保更改在下一组测试中生效。

运行这些测试时，建议使用我在第 2 至 6 章中讨论过的性能数据收集类型。这有助于你确保能够准确了解测试执行得好——或糟糕——的程度。

## 结论

通过包含分布式重放实用工具，SQL Server 现在使你能够针对数据库执行负载和功能测试。你通过使用服务器端跟踪以简单的方式捕获代码来实现这一点。然而，如果你计划利用此功能，请务必验证你根据本书提出的原则对查询所做的更改是否确实有效，并将有助于提高系统性能。你还应确保重置数据库，以尽可能避免错误。
