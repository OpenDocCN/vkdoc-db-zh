# 第五章：创建基准

如前所述，某些计数器具有绝对的好/坏值。这些计数器的当前值无需与基准值进行比较。例如，如果`Deadlocks/min`计数器的当前平均值为 10，则表明系统正遭受大量死锁。尽管如此，审查相应的基准值仍然是有利的，因为你的死锁问题可能已经存在很长时间了。拥有归档的基准日志有助于检测死锁的发生演变情况。

有些计数器没有明确的好/坏值。因为它们的值取决于应用程序，所以必须与相应的基准计数器进行相对比较。例如，`SQL Server`的`User Connections`计数器的当前值并不表明应用程序有任何好或坏的情况。但将其与相应的基准值进行比较，可能会揭示用户连接数量的大幅增加，这表明工作负载的增加。

## 比较计数器范围和时间

比较当前和基准计数器日志中计数器的值范围。计数器单个值的波动将通过值范围进行归一化处理。

比较一天中相同时段的日志。对于大多数应用程序，使用模式在一天中的不同时段会有所不同。要获得特定时间段内计数器的最小值、最大值和平均值，需要调整计数器日志的时间范围，如前所示。
