# 游标成本比较

现在您已经了解了不同类型的游标，让我们看看它们的成本。如果必须使用游标，您应该始终使用满足应用程序要求的最轻量级游标。不同游标特性之间的成本比较详述如下。

### 游标位置的成本比较

客户端游标和服务器端游标各有其自身的成本收益和开销，如下节所述。

#### 客户端游标

与服务器端游标相比，客户端游标具有以下成本收益：

*   **更高的可扩展性：** 由于游标元数据保存在连接到服务器的各个客户端机器上，维护游标元数据的开销由客户端机器承担。因此，服务更多用户的能力不受服务器资源的限制。
*   **更少的网络往返：** 由于 `SELECT` 语句返回的结果集传递给维护游标的客户端，因此从游标检索行时不需要额外的到服务器的网络往返。
*   **更快的滚动：** 由于游标在客户端本地维护，因此遍历游标行的速度更快。
*   **高度可移植：** 由于游标是使用数据访问层实现的，因此它可以在广泛的数据库中工作：SQL Server、Oracle、Sybase 等。

客户端游标具有以下成本开销或缺点：

*   **对客户端资源的压力更高：** 由于游标在客户端管理，它增加了对客户端资源的压力。但这可能并不那么糟糕，考虑到大多数时候客户端应用程序是 Web 应用程序，并且使用标准的负载均衡解决方案横向扩展 Web 应用程序（或 Web 服务器）相当容易。另一方面，横向扩展事务性 SQL Server 数据库仍然是一门艺术！
*   **支持的游标类型有限：** 不支持动态和键集驱动游标。
