# 第 10 章 ■ 数据库引擎优化顾问

SQL Server 的性能通常取决于数据库表上是否有合适的索引。然而，随着工作负载和数据随时间变化，现有的索引可能不完全合适，可能需要新的索引。决定正确索引的任务之所以复杂，是因为对一组查询有益的索引更改可能对另一组查询有害。

为了帮助你完成这个过程，SQL Server 提供了一个名为数据库引擎优化顾问的工具。该工具可以帮助为给定的工作负载识别一组最优的索引和统计信息，而无需具备数据库架构、工作负载或 SQL Server 内部机制的专家级知识。它还可以为一小部分问题查询推荐调优选项。除了该工具的优点外，我还在本章中介绍了它的局限性，因为它是一个如果使用不当可能弊大于利的工具。

在本章中，我将介绍以下主题：

-   数据库引擎优化顾问的工作原理
-   如何在有问题的查询集上使用数据库引擎优化顾问来获取索引建议，包括如何定义跟踪
-   数据库引擎优化顾问的局限性

## 数据库引擎优化顾问机制

你可以通过选择 Microsoft SQL Server 2012 ➤ 性能工具 ➤ 数据库引擎优化顾问来直接运行数据库引擎优化顾问。你也可以从命令提示符 (`dta.exe`)、SQL Profiler（工具 ➤ 数据库引擎优化顾问）、Management Studio 中的查询（突出显示所需的查询，然后选择查询 ➤ 在数据库引擎优化顾问中分析查询）或 Management Studio（选择工具 ➤ 数据库引擎优化顾问）运行它。一旦打开工具并连接到服务器，你应该会看到一个类似于图 10-1 的窗口。我将在本节介绍定义和运行分析的选项，然后在下一节通过一些详细示例进行说明。
