### 消除或减少数据库游标的开销

由于 SQL Server 被设计为处理数据集，因此使用 DML 语句处理多行数据通常比使用数据库游标逐行处理要快得多。如果你发现自己在大量使用游标，请重新检查逻辑，看看是否有办法消除游标。如果必须使用数据库游标，那么请使用开销最小的数据库游标：`FASTFORWARD` 游标类型（通常称为“只进快速游标”）。你也可以在 ADO.NET 中使用等效的 `DataReader` 对象。

数据库游标的性能开销在第 22 章中有详细解释。

### 原生编译存储过程

在你仅访问内存中表的情况下，你可以采用另一个额外的性能增强方法，那就是将你的存储过程编译成在 SQL Server 可执行文件内运行的 DLL。正如第 23 章所示，这具有相当显著的性能影响。只需确保你以正确的方式调用这些过程，通过序数位置而不是参数名来传递参数。尽管这感觉像是打破了最佳实践，但它能带来编译后过程更好的性能。

## 总结

性能优化是一个持续的过程。它需要持续关注影响性能的数据库和查询特性。本章的目标是为你提供一份这些特性的检查清单，以便在数据库应用程序的开发和维护阶段作为快速简便的参考。
