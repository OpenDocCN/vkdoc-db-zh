# 第 25 章 ■ 数据库工作负载优化

发现某种优化技术对工作负载的其他部分影响甚微，甚至产生负面影响，从而损害工作负载的整体性能，这种情况并不罕见。例如，为优化`SELECT`语句而添加的非聚集索引可能会损害修改索引列值的`UPDATE`语句的性能。`UPDATE`语句除了更新数据行外，还必须更新索引行。然而，如第 6 章所示，有时索引也能提高操作查询的性能。因此，提升特定查询的性能可能会有益或损害整体工作负载的性能。一如既往，你最好的做法是通过测试来验证任何假设。

## 工作负载优化步骤

优化数据库工作负载的过程遵循一系列特定步骤。在此过程中，你将使用前面章节介绍的一套优化技术。由于每个性能问题都是新的挑战，你可以使用不同的优化技术集来排查不同的性能问题。

请始终牢记，第一步永远是确保服务器配置良好并在可接受的限制范围内运行，正如第 2 章和第 3 章所定义的那样。

为了理解查询优化过程，你将使用一组查询来模拟一个示例工作负载。在优化此示例工作负载时，你将遵循以下优化步骤：

1.  捕获工作负载。
2.  分析工作负载。
3.  确定开销最大/调用最频繁/运行时间最长的查询。
4.  量化开销最大查询的基线资源使用情况。
5.  确定总体资源使用情况。
6.  编译资源使用的详细信息。
7.  分析并优化外部因素。
8.  分析索引的使用情况。
9.  分析应用程序使用的批处理级别选项。
10. 分析统计信息的有效性。
11. 评估碎片整理的需求。
12. 分析开销最大查询的内部行为。
13. 分析查询执行计划。
14. 确定执行计划中的高开销运算符。
15. 分析处理策略的有效性。
16. 优化开销最大的查询。
17. 分析变更对数据库工作负载的影响。
18. 迭代多个优化阶段。

如第 1 章所述，性能调优是一个迭代过程。因此，你应该多次迭代执行性能优化步骤，直到达到期望的应用程序性能目标。经过一段时间后，你需要重复此过程，以应对数据和数据库变更对工作负载造成的影响。
