## 非索引列上的统计信息

有时，你可能在连接或过滤条件中使用了没有任何索引的列。即使对于这类非索引列，如果查询优化器了解这些列的基数和数据分布（也称为*统计信息*），则更有可能做出最佳选择。

除了索引上的统计信息外，SQL Server 还可以为没有索引的列建立统计信息。关于数据分布的信息，或者某个特定值在非索引列中出现的可能性，可以帮助查询优化器确定最优的处理策略。即使优化器无法实际使用索引来定位值，这些信息也能使查询优化器受益。如果 SQL Server 认为这些信息对创建更好的执行计划有价值（通常在列被用于谓词时），它会自动为非索引列建立统计信息。默认情况下，此功能是开启的，并且可以通过数据库的“属性” -> “选项” -> “自动创建统计信息”设置进行配置。你可以使用 `ALTER DATABASE` 命令以编程方式覆盖此设置。然而，为了获得更好的性能，强烈建议你保持此功能开启。

通常，你不应禁用在非索引列上自动创建统计信息的功能。你可以考虑禁用此功能的场景之一，是在执行一系列你不会再次运行的临时的 SQL 活动时。即使在这种情况下，你也应该测试一下：是承受自动创建统计信息的成本以获得本次更好的计划更划算，还是影响其他 SQL Server 活动的性能更划算。因此，对于大多数系统，你应该保持此功能开启，无需为此担心。
