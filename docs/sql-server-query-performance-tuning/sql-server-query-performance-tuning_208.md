从执行计划中可以看出，对于第一条查询（`T1.C2 = 20`），优化器访问了两个表的数据。对于第二条查询（`T1.C2 = 30`），优化器根据 `T1.C2` 列上对应的 `CHECK` 约束了解到，该列不可能包含 `10` 到 `20` 范围之外的任何值。因此，优化器甚至不需要访问表中的数据。相应地，第二条查询的相对估计成本为 `0%`。

### 采用索引设计最佳实践

最常见的优化建议——并且通常是提升性能的最大贡献者之一——就是为数据库工作负载实施正确的索引。索引与表不同，表用于存储数据，甚至可以在不完全了解查询的情况下进行设计（只要表能正确表示业务实体）。相反，索引必须通过仔细审查数据库查询来设计。

除了常见且显而易见的情况（如主键和唯一索引）之外，请避免陷入在不了解查询的情况下设计索引的误区。即使是对于主键和唯一索引，我也建议您在开始设计数据库查询时，验证这些索引的适用性。考虑到索引对数据库性能的重要性，在设计索引时必须谨慎。
