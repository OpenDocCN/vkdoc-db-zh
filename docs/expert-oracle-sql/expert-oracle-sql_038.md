# 第 3 章

## 最终状态查询优化

现在让我们简要关注一下基于成本的优化器（CBO）的基本工作。最终状态优化涉及确定：

*   各行源的连接顺序
*   连接机制
*   查询中每个表的访问方法
*   是否使用 `in-list 迭代`

我将在第 12 章详细讨论所有这些项目。

## 本章小结

本章简要介绍了一些基本的 CBO 概念，例如成本，并在较高的层次上描述了 CBO 必须做出的决策类型以及它如何做出这些决策。本书的第 3 部分完全致力于 CBO，届时我将更详细地解释查询转换和最终状态优化。现在，让我们以三个关键信息来结束本次介绍：

*   CBO 试图在短时间内完成一项非常复杂的工作。
*   CBO 只是一个（非常）高级的猜测机器，虽然它的猜测通常是可以接受的，但常常并非如此，并且可能出现重大的性能问题。
*   当 CBO 错误地计算执行计划的成本时，这不应引起担忧。但如果它在一个明显更快的计划可用时选择了一个运行缓慢的计划，这就值得关注了。

既然我已经介绍了 CBO，是时候开始看看它的输出了。第 3 章将首次探讨执行计划。
