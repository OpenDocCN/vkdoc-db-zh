# 前言

当 Apress 的 Jonathan Gennick 找我，邀请我为本书写前言时，我没有多想就答应了，因为我早已知晓 Tony 的工作质量。他有一种很好的、实用的方法来理解和解决 Oracle 问题，以及优化低效的代码。

优化 SQL 性能并不总是意味着提供更详细的优化器统计信息以获得更快的执行计划。有时，你反而需要向基于成本的优化器提供**更少**的详细信息，并冻结统计信息，以实现更稳定的 SQL 性能。这种方法乍听起来可能有悖直觉且非主流，但如果 SQL 计划稳定性对你很重要，它可以避免日后的麻烦和深夜的电话。一如既往，你需要在正确的情况和时机下应用此类技术。Tony 解释了背后用于性能和稳定性的 SQL 优化与统计信息管理的思考过程，并提供了工具和示例来将他的方法付诸实践。

本书的另一个巨大价值在于其大量的实践示例，展示了如何通过重写或调整查询和语句来优化 SQL。你绝对需要具备实际 SQL 优化的思维模式和技能，因为拼命地重新收集优化器统计信息，或者再创建一个索引，可能不会为你的问题 SQL 带来任何显著的性能提升。因此，重写或调整是必要的，本书后半部分的多个章节通过处理各种你（或 Oracle 优化器）可以用来优化 SQL 形式的方式提供了帮助。

除了第 16 章（“重写查询”）和第 17 章（“优化排序”），你还在第 13 章中拥有多达 70 多页的优化器转换解释！这三章的内容帮助你理解，为什么一个查询的最终执行计划常常看起来与原始 SQL 的预期大相径庭，却仍然能交付正确的结果，而且更快。而这一切，反过来又有助于你从一开始就编写更高效的 SQL。

阅读本书时请记住，本书前半部分大致由许多较短的章节组成，每一章都向你介绍 Oracle 数据库 SQL 优化和运行时特性的不同部分。这些最初的章节作为构建模块，可以在本书后面的部分中投入使用，届时所有的部分将汇集在一起。

你将获得大量新知识，并且阅读本书会充满乐趣！

—Tanel Põder
