# 本书的关键信息

就像生活中的大多数问题一样，通过完全理解问题，可以更容易地（有时甚至是轻而易举地）解决一个 SQL 调优问题或生产环境不稳定问题。也许这是一个显而易见的陈述，但我已经数不清有多少次看到人们试图在*不理解*问题的情况下解决 SQL 性能问题。例如，某个性能问题的最佳解决方案可能是收集统计信息。也许您只需要停止并重新启动 SQL。也许您需要运行 SQL 调优顾问并创建 SQL 配置文件。但是，不要随机选择其中一项选项，当它不起作用时就按顺序尝试另一个。例如，如果您的 SQL 语句包含临时表，那么 SQL 调优顾问可能不会有多大用处，因为当 SQL 调优顾问运行时，临时表将是空的。您需要从审查 SQL 语句开始！
