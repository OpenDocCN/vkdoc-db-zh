# 创建或重建索引与表

*   使用 `DBMS_STATS.COPY_TABLE_STATS` 包的用户可能需要为复制目标分区的一些列设置统计信息。我们将在第 20 章讨论 `DBMS_STATS.COPY_TABLE_STATS`。
*   TSTATS 部署模型要求为某些列删除其最小值和最大值。同样，用于稳定执行计划的 TSTATS 模型也将在第 20 章讨论。
*   在临时表上收集统计信息可能很棘手，因为在收集统计信息时，临时表很可能是空的，尤其是当表设置为 `ON COMMIT DELETE ROWS` 时。
*   在我看来，所有需要的列都应该手动设置直方图。我主张手动设置直方图，以便限制测试用例的数量。本章稍后的列表 9-8 提供了一个如何直接设置直方图的示例，并更详细地解释了其好处。

在大型索引上收集对象统计信息可能非常耗时，因为需要从索引中读取数据。然而，在构建或重建索引时，可以生成索引的统计信息，而几乎不会带来额外开销，除非正在建立索引的表的统计信息被锁定，否则这是默认行为。请注意，即使为尚未加载数据的空表创建索引，也无法抑制此行为。

当运行 `CREATE TABLE ... AS SELECT` 语句时，或者使用 SQL Loader 或 `INSERT ... SELECT` 语句将数据批量加载到空表中时，也可以生成表和列的统计信息。奇怪的是，直到 12cR1 版本才出现能够在创建或加载表时生成统计信息的功能。以下是一些适用于 12cR1 及更高版本的一般提示：

*   当通过批量数据加载生成统计信息时，不会生成任何列直方图，并且任何索引的统计信息也不会被更新。
*   可以通过使用 `GATHER_OPTIMIZER_STATISTICS` 提示，为批量加载到非空表中的数据生成统计信息。
*   可以通过使用 `NO_GATHER_OPTIMIZER_STATISTICS` 提示来抑制为 `CREATE TABLE ... AS SELECT` 语句和批量加载空表生成统计信息。

以上规则可能会让你认为，如果你批量加载数据然后创建索引，那么你将不再需要收集更多统计信息。然而，`函数索引` 会向表中添加 `隐藏列`，而这些列的统计信息只能在索引创建后才能获取。此外，还涉及分区和索引组织表的复杂性。关于在表创建和批量插入期间如何以及何时收集对象统计信息的具体细节，请参阅《SQL 调优指南》。
