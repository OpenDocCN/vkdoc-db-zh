# 关于 CBO 其他输入信息的说明

虽然本章重点讨论了对象统计信息，但你应该知道，CBO（基于成本的优化器）还利用了至少四个其他信息来源：

*   优化器提示
*   初始化参数
*   系统统计信息
*   其他数据字典信息

我们将在本书中讨论优化器提示，第 18 章 专门讨论该主题。让我们依次看看其他三个信息来源，从初始化参数开始。

## 初始化参数

有大量的文档化初始化参数要么专用于 CBO，要么以某种方式影响 CBO。还有大量的隐藏初始化参数允许你影响 CBO 行为的特定方面。除非你得到第三方软件供应商的建议，否则你几乎总是应该努力使用几乎所有此类参数的默认值。

除了你不太可能通过调整与优化器相关的初始化参数的非默认值来改进 CBO 或运行时引擎的性能这一事实之外，你可能会遇到错误，因为 CBO 不一定在你的设置下经过测试。

为了说明我的观点，让我举一个初始化参数的例子，它不直接与 CBO 相关，但会影响 CBO 行为。`DB_BLOCK_SIZE` 的默认值是 8192 字节。过去专家们曾经建议为数据仓库使用更大的值。如今，大多数专家会同意 Jonathan Lewis 的《Cost-Based Oracle》一书中的以下评论：

> 你也许能找到一些特殊情况，通过将对象从一个块大小更改为另一个块大小可以获得积极的收益；但一般来说，由于优化器改变其算术运算而产生的一些副作用可能会超过你选择的块大小的感知收益。

确实如此。人们过去经常调整的另外两个初始化参数是 `OPTIMIZER_INDEX_COST_ADJ` 和 `OPTIMIZER_INDEX_CACHING`。这两个参数在系统范围内（这种概念本身就值得怀疑）工作，使得索引访问比全表扫描更优惠或不那么优惠。这些参数的一个问题是，它们的作用是减少索引访问的成本，而不是增加全表扫描的成本，因此可能会出现舍入误差。最好别动它们。
