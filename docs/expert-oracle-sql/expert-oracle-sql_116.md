# 识别必需的索引

《性能调优指南》指出：

> 索引设计在很大程度上是一个迭代过程，基于应用程序设计者生成的`SQL`。然而，通过构建用于强制主键约束的索引以及针对已知访问模式（例如人名）的索引，可以做出一个合理的开始。

索引设计是一个迭代过程，这一点非常正确，但我建议你在初始选择索引时要非常保守。很容易错误地假设一个索引是必需的，而实际上并非如此。在开始测试之前，你如何判断什么是*已知访问模式*？如果你有时间，我建议仅创建支持主键和唯一约束的索引，然后运行一些测试看看效果如何。
