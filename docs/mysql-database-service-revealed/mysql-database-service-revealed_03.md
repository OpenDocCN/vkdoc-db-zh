# 2. Oracle Cloud Infrastructure

## 云服务成本预期

使用云服务作为基础设施组件（如 IaaS 组件）最具吸引力的方面之一就是成本。你不仅可以节省建立物理信息技术实验室的成本（包括冷却和安全措施），而且无需额外投入人力资源来运营和维护硬件及服务器。

但是，在规划数据库系统时必须谨慎。Oracle 提供了多种不同的配置供你选择，例如选择**形状（计算对象的大小）**，这定义了 CPU 处理能力以及内存和磁盘大小。自然，更大（更强大）的形状比较小的形状成本更高。

使用云服务的费用因供应商而异，但所有供应商都基于可消耗处理（如虚拟计算平台）的小时费率计费，在某些情况下也按单位（如磁盘或其他存储设备）计费。此外，当你与数据库系统交互时（如上传数据、导出、网络传输等），可能会产生数据传输速率的费用。这些成本随时间累积，将决定你的节省总额。

让我们探讨一个简单的假设示例。我们将保持简单，仅计算数据库服务的成本。假设我们使用一个虚构的云服务，其计算服务收费为 $0.05/小时，存储收费为 $0.025/GB/天。这看起来真的很便宜，对吧？让我们算一下。对于一天，我们将产生 $36.00 的计算成本（30 天乘以 24 小时乘以 $0.05），如果我们每天使用 10 Gb 的磁盘存储，则需额外支付 $0.25（10 Gb 乘以 $0.025），每月总计 $36.25。显然，这比维护你自己的 MySQL 服务器所需的物理和人力资源成本要便宜得多！

虽然这个例子纯属虚构且极不准确，^(²) 但它应该让你了解云服务的计费方式以及你的账单在一个月内是如何累积的。但这不必完全出乎意料，因为你可以创建一个 OCI 账户并尝试可用的资源。事实上，如果你按照下一章的教程创建你自己的 OCI 账户，你可以看到这些费用是如何生成的，以及 Oracle 授予新账户的注册信用额度是如何抵扣账单的。也就是说，你将获得一定的美元信用额度，用于探索 OCI 资源。一旦该信用额度用完，你将按月为你使用的服务付费。

无论如何，请务必做好功课，并与 Oracle 的客户代表合作，以设定你对月度账单负债的预期。

**提示**

要获得 MDS 更准确的定价，请联系你的 Oracle 销售代表，讨论你的需求并设定成本预期。

云服务对于大多数系统架构师、基础设施规划师和信息技术专家来说已成为日常生活的一部分。利用云来满足容量、增长和创新等关键业务需求，是大多数企业未来的关键。

在本章中，我们发现了使用云服务的好处，对 Oracle Cloud Infrastructure 进行了简要介绍，并了解了使用 OCI 如何让你的业务以更少的资源、更少地依赖通常复杂的采购、安装、配置和关键基础设施组件维护而变得更加敏捷。

## 虚拟化

让我们首先了解虚拟化对于云服务意味着什么。这可能是初学者最难掌握的概念，因为有些人觉得云中没有什么是“真实”的。

简单来说，虚拟化仅仅意味着一项服务、一个功能，甚至硬件，已经被表示为一个独立的实体。在大多数情况下，这意味着所讨论的对象或资源确实作为进程或以某种形式的软件在一个庞大的计算框架中运行。

例如，一个计算资源（可以理解为服务器）可以作为虚拟机在 `虚拟机管理程序栈` 上运行。在其他情况下，它可能意味着一个概念性的、受管理的概念，比如一个虚拟网络，它向用户（你）呈现的方式就好像它是一个物理网络。

此外，一些虚拟资源只是其他虚拟资源的一部分，并非设计为单独使用。例如，一个 `块存储设备` 通常连接到一个计算资源，并通过运行在操作系统之上的操作系统（或平台服务）进行访问。

## 租户

租户是 `OCI` 为特定客户（用户）组织所有资源的方式。当你注册 `OCI` 时，系统会为你的账户分配一个 `租户`。你所有的资源、账单等都将与你的 `租户` 相关联。你 `租户` 中的资源无法与其他 `租户` 中的资源交互（除非配置了额外的资源来授予访问权限），并且你只能管理位于你 `租户` 中的那些资源。

## 隔离区

`隔离区` 是你 `租户` 的另一个组织层级。你可以创建 `隔离区` 来容纳一组相关的资源。对 `隔离区` 中资源的访问由你授予特定权限的用户组来定义。因此，`隔离区` 是一个虚拟或逻辑上的分组（而非物理分组）。有趣的是，当你使用 `OCI 云控制台` 时，你可以使用 `隔离区` 来筛选要查看的资源。

一个 `租户` 可以被视为你云资源的“根” `隔离区`，并且一旦你创建了一个 `隔离区`，创建的任何资源都需要你选择该资源将驻留于其中的 `隔离区`。

**提示**
为初学者简化起见，建议你在适应使用 `OCI` 之后再创建多个 `隔离区`。添加 `隔离区` 及其安全配置文件可能会使学习曲线更陡峭且更容易出错。

当你创建一个 `隔离区` 时，它就与你的 `租户` 关联起来。你可以根据需要创建多个 `隔离区`。每个 `隔离区` 都通过特定于该 `隔离区` 的访问策略进行控制，允许你限制用户组的访问，确保他们只能访问他们需要的资源。因此，当你处理多个 `隔离区` 时，必须仔细考虑如何为你用户对 `隔离区` 资源的访问进行授权（或限制）。

#### 安全区

每个 `隔离区` 都有一个 `安全区` 或 `安全区配置文件`。当创建一个 `隔离区` 并添加资源时，`OCI` 将使用 `安全区` 策略来验证所有针对这些资源的操作。如果操作受到限制（验证失败），该操作将被拒绝。我们将在下一节通过设置免费试用 `OCI` 账户的演练来了解更多关于安全性的内容。

#### OCI 资源分配

你可能想知道所有这些 `OCI` 资源存放在哪里，甚至它们是如何组织的。`OCI` 资源按物理和逻辑分组进行组织。`OCI` 资源可能位于一个或多个称为 `区域` 的本地数据中心，或在其中分布，这些中心具有一个或多个物理布局（地理隔离的硬件、电力、冷却等），称为 `可用域`，然后再细分为物理上相邻（同一机架）的分组，称为 `故障域`。以下尝试解释 `OCI` 内部的资源组织方式：

*   **区域**：由一个或多个数据中心组成的局部地理区域。在每个 `区域` 内，资源被分配到一个或多个 `可用域`，这允许进行额外的组织。所有资源要么是 `区域` 特定的（例如，虚拟网络），要么是 `可用性` 特定的（例如，存储和计算资源）。

*   **可用域 (AD)**：一个地理上隔离（与其他 `可用域` 隔离）、容错的物理资源分组。`可用域` 隔离通过允许你在 `可用域` 之间分配资源，从而降低故障风险。请注意，某些资源必须在同一 `可用域` 内创建。例如，如果你创建一个计算资源并想要附加存储，那么存储资源必须与计算资源位于同一 `可用域`。

*   **故障域 (FD)**：一个 `可用域` 内的硬件分组。`故障域` 允许你分配你的实例，使实例在同一个 `可用域` 内不运行在同一硬件上。这样可以防范影响一个 `故障域` 的硬件故障（相同或类似事件影响多个 `故障域` 的可能性很小），停机不会影响其他 `故障域` 中的资源。因此，如果你想要资源具有高可用性，你会希望在 `可用域` 之间以及在每个 `可用域` 内跨 `故障域` 分配你的资源。

*   **领域**：`区域` 的一个（逻辑）集合。`领域` 彼此隔离。你的 `租户` 将存在于单个 `领域` 中，并且能够访问该 `领域` 内的 `区域`。`OCI` 目前提供多个商业和政府 `领域`。

图 2-1 以视觉方式展示了 `区域`、`可用域` 和 `故障域` 的逻辑排列方式。如你所见，我们可以在 `区域` 之间分配资源，在一个 `区域` 内跨 `可用域` 分配，以及在一个 `可用域` 内跨 `故障域` 分配。每一层都增加了显著的隔离级别，以防止由硬件或外部影响引起的故障。

#### 虚拟云网络

`虚拟云网络 (VCN)` 是对传统计算机网络的虚拟化。这种虚拟化扩展到创建子网、路由表、网关等。你所有的资源都将在一个 `虚拟云网络` 上运行。该 `虚拟云网络` 与单个 `区域` 相关联，但可以连接到所有 `可用域` 和 `故障域`。此外，对于每个子网，你可以跨越一个或多个 `可用域`。在你能够启动（创建）计算实例之前，必须至少设置一个 `虚拟云网络`。因此，创建 `虚拟云网络` 是你设置账户时首先要做的事情之一。




#### 甲骨文云标识符 (`OCID`)

OCI 中的每个资源都有一个 Oracle Cloud Identifier（甲骨文云标识符）。它是一个唯一的字符串，用于标识 OCI 中的一个资源。通过命令行界面 (`CLI`)、`REST API` 或开发者 API 代码访问资源时，几乎任何要对资源执行的操作都需要用到 Oracle Cloud Identifier。以下展示了 `OCID` 的格式：
```
ocid1...[REGION][.FUTURE USE].
```
以下展示了一个 MySQL 数据库服务 `DB System` 的 `OCID` 示例。其他 OCI 资源将使用类似的格式，但前导部分通常显示 `OCID` 所引用的资源类型：
```
ocid1.mysqldbsystem.oc1.iad.aaaaaaaapuif...sbg5z7sy5imjtlclhbxwbjmrq
```

## 计算实例

也称为计算实例，是在 OCI 中运行的计算主机。换句话说，它是一个你可以用来运行软件、安装自己的应用程序等的服务器。计算实例可以是虚拟机或裸机主机。计算实例的大小或虚拟化硬件能力由一组称为**配置方案**的逻辑分组（包括 CPU 大小、核心数、性能和内存设置）来管理。服务器的配置（操作系统等）则由一个称为**镜像**的模板来管理。

##### 虚拟机

虚拟机是一种在大型系统主机（称为虚拟机管理程序服务器）上运行的硬件的软件抽象实现，该主机为服务器分配虚拟化内存、CPU，有时还有磁盘。在使用自己的服务器或 `PC` 时，你可能已经接触过虚拟机。OCI 中的虚拟机与此类似，但复杂得多，因为它们可以通过配置方案进行性能和容量的定制。

##### 裸机主机

当性能或隔离性至关重要时，OCI 提供一小部分物理（或“裸金属”）机器。这些机器直接运行在硬件上，绕过虚拟机管理程序，不使用虚拟化内存、`CPU` 等（但可以连接到虚拟化资源）。甲骨云基础设施为你提供对物理主机（“裸金属”）机器的控制。当你使用裸机主机时，你是唯一使用物理 `CPU`、内存和网络接口卡 (`NIC`) 的租户。你可以像使用任何其他计算实例一样使用它，其外观与使用虚拟机的计算实例没有区别。

##### 配置方案

配置方案是一组定义计算实例的 `CPU` 核心数、内存大小以及在某些情况下磁盘空间的配置项。这允许你“调整”你的计算实例以满足需求。例如，如果你不需要大量内存，可以选择一个“较小”的配置方案，从而在产生成本时节省一些费用。

##### 镜像

镜像是一个虚拟化磁盘的模板，定义了操作系统和默认安装的软件。当你创建计算实例时，可以选择要使用的镜像。例如，你可以选择预装了 Oracle Linux 的镜像。你也可以从已配置的计算实例创建一个镜像，这样就可以部署一个相同配置的新计算实例，为你节省设置和配置时间。很棒！

##### 关于“实例”一词的说明

实例（instance）是云计算中那些被过度使用的术语之一。在 OCI 的术语中，将计算实例简称为“实例”是不正确的。这是因为还有其他被视为实例的资源。
例如，在 MySQL 术语中，“实例”可以指运行中的 MySQL 服务器应用程序（运行在服务器平台上的 MySQL），或者在 MySQL 数据库服务中，一个 `DB System` 可以被称为 `DB System 实例`。因此，你应该始终明确“实例”的含义：计算实例、MySQL 实例或 `DB System 实例`。

## 配置

这是你开始创建资源时会遇到的一个术语。此处的配置是指 OCI 创建资源所需的一系列操作，包括其子组件以及为使用该资源配置资源。例如，配置一个新的计算实例（从高层次上讲）包括：分配一个虚拟机、通过镜像安装操作系统和基础软件，以及为你所在的区域、可用性域等配置计算实例。在使用 OCI 资源时，你可能会在状态信息中看到这个术语。

## OCI 用户界面

OCI 提供了多种交互机制。具体来说，有多种用户界面可用于创建、管理和销毁 OCI 资源。这些包括基于 Web 的 OCI 云用户界面（称为 Oracle Cloud Console 或简称 cloud console）、名为 OCI 命令行界面（`CLI`）的命令行工具，以及面向开发人员的可编程接口，包括一个表述性状态传递（`REST`）应用程序编程接口（`API`）^(³) 以及特定语言的 API，允许你直接从代码中操作 OCI。甚至还有一个名为 Cloud Shell 的基于云的终端应用程序。我们将在本章探讨基于 Web 的云控制台界面，而将 `CLI`、`REST` 和开发者 API 留到第 9 章。

## 核心服务

到目前为止，在我们对 OCI 的探索中，我们遇到了虚拟网络、计算实例、存储和访问等概念。这些资源由专用于该类型资源的云服务（有时称为云能力）管理和提供。你用来创建、管理和销毁这些资源的机制在很大程度上是隐藏的，并且涉及一系列通常密集的自动化流程。例如，当你创建一个计算实例时，会启动几个自动化机制和其他例程（称为工作流）来完成资源的配置。

这些概念被认为是 OCI 中可用服务的核心。由于它们每个都是 OCI 提供的服务，因此我们将该能力称为服务，而单个资源则按其对象名称引用。例如，计算服务用于创建计算实例。

以下列出了 OCI 提供的核心服务。我们将在后面的章节中更详细地了解每一项：

*   网络服务
*   计算服务
*   存储服务
*   身份与访问管理服务

OCI 中的这些核心服务用于构建和提供更复杂的服务和资源。事实上，OCI 中的每个复杂资源通常都是使用这些核心服务中的一个或多个构建的。例如，MySQL 数据库服务构建在计算、网络、存储和身份与访问管理服务之上。其他例子包括容器编排、额外的托管服务、无服务器计算等等。

让我们更详细地看看这些核心服务。




#### 网络服务

或许最重要的核心服务是网络服务。与其他资源一样，网络服务也是虚拟化的。这意味着传统的硬件/软件组件，例如网络、子网、路由、网关以及基于硬件的传统网络的类似构建模块，都以资源的形式呈现，您可以创建这些资源并与其他资源一起使用，以实现网络连通性。

这种虚拟网络组件的概念有时被称为 `软件定义网络（SDN）`。因此，连接到这些虚拟网络组件的云端资源也是启用了 `SDN` 的基础设施，使您可以创建和终止您的 `虚拟云网络（VCN）`，将它们组织成 `子网`，并与您的计算实例和其他云资源一起使用。OCI 网络资源包括以下内容。本节我们仅涉及高层次概念，当我们在创建网络资源的教程中看到时，会提供更多细节：

*   虚拟云网络及其子网
*   预留的公共 IP 地址
*   安全列表和安全规则
*   网关
*   路由表和规则
*   负载均衡器
*   `DNS` 区域
*   Web 应用程序防火墙（`WAF`）策略

提示

有关 OCI 网络服务和资源的更多信息，请参阅 [`https://docs.oracle.com/en-us/iaas/Content/Network/Concepts/landing.htm#top`](https://docs.oracle.com/en-us/iaas/Content/Network/Concepts/landing.htm#top)。

在使用 OCI 网络时，您通常从创建一个 `虚拟云网络（VCN）` 开始，并为其分配一个私有互联网协议（`IP`）地址范围供网络内使用。您可以创建多个 `VCN`，并可将它们配置为可通过互联网访问（或不可访问），以及云专属的（隔离的）和互连的网络。您还可以将您的 `VCN` 组织成 `子网`；这是网络地址的一个逻辑细分。此处的 `子网` 与硬件网络子网的概念相同。

当您创建需要网络连接的资源（例如计算实例）时，必须通过一个称为 `虚拟网络接口卡（vNIC）` 的云资源将它们分配到一个或多个 `子网`。连接到公共 `子网` 的 `vNIC` 可以被分配一个公共 `IP` 地址，从而允许从互联网访问。

有趣的是，OCI 使用一个动态分配的公共 `IP` 地址池。当 `vNIC` 请求公共 `IP` 地址时，它会从池中分配一个。当资源被终止时，该公共 `IP` 地址会返回到池中。这是云服务弹性的另一个例子。但是，如果您需要一个永久的公共 `IP` 地址，您可以申请一个预留的公共 `IP` 地址，该地址会分配到您的云账户（这可能也涉及费用）。

您还可以创建一个 `安全列表`，其中存储用于添加一层软件防火墙的规则，以阻止或允许访问您的资源。它的工作方式类似于端口和地址过滤器，只允许那些符合安全权限列表的连接。

类似地，您可以创建 `路由规则`，用于控制 `VCN` 的出站通信，允许数据包移动到网络中的下一跳（hop）。路由规则与 `网关` 资源相关联。

云端的网络工作与基于硬件的网络一样辛苦。您必须仔细选择所需的资源，并选择正确的规则集，以确保一切正常工作，只允许期望的流量进出 `子网`。幸运的是，OCI 提供了一个向导，我们可以用它来自动化一些更常见的配置。

#### 计算服务

计算服务是代表计算硬件的资源，例如运行应用程序或服务的服务器。回想一下，我们使用术语 `计算实例` 来指代已配置的计算资源。

当您配置一个 `计算实例` 时，您可以从多种选项中选择，包括机器类型（裸机、虚拟机等）以及一组性能设置（称为 `形状`），使您可以根据容量和性能需求定制计算实例。

#### 存储服务

这些服务是 OCI 提供的磁盘和数据存储资源。您有多种数据存储资源可供选择，以适应您的数据工作负载。这些资源具有诸如持久性/非持久性、数据类型、读写输入/输出操作每秒性能（`IOPS`）、吞吐量、持久性、连接性、存储接口协议、容量等特性。

与计算实例类似，您有许多选项可以定制存储以满足您的业务需求。以下部分简要介绍了 OCI 中存储的常见选择。

##### 块卷

一种虚拟化的硬盘。`块卷` 是为计算实例提供持久存储的磁盘。与大多数资源一样，它们是虚拟的，意味着您可以在不同的计算实例之间分离和重新连接它们（使用另一个称为 `块卷附加` 的虚拟资源）（一次只能连接到一个计算实例），而不会丢失数据。

`块卷` 的使用方式与您在 PC 上使用硬盘的方式相同。您只需读写磁盘数据或安装应用程序。您可以选择 `启动卷`（计算实例使用它来写入镜像（可以理解为操作系统））或用于数据的 `块卷`。`块卷` 是持久且耐用的。您可以从基础、平衡和高性能品种中选择，这些品种定义了性能特征。

##### 本地 NVMe

一种虚拟化的内存驱动器（类似于 `SSD`）。与 `块卷` 类似，它们可以作为块存储附加，但具有非持久性和非耐用性。这些主要用于需要高性能本地存储而不需要长期存储的应用程序。

##### 文件存储

一种分布式文件系统，您可以将其附加到计算实例并像使用本地驱动器一样使用它。通常用作共享文件系统存储，文件存储是持久且高度耐用的。

## 对象存储

与 `块卷` 不同，`对象存储` 是一种不同的架构，设计用于将数据存储为对象。因此，存储是针对对象级访问而非像 `块卷` 那样的单个文件访问进行优化的，因此您不会使用 `对象存储` 来读写您的数据文件。

数据对象存储在 `存储桶` 中，这是您创建的用于放置数据对象的特殊容器。每个 `存储桶` 可以包含无限数量的数据对象。但是，数据对象可以是任何类型，最大可达 50 GB。

您可以在两个存储层之间选择：标准对象存储（活动且可访问的，有时称为热存储）或存档存储（不使用时会被存档，有时称为冷存储，并且在您想要使用时必须使其可用）。

`对象存储` 也不是您可以直接附加到计算实例的东西。相反，您使用 `对象存储` 将数据复制到对象存储。`对象存储` 的用例包括备份数据、共享文件以及非结构化数据存储（如日志）。



## 身份和访问管理服务

身份和访问管理 (IAM) 服务是您用来控制对 OCI 资源访问的资源，这使得这些核心服务成为构建基于 OCI 的基础设施时较为复杂但又至关重要的部分。您可以创建`用户`和`组`，并应用访问策略。

最常用的概念是`委托主体`（有时称为`实例委托主体`），这是一个应用于 IAM`用户`和计算实例的资源，用于授予访问或与 OCI 资源交互的权限。经认证的`委托主体`可执行的操作和行为称为`授权`。

`用户`和`组`正如您所预期的那样，`用户`拥有名称、密码、API 签名密钥和一个或多个身份验证令牌。通过使用`策略`资源向`用户`和`组`授予权限，这些策略允许`用户`或`组`在`租户`内的特定` compartment `（隔离区）中访问资源，并可附带任何限制或条件。

例如，当`用户`想要访问 OCI 资源时，该`用户`（或应用程序/服务）必须是一个`组`的成员，并且该`组`应在同一个` compartment `和`租户`中拥有包含相应权限的`策略`。

现在我们已经简要介绍了 OCI 核心服务，让我们学习如何设置 OCI 账户并创建我们的第一个 OCI 资源。

## 为计算实例添加块存储

由于默认的启动卷是 50 GB，对于存储任何大型应用程序甚至数据来说可能都太小了。毕竟，启动卷旨在用于存储系统应用程序等，而不是用户应用程序。幸运的是，我们可以轻松地为计算实例添加一个块卷，用于我们的应用程序和数据目的。

添加块卷的过程比我们目前所见的简单点击式界面要复杂一些。简而言之，我们需要执行以下操作：
- 创建一个块卷
- 将块卷附加到计算实例

## 持久化挂载

如果您计划将附带块卷的计算实例用于实验或单次会话之外，您会希望使此映射永久生效。为此，您需要编辑 `/etc/fstab` 文件。要完成此操作，您需要块卷的 UUID。您可以使用 `sudo blkid` 命令找到 UUID。获得 UUID 后，将以下行添加到 `/etc/fstab` 文件中，然后运行 `sudo mount -a` 以检查映射：

```
UUID= /mnt/data xfs defaults,_netdev,nofail 0 2
```

完成上述操作后，驱动器将在启动时自动挂载。

## 入门指南

MySQL 是世界上最受欢迎的开源数据库系统，这有许多充分的理由。首先，它是开源的，这意味着任何人都可以免费将其用于各种任务。最重要的是，MySQL 包含在许多平台的软件仓库中，使其易于获取和安装。如果您的平台（例如 aptitude）没有在软件仓库中包含 MySQL，您可以从 MySQL 网站 ([`http://dev.mysql.com`](http://dev.mysql.com)) 下载。

Oracle 公司拥有 MySQL。Oracle 通过收购 Sun Microsystems 获得了 MySQL，而 Sun Microsystems 则是从其原始所有者 MySQL AB 手中收购了 MySQL。尽管存在相反的担忧，但 Oracle 通过持续投资于 MySQL 的演进和新功能开发，并忠实维护其开源传统，展现了出色的管理。尽管 Oracle 也提供 MySQL 的商业许可证——正如其以前的所有者过去所做的那样——MySQL 仍然是开源的，并对所有人开放。

开源真的是免费的吗？

开源软件起源于对公司财产心态的有意识抵制。Richard Stallman 被认为是自由软件运动之父，他开创了一种许可机制，以帮助保护软件的所有权，同时让软件的使用以及一定程度上的修改对所有人免费。目标是重建一个开发者合作的社区，只有一个共同的使命——保障自由而非限制自由。

这最终导致了一些措辞巧妙（即具有法律约束力）的许可协议的发明，这些协议允许代码不受限制地复制和修改，并规定衍生作品（修改后的副本）必须根据与原始版本相同的许可证分发，不得附加任何额外限制。其中一种许可证（由 Stallman 创建）称为 GNU 通用公共许可证（GPL）。Oracle 正是使用此许可证来授权 MySQL，因此它确实可供任何人免费使用。

然而，GPL 及类似许可证旨在保障使用、修改和分发的自由；大多数情况下，它们从未将“自由”意指为“免费”或“免费赠予”。为了消除这种误解，开源促进组织（OSI）成立，后来采纳并推广了“开源”一词来描述 GPL 许可证所保障的自由。有关开源软件和 GPL 的更多信息，请访问 [`www.opensource.org`](http://www.opensource.org)。

MySQL 作为后台进程（或者如果您从命令行启动它，也可以作为前台进程）在您的系统上运行。与大多数数据库系统一样，MySQL 支持结构化查询语言（SQL）。您可以使用 SQL 来创建数据库和对象（使用数据定义语言 [DDL]），写入或更改数据（使用数据操纵语言 [DML]），以及执行用于管理服务器的各种命令。

连接到 MySQL 有两种机制。您可以使用像 MySQL Shell 或 MySQL Workbench 这样的应用程序，或者您可以使用较旧的 MySQL 客户端。让我们看看这两种选项。



### 连接 MySQL Shell

MySQL Shell 是 MySQL 产品组合中新增的一个令人兴奋的组件。它代表了首个用于连接和交互 MySQL 的现代化高级客户端。该 Shell 可作为脚本环境，用于开发处理数据的新工具和应用程序。虽然它确实支持 SQL 模式，但其主要目的是允许使用 JavaScript 和 Python 语言访问数据。没错，你可以编写 JavaScript 和 Python 脚本，并在 Shell 中以交互方式或批处理方式执行它们。很酷吧！

**注意**

我使用术语 “shell” 指代 MySQL Shell 支持的功能或对象。我使用 “MySQL Shell” 指代产品本身。

MySQL Shell 设计用于通过 X Plugin 使用新的 X Protocol 与服务器通信。但是，Shell 也可以使用较旧的协议连接到服务器，尽管在脚本模式下功能有限。这意味着，Shell 允许你同时处理关系型（SQL）、JSON 文档（NoSQL）或两者。

NoSQL 接口基于一个称为 X DevAPI（[`https://dev.mysql.com/doc/x-devapi-userguide/en/`](https://dev.mysql.com/doc/x-devapi-userguide/en/)）的应用程序编程接口。管理操作则由另一个称为 AdminAPI 的 API 支持。如果你计划使用 Shell 进行开发工作，请参阅在线文档以获取有关这些 API 的更多信息（[`https://dev.mysql.com/doc/mysql-shell/8.0/en/`](https://dev.mysql.com/doc/mysql-shell/8.0/en/)）。

#### 功能特性

MySQL Shell 具有许多功能，包括支持传统 SQL 命令处理、脚本原型设计，甚至支持自定义 Shell。以下列出了 Shell 的一些主要功能。大多数功能可以通过命令行选项或特殊的 Shell 命令来控制。我们将在后续章节中更深入地探讨一些更关键的功能：

*   *日志记录*：你可以为会话创建日志以供后续分析或保存消息记录。你可以使用 `--log-level` 选项设置详细级别，范围从 1（无日志记录）到 8（最大调试信息）。

*   *输出格式*：Shell 支持三种格式选项：表格（`--table`），这是你从旧客户端习惯的传统网格格式；制表符分隔（tabbed），它使用制表符进行间距并用于批处理执行；以及 JSON（`--json`），它以更易于阅读的方式格式化 JSON 文档。这些是你在启动 Shell 时指定的命令行选项。

*   *交互式代码执行*：使用 Shell 的默认模式是交互模式，它像传统客户端一样工作，你输入命令并获得响应。

*   *批处理代码执行*：如果你想在没有交互会话的情况下运行脚本，可以使用 Shell 以批处理模式运行脚本。但是，输出仅限于非格式化输出（但可以使用 `--interactive` 选项覆盖）。

*   *脚本语言*：Shell 同时支持 JavaScript 和 Python，但一次只能使用一种。

*   *会话*：会话本质上是到服务器的连接。Shell 允许你存储和删除会话。我们将在后面的章节中看到更多关于会话的内容。

*   *启动脚本*：你可以定义一个脚本在 Shell 启动时执行。你可以用 JavaScript 或 Python 编写该脚本。

*   *命令历史和命令补全*：Shell 会保存你输入的命令，允许你使用上下箭头键调用它们。Shell 还为已知关键字、API 函数和 SQL 关键字提供代码补全功能。

*   *全局变量*：Shell 提供了一些全局变量，你可以在交互模式下访问它们。这些包括：
    *   `session`：如果已建立，则为全局会话对象
    *   `db`：如果通过连接建立，则为模式对象
    *   `dba`：用于处理 InnoDB 集群的 AdminAPI 对象
    *   `shell`：用于使用 Shell 的通用函数
    *   `util`：用于处理服务器的实用函数

*   *自定义提示符*：你还可以通过使用特殊格式更新名为 `~/.mysqlsh/prompt.json` 的配置文件或定义名为 `MYSQLSH_PROMPT_THEME` 的环境变量来更改默认提示符。有关更改提示符的更多详细信息，请参阅 MySQL Shell 参考手册。

*   *自动补全*：从 8.0.4 版本开始，Shell 允许用户按 *TAB* 键在 SQL 模式下自动补全关键字，以及在 JavaScript 和 Python 模式下自动补全主要类和方法。

*   *扩展*：你可以以报告和扩展对象的形式定义扩展。报告和扩展对象可以使用 JavaScript 或 Python 创建。

*   *实用程序*：Shell 还提供了几个你可能会觉得有用的实用程序，包括以下。你可以通过 API 调用访问它们，如示例所示。
    *   *升级检查器*：用于验证 MySQL 服务器实例是否已准备好进行升级（例如，`util.checkForServerUpgrade()`）。
    *   *JSON 导入*：将 JSON 文档导入到 MySQL 服务器的集合或表中（例如，`util.importJSON()`）。
    *   *并行表导入*：拆分单个数据文件并使用多个线程将数据块加载到 MySQL 表中。

*   *Shell 命令*：与原始的 MySQL 客户端一样，有一些特殊命令用于控制应用程序本身，而不是与数据交互。要执行 Shell 命令，请使用反斜杠（`\`）发出命令。例如，`\help` 会打印所有 Shell 命令的帮助信息。

*   *选项*：可以使用多个启动选项来启动 Shell，这些选项控制模式、连接、行为等。常见选项包括模式（SQL、JavaScript 或 Python）和连接参数。有关选项的完整列表，请使用 `mysqlsh --help` 执行 Shell。

如你所见，MySQL Shell 有一个长长的功能列表。有关完整的功能列表，请参阅在线参考手册。

## 模式支持

MySQL Shell 支持三种模式（也称为语言支持或活动语言）：SQL、JavaScript 和 Python。我们可以通过 Shell 命令来启动其中任何一种模式。你可以根据需要随时切换模式（语言），而无需每次断开连接。以下是三种模式及其切换方式：

*   `\sql`：切换到 SQL 语言

*   `\js`：切换到 JavaScript 语言（默认模式）

*   `\py`：切换到 Python 语言

你也可以使用 `--` 代替 `\` 作为启动命令（`--sql`、`--js`、`--py`）。

## 建立连接

在 Shell 中建立连接是一个需要与原来 MySQL 客户端（`mysql`）不同操作习惯的领域。你可以使用特殊格式的 URI 字符串，或者像旧客户端那样通过指定名称的各个选项来连接到服务器。也支持 SSL 连接。连接可以通过启动选项、Shell 命令和脚本建立。但是，所有连接都需要密码。因此，除非另有说明，否则如果未提供密码，Shell 将提示输入密码。

> **注意**
>
> 如果你想使用无密码连接（不推荐），必须使用 `--password` 选项，或者如果使用 URI，则需要额外添加一个冒号来代替密码。

与其讨论所有可用的连接方式及相应选项，不如在下面的章节中介绍每种连接方法的一个示例。

### 使用 URI

在 MySQL Shell 连接的场景下，URI 是一个使用以下格式编码的特殊字符串：`<dbuser>[:<dbpassword>]@host[:port][/schema/]`，其中 `<>` 表示各个参数的字符串值。注意，密码、端口和模式是可选的，但用户和主机是必需的。这里的模式是指连接时要使用的默认模式（数据库）。

> **注意**
>
> X 协议的默认端口是 `33060`。

要在启动 Shell 时在命令行上使用 URI 连接到服务器，请使用 `--uri` 选项指定它，如下所示：

```bash
$ mysqlsh --uri root:secret@localhost:33060
```

你还可以创建 SSL 连接以实现到服务器的安全连接。要使用 SSL，必须将服务器配置为使用 SSL。要在运行 MySQL 的同一台机器上使用 SSL，可以使用 `--ssl-mode=REQUIRED` 选项。你也可以按照在线参考手册（[`https://dev.mysql.com/doc/mysql-shell/8.0/en/mysql-shell-connections.xhtml`](https://dev.mysql.com/doc/mysql-shell/8.0/en/mysql-shell-connections.xhtml)）所示指定 SSL 选项。可以使用命令行选项在命令行上指定它们，也可以作为 `\connect` Shell 命令的扩展。以下是如何使用 SSL 和命令行选项连接到服务器。

### 连接 MySQL 客户端

虽然 MySQL Shell 已推出数年，并具备包括脚本编写和开发能力在内的诸多功能，但另一个客户端在 MySQL 中已存在数十年。它是一个名为 `mysql` 的应用程序，可让你连接到服务器并在其上运行 SQL 命令。有趣的是，这个 MySQL 客户端最初被称为 MySQL 监视器，但长期以来被简称为“MySQL 客户端”、终端监视器，甚至 MySQL 命令窗口。

你只有在安装 MySQL 服务器（在 Windows 或 macOS 上）或某个软件包组件时才能获得 `mysql` 客户端。例如，在 Linux 上，你可以仅安装客户端软件包，这样也会获得该客户端。我们将把客户端的安装留到后续在本地 PC 上安装 MySQL 的章节。

#### 新的默认认证方式

在 MySQL 8.0.4 版本之前，默认的认证机制使用了一个名为 `mysql_native_password` 插件的认证插件，该插件使用 SHA1 算法。这种机制速度快，且不需要加密连接。然而，由于 NIST 已建议组织停止使用 SHA1 算法，Oracle 在 MySQL 8.0.4 版本中将默认认证插件更改为了 `caching_sha2_password` 插件。

如果你想了解更多关于此变更的信息，包括 Oracle 为何做出此变更以及为用户带来的优势，请参阅 [`https://mysqlserverteam.com/mysql-8-0-4-new-default-authentication-plugin-caching_sha2_password/`](https://mysqlserverteam.com/mysql-8-0-4-new-default-authentication-plugin-caching_sha2_password/)。

要使用 MySQL 客户端 (`mysql`) 连接到服务器，你必须指定一个用户账户和要连接的服务器。如果你连接的是同一台机器上的服务器，可以省略服务器信息（主机和端口），因为它们默认为 localhost 和端口 3306。用户通过 `--user`（或 `-u`）选项指定。你可以在命令行上指定用户的密码，但更安全的做法是指定 `--password`（或 `-p`），客户端会提示你输入密码。如果你确实在命令行上指定了密码，系统会提示一条警告，建议你不要这样做。

在同一台机器上使用 `mysql` 客户端而不指定 `--host`（或 `-h`）和 `--port` 选项时，不会使用网络连接。如果你想通过网络连接或使用不同端口连接，必须使用回环地址。例如，要连接到同一台机器上运行在端口 3307 的服务器，请使用命令 `mysql -uroot -p –h127.0.0.1 --port=3306`。清单 3-3 展示了一个使用 `mysql` 客户端连接（并断开连接）到 MySQL 服务器的示例。

接下来，你看到的是创建数据库和表来存储数据，在表中添加一行数据，最后检索表中的数据。注意我是如何使用大写字母来书写 SQL 命令关键字的。这是一种常见的做法，有助于使 SQL 命令更易于阅读，也更容易找到用户提供的选项或数据。

MySQL 中有非常多的命令。幸运的是，你只需要掌握少数几个更常用的命令即可。以下是你将最常使用的命令。用 `<>` 括起来的部分表示命令中由用户提供的组件，`[...]` 表示需要其他选项：

*   `CREATE DATABASE <database_name>`：创建一个数据库
*   `USE <database>`：设置默认数据库（非 SQL 命令）
*   `CREATE TABLE <table_name> [...]`：创建一个用于存储数据的表或结构
*   `INSERT INTO <table_name> [...]`：向表中添加数据
*   `UPDATE [...]`：更改特定行的一个或多个值
*   `DELETE FROM <table_name> [...]`：从表中删除数据
*   `SELECT [...]`：从表中检索数据（行）
*   `SHOW [...]`：显示对象列表

注意：你必须以分号 (`;`) 或 `\G` 终止每条命令。

尽管此列表只是一个简短的介绍，远非完整的语法指南，但有一份优秀的在线参考手册，其中更详细地解释了每条命令（以及更多内容）。当你对 MySQL 中的任何内容有疑问时，应查阅该在线参考手册。你可以在 [`http://dev.mysql.com/doc/`](http://dev.mysql.com/doc/) 找到它。

其中一个更有趣的命令是允许你查看对象列表的命令。例如，你可以使用 `SHOW DATABASES` 查看数据库，在切换到某个数据库后使用 `SHOW TABLES` 查看表列表，甚至使用 `SHOW GRANTS` 查看用户的权限。我发现我自己经常使用这些命令。

如果你认为 MySQL 的内容远不止这几个简单的命令，那你是对的。尽管它易于使用且启动速度快，但 MySQL 是一个功能齐全的关系数据库管理系统（RDBMS）。它的内容远比你在此处看到的要多。有关 MySQL 的更多信息，包括所有高级功能，请参阅参考手册。
