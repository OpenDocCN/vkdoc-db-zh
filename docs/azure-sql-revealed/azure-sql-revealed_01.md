# 前言

如果在 2018 年底你告诉我，有一天我会为 Bob Ward 所著的书撰写前言，我可能会笑出声来，或许还会加上一句“嗯，得了吧。”那时，我正在阅读 Bob 的书 `Pro SQL Server on Linux`。实际上，我是在**钻研**那本书，试图尽可能多地学习关于 SQL Server 的知识。我尝试了所有的练习，有时甚至重复多次，以确保我真正掌握了*问题*和*解决方案*——幸运的是，这正是 Bob 经常采用的叙事方式。理解这些场景对于我理解 SQL 团队为何以及如何构建产品的各个方面至关重要。

为什么我当时要**钻研** `Pro SQL Server on Linux` 呢？2018 年，我在 Microsoft 内部转岗，加入了 SQL Server 和 Azure SQL 产品组。我对 SQL Server 或管理数据库一无所知。一无所知。Microsoft SQL 副总裁 Asad Khan 给了我这个机会。Bob 的 `Pro SQL Server on Linux` 一书之所以了不起，是因为 Bob “读懂了读者的心思”。他特意花时间解释了 SQL Server 的基本概念，因为 Linux 用户不一定了解 SQL Server。通过使用这本书作为工具，我学到了很多。它为我未来的成功奠定了基础。

2019 年，当 Asad 的职责范围不仅包括 SQL Server，还扩展到了 Azure SQL 和迁移时，这并不令人意外。就在那时，Asad 让我研究向 Azure SQL 的迁移。Bob、Buck Woody 和我一起构建了 SQL Ground to Cloud，这是一套资源（幻灯片、实验），旨在帮助你理解关于 SQL Server 2019、Azure SQL 和迁移的一切。我们进行了一次全球“巡演”，参加了许多 SQL Saturday 活动、会议和 Microsoft 办公室活动，以赋能 Microsoft 内部员工和客户。在 2019 年 7 月到 12 月期间，我们每次在家停留的时间从未超过三天。在此期间，Bob 撰写了 `SQL Server 2019 Revealed`。

对于任何做过演讲的人来说，你知道练习非常有帮助。我认为 Bob 的书之所以既内容丰富又易于理解，其中一个原因在于 Bob 在传达信息方面拥有丰富的经验。例如，在他的 `SQL Server 2019 Revealed` 一书发布之前，毫不夸张地说，Bob 可能已经向超过 10 万人做过超过 50 次相关内容的演讲。这让 Bob 有机会观察哪些内容能引起共鸣，倾听所有问题，并相应地调整他的表达方式。其影响是，你拿起 Bob 的任何一本书来读，包括这一本，都会发现它易于消化，而且通常与你当前可能面临的问题直接相关。

到 2019 年底，我意识到我们需要一种方式，将 SQL Server 的从业人员“带上”云端和 Azure SQL。我曾希望我们能进行一次类似于 SQL Ground to Cloud 的路演，但更侧重于将 SQL Server 的知识转化到 Azure。我记得在 SQL Server 2019 产品发布后，和 Bob 喝了一杯老式鸡尾酒（Bob 的最爱），他告诉我他要开始研究 Azure SQL 了。这真是梦想成真。Bob Ward，这位社区中值得信赖的 SQL Server 专家，要去研究 Azure SQL 了。于是我问 Bob，是否愿意将他深入研究 Azure SQL 的成果，与我合作构建一个叫做“Azure SQL 基础”的内容。Bob 和我一起承担了这个风险，尽管我不确定他当时是否完全清楚自己答应了什么。我们共同制作了一个拥有超过 50 万观看量的 61 集视频系列，一个包含六个模块的学习路径（`https://aka.ms/azuresqlfundamentals`）观看量超过 100 万，以及一个高度完善的演示文稿，我们可以（或任何人，我们已将其公开）用来一次演示一个模块，或者举办为期一天的研讨会。

在此期间，我得以与 Bob 密切合作。Bob 总是极度专注于理解 Azure SQL 和 SQL Server 之间任何差异的**确切细节和原因**。Bob 与产品经理、工程师、前 SQL 团队成员以及现场人员通过数百封电子邮件、会议和 Teams 消息进行沟通，以理解每一个差异和每一个细微差别。他还使用了“信任，但要核实”这句话，我认为这实际上也出现在了这本书里。Bob 拥有极高的尊重和融洽关系，亲眼目睹他如何专业且得体地理解*为什么*以及*如何做*，这非常酷。然后 Bob 会将其转化为易于理解的*问题*和*解决方案*。

在此期间，与他撰写 `SQL Server 2019 Revealed` 时的疯狂状态类似，Bob 撰写了 `Azure SQL Revealed`，你现在阅读的是它的第二版。在这里，Bob 捕捉并提炼了所有对话和学习成果，整合成一个全面的作品。如果你有任何使用 SQL Server、Azure SQL 或 Azure 本身的历史，我保证这将是一本易于阅读、实际上既有趣又引人入胜的书。如果你读过第一版，你可以确信 Bob 已经确保每一个细节都经过了核实或更新，并且 Bob 能提交编辑的最后一天之前的所有新功能都已添加（我们以云的速度前进！）。

我提到了 Asad 和 Bob 如何给了我机会，但我认为更重要的是我们要给自己机会——我们要相信自己。我的行动号召是：**给自己一个机会，拿起这本书**。将其视为一个工具，而不是一本书。完成练习，确保你理解了“核心内容”（如果你还不认识 Bob 的这个短语，你很快就会认识的）。如果你做到了，那些为你专业甚至个人层面打开的机会和大门可能会让你惊喜。而这仅仅是个开始。

Anna Hoffman

Microsoft 首席组项目经理兼 Data Exposed 主持人
