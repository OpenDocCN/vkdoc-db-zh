# 引言




不敢相信距离《深入解析 Azure SQL》第一版出版已经四年了。2023 年我暂停了写作，但在 2023 年 11 月的 PASS 数据社区峰会上，我遇到了 Apress 出版社的 Shaul Elson。他问我是否有兴趣再写一本书。我的第一反应是《深入解析 Azure SQL》第二版，因为这四年间的每一次改进和变化我都亲身经历、全心投入。我知道将所有这些更新整合到书的新版本中会很有吸引力。

本书第一版写于 2020 年疫情期间，全部完成于我位于德克萨斯州北里奇兰山的家中。但 2024 年对我而言是职业和个人生活都频繁出行的一年，所以回顾在奥斯汀、拉斯维加斯（多次）、奥兰多、亚特兰大、达拉斯、范堡罗（英国）、伦敦、都柏林和科克（爱尔兰）、劳德代尔堡、蒙特雷（加利福尼亚州）、西雅图、雷德蒙德（微软园区）、华盛顿特区、纽约市、伯利兹、芝加哥、丹佛和格内西（科罗拉多州）的飞机、汽车、火车、酒店和会议中心里断断续续写成这本书的点点滴滴，感觉很有趣。

这一版的写作意图与第一版相同：我希望从开发人员到管理员再到架构师，所有了解 SQL Server 的 SQL 专业人士都能学习 Azure SQL。我一直倾听各种反馈，无论是正面的还是负面的。我阅读了大量关于第一版的评论反馈。所有反馈都非常积极，其中一些建设性的意见我认为帮助塑造了本书中的一些改动。

阅读本书各章时，请记住以下几点：

*   第一章是“纯粹的历史”。我写的每本书都从历史开始。Azure SQL 历史悠久，因此本章可能显得冗长。如果您对此不感兴趣，可以先跳过这一章，以后再回来看。
*   第二章全是基础知识。如果您认为自己对“什么是 Azure 和 Azure SQL？”的基础知识掌握得很好，那么可以直接阅读其余章节。不过，我认为值得您花时间浏览一下。您永远不知道会发现自己还有什么不知道的。
*   我喜欢在书中收录参考资料。我希望读者有进一步的材料来佐证我的论述。然而，这可能会让一些人对书中出现的 URL 链接感到厌烦。在这一版中，它们仍然保留，但我从第一版中删除了一些我认为没有增加价值的链接。对于电子书的读者，这些链接提供了方便的“跳转”参考以获取更多信息。如果您阅读的是印刷版，有时这些链接可能看起来有些“碍事”，但我相信您会发现这些参考点对于扩展所需知识很有帮助。
*   我仔细审阅了第一版的每一章，并对其进行了全面修改，以确保所有最新细节都是最新的，包括添加了新的图表，删除了不再存在的功能，并增加了新的能力。
*   本书中的示例脚本不像我的其他书那么多。许多示例是使用`Azure Portal`进行的指导性练习。我确实推荐一个我为您构建的带有实验的免费工作坊，您可以在[`https://aka.ms/cloudsqlworkshop`](https://aka.ms/cloudsqlworkshop)使用，以扩展您的实践经验。

牢记这些细节，以下是我关于如何最佳使用和阅读本书的指南。

第三章全面涵盖了 Azure 虚拟机中的 SQL Server，这提供了一种将 SQL Server 迁移到 Azure 的绝佳“直接迁移”方法。本书的其余部分则侧重于两个 PaaS 服务：Azure SQL 托管实例和 Azure SQL 数据库。

第 4 章和第 5 章是部署和配置 Azure SQL 托管实例和 Azure SQL 数据库的指南。即使您已经有部署经验，我保证您仍会在这些章节中发现隐藏的瑰宝。

第 6 至 8 章是 Azure SQL 知识的核心，涵盖了安全性、性能和可用性。这些是我所谓的 SQL“核心内容”，所以我想您会喜欢看到其中的异同。我认为您也会喜欢这些章节中使用托管标识、Copilot 和 Database Watcher 等技术构建的新场景。

第 9 章涵盖了一系列主题，扩展了本书其他地方未涉及的知识，例如如何调度作业、使用数据虚拟化或自己处理 Azure SQL 的问题。

我完全替换了第一版的第 10 章，包含了与我称之为`超越关系数据库`相关的主题，包括生成式 AI 应用程序以及与 Microsoft Fabric 的集成。

如果您想要本书中的任何示例脚本，发现出版后的任何勘误，或想了解任何重要更新，请收藏链接[`https://aka.ms/azuresqlbook2`](https://aka.ms/azuresqlbook2)。

我的个人理念是“赋能他人，成就非凡”，因此这也是我对本书读者的终极目标：我希望用学习 Azure SQL 的知识赋能于您，使您能在云端取得成功。欢迎来到世界的数据库：Azure SQL。

Bob Ward
北里奇兰山，德克萨斯州
2024 年 9 月
