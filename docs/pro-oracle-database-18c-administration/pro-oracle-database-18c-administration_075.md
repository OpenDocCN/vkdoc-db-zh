# Oracle Scheduler 与 cron 对比

DBA 们经常争论在安排和自动化任务时应该使用 Oracle Scheduler 还是 Linux/Unix 的 `cron` 实用程序。以下是 Oracle Scheduler 相对于 `cron` 的一些优势：

*   可以使作业的执行依赖于另一个作业的完成
*   健壮的资源平衡和灵活的调度功能
*   可以基于数据库事件运行作业
*   作业不需要检查数据库是否启动并运行
*   程序的 `DBMS_SCHEDULER` PL/SQL 包语法无论操作系统如何都保持一致
*   可以使用数据字典运行状态报告
*   作业不需要通过 cron 作业传递密码
*   如果在集群环境中工作，无需担心同步集群中每个节点的多个 `con` 表
*   可以通过企业管理器进行维护和监控

Oracle Scheduler 通过 `DBMS_SCHEDULER` PL/SQL 包实现。如前所述，使用此实用程序创建和维护作业相当简单。然而，尽管 Oracle Scheduler 有许多优点，许多 DBA 仍倾向于使用诸如 `cron` 之类的调度实用程序。以下是 `cron` 的一些优势：

*   易于使用；简单、久经考验；创建或修改作业只需几秒钟
*   几乎在所有 Linux/Unix 机器上普遍可用；在很大程度上，无论 Linux/Unix 平台如何，其运行方式几乎相同（是的，存在细微差异）
*   与数据库无关；独立于数据库运行，并且无论数据库供应商或版本如何，其工作方式相同
*   无论数据库是否可用都可以工作

先前的列表并非详尽无遗，但应能让你了解每种调度工具的用途。最好让所有服务器上的作业保持一致，但更好的是拥有集中管理作业和执行的方式，例如 Cloud Control 或企业调度工具。本章后续部分提供了有关如何通过 `cron` 实现和调度自动作业的信息。

## 注意

如果你在 Windows 环境中，请使用任务计划程序实用程序来自动运行批处理作业。你可以通过控制面板，然后进入管理工具来访问任务计划程序。
