# Oracle CDB 与 PDB 架构详解

## CDB 与 PDB 的管理特点

- 一个 CDB 对应一个告警日志和一组跟踪文件。适用于关联 PDB 的任何数据库消息都会写入公共的 CDB 告警日志。

- 每个容器都被分配一个唯一的容器 ID。根容器被分配容器 ID 为 1；种子数据库被分配容器 ID 为 2。每个后续创建的 PDB 都会被分配一个唯一的顺序容器 ID。

- CDB 有一个 FRA。在 FRA 内不会为 PDB 创建单独的目录。RMAN 备份文件、控制文件和在线重做日志都放置在与 CDB 关联的目录中，不会按 PDB 隔离。如果仅备份 PDB，备份文件可以被分离开，但它们仍然位于同一个 FRA 中。

- “闪回数据库”功能通过到根容器的特权连接开启和关闭。你无法在 PDB 级别启用闪回。

- AWR、ADD M 和 ASH 报告是在 CDB 中所有 PDB 范围内发布的。资源消耗按 PDB 识别。

- 在解决 SQL 性能问题时，查询通过视图（如 `V$SQL` 和 `V$SQLAREA`）中的 `CON_ID` 列与特定的 PDB 相关联。

- 数据库保险库等安全选项可以在 PDB 级别启用。CDB 和 PDB 中的权限提供了另一级别的安全性和职责分离。

上述列表很长，包含大量信息，但一旦你消化了 PDB 环境的细微差别，就能有效地实施和管理这项技术。这里的一个要点是，你可以在一个 CDB 内安全地隔离数十个甚至更多的 PDB，而只需管理一个实例（内存和后台进程）、一个重做线程和一个控制文件集。本章还有更多内容需要讨论以填补更多细节。

## 范式转变

通常，特定的应用程序会要求将其数据库对象（用户、表、索引等）放置在与其他应用程序隔离的数据库中。这样做的原因通常是安全或性能问题。在 PDB 出现之前，想想你是如何解决为不同应用程序和开发团队提供独立环境的需求的。两种常见的解决方案如下：

- 为每个需要环境的团队/应用程序创建一个单独的数据库。有时这种方法是每个服务器一个数据库，这通常意味着额外的硬件和许可成本。

- 在一个数据库内创建独立的环境。通常这是通过独立的模式和不同的表空间来实现的。这种方法要求应用程序之间不能有任何数据库对象命名冲突，例如用户名、表空间名和公共同义词等对象。

从 Oracle Database 12c 开始，PDB 为你提供了另一种工具来满足先前的需求。PDB 提供了安全隔离要求；一个 PDB 不能直接访问另一个 PDB。即使一个用户以 `SYSDBA` 特权连接到给定的 PDB，也不能直接 SQL 访问 CDB 中的其他 PDB。这与非 CDB 数据库类似。从安全和应用程序的角度来看，你在更大的 CDB 内拥有完全隔离的 PDB。

作为 DBA，你无需再实施和维护数十个独立的数据库及相关运维任务（例如配置新数据库、安装、升级、调优、可用性、监控、复制、灾难恢复以及备份与恢复），而是可以像管理一个数据库那样管理任意数量的 PDB。从根容器的角度来看，这类似于管理一个非 CDB 数据库。

可插拔架构的另一个显著优势是，PDB 可以轻松地从一个 CDB 克隆或传输到另一个 CDB。这为执行诸如配置新数据库、升级数据库、负载均衡或将应用程序数据从一个环境移动到另一个环境（例如从开发数据库到测试数据库）等任务提供了更多选项。可以通过克隆另一个 PDB 来创建新环境。而将 PDB 从 CDB 中移出，只需通过 SQL 命令将该 PDB 从 CDB 中拔出，然后将元数据和数据文件与新的 CDB 关联（插入）。

如前面章节所多次提到的，新的数据库结构应该是一个至少包含一个 PDB 的 CDB。这支持了新的转变，即让一个 CDB 管理多个 PDB，实现集中资源管理和规划。

## Oracle 云中的 PDB：Exadata Express

测试、开发和开始向云迁移的一个好方法是使用 Oracle 云中的 Exadata Express 服务。这是在 Exadata 上运行的一个 PDB。它具备 PDB 能做的所有事情，本质上是一个平台即服务，具有 Exadata 特性和可用的数据库企业版。这不仅是测试一些应用程序的好方法，而且能让你习惯 PDB 的管理而无需担心其他事情。

基础设施和 CDB 已经准备就绪，配置 PDB 只需要几分钟。维护脚本和应用程序可以快速进行测试，同时使用云服务作为数据库。

这个 PDB 和 Oracle 云的切入点对于熟悉所需服务和管理 PDB 非常有用。添加用户、对象和数据，花时间探索云环境中的 PDB 数据字典。

## 备份与恢复的影响

可插拔环境在备份和恢复架构方面有一些有趣的方面。以下列表重点介绍了这些特性：

- 当以 `SYSDBA` 或 `SYSBACKUP` 特权连接到根容器时，你可以选择通过一个备份命令备份 CDB 内的所有数据文件（根、种子和所有 PDB）。你还可以选择在 PDB 粒度级别执行备份和恢复任务。

- 当直接以 `SYSDBA` 或 `SYSBACKUP` 特权连接到 PDB 时，你只能备份和恢复与当前连接的 PDB 相关联的数据文件；你无法查看或操作根容器数据文件或其他 PDB 数据文件。

- 对整个 CDB 的不完全恢复必须通过以 `SYSDBA` 或 `SYSBACKUP` 特权连接到根容器来执行。在执行整个 CDB 的不完全恢复期间，CDB 内的所有数据文件（根容器和关联的 PDB）都不可用。

- 以 `SYSDBA` 或 `SYSBACKUP` 特权直接连接到 PDB 可以仅对当前连接的 PDB 执行不完全恢复，而不影响 CDB 内的任何其他 PDB。
