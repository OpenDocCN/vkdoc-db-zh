# 术语定义

| 术语 | 描述 |
| :--- | :--- |
| 可插拔数据库 (PDB) | 一套数据文件和元数据，可以无缝地从一个 CDB 转移到另一个 CDB，是面向应用程序的用户数据库。 |
| 根容器 | 一个主数据文件和元数据集合，包含关于 CDB 内所有容器的信息。根容器名为`CDB$ROOT`。 |
| 容器 | 一个数据文件和元数据的集合。可以是根容器、种子容器或可插拔数据库。 |
| 种子可插拔数据库 | 一个用于创建新可插拔数据库的数据文件和元数据模板。种子可插拔数据库名为`PDB$SEED`。 |
| 插入 (Plugging) | 将可插拔数据库的元数据和数据文件与 CDB 关联。 |
| 拔出 (Unplugging) | 将可插拔数据库的元数据和数据文件与 CDB 解除关联。 |
| 克隆 (Cloning) | 从另一个数据库（种子、PDB 或非 CDB）的副本创建可插拔数据库。 |
| `CON_ID` | CDB 中每个容器的唯一标识符。CDB 级别的视图包含一个`CON_ID`列，用于标识正在查看的信息所属的容器。 |
| CDB 数据字典视图 | 包含关于 CDB 内所有可插拔数据库元数据的视图。这些视图只有通过根容器的特权连接查询时，才会显示有意义的信息。可插拔数据库必须处于打开状态才能使用。 |
| 非 CDB 数据库 | 一个未启用可插拔数据库功能创建的 Oracle 数据库（在 12c 之前唯一可用的数据库类型）。 |
