# 可插拔的集装箱：数量优势

**安全性：** CDB 中的 PDB 类似于大型商船上的可插拔集装箱。每个容器都是安全隔离且自包含的。如果你发现自己在一个特定的集装箱内，你不一定知道母船上的其他任何容器。船长（通过 `SYS` 连接到根容器）知道船上的所有容器，并可以通过集装箱货运清单（CDB 级别的元数据视图）查看每个容器的内容。
**粒度：** 如果需要，可以在容器级别执行维修和维护。同样地，你可以将 PDB 离线进行维护或移动，而不会影响 CDB 内的任何其他 PDB。
**协同作用：** 每个容器高效地共享船舶的公共资源，如引擎、船员和导航系统。这类似于 PDB 共享一个公共的 SGA、`UNDO` 表空间、FRA、参数文件、重做日志和归档日志以及后台进程和控制文件。通过共享共同的数据库基础设施产生规模经济；硬件和人力资源（DBA）成本分散到许多数据库上。
**供应：** 在这个类比中，假设你可以轻松复制一个集装箱（可能使用一个巨大的三维扫描仪/打印机）；这将允许你创建现有容器的副本。同样地，PDB 可以从现有数据库（种子、PDB 或非 CDB）中快速高效地创建。
**可移植性：** 当需要时，一个集装箱可以轻松地转移到另一种运输工具上（另一艘船、一列货运火车或由一辆半挂卡车驮运）。同样地，PDB 可以轻松地拔出并插入其他 CDB。目标 CDB 是否与源 CDB 版本和类型相同并不一定重要。
