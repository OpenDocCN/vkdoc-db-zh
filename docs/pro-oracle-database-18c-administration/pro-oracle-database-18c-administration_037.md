# 配置 RMAN

Oracle Recovery Manager (RMAN) 在您安装 Oracle 软件（标准版和企业版）时默认提供。RMAN 提供了一套健壮且灵活的备份与恢复功能。以下列表突出了一些最显著的特点：

*   易于使用的备份、恢复和恢复命令。
*   能够跟踪哪些文件已备份以及备份到何处。
*   管理过时备份和归档日志的删除。
*   并行化：可以使用多个进程进行备份、恢复和恢复。
*   增量备份，仅备份自上次备份以来的更改。
*   能够将增量备份应用到映像副本。
*   支持在数据库、表空间、数据文件、表或块级别进行恢复。
*   高级压缩和加密功能。
*   与介质管理器集成以进行磁带备份。
*   备份验证和测试。恢复验证和测试。
*   跨平台数据转换。
*   数据恢复顾问，可协助诊断故障并提出解决方案。
*   能够检测数据文件中的损坏块。
*   来自 RMAN 命令行的高级报告功能。

本章的目标是提供足够的关于 RMAN 的信息，使您能够就如何实施可靠的备份策略做出合理的决策。首先描述基本的 RMAN 组件，之后您将逐步了解实施 RMAN 涉及的许多决策点。

> 注意
> 本书中与 RMAN 相关的章节并非旨在作为所有备份和恢复方面的完整参考。那将需要一整本书。这些章节包含了成功使用 RMAN 所需的基本信息。如果您需要有关备份、恢复和恢复的高级 RMAN 信息，请参阅 Darl Kuhn, Sam Alapati 和 Arup Nanda 所著的 *RMAN Recipes for Oracle Database 12c* 第二版 (Apress, 2013)。

## 理解 RMAN

RMAN 生态系统由许多不同的组件组成。图 17-1 显示了主要 RMAN 组件的交互。在阅读本节时请参考此图。

图 17-1 RMAN 架构组件

以下列表描述了 RMAN 架构组件：

*   **DBA**：人工交互以确保备份和恢复成功。
*   **目标数据库**：被 RMAN 备份的数据库。您使用 RMAN 命令行的 `TARGET` 参数连接到目标数据库（有关更多详细信息，请参见下一节）。
*   **RMAN 客户端**：您从中发出 `BACKUP`、`RESTORE` 和 `RECOVER` 命令的 `rman` 实用程序。在大多数数据库服务器上，`rman` 实用程序位于 `ORACLE_HOME/bin` 目录中（与其他所有 Oracle 实用程序一起，例如 `sqlplus` 和 `expdp`）。
*   **Oracle 服务器进程**：当您执行 `rman` 客户端并连接到目标数据库时，将启动两个 Oracle 服务器后台进程。第一个默认服务器进程与 PL/SQL 包交互以协调备份活动。辅助轮询进程偶尔更新 Oracle 数据字典结构。
*   **通道**：用于处理正在备份（或恢复）的文件与备份设备（磁盘或磁带）之间 I/O 的 Oracle 服务器进程。
*   **PL/SQL 包**：RMAN 使用两个内部 PL/SQL 包（由 `SYS` 拥有）来执行备份和恢复任务：`DBMS_RCVMAN` 和 `DBMS_BACKUP_RESTORE`。`DBMS_RCVMAN` 访问控制文件中的信息并将其传递给 RMAN 服务器进程。`DBMS_BACKUP_RESTORE` 包执行了 RMAN 的大部分工作。例如，此包创建系统调用，指导通道进程执行 B&R 操作。
*   **内存缓冲区（PGA 或 SGA）**：RMAN 在 PGA（有时在 SGA）中使用一个内存区域作为缓冲区，用于从数据文件读取并将后续块复制到备份文件。
*   **辅助数据库**：一个数据库，RMAN 将目标数据库数据文件恢复到此数据库，用于复制数据库、创建 Data Guard 备用数据库或执行数据库时间点恢复（DBPITR）。
*   **备份**：可以是名词或动词。存储备份文件的物理文件（备份）；或者，复制和归档（备份）文件的行为。备份可以由备份集和备份片或映像副本组成。
