# 系统监控工具简介

每当出现应用程序性能问题或可用性故障时（从数据库管理员的视角看），第一个被问到的问题通常是：“数据库怎么了？”无论问题根源何在，责任往往落在数据库管理员身上，需要确认数据库运行是否良好。我通常通过确定系统范围内的资源消耗情况来处理这个问题。有两个 Linux/Unix 操作系统工具对于显示系统范围内的资源使用情况特别有用：

*   `vmstat`
*   `top`

`vmstat`（虚拟内存统计）工具旨在帮助您快速识别服务器上的瓶颈。`top` 实用程序则提供了系统资源使用的动态、实时视图。这两个实用工具将在接下来的两节中讨论。

## 使用 vmstat

`vmstat` 实用程序显示关于进程、内存、分页、磁盘 I/O 和 CPU 使用情况的实时性能信息。此示例展示了在不指定任何选项的情况下，使用 `vmstat` 显示默认输出：

```
$ vmstat
procs  -----------memory---------- ---swap-- -----io---- --system-- ----cpu----
r  b   swpd   free   buff  cache    si   so    bi    bo   in    cs us sy id wa
14  0  52340  25272   3068 1662704    0    0    63    76    9    31 15  1 84  0
```

在解释 `vmstat` 输出时，可以使用以下一些通用启发式方法：

*   如果 `wa`（等待 I/O 的时间）列的值很高，这通常表明存储子系统过载。
*   如果 `b`（被阻塞的进程）持续大于 0，则可能表示 CPU 处理能力不足。
*   如果 `so`（换出到磁盘的内存）和 `si`（从磁盘换入的内存）持续大于 0，则可能存在内存瓶颈。

默认情况下，运行 `vmstat`（不提供任何选项）时，仅显示一行服务器统计数据。这一行输出给出了自上次系统重启以来计算出的平均统计数据。这对于快速快照来说是足够的。但是，如果您想在一段时间内收集指标，请使用以下语法的 `vmstat`：

```
$ vmstat 间隔秒数 报告次数
```

在此模式下，`vmstat` 会报告统计数据，从一个间隔到下一个间隔进行采样。例如，如果您希望每 2 秒报告一次系统统计数据，总共报告 10 次，可以输入以下命令：

```
$ vmstat 2 10
```

您还可以将 `vmstat` 输出发送到一个文件。这对于分析一段时间的历史性能非常有用。此示例每 5 秒采样一次统计数据，总共生成 60 份报告，并将输出记录到文件中：

```
$ vmstat 5 60 > vmout.perf
```

此外，`vmstat` 实用程序可以与 `watch` 工具结合使用。`watch` 命令用于定期执行另一个程序。在此示例中，`watch` 每 5 秒运行一次 `vmstat` 命令，并在屏幕上高亮显示每个快照之间的任何差异：

```
$ watch –n 5 –d vmstat
Every 5.0s: vmstat                                    Thu Aug  9 13:27:57 2007
procs -----------memory---------- ---swap-- -----io---- --system-- ----cpu----
r  b   swpd   free   buff  cache  si   so    bi    bo   in    cs us sy id wa
0  0    144  15900  64620 1655100   0    0     1     7   16     4  0  0 99  0
```

在 `watch -d`（差异）模式下运行 `vmstat` 时，您将在屏幕上直观地看到从快照到快照的变化。要退出 `watch`，请按 `Ctrl+C`。

请注意，`vmstat` 内存列的默认计量单位是千字节。如果您想以兆字节为单位查看内存统计信息，请使用 `S m`（以兆字节显示统计信息）选项：

```
$ vmstat –S m
```

**提示**
使用 `man vmstat` 或 `vmstat --help` 命令获取有关此实用程序的更多文档。

## 使用 top

另一个用于识别资源密集型进程的工具是 `top` 命令。使用此实用程序可以快速识别哪些进程是服务器上资源的最大消耗者。默认情况下，`top` 会重复刷新（每 3 秒）有关最消耗 CPU 的进程的信息。运行 `top` 的最简单方式如下：

```
$ top
```

以下是输出的一个片段：

```
top - 13:34:32 up 19 min,  2 users,  load average: 0.05, 0.16, 0.24
Tasks: 176 total,   1 running, 175 sleeping,   0 stopped,   0 zombie
Cpu(s):  2.0%us,  0.7%sy,  0.0%ni, 91.6%id,  5.7%wa,  0.0%hi,  0.0%si,  0.0%st
Mem:    787028k total,   748744k used,    38284k free,      1836k buffers
Swap:  1605624k total,    31896k used,  1573728k free,      377668k cached
PID USER      PR  NI  VIRT  RES  SHR S %CPU %MEM    TIME+  COMMAND
4683 root      20   0  279m  14m 7268 S  2.0  1.9   0:01.61 gnome-terminal
3826 root      20   0  218m 9944 4200 S  1.7  1.3   0:02.80 Xorg
3592 oracle    -2   0  601m  18m  15m S  0.7  2.4   0:09.36 ora_vktm_o12c
```

最高消耗会话的进程标识符 (PID) 列在第一列 (`PID`)。您可以使用此信息查看 PID 是否映射到数据库进程（有关将 PID 映射到数据库进程的详细信息，请参阅下一节“将操作系统进程映射到 SQL 语句”）。

在 `top` 运行时，您可以交互式地更改其输出。例如，如果您键入重定向字符 (`>`)，这将使 `top` 排序的列向右移动一个位置。下表列出了一些可用于将 `top` 显示更改为您所需格式的关键功能。

表 21-2：交互式更改 `top` 输出的命令
| 命令 | 功能 |
| :--- | :--- |
| 空格键 | 立即刷新输出 |
| `<` 或 `>` | 将排序列向左或向右移动一个位置。默认情况下，`top` 按 CPU 列排序。 |
| `D` | 更改刷新时间 |
| `R` | 反转排序顺序 |
| `Z` | 切换彩色输出 |
| `H` | 显示帮助菜单 |
| `F` 或 `O` | 选择排序列 |

键入 `q`，或按 `Ctrl+C` 以退出 `top`。下表描述了 `top` 默认输出中显示的几个列。

表 21-3：`top` 输出的列描述
| 列 | 描述 |
| :--- | :--- |
| `PID` | 唯一的进程标识符 |
| `USER` | 运行进程的操作系统用户名 |
| `PR` | 进程的优先级 |
| `NI` | 进程的 Nice 值。负值表示高优先级；正值表示低优先级。 |
| `VIRT` | 进程使用的虚拟内存总量 |
| `RES` | 进程使用的非交换物理内存 |
| `SHR` | 进程使用的共享内存 |
| `S` | 进程状态 |
| `CPU` | 自上次屏幕刷新以来进程消耗的 CPU 百分比 |
| `MEM` | 进程消耗的物理内存百分比 |
| `TIME` | 进程使用的 CPU 总时间 |
| `TIME+` | 进程使用的 CPU 总时间，显示到百分之一秒 |
| `COMMAND` | 用于启动进程的命令行 |

您也可以使用 `b`（批处理模式）选项运行 `top`，并将输出发送到文件以供以后分析：

```
$ top –b > tophat.out
```

在批处理模式下运行时，`top` 命令将持续运行，直到您终止它（使用 `Ctrl+C`）或达到指定的迭代次数。您可以将前面的 `top` 命令与 `nohup` 和 `&` 结合使用，以批处理模式运行它，无论您是否登录系统都能保持其运行。这样做的危险在于，您可能会忘记它，最终创建一个非常大的输出文件（并惹恼系统管理员）。

如果您有一个特定的进程需要监控，请使用 `p` 选项监控某个 PID，或使用 `U` 选项监控某个用户名。您还可以使用 `d` 和 `-n` 选项指定延迟和迭代次数。以下示例监控 `oracle` 用户，延迟为 5 秒，共 25 次迭代：
