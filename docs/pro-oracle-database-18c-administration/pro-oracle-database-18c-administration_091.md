# 22. 可插拔数据库

Oracle 数据库 12c 的新特性是 Oracle 多租户。此功能允许您在一个总体的多租户容器数据库内创建和维护多个可插拔数据库。以下是关于可插拔术语的简明介绍。

一个多租户容器数据库被定义为一个能够容纳一个或多个可插拔数据库的数据库。容器被定义为存在于 CDB 内的一组数据文件和元数据的集合。PDB 是一种特殊类型的容器，可以通过克隆另一个数据库轻松配置。如果需要，PDB 也可以从一个 CDB 转移到另一个 CDB。

CDB 将是数据库资源的容器，您应该将其视为进程、全部内存和 CPU 分配的数据库。每个 CDB 启动时都有一组进程，内存分配给 CDB 供每个 PDB 使用。我们稍后将介绍资源管理，它允许为每个 PDB 分配特定的内存和 CPU；否则，内存和 CPU 将在单个 CDB 内的所有 PDB 之间按需共享。

不是在一个服务器上拥有多个数据库实例，而是可以创建一个或几个 CDB 来容纳所有的 PDB。PDB 彼此之间在数据和用户上是隔离的，因此它们不需要各自拥有自己的 CDB 来实现隔离。

每个 CDB 都包含一组主数据文件和元数据，称为主容器。每个 CDB 还包含一个种子 PDB，它被用作创建其他 PDB 的模板。每个 CDB 由一个主根容器、一个种子 PDB 和一个或多个 PDB 组成。

必须使用 `ENABLE PLUGGABLE DATABASE` 子句创建支持可插拔功能的 CDB。未以此方式创建的数据库不能包含 PDB。在 Oracle 12c 之前，唯一可用的数据库类型是非 CDB，目前仍可以这种方式创建。使用 `dbca` 实用程序时，默认设置是创建一个带有一个 PDB 的 CDB。每个 CDB 包含以下元素：

*   一个根容器，名为 `CDB$ROOT`。根容器包含主数据字典视图，这些视图具有关于根容器以及 CDB 内每个子 PDB 的元数据。
*   一个静态的种子容器，名为 `PDB$SEED`。此容器仅作为模板存在，用于提供创建 CDB 内新 PDB 所需的数据文件和元数据。
*   零个，或一个或多个 PDB（最多 4096 个）。每个 PDB 是独立的，功能类似于一个孤立的非 CDB 数据库。此外，每个 PDB 包含自己的数据文件和应用程序对象（用户、表、索引等）。连接到 PDB 时，无法看到根容器或 CDB 中存在的任何其他 PDB。

Oracle 数据库 12c 新增了一个 CDB 级别的数据字典视图层，它涵盖了 `DBA/ALL/USER` 级别的视图。CDB 级视图报告 CDB 中所有容器（根、种子和所有可插拔数据库）的信息。例如，如果您想查看 CDB 数据库中的所有用户，可以从根容器查询 `CDB_USERS` 来实现。如果您不使用 CDB，那么 `DBA_USERS` 仍然是报告所有用户信息的准确视图。许多数据字典视图现在包含一个名为 `CON_ID` 的新列，它是 CDB 内每个容器的唯一标识符。根容器的 `CON_ID` 为 1。种子的 `CON_ID` 为 2。在 CDB 内创建的每个新可插拔数据库都会被分配一个唯一的顺序容器 ID。
