# 调优的细微差别

在可插拔环境中进行调优带来了新的挑战，但与在具有多个数据库实例的服务器上调优类似。优势在于，CDB 拥有资源并能查看其他已插入并消耗资源的 PDB 的活动情况。性能调优可以从 PDB 级别开始，然后继续到 CDB 级别，以理解其他 PDB 的活动。此外，还可以根据不同的服务级别、优先级或数据层来管理 PDB 的资源。

AWR、ADDM 和 ASH 报告现在显示 PDB 级别的资源使用情况；这将使你能够将调优工作集中在适当的 PDB 上。对于你编写的任何自定义调优 SQL 查询，你需要在适当的地方修改它们以报告 CDB 级别的视图。同样，在分析 SQL 语句时，你必须考虑该 SQL 语句是在哪个 PDB 中执行的。

本章前面的部分重点讨论了可插拔架构。在创建 CDB 之前，理解这些架构基础至关重要；因此，本主题的介绍篇幅较长。现在你已经掌握了可插拔基础，可以准备将理论概念转化为可用且可持续的数据库技术了。手头的第一个实际任务是创建一个 CDB。
