## 创建远程物化视图刷新

您可以创建从远程表、物化视图或视图中选择的物化视图，或这些对象的组合。这样做可以快速高效地复制数据。基于远程对象创建物化视图的设置如下：

1.  确保从复制的数据库环境到包含主表的数据库之间存在 Oracle Net 连接。如果没有此连接，则无法使用物化视图进行复制。
2.  获取对远程数据库中用户帐户的访问权限，该帐户应有权访问您想要复制的远程表、物化视图或视图。
3.  对于快速刷新，请在主（基）表上创建物化视图日志。仅当您打算执行快速刷新时才需要执行此操作。
4.  在复制的数据库环境中创建一个数据库链接，指向主数据库。
5.  在复制的数据库环境中创建物化视图，通过步骤 4 中创建的数据库链接访问远程主对象。

这里是一个简单的示例。首先，确保您可以从复制环境建立到主数据库的 Oracle Net 连接。您可以通过从复制的数据库环境使用 `SQL*Plus` 连接到远程主数据库来验证连接并确保可以登录到主数据库。在将要包含物化视图的数据库的命令提示符下，尝试连接到名为 `ENGDEV`、位于 `XENGDB` 服务器上的主数据库中的用户 `REP_MV`：

```
$ sqlplus rep_mv/foo@'xengdb:1522/engdev'
```

当您连接到远程主数据库后，您就可以访问作为物化视图基础的表。在此示例中，远程主表的名称是 `SALES`：

```
SQL> select count(*) from sales;
```

接下来，在将包含物化视图的数据库中创建一个数据库链接。该数据库链接指向远程主数据库中的用户：

```
SQL> create database link engdev
connect to rep_mv identified by foo
using 'xengdb:1522/engdev';
```

现在，创建一个访问主 `SALES` 表的物化视图：

```
SQL> create materialized view sales_mv
refresh complete on demand
as
select
sales_id
,sales_amt
from sales@engdev;
```

您可以通过在表名后附加 `@<database_link_name>` 来访问远程数据库。这指示 Oracle 从远程表中进行选择。远程表的位置在 `CREATE DATABASE LINK` 语句中定义。

## 理解远程刷新架构

您可以使用多种配置进行远程刷新的物化视图。本节详细介绍了三种常见场景；您可以基于它们进行构建以满足大多数远程复制需求。

图 15-3 展示了一种常见配置，它在主 OLTP 数据库上使用物化视图日志。远程数据库使用物化视图日志来实现快速刷新。当您担心报表活动会严重阻碍生产性能而无法直接从 OLTP 数据库进行报表时，通常使用此配置。当您在地球另一端有用户，并且希望将数据复制到物理上更靠近他们的数据库以获得可接受的报表性能时，此架构也很有用。

图 15-3 在主站点使用物化视图日志的远程刷新

图 15-4 说明了一种场景，您不被允许在主基表上创建物化视图日志。这可能是因为另一个团队或组织拥有主（基础）数据库，并且所有者不愿意让您在主环境中创建物化视图日志。在这种情况下，您必须对远程报表数据库使用完全物化视图刷新。当基表记录中很大比例每天被修改时，此架构也是合适的。在这种情况下，完全刷新可能比快速刷新更有效（因为您正在复制大部分数据，而不仅仅是一小部分）。

![../images/214899_3_En_15_Chapter/214899_3_En_15_Fig4_HTML.jpg](img/214899_3_En_15_Fig4_HTML.jpg)
图 15-4 使用完全物化视图刷新的远程刷新

图 15-5 展示了一种场景，您将基表复制到一个临时数据库，然后从临时数据库复制到报表数据库。当网络架构配置使得报表数据库位于无法直接连接到加固的生产环境的网段上时，这种情况很常见。在这种情况下，您可以构建一个中间数据库，该数据库驻留在可以连接到 OLTP 数据库和报表数据库的网络中。请注意，物化视图日志是构建在安全临时数据库中的物化视图上的。在此配置中刷新时，您必须协调临时数据库和报表数据库的刷新时间，以确保在刷新期间没有重叠。

## 查看物化视图基表信息

当您诊断物化视图的问题时，查看物化视图及其关联的远程主表很有用。在包含物化视图的数据库上运行以下查询以提取主所有者和表信息：

```
SQL> select
owner        mv_owner
,name         mv_name
,master_owner mast_owner
,master       mast_table
from dba_mview_refresh_times
order by 1,2;
```

前面的查询报告每个物化视图及其所基于的主表。基表可以是本地的，也可以是远程的。

## 确定有多少物化视图引用中央物化视图日志

假设您有一个带物化视图日志的主表。此外，多个远程物化视图使用中央主物化视图日志。图 15-6 说明了这种配置。

![../images/214899_3_En_15_Chapter/214899_3_En_15_Fig6_HTML.jpg](img/214899_3_En_15_Fig6_HTML.jpg)
图 15-6 多个远程物化视图使用同一集中式物化视图日志

在这种情况下，Oracle 会在物化视图日志中保留记录，直到所有物化视图都已刷新。例如，假设物化视图 `A` 的 `LAST_REFRESH_DATE` 为 2013 年 7 月 1 日，物化视图 `B` 的 `LAST_REFRESH_DATE` 为 2013 年 9 月 1 日。然后，物化视图 `A` 在 2013 年 10 月 1 日刷新。主日志仅清除早于 2013 年 9 月 1 日的记录（因为较新的日志记录仍被物化视图 `B` 需要）。

如果某个物化视图被删除并且无法从主物化视图日志表中注销自己，那么主物化视图日志表中的记录将无限增长。要解决此问题，您需要有关哪些物化视图与哪些物化视图日志相关联的信息。此查询显示所有依赖物化视图的主表所有者信息和 `SNAPID`（物化视图 ID）：

```
SQL> select mowner
,master base_table
,snapid, snaptime
from sys.slog$;
```

以下是一些示例输出，显示两个物化视图连接到一个物化视图日志：

```
MOWNER          BASE_TABLE               SNAPID SNAPTIME
--------------- -------------------- ---------- ---------
INV_MGMT        PRODUCT_TAXONOMY            653 28-JAN-13
INV_MGMT        COMPANY_ACCOUNTS            650 28-JAN-13
INV_MGMT        CMP_GRP_ASSOC               651 28-JAN-13
```
