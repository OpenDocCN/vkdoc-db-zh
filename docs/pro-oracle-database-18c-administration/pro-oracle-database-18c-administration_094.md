# 理解可插拔架构

PDB 与非 CDB 数据库环境有一些重要的架构差异。图 22-2 展示了一个名为`CDB`的容器数据库，它包含一个根容器、一个种子数据库和两个名为`SALESPDB`及`HRPDB`的 PDB。

![../images/214899_3_En_22_Chapter/214899_3_En_22_Fig2_HTML.jpg](img/214899_3_En_22_Fig2_HTML.jpg)

**图 22-2** 可插拔数据库架构

花点时间看看图 22-2。如果你是一个好奇的 DBA，很可能立刻涌现出许多想法。以下列表重点介绍了理解图 22-2 中新架构需要理解的一些关键点：

*   连接到`CDB`数据库等同于连接到`CDB$ROOT`根容器。根容器的主要目的是为任何关联的 PDB 提供资源和存放元数据。
*   你可以通过`SYS`用户访问根容器，就像访问非 CDB 数据库一样。换句话说，当登录到数据库服务器时，你可以使用操作系统认证直接连接到根容器，无需指定用户名和密码（`sqlplus / as sysdba`）。
*   种子 PDB (`PDB$SEED`) 仅作为创建新 PDB 的模板存在。你可以连接到种子，但它是只读的，这意味着你不能对其执行事务。
*   除了两个默认容器（根和种子）外，这个特定的 CDB 还手动创建了两个额外的 PDB，名为`SALESPDB`和`HRPDB`（本章后面会详细介绍创建 PDB）。
*   PDB 存在于独立的命名空间中。PDB 在 CDB 内必须是唯一的，但 PDB 内的对象遵循非 CDB 数据库的命名空间规则。例如，表空间名称和用户名必须在各个 PDB 内唯一，但在 CDB 内的其他 PDB 之间不必唯一。
*   每个 PDB 都有自己的`SYSTEM`和`SYSAUX`表空间，以及可选的`TEMP`表空间。
*   如果 PDB 没有自己的`TEMP`文件，它可以使用根容器`TEMP`文件中的资源。
*   每个 PDB 的`SYSTEM`表空间包含有关 PDB 元数据的信息，例如其用户和对象；这些元数据可通过 PDB 内的`DBA/ALL/USER`级视图访问，并可通过根容器的 CDB 级视图看到。
*   CDB 可以容纳不同字符集的 PDB（自 12.2 起）。
*   你可以为 CDB 和所有关联的 PDB 设置时区，也可以为每个 PDB 单独设置时区。
*   CDB 实例在通过`SYS`连接到根容器时启动和停止。你不能在连接到 PDB 时启动/停止 CDB 实例（系统 DBA 和应用程序 DBA 的职责分离）。
*   实例启动时读取一个初始化参数文件。连接到根容器的特权用户可以修改所有初始化参数。相比之下，连接到 PDB 的特权用户只能修改适用于当前连接的 PDB 的参数。
*   当连接到 PDB 并修改初始化参数时，这些修改仅适用于当前连接的 PDB，并在数据库重启后对该 PDB 持久生效。`V$PARAMETER`中的`ISPDB_MODIFIABLE`列显示了哪些参数可以在以 DBA 身份连接到 PDB 时修改。（锁定参数和配置还有额外的安全权限。）
*   应用程序用户只能通过网络连接访问 PDB。因此，必须有一个侦听器正在运行，并侦听与关联 PDB 对应的服务名。如果没有运行侦听器，应用程序用户就无法连接到 PDB。
*   单个 PDB 本身不会停止或启动（不是数据库实例的术语）。当你启动/停止 PDB 时，你并不是在分配内存或启动/停止后台进程。相反，PDB 要么变为可用，要么不可用（打开或关闭）。
*   CDB 有一组控制文件。控制文件在以特权用户身份连接到根容器时进行管理。
*   CDB 有一个`UNDO`表空间。CDB 中的所有 PDB 使用相同的`UNDO`表空间（如果是 RAC，则每个实例一个活动的 undo 表空间）。
*   有一条重做线程（每个实例），在以具有适当特权的用户身份连接到根容器时进行管理。只有连接到根的特权连接才能启用归档或切换联机日志。具有 SYSDBA 特权的用户连接到 PDB 时不能更改联机重做或归档设置。
