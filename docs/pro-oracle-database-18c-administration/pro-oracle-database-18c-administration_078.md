# 理解 Cron 表条目

你的 `cron` 表是一个包含数字和命令的列表，`cron` 后台进程（`cron` 守护进程）将按照指定的时间和计划运行它们。`crontab` 工具要求条目遵循明确定义的格式。在你的 `crontab` 文件开头添加一个注释行来记录所需格式是个好主意，像这样：

```
# 分(0-59) 时(0-23) 日(1-31) 月(1-12) 周(0/7-6) 命令或脚本
```

在上面的例子中，`cron` 文件中的井号 (`#`) 代表注释的开始。在 `#` 之后输入的任何文本都会被 `cron` 忽略。

`crontab` 中的每个条目是一个由六个字段组成的单行。前五个字段指定了执行时间和频率。这些字段中的条目可以用逗号或连字符分隔。逗号表示一个条目的多个值，而连字符表示一个值范围。条目也可以是一个星号 (`*`)，表示所有可能的值都生效。这里有一个例子来帮助你理解。下面的条目在每周一至周五，从早上 8 点到下午 4:30，每半小时发送一封内容为“Wake up”的电子邮件：

```
0,30 8-16 * * 1-5 echo "wake up" | mailx -s "wake up" dkuhn@gmail.com
```

在某些 Linux 系统上，你可以在范围内的值后面跟上 `/<整数>` 来跳过一个值。例如，如果你想每隔一分钟运行一次任务，可以在分钟列中使用 `0-59/2`。你也可以将斜杠 (`/`) 与星号一起使用来跳过值。例如，要每四分钟运行一次任务，你可以在分钟列中使用 `*/4`。

`crontab` 中的第六个字段可以是一个或多个 Linux 命令或一个 shell 脚本。或者换句话说，第六列可以是任何你能在 Linux 命令行上在一行中运行的命令或脚本的组合。

`cron` 工具有几个需要进一步解释的特殊之处。第五列是星期几。星期日用 0 或 7 表示；星期一用 1 表示；星期二用 2 表示；依此类推，星期六用 6 表示。

第二列中的小时数采用军用时间格式，范围从 0 到 23。第四列（月份）和第五列（星期几）可以用数值或三个字母的缩写来表示。例如，下面的 `crontab` 条目使用了月份和星期几的三字母缩写：

```
0,30 8-16 * Jan-Dec Mon-Fri echo "wake up" | mailx -s "get up" dkuhn@gmail.com
```

此外，列之间似乎有重叠，比如第三列（日期）和第五列（星期几）。这些列允许你为需要在每月 1 号和 15 号或每周二等时间表上运行的任务创建灵活的计划。在你没有使用的列中填入一个星号。如果你需要在 1 号、15 号以及每周二运行一个任务，那么就同时填写这两列。

如果你从 `cron` 运行一个包含对 Oracle 实用程序（如 `sqlplus` 或 `rman`）调用的 shell 脚本，请确保你在脚本内实例化任何所需的 OS 变量，例如 `ORACLE_SID` 和 `ORACLE_HOME`。如果你没有源引（source）这些变量，当你的 shell 脚本从 `cron` 运行时，你会看到如下错误：

```
sqlplus: command not found
```

当 `cron` 运行一个脚本（在用户的 `crontab` 中）时，它不会运行用户的启动或登录文件（例如 `.bashrc`）。因此，任何（从 `cron` 运行的）脚本都需要显式设置任何所需的变量。你可以直接在脚本中设置变量，或者调用另一个导出这些变量的脚本（例如 Oracle 的 `oraenv` 脚本）。

> **提示**
> 不要将你在 `cron` 中设置的任务都安排在同一时间运行。相反，将它们分散开，以免在任何特定时间点拖慢 `cron` 或系统。
