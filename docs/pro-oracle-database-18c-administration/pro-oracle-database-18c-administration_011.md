# OS 效率工具：在压力下快速定位环境

当你处于一个紧张的环境时，拥有一个能让你快速辨别当前位置、所用账户的环境，以及拥有能帮助你快速识别问题的工具至关重要。本章所描述的技术就像杠杆：它们为快速完成大量工作提供了助力。这些工具让你能够专注于可能面临的问题，而不是验证你的位置或担心命令语法。

本章首先详细介绍实现最高效率的操作系统技术。后续章节将展示如何使用这些工具自动显示环境细节、导航文件系统、主动监控数据库以及进行故障分诊。

> **提示**
> 在数据库服务器上工作时，请始终使用一个操作系统 Shell。
> 我推荐使用 Bash Shell；它包含了其他 Shell（如 Korn 和 C）的所有最有用功能，并且还有一些额外功能，进一步增强了其易用性。

## 定制操作系统命令提示符

通常，数据库管理员（DBA）需要操作多个服务器和多个数据库。在这种情况下，你的屏幕上可能打开了许多终端会话。你可以运行以下类型的命令来识别当前工作环境：

```bash
$ hostname -a
$ id
$ who am i
$ echo $ORACLE_SID
$ pwd
```

为了避免混淆正在操作的服务器，通常需要将命令提示符配置为显示有关其环境的信息，例如主机名和数据库 SID。在这个示例中，命令提示符被自定义为包含主机名、用户和 Oracle SID：

```bash
$ PS1='[\h:\u:${ORACLE_SID}]$ '
```

`\h` 指定了主机名。`\u` 指定了当前操作系统用户。`${ORACLE_SID}` 包含了你的 Oracle 实例标识符的当前设置。以下是此示例的命令提示符：

```
[oracle18c:oracle:o18c]$
```

该命令提示符包含了关于环境的三个重要信息：服务器名、操作系统用户名和数据库名。当你在多个环境之间导航时，设置命令提示符可以成为一个宝贵的工具，帮助你跟踪所在位置和所处环境。

如果你希望登录时操作系统提示符能自动配置，则需要在启动文件中进行设置。在 Bash Shell 环境中，通常使用 `.bashrc` 文件。该文件通常位于你的 `$HOME` 目录中。在 `.bashrc` 文件中添加以下代码行：

```bash
PS1='[\h:\u:${ORACLE_SID}]$ '
```

当将此代码行放入启动文件后，每次登录服务器时，你的操作系统提示符都会自动设置好。在其他 Shell（如 Korn Shell）中，启动文件是 `.profile`。

根据个人偏好，你可能需要修改命令提示符以满足特定需求。例如，许多 DBA 喜欢在命令提示符中显示当前工作目录。要显示当前工作目录信息，可以添加 `\w` 变量：

```bash
$ PS1='[\h:\u:\w:${ORACLE_SID}]$ '
```

可以想象，命令提示符中显示的信息有多种选项。这是另一种流行的格式：

```bash
$ PS1='[\u@${ORACLE_SID}@\h:\W]$ '
```

表 3-1 列出了许多可用于自定义操作系统命令提示符的 Bash Shell 变量。

**表 3-1** 用于自定义命令提示符的 Bash Shell 反斜杠转义变量

| **变量** | **描述** |
| --- | --- |
| `\a` | ASCII 响铃字符 |
| `\d` | “星期 月份 日期”格式的日期 |
| `\h` | 主机名 |
| `\e` | ASCII 转义字符 |
| `\j` | Shell 管理的作业数量 |
| `\l` | Shell 终端设备的基本名称 |
| `\n` | 换行符 |
| `\r` | 回车符 |
| `\s` | Shell 的名称 |
| `\t` | 24 小时制 HH:MM:SS 格式的时间 |
| `\T` | 12 小时制 HH:MM:SS 格式的时间 |
| `\@` | 12 小时制 am/pm 格式的时间 |
| `\A` | 24 小时制 HH:MM 格式的时间 |
| `\u` | 当前 Shell 用户的用户名 |
| `\w` | 当前工作目录的完整路径 |
| `\W` | 当前工作目录的基本名称 |
| `\!` | 此 Shell 中的历史命令编号 |
| `\#` | 此 Shell 的命令编号 |
| `\$` | 如果有效用户 ID 为 0，则为 `#`；否则为 `$` |
| `\\` | 反斜杠本身 |
| `\[` | 非打印字符序列的开始（用于在提示符中包装颜色转义码） |
| `\]` | 非打印字符序列的结束 |
