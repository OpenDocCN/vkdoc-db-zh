# 总结与环境配置

## 总结

在安装好 Oracle 二进制文件后，你就可以创建数据库了。在创建数据库之前，请确保已正确设置必需的操作系统变量。你还需要一个初始化文件并预先创建任何必要的目录。你应仔细考虑哪些初始化参数应设置为非默认值。通常，我尝试尽可能多地使用默认值，仅在有充分理由时才更改初始化参数。如果执行过多手动流程和步骤，则需要重新审视该流程。对于最新版本的数据库，许多环境变量已设置好；使用配置助手（如 `dbca` 和 `netca`）时会自动创建目录。使用响应文件是自动化创建流程的另一种方式。

本章重点介绍了使用 SQL*Plus 创建数据库。这是一种高效且可重复的创建数据库方法。在编写 `CREATE DATABASE` 语句时，请考虑数据文件和联机重做日志的大小，以满足数据库的放置和存储需求。内部参数和大小调整应作为数据库内部知识的一部分被理解，以便于后续故障排除和其他配置。使用最新版本的新特性将提高数据库的效率。某些环境可能仍在使用旧版本，这使得理解创建数据库所需的内部知识变得更加重要。

我曾在一些环境中工作，管理层规定每台服务器只能有一个数据库；除非这是一个包含多个可插拔数据库的容器数据库，否则服务器上存在未利用的资源。一台内存容量大、CPU 多的快速服务器应能托管几个不同的数据库。在决定一台服务器上放置多少个数据库时，你必须确定哪种架构能满足你的业务需求。

创建数据库后，下一步是配置环境，以便能够高效地导航、操作和监控数据库。这些任务将在下一章中描述。

## 3. 配置高效环境

安装好 Oracle 二进制文件并创建数据库后，你应该配置你的环境以实现高效操作。无论图形化数据库管理工具的功能如何，DBA 仍然需要从操作系统命令行执行许多任务并手动执行 SQL 语句。善于利用操作系统和 SQL 的 DBA 比不善利用的 DBA 具有明显的优势。

在任何数据库环境（Oracle、MySQL 等）中，有效的 DBA 会利用高级操作系统特性来快速导航目录、定位文件、重复命令、显示系统瓶颈等。要实现这种效率，你必须熟悉承载数据库的操作系统。

除了精通操作系统，你还必须熟练掌握进入数据库的 SQL 接口。尽管你可以从图形界面获取大量诊断信息，但 SQL 使你能够更深入地探索内部机制，进行高级故障排除并获取数据库情报。

本章为高效使用操作系统和 SQL 来管理数据库奠定了基础。你可以使用以下操作系统和数据库特性来配置你的环境以提高效率：

*   操作系统变量
*   Shell 别名
*   Shell 函数
*   Shell 脚本
*   SQL 脚本
