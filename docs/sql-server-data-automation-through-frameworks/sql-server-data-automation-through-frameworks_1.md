# 引言

框架与人类构建事物的历史一样悠久。当人们开始使用计算机开发解决方案时，框架也随之出现。如同木工夹具，框架的存在是为了简化工作并加速解决方案的开发过程，无论是完成一个木工项目还是交付一个企业数据集成解决方案。

## 本书适合谁阅读？

任何对数据相关自动化或通过单条命令执行多个代码块感兴趣的人都能从本书中找到价值。三类读者将尤其受益：

1.  数据库开发人员
2.  数据工程师和数据集成开发人员
3.  商业智能标记语言（`Biml`）开发人员

数据库开发人员将学习如何通过存储过程驱动数据库对象执行，并会看到一个数据库框架的示例。

数据工程师和数据集成开发人员将学习如何使用 `SQL Server Integration Services`（`SSIS`）来实现一个元数据驱动的数据集成框架。

`Biml` 开发人员将看到一个使用 `BimlExpress` 和元数据在选定的设计模式下快速生成多个 `SSIS` 包的示例，*以及*一个通过 `Biml` 查询平面文件以生成平面文件格式和平面文件连接管理器的示例。

## 本书结构

在接下来的章节中，我们将探讨本书的章节布局。本书分为两部分，引导你完成使用存储过程和 `SSIS` 框架构建数据库框架的过程。每一部分都由若干章节组成。

## 第一部分：基于存储过程的数据库框架

在本书的第一部分，我们将重点介绍如何使用存储过程构建数据库框架。这些章节主要面向开发数据库解决方案的人员。我们并不试图创建涵盖数据库开发各个方面的综合性入门读物；但我们很好地涵盖了基础知识。

### 第 1 章：存储过程 101

在我们深入数据库开发之前，让我们先介绍一些存储过程的基础知识。

### 第 2 章：使用存储过程实现自动化

一旦你牢固掌握了存储过程，我们将利用所学知识，使用控制器模式（执行一个或多个存储过程）来实现自动化执行。

### 第 3 章：存储过程协调器

在本章中，我们将探讨使用协调器模式（执行一个或多个控制器存储过程）进行高层执行管理。

### 第 4 章：一个基于存储过程的元数据驱动框架

元数据驱动框架是本书反复出现的主题。在本章中，我们将存储有关存储过程、控制器和协调器的信息，以执行框架应用程序。

## 第二部分：SSIS 框架

如果你的企业只执行几个 `SSIS` 包，则不需要 `SSIS` 框架。然而，如果你的企业要执行数百（或数千）个 `SSIS` 包，你*绝对*需要一个 `SSIS` 框架。

#### 第 5 章：一个简单的、自定义的、基于文件的 SSIS 框架

在本章中，我们定义了一个数据集成/工程框架元数据数据库，用于管理配置和执行。我们介绍了一个 `SSIS` 框架的版本，用于管理在本地执行的 `SSIS` 的这些主题领域。

### 第 6 章：框架执行引擎

在本章中，我们以 `Information` 和 `Error` 事件的形式添加了插桩。内置的 `SSIS` 日志记录将显示这些消息，这些消息在排查 `SSIS` 框架应用程序问题时非常有用。

### 第 7 章：框架日志记录

在本章中，我们将执行元数据持久化到“实例”表中。`ApplicationInstance` 表为每个 `SSIS` 框架应用程序执行实例存储一条记录。`ApplicationPackageInstance` 表为每个 `SSIS` 框架应用程序包执行实例存储一条记录。重要的执行实例属性——如开始和结束时间、执行状态——都在“实例”表中维护。

### 第 8 章：Azure-SSIS 集成运行时

在本章中，我们通过逐步介绍配置流程，引入 `Azure Data Factory`（`ADF`）和 `Azure-SSIS` 集成运行时。

#### 第 9 章：部署一个简单的、自定义的、基于文件的 Azure-SSIS 框架

在第 5-7 章中，我们设计并构建了一个针对本地 `SSIS` 执行、配置和日志记录的 `SSIS` 框架。在本章中，我们开始迁移在第 5-7 章中设计的 `SSIS` 框架：首先配置一个新的 `Azure SQL` 数据库，然后将元数据数据库部署到我们的新 `Azure SQL` 数据库中。我们配置一个 `Azure 文件共享`，并将测试 `SSIS` 包迁移到 `Azure 文件共享`。最后，我们开始构建 `Azure Data Factory` 父管道，该管道将作为 `ADF` 版本 `SSIS` 框架的执行引擎。

### 第 10 章：ADF 中的框架日志记录

在本章中，我们向父 `ADF` 管道添加日志记录功能，非常类似于我们在第 7 章为本地版本的 `SSIS` 框架添加的功能。

### 第 11 章：ADF 框架中的容错

在本章中，我们通过实现容错来完成 `ADF` 执行引擎的功能，该容错将根据 `SSISConfig` 元数据配置以编程方式停止（或不停止）管道执行。



### 结论

本书适合希望更深入钻研 SQL Server 与 SSIS 自动化的读者。我们将探讨并演示数据库与集成服务框架，并涵盖诸如 SQL Server、SSIS、Azure 和 Azure 数据工厂等主题。

如你所见，前方是一段充满学习与新见解的漫长旅程。让我们开始吧！
