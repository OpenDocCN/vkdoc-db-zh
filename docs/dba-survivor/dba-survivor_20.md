# 第三章：一些基础知识

当人们经历这三个阶段时，他们的反应各不相同。想想你可能已经认识的以下几类人群：

*   创新者
*   早期采用者
*   早期大众
*   晚期大众
*   落后者
*   顽固派

听起来是不是就像你认识的几乎每个人？而且很有可能，这些分类也适用于你自己，只不过针对不同的事物，你可能属于不同的类别。也许你永远要等到第一个服务包发布后，才会升级到新版本的软件。听起来耳熟吗？但或许你也会第一个排队购买最新的 iPhone。在某些情况下你拥抱变化，在另一些情况下则更为谨慎。

一个人对最熟悉的事物所赋予的价值，将引导他们拥抱或抗拒变化。思考一下。如果你赋予已知事物的价值高于未知事物，难道你不会抗拒改变吗？你所支持的最终用户也是如此。如果你提议他们开始以许多不同的方式做事，你最好准备好向他们展示未来的状态将比现状有价值多少。

你为什么排队购买新款 iPhone？因为你认为新型号比你当前的手机更有价值。你为什么在考虑升级软件前，一定要等到第一个服务包发布？因为你认为当前系统的稳定性比新版本更有价值。

作为一名数据库管理员，你将在许多场合被要求领导变革。而且，在某些场合，你不会被*要求*领导变革——你需要主动从你的小隔间里站起来，无需他人告知就自己动手去做。当那一刻到来时，请遵循这些简单的准则：

1.  从组织的角度定义需求。
2.  从个人（或最终用户）的角度定义需求。
3.  描述期望的结果。
4.  定义一个时间表。

相信我，这需要练习。很少有人天生具备那种能让人轻松拥抱变革的软技能。但对于那些天生具备或后天习得这些技能的人来说，结果是强大的——能激励和鼓舞人们走向新的境地。

你还应该意识到抵制行为的类型。有两种：消极抵制和积极抵制。消极抵制是指一个人口头同意你所说的一切，但从不履行他们同意的事，或者假装无知，也许还隐瞒信息。积极抵制是指一个人寻找缺陷并嘲笑变革；操纵他人并诉诸恐惧感。

如果你在与人打交道时意识到这些行为，你就可以在问题变得更大之前就将其扼杀。

### 拥有标准和流程并非坏事

在某个时候，你不可避免地会遇到有人表示不理解你为什么以特定的方式做事。这很可能是因为你已经实施了一些标准，以帮助你提供更稳定的环境。而这些人会目瞪口呆地发现：（1）你需要有这样的标准，以及（2）这个标准竟然真的要适用于*他们*。

例如，假设你是一个之前没有数据库管理员的新单位的数据库管理员。每个人都对数据库服务器拥有完全访问权限，可以为所欲为。说一团糟可能都太客气了。作为优秀的数据库管理员，你检查了清单，找到了几种可以立即改进并给整个公司带来巨大利益的方法。

首先，你逐一检查每个服务器和数据库，将数据和日志文件正确地放置在不同的驱动器上，以避免重大灾难。在移动文件的过程中，你撤销了除你自己之外任何人创建新数据库的权限。你这样做是因为你对问题的根本原因分析得出结论：有如此多的数据库设置不正确，是因为太多人不了解其行为的后果。为了帮助他们避免伤害自己，你做出了必要的更改。

在你开始撤销权限大约一小时后，你接到了一位开发人员的电话，他不明白为什么自己不能创建数据库了。“不就是个数据库吗——我自己能创建，为什么还要打电话叫你来创建？” 你会忍住想要回应说“如果你自己能搞定，那我就不用花一周时间跟在你们屁股后面收拾烂摊子了，就像独立日后跟在马后面清扫的人一样”的冲动，并会尽力解释：为了拥有一个稳定的环境，数据库创建权掌握在少数人手中比掌握在多数人手中更好。

试着去理解，大多数人在日常生活中持有的是一种横向视角或体验。开发者需要上班，登录网络，访问数据库服务器，进行更改，测试他们的更改（希望如此），并将更改部署到生产环境。为什么这是一种横向视角？嗯，想想他们一天中经历的所有纵向壁垒：他们开车上班，停车，进入大楼（设施）。他们登录网络（域控制器，网络）。他们访问数据库服务器进行更改（你这里）。他们测试更改（获取客户确认）。他们将更改部署到生产环境（又是你这里）。

你与他们进入大楼或访问网络通常无关，但你对他们如何被允许更改数据库服务器确实有一定控制权。在这种情况下，你的视角是纵向的：你主要关心自己的领域（数据库服务器），完全不知道横向视角是什么样子。这可能是一件很难调和的事情；你真的需要与你的最终用户合作，并经常与他们沟通，以理解他们所有的痛点。

一旦你开始理解你的大多数最终用户都是从横向视角看问题，就很容易明白为什么当你开始强制执行政策、指南和标准时，他们可能会感到沮丧。记住，人们可能抵制变化，并利用这个机会来领导变革。随着时间的推移，你的软技能会提高到一定程度，人们会开始相信你的行为是为了公司的整体利益，而不是针对任何个人。

### 人们会责备他们不理解的事物

这一点相当不言自明。人们总是倾向于责备他们知道但不完全理解的东西。这很自然，对吧？我的意思是，如果你认为自己对十件事中的九件都已经了如指掌，那么你的注意力就会集中在第十件事上，并且你会花太多时间去思考为什么这第十件事此刻让你如此头疼。

你会经常听到这样的话：“我们的代码多年没变过了”，“昨晚一切都运行良好”，以及“你们昨天肯定做了什么，因为现在我们所有东西运行得都慢多了”。习惯成为任何系统任何问题的焦点吧。开始培养一些厚脸皮，因为你很快就会需要它。
