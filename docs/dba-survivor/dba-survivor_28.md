# 第四章：开发服务器对开发者而言就是生产服务器

另一个缓解压力的关键在于识别那些你感到压力并知道需要起身做些能让你精力充沛的事情的时刻。这对每个人来说并不相同；你需要找到适合自己的方法，并在压力症状出现时能够识别它们。

现在，我经常被问到关于缓解压力和时间管理的这个问题：“但是，从工作中抽出时间难道不是在浪费宝贵的工作时间，并且让在规定时间内完成任务变得更困难吗？” 绝对不是。缓解压力能让你保持高效；当你高效时，你也同时具备效率；当你有效率时，你就在明智地最大化利用你的时间。道理就是这么简单。你是愿意花一个小时拖拖拉拉地完成一些任务，还是愿意花 30 分钟呼吸新鲜空气，然后用清晰的头脑再花 30 分钟完成任务？

当我担任篮球教练时，我总是强调，如果我们在心理、身体和情感上都做好了准备，那么我们在任何一个晚上都将难以被击败。你在工作时也是如此。如果你感到压力袭来，并以某种方式消耗着你，那就找些能让你重新振作起来的事情，你会惊讶于自己能变得多么高效。

### 待办清单

我在研究生院时开始使用待办清单。出于某种原因，我开始写下一天中我需要处理的所有事情。大事如考试，小事如“去吃午饭”。因此，我的清单平均大约有十几项或更多，并且通常是按照我期望在一天中完成的顺序写下的。每完成一项，我就会把它从清单上划掉。任何未完成的事项都会被移到明天的清单上。

我这样做了三年，离开研究生院后就停止了。从那以后，我断断续续地使用它。在我人生的某些阶段，我感到需要清单来帮助我保持条理。这反过来又降低了我的压力水平，而我们已知道这对时间管理很重要。而且，当你划掉清单上的一项时，感觉总是很棒。

在我只有四五件事情要处理的时期，我发现不用待办清单、仅靠例行安排来管理时间会更容易。每当我开始感觉自己承担了超出管理能力的事情时，我就会拿出一些纸，开始把事情写下来。一旦你把所有事情都写在一张纸上，它真的能帮你让所有工作看起来更易于管理。看似压倒性的工作量，可以开始变得像是你能够轻松应对的事情。

这个技巧并不适合所有人。我妻子尝试过清单，但效果不如她希望的那样好。但这里重要的是她尝试过了，对吧？我鼓励你也去尝试，你可能会惊讶于它能带来的帮助。

### 任务分块

有时你会发现自己“有心情”一次性完成很多事情。也许你有一个任务清单，但你决定在一两个任务上多做一点，而不是继续推进。有些人可能会开始觉得他们会落后于清单上的事项。然而，另一些人则会因为知道在某项任务上取得了更多进展而感觉良好。如果你清单上有一个你知道是持续性的任务，这种方法尤其有用。

虽然你可以专注于一项任务的特定分块，但你也可以在一天中处理多项任务的小块内容。对某些人来说，整天处理各种各样的任务能让他们保持投入和精力充沛。有很多天，我喜欢同时处理很多事项，在每件上都完成一点点。当然，也有其他一些天，我喜欢非常专注于某一个特定的事项。

我有效地管理我的时间，使用了刚才描述的每一种技巧。嗯，除了这些，还有一种我们尚未讨论的技巧。

### 学会拒绝

你去过自助餐厅吃饭吗？如果去过，你是否曾发现自己品尝了很多不同的食物，每次只拿一点点——但当你回到餐桌时，盘子里已经有了大约 5 磅的食物？你知道这太多了，但你觉得必须吃掉你拿的所有东西——否则就是浪费。嗯，当涉及到你的工作时，也会发生同样的事情。你最终会一件接一件地堆积小任务，不知不觉中，你就一次性地承担了远超出你舒适处理能力的事情。

我知道我经常承担了太多的副项目。我相信大多数人在专注于两件或更少的事情时，可以发挥出最高的效率。当然，我每天要处理的事项远不止两件，但当我盯着电脑屏幕说“我从哪里开始？”时，这会让人不堪重负。这时候，我就会拿出待办清单，让所有事情变得可管理；我可以为手头的任务排定优先级，将它们分解成我想要的任何大小的分块，建立例行安排，并在需要提神时去慢跑一下。

## 代码评审与克里斯·汉森

好的代码评审是一项必要的“邪恶”。它们应该在固定的时间间隔进行，也许是在项目的里程碑或关卡处。代码评审是你向同行解释你思维过程的时间，同时也是就你的代码和设计获得反馈的时间。最终的结果是更好的代码，这带来更稳定的系统，从而减少生产支持问题。那么，为什么大多数公司都不愿进行代码评审呢？

因为每个人都害怕代码评审。

大多数人都不擅长展示。更糟的是，他们知道自己不擅长，这让他们表现得更差。有些人可能本来可以做得好，但在一群同行面前讲话时会紧张。而那些代码被评审的人，感觉就像是被《to Catch a Predator》节目中的克里斯·汉森审问一样。所有这些加起来，就成了你能想象到的最令人畏惧的员工集会。

所以我们知道代码评审很重要，对吧？而且我们知道每个人都害怕它们，结果就是现在没人再做代码评审了，对吧？现在，我希望你想象一下，是克里斯·汉森在主持代码评审，而你就是当前正在做展示的开发者。你刚解释完你在做什么，克里斯开始问你一些问题，比如：

*CH*: 你知道`DTS`有多旧了吗？你是怎么想的？而且你不打算批量处理你的事务吗？你知道这会对你的日志文件造成什么影响吗？

*你*: 我发誓，伙计，那只是说说而已，仅此而已。我没打算做任何事。我来这里是为了告诉我的`DBA`我们需要分道扬镳了。

*CH*: 只是说说？这话说得可不少。我这里有完整的记录。你在这里说，“`I want to cursor through all your rows`”。伙计，这简直*大错特错*。

*你*: 我知道，我知道。我正在寻求帮助。前几天我买了一本关于`SQL Server 2008`的书。而且我愿意尽我所能来帮助你们。告诉我你们希望我做什么。

*CH*: 帮助我们？

*你*: 是的，无论需要做什么。

*CH*: 门在那儿。去告诉你的朋友们，我们在看着。下次他们给我们一份比`NASA`发射流程还复杂的部署说明时，我们会去找他们的。
