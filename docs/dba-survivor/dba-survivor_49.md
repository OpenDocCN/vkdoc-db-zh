# 第 8 章：培训，提升自我

### 认证

认证本身并非一种资源；你从上述渠道获得的培训才是你将（与工作经验一起）用于获得认证的实际资源。将认证视为目标，而前述资源则是实现目标的手段。切勿相信仅凭认证就能轻松获得名利。

**提示：** 认证本身证明力有限。认证与工作经验相结合，才能讲述一个更完整的故事。

**优点：** 认证能极大增强你现有的技能，并且与你的经验结合后，能向他人传达你的技能组合。

**缺点：** 它可能会被负面看待，尤其是当你持有很多认证但缺乏经验时。市场上存在*大量*的“题库速成”机构，提供获取认证的捷径；长远来看，这可能会削弱人们对认证价值的信心。

## 构建你的论证基础

你现在应该已经认识到培训的必要性，并列出了一些可能的培训资源。让你的经理相信派你去参加一些培训课程是合乎逻辑的，这应该相当容易，对吧？

错了。

切勿将逻辑与`商业智慧`混为一谈。你的经理很可能也有他的经理，而那位经理无论培训费用高低，都会希望为这笔开支找到一些正当理由。很可能需要由你提出一个坚实的商业论证，才能确保获得这笔资金。

要为培训构建一个坚实的商业论证，首先要理解企业最关心什么：`收入`、`支出`和`利润`。尽管我们都希望热爱自己的工作，并在一个充满欢乐、友爱、独角兽和彩虹的环境中工作，但事实是金钱让世界运转。如果你想确保获得培训经费，那么你需要展示企业通过花费这笔钱将如何获益。如果这听起来像是我在说“赚钱需要本钱”，那确实如此。但这没关系，因为大多数公司正是遵循这一理念运作；你需要做的，就是向他们展示如何实现。

**提示：** 你需要与之讨论资金问题的职位越高，就越重要的是，你能证明他们为你所请求的任何投资带来财务回报。

以下是一份需要开始收集信息的具体项目清单：
1.  确定你的培训需求。
2.  确定有多少人需要培训。
3.  确定将使用哪些选项。
4.  计算相关的成本/支出。
5.  确定培训的收益。
6.  确定成本节约。
7.  衡量你的成果！

### 确定你的培训需求

你需要清晰地识别你的培训需求，并且这些需求应以某种方式与业务目标或计划保持一致。例如，假设公司正致力于将数据库服务器虚拟化。你可以列出对虚拟化某个方面的培训需求。或者，也许你想学习某个特定工具，而你的公司也正希望构建一个数据仓库。如果你的培训需求与任何业务需求不符，那么你拥有的就不是培训`需求`，而是培训`愿望`。公司没有理由仅仅满足你的愿望；必须在某种程度上对公司有利。

## 确定有多少人需要培训

这在后续权衡培训选项和计算成本时将非常重要。有时你会发现，如果你能证明培训也能使他人受益，而不仅仅是你自己，那么获得培训批准会更容易。记住，你总是在为培训经费而竞争——你的同事也想要那笔培训资金。你可能需要通过承诺在培训归来后培训其他员工来为你的培训预算做辩护。此外，如果只有你一个人参加了培训，那么你可能是唯一掌握该知识的人，结果可能是你发现自己需求激增，这很容易侵蚀你的工作/生活平衡（如果它存在的话）。

### 确定将使用哪些选项

回顾之前列出的所有可用培训选项，看看哪些是可行的。

哪些选项能满足你的需求，并容纳所有需要培训的人？

你或许可以混合搭配几个选项。也许一个人可以参加课程，另一个人可以参加会议，还有一个人可以找一个副业项目进行一些自我培训。然后你们三人可以聚在一起交流、学习和分享经验。

在我们的团队中，我们倾向于让每个成员专注于数据库管理的一个特定方面。其他成员知道谁专注于什么，并在必要时利用这些知识。随着时间的推移，我们在许多不同领域都变得知识更丰富一些。这减少了全年派我们所有人参加许多不同培训课程的需求。

## 计算相关的成本/支出

开始累加你能想到的所有相关成本。容易识别的成本是那些最大的开销，例如课程费用、会议费用或聘请培训师现场授课的费用。差旅和住宿也是培训支出中容易识别的成本。一些较难识别的隐性成本包括：如果你必须开车去培训中心，可能会产生汽车里程费；或者购买先决培训材料，如书籍。还有你离开办公室的时间，你可以用损失的工作小时数来表示。如果培训班上人数众多，你可能更容易用全职人力当量（`FTE`）来表示这个数字。

**提示：** 没有任何支出是微不足道的；写下你能想到的所有可能的支出项目。

尽你所能考虑每一项可能的支出。你可能无法涵盖所有项目，但你需要让你的估算尽可能接近最终账单。

## 确定培训的收益

首先写下你能想到的培训可能带来的每一个可能的好处。不要像列出培训需求时那样，将自己局限于仅与业务相关的结果。一些好处可能包括：降低员工流失率、减少使用系统时出现的错误，或者仅仅是让你有能力与其他专业人士建立联系。
