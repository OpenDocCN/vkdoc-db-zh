# 第五章：生产支持

当然，你可能会漏接一个电话，这一点毫无疑问。但当你想想一天内收到的电子邮件数量与电话数量的对比，你就会发现电话是*不同寻常*的东西，你会更注意电话铃声，而不是一封电子邮件。

如果我们假设有人打电话并留言，你有多长时间来响应？答案当然是，你的响应时间由 SLA（服务级别协议）来定义。

这意味着它取决于紧急情况的性质。也就是说，你需要建立一个流程来满足可能的最高 SLA 响应时间，而凌晨三点的一封邮件很可能是不够的。

**提示：** 我不确定你怎么样，但我喜欢睡觉。当你开始构建系统并定义 SLA 时，请记住其他人也同样喜欢睡觉。

如果你有幸能够雇用一个电话应答服务，我建议你这样做。这样你就能放心，总会有人接电话，并且他们会立即尝试联系你或你的团队。无论你是否拥有电话应答服务，你都应该确保只使用一个电话号码作为主要联系电话。

一个包含上述许多细节的流程示例如下：

1.  客户拨打生产支持电话号码。
2.  电话由应答服务接听。
3.  记录留言，包括客户详细信息。
4.  应答服务拨打指定的电话号码。
5.  电话由负责支持的 DBA 接听。

### 发布流程

你需要发布你定义的**生产支持**流程。它必须被清晰地传达，并且需要经常传达。无论你向客户传达了多少次这个流程，总会有人说他们对这个流程一无所知。记住，*态度要友好*。

寻找有创意的方式来宣传支持流程。与其发送详细描述流程的全公司范围的电子邮件，你可以更巧妙一些，只需在电子邮件签名中包含正确的电话号码。你也可以将流程发布到内部网站上，并在适当时引导人们访问该链接。另一个选择是发布关于你每月处理支持呼叫量的报告，并将流程详情包含在报告中（而不是仅仅报告数字并假设每个人都知道流程）。

**提示：** 即使你发布十遍这个信息，人们仍然会忽略或否认看过。当他们声称不知道时，请耐心对待，只需再次将材料转发给他们。

你需要在任何可能的地方发布，定期（以某种方式）提醒人们，并确保人们被通知到这个流程，尤其是在流程发生变更时。

## 提醒和执行

你需要不断提醒人们你的**生产支持**流程。大多数人不会关注他们不经常使用的任何流程。如果你的系统运行良好，那么人们就不需要联系你寻求帮助，这意味着他们很可能不像应有的那样熟悉这个流程。

除了提醒人们这个流程外，你还需要**执行**这个流程。

这正是态度友好能带来好处的地方。如果有人联系你进行生产支持但没有遵循既定的流程，你有两个选择。你可以帮助他们并提醒他们流程，或者你可以先提醒他们流程，并要求他们遵循流程后你再提供帮助。

虽然你可能认为正确的选择是先帮助这个人，然后提醒他们正确的流程，但请停下来想一下这个场景。如果他们不需要遵循流程而你还是帮助了他们，下次他们有什么理由要改变做法呢？记住，如果你的系统运行良好，那么大多数人不需要经常联系你。那么，他们到底什么时候才会开始熟悉这个流程呢？

尽管这样做可能很痛苦，甚至冒着你的人缘变差的风险，但有时最好提醒人们正确的流程。通常我会权衡手头的情况，以决定是执行流程，还是允许这个人完全绕过流程。无论我选择哪一种，我都尽力保持友好的态度。

## 工作/生活平衡

在设计**生产支持**流程时，重要的是要记住你的工作之外还有生活。未能认识到这一事实，可能会导致你设计出一个长期来看不可持续的流程。即使你认为自己没有个人生活，也请尽力假设你有；否则，你的流程设计将建立在“你随时有空”的假设之上。这意味着你是一个单点故障，这将导致在某个时间点为某人提供糟糕的服务。

一个往往被忽视的方面是知道何时说“不”的能力。在当今的企业文化中，许多人觉得当被要求承担额外任务时，他们不被允许说“不”。如果你经常说“不”，那么很快人们就不再问你了，不久之后，你的服务将不再被需要。

从工作职责中抽身休息一下是很重要的。这可以让你，可以说，给自己充充电。你可以降低压力水平，得到一些必要的休息，当你重返工作岗位时，会发现自己精神焕发。但如果工作要求太高，你随时都需要待命，你该如何平衡工作和生活呢？

## 沟通

就像生活中的大多数事情一样，良好的沟通对于一切顺利运行至关重要。这里也不例外。在努力平衡工作和个人时间时，你、你的经理和你的家人之间的良好沟通至关重要。

如果你需要加班或在周末工作，那么一旦你知道了就通知你的家人。你能做到足够及时，达到他们的期望吗？不，不一定。但至少你在尝试这样做。我发现与家人共享一个日历很有用，这样当我知道我的工作职责会占用家庭时间时，我总能了解家人的活动安排。

**提示：** 知道何时说“不”，但对你的家人永远要说“是”。永远如此。

如果你的家人需要你，那么一旦你知道了就告诉你的经理。如果你需要早退，比如孩子病了，就告诉你的经理。如今大多数公司都提供所谓的弹性工作时间；没有打卡钟，所以如果你只是偶尔需要早退，你可以这样做，只要之后补上工作时间。

### 保护你的私人时间

当你在家，既不需要待命，也没有工作事务需要回应时，请把黑莓手机放到另一个房间。相信我，如果你直到早上才阅读或回复邮件，世界不会因此毁灭。如果事情真的重要，应该有人打你的手机，而不是发邮件，所以只需确保你能听到电话铃声。

你的工作表现应该与可量化的成果挂钩，而不是你投入的工时数量。如果似乎只看重工时，那么你会发现个人时间越来越少，而公司也很可能从你的表现中获得不理想的结果。让人们知道，当你在家时，你就是在家。许多人相信，通过保护家庭生活，他们最终能提升在正常工作时间内产出的质量。

## 愉快的玩乐时间

你需要保持低压力水平，而做到这一点的方法之一是参与你觉得有趣和愉快的活动。有爱好吗？每周留出一些时间去追求它。找到那些对你重要的事情；重要到值得你专门留出时间的事情。你是掌控自己日程的人，所以要善用这一点。

我寻找的是能让我充满活力的事情。它可以是任何事——写作、调优查询、慢跑——无论什么。重点是，我知道有些活动能在一天或一周的不同时间吸引我，这些活动给我能量，而不是让我在身体、精神和情感上筋疲力尽。

你并不总是能为工作和生活活动分配等量的时间。但你*可以*做的是，只为那些能让你焕发活力的工作和生活活动找时间。

工作和生活是流动的事件——有时你需要多做一方而少做另一方，这通常让你感到沮丧。但如果这些活动*给予*你能量，那么你就不太可能感到倦怠。

你的日程安排应该反映这一点；每天给自己留一些回旋余地，以应对工作或生活超出预期的情况。这样，你就永远不会觉得遗漏了什么。更重要的是，当你完成其中一方的任务时，你会感到充满活力，而不是精疲力尽。你应该感到自己有所成就，并期待下一个任务。

**作者：乔纳森·詹尼克**

我每天开始工作时，都会为当天创建一个待办事项清单。几年前，我会试图把自认为能完成的所有事项都塞进清单里。别这么做。这是个陷阱。正如汤姆指出的，工作日绝不会完全按计划进行，你需要为错误留出一些余地。我现在仍然创建每日清单，但会留出充裕的余地来处理一天中出现的所有临时任务。待办清单给了我一个目标——如果我能完成它——就能让我自豪地回家，感觉付出了良好努力并取得了一些切实成果。

工作质量差

担心失业

## 工作质量

说到外包，许多人最关心的问题是担心工作质量差。对许多人来说，这无疑是真实的。但我们决定探究为何会有质量差的看法。我们四处打听，寻找其他使用外包的公司，试图了解对每个特定公司来说，哪些做法有效，哪些无效。

我们发现，被识别为“操作型”的任务通常被认为完成质量高。被定义为“非操作型”的任务通常被认为质量差。相信我，就是这么简单。只需要四处打听，和一些同行交流，弄清楚哪些有效、哪些无效，并将它们分成两组。

作为一名数据库管理员，我们有许多本质上被认为是操作型的任务。对我来说，操作型任务是可以通过重复遵循的事情——它很少改变，结果通常是已知的。考虑数据库备份。配置备份——确保它们成功完成——是一项操作型任务。数据库恢复也是。密码重置、生产环境变更部署、数据迁移，甚至一些基本的故障排除也是如此。

**提示：** 如果你已经很好地记录了流程，那么很容易判断外包是否导致了质量下降。

第一步是准确记录这类任务在你的环境中是如何完成的。一旦有了这份文档，你就将其交给你的外包团队，给他们一个提供同等水平支持的机会。要么他们能做到，那就很好；要么他们做不到，你就继续寻找其他人。

如果这一切最坏的结果是你最终花时间记录了一些日常任务，那么无论如何你都会处于更好的状态。毕竟，你知道自己已经在工作重压下挣扎，需要他人的帮助。为了确保任何新员工能在最短时间内上手，将流程写下来是必要的。

每当我们注意到我们的离岸团队无法为客户提供所需的支持水平时，我们的反应不是说“我们需要找更好的人”。我们的第一反应是：“我们如何能简化流程并改进我们的文档？”

随着时间的推移，我们能够精简流程并改进文档，我们的离岸团队也能够为我们归类为操作型的所有任务提供必要的支持。
