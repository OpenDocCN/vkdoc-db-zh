## 示例中使用的数据

请记住，示例中生成的数据是随机生成的。我尽了最大努力去除了重复项和无意义的组合。在汇总账户余额和手头库存数量的章节中，我也非常自由地进行了处理，以便演示函数的行为。

例如，在财务数据库示例中：
- 假设月度值是月底扣除所有贷项和借项后账户中剩余的金额。这适用于与账户和投资组合相关的表。
- 假设滚动月度值将当月（扣除所有借项和贷项后）的值与上个月的值相加，以表示当月的总价值。

在某些情况下，我应用了以下 `ROWS` 窗口规范来处理两个月的滚动场景：

```
ROWS BETWEEN 1 PRECEDING AND CURRENT ROW
```

例如，在现实生活中，一个账户在 1 月可能有 100 美元，在 2 月有 50 美元。如果您认为总额是 150 美元，但如果 1 月或 2 月发生了借项，那就错了。如果没有发生借项，那么 150 美元就是总额，因为它包括了从 1 月结转到账户的现金。因此，有几个因素在起作用，比如账户的借项和贷项。

这是另一种情况。如果 2 月从账户中借记了 50 美元，并且没有向账户存入任何款项，那么剩下的就只有 50 美元（当月至今）。

同样的处理方式也适用于投资组合余额（在财务数据库章节）以及处理库存移动的三章。与财务场景一样，库存需要考虑进出库的流动，以实际报告月度库存总额。

代码有详细的注释，所以一切应该都很清楚。目标是理解窗口函数的工作原理，而不是模拟真实世界的业务场景或现实的数据模式。

### DBCC

我们使用这个简单但重要的工具来清除查询用于存储数据和计划的内存缓存。每次需要进行性能分析时，请先运行此命令，以便清除缓存并获得准确的一组统计信息。请参考代码清单 14-1。

```
DBCC dropcleanbuffers;
CHECKPOINT;
GO
```
代码清单 14-1
运行 DBCC

如果不运行此工具，数据和计划信息将存储在内存中，因此您正在分析的查询的后续运行将返回不正确的结果（就 `IO` 和 `TIME` 统计信息而言），因为该查询利用了内存中的数据和查询信息。

### IO 和 TIME 统计信息

在我们的性能调优会话中，我们多次依赖这组统计信息。它们告诉我们发生了多少操作，例如磁盘扫描和物理读取，以及查询解析和运行所需的时间。

注意
解析步骤是查询引擎确保语法正确并知道需要检索的数据位置所在的时候。还会创建称为解析树的对象。这就是当我们查看计划时估计查询计划工具所显示的内容。请参考代码清单 14-2。

```
SET STATISTICS IO ON
GO
SET STATISTICS TIME ON
GO
```
代码清单 14-2
开启 IO 和 TIME 统计信息

在运行 `DBCC` 后运行这些命令以开启统计信息，然后执行查询。当您完成并希望停止显示统计信息时，将 `ON` 更改为 `OFF`。请参考图 14-18。

![](img/527021_1_En_14_Fig18_HTML.png)
一个 SSMS 窗口的截图。左侧面板展示了对象资源管理器。右侧面板有代码以及 IO 和 TIME 统计信息。CPU 和已用时间的解析与编译时间均为 8 毫秒。CPU 和已用时间的执行时间均为 0 毫秒。影响了 12 行。
图 14-18
IO 和 TIME 统计信息
