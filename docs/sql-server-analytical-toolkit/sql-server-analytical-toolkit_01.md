# 引言

欢迎阅读我的书，*SQL Server 分析工具集*。

## 本书内容

这是一本关于如何在各行业中应用 Microsoft SQL Server 聚合、分析和排名函数的书，旨在使用一系列内置的 SQL Server 函数——亲切地称为窗口函数！——进行统计、报表、分析和历史性能分析。

不，不是像 C# 或其他 Microsoft Windows 应用程序编程中使用的那种窗口。它们被称为窗口函数，是因为它们在查询生成的数据集上实现了“窗口”。这些窗口通过在查询数据集中创建分区，来控制函数应用于数据的位置。

## 什么是分区？

你可能会问：“什么是分区？”这是一个关键概念，理解它才能最大限度地从本书获益。假设你有一个数据集，其中产品类别 A 有六行，产品类别 B 有六行。每一行都包含一个你希望分析的销售额的列。该数据集可以划分为两个部分，每个产品类别一个。这些就是窗口函数所使用的分区。你可以通过应用窗口函数来分析每个分区（更多内容见第 1 章）。

我们将看到，每个分区中的窗口可以进一步划分为更小的窗口。窗口帧机制使你能够控制分区中哪些行被提交给窗口函数，相对于当前正在处理的行。例如，将像 `SUM()` 这样的函数应用于当前正在处理的行以及分区中的所有先行行，以按月计算累计总额。移动到分区中的下一行，其行为相同。

本书重点介绍如何在四个关键行业应用这些函数：销售、金融、工程和库存控制。我这样做是为了让这些行业的读者可以在日常工作中找到熟悉的内容。即使你不在这四个行业工作，你仍然可以通过学习窗口函数及其应用方式而获益。

也许你想面试金融领域的开发职位？或者你从事工程、电信行业，或是零售产品的制造商。本书将帮助你获得一些宝贵的技能，助你通过工作面试。

虽然你可以使用像 Power BI 这样的工具来执行这些函数，但在 SQL 层面执行这些函数可以预先计算结果并提高性能，从而让报表工具使用预先计算好的数据。

顺便说一句，市面上有很多关于 SQL Server 和窗口（或窗口化）函数的书。这本书有什么不同？


## 方法

本书采用烹饪手册式的方法。不仅向你展示如何使用这些函数，还教你如何将它们应用于销售、财务、库存控制和工程场景中。

这些函数被分为三类，因此，对于我们审视的每一个行业用例，我们将为每个函数类别分配一章：

*   聚合函数
*   分析函数
*   排名函数

对于每个函数，都会创建并解释一个查询。接下来，会审查和分析结果。

在适用的情况下，结果会用于在 Microsoft Excel 电子表格中生成一些有趣的图表，例如为销售数据创建正态分布图。

附录 A 包含了这些函数的描述和语法，以防你不熟悉它们，所以在深入阅读本书前可以随意查阅。

掌握本书概念的关键在于理解 `OVER()` 子句的作用。第 1 章首先定义了 `OVER()` 子句是什么，以及它如何与窗口函数一起使用。

几张图表清晰地解释了数据集、分区和窗口框架是什么，以及它们如何是使用窗口函数的关键。

我们之前确定的每个行业都有三个专门的章节，分别对应一种窗口函数类别。每个章节都提供了要编写的查询的规范、满足该规范的代码，然后是一个或多个展示结果的图表。

本书的独特之处在于，它不仅仅展示每个函数的工作原理；还提供了与统计分析、数据分析和 BI（顺便说一下，BI 代表商业智能）相关的用例场景。

本书还提供了所有代码示例，包括通过出版商的 Google 网站创建和加载四个数据库的代码。

最后，书中包含了足够的理论知识，为你介绍统计分析，以防你不熟悉标准差、均值、正态分布和方差等术语。这些知识很重要，因为它们将为你提供宝贵的技能，以支持你的业务用户并增强你的技能组合。嘿，一点商业理论总没坏处！

附录 B 有一篇简短的统计学入门，所以如果你对这些主题不熟悉，请务必查看一下。它讨论了标准差、方差、正态分布、其他统计计算以及钟形曲线。

回到窗口函数。这些函数生成大量数值数据。生成带小数点的数字固然好，但更有趣的是将其可视化成图表并理解其含义。一图胜千言。看到一张显示销售额逐月下降的图表绝对值得一看，并且应该敲响警钟！

你还可以使用 Excel 电子表格，通过其内置函数来验证你的结果，确保它们与你的查询结果一致。始终用一组已知正确的结果来测试你的数据（你可能还会顺便学到一点 Microsoft Excel 的知识！）。本书中使用的电子表格也将在出版商的 Google 网站上提供。

本书包含提示和讨论，引导你完成学习 SQL Server 聚合、排名和分析函数的过程。这些内容采用循序渐进的方式呈现，以便你能轻松掌握概念。数据结果经过分析，以便你理解函数的功能以及窗口如何用于分析数据工作。

除了各章节中讨论的查询外，还包含了用于创建四个练习数据库并加载其中表的 DDL（数据定义语言）代码。这些脚本清晰地指明了创建和加载所有数据对象所需的步骤与顺序（其中一些会生成数百万行数据）。

一旦数据库创建完成且表加载完毕，你就可以在阅读各章节的同时，通过执行提供的查询进行练习。你也可以通过创建索引并检查 SSMS 生成的可用查询计划来练习性能调优。

最后，我也会稍微涉及性能调优，以确保你创建的查询能够快速高效地运行。你将学习的函数是资源密集型的，因此我们需要编写不会过度占用服务器资源（如 CPU、内存和磁盘）的查询！（或者避免让用户因为一个查询要运行数小时或数天而抓狂……）。

作为附加价值，你将看到如何应用良好的编码标准来提高代码的可读性。易于阅读的代码也易于维护、修复和修改。我们希望成为好队友，以防你的代码在你升职（或找到更好的工作后）交给另一位开发人员！基于这些原因，我们将使用微软公司为 SQL Server 建议的编码标准。

这些标准易于学习和采用。所有 `TSQL` 命令和关键字均使用大写，引用的数据对象名称由一个或多个描述该数据对象的单词组成。构成数据对象名称的单词的首字母大写，其余字母小写，例如用于部门标识符的 ``DeptId``。

遵循此编码标准的示例查询如下：

```sql
SELECT DeptId, MgrId, MgrLastName
FROM Department
GO
```

非常简单且易于阅读。从所使用的名称可以清楚地看出，该查询检索部门信息，具体是部门标识符、经理标识符和经理的姓氏。注意表的名称。毫无疑问，该表包含什么内容。
