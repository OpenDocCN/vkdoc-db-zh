# 工作原理

并行 DML 可适用于任何 DML 操作——`INSERT`、`UPDATE`、`MERGE` 和 `DELETE`。根据你运行的 DML 操作类型，规则略有不同。例如，若想并行运行 `INSERT` 语句，首先需为会话启用并行处理，然后使用适当的机制执行 `INSERT` 语句，以使 DML 能够并行运行：

`ALTER SESSION ENABLE PARALLEL DML;`

`INSERT /*+ PARALLEL(DEPT,4) */ INTO DEPT SELECT /*+ PARALLEL(DEPT_COPY,4) */ * FROM DEPT_COPY;`

通过上述语句，我们在 `INSERT` 语句中加入了一个并行提示，同时在该语句的 `SELECT` 部分也加入了一个并行提示。需要记住一个非常重要的点：即使你的 DML 语句启用了并行处理，它也不会直接影响同一语句中相关查询的并行性。例如，以下语句的 DML 操作可以并行运行，但相应的 `SELECT` 语句将以串行模式运行，因为查询本身未指定并行性。

`ALTER SESSION ENABLE PARALLEL DML;`

`INSERT /*+ PARALLEL(DEPT,4) */ INTO DEPT SELECT * FROM DEPT_COPY;`

为了充分利用并行能力，尝试将语句的所有部分都并行化。如果你并行化了 `INSERT` 但没有并行化 `SELECT`，那么 `SELECT` 部分就成了 `INSERT` 的瓶颈，导致 `INSERT` 性能下降。

并行 DML 操作也适用于 `UPDATE`、`MERGE` 和 `DELETE` 语句。假设你的公司很慷慨，决定给会计部门的每个员工加薪 1%：

`UPDATE /*+ PARALLEL(EMP,4) */ EMP SET SAL = SAL*1.01 WHERE DEPTNO=10;`

然后，几个月后，你的公司决定解雇那些在会计部门加过薪的员工：

`DELETE /*+ PARALLEL(EMP,4) */ FROM EMP WHERE DEPTNO=10;`

在数据库中并行化 DML 事务的另一种方法是使用 `DBMS_PARALLEL_EXECUTE` PL/SQL 包。虽然配置更复杂，但使用此包运行并行化事务有一些关键优势：

*   整个事务被分割成若干部分，每部分都有自己的提交点。
*   事务可重新启动。
*   锁定仅作用于受影响的行。
*   减少了 Undo 空间使用。
*   你可以更精细地控制工作如何划分。你可以通过多种方式划分工作：
    *   按列
    *   按 ROWID
    *   按 SQL 语句

使用 `DBMS_PARALLEL_EXECUTE` 包的明显好处是能更好地控制大型事务的运行方式、增加功能性以及更高效地利用数据库资源。使用此包的主要权衡取舍是它配置、设置和运行起来更为复杂——但在处理海量数据时，这可能是非常值得的。

## 并行 DML 的限制

使用并行 DML 存在许多限制，你需要了解，即使处理大量数据，在某些情况下并行 DML 也可能无法实现：

*   单行插入（使用 `VALUES` 子句）无法并行运行。
*   在一个事务中，你只能对一个表进行一次修改。
*   无法对带有触发器的表运行。
*   带有某些约束的表可能不符合条件。
*   对带有对象或 LOB 列的表，并行 DML 功能有限。
*   对临时表，并行 DML 功能有限。
*   分布式事务无法并行化。

## 并行度

一旦你提交一个并行化的 DML 操作以供执行，Oracle 会根据一组优先规则，确定将用于整个提交语句的并行度。理解这些规则很重要，这样你才能获得事务所期望的并行度。

对于 DML 事务，Oracle 应用以下基本优先规则来确定并行度：

1.  检查 `INSERT`、`UPDATE`、`MERGE` 或 `DELETE` 语句上是否指定了提示。
2.  检查是否存在任何会话级指令。
3.  检查目标对象上的对象级并行性。
4.  在语句查询部分所涉及的查询表或任何关联索引中，选择指定的最大并行度（仅适用于插入）。

为语句的插入和查询部分选择合适的并行度后，查询即被执行。请注意，为语句每个部分选择的并行度可以不同。

## 其他注意事项

使用并行 DML 可能很复杂，因为涉及的对象类型、是否是分区表、对象上指定的并行度、语句中指定的提示以及并行参数设置等，存在多种可能的排列组合。

以下是使用并行 DML 时需要考虑的其他一些因素：

*   对于并行化的插入事务，会执行直接路径加载。
*   在决定是否使用并行 DML 时，你必须权衡你将获得的性能提升与该操作的空间使用量。并行化的 `INSERT` 语句速度快，但占用更多空间。如果你为插入事务指定了并行度为 4，那么将为该操作分配 4 个区。你必须根据自身需求判断哪个更重要。
*   如果对象是分区表，可能会影响并行 DML 事务的运行方式。
