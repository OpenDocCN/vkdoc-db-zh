# 工作原理

Oracle 尝试在内存（PGA）中执行排序和哈希操作，但如果排序操作太大无法放入内存，它会使用临时表空间来完成工作。重要的是要理解，即使是一个大型排序操作也有可能用尽整个临时表空间。由于所有数据库会话共享临时表空间，运行大型排序操作的会话可能导致其他会话因空间不足而收到错误。一旦临时表空间填满，所有试图使用临时表空间的 SQL 语句都将失败，并出现 `ORA-1652: unable to extend temp segment` 错误。新会话可能无法连接，查询有时会挂起，用户可能无法发出新查询。你试图查找任何阻塞锁，但并不存在。如果临时表空间被填满，事务将无法完成。如果你查看告警日志，会发现临时表空间空间已耗尽。

使用 `ORDER BY` 或 `GROUP BY` 子句的操作经常使用临时表空间来完成工作。你还必须记住，创建索引或重建索引也会使用临时表空间对索引进行排序。

Oracle 使用 PGA 内存来执行排序和哈希操作。因此，你首先要做的是检查为 `PGA_AGGREGATE_TARGET` 初始化参数设置的当前值，并看看提高它是否有所帮助。然而，即使调大 `PGA_AGGREGATE_TARGET` 参数的设置，也不能保证 Oracle 会完全在内存中执行巨大的排序操作。Oracle 为每个会话分配一定量的 PGA 内存，其分配量由内部根据 `PGA_AGGREGATE_TARGET` 参数的值决定。一旦一个大型操作使用了其分配的 PGA 内存份额，Oracle 就会将中间结果写入到磁盘上的临时表空间中。这类操作被称为 `one-pass` 或 `multi-pass` 操作，由于它们是在磁盘上执行的，因此比完全在 PGA 中执行的操作要慢得多。

如果你的数据库临时表空间空间不足，你必须通过添加临时文件来增加其大小。为临时表空间启用 `autoextend` 也有助于防止“空间不足”错误。由于 Oracle 会为执行排序操作的用户分配其指定的临时表空间中的空间，因此你可以为需要执行大量排序的用户分配一个不同于其他用户的临时表空间，从而防止大量排序活动损害数据库性能。

请注意，与每个对象有多个表或索引段不同，临时表空间只有一个称为排序段的段。所有会话共享此排序段。单个 SQL 语句可以使用多个排序和哈希操作。此外，同一个会话可以同时执行多个 SQL 语句，每个语句使用多个排序和哈希操作。一旦一个排序操作完成，数据库会立即将该操作使用的块标记为可用，并将它们分配给另一个排序操作。随着排序操作变大，数据库会向排序段添加区，但如果临时表空间中没有更多空闲空间来分配额外的区，它将发出 `ORA-1652:unable to extend temp segment` 错误。使用该排序操作的 SQL 语句将因此失败。

## 7-6. 解决“无法扩展临时段”错误

### 问题

在创建大型索引时，你收到一个 Oracle 错误，指出数据库无法扩展 `TEMP` 段。然而，你的临时表空间中有大量可用空间。

### 解决方案

当你遇到如下错误时，你的第一反应可能是认为临时表空间中没有可用空间。

```
ORA-01652: unable to extend temp segment by 1024 in tablespace INDX_01
```

你无法通过向临时表空间添加空间来解决此问题。错误信息明确指出了空间不足的表空间。在本例中，出问题的表空间是 `INDX_01`，而不是 `TEMP` 表空间。显然，索引创建过程失败是因为 `INDX_01` 表空间空间不足。你可以通过向 `INDX_01` 表空间添加一个数据文件来修复问题，如下所示：

```
SQL>alter tablespace INDX_01 add datafile '/u01/app/oracle/data/indx_01_02.dbf'
  2 size 1000m;
```
