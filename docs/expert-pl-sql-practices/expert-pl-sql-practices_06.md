## 关于作者

![images](img/square.jpg) **John Beresniewicz** 是 Oracle 位于加州红木城总部的技术员工顾问成员。他于 2002 年加入 Oracle，从事数据库性能领域的企业管理器工作，并在诊断与调优包、真实应用测试、支持工作台和 Exadata 的设计中扮演了重要角色。多年来，他经常在 Oracle Openworld 和其他会议上就数据库性能和 PL/SQL 编程发表演讲。他是 *Oracle Built-in Packages*（O'Reilly & Associates, 1998）的合著者之一，与 Steven Feuerstein 合作，并且是 OakTable 网络的创始成员。

![images](img/square.jpg) **Adrian Billington** 是一名从事应用设计、开发和性能调优的顾问，自 1999 年以来一直使用 Oracle 数据库。他是 [www.oracle-developer.net](http://www.oracle-developer.net) 网站的创建者，该网站为 Oracle 开发人员提供了大量关于 SQL 和 PL/SQL 特性、实用程序和技术的资源。Adrian 也是 Oracle ACE 和 OakTable 网络的成员。他与妻子 Anji 以及三个孩子 Georgia、Oliver 和 Isabella 居住在英国。

![images](img/square.jpg) **Martin Büchi** 自 2004 年起担任 Avaloq 的首席软件架构师。Avaloq 是一家基于 Oracle RDBMS 构建标准化银行软件的供应商，其代码库包含 1100 万行 PL/SQL。他与两位同事一起定义系统架构，并审查 170 名全职 PL/SQL 开发人员的构架和代码，追求简洁、高效和健壮性。Martin 定期在 Oracle 会议上发表演讲。2009 年，他被 *Oracle Magazine* 评为年度 PL/SQL 开发者。在接触 Oracle 数据库之前，Martin 从事面向对象系统、形式化方法和近似记录匹配方面的工作。他拥有瑞士联邦理工学院的硕士学位和芬兰图尔库计算机科学中心的博士学位。闲暇时，Martin 喜欢与家人进行各种户外运动。

![images](img/square.jpg) **Melanie Caffrey** 是 Oracle 公司的高级开发经理，为不同客户的业务需求提供前端和后端 Oracle 解决方案。她是多本技术出版物的合著者，包括 *Oracle Web Application Programming for PL/SQL Developers, Oracle DBA Interactive Workbook,* 以及 *Oracle Database Administration: The Complete Video Course*，均由 Prentice Hall 出版。她曾在纽约市哥伦比亚大学的计算机技术与应用项目中授课，教授高级 Oracle 数据库管理和 PL/SQL 开发课程。她也是 Oracle 会议上的常客。

![images](img/square.jpg) **Ron Crisco** 28 年来一直担任软件设计师、开发人员和项目负责人，并有 21 年使用 Oracle 数据库的经验。他在 Method R Corporation 工作，负责设计和开发软件、管理软件产品（如 Method R Profiler, MR Tools, 和 MR Trace）、提供咨询和授课。他的专长是简化复杂工作，这在帮助周围的人完成非凡任务时尤其有价值。

## 卢·坎宁安

**卢·坎宁安**在 IT 领域工作已超过 20 年，自 1993 年起便与`Oracle`数据库打交道。他的专长是应用程序设计、数据库设计以及高吞吐量`VLDB`数据库的编码工作。他目前是佛罗里达州圣彼得堡一家金融服务公司的高级数据库架构师，负责处理规模庞大、高事务率的分析型数据库和应用程序。他花费大量时间紧跟当前技术与趋势，并在用户组会议上发表演讲和举办网络研讨会。Lewis 也是一名`Oracle ACE Director`和`Oracle Certified Professional`。他为`Oracle Technology Network`撰写过多篇文章，并维护着一个`Oracle`技术博客：<http://it.toolbox.com/blogs/oracle-guide>。Lewis 著有两本书：`EnterpriseDB: The Definitive Reference` (`Rampant Techpress`, 2007) 和 `SQL DML: The SQL Starter Series` (`CreateSpace`, 2008)。他与妻子和两个儿子住在`佛罗里达州`。可通过 <lewisc@databasewisdom.com> 联系他。

## 康纳·麦克唐纳

**康纳·麦克唐纳**自 1990 年代初便开始使用`Oracle`，初露头角于`Oracle`的`6.0.36`和`7.0.12`版本。在过去的 11 年里，康纳曾在`澳大利亚`、`英国`、`东南亚`、`西欧`和`美国`的系统上工作。他逐渐认识到，尽管世界各地的系统和方法论多种多样，但在开发运行在`Oracle`上的系统时，往往存在两个共同主题：要么刻意避开`Oracle`特有的功能，要么以杂乱无章或不够优化的方式使用它们。正是这一观察，促使他创建了一个个人技巧与提示网站 (<http://www.oracledba.co.uk>)，并更多地在`Oracle`演讲圈中发表演讲，以期改善业界对`PL/SQL`的认知和使用。

## 阿鲁普·南达

**阿鲁普·南达**自 1993 年起便担任`Oracle DBA`，这让他接触了数据库管理的方方面面，从建模到灾难恢复。他目前领导着位于`纽约州白原市`的`喜达屋酒店`（`喜来登`和`威斯汀`等连锁酒店的母公司）的全球`DBA`团队。他担任`独立 Oracle 用户组` (`IOUG`) 的出版物`SELECT Journal`的特约编辑；在`Oracle World`等众多`Oracle`技术活动以及`纽约 Oracle 用户组`等本地用户组会议上发表演讲；并为`Oracle Magazine`等印刷出版物以及`Oracle Technology Network`等在线出版物撰写了许多文章。阿鲁普合著了两本书：`Oracle Privacy Security Auditing` (`Rampant`, 2003) 和 `Oracle PL/SQL for DBAs` (`O'Reilly`, 2005)。鉴于其专业成就及对用户社区的贡献，`Oracle`于 2003 年授予他`年度 DBA`称号。阿鲁普与妻子`Anindita`和儿子`Anish`居住在`康涅狄格州丹伯里`。可通过 <arup@proligence.com> 联系他。

## 斯蒂芬·珀蒂

**斯蒂芬·珀蒂**于 1995 年在`瑞士日内瓦`的`欧洲核子研究组织`开始了他的职业生涯，该组织是欧洲粒子物理实验室。他现在负责带领一个由软件工程师和学生组成的团队，为实验室及更广泛的领域交付应用程序和工具。其中一种工具是`工程与设备数据管理系统`，也称为`CERN EDMS`。像`CERN`的`大型强子对撞机` (`LHC`) 这样的项目生命周期长达 40 年或更久。`EDMS`是实验室的数字工程记忆。超过一百万份与超过一百万件设备相关的文档存储在`EDMS`中，该系统也被用作`CERN`的`产品生命周期管理` (`PL/SQL`) 和`资产跟踪`系统。`EDMS`几乎完全基于`PL/SQL`构建，其预期寿命至少与`LHC`一样长。

斯蒂芬和他的团队一直在打磨`PL/SQL`中的编码规范和最佳实践，以应对其所面对的非常有趣且混合的挑战：数十年的可维护性、可靠性、高效的错误处理、可扩展性以及模块的可复用性。这些挑战因团队成员的频繁轮换而加剧，其中大多数学生只是在`CERN`短暂工作。最古老的代码片段写于 1995 年，至今仍在使用——并且成功运行！除了钻研`PL/SQL`，斯蒂芬也喜欢时不时登上舞台，在`CERN`的摇滚夏日音乐节上担任摇滚乐队主唱，或在各种戏剧中扮演角色。
