# 游标变量限制清单

关于游标变量，请务必考虑以下关键点：

*   不能在包（无论是规范还是主体）中声明游标变量。它们只能在过程或函数中声明。
*   不能通过数据库链接来回传递游标变量。如果您将宿主游标变量传递给 PL/SQL，则无法在服务器端从中提取数据，除非您在同一服务器调用中也打开它。
*   游标和游标变量不兼容。不能在预期使用一种游标的地方使用另一种。
*   不能使用比较运算符来测试游标变量是否相等、不等或为空。
*   不能使用 `REF CURSOR` 类型来指定集合的元素类型。因此，集合元素无法存储游标变量的值。在计划使用 `REF` 游标辅助批量处理等场景时，这一点尤其需要考虑。
*   同样，不能在 `CREATE TABLE` 或 `CREATE VIEW` 语句中使用 `REF CURSOR` 类型来指定列类型。换言之，数据库列无法存储游标变量的值。
*   不能将 null 值赋给游标变量。

需要考虑的限制确实很多。鉴于使用游标变量存在所有这些限制，因此在选择使用它们时必须经过深思熟虑。然而，使用游标变量绝对比将所有数据库访问逻辑直接放入应用程序中要好。因此，如果选择使用游标变量，重要的是不仅要考虑如何构建 PL/SQL 代码部分，还要考虑如何构建 Java 代码部分。
