### 数据加密

您可以在两种环境中加密数据：本地或通过服务在`Windows Azure`中进行。如前所述，`SQL 数据库`目前不支持加密（尽管支持哈希处理）。如果您需要加密社会安全号码或电话号码，应考虑在何处进行加密是合理的。

一般来说，除非您的应用程序运行在私钥可能面临风险的公共环境中，否则应考虑在数据通过互联网传输之前在本地进行加密。但如果您需要在`Windows Azure`中解密，或者需要在不共享密钥的多个消费者之间加密和解密数据，那么您可能需要在将数据存储到`SQL 数据库`之前，先在`Windows Azure`中对其进行加密。

### SaaS 应用程序与联合

设计 SaaS 应用程序通常需要使用多个数据库，以扩展数据存储和性能。为了构建可靠的 SaaS 解决方案，开发者通常会开发一个自定义框架，提供应用程序所需的连接路由逻辑，以确定它应连接到哪个数据库。

如果您正在构建 SaaS 应用程序，应研究使用联合（Federations），这是`SQL 数据库`的一项功能。

联合通过使用管理`SQL`命令将数据分布到多个数据库中，有助于提高可扩展性。我将在第 11 章更详细地探讨联合，包括其工作原理和当前的一些限制。

## 总结

本章回顾了许多分布式计算所特有的设计概念，而云计算正是这些概念的天然试验场。请记住，设计云应用程序可以像从本地应用程序连接到`SQL 数据库`实例一样简单，也可以根据需要（使用分布式边缘缓存和分片）构建企业级系统的复杂设计。

本章提供了在设计应用程序时应考虑的多个参数，每个参数都可能显著影响您的最终设计，例如性能、计费、安全性、存储选项和限流。在做出最终选择之前，您应考虑创建两个或多个基于云的设计方案，并与其他设计师一起审查，讨论其优缺点。如果有时间，还可以构建概念验证来验证您的假设并衡量解决方案的有效性。
