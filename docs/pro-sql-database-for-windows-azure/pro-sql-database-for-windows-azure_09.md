# 长时间运行的单一事务
- 空闲连接
- 因服务器故障导致的故障转移

## 不允许使用的用户名
出于安全原因，某些用户名无法创建：
- `sa`
- `admin`
- `administrator`
- `guest`
- `root`

## 登录名
在某些情况下，你可能需要在登录名后附加服务器名称才能成功登录，格式如下：`[loginName]@[servername]`。因此，请避免在登录名中使用 at 符号 (`@`)。

## TCP 端口 1433
仅允许使用 TCP 端口 `1433`。无法为 SQL 数据库定义其他侦听端口。

#### 备份
备份你的 SQL 数据库实例与备份传统的 SQL Server 数据库有些不同。你无法以传统方式备份 SQL 数据库实例，也无法在 SQL 数据库中还原 SQL Server 数据库。但是，你可以在导出/导入数据之前，创建 SQL 数据库实例的事务一致性克隆。关于备份，你可以期待以下内容：

- **备份/还原操作。** 这些操作不可用。此外，你无法附加或分离 SQL 数据库实例。
- **克隆操作。** 你可以使用 `CREATE DATABASE` 语句将 SQL 数据库实例克隆到另一个实例中。
- **日志文件。** 你无法访问数据库日志文件，也无法创建日志备份。
- **导出/导入数据。** 你可以使用导入和导出功能执行这些操作。
- **还原。** 你可以使用“时间点还原”功能。

#### 对象
SQL Server 中的某些对象在 SQL 数据库中不可用。如果你的应用程序严重依赖这些功能，可能会在使用 SQL 数据库时遇到困难，并且你可能需要重新考虑应用程序设计以适应这些限制。以下是目前适用于 SQL 数据库的一些限制：

- **CLR。** SQL 数据库中不提供 .NET CLR。因此，你无法创建扩展存储过程或扩展函数。
- **系统函数。** SQL 数据库支持许多系统函数，包括聚合函数和排名函数。但是，SQL 数据库不支持 RowSet 函数，包括：
  - `OPENQUERY`
  - `OPENXML`
  - `OPENROWSET`
  - `OPENDATASOURCE`
- **系统存储过程。** SQL 数据库仅提供一小部分系统存储过程，分为以下几类：
  - 目录存储过程
  - 数据库引擎存储过程
  - 安全存储过程
- **系统表。** 所有系统表均不可用。
- **系统视图。** 提供一部分系统视图；你可以从 master 数据库访问其中一些，从用户数据库访问另一些。以下是一些可用的系统视图（完整列表请参阅 SQL 数据库的在线 MSDN 库）：
  - `sys.sql_logins`
  - `sys.views`
  - `sys.databases`
  - `sys.columns`
  - `sys.objects`
- **堆表。** SQL 数据库不允许使用堆表。所有表都必须具有聚集索引。

#### 其他
除了上述限制之外，SQL Server 提供的其他组件和选项在 SQL 数据库中不可用。在大多数情况下，这些限制不会影响你的应用程序设计，但最好记住它们：

- **最大数据库数量。** 你最多只能创建一百五十个用户数据库。
- **分布式事务。** 虽然支持 SQL 事务，但不支持跨 SQL 数据库实例的分布式事务。
- **排序规则。** SQL 数据库在列和数据库级别支持排序规则，或在执行时使用表达式。服务器级别的排序规则无法更改，并设置为 `SQL_LATIN1_GENERAL_CP1_CI_AS`。在数据库级别设置后，无法使用 `ALTER DATABASE` 命令更改排序规则。
- **英语。** SQL 数据库仅支持英语。
- **数据库大小。** 你只能创建特定大小的数据库，如前所述。
- **基于行版本控制的隔离级别。** `READ_COMMITTED_SNAPSHOT` 和 `ALLOW_SNAPSHOT_ISOLATION` 选项在数据库级别设置，且无法更改。
- **数据库文件放置。** 你无法选择数据库文件的物理部署方式；也无法控制文件组。这由 Microsoft 数据中心自动处理以实现最佳性能。
- **跟踪标志。** 跟踪标志不可用。
- **SQL Server 配置选项。** 所有常规 SQL Server 选项均不可用，包括 CPU 和 I/O 亲和性。
- **Service Broker。** Service Broker 不可用。
- **全局临时表。** 全局临时表不可用。但是，你可以使用局部临时表。
- **SQL Server Agent。** SQL Server Agent 不可用。

#### 驱动程序和协议
你还应该知道，访问 SQL 数据库只能使用特定的库执行。如果你的编程栈中不使用 ADO.NET，这可能相关。例如，旧版本的 Delphi 无法连接到 SQL 数据库。以下是支持的数据库的摘要：

- **TDS 版本 7.3 或更高版本。** 不支持使用低于 7.3 版本的 TDS 的任何客户端。
- **驱动程序和库。** 允许使用以下驱动程序和库：
  - .NET Framework Data Provider for SQL Server（来自 .NET 3.5 SP1）
  - SQL Server 2008 Native Client ODBC 驱动程序或更高版本
  - SQL Server 2008 driver for PHP 版本 1.1 或更高版本
