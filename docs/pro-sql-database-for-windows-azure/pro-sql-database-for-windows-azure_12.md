# 设计注意事项
为了将 Azure 平台的云计算用于不仅仅是简单的托管，你必须探索可用于设计解决方案的广泛选项。除了理解本章介绍的设计选项外，你还需要牢固掌握云计算当前的不足之处，这可能会影响你的设计选择。

## 设计因素
在审查各种设计模式之前，让我们从影响设计选择的一些机会和限制开始。请记住，尽管本书主要关注 SQL 数据库，但本章的许多概念普遍适用于 Azure 开发。

### 异地存储
如第 1 章所述，Azure 平台提供四种不同的存储模型：Blob、表、队列和 SQL 数据库。在 SQL 数据库中存储数据类似于在 SQL Server 中存储数据。你所需要做的就是发出 T-SQL 语句，并检查 SQL 数据库特有的语法限制，然后就可以开始了！

使用 T-SQL 在 SQL 数据库中存储数据的能力提供了独特的机会。在许多情况下，你可以轻松扩展或移植某些类型的应用程序到 SQL 数据库中，而无需（或只需有限）修改。这种可移植性使你可以实施直接依赖 SQL 数据库存储的解决方案，或者使用本地数据库同时透明地使用 SQL 数据库来满足额外的存储需求（例如报表）。

但是，请记住，你可存储在单个 SQL 数据库实例中的数据量是有限制的。
