# 包 dbms_xplan

您在本章前面已经看到，包 `dbms_xplan` 显示存储在三个不同位置的执行计划：计划表、库缓存和 AWR。以下部分描述了为此目的在该包中可用的函数。首先，让我们看看它们生成的输出。

## 输出

本节的目的是解释包 `dbms_xplan` 中函数返回的输出所包含的信息。为此，我将使用一个示例输出，该输出由脚本 `dbms_xplan_output.sql` 生成，包含所有可用部分，但每个部分并未包含全部信息。问题在于一个书籍页面宽度不足以显示所有信息。因此，我将选择一个仅显示关键信息的输出。然后，如果有缺失的内容，我会指出来。

输出的第一部分如下：

```
SQL_ID cmm8zrzv2v2s5, child number 0
------------------------------------
SELECT t2.* FROM t t1, t t2 WHERE t1.n = t2.n AND t1.id > 6 AND t2.id
BETWEEN 6 AND 19
```

此部分提供有关 SQL 语句的以下信息：

*   `sql_id` 标识父游标。此信息仅在输出由 `display_cursor` 和 `display_awr` 函数生成时才可用。
*   子游标编号与 `sql_id` 一起标识子游标。此信息仅在输出由 `display_cursor` 函数生成时才可用。
*   SQL 语句的文本仅在输出由 `display_cursor` 和 `display_awr` 函数生成时才可用。

第二部分显示执行计划的哈希值，并在表格中显示执行计划本身。示例如下：

```
Plan hash value: 1338433605

------------------------------------------------------------------------------------
| Id  | Operation                    | Name | Rows | Bytes | Cost (%CPU)| Time     |
------------------------------------------------------------------------------------
|   0 | SELECT STATEMENT             |      |      |       |    49 (100)|          |
|*  1 |  HASH JOIN                   |      |   14 |  7756 |    49   (3)| 00:00:01 |
|   2 |   TABLE ACCESS BY INDEX ROWID| T    |   14 |  7392 |     4   (0)| 00:00:01 |
|*  3 |    INDEX RANGE SCAN          | T_PK |   14 |       |     2   (0)| 00:00:01 |
|*  4 |   TABLE ACCESS FULL          | T    |  994 | 25844 |    44   (0)| 00:00:01 |
------------------------------------------------------------------------------------
```

在表格中，为每个操作提供了估算值和执行统计信息。表格中的列数直接取决于可用信息的多少。例如，仅当信息可用时才会显示有关分区、并行处理或执行统计信息的信息。因此，使用完全相同参数由同一函数生成的两个输出可能不同。在这种情况下，您将看到默认可用的列。表 6-1 总结了您可能看到的所有列。
