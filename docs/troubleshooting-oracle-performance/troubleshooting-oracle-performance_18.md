# 直方图

查询优化器始于数据均匀分布的假设。均匀分布数据的一个例子是测试表中 `id` 列存储的数据（该测试表在前几节中使用）。实际上，它恰好存储了从 1 到 1,000 的所有数字各一次。在这种情况下，要找出基于该列的谓词（例如 `id BETWEEN 6 AND 19`）过滤掉的行数，查询优化器仅需要上一节描述的对象统计信息：最小值、最大值和不同值的数量。

如果数据分布不均匀，查询优化器没有额外信息就无法计算出可接受的估计值。例如，给定存储在列 `val2` 中的数据集（参见以下查询的输出），查询优化器如何对 `val2=105` 这样的谓词做出有意义的估计？它不能，因为它不知道大约 50% 的行满足该谓词。

```sql
SQL> SELECT val2 AS val2, count(*)
  2 FROM t
  3 GROUP BY val2
  4 ORDER BY val2;
```

```
VAL2   COUNT(*)
---------- ----------
       101          8
       102         25
       103         68
       104        185
       105        502
       106        212
```

查询优化器需要的关于数据非均匀分布的额外信息称为 `直方图`。有两种类型的直方图可用：`频率直方图` 和 `高度平衡直方图`。
