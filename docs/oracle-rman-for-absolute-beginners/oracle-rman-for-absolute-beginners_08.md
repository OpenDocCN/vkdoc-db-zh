# 第 1 章 入门

![image](img/frontdot.jpg)

备份与恢复技能是 DBA 能力清单上的首要技能。保护数据并保持其可用性构成了数据库管理的基础。你的 DBA 工作依赖于你定期执行备份的能力，并在必要时还原和恢复数据库。恢复管理器（RMAN）是 Oracle 旗舰级的备份与恢复工具。此工具是保护数据的高效且有效的方式。每位 DBA 都必须知道如何实施和使用 RMAN。

RMAN 是一个灵活的工具，包含各种各样的备份与恢复功能。本书的目的是让你扎实了解使用 RMAN 备份、还原和恢复数据库的最常见方法。我将展示许多实际示例以及实现各种功能所需的代码。有了这个基础，你将能够在任何环境中实施 RMAN。

我还将介绍用户管理备份技术（热备份和冷备份）。我发现，理解用户管理备份机制的 DBA 能更好地排查和解决任何类型的数据库问题（包括但不限于备份与恢复问题）。我还将介绍 Data Pump 的使用。DBA 通常会实施 Data Pump 功能来满足备份与恢复需求（例如，快速备份单个表）。简而言之，每位 DBA 都应熟悉所有用于保护数据的 Oracle 工具。

要开始使用，在处理 Oracle 备份与恢复时，有一些基本任务你应该熟悉，即：

*   连接到你的数据库
*   启动/停止你的数据库

理解这些主题是使用 RMAN（以及其他 Oracle 工具）的前提。本章的目的是让你熟悉这些初始任务。首先是连接到你的数据库。

![Image](img/sq.jpg) **注意** 本章假设你已安装 Oracle，并且有一个已创建且可用的数据库。

### 连接到你的数据库

在连接到你的数据库之前，你必须建立所需的操作系统变量。此外，如果你要运行备份和恢复命令，你需要有权访问特权操作系统（OS）账户或已被授予适当权限（通过密码文件）的数据库用户。这些主题将在以下小节中讨论。

### 建立操作系统变量

在通过 SQL*Plus、RMAN、Data Pump（或任何其他 Oracle 实用程序）连接到你的数据库之前，必须首先设置几个操作系统变量：

*   `ORACLE_HOME`
*   `ORACLE_SID`
*   `LD_LIBRARY_PATH`
*   `PATH`

`ORACLE_HOME` 变量很重要，因为它定义了定位 Oracle 二进制文件（如 `sqlplus`、`dbca`、`netca`、`rman` 等）的起始目录，这些文件位于 `ORACLE_HOME/bin` 中。
